<span data-ttu-id="bdac4-325">Gdy wyrażenie zawiera wiele operatorów, *pierwszeństwo* operatorów kontroluje kolejność, w jakiej są oceniane poszczególne operatory.</span><span class="sxs-lookup"><span data-stu-id="bdac4-325">When an expression contains multiple operators, the *precedence* of the operators controls the order in which the individual operators are evaluated.</span></span> <span data-ttu-id="bdac4-326">Na przykład wyrażenie `x + y * z` jest oceniane tak, jakby `x + (y * z)` `*` operator miał wyższy priorytet niż `+` operator.</span><span class="sxs-lookup"><span data-stu-id="bdac4-326">For example, the expression `x + y * z` is evaluated as `x + (y * z)` because the `*` operator has higher precedence than the `+` operator.</span></span>

<span data-ttu-id="bdac4-327">Gdy operand występuje między dwoma operatorami o takim samym priorytecie, *łączność* operatorów kontroluje kolejność wykonywania operacji:</span><span class="sxs-lookup"><span data-stu-id="bdac4-327">When an operand occurs between two operators with the same precedence, the *associativity* of the operators controls the order in which the operations are performed:</span></span>
