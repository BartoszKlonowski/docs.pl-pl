# <a name="expressions"></a><span data-ttu-id="9a054-103">Wyrażenia</span><span class="sxs-lookup"><span data-stu-id="9a054-103">Expressions</span></span>

<span data-ttu-id="9a054-104">*Wyrażenia* są zbudowane z *argumentów operacji* i *operatorów*.</span><span class="sxs-lookup"><span data-stu-id="9a054-104">*Expressions* are constructed from *operands* and *operators*.</span></span> <span data-ttu-id="9a054-105">Operatory wyrażenia wskazują, które operacje mają być stosowane do operandów.</span><span class="sxs-lookup"><span data-stu-id="9a054-105">The operators of an expression indicate which operations to apply to the operands.</span></span> <span data-ttu-id="9a054-106">Przykłady operatorów obejmują `+`, `-`, `*`, `/` i `new`.</span><span class="sxs-lookup"><span data-stu-id="9a054-106">Examples of operators include `+`, `-`, `*`, `/`, and `new`.</span></span> <span data-ttu-id="9a054-107">Przykłady operandów obejmują literały, pola, zmienne lokalne i wyrażenia.</span><span class="sxs-lookup"><span data-stu-id="9a054-107">Examples of operands include literals, fields, local variables, and expressions.</span></span>

<span data-ttu-id="9a054-108">Gdy wyrażenie zawiera wiele operatorów, *pierwszeństwo* operatorów określa kolejność, w której są oceniane poszczególne operatory.</span><span class="sxs-lookup"><span data-stu-id="9a054-108">When an expression contains multiple operators, the *precedence* of the operators controls the order in which the individual operators are evaluated.</span></span> <span data-ttu-id="9a054-109">Na przykład wyrażenie `x + y * z` jest oceniane jako `x + (y * z)`, ponieważ operator `*` ma wyższy priorytet niż operator `+`.</span><span class="sxs-lookup"><span data-stu-id="9a054-109">For example, the expression `x + y * z` is evaluated as `x + (y * z)` because the `*` operator has higher precedence than the `+` operator.</span></span>

<span data-ttu-id="9a054-110">Gdy operand występuje między dwoma operatorami o takim samym priorytecie, *łączność* operatorów kontroluje kolejność wykonywania operacji:</span><span class="sxs-lookup"><span data-stu-id="9a054-110">When an operand occurs between two operators with the same precedence, the *associativity* of the operators controls the order in which the operations are performed:</span></span>

* <span data-ttu-id="9a054-111">Z wyjątkiem operatorów przypisania i łączenia wartości null wszystkie operatory binarne są z *lewej strony skojarzenia*, co oznacza, że operacje są wykonywane od lewej do prawej.</span><span class="sxs-lookup"><span data-stu-id="9a054-111">Except for the assignment and null-coalescing operators, all binary operators are *left-associative*, meaning that operations are performed from left to right.</span></span> <span data-ttu-id="9a054-112">Na przykład `x + y + z` jest oceniane jako `(x + y) + z`.</span><span class="sxs-lookup"><span data-stu-id="9a054-112">For example, `x + y + z` is evaluated as `(x + y) + z`.</span></span>
* <span data-ttu-id="9a054-113">Operatory przypisania, łączenie wartości null `??` i operatory `??=` i operator warunkowy `?:` są z prawej strony *skojarzenia*, co oznacza, że operacje są wykonywane od prawej do lewej.</span><span class="sxs-lookup"><span data-stu-id="9a054-113">The assignment operators, the null-coalescing `??` and `??=` operators, and the conditional operator `?:` are *right-associative*, meaning that operations are performed from right to left.</span></span> <span data-ttu-id="9a054-114">Na przykład `x = y = z` jest oceniane jako `x = (y = z)`.</span><span class="sxs-lookup"><span data-stu-id="9a054-114">For example, `x = y = z` is evaluated as `x = (y = z)`.</span></span>

<span data-ttu-id="9a054-115">Pierwszeństwo i łączność można kontrolować za pomocą nawiasów.</span><span class="sxs-lookup"><span data-stu-id="9a054-115">Precedence and associativity can be controlled using parentheses.</span></span> <span data-ttu-id="9a054-116">Na przykład `x + y * z` najpierw mnoży `y` przez `z`, a następnie dodaje wynik do `x`, ale `(x + y) * z` dodaje `x` i `y`, a następnie mnoży wynik przez `z`.</span><span class="sxs-lookup"><span data-stu-id="9a054-116">For example, `x + y * z` first multiplies `y` by `z` and then adds the result to `x`, but `(x + y) * z` first adds `x` and `y` and then multiplies the result by `z`.</span></span>

<span data-ttu-id="9a054-117">Większość operatorów może być [*przeciążona*](../language-reference/operators/operator-overloading.md).</span><span class="sxs-lookup"><span data-stu-id="9a054-117">Most operators can be [*overloaded*](../language-reference/operators/operator-overloading.md).</span></span> <span data-ttu-id="9a054-118">Przeciążanie operatora umożliwia określenie implementacji operatora zdefiniowanego przez użytkownika dla operacji, w których jeden lub oba operandy są klasy lub typu struktury zdefiniowanej przez użytkownika.</span><span class="sxs-lookup"><span data-stu-id="9a054-118">Operator overloading permits user-defined operator implementations to be specified for operations where one or both of the operands are of a user-defined class or struct type.</span></span>
