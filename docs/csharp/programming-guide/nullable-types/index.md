<span data-ttu-id="ea941-108">Na przykład możesz przypisać `null` lub dowolnej wartości liczby całkowitej z <xref:System.Int32.MinValue?displayProperty=nameWithType> do <xref:System.Int32.MaxValue?displayProperty=nameWithType> do `Nullable<int>` i [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md), lub `null` do `Nullable<bool>`.</span><span class="sxs-lookup"><span data-stu-id="ea941-108">For example, you can assign `null` or any integer value from <xref:System.Int32.MinValue?displayProperty=nameWithType> to <xref:System.Int32.MaxValue?displayProperty=nameWithType> to a `Nullable<int>` and [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md), or `null` to a `Nullable<bool>`.</span></span>

<span data-ttu-id="ea941-109">Używasz typu dopuszczającego wartość null, jeśli potrzebujesz do reprezentowania niezdefiniowane wartości typu podstawowego.</span><span class="sxs-lookup"><span data-stu-id="ea941-109">You use a nullable type when you need to represent the undefined value of an underlying type.</span></span> <span data-ttu-id="ea941-110">Wartość logiczna może mieć tylko dwie wartości: true i false.</span><span class="sxs-lookup"><span data-stu-id="ea941-110">A Boolean variable can have only two values: true and false.</span></span> <span data-ttu-id="ea941-111">Nie ma żadnej wartości "undefined".</span><span class="sxs-lookup"><span data-stu-id="ea941-111">There is no "undefined" value.</span></span> <span data-ttu-id="ea941-112">W wielu aplikacjach programowania głównie interakcji bazy danych, wartość zmiennej może być niezdefiniowana lub jego brak.</span><span class="sxs-lookup"><span data-stu-id="ea941-112">In many programming applications, most notably database interactions, a variable value can be undefined or missing.</span></span> <span data-ttu-id="ea941-113">Na przykład pola w bazie danych może zawierać wartości true lub false lub może zawierać żadnej wartości w ogóle.</span><span class="sxs-lookup"><span data-stu-id="ea941-113">For example, a field in a database may contain the values true or false, or it may contain no value at all.</span></span> <span data-ttu-id="ea941-114">Możesz użyć `Nullable<bool>` typu w takiej sytuacji.</span><span class="sxs-lookup"><span data-stu-id="ea941-114">You use a `Nullable<bool>` type in that case.</span></span>

<span data-ttu-id="ea941-115">Typy dopuszczające wartości zerowe mają następującą charakterystykę:</span><span class="sxs-lookup"><span data-stu-id="ea941-115">Nullable types have the following characteristics:</span></span>
  
- <span data-ttu-id="ea941-116">Typy dopuszczające wartości zerowe reprezentują zmienne typu wartości, które można przypisać `null` wartość.</span><span class="sxs-lookup"><span data-stu-id="ea941-116">Nullable types represent value-type variables that can be assigned the `null` value.</span></span> <span data-ttu-id="ea941-117">Nie można utworzyć typu dopuszczającego wartość null, w oparciu o typ odwołania.</span><span class="sxs-lookup"><span data-stu-id="ea941-117">You cannot create a nullable type based on a reference type.</span></span> <span data-ttu-id="ea941-118">(Już obsługuje typy odwołań `null` wartości.)</span><span class="sxs-lookup"><span data-stu-id="ea941-118">(Reference types already support the `null` value.)</span></span>  
  
- <span data-ttu-id="ea941-119">Składnia `T?` jest skrótem `Nullable<T>`.</span><span class="sxs-lookup"><span data-stu-id="ea941-119">The syntax `T?` is shorthand for `Nullable<T>`.</span></span> <span data-ttu-id="ea941-120">Dwa formularze są wymienne.</span><span class="sxs-lookup"><span data-stu-id="ea941-120">The two forms are interchangeable.</span></span>  
  
- <span data-ttu-id="ea941-121">Przypisania wartości do typu dopuszczającego wartość null, tak samo jak dla podstawowego typu wartości: `int? x = 10;` lub `double? d = 4.108;`.</span><span class="sxs-lookup"><span data-stu-id="ea941-121">Assign a value to a nullable type just as you would for an underlying value type: `int? x = 10;` or `double? d = 4.108;`.</span></span> <span data-ttu-id="ea941-122">Możesz również przypisać `null` wartość: `int? x = null;`.</span><span class="sxs-lookup"><span data-stu-id="ea941-122">You also can assign the `null` value: `int? x = null;`.</span></span>  
  
- <span data-ttu-id="ea941-123">Użyj <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> i <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> właściwości tylko do odczytu, aby sprawdzić wartość null i pobierać wartości, jak pokazano w poniższym przykładzie: `if (x.HasValue) y = x.Value;`</span><span class="sxs-lookup"><span data-stu-id="ea941-123">Use the <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> and <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> readonly properties to test for null and retrieve the value, as shown in the following example: `if (x.HasValue) y = x.Value;`</span></span>  
  
  - <span data-ttu-id="ea941-124"><xref:System.Nullable%601.HasValue%2A> Właściwość zwraca `true` Jeśli zmienna zawiera wartość, lub `false` , gdy jest `null`.</span><span class="sxs-lookup"><span data-stu-id="ea941-124">The <xref:System.Nullable%601.HasValue%2A> property returns `true` if the variable contains a value, or `false` if it's `null`.</span></span>
  
  - <span data-ttu-id="ea941-125"><xref:System.Nullable%601.Value%2A> Właściwość zwraca wartość, jeśli <xref:System.Nullable%601.HasValue%2A> zwraca `true`.</span><span class="sxs-lookup"><span data-stu-id="ea941-125">The <xref:System.Nullable%601.Value%2A> property returns a value if <xref:System.Nullable%601.HasValue%2A> returns `true`.</span></span> <span data-ttu-id="ea941-126">W przeciwnym razie <xref:System.InvalidOperationException> zgłaszany.</span><span class="sxs-lookup"><span data-stu-id="ea941-126">Otherwise, an <xref:System.InvalidOperationException> is thrown.</span></span>  
  
- <span data-ttu-id="ea941-127">Można również użyć `==` i `!=` operatory o typu dopuszczającego wartość null, jak pokazano w poniższym przykładzie: `if (x != null) y = x.Value;`.</span><span class="sxs-lookup"><span data-stu-id="ea941-127">You can also use the `==` and `!=` operators with a nullable type, as shown in the following example: `if (x != null) y = x.Value;`.</span></span> <span data-ttu-id="ea941-128">Jeśli `a` i `b` są ma wartość null, `a == b` daje w wyniku `true`.</span><span class="sxs-lookup"><span data-stu-id="ea941-128">If `a` and `b` are both null, `a == b` evaluates to `true`.</span></span>  

- <span data-ttu-id="ea941-129">Począwszy od języka C# 7.0, można użyć [dopasowywania do wzorca](../../pattern-matching.md#the-is-type-pattern-expression) do zbadania i Pobierz wartość typu dopuszczającego wartość null: `if (x is int valueOfX) y = valueOfX;`.</span><span class="sxs-lookup"><span data-stu-id="ea941-129">Beginning with C# 7.0, you can use [pattern matching](../../pattern-matching.md#the-is-type-pattern-expression) to both examine and get a value of a nullable type: `if (x is int valueOfX) y = valueOfX;`.</span></span>
  
- <span data-ttu-id="ea941-130">Wartość domyślna `T?` jest wystąpieniem którego <xref:System.Nullable%601.HasValue%2A> właściwość zwraca `false`.</span><span class="sxs-lookup"><span data-stu-id="ea941-130">The default value of `T?` is an instance whose <xref:System.Nullable%601.HasValue%2A> property returns `false`.</span></span>  

- <span data-ttu-id="ea941-131">Użyj <xref:System.Nullable%601.GetValueOrDefault> metodę, aby zwrócić albo przypisaną wartość, lub [domyślne](../../language-reference/keywords/default-values-table.md) wartość bazowego typu wartości, jeśli wartość typu dopuszczającego wartość null jest `null`.</span><span class="sxs-lookup"><span data-stu-id="ea941-131">Use the <xref:System.Nullable%601.GetValueOrDefault> method to return either the assigned value, or the [default](../../language-reference/keywords/default-values-table.md) value of the underlying value type if the value of the nullable type is `null`.</span></span>  

- <span data-ttu-id="ea941-132">Użyj <xref:System.Nullable%601.GetValueOrDefault(%600)> metody do zwrócenia przypisaną wartość lub wartość domyślną podany, jeśli wartość typu dopuszczającego wartość null jest `null`.</span><span class="sxs-lookup"><span data-stu-id="ea941-132">Use the <xref:System.Nullable%601.GetValueOrDefault(%600)> method to return either the assigned value, or the provided default value if the value of the nullable type is `null`.</span></span>
  
- <span data-ttu-id="ea941-133">Użyj [operatorem łączenia wartości null](../../language-reference/operators/null-coalescing-operator.md), `??`, aby przypisać wartość do typu podstawowego, na podstawie wartości typu dopuszczającego wartość null: `int? x = null; int y = x ?? -1;`.</span><span class="sxs-lookup"><span data-stu-id="ea941-133">Use the [null-coalescing operator](../../language-reference/operators/null-coalescing-operator.md), `??`, to assign a value to an underlying type based on a value of the nullable type: `int? x = null; int y = x ?? -1;`.</span></span> <span data-ttu-id="ea941-134">W tym przykładzie ponieważ `x` jest null, wartość wyniku `y` jest `-1`.</span><span class="sxs-lookup"><span data-stu-id="ea941-134">In the example, since `x` is null, the result value of `y` is `-1`.</span></span>

- <span data-ttu-id="ea941-135">Jeśli nie zdefiniowano konwersji zdefiniowanej przez użytkownika między dwoma typami danych, tej samej konwersji można również z tych typów danych w wersji dopuszczającego wartość null.</span><span class="sxs-lookup"><span data-stu-id="ea941-135">If a user-defined conversion is defined between two data types, the same conversion can also be used with the nullable versions of these data types.</span></span>
