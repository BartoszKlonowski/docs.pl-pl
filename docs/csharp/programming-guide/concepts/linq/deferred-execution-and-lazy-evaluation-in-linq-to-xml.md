## <a name="deferred-execution"></a><span data-ttu-id="be6e2-107">Wykonanie odroczone</span><span class="sxs-lookup"><span data-stu-id="be6e2-107">Deferred Execution</span></span>  
 <span data-ttu-id="be6e2-108">Wykonywanie odroczone oznacza, że Obliczanie wyrażenia jest opóźnione do momentu, gdy wartość *rzeczywista* jest wymagana.</span><span class="sxs-lookup"><span data-stu-id="be6e2-108">Deferred execution means that the evaluation of an expression is delayed until its *realized* value is actually required.</span></span> <span data-ttu-id="be6e2-109">Wykonywanie odroczone może znacznie poprawić wydajność, gdy trzeba manipulować dużymi kolekcjami danych, szczególnie w programach, które zawierają serię zapytań lub manipulacji łańcucha.</span><span class="sxs-lookup"><span data-stu-id="be6e2-109">Deferred execution can greatly improve performance when you have to manipulate large data collections, especially in programs that contain a series of chained queries or manipulations.</span></span> <span data-ttu-id="be6e2-110">W najlepszym przypadku wykonywanie odroczone włącza tylko jedną iterację za pomocą kolekcji źródłowej.</span><span class="sxs-lookup"><span data-stu-id="be6e2-110">In the best case, deferred execution enables only a single iteration through the source collection.</span></span>  
  
 <span data-ttu-id="be6e2-111">Technologie LINQ umożliwiają użycie odroczonego wykonania w obu elementach członkowskich <xref:System.Linq?displayProperty=nameWithType> klas podstawowych i w metodach rozszerzających w różnych przestrzeniach nazw LINQ, takich jak <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType> .</span><span class="sxs-lookup"><span data-stu-id="be6e2-111">The LINQ technologies make extensive use of deferred execution in both the members of core <xref:System.Linq?displayProperty=nameWithType> classes and in the extension methods in the various LINQ namespaces, such as <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="be6e2-112">Wykonywanie odroczone jest obsługiwane bezpośrednio w języku C# za pomocą słowa kluczowego [Yield](../../../language-reference/keywords/yield.md) (w formie `yield-return` instrukcji), gdy jest używana w bloku iteratora.</span><span class="sxs-lookup"><span data-stu-id="be6e2-112">Deferred execution is supported directly in the C# language by the [yield](../../../language-reference/keywords/yield.md) keyword (in the form of the `yield-return` statement) when used within an iterator block.</span></span> <span data-ttu-id="be6e2-113">Taki iterator musi zwracać kolekcję typu <xref:System.Collections.IEnumerator> lub <xref:System.Collections.Generic.IEnumerator%601> (lub typu pochodnego).</span><span class="sxs-lookup"><span data-stu-id="be6e2-113">Such an iterator must return a collection of type <xref:System.Collections.IEnumerator> or <xref:System.Collections.Generic.IEnumerator%601> (or a derived type).</span></span>  
  
## <a name="eager-vs-lazy-evaluation"></a><span data-ttu-id="be6e2-114">Ocena eager a z opóźnieniem</span><span class="sxs-lookup"><span data-stu-id="be6e2-114">Eager vs. Lazy Evaluation</span></span>  
 <span data-ttu-id="be6e2-115">Podczas pisania metody, która implementuje wykonywanie odroczone, należy również zdecydować, czy zaimplementować metodę przy użyciu oceny z opóźnieniem czy oceny eager.</span><span class="sxs-lookup"><span data-stu-id="be6e2-115">When you write a method that implements deferred execution, you also have to decide whether to implement the method using lazy evaluation or eager evaluation.</span></span>  
  
- <span data-ttu-id="be6e2-116">W *ocenie z opóźnieniem*pojedynczy element kolekcji źródłowej jest przetwarzany podczas każdego wywołania iteratora.</span><span class="sxs-lookup"><span data-stu-id="be6e2-116">In *lazy evaluation*, a single element of the source collection is processed during each call to the iterator.</span></span> <span data-ttu-id="be6e2-117">Jest to typowy sposób implementacji iteratorów.</span><span class="sxs-lookup"><span data-stu-id="be6e2-117">This is the typical way in which iterators are implemented.</span></span>  
  
- <span data-ttu-id="be6e2-118">W *ocenie eager*pierwsze wywołanie iteratora spowoduje przetworzenie całej kolekcji.</span><span class="sxs-lookup"><span data-stu-id="be6e2-118">In *eager evaluation*, the first call to the iterator will result in the entire collection being processed.</span></span> <span data-ttu-id="be6e2-119">Może być również wymagana tymczasowa kopia kolekcji źródłowej.</span><span class="sxs-lookup"><span data-stu-id="be6e2-119">A temporary copy of the source collection might also be required.</span></span> <span data-ttu-id="be6e2-120">Na przykład <xref:System.Linq.Enumerable.OrderBy%2A> Metoda musi sortować całą kolekcję przed zwróceniem pierwszego elementu.</span><span class="sxs-lookup"><span data-stu-id="be6e2-120">For example, the <xref:System.Linq.Enumerable.OrderBy%2A> method has to sort the entire collection before it returns the first element.</span></span>  
  
 <span data-ttu-id="be6e2-121">Ocena z opóźnieniem zwykle zapewnia lepszą wydajność, ponieważ dystrybuuje przetwarzanie narzutowe równomiernie przez oszacowanie kolekcji i minimalizuje użycie danych tymczasowych.</span><span class="sxs-lookup"><span data-stu-id="be6e2-121">Lazy evaluation usually yields better performance because it distributes overhead processing evenly throughout the evaluation of the collection and minimizes the use of temporary data.</span></span> <span data-ttu-id="be6e2-122">Oczywiście w przypadku niektórych operacji nie ma żadnej innej opcji, aby zmaterializowania wyniki pośrednie.</span><span class="sxs-lookup"><span data-stu-id="be6e2-122">Of course, for some operations, there is no other option than to materialize intermediate results.</span></span>
