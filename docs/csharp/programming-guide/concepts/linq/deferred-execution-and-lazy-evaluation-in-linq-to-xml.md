## <a name="deferred-execution"></a><span data-ttu-id="4b6c0-105">Wykonanie odroczone</span><span class="sxs-lookup"><span data-stu-id="4b6c0-105">Deferred Execution</span></span>  
 <span data-ttu-id="4b6c0-106">Odroczone wykonania oznacza wyniku obliczenia wyrażenia jest opóźnione aż do jego *zrealizowany* faktycznie wymagana jest wartość.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-106">Deferred execution means that the evaluation of an expression is delayed until its *realized* value is actually required.</span></span> <span data-ttu-id="4b6c0-107">Wykonanie odroczone może znacznie zwiększyć wydajność, gdy masz do manipulowania kolekcji dużych ilości danych, szczególnie w programach, które zawierają szereg zapytań łańcuchowych lub manipulacje.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-107">Deferred execution can greatly improve performance when you have to manipulate large data collections, especially in programs that contain a series of chained queries or manipulations.</span></span> <span data-ttu-id="4b6c0-108">W przypadku najlepszych odroczonego wykonania umożliwia jednej iteracji za pomocą kolekcji źródłowej.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-108">In the best case, deferred execution enables only a single iteration through the source collection.</span></span>  
  
 <span data-ttu-id="4b6c0-109">Technologii LINQ wykorzystać rozbudowane odroczonego wykonania w przypadku członków podstawowych <xref:System.Linq?displayProperty=nameWithType> klas i metod rozszerzenia w różnych obszarach nazw LINQ, takich jak <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-109">The LINQ technologies make extensive use of deferred execution in both the members of core <xref:System.Linq?displayProperty=nameWithType> classes and in the extension methods in the various LINQ namespaces, such as <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="4b6c0-110">Wykonanie odroczone jest obsługiwane bezpośrednio w języku C#, [uzyskanie](../../../../csharp/language-reference/keywords/yield.md) — słowo kluczowe (w postaci `yield-return` instrukcji) używanego w ramach blokiem iteratora.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-110">Deferred execution is supported directly in the C# language by the [yield](../../../../csharp/language-reference/keywords/yield.md) keyword (in the form of the `yield-return` statement) when used within an iterator block.</span></span> <span data-ttu-id="4b6c0-111">Takie iteratora musi zwracać kolekcję typu <xref:System.Collections.IEnumerator> lub <xref:System.Collections.Generic.IEnumerator%601> (lub typem pochodnym).</span><span class="sxs-lookup"><span data-stu-id="4b6c0-111">Such an iterator must return a collection of type <xref:System.Collections.IEnumerator> or <xref:System.Collections.Generic.IEnumerator%601> (or a derived type).</span></span>  
  
## <a name="eager-vs-lazy-evaluation"></a><span data-ttu-id="4b6c0-112">Zapoznamy programu vs. Obliczanie z opóźnieniem</span><span class="sxs-lookup"><span data-stu-id="4b6c0-112">Eager vs. Lazy Evaluation</span></span>  
 <span data-ttu-id="4b6c0-113">Podczas pisania metody, która implementuje odroczonym należy zdecydować, czy zaimplementować metodę, przy użyciu opóźnieniem lub eager oceny.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-113">When you write a method that implements deferred execution, you also have to decide whether to implement the method using lazy evaluation or eager evaluation.</span></span>  
  
-   <span data-ttu-id="4b6c0-114">W *opóźnieniem*, jeden element z kolekcji źródłowej jest przetwarzany w każdym wywołaniu iteratora.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-114">In *lazy evaluation*, a single element of the source collection is processed during each call to the iterator.</span></span> <span data-ttu-id="4b6c0-115">Jest to typowy sposób, w którym są implementowane Iteratory.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-115">This is the typical way in which iterators are implemented.</span></span>  
  
-   <span data-ttu-id="4b6c0-116">W *eager oceny*, pierwsze wywołanie funkcji iteratora spowoduje całą kolekcję przetwarzany.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-116">In *eager evaluation*, the first call to the iterator will result in the entire collection being processed.</span></span> <span data-ttu-id="4b6c0-117">Tymczasową kopię kolekcji źródłowej, mogą być także wymagane.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-117">A temporary copy of the source collection might also be required.</span></span> <span data-ttu-id="4b6c0-118">Na przykład <xref:System.Linq.Enumerable.OrderBy%2A> metoda ma sortowania całą kolekcję przed zwróceniem pierwszego elementu.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-118">For example, the <xref:System.Linq.Enumerable.OrderBy%2A> method has to sort the entire collection before it returns the first element.</span></span>  
  
 <span data-ttu-id="4b6c0-119">Obliczanie leniwe zwykle daje lepszą wydajność, ponieważ dystrybuuje obciążenie przetwarzania równomiernie w całej oceny kolekcji, a minimalizuje wykorzystanie danych tymczasowych.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-119">Lazy evaluation usually yields better performance because it distributes overhead processing evenly throughout the evaluation of the collection and minimizes the use of temporary data.</span></span> <span data-ttu-id="4b6c0-120">Oczywiście w przypadku niektórych operacji nie ma innych opcji niż celu zmaterializowania wyników pośrednich.</span><span class="sxs-lookup"><span data-stu-id="4b6c0-120">Of course, for some operations, there is no other option than to materialize intermediate results.</span></span>
