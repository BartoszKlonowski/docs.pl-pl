<span data-ttu-id="5bb72-103">Wątkowość Włącza program C# do wykonania równoczesnych przetwarzania, aby zrobić więcej niż jedną operację naraz.</span><span class="sxs-lookup"><span data-stu-id="5bb72-103">Threading enables your C# program to perform concurrent processing so that you can do more than one operation at a time.</span></span> <span data-ttu-id="5bb72-104">Można na przykład użyć wątkowość do monitorowania danych wejściowych od użytkownika, wykonaj zadania w tle i obsługi jednoczesnych strumieni danych wejściowych.</span><span class="sxs-lookup"><span data-stu-id="5bb72-104">For example, you can use threading to monitor input from the user, perform background tasks, and handle simultaneous streams of input.</span></span>  
  
 <span data-ttu-id="5bb72-105">Wątki mieć następujące właściwości:</span><span class="sxs-lookup"><span data-stu-id="5bb72-105">Threads have the following properties:</span></span>  
  
-   <span data-ttu-id="5bb72-106">Wątki włączyć program do przetwarzania współbieżnych.</span><span class="sxs-lookup"><span data-stu-id="5bb72-106">Threads enable your program to perform concurrent processing.</span></span>

 <span data-ttu-id="5bb72-110">Domyślnie program C# ma jeden wątek.</span><span class="sxs-lookup"><span data-stu-id="5bb72-110">By default, a C# program has one thread.</span></span> <span data-ttu-id="5bb72-111">Jednak pomocnicze wątków można tworzyć i używane do wykonywania kodu równolegle z podstawowym wątku.</span><span class="sxs-lookup"><span data-stu-id="5bb72-111">However, auxiliary threads can be created and used to execute code in parallel with the primary thread.</span></span> <span data-ttu-id="5bb72-112">Wątki te są często nazywane *wątków roboczych*.</span><span class="sxs-lookup"><span data-stu-id="5bb72-112">These threads are often called *worker threads*.</span></span>  
  
 <span data-ttu-id="5bb72-113">Wątki robocze może służyć do wykonywania zadań czasochłonne lub wrażliwego na czas bez angażowania podstawowy wątku.</span><span class="sxs-lookup"><span data-stu-id="5bb72-113">Worker threads can be used to perform time-consuming or time-critical tasks without tying up the primary thread.</span></span> <span data-ttu-id="5bb72-114">Na przykład wątków roboczych są często używane w aplikacji serwerowych do spełnienia żądań przychodzących bez oczekiwania na ukończenie poprzedniego żądania.</span><span class="sxs-lookup"><span data-stu-id="5bb72-114">For example, worker threads are often used in server applications to fulfill incoming requests without waiting for the previous request to be completed.</span></span> <span data-ttu-id="5bb72-115">Wątki robocze są również używane do wykonywania zadań "tła" w aplikacjach komputerowych, aby wątku głównego--dyski elementy interfejsu użytkownika — pozostaje reakcji na działanie użytkownika.</span><span class="sxs-lookup"><span data-stu-id="5bb72-115">Worker threads are also used to perform "background" tasks in desktop applications so that the main thread--which drives user interface elements--remains responsive to user actions.</span></span>  
  
 <span data-ttu-id="5bb72-116">Wątkowość rozwiązuje problemy z przepływność i elastyczność, ale mogą też stać udostępniania zasobów problemy, takie jak zakleszczenie i sytuacja wyścigu.</span><span class="sxs-lookup"><span data-stu-id="5bb72-116">Threading solves problems with throughput and responsiveness, but it can also introduce resource-sharing issues such as deadlocks and race conditions.</span></span> <span data-ttu-id="5bb72-117">Wiele wątków są najlepsze w przypadku zadania, które wymagają różnych zasobów, takich jak dojścia do plików i połączeń sieciowych.</span><span class="sxs-lookup"><span data-stu-id="5bb72-117">Multiple threads are best for tasks that require different resources such as file handles and network connections.</span></span> <span data-ttu-id="5bb72-118">Przypisywanie wiele wątków do pojedynczego zasobu jest może spowodować problemy z synchronizacją, a o wątków często blokowane podczas oczekiwania na inne wątki pozbawia sensu używanie wielu wątków.</span><span class="sxs-lookup"><span data-stu-id="5bb72-118">Assigning multiple threads to a single resource is likely to cause synchronization issues, and having threads frequently blocked when waiting for other threads defeats the purpose of using multiple threads.</span></span>  
  
 <span data-ttu-id="5bb72-119">Wspólna strategia polega na użyciu wątków roboczych do wykonania czasochłonne wrażliwego na czas zadania, które nie wymagają wielu zasoby używane przez inne wątki.</span><span class="sxs-lookup"><span data-stu-id="5bb72-119">A common strategy is to use worker threads to perform time-consuming or time-critical tasks that do not require many of the resources used by other threads.</span></span> <span data-ttu-id="5bb72-120">Oczywiście niektórych zasobów w programie muszą być dostępne przez wiele wątków.</span><span class="sxs-lookup"><span data-stu-id="5bb72-120">Naturally, some resources in your program must be accessed by multiple threads.</span></span> <span data-ttu-id="5bb72-121">W tych przypadkach <xref:System.Threading> przestrzeń nazw zawiera klasy dla synchronizacja wątków.</span><span class="sxs-lookup"><span data-stu-id="5bb72-121">For these cases, the <xref:System.Threading> namespace provides classes for synchronizing threads.</span></span> <span data-ttu-id="5bb72-122">Te klasy mają <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, i <xref:System.Threading.ManualResetEvent>.</span><span class="sxs-lookup"><span data-stu-id="5bb72-122">These classes include <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, and <xref:System.Threading.ManualResetEvent>.</span></span>  
  
 <span data-ttu-id="5bb72-123">Można zsynchronizować działania wiele wątków niektórych lub wszystkich tych klas, ale niektóre Obsługa wątkowości jest obsługiwany przez język C#.</span><span class="sxs-lookup"><span data-stu-id="5bb72-123">You can use some or all these classes to synchronize the activities of multiple threads, but some support for threading is supported by the C# language.</span></span> <span data-ttu-id="5bb72-124">Na przykład [instrukcji "Lock"](../../../../csharp/language-reference/keywords/lock-statement.md) udostępnia funkcje synchronizacji przy użyciu niejawnego <xref:System.Threading.Monitor>.</span><span class="sxs-lookup"><span data-stu-id="5bb72-124">For example, the [Lock Statement](../../../../csharp/language-reference/keywords/lock-statement.md) provides synchronization features through implicit use of <xref:System.Threading.Monitor>.</span></span>
