 <span data-ttu-id="1a5c1-113">Wątki robocze może służyć do wykonywania zadań czasochłonne lub wrażliwego na czas bez zajmowania wątku głównego.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-113">Worker threads can be used to perform time-consuming or time-critical tasks without tying up the primary thread.</span></span> <span data-ttu-id="1a5c1-114">Na przykład wątki robocze są często używane w aplikacjach serwerowych do spełnienia żądania przychodzące bez oczekiwania na ukończenie poprzedniego żądania.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-114">For example, worker threads are often used in server applications to fulfill incoming requests without waiting for the previous request to be completed.</span></span> <span data-ttu-id="1a5c1-115">Wątki robocze są również używane do wykonywania zadań "tła" w aplikacjach pulpitu tak, aby główny wątek — dyski elementy interfejsu użytkownika — pozostaje reagują na czynności użytkownika.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-115">Worker threads are also used to perform "background" tasks in desktop applications so that the main thread--which drives user interface elements--remains responsive to user actions.</span></span>  
  
 <span data-ttu-id="1a5c1-116">Wątkowość rozwiązuje problemy z przepływności i czasu odpowiedzi, ale również mogą wprowadzać udostępnianie zasobów problemy, takie jak zakleszczeń i sytuacje wyścigu.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-116">Threading solves problems with throughput and responsiveness, but it can also introduce resource-sharing issues such as deadlocks and race conditions.</span></span> <span data-ttu-id="1a5c1-117">Wiele wątków są najlepsze w przypadku zadań, które wymagają różnych zasobów, takich jak dojścia do plików i połączeń sieciowych.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-117">Multiple threads are best for tasks that require different resources such as file handles and network connections.</span></span> <span data-ttu-id="1a5c1-118">Przypisywanie wielu wątków do pojedynczego zasobu jest może spowodować problemy z synchronizacją i konieczności często blokowane podczas oczekiwania na inne wątki wątki pozbawia przy użyciu wielu wątków.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-118">Assigning multiple threads to a single resource is likely to cause synchronization issues, and having threads frequently blocked when waiting for other threads defeats the purpose of using multiple threads.</span></span>  
  
 <span data-ttu-id="1a5c1-119">Typowe strategia polega na wątki robocze do wykonania czasochłonnych lub wrażliwego na czas zadania, które nie wymagają wielu zasoby używane przez inne wątki.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-119">A common strategy is to use worker threads to perform time-consuming or time-critical tasks that do not require many of the resources used by other threads.</span></span> <span data-ttu-id="1a5c1-120">Oczywiście niektórych zasobów w programie muszą być dostępne w wielu wątkach.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-120">Naturally, some resources in your program must be accessed by multiple threads.</span></span> <span data-ttu-id="1a5c1-121">W takich przypadkach <xref:System.Threading> przestrzeń nazwy udostępnia klasy do synchronizacji wątków.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-121">For these cases, the <xref:System.Threading> namespace provides classes for synchronizing threads.</span></span> <span data-ttu-id="1a5c1-122">Te klasy mają <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, i <xref:System.Threading.ManualResetEvent>.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-122">These classes include <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, and <xref:System.Threading.ManualResetEvent>.</span></span>  
  
 <span data-ttu-id="1a5c1-123">Niektórych lub wszystkich tych klas służy do synchronizowania działania wielu wątków, ale niektóre Obsługa wątkowości jest obsługiwany przez język C#.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-123">You can use some or all these classes to synchronize the activities of multiple threads, but some support for threading is supported by the C# language.</span></span> <span data-ttu-id="1a5c1-124">Na przykład [instrukcji "Lock"](../../../../csharp/language-reference/keywords/lock-statement.md) udostępnia funkcje synchronizacji przy użyciu niejawnego <xref:System.Threading.Monitor>.</span><span class="sxs-lookup"><span data-stu-id="1a5c1-124">For example, the [Lock Statement](../../../../csharp/language-reference/keywords/lock-statement.md) provides synchronization features through implicit use of <xref:System.Threading.Monitor>.</span></span>
