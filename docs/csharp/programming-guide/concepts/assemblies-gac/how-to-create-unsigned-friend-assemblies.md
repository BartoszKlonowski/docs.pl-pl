# <a name="how-to-create-unsigned-friend-assemblies-c"></a><span data-ttu-id="48af3-102">Instrukcje: Utwórz niepodpisane zestawy zaprzyjaźnione (C#)</span><span class="sxs-lookup"><span data-stu-id="48af3-102">How to: Create Unsigned Friend Assemblies (C#)</span></span>
<span data-ttu-id="48af3-103">Ten przykład pokazuje, jak używać zespołów zaprzyjaźnionych z zestawami, które nie są podpisane.</span><span class="sxs-lookup"><span data-stu-id="48af3-103">This example shows how to use friend assemblies with assemblies that are unsigned.</span></span>  
  
### <a name="to-create-an-assembly-and-a-friend-assembly"></a><span data-ttu-id="48af3-104">Aby utworzyć zestaw i zestaw zaprzyjaźniony</span><span class="sxs-lookup"><span data-stu-id="48af3-104">To create an assembly and a friend assembly</span></span>  
  
1. <span data-ttu-id="48af3-105">Otwórz wiersz polecenia.</span><span class="sxs-lookup"><span data-stu-id="48af3-105">Open a command prompt.</span></span>  
  
2. <span data-ttu-id="48af3-106">Utwórz C# plik o nazwie `friend_unsigned_A.` , który zawiera następujący kod.</span><span class="sxs-lookup"><span data-stu-id="48af3-106">Create a C# file named `friend_unsigned_A.` that contains the following code.</span></span> <span data-ttu-id="48af3-107">Kod używa <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> atrybutu, aby zadeklarować friend_unsigned_B jako zestaw zaprzyjaźniony.</span><span class="sxs-lookup"><span data-stu-id="48af3-107">The code uses the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute to declare friend_unsigned_B as a friend assembly.</span></span>  
  
    ```csharp  
    // friend_unsigned_A.cs  
    // Compile with:   
    // csc /target:library friend_unsigned_A.cs  
    using System.Runtime.CompilerServices;  
    using System;  
  
    [assembly: InternalsVisibleTo("friend_unsigned_B")]  
  
    // Type is internal by default.  
    class Class1  
    {  
        public void Test()  
        {  
            Console.WriteLine("Class1.Test");  
        }  
    }  
  
    // Public type with internal member.  
    public class Class2  
    {  
        internal void Test()  
        {  
            Console.WriteLine("Class2.Test");  
        }  
    }  
    ```  
  
3. <span data-ttu-id="48af3-108">Kompiluj i podpisz friend_unsigned_A za pomocą następującego polecenia.</span><span class="sxs-lookup"><span data-stu-id="48af3-108">Compile and sign friend_unsigned_A by using the following command.</span></span>  
  
    ```csharp  
    csc /target:library friend_unsigned_A.cs  
    ```  
  
4. <span data-ttu-id="48af3-109">Utwórz C# plik o nazwie `friend_unsigned_B` , który zawiera następujący kod.</span><span class="sxs-lookup"><span data-stu-id="48af3-109">Create a C# file named `friend_unsigned_B` that contains the following code.</span></span> <span data-ttu-id="48af3-110">Ponieważ friend_unsigned_A określa friend_unsigned_B jako zestaw zaprzyjaźniony, kod w friend_unsigned_B ma dostęp do `internal` typów i członków z friend_unsigned_A.</span><span class="sxs-lookup"><span data-stu-id="48af3-110">Because friend_unsigned_A specifies friend_unsigned_B as a friend assembly, the code in friend_unsigned_B can access `internal` types and members from friend_unsigned_A.</span></span>  
  
    ```csharp  
    // friend_unsigned_B.cs  
    // Compile with:   
    // csc /r:friend_unsigned_A.dll /out:friend_unsigned_B.exe friend_unsigned_B.cs  
    public class Program  
    {  
        static void Main()  
        {  
            // Access an internal type.  
            Class1 inst1 = new Class1();  
            inst1.Test();  
  
            Class2 inst2 = new Class2();  
            // Access an internal member of a public type.  
            inst2.Test();  
  
            System.Console.ReadLine();  
        }  
    }  
    ```  
  
5. <span data-ttu-id="48af3-111">Kompiluj friend_unsigned_B przy użyciu następującego polecenia.</span><span class="sxs-lookup"><span data-stu-id="48af3-111">Compile friend_unsigned_B by using the following command.</span></span>  
  
    ```csharp  
    csc /r:friend_unsigned_A.dll /out:friend_unsigned_B.exe friend_unsigned_B.cs  
    ```  
  
     <span data-ttu-id="48af3-112">Nazwa zestawu, który jest generowany przez kompilator musi być zgodna z nazwą zestawu zaprzyjaźnionego, który jest przesyłany do <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> atrybutu.</span><span class="sxs-lookup"><span data-stu-id="48af3-112">The name of the assembly that is generated by the compiler must match the friend assembly name that is passed to the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute.</span></span> <span data-ttu-id="48af3-113">Należy jawnie określić nazwę zestawu wyjściowego (. exe lub. dll) przy użyciu `/out` opcji kompilatora.</span><span class="sxs-lookup"><span data-stu-id="48af3-113">You must explicitly specify the name of the output assembly (.exe or .dll) by using the `/out` compiler option.</span></span> <span data-ttu-id="48af3-114">Aby uzyskać więcej informacji, zobacz [/outC# (opcje kompilatora)](../../../language-reference/compiler-options/out-compiler-option.md).</span><span class="sxs-lookup"><span data-stu-id="48af3-114">For more information, see [/out (C# Compiler Options)](../../../language-reference/compiler-options/out-compiler-option.md).</span></span>  
  
6. <span data-ttu-id="48af3-115">Uruchom plik friend_unsigned_B. exe.</span><span class="sxs-lookup"><span data-stu-id="48af3-115">Run the friend_unsigned_B.exe file.</span></span>  
  
     <span data-ttu-id="48af3-116">Program drukuje dwa ciągi: "Class1. test" i "'Klasa. test".</span><span class="sxs-lookup"><span data-stu-id="48af3-116">The program prints two strings: "Class1.Test" and "Class2.Test".</span></span>  
  
## <a name="net-framework-security"></a><span data-ttu-id="48af3-117">Zabezpieczenia.NET Framework</span><span class="sxs-lookup"><span data-stu-id="48af3-117">.NET Framework Security</span></span>  
 <span data-ttu-id="48af3-118">Istnieją podobieństwa między <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> atrybutem <xref:System.Security.Permissions.StrongNameIdentityPermission> a klasą.</span><span class="sxs-lookup"><span data-stu-id="48af3-118">There are similarities between the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute and the <xref:System.Security.Permissions.StrongNameIdentityPermission> class.</span></span> <span data-ttu-id="48af3-119">Główną różnicą jest to <xref:System.Security.Permissions.StrongNameIdentityPermission> , że może to wymagać uprawnień zabezpieczeń do uruchomienia określonej sekcji kodu, <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> natomiast `internal` atrybut kontroluje widoczność typów i elementów członkowskich.</span><span class="sxs-lookup"><span data-stu-id="48af3-119">The main difference is that <xref:System.Security.Permissions.StrongNameIdentityPermission> can demand security permissions to run a particular section of code, whereas the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute controls the visibility of `internal` types and members.</span></span>
