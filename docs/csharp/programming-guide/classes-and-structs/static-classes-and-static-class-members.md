<span data-ttu-id="e2edc-103">Klasa [statyczna](../../language-reference/keywords/static.md) jest zasadniczo taka sama jak Klasa niestatyczna, ale istnieje jedna różnica: nie można utworzyć wystąpienia klasy statycznej.</span><span class="sxs-lookup"><span data-stu-id="e2edc-103">A [static](../../language-reference/keywords/static.md) class is basically the same as a non-static class, but there is one difference: a static class cannot be instantiated.</span></span> <span data-ttu-id="e2edc-104">Innymi słowy, nie można użyć operatora [New](../../language-reference/operators/new-operator.md) , aby utworzyć zmienną typu klasy.</span><span class="sxs-lookup"><span data-stu-id="e2edc-104">In other words, you cannot use the [new](../../language-reference/operators/new-operator.md) operator to create a variable of the class type.</span></span> <span data-ttu-id="e2edc-105">Ponieważ nie istnieje zmienna wystąpienia, dostęp do elementów członkowskich klasy statycznej można uzyskać przy użyciu samej nazwy klasy.</span><span class="sxs-lookup"><span data-stu-id="e2edc-105">Because there is no instance variable, you access the members of a static class by using the class name itself.</span></span> <span data-ttu-id="e2edc-106">Na przykład jeśli masz klasę statyczną o nazwie `UtilityClass`, która ma publiczną metodę statyczną o nazwie `MethodA`, wywoływana jest metoda, jak pokazano w następującym przykładzie:</span><span class="sxs-lookup"><span data-stu-id="e2edc-106">For example, if you have a static class that is named `UtilityClass` that has a public static method named `MethodA`, you call the method as shown in the following example:</span></span>

```csharp  
UtilityClass.MethodA();  
```  
  
 <span data-ttu-id="e2edc-107">Klasa statyczna może być używana jako wygodny kontener dla zestawów metod, które po prostu działają w parametrach wejściowych i nie muszą pobierać ani ustawiać żadnych pól wystąpienia wewnętrznego.</span><span class="sxs-lookup"><span data-stu-id="e2edc-107">A static class can be used as a convenient container for sets of methods that just operate on input parameters and do not have to get or set any internal instance fields.</span></span> <span data-ttu-id="e2edc-108">Na przykład w bibliotece klas .NET Framework, Klasa statyczna <xref:System.Math?displayProperty=nameWithType> zawiera metody, które wykonują operacje matematyczne, bez konieczności przechowywania lub pobierania danych, które są unikatowe dla określonego wystąpienia klasy <xref:System.Math>.</span><span class="sxs-lookup"><span data-stu-id="e2edc-108">For example, in the .NET Framework Class Library, the static <xref:System.Math?displayProperty=nameWithType> class contains methods that perform mathematical operations, without any requirement to store or retrieve data that is unique to a particular instance of the <xref:System.Math> class.</span></span> <span data-ttu-id="e2edc-109">Oznacza to, że należy zastosować elementy członkowskie klasy, określając nazwę klasy i nazwę metody, jak pokazano w poniższym przykładzie.</span><span class="sxs-lookup"><span data-stu-id="e2edc-109">That is, you apply the members of the class by specifying the class name and the method name, as shown in the following example.</span></span>  
  
```csharp

Console.WriteLine(Math.Abs(dub));  
Console.WriteLine(Math.Floor(dub));  
Console.WriteLine(Math.Round(Math.Abs(dub)));  
  
// Output:  
// 3.14  
// -4  
// 3  
```  
  
 <span data-ttu-id="e2edc-110">Podobnie jak w przypadku wszystkich typów klas, informacje o typie klasy statycznej są ładowane przez .NET Framework środowiska uruchomieniowego języka wspólnego (CLR) podczas ładowania programu, który odwołuje się do klasy.</span><span class="sxs-lookup"><span data-stu-id="e2edc-110">As is the case with all class types, the type information for a static class is loaded by the .NET Framework common language runtime (CLR) when the program that references the class is loaded.</span></span> <span data-ttu-id="e2edc-111">Program nie może określić dokładnie, gdy Klasa jest załadowana.</span><span class="sxs-lookup"><span data-stu-id="e2edc-111">The program cannot specify exactly when the class is loaded.</span></span> <span data-ttu-id="e2edc-112">Jednak zagwarantowane jest załadowanie i zainicjowanie jego pól oraz jego konstruktora statycznego wywołanego przed odwołaniem do klasy po raz pierwszy w programie.</span><span class="sxs-lookup"><span data-stu-id="e2edc-112">However, it is guaranteed to be loaded and to have its fields initialized and its static constructor called before the class is referenced for the first time in your program.</span></span> <span data-ttu-id="e2edc-113">Konstruktor statyczny jest wywoływany tylko jednokrotnie, a Klasa statyczna pozostaje w pamięci przez okres istnienia domeny aplikacji, w której znajduje się program.</span><span class="sxs-lookup"><span data-stu-id="e2edc-113">A static constructor is only called one time, and a static class remains in memory for the lifetime of the application domain in which your program resides.</span></span>
