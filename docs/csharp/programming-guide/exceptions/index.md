## <a name="exceptions-overview"></a><span data-ttu-id="04217-112">Omówienie wyjątków</span><span class="sxs-lookup"><span data-stu-id="04217-112">Exceptions Overview</span></span>  
 <span data-ttu-id="04217-113">Wyjątki mają następujące właściwości:</span><span class="sxs-lookup"><span data-stu-id="04217-113">Exceptions have the following properties:</span></span>  
  
-   <span data-ttu-id="04217-114">Wyjątki są typów wyprowadzonych wszystkie ostatecznie z `System.Exception`.</span><span class="sxs-lookup"><span data-stu-id="04217-114">Exceptions are types that all ultimately derive from `System.Exception`.</span></span>  
  
-   <span data-ttu-id="04217-115">Użyj `try` bloku wokół instrukcji, które mogą zgłaszać wyjątków.</span><span class="sxs-lookup"><span data-stu-id="04217-115">Use a `try` block around the statements that might throw exceptions.</span></span>  
  
-   <span data-ttu-id="04217-116">Gdy wystąpi wyjątek w `try` zablokować przepływu sterowania przechodzi do pierwszego programu obsługi skojarzony wyjątek, który znajduje się w dowolnym miejscu w stosie wywołań.</span><span class="sxs-lookup"><span data-stu-id="04217-116">Once an exception occurs in the `try` block, the flow of control jumps to the first associated exception handler that is present anywhere in the call stack.</span></span> <span data-ttu-id="04217-117">W języku C# `catch` — słowo kluczowe jest używane do definiowania obsługi wyjątków.</span><span class="sxs-lookup"><span data-stu-id="04217-117">In C#, the `catch` keyword is used to define an exception handler.</span></span>  
  
-   <span data-ttu-id="04217-118">Jeśli żaden moduł obsługi wyjątku dla danego wyjątku, program zatrzymuje wykonywanie komunikat o błędzie.</span><span class="sxs-lookup"><span data-stu-id="04217-118">If no exception handler for a given exception is present, the program stops executing with an error message.</span></span>  
  
-   <span data-ttu-id="04217-119">Nie przechwytuj wyjątek, chyba że można go obsłużyć i pozostawić aplikacji w znanego stanu.</span><span class="sxs-lookup"><span data-stu-id="04217-119">Do not catch an exception unless you can handle it and leave the application in a known state.</span></span> <span data-ttu-id="04217-120">Jeśli zostaną wychwycone `System.Exception`, za pomocą rethrow `throw` — słowo kluczowe na końcu `catch` bloku.</span><span class="sxs-lookup"><span data-stu-id="04217-120">If you catch `System.Exception`, rethrow it using the `throw` keyword at the end of the `catch` block.</span></span>  
  
-   <span data-ttu-id="04217-121">Jeśli `catch` bloku definiuje zmienną wyjątek, można go uzyskać więcej informacji na temat typ wyjątku, który wystąpił.</span><span class="sxs-lookup"><span data-stu-id="04217-121">If a `catch` block defines an exception variable, you can use it to obtain more information about the type of exception that occurred.</span></span>  
  
-   <span data-ttu-id="04217-122">Wyjątki można jawnie wygenerowanych przez program przy użyciu `throw` — słowo kluczowe.</span><span class="sxs-lookup"><span data-stu-id="04217-122">Exceptions can be explicitly generated by a program by using the `throw` keyword.</span></span>  
  
-   <span data-ttu-id="04217-123">Obiekty wyjątków zawiera szczegółowe informacje na temat tego błędu, takie jak stan stosu wywołań i opis błędu.</span><span class="sxs-lookup"><span data-stu-id="04217-123">Exception objects contain detailed information about the error, such as the state of the call stack and a text description of the error.</span></span>  
  
-   <span data-ttu-id="04217-124">Kod w `finally` blok jest wykonywana nawet wtedy, gdy jest zgłaszany wyjątek.</span><span class="sxs-lookup"><span data-stu-id="04217-124">Code in a `finally` block is executed even if an exception is thrown.</span></span> <span data-ttu-id="04217-125">Użyj `finally` bloku, aby zwolnić zasoby, na przykład zamknąć wszystkie strumienie lub pliki, które były otwarte w `try` bloku.</span><span class="sxs-lookup"><span data-stu-id="04217-125">Use a `finally` block to release resources, for example to close any streams or files that were opened in the `try` block.</span></span>  
  
-   <span data-ttu-id="04217-126">Zarządzane wyjątki w programie .NET Framework są wdrażane na podstawie mechanizmu obsługi wyjątków strukturalnych Win32.</span><span class="sxs-lookup"><span data-stu-id="04217-126">Managed exceptions in the .NET Framework are implemented on top of the Win32 structured exception handling mechanism.</span></span> <span data-ttu-id="04217-127">Aby uzyskać więcej informacji, zobacz [strukturalnych obsługi wyjątków (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) i [A awarii kursu na głębokości systemu Win32 strukturalnych obsługi wyjątków](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm).</span><span class="sxs-lookup"><span data-stu-id="04217-127">For more information, see [Structured Exception Handling (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) and [A Crash Course on the Depths of Win32 Structured Exception Handling](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm).</span></span>
