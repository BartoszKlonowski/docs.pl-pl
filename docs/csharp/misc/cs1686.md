# <a name="compiler-error-cs1686"></a><span data-ttu-id="4c091-102">Błąd kompilatora CS1686</span><span class="sxs-lookup"><span data-stu-id="4c091-102">Compiler Error CS1686</span></span>

<span data-ttu-id="4c091-103">Lokalna "zmienna" lub jej składowe nie mogą mieć adresu i są używane wewnątrz metody anonimowej lub wyrażenia lambda</span><span class="sxs-lookup"><span data-stu-id="4c091-103">Local 'variable' or its members cannot have their address taken and be used inside an anonymous method or lambda expression</span></span>  
  
 <span data-ttu-id="4c091-104">Ten błąd jest generowany, gdy używana jest zmienna i próbuje uzyskać swój adres, a jedna z tych akcji jest wykonywana wewnątrz metody anonimowej.</span><span class="sxs-lookup"><span data-stu-id="4c091-104">This error is generated when you use a variable, and attempt to take its address, and one of these actions is done inside an anonymous method.</span></span>  
  
## <a name="example"></a><span data-ttu-id="4c091-105">Przykład</span><span class="sxs-lookup"><span data-stu-id="4c091-105">Example</span></span>  

 <span data-ttu-id="4c091-106">Poniższy przykład generuje CS1686.</span><span class="sxs-lookup"><span data-stu-id="4c091-106">The following sample generates CS1686.</span></span>  
  
```csharp  
// CS1686.cs  
// compile with: /unsafe /target:library  
class MyClass  
{  
   public unsafe delegate int * MyDelegate();  
  
   public unsafe int * Test()  
   {  
      int j = 0;  
      MyDelegate d = delegate { return &j; };   // CS1686  
      return &j;   // OK  
   }  
}  
```
