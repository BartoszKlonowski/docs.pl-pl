# <a name="compiler-error-cs0837"></a>CS0837 błąd kompilatora
Pierwszy argument operacji "is" lub "as" operator nie może być wyrażenie lambda lub metodę anonimową.  
  
 Wyrażenia lambda i metod anonimowych, nie mogą zostać użyte w lewej części [jest](../../csharp/language-reference/keywords/is.md) lub [jako](../../csharp/language-reference/keywords/as.md).  
  
## <a name="to-correct-this-error"></a>Aby poprawić ten błąd  
  
- Jeśli ten błąd obejmuje `is` operatora, należy pamiętać, że `is` przyjmuje wartość i typ i informuje, czy wartość może się do tego typu przez odwołanie, pakowanie i rozpakowywanie konwersji. Ponieważ wyrażenia lambda nie są wartościami i mają żadne odwołanie, pakowanie i rozpakowywanie konwersji, wyrażenia lambda nie są kandydatami do `is`.  
  
- Jeśli niewłaściwym kodu `as`, naprawianie jest prawdopodobnie można go zmienić na rzutowania.  
  
## <a name="example"></a>Przykład  
 Poniższy przykład generuje CS0837:  
  
```csharp  
// cs0837.cs  
namespace TestNamespace  
{  
    public delegate void Del();  
  
    class Test  
    {  
        static int Main()  
        {  
            bool b1 = (() => { }) is Del;   // CS0837  
            bool b2 = delegate() { } is Del;// CS0837  
            Del d1 = () => { } as Del;      // CS0837  
            Del d2 = delegate() { } as Del; // CS0837  
            return 1;  
        }  
    }  
}  
```
