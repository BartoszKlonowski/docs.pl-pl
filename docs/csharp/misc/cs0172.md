# <a name="compiler-error-cs0172"></a><span data-ttu-id="45afd-103">Błąd kompilatora CS0172</span><span class="sxs-lookup"><span data-stu-id="45afd-103">Compiler Error CS0172</span></span>
<span data-ttu-id="45afd-104">Nie można określić typu wyrażenia warunkowego, ponieważ wartości "type1" i "type2" niejawnie przekonwertują się na siebie</span><span class="sxs-lookup"><span data-stu-id="45afd-104">Type of conditional expression cannot be determined because 'type1' and 'type2' implicitly convert to one another</span></span>  
  
<span data-ttu-id="45afd-105">W instrukcji warunkowej, musisz być w stanie niejawnie skonwertować typy po obu stronach `:` tokenu.</span><span class="sxs-lookup"><span data-stu-id="45afd-105">In a conditional statement, you must be able to implicitly convert the types on either side of the `:` token.</span></span> <span data-ttu-id="45afd-106">Ponadto nie mogą istnieć wzajemnie niejawne konwersje; potrzebna jest tylko jedna konwersja.</span><span class="sxs-lookup"><span data-stu-id="45afd-106">Also, there cannot be mutual implicit conversions; you only need one conversion.</span></span>
  
<span data-ttu-id="45afd-107">Poniższy przykład generuje CS0172:</span><span class="sxs-lookup"><span data-stu-id="45afd-107">The following sample generates CS0172:</span></span>
  
```csharp  
// CS0172.cs  
public class Square  
{  
   public class Circle  
   {  
      public static implicit operator Circle(Square aa)  
      {  
         return null;  
      }  
  
      public static implicit operator Square(Circle aa)  
      // using explicit resolves this error  
      // public static explicit operator Square(Circle aa)  
      {  
         return null;  
      }  
   }  
  
   public static void Main()  
   {  
      Circle aa = new Circle();  
      Square ii = new Square();  
      object o = (1 == 1) ? aa : ii;   // CS0172  
      // the following cast would resolve this error  
      // (1 == 1) ? aa : (Circle)ii;  
   }  
}  
```
