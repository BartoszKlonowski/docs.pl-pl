# <a name="compiler-error-cs0172"></a><span data-ttu-id="b4740-102">CS0172 błąd kompilatora</span><span class="sxs-lookup"><span data-stu-id="b4740-102">Compiler Error CS0172</span></span>
<span data-ttu-id="b4740-103">Nie można określić typu wyrażenia warunkowego, ponieważ 'Typ1' i 'Typ2' niejawnie konwertowane ze sobą</span><span class="sxs-lookup"><span data-stu-id="b4740-103">Type of conditional expression cannot be determined because 'type1' and 'type2' implicitly convert to one another</span></span>  
  
<span data-ttu-id="b4740-104">W instrukcji warunkowej, musi mieć możliwość niejawnej konwersji typów po obu stronach `:` tokenu.</span><span class="sxs-lookup"><span data-stu-id="b4740-104">In a conditional statement, you must be able to implicitly convert the types on either side of the `:` token.</span></span> <span data-ttu-id="b4740-105">Ponadto nie może być wzajemnej konwersji niejawnych; wystarczy tylko jednej konwersji.</span><span class="sxs-lookup"><span data-stu-id="b4740-105">Also, there cannot be mutual implicit conversions; you only need one conversion.</span></span>
  
<span data-ttu-id="b4740-106">Poniższy przykład spowoduje wygenerowanie CS0172:</span><span class="sxs-lookup"><span data-stu-id="b4740-106">The following sample generates CS0172:</span></span>
  
```csharp  
// CS0172.cs  
public class Square  
{  
   public class Circle  
   {  
      public static implicit operator Circle(Square aa)  
      {  
         return null;  
      }  
  
      public static implicit operator Square(Circle aa)  
      // using explicit resolves this error  
      // public static explicit operator Square(Circle aa)  
      {  
         return null;  
      }  
   }  
  
   public static void Main()  
   {  
      Circle aa = new Circle();  
      Square ii = new Square();  
      object o = (1 == 1) ? aa : ii;   // CS0172  
      // the following cast would resolve this error  
      // (1 == 1) ? aa : (Circle)ii;  
   }  
}  
```
