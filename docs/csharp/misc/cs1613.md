# <a name="compiler-error-cs1613"></a><span data-ttu-id="dafa1-102">CS1613 błąd kompilatora</span><span class="sxs-lookup"><span data-stu-id="dafa1-102">Compiler Error CS1613</span></span>
<span data-ttu-id="dafa1-103">Nie można odnaleźć klasy otoki coclass zarządzanego "class" dla interfejsu "interface" (czy nie brakuje odwołania do zestawu?)</span><span class="sxs-lookup"><span data-stu-id="dafa1-103">The managed coclass wrapper class 'class' for interface 'interface' cannot be found (are you missing an assembly reference?)</span></span>  
  
 <span data-ttu-id="dafa1-104">Nastąpiła próba tworzenia wystąpienia obiektu COM z interfejsu.</span><span class="sxs-lookup"><span data-stu-id="dafa1-104">An attempt was made to instantiate a COM object from an interface.</span></span> <span data-ttu-id="dafa1-105">Interfejs ma **ComImport** i `CoClass` atrybuty, ale kompilator nie można odnaleźć typu danych dotyczących `CoClass` atrybutu.</span><span class="sxs-lookup"><span data-stu-id="dafa1-105">The interface has the **ComImport** and `CoClass` attributes, but the compiler cannot find the type given for the `CoClass` attribute.</span></span>  
  
 <span data-ttu-id="dafa1-106">Aby rozwiązać ten problem, możesz wypróbować jedną z następujących czynności:</span><span class="sxs-lookup"><span data-stu-id="dafa1-106">To resolve this error, you can try one of the following:</span></span>  
  
-   <span data-ttu-id="dafa1-107">Dodaj odwołanie do zestawu, który ma coclass (w większości przypadków interfejsu i coclass musi należeć do tego samego zestawu).</span><span class="sxs-lookup"><span data-stu-id="dafa1-107">Add a reference to the assembly that has the coclass (most of the time the interface and coclass should be in the same assembly).</span></span> <span data-ttu-id="dafa1-108">Zobacz [/reference](../../csharp/language-reference/compiler-options/reference-compiler-option.md) lub [odwołania okno dialogowe Dodawanie](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) informacji.</span><span class="sxs-lookup"><span data-stu-id="dafa1-108">See [/reference](../../csharp/language-reference/compiler-options/reference-compiler-option.md) or [Add Reference Dialog Box](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) for information.</span></span>  
  
-   <span data-ttu-id="dafa1-109">Napraw `CoClass` atrybutu w interfejsie.</span><span class="sxs-lookup"><span data-stu-id="dafa1-109">Fix the `CoClass` attribute on the interface.</span></span>  
  
 <span data-ttu-id="dafa1-110">W poniższym przykładzie pokazano poprawne użycie **CoClassAttribute**:</span><span class="sxs-lookup"><span data-stu-id="dafa1-110">The following sample demonstrates correct usage of **CoClassAttribute**:</span></span>  
  
```csharp  
// CS1613.cs  
using System;  
using System.Runtime.InteropServices;  
  
[Guid("1FFD7840-E82D-4268-875C-80A160C23296")]  
[ComImport()]  
[CoClass(typeof(A))]  
public interface IA{}  
public class A : IA {}  
  
public class AA  
{  
   public static void Main()  
   {  
      IA i;  
      i = new IA(); // This is equivalent to new A().  
                    // because of the CoClass attribute on IA  
   }  
}  
```
