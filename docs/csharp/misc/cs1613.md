# <a name="compiler-error-cs1613"></a>Błąd kompilatora CS1613
Nie można znaleźć zarządzanej klasy otoki coclass "Class" dla interfejsu "Interface" (czy nie brakuje odwołania do zestawu?)  
  
 Podjęto próbę utworzenia wystąpienia obiektu COM z interfejsu. Interfejs ma **atrybutem ComImport** i `CoClass` atrybuty, ale kompilator nie może znaleźć `CoClass` typu podanych dla atrybutu.  
  
 Aby rozwiązać ten problem, możesz wypróbować jedną z następujących czynności:  
  
- Dodaj odwołanie do zestawu, który ma klasę coclass (większość czasu interfejs i Klasa coclass powinny znajdować się w tym samym zestawie). Aby uzyskać informacje, zobacz okno dialogowe [/Reference](../language-reference/compiler-options/reference-compiler-option.md) lub [Dodawanie odwołania](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) .  
  
- `CoClass` Popraw atrybut w interfejsie.  
  
 W poniższym przykładzie pokazano poprawne użycie **klasy CoClassAttribute**:  
  
```csharp  
// CS1613.cs  
using System;  
using System.Runtime.InteropServices;  
  
[Guid("1FFD7840-E82D-4268-875C-80A160C23296")]  
[ComImport()]  
[CoClass(typeof(A))]  
public interface IA{}  
public class A : IA {}  
  
public class AA  
{  
   public static void Main()  
   {  
      IA i;  
      i = new IA(); // This is equivalent to new A().  
                    // because of the CoClass attribute on IA  
   }  
}  
```
