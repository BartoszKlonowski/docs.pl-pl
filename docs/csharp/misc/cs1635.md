# <a name="compiler-warning-level-1-cs1635"></a>Kompilatora (poziom 1) ostrzeżenie CS1635
Nie można przywrócić ostrzeżenia "Kod ostrzeżenia", ponieważ zostało globalnie wyłączone  
  
 To ostrzeżenie występuje, jeśli używasz **/nowarn** polecenia opcji lub projektu, ustawienie wyłączania ostrzeżenie dla jednostki całej kompilacji, ale użyć `#pragma warning restore` próbę przywrócić tego ostrzeżenia. Aby rozwiązać ten problem, Usuń/nowarn — opcja wiersza polecenia lub ustawienie projektu lub usuń `#pragma warning restore` dla dowolnego ostrzeżenia wyłączania za pośrednictwem wiersza polecenia lub w ustawieniach projektu. Aby uzyskać więcej informacji, zobacz [ostrzeżenie #pragma](../../csharp/language-reference/preprocessor-directives/preprocessor-pragma-warning.md) tematu.  
  
 Poniższy przykład spowoduje wygenerowanie CS1635:  
  
```csharp  
// CS1635.cs  
// compile with: /w:1 /nowarn:162  
  
enum MyEnum {one=1,two=2,three=3};  
  
class MyClass  
{  
    public static void Main()  
    {  
#pragma warning disable 162  
  
    if (MyEnum.three == MyEnum.two)  
        System.Console.WriteLine("Duplicate");  
  
#pragma warning restore 162  
    }  
}  
```
