# <a name="compiler-error-cs0559"></a>CS0559 błąd kompilatora
Typ parametru ++ lub--operatora musi być typem zawierającym  
  
 Deklaracja metoda przeciążenia operatora musi występować po niektórych wytycznych. Dla ++ i--operatorów, jest wymagany parametr stosowania tego samego typu jako typu, w którym operator jest przeciążona.  
  
## <a name="example"></a>Przykład  
 Poniższy przykład generuje CS0559:  
  
```  
// CS0559.cs  
// compile with: /target:library  
public class iii  
{  
   public static implicit operator int(iii x)  
   {  
      return 0;  
   }  
  
   public static implicit operator iii(int x)  
   {  
      return null;  
   }  
  
   public static int operator ++(int aa)   // CS0559  
   // try the following line instead  
   // public static iii operator ++(iii aa)  
   {  
      return (iii)0;  
   }  
}  
```  
  
## <a name="example"></a>Przykład  
 Poniższy przykład generuje CS0559.  
  
```  
// CS0559_b.cs  
// compile with: /target:library  
public struct S  
{  
   public static S operator ++(S? s) { return new S(); }   // CS0559  
   public static S operator --(S? s) { return new S(); }   // CS0559  
}  
  
public struct T  
{  
// OK  
   public static T operator --(T t) { return new T(); }  
   public static T operator ++(T t) { return new T(); }  
  
   public static T? operator --(T? t) { return new T(); }  
   public static T? operator ++(T? t) { return new T(); }  
}  
```
