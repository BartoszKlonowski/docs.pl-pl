# <a name="compiler-error-cs0012"></a>Błąd kompilatora CS0012
Typ "Type" jest zdefiniowany w zestawie, do którego nie istnieje odwołanie. Należy dodać odwołanie do zestawu "Assembly".  
  
 Nie znaleziono definicji dla typu, którego dotyczy odwołanie. Taka sytuacja może wystąpić, jeśli jest to wymagane. Plik DLL nie jest uwzględniony w kompilacji. Aby uzyskać więcej informacji, zobacz okno [dialogowe Dodawanie odwołania](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) i [/ReferenceC# (opcje kompilatora)](../language-reference/compiler-options/reference-compiler-option.md).  
  
 Następująca sekwencja kompilacji spowoduje CS0012:  
  
```csharp  
// cs0012a.cs  
// compile with: /target:library  
public class A {}  
```  
  
 Następnie  
  
```csharp  
// cs0012b.cs  
// compile with: /target:library /reference:cs0012a.dll  
public class B  
{  
   public static A f()  
   {  
      return new A();  
   }  
}  
```  
  
 Następnie  
  
```csharp  
// cs0012c.cs  
// compile with: /reference:cs0012b.dll  
class C  
{  
   public static void Main()  
   {  
      object o = B.f();   // CS0012  
   }  
}  
```  
  
 Można rozwiązać ten CS0012 przez skompilowanie z `/reference:cs0012b.dll;cs0012a.dll`lub w programie Visual Studio za pomocą [okna dialogowego Dodaj odwołanie](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) , aby dodać odwołanie do cs0012a. dll oprócz cs0012b. dll.
