<span data-ttu-id="294ab-103">Metoda, operator lub metoda dostępu "method" jest oznaczona jako zewnętrzna i nie ma na nim żadnych atrybutów.</span><span class="sxs-lookup"><span data-stu-id="294ab-103">Method, operator, or accessor 'method' is marked external and has no attributes on it.</span></span> <span data-ttu-id="294ab-104">Rozważ dodanie atrybutu DllImport w celu określenia implementacji zewnętrznej.</span><span class="sxs-lookup"><span data-stu-id="294ab-104">Consider adding a DllImport attribute to specify the external implementation.</span></span>
  
 <span data-ttu-id="294ab-105">Metoda oznaczona `extern` powinien również być oznaczone atrybutem, na przykład, [DllImport](xref:System.Runtime.InteropServices.DllImportAttribute) atrybutu.</span><span class="sxs-lookup"><span data-stu-id="294ab-105">A method marked `extern` should also be marked with an attribute, for example, the [DllImport](xref:System.Runtime.InteropServices.DllImportAttribute) attribute.</span></span>
  
 <span data-ttu-id="294ab-106">Ten atrybut określa, w którym zaimplementowano metody.</span><span class="sxs-lookup"><span data-stu-id="294ab-106">The attribute specifies where the method is implemented.</span></span> <span data-ttu-id="294ab-107">W czasie wykonywania program będzie to potrzebne.</span><span class="sxs-lookup"><span data-stu-id="294ab-107">At run time, the program will need this information.</span></span>

```csharp

public class MyClass  
{  
   static extern public void M(); // CS0626  
   // try the following line  
   // [DllImport("mydll.dll")] static extern public void M();

   {  
   }  
}  
```
