# <a name="compiler-warning-level-2-cs3021"></a>CS3021 ostrzeżenie (poziom 2) kompilator
"type" nie wymaga atrybutu CLSCompliant, ponieważ zestaw nie ma atrybutu CLSCompliant  
  
 To ostrzeżenie występuje, gdy `[CLSCompliant(false)]` pojawia się na klasy w zestawie, który nie ma wartości atrybutu CLSCompliant poziomie zestawu true (czyli wiersz `[assembly: CLSCompliant(true)]`). Ponieważ zestaw jest nie deklarując sam zgodny ze specyfikacją zgodne, nie ma potrzeby dla wszystkich elementów w obrębie zestawu można zadeklarować sam niezgodne, ponieważ zakłada się niezgodne. Aby uzyskać więcej informacji na temat zgodności ze specyfikacją CLS, zobacz [pisanie kodu zgodne ze specyfikacją CLS](http://msdn.microsoft.com/library/4c705105-69a2-4e5e-b24e-0633bc32c7f3).  
  
 Aby usunąć to ostrzeżenie, usuń atrybut lub Dodaj atrybut poziomu zestawu.  
  
## <a name="example"></a>Przykład  
 Poniższy przykład generuje CS3021:  
  
```csharp  
// CS3021.cs  
using System;  
// Uncomment the following line to declare the assembly CLS Compliant,  
// and avoid the warning without removing the attribute on the class.  
//[assembly: CLSCompliant(true)]  
  
// Remove the next line to avoid the warning.  
[CLSCompliant(false)]               // CS3021  
public class C  
{  
    public static void Main()  
    {  
    }  
}  
```
