# <a name="compiler-warning-level-3-cs1718"></a>Ostrzeżenie o kompilatorze (poziom 3) CS1718
Porównanie do tej samej zmiennej; czy chciałeś porównać coś innego?  
  
 Jeśli chcesz porównać do czegoś innego, to należy po prostu poprawić oświadczenie.  
  
 Ale inną możliwością jest to, że były badania dla prawdziwych `if (a == a) (true)` `if (a < a) (false)`lub fałszywych, i robili to przez oświadczenia, takie jak lub . Lepiej po prostu `if (true)` powiedzieć `if (false)`lub . Istnieją dwa powody takiego działania:  
  
- To jest prostsze: zawsze jaśniejsze jest po prostu powiedzieć, co masz na myśli.  
  
- Pomaga uniknąć nieporozumień: nowa funkcja C# 2.0 jest typy wartości nullable, które są analogiczne do wartości `null` w T-SQL, język programowania używany przez program SQL Server. Deweloperzy zaznajomieni z T-SQL może być zaniepokojony wpływ `if (a == a)`typów nullable na wyrażenia, takie jak , ze względu na użycie logiki ternary w T-SQL. Jeśli używasz `true` `false`lub , można uniknąć tego możliwego zamieszania.  
  
## <a name="example"></a>Przykład  
 Poniższy kod generuje ostrzeżenie CS1718.  
  
```csharp  
// CS1718.cs  
using System;  
public class Tester
{  
    public static void Main()
    {
        int i = 0;  
        if (i == i) { // CS1718.cs  
        //if (true) {
            i++;  
        }  
    }  
}  
```
