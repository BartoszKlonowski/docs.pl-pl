# <a name="compiler-error-cs5001"></a>Błąd kompilatora CS5001

Program "program" nie zawiera statycznej metody "Main" odpowiedniej dla punktu wejścia  
  
Ten błąd występuje, gdy w `Main` kodzie, który tworzy plik wykonywalny, nie zostanie znaleziona żadna metoda statyczna o poprawnym podpisie. Ten błąd występuje również, gdy funkcja punktu wejścia, `Main`,, jest zdefiniowana z niewłaściwym przypadkiem, na przykład `main`w przypadku małych liter. Aby uzyskać informacje o regułach, które są `Main` stosowane do metody, zobacz [Main () i argumenty wiersza polecenia](../programming-guide/main-and-command-args/index.md).

Jeśli metoda ma modyfikator, upewnij się, że [wybrana C# wersja językowa](../language-reference/configure-language-version.md) to 7,1 lub nowsza. `async` `Main`

Metoda jest wymagana tylko w przypadku kompilowania pliku wykonywalnego, to jest, kiedy [-target: exe](/dotnet/csharp/language-reference/compiler-options/target-exe-compiler-option) lub [-target: winexe](/dotnet/csharp/language-reference/compiler-options/target-winexe-compiler-option) opcja kompilatora jest określona. `Main` Następujące typy projektów programu Visual Studio domyślnie określają jedną z następujących opcji:

- Aplikacja konsolowa
- Aplikacja ASP.NET Core
- aplikacja WPF
- aplikacje Formularzy systemu Windows

## <a name="example"></a>Przykład  
 Poniższy przykład generuje CS5001:  
  
```csharp  
// CS5001.cs  
// CS5001 expected when compiled with -target:exe or -target:winexe 
public class a  
{  
   // Uncomment the following line to resolve.  
   // static void Main() {}  
}  
```
