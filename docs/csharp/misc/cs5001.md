# <a name="compiler-error-cs5001"></a>Błąd kompilatora CS5001

Program nie zawiera statycznej metody "Main" odpowiedniej dla punktu wejścia

Ten błąd występuje, gdy w kodzie, który tworzy plik wykonywalny, nie zostanie znaleziona statyczna metoda `Main` z prawidłową sygnaturą. Występuje również, gdy funkcja punktu wejścia, `Main`, jest zdefiniowana z niewłaściwym przypadkiem, na przykład w przypadku małych liter `main`. Aby uzyskać informacje o regułach, które dotyczą metody `Main`, zobacz [Main () i argumenty wiersza polecenia](../programming-guide/main-and-command-args/index.md).

Jeśli metoda `Main` ma modyfikator `async`, upewnij się, że [wybrana C# wersja językowa](../language-reference/configure-language-version.md) to 7,1 lub nowsza.

Metoda `Main` jest wymagana tylko w przypadku kompilowania pliku wykonywalnego, to jest, gdy jest określona opcja kompilatora [-target: exe](/dotnet/csharp/language-reference/compiler-options/target-exe-compiler-option) lub [-target: winexe](/dotnet/csharp/language-reference/compiler-options/target-winexe-compiler-option) . Następujące typy projektów programu Visual Studio domyślnie określają jedną z następujących opcji:

- Aplikacja konsolowa
- Aplikacja ASP.NET Core
- aplikacja WPF
- aplikacje Formularzy systemu Windows

## <a name="example"></a>Przykład

Poniższy przykład generuje CS5001:
  
```csharp
// CS5001.cs
// CS5001 expected when compiled with -target:exe or -target:winexe
public class Program
{
   // Uncomment the following line to resolve.
   // static void Main() {}
}
```
