# <a name="compiler-warning-level-3-cs0414"></a><span data-ttu-id="7cf41-102">Kompilatora (poziom 3) ostrzeżenie CS0414</span><span class="sxs-lookup"><span data-stu-id="7cf41-102">Compiler Warning (level 3) CS0414</span></span>
<span data-ttu-id="7cf41-103">Przypisano pole private "field", ale jego wartość nie jest nigdy używana</span><span class="sxs-lookup"><span data-stu-id="7cf41-103">The private field 'field' is assigned but its value is never used</span></span>  
  
 <span data-ttu-id="7cf41-104">To ostrzeżenie może wystąpić w kilku scenariuszach, w których kompilator można sprawdzić, czy nigdy nie odwołuje się zmienną:</span><span class="sxs-lookup"><span data-stu-id="7cf41-104">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>  
  
-   <span data-ttu-id="7cf41-105">Pole prywatne jest przypisywana wartość stałą, ale to nigdy nie odczytana.</span><span class="sxs-lookup"><span data-stu-id="7cf41-105">A private field is assigned a constant value but is never subsequently read.</span></span> <span data-ttu-id="7cf41-106">Niepotrzebne przypisania można wpływ na wydajność.</span><span class="sxs-lookup"><span data-stu-id="7cf41-106">The unnecessary assignment could effect performance.</span></span> <span data-ttu-id="7cf41-107">Rozważ usunięcie pola.</span><span class="sxs-lookup"><span data-stu-id="7cf41-107">Consider removing the field.</span></span>  
  
-   <span data-ttu-id="7cf41-108">Prywatne lub wewnętrzne pole statyczne jest przypisywana wartość stałej tylko w inicjatorze.</span><span class="sxs-lookup"><span data-stu-id="7cf41-108">A private or internal static field is assigned a constant value only in the initializer.</span></span> <span data-ttu-id="7cf41-109">Należy rozważyć zmianę pola na stałą.</span><span class="sxs-lookup"><span data-stu-id="7cf41-109">Consider changing the field to a const.</span></span>  
  
-   <span data-ttu-id="7cf41-110">Pole prywatne lub wewnętrzne przypisane wartości stałych i używane tylko w blokach, które są wykluczone przez dyrektywy #ifdef.</span><span class="sxs-lookup"><span data-stu-id="7cf41-110">A private or internal field is assigned constant values and only used in blocks that are excluded by #ifdef directives.</span></span> <span data-ttu-id="7cf41-111">Rozważ umieszczenie pola wewnątrz bloku #ifdef.</span><span class="sxs-lookup"><span data-stu-id="7cf41-111">Consider putting the field inside the #ifdef block.</span></span>  
  
-   <span data-ttu-id="7cf41-112">Pole prywatne lub wewnętrzne jest przypisane wartości stałych w wielu lokalizacjach, ale nie jest dostępny.</span><span class="sxs-lookup"><span data-stu-id="7cf41-112">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="7cf41-113">Jeśli pole nie jest konieczne, rozważ usunięcie go.</span><span class="sxs-lookup"><span data-stu-id="7cf41-113">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="7cf41-114">W przeciwnym razie użyj je w odpowiedni sposób.</span><span class="sxs-lookup"><span data-stu-id="7cf41-114">Otherwise, use it in some appropriate way.</span></span>  
  
 <span data-ttu-id="7cf41-115">W innych sytuacjach lub gdy nie jest dopuszczalne, sugerowane rozwiązania za pomocą #pragma 0414.</span><span class="sxs-lookup"><span data-stu-id="7cf41-115">In other situations, or where the suggested workaround is not acceptable, use #pragma 0414.</span></span>  
  
 <span data-ttu-id="7cf41-116">Poniższy przykład pokazuje jeden ze sposobów, w którym zostaną wygenerowane CS0414:</span><span class="sxs-lookup"><span data-stu-id="7cf41-116">The following sample shows one way in which CS0414 will be generated:</span></span>  
  
```csharp

{  
   private int i = 1;  // CS0414  
  
   public static void Main()  
   { }  
}  
```  
  
 <span data-ttu-id="7cf41-117">**Uwaga** Jeśli zmienna `i` jest zadeklarowany jako `protected or public`, nie zostanie wygenerowany błąd, ponieważ kompilator nie wie, czy użyć jej klasę pochodną lub inny kod klienta może utworzyć wystąpienie klasy i odwoływać się do zmiennej</span><span class="sxs-lookup"><span data-stu-id="7cf41-117">**Note** If the variable `i` is declared as `protected or public`, no error will be generated because the compiler cannot know whether a derived class might use it or some other client code might instantiate the class and reference the variable</span></span>
