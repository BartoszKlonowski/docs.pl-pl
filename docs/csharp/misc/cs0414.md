 <span data-ttu-id="cdc78-104">To ostrzeżenie może wystąpić w kilku scenariuszach, w których kompilator można sprawdzić, czy zmienna jest nigdy nie odwołuje się do:</span><span class="sxs-lookup"><span data-stu-id="cdc78-104">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>  
  
-   <span data-ttu-id="cdc78-105">Pole prywatne przypisano wartość stałą, ale jest nigdy nie odczytana.</span><span class="sxs-lookup"><span data-stu-id="cdc78-105">A private field is assigned a constant value but is never subsequently read.</span></span> <span data-ttu-id="cdc78-106">Przypisanie niepotrzebnych można dokonać wydajności.</span><span class="sxs-lookup"><span data-stu-id="cdc78-106">The unnecessary assignment could effect performance.</span></span> <span data-ttu-id="cdc78-107">Rozważ usunięcie pola.</span><span class="sxs-lookup"><span data-stu-id="cdc78-107">Consider removing the field.</span></span>  
  
-   <span data-ttu-id="cdc78-108">Prywatny lub wewnętrzny pola statycznego jest przypisywana wartość stałej tylko w inicjatorze.</span><span class="sxs-lookup"><span data-stu-id="cdc78-108">A private or internal static field is assigned a constant value only in the initializer.</span></span> <span data-ttu-id="cdc78-109">Rozważ zmianę typu const pola.</span><span class="sxs-lookup"><span data-stu-id="cdc78-109">Consider changing the field to a const.</span></span>  
  
-   <span data-ttu-id="cdc78-110">Pole prywatne lub wewnętrzne jest przypisane wartości stałych i tylko używane w blokach, które są wykluczone przez dyrektywy #ifdef.</span><span class="sxs-lookup"><span data-stu-id="cdc78-110">A private or internal field is assigned constant values and only used in blocks that are excluded by #ifdef directives.</span></span> <span data-ttu-id="cdc78-111">Należy rozważyć umieszczenie w polu wewnątrz bloku #ifdef.</span><span class="sxs-lookup"><span data-stu-id="cdc78-111">Consider putting the field inside the #ifdef block.</span></span>  
  
-   <span data-ttu-id="cdc78-112">Pole prywatne lub wewnętrzne jest przypisane wartości stałych w wielu lokalizacjach, ale nie jest dostępne.</span><span class="sxs-lookup"><span data-stu-id="cdc78-112">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="cdc78-113">Jeśli pole nie jest konieczne, należy rozważyć usunięcie go.</span><span class="sxs-lookup"><span data-stu-id="cdc78-113">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="cdc78-114">W przeciwnym razie użyj go w odpowiedni sposób.</span><span class="sxs-lookup"><span data-stu-id="cdc78-114">Otherwise, use it in some appropriate way.</span></span>  
  
 <span data-ttu-id="cdc78-115">W innych sytuacjach lub gdy sugerowane rozwiązania nie jest dopuszczalne Użyj #pragma 0414.</span><span class="sxs-lookup"><span data-stu-id="cdc78-115">In other situations, or where the suggested workaround is not acceptable, use #pragma 0414.</span></span>  
  
 <span data-ttu-id="cdc78-116">Poniższy przykład przedstawia sposób w którym zostanie wygenerowany CS0414:</span><span class="sxs-lookup"><span data-stu-id="cdc78-116">The following sample shows one way in which CS0414 will be generated:</span></span>  
  
```csharp  
// CS0414

{  
   private int i = 1;  // CS0414  
  
   public static void Main()  
   { }  
}
```  

 <span data-ttu-id="cdc78-117">**Uwaga** Jeśli zmienna `i` jest zadeklarowany jako `protected or public`, nie zostanie wygenerowany błąd, ponieważ kompilator nie wie, czy klasa pochodna może używać jej lub inny kod klienta może utworzyć wystąpienia klasy i odwoływać się do zmiennej</span><span class="sxs-lookup"><span data-stu-id="cdc78-117">**Note** If the variable `i` is declared as `protected or public`, no error will be generated because the compiler cannot know whether a derived class might use it or some other client code might instantiate the class and reference the variable</span></span>
