# <a name="compiler-warning-level-3-cs0414"></a><span data-ttu-id="bc8d4-102">Ostrzeżenie kompilatora (poziom 3) CS0414</span><span class="sxs-lookup"><span data-stu-id="bc8d4-102">Compiler Warning (level 3) CS0414</span></span>

 <span data-ttu-id="bc8d4-104">To ostrzeżenie może wystąpić w kilku scenariuszach, w których kompilator może sprawdzić, czy zmienna nie jest nigdy przywoływana:</span><span class="sxs-lookup"><span data-stu-id="bc8d4-104">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>  
  
- <span data-ttu-id="bc8d4-105">Do pola prywatnego jest przypisywana stała wartość, ale nie jest ona nigdy odczytywana.</span><span class="sxs-lookup"><span data-stu-id="bc8d4-105">A private field is assigned a constant value but is never subsequently read.</span></span> <span data-ttu-id="bc8d4-106">Niepotrzebne przypisanie może mieć wpływ na wydajność.</span><span class="sxs-lookup"><span data-stu-id="bc8d4-106">The unnecessary assignment could effect performance.</span></span> <span data-ttu-id="bc8d4-107">Rozważ usunięcie pola.</span><span class="sxs-lookup"><span data-stu-id="bc8d4-107">Consider removing the field.</span></span>  
  
- <span data-ttu-id="bc8d4-108">Do prywatnego lub wewnętrznego pola statycznego przypisywana jest stała wartość tylko w inicjatorze.</span><span class="sxs-lookup"><span data-stu-id="bc8d4-108">A private or internal static field is assigned a constant value only in the initializer.</span></span> <span data-ttu-id="bc8d4-109">Rozważ zmianę pola na const.</span><span class="sxs-lookup"><span data-stu-id="bc8d4-109">Consider changing the field to a const.</span></span>  
  
- <span data-ttu-id="bc8d4-110">Do pola prywatnego lub wewnętrznego są przypisywane wartości stałe i używane tylko w blokach, które są wykluczone przez dyrektywy #ifdef.</span><span class="sxs-lookup"><span data-stu-id="bc8d4-110">A private or internal field is assigned constant values and only used in blocks that are excluded by #ifdef directives.</span></span> <span data-ttu-id="bc8d4-111">Rozważ umieszczenie pola wewnątrz bloku #ifdef.</span><span class="sxs-lookup"><span data-stu-id="bc8d4-111">Consider putting the field inside the #ifdef block.</span></span>  
  
- <span data-ttu-id="bc8d4-112">Do pola prywatnego lub wewnętrznego są przypisywane wartości stałe w wielu lokalizacjach, ale nie są używane w inny sposób.</span><span class="sxs-lookup"><span data-stu-id="bc8d4-112">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="bc8d4-113">Jeśli pole nie jest potrzebne, rozważ jego usunięcie.</span><span class="sxs-lookup"><span data-stu-id="bc8d4-113">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="bc8d4-114">W przeciwnym razie użyj go w odpowiedni sposób.</span><span class="sxs-lookup"><span data-stu-id="bc8d4-114">Otherwise, use it in some appropriate way.</span></span>  
  
 <span data-ttu-id="bc8d4-115">W innych sytuacjach lub w przypadku, gdy sugerowane obejście nie jest akceptowalne, użyj #pragma 0414.</span><span class="sxs-lookup"><span data-stu-id="bc8d4-115">In other situations, or where the suggested workaround is not acceptable, use #pragma 0414.</span></span>  
  
 <span data-ttu-id="bc8d4-116">Poniższy przykład pokazuje, w jaki sposób zostanie wygenerowany CS0414:</span><span class="sxs-lookup"><span data-stu-id="bc8d4-116">The following sample shows one way in which CS0414 will be generated:</span></span>  
  
```csharp  
// CS0414

{  
   private int i = 1;  // CS0414  
  
   public static void Main()  
   { }  
}
```  

 <span data-ttu-id="bc8d4-117">**Uwaga** Jeśli zmienna `i` jest zadeklarowana jako `protected or public`, żaden błąd nie zostanie wygenerowany, ponieważ kompilator nie wie, czy Klasa pochodna może go używać lub inny kod klienta może utworzyć wystąpienie klasy i odwołać się do zmiennej</span><span class="sxs-lookup"><span data-stu-id="bc8d4-117">**Note** If the variable `i` is declared as `protected or public`, no error will be generated because the compiler cannot know whether a derived class might use it or some other client code might instantiate the class and reference the variable</span></span>
