# <a name="compiler-error-cs0075"></a><span data-ttu-id="6b3df-103">Błąd kompilatora CS0075</span><span class="sxs-lookup"><span data-stu-id="6b3df-103">Compiler Error CS0075</span></span>

<span data-ttu-id="6b3df-104">Aby rzutować wartość ujemną, musisz ją ująć w nawiasy.</span><span class="sxs-lookup"><span data-stu-id="6b3df-104">To cast a negative value, you must enclose the value in parentheses</span></span>  
  
 <span data-ttu-id="6b3df-105">W przypadku rzutowania za pomocą słowa kluczowego, które identyfikuje wstępnie zdefiniowany typ, nie trzeba używać nawiasów.</span><span class="sxs-lookup"><span data-stu-id="6b3df-105">If you are casting using a keyword that identifies a predefined type, then you do not need parentheses.</span></span> <span data-ttu-id="6b3df-106">W przeciwnym razie należy umieścić nawiasy, ponieważ (x) – y nie będą traktowane jako wyrażenie rzutowania.</span><span class="sxs-lookup"><span data-stu-id="6b3df-106">Otherwise, you must put the parentheses because (x) –y will not be considered a cast expression.</span></span> <span data-ttu-id="6b3df-107">W specyfikacji języka C# sekcja 7.6.6:</span><span class="sxs-lookup"><span data-stu-id="6b3df-107">From the C# Specification, Section 7.6.6:</span></span>  
  
 <span data-ttu-id="6b3df-108">*Z poziomu reguły uściślania, która następuje, jeśli x i y są identyfikatorami, (x) y, (x) (y) i (x) (-y) są wyrażeniami rzutowania, ale (x)-y, nawet jeśli x identyfikuje typ. Jeśli jednak x jest słowem kluczowym, który identyfikuje wstępnie zdefiniowany typ (na przykład int), wszystkie cztery formy są wyrażeniami rzutowania (ponieważ takie słowo kluczowe nie może być wyrażeniem przez siebie).*</span><span class="sxs-lookup"><span data-stu-id="6b3df-108">*From the disambiguation rule it follows that, if x and y are identifiers, (x)y, (x)(y), and (x)(-y) are cast-expressions, but (x)-y is not, even if x identifies a type. However, if x is a keyword that identifies a predefined type (such as int), then all four forms are cast-expressions (because such a keyword could not possibly be an expression by itself).*</span></span>  
  
 <span data-ttu-id="6b3df-109">Poniższy kod generuje CS0075:</span><span class="sxs-lookup"><span data-stu-id="6b3df-109">The following code generates CS0075:</span></span>  
  
```csharp  
// CS0075  
namespace MyNamespace  
{  
    enum MyEnum { }  
    public class MyClass  
    {  
        public static void Main()  
        {  
            // To fix the error, place the negative  
            // values below in parentheses  
            int i = (System.Int32) - 4; //CS0075  
            MyEnum e = (MyEnum) - 1;    //CS0075  
            System.Console.WriteLine(i); //to avoid warning  
            System.Console.WriteLine(e); //to avoid warning  
        }  
    }  
}  
```
