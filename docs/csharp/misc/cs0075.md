# <a name="compiler-error-cs0075"></a><span data-ttu-id="3d92b-102">CS0075 błąd kompilatora</span><span class="sxs-lookup"><span data-stu-id="3d92b-102">Compiler Error CS0075</span></span>
<span data-ttu-id="3d92b-103">Aby rzutować wartość ujemną, wartość należy ująć w nawiasy</span><span class="sxs-lookup"><span data-stu-id="3d92b-103">To cast a negative value, you must enclose the value in parentheses</span></span>  
  
 <span data-ttu-id="3d92b-104">Jeśli są rzutowaniem, za pomocą słowa kluczowego, który identyfikuje wstępnie zdefiniowany typ, nie potrzebujesz nawiasów.</span><span class="sxs-lookup"><span data-stu-id="3d92b-104">If you are casting using a keyword that identifies a predefined type, then you do not need parentheses.</span></span> <span data-ttu-id="3d92b-105">W przeciwnym razie należy umieść nawiasy, ponieważ (x) – y nie zostanie uwzględniony wyrażenia rzutowania.</span><span class="sxs-lookup"><span data-stu-id="3d92b-105">Otherwise, you must put the parentheses because (x) –y will not be considered a cast expression.</span></span> <span data-ttu-id="3d92b-106">W specyfikacji języka C# sekcji 7.6.6:</span><span class="sxs-lookup"><span data-stu-id="3d92b-106">From the C# Specification, Section 7.6.6:</span></span>  
  
 <span data-ttu-id="3d92b-107">*Z reguły uściślania następuje, jeśli x i y są identyfikatory (x), y, (x)(y), i (x)(-y) są — wyrażenia cast, ale (x)-y nie jest nawet wtedy, gdy x identyfikuje typ. Jednak jeśli x jest słowem kluczowym, który identyfikuje wstępnie zdefiniowany typ (np. int), następnie wszystkie cztery formularze są wyrażenia cast (ponieważ słów kluczowych nie można prawdopodobnie wyrażenia przez siebie).*</span><span class="sxs-lookup"><span data-stu-id="3d92b-107">*From the disambiguation rule it follows that, if x and y are identifiers, (x)y, (x)(y), and (x)(-y) are cast-expressions, but (x)-y is not, even if x identifies a type. However, if x is a keyword that identifies a predefined type (such as int), then all four forms are cast-expressions (because such a keyword could not possibly be an expression by itself).*</span></span>  
  
 <span data-ttu-id="3d92b-108">Poniższy kod generuje CS0075:</span><span class="sxs-lookup"><span data-stu-id="3d92b-108">The following code generates CS0075:</span></span>  
  
```csharp  
// CS0075  
namespace MyNamespace  
{  
    enum MyEnum { }  
    public class MyClass  
    {  
        public static void Main()  
        {  
            // To fix the error, place the negative  
            // values below in parentheses  
            int i = (System.Int32) - 4; //CS0075  
            MyEnum e = (MyEnum) - 1;    //CS0075  
            System.Console.WriteLine(i); //to avoid warning  
            System.Console.WriteLine(e); //to avoid warning  
        }  
    }  
}  
```
