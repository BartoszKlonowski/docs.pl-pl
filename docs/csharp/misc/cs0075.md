# <a name="compiler-error-cs0075"></a><span data-ttu-id="07954-102">CS0075 błąd kompilatora</span><span class="sxs-lookup"><span data-stu-id="07954-102">Compiler Error CS0075</span></span>
<span data-ttu-id="07954-103">Wartość do rzutowania wartości ujemnej, należy ująć w nawiasach</span><span class="sxs-lookup"><span data-stu-id="07954-103">To cast a negative value, you must enclose the value in parentheses</span></span>  
  
 <span data-ttu-id="07954-104">Jeśli są rzutowanie, za pomocą słowa kluczowego, który identyfikuje wstępnie zdefiniowanego typu, nie wymagają nawiasów.</span><span class="sxs-lookup"><span data-stu-id="07954-104">If you are casting using a keyword that identifies a predefined type, then you do not need parentheses.</span></span> <span data-ttu-id="07954-105">W przeciwnym razie możesz umieścić nawiasy ponieważ (x) — y nie zostanie uwzględniony wyrażeniem rzutowania.</span><span class="sxs-lookup"><span data-stu-id="07954-105">Otherwise, you must put the parentheses because (x) –y will not be considered a cast expression.</span></span> <span data-ttu-id="07954-106">W specyfikacji języka C# sekcji 7.6.6:</span><span class="sxs-lookup"><span data-stu-id="07954-106">From the C# Specification, Section 7.6.6:</span></span>  
  
 <span data-ttu-id="07954-107">*Z reguły Uściślanie następuje, jeśli x i y to identyfikatory, (y, x) (x)(y), i (x)(-y) są rzutowania wyrażenia, ale (x)-y nie jest nawet wtedy, gdy typem identyfikuje x. Jednak jeśli x to — słowo kluczowe, który identyfikuje wstępnie zdefiniowany typ (na przykład int), następnie wszystkie cztery formularze są rzutowania wyrażenia (ponieważ takie słowo kluczowe nie można prawdopodobnie wyrażenia przez samego siebie).*</span><span class="sxs-lookup"><span data-stu-id="07954-107">*From the disambiguation rule it follows that, if x and y are identifiers, (x)y, (x)(y), and (x)(-y) are cast-expressions, but (x)-y is not, even if x identifies a type. However, if x is a keyword that identifies a predefined type (such as int), then all four forms are cast-expressions (because such a keyword could not possibly be an expression by itself).*</span></span>  
  
 <span data-ttu-id="07954-108">Poniższy kod generuje CS0075:</span><span class="sxs-lookup"><span data-stu-id="07954-108">The following code generates CS0075:</span></span>  
  
```csharp  
// CS0075  
namespace MyNamespace  
{  
    enum MyEnum { }  
    public class MyClass  
    {  
        public static void Main()  
        {  
            // To fix the error, place the negative  
            // values below in parentheses  
            int i = (System.Int32) - 4; //CS0075  
            MyEnum e = (MyEnum) - 1;    //CS0075  
            System.Console.WriteLine(i); //to avoid warning  
            System.Console.WriteLine(e); //to avoid warning  
        }  
    }  
}  
```
