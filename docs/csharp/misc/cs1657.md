# <a name="compiler-error-cs1657"></a>CS1657 błąd kompilatora
Nie można przekazać "parametru" ref lub out argument ponieważ "Przyczyna ''  
  
 Ten błąd występuje, gdy zmienna jest przekazywana jako [ref](../../csharp/language-reference/keywords/ref.md) lub [się](../../csharp/language-reference/keywords/out-parameter-modifier.md) argument w kontekście, w którym zmienna jest tylko do odczytu. Konteksty tylko do odczytu obejmują [foreach](../../csharp/language-reference/keywords/foreach-in.md) zmienne iteracji [przy użyciu](../../csharp/language-reference/keywords/using-statement.md) zmienne, i `fixed` zmiennych. Aby rozwiązać ten problem, nie należy wywoływać funkcje, które przyjmują `foreach`, `using` lub `fixed` zmiennej jako `ref` lub `out` parametru w `using` bloki, `foreach` instrukcji i `fixed` instrukcje.  
  
## <a name="example"></a>Przykład  
 Poniższy przykład spowoduje wygenerowanie CS1657:  
  
```csharp  
// CS1657.cs  
using System;  
class C : IDisposable  
{  
    public int i;  
    public void Dispose() {}  
}  
  
class CMain  
{  
    static void f(ref C c)  
    {  
    }  
    static void Main()  
    {  
        using (C c = new C())  
        {  
            f(ref c);  // CS1657  
        }  
    }  
}  
```  
  
## <a name="example"></a>Przykład  
 Poniższy kod ilustruje ten sam problem w `fixed` instrukcji:  
  
```csharp  
// CS1657b.cs  
// compile with: /unsafe  
unsafe class C  
{  
    static void F(ref int* p)  
    {  
    }  
  
    static void Main()  
    {  
        int[] a = new int[5];  
        fixed(int* p = a) F(ref p); // CS1657  
    }  
}  
```
