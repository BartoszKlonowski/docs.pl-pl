<span data-ttu-id="bf2c0-103">"class" definiuje operator == lub! =, lecz nie przesłania metody Object.GetHashCode()</span><span class="sxs-lookup"><span data-stu-id="bf2c0-103">'class' defines operator == or operator != but does not override Object.GetHashCode()</span></span>  
  
 <span data-ttu-id="bf2c0-104">Kompilator wykryto równości zdefiniowane przez użytkownika lub operator nierówności, ale nie zastąpienie **GetHashCode** funkcji.</span><span class="sxs-lookup"><span data-stu-id="bf2c0-104">The compiler detected the user-defined equality or inequality operator, but no override for the **GetHashCode** function.</span></span> <span data-ttu-id="bf2c0-105">Operator równości i nierówności zdefiniowane przez użytkownika wskazuje, czy też chcesz zastąpić **GetHashCode** funkcji.</span><span class="sxs-lookup"><span data-stu-id="bf2c0-105">A user-defined equality or inequality operator implies that you also want to override the **GetHashCode** function.</span></span>  
  
 <span data-ttu-id="bf2c0-106">Poniższy przykład generuje CS0661:</span><span class="sxs-lookup"><span data-stu-id="bf2c0-106">The following sample generates CS0661:</span></span>  
  
```csharp  
// CS0661.cs  
// compile with: /W:3  
class Test   // CS0661  
{  
   public static bool operator == (object o, Test t)  
   {  
      return true;  
   }  
  
   public static bool operator != (object o, Test t)  
   {  
      return true;  
   }  
  
   public override bool Equals(object o)  
   {  
      return true;  
   }  
  
   // uncomment the GetHashCode function to resolve  
   // public override int GetHashCode()  
   // {  
   //    return 0;  
   // }  
  
   public static void Main()  
   {  
   }  
}  
```
