# <a name="compiler-error-cs0695"></a><span data-ttu-id="0e16a-102">CS0695 błąd kompilatora</span><span class="sxs-lookup"><span data-stu-id="0e16a-102">Compiler Error CS0695</span></span>
<span data-ttu-id="0e16a-103">"typu ogólnego" nie może implementować zarówno "Ogólne" i "ogólny interfejsem", ponieważ mogą one łączyć przy niektórych podstawieniach parametrów typu.</span><span class="sxs-lookup"><span data-stu-id="0e16a-103">'generic type' cannot implement both 'generic interface' and 'generic interface' because they may unify for some type parameter substitutions</span></span>  
  
 <span data-ttu-id="0e16a-104">Ten błąd występuje, gdy więcej niż jeden parametryzacja tego samego interfejsu ogólnego implementuje klasy ogólnej, jak i podstawienia parametru typu, co może uniemożliwić dwa interfejsy identyczne.</span><span class="sxs-lookup"><span data-stu-id="0e16a-104">This error occurs when a generic class implements more than one parameterization of the same generic interface, and there exists a type parameter substitution which would make the two interfaces identical.</span></span> <span data-ttu-id="0e16a-105">Aby uniknąć tego błędu, wdrożenie tylko jednego z interfejsów, lub zmień parametry typu, aby uniknąć konfliktu.</span><span class="sxs-lookup"><span data-stu-id="0e16a-105">To avoid this error, implement only one of the interfaces, or change the type parameters to avoid the conflict.</span></span>  
  
 <span data-ttu-id="0e16a-106">Poniższy przykład generuje CS0695:</span><span class="sxs-lookup"><span data-stu-id="0e16a-106">The following sample generates CS0695:</span></span>  
  
```csharp  
// CS0695.cs  
// compile with: /target:library  
  
interface I<T>  
{  
}  
  
class G<T1, T2> : I<T1>, I<T2>  // CS0695  
{  
}  
```
