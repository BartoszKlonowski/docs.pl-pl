# <a name="compiler-error-cs0695"></a><span data-ttu-id="fc5e7-102">CS0695 błąd kompilatora</span><span class="sxs-lookup"><span data-stu-id="fc5e7-102">Compiler Error CS0695</span></span>
<span data-ttu-id="fc5e7-103">"typ ogólny" nie może implementować "ogólny interfejs" i "Interfejs ogólny", ponieważ mogą one łączyć przy niektórych podstawieniach parametrów typu</span><span class="sxs-lookup"><span data-stu-id="fc5e7-103">'generic type' cannot implement both 'generic interface' and 'generic interface' because they may unify for some type parameter substitutions</span></span>  
  
 <span data-ttu-id="fc5e7-104">Ten błąd występuje, gdy klasy ogólnej implementuje więcej niż jeden parametryzacji ten sam interfejs ogólny, jak i Podstawienie parametru typu, co spowoduje, że dwa interfejsy identyczne.</span><span class="sxs-lookup"><span data-stu-id="fc5e7-104">This error occurs when a generic class implements more than one parameterization of the same generic interface, and there exists a type parameter substitution which would make the two interfaces identical.</span></span> <span data-ttu-id="fc5e7-105">Aby uniknąć tego błędu, implementować tylko jeden z interfejsów, lub zmień parametry typu, aby uniknąć konfliktu.</span><span class="sxs-lookup"><span data-stu-id="fc5e7-105">To avoid this error, implement only one of the interfaces, or change the type parameters to avoid the conflict.</span></span>  
  
 <span data-ttu-id="fc5e7-106">Poniższy przykład spowoduje wygenerowanie CS0695:</span><span class="sxs-lookup"><span data-stu-id="fc5e7-106">The following sample generates CS0695:</span></span>  
  
```csharp  
// CS0695.cs  
// compile with: /target:library  
  
interface I<T>  
{  
}  
  
class G<T1, T2> : I<T1>, I<T2>  // CS0695  
{  
}  
```
