# <a name="compiler-warning-level-1-cs0688"></a>Kompilatora (poziom 1) ostrzeżenie CS0688
"metoda1" zawiera żądanie łącza, ale zastępuje lub implementuje metodę "2", która nie ma zapotrzebowania na łącza. Być może istnieje luka w zabezpieczeniach.  
  
 W klasie pochodnej metoda zapotrzebowania na łącza można łatwo obejść, przez wywołanie metody klasy bazowej. Aby zamknąć lukę w zabezpieczeniach, metody klasy bazowej musi również użyć zapotrzebowania na łącza. Aby uzyskać więcej informacji, zobacz [popytu programu vs. Żądanie LinkDemand](https://msdn.microsoft.com/library/1ab877f2-70f4-4e0d-8116-943999dfe8f5).  
  
## <a name="example"></a>Przykład  
 Poniższy przykład spowoduje wygenerowanie CS0688. Aby rozwiązać ostrzeżenie bez modyfikowania klasy bazowej, usuń atrybut zabezpieczeń z metoda przesłaniania. To nie rozwiąże problem z zabezpieczeniami.  
  
```csharp  
// CS0688.cs  
// compile with: /W:1  
using System;  
using System.Security.Permissions;  
  
class Base   
{  
    //Uncomment the following line to close the security hole  
    //[FileIOPermission(SecurityAction.LinkDemand, All=@"C:\\")]  
    public virtual void DoScaryFileStuff()  
    {  
    }  
}  
  
class Derived: Base  
{  
    [FileIOPermission(SecurityAction.LinkDemand, All=@"C:\\")] // CS0688  
    public override void DoScaryFileStuff()  
    {  
    }  
    static void Main()  
    {  
    }  
}  
```
