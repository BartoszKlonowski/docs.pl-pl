# <a name="compiler-warning-level-3-cs0659"></a>Kompilatora (poziom 3) ostrzeżenie CS0659
"class" przesłania metodę Object.Equals (object o), ale nie przesłania metody Object.GetHashCode()  
  
 Kompilator wykrył nadpisanie **jest równa** funkcji, ale nie zostać zastąpiona w przypadku **GetHashCode**. Zastępowanie **jest równa** wskazuje, czy też chcesz zastąpić **GetHashCode**.  
  
 Aby uzyskać więcej informacji, zobacz artykuł  
  
-   <xref:System.Collections.Hashtable>.  
  
-   [Operatory równości](../../standard/design-guidelines/equality-operators.md)  
  
-   [NIB: Implementacja metody Equals](https://msdn.microsoft.com/library/30f28aaf-8b9e-46cd-a746-58a12473af2c)  
  
-   <xref:System.Object.GetHashCode%2A>  
  
 Poniższy przykład spowoduje wygenerowanie CS0659:  
  
```csharp  
// CS0659.cs  
// compile with: /W:3 /target:library  
class Test     
{  
   public override bool Equals(object o) { return true; }   // CS0659  
}  
  
// OK  
class Test2  
{  
   public override bool Equals(object o) { return true; }  
   public override int GetHashCode() { return 0; }  
}  
```
