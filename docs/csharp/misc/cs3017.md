# <a name="compiler-warning-level-1-cs3017"></a>Kompilator CS3017 ostrzegawcze (poziom 1)
Nie można określić atrybutu CLSCompliant w module, który różni się od atrybutu CLSCompliant w zestawie  
  
 To ostrzeżenie występuje, gdy powodujący konflikt atrybutu CLSCompliant zestawu z atrybutem CLSCompliant modułu. Zestaw, który jest zgodny z CLS nie może zawierać modułów, które nie są zgodne ze specyfikacją CLS. Aby usunąć to ostrzeżenie, upewnij się, zestaw i moduł CLSCompliant atrybuty są zarówno wartość true lub false zarówno, albo usuń jeden z atrybutów. Aby uzyskać więcej informacji dotyczących zgodności ze specyfikacją CLS, zobacz [pisanie kodu zgodne ze specyfikacją CLS](http://msdn.microsoft.com/en-us/4c705105-69a2-4e5e-b24e-0633bc32c7f3) i [niezależność od języka i elementy niezależne od języka](https://msdn.microsoft.com/library/12a7a7h3).  
  
## <a name="example"></a>Przykład  
 Poniższy przykład generuje CS3017:  
  
```  
// CS3017.cs  
// compile with: /target:module  
  
using System;  
  
[module: CLSCompliant(true)]  
[assembly: CLSCompliant(false)]  // CS3017  
// Try this line instead:  
// [assembly: CLSCompliant(true)]  
class C  
{  
    static void Main() {}  
}  
```
