# <a name="compiler-warning-level-3-cs0219"></a><span data-ttu-id="124ad-102">Ostrzeżenie o kompilatorze (poziom 3) CS0219</span><span class="sxs-lookup"><span data-stu-id="124ad-102">Compiler Warning (level 3) CS0219</span></span>
<span data-ttu-id="124ad-103">Zmienna "zmienna" jest przypisana, ale jej wartość nigdy nie jest używana</span><span class="sxs-lookup"><span data-stu-id="124ad-103">The variable 'variable' is assigned but its value is never used</span></span>

 <span data-ttu-id="124ad-104">Kompilator generuje ostrzeżenie poziomu trzeciego, gdy deklarujesz i przypisujesz zmienną, ale jej nie używasz.</span><span class="sxs-lookup"><span data-stu-id="124ad-104">The compiler issues a level-three warning, when you declare and assign a variable, but do not use it.</span></span>

 > [!NOTE]
 > <span data-ttu-id="124ad-105">Kompilator generuje to ostrzeżenie tylko wtedy, gdy wartość zmiennej jest stałą czasu kompilacji.</span><span class="sxs-lookup"><span data-stu-id="124ad-105">The compiler generates this warning only when the variable value is a compile-time constant.</span></span> <span data-ttu-id="124ad-106">Przypisywanie wyrażenia niestałego lub wyniku metody do zmiennej lokalnej ułatwia obserwowanie tych wyrażeń w debugerze.</span><span class="sxs-lookup"><span data-stu-id="124ad-106">Assigning a non-constant expression or method result to a local variable makes it easier to observe those expressions in the debugger.</span></span> <span data-ttu-id="124ad-107">To również sprawia, że wynik osiągalny, zapobieganie wyrzucania elementów bezużytecznych, podczas gdy ta zmienna jest osiągalna.</span><span class="sxs-lookup"><span data-stu-id="124ad-107">It also makes the result reachable, preventing garbage collection while that variable is reachable.</span></span>

 <span data-ttu-id="124ad-108">W poniższym przykładzie przedstawiono przypadki, kiedy i kiedy nie jest generowane ostrzeżenie:</span><span class="sxs-lookup"><span data-stu-id="124ad-108">The following sample shows the cases when and when not the warning is generated:</span></span>

```csharp
// CS0219.cs
// compile with: /W:3
public class MyClass
{
   public static void Main()
   {
      var interpolated = "Interpolated";
      var a = 0; // CS0219
      int b = GetZero(); // Doesn't generate a warning.
      var c = "Regular string"; // CS0219
      var d = $"Constant interpolated string"; // Doesn't generate a warning.
      var e = $"{interpolated} string"; // Doesn't generate a warning.
   }

   private static int GetZero()
   {
       return 0;
   }
}  
```
