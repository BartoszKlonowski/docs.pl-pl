# <a name="compiler-error-cs1649"></a>Błąd kompilatora CS1649
Elementy członkowskie pola tylko do odczytu "identyfikator" nie mogą być przekazywane ref lub out (z wyjątkiem konstruktora)  
  
 Ten błąd występuje, jeśli przekazać zmienną do `readonly` funkcji, `ref` która `out` jest członkiem pola jako lub argumentu. Ponieważ `ref` `out` i parametry mogą być modyfikowane przez funkcję, nie jest to dozwolone. Aby rozwiązać ten błąd, usuń `readonly` słowo kluczowe w polu lub `readonly` nie należy przekazywać elementów członkowskich pola do funkcji. Na przykład można spróbować utworzyć zmienną tymczasową, która może `ref` być modyfikowana i przekazywanie tymczasowe jako argument, jak pokazano w poniższym przykładzie.  
  
## <a name="example"></a>Przykład  
 Poniższy przykład generuje CS1649:  
  
```csharp  
// CS1649.cs  
public struct Inner  
    {  
        public int i;  
    }  
  
class Outer  
{  
    public readonly Inner inner = new Inner();  
}  
  
class D  
{  
    static void f(ref int iref)  
    {  
    }  
  
    static void Main()  
    {  
        Outer outer = new Outer();
        f(ref outer.inner.i);  // CS1649  
        // Try this code instead:  
        // int tmp = outer.inner.i;  
        // f(ref tmp);  
    }  
}  
```
