# <a name="compiler-error-cs0017"></a><span data-ttu-id="22910-102">CS0017 błąd kompilatora</span><span class="sxs-lookup"><span data-stu-id="22910-102">Compiler Error CS0017</span></span>
<span data-ttu-id="22910-103">Program "Nazwa pliku wyjściowego" ma więcej niż jeden punkt wejścia zdefiniowane.</span><span class="sxs-lookup"><span data-stu-id="22910-103">Program 'output file name' has more than one entry point defined.</span></span> <span data-ttu-id="22910-104">Skompiluj z opcją/Main, aby określić typ zawierający punkt wejścia.</span><span class="sxs-lookup"><span data-stu-id="22910-104">Compile with /main to specify the type that contains the entry point.</span></span>  
  
 <span data-ttu-id="22910-105">Program może mieć tylko jedną [Main](../../csharp/programming-guide/main-and-command-args/index.md) metody.</span><span class="sxs-lookup"><span data-stu-id="22910-105">A program can only have one [Main](../../csharp/programming-guide/main-and-command-args/index.md) method.</span></span>  
  
 <span data-ttu-id="22910-106">Aby rozwiązać ten problem, możesz usunąć wszystkie metody Main w kodzie, z wyjątkiem jednego, możesz też [/main](../../csharp/language-reference/compiler-options/main-compiler-option.md) opcję kompilatora, aby określić które metody Main, którego chcesz użyć.</span><span class="sxs-lookup"><span data-stu-id="22910-106">To resolve this error, you can either delete all Main methods in your code, except one, or you can use the [/main](../../csharp/language-reference/compiler-options/main-compiler-option.md) compiler option to specify which Main method you want to use.</span></span>  
  
 <span data-ttu-id="22910-107">Poniższy przykład spowoduje wygenerowanie CS0017:</span><span class="sxs-lookup"><span data-stu-id="22910-107">The following sample generates CS0017:</span></span>  
  
```csharp  
// CS0017.cs  
// compile with: /target:exe  
public class clx  
{  
   static public void Main()  
   {  
   }  
}  
  
public class cly  
{  
   public static void Main()   // CS0017, delete one Main or use /main  
   {  
   }  
}  
```
