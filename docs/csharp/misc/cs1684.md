<span data-ttu-id="e24b0-103">Odwołanie do typu "nazwa typu" oświadczenia jest zdefiniowane w elemencie "namespace", ale nie można go znaleźć</span><span class="sxs-lookup"><span data-stu-id="e24b0-103">Reference to type 'Type Name' claims it is defined in 'Namespace', but it could not be found</span></span>  
  
 <span data-ttu-id="e24b0-104">Ten błąd może być spowodowany przez odwołanie wewnątrz przestrzeni nazw odwołujące się do typu, który istnieje w drugim obszarze nazw, ale ten typ nie istnieje.</span><span class="sxs-lookup"><span data-stu-id="e24b0-104">This error can be caused by a reference inside one namespace referring to a type that it says exists inside a second namespace, but the type does not exist.</span></span> <span data-ttu-id="e24b0-105">Na przykład mydll.dll mówi, że typ `A` istnieje wewnątrz yourdll.dll, ale ten typ nie istnieje w yourdll.dll.</span><span class="sxs-lookup"><span data-stu-id="e24b0-105">For example, mydll.dll says that type `A` exists inside yourdll.dll, but no such type exists inside yourdll.dll.</span></span> <span data-ttu-id="e24b0-106">Jedną z możliwych przyczyn tego błędu jest to, że używana wersja yourdll.dll jest zbyt stara i `A` nie została jeszcze zdefiniowana.</span><span class="sxs-lookup"><span data-stu-id="e24b0-106">One possible cause of this error is that the version of yourdll.dll you are using is too old and `A` has not yet been defined.</span></span>  
  
 <span data-ttu-id="e24b0-107">Poniższy przykład generuje CS1684.</span><span class="sxs-lookup"><span data-stu-id="e24b0-107">The following sample generates CS1684.</span></span>  
  
## <a name="example"></a><span data-ttu-id="e24b0-108">Przykład</span><span class="sxs-lookup"><span data-stu-id="e24b0-108">Example</span></span>  
  
```csharp  
// CS1684_a.cs  
// compile with: /target:library /keyfile:CS1684.key  
public class A {  
   public void Test() {}  
}  
  
public class C2 {}  
```  
  
## <a name="example"></a><span data-ttu-id="e24b0-109">Przykład</span><span class="sxs-lookup"><span data-stu-id="e24b0-109">Example</span></span>  
  
```csharp  
// CS1684_b.cs  
// compile with: /target:library /r:cs1684_a.dll  
// post-build command: del /f CS1684_a.dll  
using System;  
public class Ref
{  
   public static A GetA() { return new A(); }  
   public static C2 GetC() { return new C2(); }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="e24b0-110">Przykład</span><span class="sxs-lookup"><span data-stu-id="e24b0-110">Example</span></span>  

 <span data-ttu-id="e24b0-111">Teraz kompilujemy pierwszy zestaw, pozostawiając definicję klasy C2, która nie ma być zdefiniowana w ponownej kompilacji.</span><span class="sxs-lookup"><span data-stu-id="e24b0-111">We now rebuild the first assembly, leaving out the definition of the class C2 not to be defined in the recompilation.</span></span>

```csharp  
// compile with: /target:library /keyfile:CS1684.key /out:CS1684_a.dll

```  
  
## <a name="example"></a><span data-ttu-id="e24b0-112">Przykład</span><span class="sxs-lookup"><span data-stu-id="e24b0-112">Example</span></span>  

 <span data-ttu-id="e24b0-113">Ten moduł odwołuje się do drugiego modułu za pomocą identyfikatora `Ref` .</span><span class="sxs-lookup"><span data-stu-id="e24b0-113">This module references the second module by means of the identifier `Ref`.</span></span> <span data-ttu-id="e24b0-114">Drugi moduł zawiera odwołanie do klasy `C2` , która już nie istnieje ze względu na kompilację w poprzednim kroku, a tym samym komunikat o błędzie CS1684 jest zwracany z kompilacji tego modułu.</span><span class="sxs-lookup"><span data-stu-id="e24b0-114">But the second module contains a reference to the class `C2`, which no longer exists because of the compilation in the previous step, and therefore the CS1684 error message is returned from the compilation of this module.</span></span>
