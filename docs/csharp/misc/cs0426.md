 <span data-ttu-id="5210c-104">Ten błąd występuje, gdy odwołanie typu zagnieżdżone w obrębie innego typu, ale nie ma takiej zagnieżdżone się, że istnieje typu.</span><span class="sxs-lookup"><span data-stu-id="5210c-104">This error occurs when you reference a type nested within another type, but no such nested type exists.</span></span> <span data-ttu-id="5210c-105">Może to wystąpić, jeśli wpisujesz nazwę typu zagnieżdżonego.</span><span class="sxs-lookup"><span data-stu-id="5210c-105">This can occur if you mistype the name of the nested type.</span></span> <span data-ttu-id="5210c-106">Sprawdź pisownię nazwy używanych i sprawdź, czy typ otaczający ma oczekiwanego elementu członkowskiego.</span><span class="sxs-lookup"><span data-stu-id="5210c-106">Check the spelling of the names used, and verify that the enclosing type has the expected member.</span></span>  
  
 <span data-ttu-id="5210c-107">Poniższy przykład generuje CS0426, ponieważ klasa C nie ma zagnieżdżonych typu A:</span><span class="sxs-lookup"><span data-stu-id="5210c-107">The following sample generates CS0426 because class C has no nested type A:</span></span>  
  
```csharp  
// CS0426.cs  
  
class C  
{  
    // No nested types are declared.     
}  
  
class D  
{  
   public static void Main()  
   {  
       C c = new C();  
       // Attempt to reference a nested type A:  
       C.A a; // CS0426 because no such type C.A  
   }  
}  
```
