# <a name="compiler-error-cs0426"></a><span data-ttu-id="309a5-102">Błąd kompilatora CS0426</span><span class="sxs-lookup"><span data-stu-id="309a5-102">Compiler Error CS0426</span></span>
<span data-ttu-id="309a5-103">Nazwa typu "identyfikator" nie istnieje w typie "typ"</span><span class="sxs-lookup"><span data-stu-id="309a5-103">The type name 'identifier' does not exist in the type 'type'</span></span>  
  
 <span data-ttu-id="309a5-104">Ten błąd występuje, gdy odwołujesz się do typu zagnieżdżonego w innym typie, ale nie istnieje taki typ zagnieżdżony.</span><span class="sxs-lookup"><span data-stu-id="309a5-104">This error occurs when you reference a type nested within another type, but no such nested type exists.</span></span> <span data-ttu-id="309a5-105">Taka sytuacja może wystąpić w przypadku błędnego wpisania nazwy typu zagnieżdżonego.</span><span class="sxs-lookup"><span data-stu-id="309a5-105">This can occur if you mistype the name of the nested type.</span></span> <span data-ttu-id="309a5-106">Sprawdź pisownię używanych nazw i sprawdź, czy typ otaczający ma oczekiwany element członkowski.</span><span class="sxs-lookup"><span data-stu-id="309a5-106">Check the spelling of the names used, and verify that the enclosing type has the expected member.</span></span>  
  
 <span data-ttu-id="309a5-107">Poniższy przykład generuje CS0426, ponieważ klasa C nie ma zagnieżdżonego typu A:</span><span class="sxs-lookup"><span data-stu-id="309a5-107">The following sample generates CS0426 because class C has no nested type A:</span></span>  
  
```csharp  
// CS0426.cs  
  
class C  
{  
    // No nested types are declared.
}  
  
class D  
{  
   public static void Main()  
   {  
       C c = new C();  
       // Attempt to reference a nested type A:  
       C.A a; // CS0426 because no such type C.A  
   }  
}  
```
