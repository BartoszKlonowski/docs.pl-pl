# <a name="compiler-error-cs0434"></a><span data-ttu-id="1ad9f-102">CS0434 błąd kompilatora</span><span class="sxs-lookup"><span data-stu-id="1ad9f-102">Compiler Error CS0434</span></span>

 <span data-ttu-id="1ad9f-104">Ten błąd występuje, gdy typu zaimportowany i zaimportowane zagnieżdżone przestrzenie nazw mają taką samą nazwę FQDN.</span><span class="sxs-lookup"><span data-stu-id="1ad9f-104">This error occurs when an imported type and an imported nested namespace have the same fully qualified name.</span></span> <span data-ttu-id="1ad9f-105">Gdy odwołuje się do tej nazwy, kompilator nie może rozróżnić.</span><span class="sxs-lookup"><span data-stu-id="1ad9f-105">When that name is referenced, the compiler is unable to distinguish between the two.</span></span> <span data-ttu-id="1ad9f-106">Jeśli zmienisz kod źródłowy zaimportowany, ten błąd można rozwiązać przez zmianę nazwy typu lub przestrzeni nazw, tak aby oba są unikatowe w obrębie zestawu.</span><span class="sxs-lookup"><span data-stu-id="1ad9f-106">If you can change the imported source code, you can resolve the error by changing the name of either the type or the namespace so that both are unique within the assembly.</span></span>  
  
 <span data-ttu-id="1ad9f-107">Poniższy kod generuje błąd CS0434.</span><span class="sxs-lookup"><span data-stu-id="1ad9f-107">The following code generates error CS0434.</span></span>  
  
## <a name="example"></a><span data-ttu-id="1ad9f-108">Przykład</span><span class="sxs-lookup"><span data-stu-id="1ad9f-108">Example</span></span>  
 <span data-ttu-id="1ad9f-109">Ten kod tworzy pierwszy kopię używanego typu o identycznej nazwie FQDN.</span><span class="sxs-lookup"><span data-stu-id="1ad9f-109">This code creates the first copy of the type with the identical fully qualified name.</span></span>  
  
```csharp  
// CS0434_1.cs  
// compile with: /t:library  
namespace TypeBindConflicts   
{  
    namespace NsImpAggPubImp   
    {  
        public class X { }  
    }  
}  
```
