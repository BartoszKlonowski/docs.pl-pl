# <a name="compiler-error-cs1112"></a>Błąd kompilatora CS1112
Nie używaj "System.Runtime.CompilerServices.ExtensionAttribute". Zamiast tego użyj słowa kluczowego "this".  
  
 Ten błąd jest generowany, gdy <xref:System.Runtime.CompilerServices.ExtensionAttribute> jest używana na niestatycznych klas, który zawiera metody rozszerzenia. Jeśli ten atrybut jest używany w klasie statycznej inny błąd, takich jak CS0708: "Nie można zadeklarować składowych wystąpienia w klasie statycznej" mogą wystąpić.  
  
 W C#metody rozszerzenia muszą być zdefiniowane w klasie statycznej i pierwszy parametr metody jest modyfikowana za pomocą `this` — słowo kluczowe. Nie używaj atrybutu na wszystkich w kodzie źródłowym. Aby uzyskać więcej informacji, zobacz [metody rozszerzenia](../../../csharp/programming-guide/classes-and-structs/extension-methods.md).  
  
## <a name="to-correct-this-error"></a>Aby poprawić ten błąd  
  
1. Usuń atrybut i zastosować `this` modyfikatora pierwszy parametr metody.  
  
## <a name="example"></a>Przykład  
 Poniższy przykład generuje CS1112:  
  
```csharp  
// cs1112.cs  
[System.Runtime.CompilerServices.ExtensionAttribute] // CS1112  
public class Extensions  
{  
    public bool A(bool b) { return b; }  
}  
  
class A { }   
```
