# <a name="compiler-error-cs1704"></a>Błąd kompilatora CS1704
Zaimportowano już zestaw o tej samej prostej nazwie "Nazwa zestawu". Spróbuj usunąć jedno z odwołań lub podpisać je, aby włączyć obok siebie.  
  
 Ten błąd wskazuje, że dwa odwołania mają tę samą tożsamość zestawu, ponieważ zestawy, o których mowa brak silnych nazw, nie zostały podpisane, a zatem kompilator nie ma możliwości rozróżnienia między nimi w metadanych. W związku z tym czas wykonywania ignoruje właściwości nazwy zestawu wersji i kultury. Użytkownik powinien usunąć nadmiarowe odwołanie, zmienić nazwę jednego z odwołań lub podać silną nazwę dla nich.  
  
## <a name="example"></a>Przykład  
 Ten przykład tworzy zestaw i zapisuje go w katalogu głównym.  
  
```csharp  
// CS1704_a.cs  
// compile with: /target:library /out:c:\\cs1704.dll  
public class A {}  
```  
  
## <a name="example"></a>Przykład  
 Ten przykład tworzy zestaw o takiej samej nazwie jak poprzedni przykład, ale zapisuje go w innej lokalizacji.  
  
```csharp  
// CS1704_b.cs  
// compile with: /target:library /out:cs1704.dll  
public class A {}  
```  
  
## <a name="example"></a>Przykład  
 W tym przykładzie próbuje odwoływać się do obu zestawów. Poniższy przykład generuje CS1704.  
  
```csharp  
// CS1704_c.cs  
// compile with: /target:library /r:A2=cs1704.dll /r:A1=c:\\cs1704.dll  
// CS1704 expected  
extern alias A1;  
extern alias A2;  
```
