  
## <a name="example"></a><span data-ttu-id="3f989-110">Przykład</span><span class="sxs-lookup"><span data-stu-id="3f989-110">Example</span></span>  
 <span data-ttu-id="3f989-111">Poniższy przykład generuje CS0686:</span><span class="sxs-lookup"><span data-stu-id="3f989-111">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="3f989-112">Przykład</span><span class="sxs-lookup"><span data-stu-id="3f989-112">Example</span></span>  
 <span data-ttu-id="3f989-113">Ten błąd może również wystąpić podczas deklarowania zdarzeń.</span><span class="sxs-lookup"><span data-stu-id="3f989-113">This error can also occur when declaring events.</span></span>  <span data-ttu-id="3f989-114">Konstrukcja zdarzenia automatycznie generuje `add_event` i `remove_event` metody, które mogą kolidować z metodami o tej samej nazwie w interfejsie, jak w poniższym przykładzie:</span><span class="sxs-lookup"><span data-stu-id="3f989-114">The event construct automatically generates the `add_event` and `remove_event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }  
}  
  
// Correct (using explicit interface implementation):  
class D : I  
{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```
