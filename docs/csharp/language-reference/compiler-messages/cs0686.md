 <span data-ttu-id="5aabe-106">Sugerowane: ten błąd może wystąpić podczas implementowania interfejsu, który zawiera nazwy metod, które powodują konflikt z automatycznie generowanymi metodami skojarzonymi z właściwością lub zdarzeniem.</span><span class="sxs-lookup"><span data-stu-id="5aabe-106">Suggested: This error can occur when implementing an interface that contains method names which conflict with the auto-generated methods associated with a property or event.</span></span> <span data-ttu-id="5aabe-107">Metody get/set dla właściwości są generowane jako get_property i set_property, a metody dodawania/usuwania zdarzeń są generowane jako add_event i remove_event.</span><span class="sxs-lookup"><span data-stu-id="5aabe-107">The get/set methods for properties are generated as get_property and set_property, and the add/remove methods for events are generated as add_event and remove_event.</span></span> <span data-ttu-id="5aabe-108">Jeśli interfejs zawiera jedną z tych metod, występuje konflikt.</span><span class="sxs-lookup"><span data-stu-id="5aabe-108">If an interface contains either of these methods, a conflict occurs.</span></span> <span data-ttu-id="5aabe-109">Aby uniknąć tego błędu, zaimplementuj metody przy użyciu jawnej implementacji interfejsu.</span><span class="sxs-lookup"><span data-stu-id="5aabe-109">To avoid this error, implement the methods using an explicit interface implementation.</span></span> <span data-ttu-id="5aabe-110">W tym celu należy określić funkcję jako:</span><span class="sxs-lookup"><span data-stu-id="5aabe-110">To do this, specify the function as:</span></span>  
  
```csharp  
Interface.get_property() { /* */ }  
Interface.set_property() { /* */ }  
```  
  
## <a name="example"></a><span data-ttu-id="5aabe-111">Przykład</span><span class="sxs-lookup"><span data-stu-id="5aabe-111">Example</span></span>  

 <span data-ttu-id="5aabe-112">Poniższy przykład generuje CS0686:</span><span class="sxs-lookup"><span data-stu-id="5aabe-112">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="5aabe-113">Przykład</span><span class="sxs-lookup"><span data-stu-id="5aabe-113">Example</span></span>  

 <span data-ttu-id="5aabe-114">Ten błąd może również wystąpić podczas deklarowania zdarzeń.</span><span class="sxs-lookup"><span data-stu-id="5aabe-114">This error can also occur when declaring events.</span></span>  <span data-ttu-id="5aabe-115">Konstrukcja zdarzenia automatycznie generuje `add_event` `remove_event` metody i, które mogą powodować konflikt z metodami o tej samej nazwie w interfejsie, jak w poniższym przykładzie:</span><span class="sxs-lookup"><span data-stu-id="5aabe-115">The event construct automatically generates the `add_event` and `remove_event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }  
}  
  
// Correct (using explicit interface implementation):  
class D : I  
{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```
