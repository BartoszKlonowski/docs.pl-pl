 <span data-ttu-id="4bc19-106">Jeśli ten błąd wystąpi w przypadku typu klasy, będzie to wynikać z tego, że klasa nie przeciąża operatora.</span><span class="sxs-lookup"><span data-stu-id="4bc19-106">If you encounter this error with a class type, it is because the class does not overload the operator.</span></span> <span data-ttu-id="4bc19-107">Aby uzyskać więcej informacji, zobacz [operatory z możliwością przeciążenia](../../../csharp/programming-guide/statements-expressions-operators/overloadable-operators.md).</span><span class="sxs-lookup"><span data-stu-id="4bc19-107">For more information, see [Overloadable Operators](../../../csharp/programming-guide/statements-expressions-operators/overloadable-operators.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="4bc19-108">Przykład</span><span class="sxs-lookup"><span data-stu-id="4bc19-108">Example</span></span>  
 <span data-ttu-id="4bc19-109">W poniższym przykładzie błąd CS0019 jest generowany w dwóch miejscach, ponieważ w języku C# nie można przekonwertować wartości [bool](../../../csharp/language-reference/keywords/bool.md) na [int](../../../csharp/language-reference/keywords/int.md). Błąd CS0019 występuje również, gdy operator odejmowania jest stosowany dla ciągu znakowego.</span><span class="sxs-lookup"><span data-stu-id="4bc19-109">In the following example, CS0019 is generated in two places because [bool](../../../csharp/language-reference/keywords/bool.md) in C# is not convertible to [int](../../../csharp/language-reference/keywords/int.md). CS0019 also is generated when the subtraction operator is applied to a string.</span></span> <span data-ttu-id="4bc19-110">Operatora dodawania (+) można używać z tekstowym argumentami operacji, ponieważ ten operator jest przeciążony przez klasę `String` w celu połączenia ciągu znaków.
</span><span class="sxs-lookup"><span data-stu-id="4bc19-110">The addition operator (+) can be used with string operands because that operator is overloaded by the `String` class to perform string concatenation.</span></span>  
  
```csharp  
static void Main()  
{  
    bool result = true;  
    if (result > 0) //CS0019  
    {  
        // Do something.  
    }  
  
    int i = 1;  
    // You cannot compare an integer and a boolean value.  
    if (i == true) //CS0019  
    {  
        //Do something...  
    }  
  
    // The following use of == causes no error. It is the comparison of  
    // an integer and a boolean value that causes the error in the   
    // previous if statement.  
    if (result == true)  
    {  
        //Do something...  
    }  
  
    string s = "Just try to subtract me.";  
    float f = 100 - s; // CS0019  
}  
```  
  
## <a name="example"></a><span data-ttu-id="4bc19-111">Przykład</span><span class="sxs-lookup"><span data-stu-id="4bc19-111">Example</span></span>  
 <span data-ttu-id="4bc19-112">W poniższym przykładzie należy określić logikę warunkową poza klasą <xref:System.Diagnostics.ConditionalAttribute>. </span><span class="sxs-lookup"><span data-stu-id="4bc19-112">In the following example, conditional logic must be specified outside the <xref:System.Diagnostics.ConditionalAttribute>.</span></span> <span data-ttu-id="4bc19-113">Do klasy <xref:System.Diagnostics.ConditionalAttribute> można przekazać tylko jeden wstępnie zdefiniowany symbol.</span><span class="sxs-lookup"><span data-stu-id="4bc19-113">You can pass only one predefined symbol to the <xref:System.Diagnostics.ConditionalAttribute>.</span></span>  
  
 <span data-ttu-id="4bc19-114">Poniższy przykład generuje CS0019.</span><span class="sxs-lookup"><span data-stu-id="4bc19-114">The following sample generates CS0019.</span></span>  
  
```csharp  
// CS0019_a.cs

public class MyClass  
{  
   [ConditionalAttribute("DEBUG" || "TRACE")]   // CS0019

   [ConditionalAttribute("DEBUG"), ConditionalAttribute("TRACE")]  
   public void TestMethod2() {}  
}  
```
