# <a name="compiler-error-cs0019"></a>Błąd kompilatora CS0019

Nie można zastosować operatora "operator" do argumentów operacji typu "type" i "type"  
  
 Operator binarny jest stosowany do typów danych, które go nie obsługują. Na przykład nie można użyć [ &#124; &#124; ](../../../csharp/language-reference/operators/boolean-logical-operators.md#conditional-logical-or-operator-) operatora na ciągi znaków, nie można użyć [ + ](../../../csharp/language-reference/operators/addition-operator.md), [ - ](../../../csharp/language-reference/operators/subtraction-operator.md), [ \< ](../../../csharp/language-reference/operators/comparison-operators.md#less-than-operator-), lub [ > ](../../../csharp/language-reference/operators/comparison-operators.md#greater-than-operator-) operatory na [bool](../../../csharp/language-reference/keywords/bool.md) zmienne, a nie można użyć [ == ](../../../csharp/language-reference/operators/equality-operators.md#equality-operator-) operator `struct`typu, chyba że typ jawnie przeciążenia operatora.  
  
 Możesz doprowadzić do przeciążenia operatora ułatwiają obsługuje niektóre typy argumentów operacji. Aby uzyskać więcej informacji, zobacz [przeciążania operatora](../../../csharp/language-reference/operators/operator-overloading.md).
  
## <a name="example"></a>Przykład

 W poniższym przykładzie błąd CS0019 jest generowany w dwóch miejscach, ponieważ w języku C# nie można przekonwertować wartości [bool](../../../csharp/language-reference/keywords/bool.md) na [int](../../../csharp/language-reference/builtin-types/integral-numeric-types.md). Błąd CS0019 występuje również, gdy operator odejmowania jest stosowany dla ciągu znakowego. Operatora dodawania (+) można używać z tekstowym argumentami operacji, ponieważ ten operator jest przeciążony przez klasę `String` w celu połączenia ciągu znaków.  

```csharp
static void Main()  
{  
    bool result = true;  
    if (result > 0) //CS0019  
    {  
        // Do something.  
    }  
  
    int i = 1;  
    // You cannot compare an integer and a boolean value.  
    if (i == true) //CS0019  
    {  
        //Do something...  
    }  
  
    // The following use of == causes no error. It is the comparison of
    // an integer and a boolean value that causes the error in the
    // previous if statement.  
    if (result == true)  
    {  
        //Do something...  
    }  
  
    string s = "Just try to subtract me.";  
    float f = 100 - s; // CS0019  
}  
```

## <a name="example"></a>Przykład

 W poniższym przykładzie należy określić logikę warunkową poza klasą <xref:System.Diagnostics.ConditionalAttribute>. Do klasy <xref:System.Diagnostics.ConditionalAttribute> można przekazać tylko jeden wstępnie zdefiniowany symbol.  
  
 Poniższy przykład spowoduje wygenerowanie CS0019.  

```csharp
// CS0019_a.cs  
// compile with: /target:library  
using System.Diagnostics;  
public class MyClass  
{  
   [ConditionalAttribute("DEBUG" || "TRACE")]   // CS0019  
   public void TestMethod() {}  
  
   // OK  
   [ConditionalAttribute("DEBUG"), ConditionalAttribute("TRACE")]  
   public void TestMethod2() {}  
}  
```
