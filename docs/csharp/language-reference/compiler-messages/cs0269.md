# <a name="compiler-error-cs0269"></a><span data-ttu-id="33340-103">Błąd kompilatora CS0269</span><span class="sxs-lookup"><span data-stu-id="33340-103">Compiler Error CS0269</span></span>

<span data-ttu-id="33340-104">Użycie nieprzypisanego parametru out "parameter"</span><span class="sxs-lookup"><span data-stu-id="33340-104">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="33340-105">Kompilator nie może sprawdzić, czy parametr out ma przypisaną wartość przed użyciem; jego wartość może być niezdefiniowana po przypisaniu.</span><span class="sxs-lookup"><span data-stu-id="33340-105">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="33340-106">Należy pamiętać o przypisaniu wartości do `out` parametrów w wywołanej metodzie przed uzyskaniem dostępu do wartości.</span><span class="sxs-lookup"><span data-stu-id="33340-106">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="33340-107">Jeśli musisz użyć wartości przekazaną przez zmienną, użyj `ref` zamiast niego parametru.</span><span class="sxs-lookup"><span data-stu-id="33340-107">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="33340-108">Aby uzyskać więcej informacji, zobacz [przekazywanie parametrów](../../programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="33340-108">For more information, see [Passing Parameters](../../programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="33340-109">Przykład</span><span class="sxs-lookup"><span data-stu-id="33340-109">Example</span></span>  

 <span data-ttu-id="33340-110">Poniższy przykład generuje CS0269:</span><span class="sxs-lookup"><span data-stu-id="33340-110">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="33340-111">Przykład</span><span class="sxs-lookup"><span data-stu-id="33340-111">Example</span></span>  

 <span data-ttu-id="33340-112">Może to również wystąpić, Jeśli inicjalizacja zmiennej występuje w bloku try, którego kompilator nie może zweryfikować, zostanie wykonany pomyślnie:</span><span class="sxs-lookup"><span data-stu-id="33340-112">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
