# <a name="compiler-error-cs0269"></a><span data-ttu-id="a136c-102">Błąd kompilatora CS0269</span><span class="sxs-lookup"><span data-stu-id="a136c-102">Compiler Error CS0269</span></span>
<span data-ttu-id="a136c-103">Użycie nieprzypisanego parametru "parametr"</span><span class="sxs-lookup"><span data-stu-id="a136c-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="a136c-104">Kompilator nie mógł sprawdzić, czy out parametr został przypisany wartość przed jego użyciem; jego wartość może być niezdefiniowana po przypisaniu.</span><span class="sxs-lookup"><span data-stu-id="a136c-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="a136c-105">Pamiętaj, aby przypisać wartość `out` do parametrów w wywoływanej metody przed uzyskując dostęp do wartości.</span><span class="sxs-lookup"><span data-stu-id="a136c-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="a136c-106">Jeśli trzeba użyć wartości zmiennej przekazywane, należy `ref` użyć parametru zamiast.</span><span class="sxs-lookup"><span data-stu-id="a136c-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="a136c-107">Aby uzyskać więcej informacji, zobacz [Przekazywanie parametrów](../../programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="a136c-107">For more information, see [Passing Parameters](../../programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="a136c-108">Przykład</span><span class="sxs-lookup"><span data-stu-id="a136c-108">Example</span></span>  
 <span data-ttu-id="a136c-109">Poniższy przykład generuje CS0269:</span><span class="sxs-lookup"><span data-stu-id="a136c-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="a136c-110">Przykład</span><span class="sxs-lookup"><span data-stu-id="a136c-110">Example</span></span>  
 <span data-ttu-id="a136c-111">Może to również wystąpić, jeśli inicjowanie zmiennej występuje w try bloku, który kompilator nie może sprawdzić zostanie pomyślnie wykonana:</span><span class="sxs-lookup"><span data-stu-id="a136c-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
