# <a name="compiler-error-cs0269"></a><span data-ttu-id="97a2d-102">Błąd kompilatora CS0269</span><span class="sxs-lookup"><span data-stu-id="97a2d-102">Compiler Error CS0269</span></span>
<span data-ttu-id="97a2d-103">Użycie nieprzypisanego parametru "parametru" out</span><span class="sxs-lookup"><span data-stu-id="97a2d-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="97a2d-104">Kompilator nie może zweryfikować, czy parametr out została przypisana wartość, zanim został on użyty; wartość może być Niezdefiniowany, po przypisaniu.</span><span class="sxs-lookup"><span data-stu-id="97a2d-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="97a2d-105">Pamiętaj przypisać wartość do `out` parametrów w metodzie wywoływanej przed uzyskaniem dostępu do wartości.</span><span class="sxs-lookup"><span data-stu-id="97a2d-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="97a2d-106">Jeśli chcesz użyć wartości zmiennej przekazanej, użyj `ref` parametru zamiast tego.</span><span class="sxs-lookup"><span data-stu-id="97a2d-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="97a2d-107">Aby uzyskać więcej informacji, zobacz [przekazanie parametrów](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="97a2d-107">For more information, see [Passing Parameters](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="97a2d-108">Przykład</span><span class="sxs-lookup"><span data-stu-id="97a2d-108">Example</span></span>  
 <span data-ttu-id="97a2d-109">Poniższy przykład spowoduje wygenerowanie CS0269:</span><span class="sxs-lookup"><span data-stu-id="97a2d-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="97a2d-110">Przykład</span><span class="sxs-lookup"><span data-stu-id="97a2d-110">Example</span></span>  
 <span data-ttu-id="97a2d-111">To może również wystąpić, jeśli wystąpi inicjowania zmiennej w bloku try, czyli kompilator nie można zweryfikować wykona pomyślnie:</span><span class="sxs-lookup"><span data-stu-id="97a2d-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
