# <a name="compiler-error-cs1612"></a><span data-ttu-id="7ce44-102">Błąd kompilatora CS1612</span><span class="sxs-lookup"><span data-stu-id="7ce44-102">Compiler Error CS1612</span></span>
<span data-ttu-id="7ce44-103">Nie można zmodyfikować zwracanej wartości "wyrażenia", ponieważ nie jest zmienną</span><span class="sxs-lookup"><span data-stu-id="7ce44-103">Cannot modify the return value of 'expression' because it is not a variable</span></span>  
  
 <span data-ttu-id="7ce44-104">Podjęto próbę zmodyfikowania typu wartości, który jest tworzony w wyniku wyrażenia pośredniego, ale nie jest przechowywany w zmiennej.</span><span class="sxs-lookup"><span data-stu-id="7ce44-104">An attempt was made to modify a value type that is produced as the result of an intermediate expression but is not stored in a variable.</span></span> <span data-ttu-id="7ce44-105">Ten błąd może wystąpić podczas próby bezpośredniego modyfikowania struktury w kolekcji ogólnej, jak pokazano w poniższym przykładzie:</span><span class="sxs-lookup"><span data-stu-id="7ce44-105">This error can occur when you attempt to directly modify a struct in a generic collection, as shown in the following example:</span></span>  
  
```csharp  
List<myStruct> list = {…};  
list[0].Name = "MyStruct42"; //CS1612  
```  
  
 <span data-ttu-id="7ce44-106">Aby zmodyfikować strukturę, najpierw przypisz ją do zmiennej lokalnej, zmodyfikuj zmienną, a następnie przypisz zmienną z powrotem do elementu w kolekcji.</span><span class="sxs-lookup"><span data-stu-id="7ce44-106">To modify the struct, first assign it to a local variable, modify the variable, then assign the variable back to the item in the collection.</span></span>  
  
```csharp  
List<myStruct> list = {…};  
MyStruct ms = list[0];  
ms.Name = "MyStruct42";  
list[0] = ms;  
```  
  
 <span data-ttu-id="7ce44-107">Ten błąd występuje, ponieważ typy wartości są kopiowane na przypisanie.</span><span class="sxs-lookup"><span data-stu-id="7ce44-107">This error occurs because value types are copied on assignment.</span></span> <span data-ttu-id="7ce44-108">Podczas pobierania typu wartości z właściwości lub indeksatora, otrzymujesz kopię obiektu, a nie odwołanie do samego obiektu.</span><span class="sxs-lookup"><span data-stu-id="7ce44-108">When you retrieve a value type from a property or indexer, you are getting a copy of the object, not a reference to the object itself.</span></span> <span data-ttu-id="7ce44-109">Zwracana kopia nie jest przechowywana przez właściwość lub indeksatora, ponieważ są one faktycznie metody, a nie lokalizacji magazynu (zmienne).</span><span class="sxs-lookup"><span data-stu-id="7ce44-109">The copy that is returned is not stored by the property or indexer because they are actually methods, not storage locations (variables).</span></span> <span data-ttu-id="7ce44-110">Należy zapisać kopię w zmiennej, która deklarujesz, zanim będzie można go zmodyfikować.</span><span class="sxs-lookup"><span data-stu-id="7ce44-110">You must store the copy into a variable that you declare before you can modify it.</span></span>  
  
 <span data-ttu-id="7ce44-111">Błąd nie występuje w przypadku typów odwołań, ponieważ właściwość lub indeksator w takim przypadku zwraca odwołanie do istniejącego obiektu, który jest lokalizacją magazynu.</span><span class="sxs-lookup"><span data-stu-id="7ce44-111">The error does not occur with reference types because a property or indexer in that case returns a reference to an existing object, which is a storage location.</span></span>  
  
 <span data-ttu-id="7ce44-112">Jeśli definiujesz klasę lub strukturę, można rozwiązać ten błąd, modyfikując deklarację właściwości, aby zapewnić dostęp do elementów członkowskich struktury.</span><span class="sxs-lookup"><span data-stu-id="7ce44-112">If you are defining the class or struct, you can resolve this error by modifying your property declaration to provide access to the members of a struct.</span></span> <span data-ttu-id="7ce44-113">Jeśli piszesz kod klienta, można rozwiązać błąd, tworząc własne wystąpienie struktury, modyfikując jego pola, a następnie przypisując całą strukturę z powrotem do właściwości.</span><span class="sxs-lookup"><span data-stu-id="7ce44-113">If you are writing client code, you can resolve the error by creating your own instance of the struct, modifying its fields, and then assigning the entire struct back to the property.</span></span> <span data-ttu-id="7ce44-114">Jako trzecia alternatywa, można zmienić strukturę do klasy.</span><span class="sxs-lookup"><span data-stu-id="7ce44-114">As a third alternative, you can change your struct to a class.</span></span>  
  
## <a name="example"></a><span data-ttu-id="7ce44-115">Przykład</span><span class="sxs-lookup"><span data-stu-id="7ce44-115">Example</span></span>  
 <span data-ttu-id="7ce44-116">CS1612 występuje również podczas próby uzyskania dostępu do elementu członkowskiego struktury za pośrednictwem właściwości na otaczającej klasy, która zwraca całą strukturę, jak pokazano w poniższym przykładzie:</span><span class="sxs-lookup"><span data-stu-id="7ce44-116">CS1612 also occurs when you attempt to access the member of a struct through a property on an enclosing class that is returning the entire struct, as shown in the following example:</span></span>  
  
```csharp  
// CS1612.cs  
using System;  
  
public struct MyStruct  
{  
    public int Width;  
}  
  
public class ListView  
{  
    MyStruct ms;  
    public MyStruct Size  
    {  
        get { return ms; }  
        set { ms = value; }  
    }  
}  
  
public class MyClass  
{  
    public MyClass()  
    {  
        ListView lvi;  
        lvi = new ListView();  
        lvi.Size.Width = 5; // CS1612  
  
        // You can use the following lines instead.  
        // MyStruct ms;  
        // ms.Width = 5;  
        // lvi.Size = ms;
    }  
  
    public static void Main()
    {  
        MyClass mc = new MyClass();  
        // Keep the console open in debug mode.  
        Console.WriteLine("Press any key to exit.");  
        Console.ReadKey();
    }  
}  
```
