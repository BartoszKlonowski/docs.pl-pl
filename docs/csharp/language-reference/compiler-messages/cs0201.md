# <a name="compiler-error-cs0201"></a>Błąd kompilatora CS0201

Tylko przypisywanie, wywołanie, przyrost, zmniejszanie i nowe wyrażenia obiektów mogą być używane jako instrukcja  
  
 Kompilator generuje błąd, gdy napotka nieprawidłową instrukcję. Nieprawidłowa instrukcja to dowolny wiersz lub seria wierszy kończących[=](../operators/assignment-operator.md)się średnikiem, który [++](../operators/arithmetic-operators.md#increment-operator-) nie reprezentuje przypisania (), wywołania metody [()](../operators/member-access-operators.md#invocation-operator-), [nowego](../operators/new-operator.md) [--](../operators/arithmetic-operators.md#decrement-operator---) lub operacji. Aby uzyskać więcej informacji, zobacz [Instrukcje, wyrażenia i operatory](../../programming-guide/statements-expressions-operators/index.md).  
  
## <a name="example"></a>Przykład

 Poniższy przykład generuje CS0201, ponieważ 2 * 3 jest wyrażeniem, a nie instrukcją. Aby skompilować kod, spróbuj przypisać wartość wyrażenia do zmiennej.  

```csharp
// CS0201.cs  
public class MainClass  
{  
   public static void Main()  
   {  
      2 * 3;   // CS0201  
      // Try the following line instead.  
      //   int i = 2 * 3;  
   }  
}  
```

## <a name="example"></a>Przykład

 Poniższy przykład generuje CS0201, ponieważ sprawdzane przez siebie nie jest instrukcją, mimo że jest sparametryzowana przez operację przyrostu.  

```csharp
// CS0201_b.cs  
// compile with: /target:library  
public class MyList<T>
{  
   public void Add(T x)  
   {  
      int i = 0;  
      if ( (object)x == null)  
      {  
         checked(i++);   // CS0201  
  
         // OK  
         checked {  
            i++;
         }  
      }  
   }  
}  
```
