# <a name="compiler-error-cs0201"></a>Błąd kompilatora CS0201

Tylko przypisania, wywołanie, inkrementacja, dekrementacja i wyrażenia nowego obiektu może służyć jako instrukcji  
  
 Kompilator generuje błąd, po napotkaniu nieprawidłowa instrukcja. Nieprawidłowa instrukcja jest dowolnym wierszu lub szeregu wiersze kończące się średnikiem, który nie reprezentuje przypisania ([=](../operators/assignment-operator.md)), wywołanie metody [()](../operators/member-access-operators.md#invocation-operator-), [nowe](../keywords/new.md), [ -- ](../operators/arithmetic-operators.md#decrement-operator---) lub [ ++ ](../operators/arithmetic-operators.md#increment-operator-) operacji. Aby uzyskać więcej informacji, zobacz [instrukcje, wyrażenia i operatory](../../programming-guide/statements-expressions-operators/index.md).  
  
## <a name="example"></a>Przykład

 Poniższy przykład generuje CS0201, ponieważ wyrażenie nie instrukcję 2 * 3. Uczynienie go skompilować, spróbuj przypisać wartość wyrażenia do zmiennej.  

```csharp
// CS0201.cs  
public class MainClass  
{  
   public static void Main()  
   {  
      2 * 3;   // CS0201  
      // Try the following line instead.  
      //   int i = 2 * 3;  
   }  
}  
```

## <a name="example"></a>Przykład

 Poniższy przykład generuje CS0201, ponieważ sprawdzane przez sam nie jest instrukcja, nawet jeśli jest nadać operacja zwiększenia wartości.  

```csharp
// CS0201_b.cs  
// compile with: /target:library  
public class MyList<T>
{  
   public void Add(T x)  
   {  
      int i = 0;  
      if ( (object)x == null)  
      {  
         checked(i++);   // CS0201  
  
         // OK  
         checked {  
            i++;
         }  
      }  
   }  
}  
```
