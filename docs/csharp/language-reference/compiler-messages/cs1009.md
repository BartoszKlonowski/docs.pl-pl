# <a name="compiler-error-cs1009"></a><span data-ttu-id="def8a-103">Błąd kompilatora CS1009</span><span class="sxs-lookup"><span data-stu-id="def8a-103">Compiler Error CS1009</span></span>

<span data-ttu-id="def8a-104">Nierozpoznana sekwencja ucieczki</span><span class="sxs-lookup"><span data-stu-id="def8a-104">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="def8a-105">Nieoczekiwany znak następuje po ukośniku odwrotnym ( \\ ) w [ciągu](../builtin-types/reference-types.md#the-string-type).</span><span class="sxs-lookup"><span data-stu-id="def8a-105">An unexpected character follows a backslash (\\) in a [string](../builtin-types/reference-types.md#the-string-type).</span></span> <span data-ttu-id="def8a-106">Kompilator oczekuje jednego z prawidłowych znaków ucieczki.</span><span class="sxs-lookup"><span data-stu-id="def8a-106">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="def8a-107">Aby uzyskać więcej informacji, zobacz [znak ucieczki](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span><span class="sxs-lookup"><span data-stu-id="def8a-107">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="def8a-108">Poniższy przykład generuje CS1009.</span><span class="sxs-lookup"><span data-stu-id="def8a-108">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="def8a-109">Typową przyczyną tego błędu jest użycie znaku ukośnika odwrotnego w nazwie pliku, jak pokazano w poniższym przykładzie.</span><span class="sxs-lookup"><span data-stu-id="def8a-109">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="def8a-110">Aby rozwiązać ten problem, należy użyć " \\ \\ " lub @-quoted literału ciągu, jak pokazano w poniższym przykładzie.</span><span class="sxs-lookup"><span data-stu-id="def8a-110">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
