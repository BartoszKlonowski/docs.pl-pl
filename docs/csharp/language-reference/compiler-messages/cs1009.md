# <a name="compiler-error-cs1009"></a><span data-ttu-id="c8f64-102">Błąd kompilatora CS1009</span><span class="sxs-lookup"><span data-stu-id="c8f64-102">Compiler Error CS1009</span></span>
<span data-ttu-id="c8f64-103">Nierozpoznana sekwencja specjalna</span><span class="sxs-lookup"><span data-stu-id="c8f64-103">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="c8f64-104">Nieoczekiwany znak następuje ukośnik odwrotny (\\) w [ciąg](../../../csharp/language-reference/keywords/string.md).</span><span class="sxs-lookup"><span data-stu-id="c8f64-104">An unexpected character follows a backslash (\\) in a [string](../../../csharp/language-reference/keywords/string.md).</span></span> <span data-ttu-id="c8f64-105">Kompilator oczekuje jeden ze znaków ucieczki prawidłowe.</span><span class="sxs-lookup"><span data-stu-id="c8f64-105">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="c8f64-106">Aby uzyskać więcej informacji, zobacz [znaki specjalne](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span><span class="sxs-lookup"><span data-stu-id="c8f64-106">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="c8f64-107">Poniższy przykład generuje CS1009.</span><span class="sxs-lookup"><span data-stu-id="c8f64-107">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";     
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="c8f64-108">Typową przyczyną tego błędu jest przy użyciu ukośnika w nazwie pliku, jak w poniższym przykładzie pokazano.</span><span class="sxs-lookup"><span data-stu-id="c8f64-108">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="c8f64-109">Aby rozwiązać ten problem, należy użyć "\\\\" lub @-quoted literału ciągu, jak przedstawiono na poniższym przykładzie.</span><span class="sxs-lookup"><span data-stu-id="c8f64-109">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";     
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
