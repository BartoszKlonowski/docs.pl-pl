# <a name="compiler-error-cs1019"></a>Błąd kompilatora CS1019
Oczekiwano operatora jednoargumentowego z możliwością przeciążenia  
  
 Coś co wygląda na przeciążony operator jednoargumentowy zostało zadeklarowane, ale operator nie istnieje lub jest w niewłaściwej lokalizacji w podpisie.  
  
 *Operator jednoargumentowy* jest operatorem, który działa na jeden argument. Na przykład `++` jest operatorem jednoargumentowym. Niektóre operatory jednoargumentowe można przeciążać przy użyciu słowa kluczowego - `operator`, i określenie jednego parametru typu na którym działa operator. Na przykład, jeśli chcesz przeciążyć operator `++` dla klasy `Temperature` zdefiniowanej przez użytkownika, aby mogła używać `Temperature++`, można to zrobić w ten sposób:  
  
```csharp  
public static  Temperature operator ++ (Temperature temp)  
{  
    temp.Degrees++;  
    return temp;  
}  
```  
  
 Gdy ten błąd zostanie wyświetlony po zadeklarowaniu czego co, wygląda jak przeciążony operator jednoargumentowy, oznacza to że brakuje samego operatora lub jest w niewłaściwej lokalizacji w podpisie. Jeśli usuniesz `++` z podpisu w poprzednim przykładzie, spowoduje to wygenerowanie CS1019.  
  
 Poniższy kod generuje CS1019:  
  
```csharp  
// CS1019.cs  
public class ii  
{  
   int i  
   {  
      get  
      {  
         return 0;  
      }  
   }  
}  
  
public class a  
{  
    public int i;  
// Generuje CS1019: "ii" nie jest operatorem jednoargumentowym.  
   public static a operator ii(a aa)     
  
   // Użyj tej linii kodu:  
   //public static a operator ++(a aa)  
   {  
      aa.i++;  
      return aa;   
   }  
  
   public static void Main()  
   {  
   }  
}  
```
