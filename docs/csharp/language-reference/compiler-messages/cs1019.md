# <a name="compiler-error-cs1019"></a><span data-ttu-id="8e4b3-102">Błąd kompilatora CS1019</span><span class="sxs-lookup"><span data-stu-id="8e4b3-102">Compiler Error CS1019</span></span>
<span data-ttu-id="8e4b3-103">Oczekuje się, że operator unary z przeładem</span><span class="sxs-lookup"><span data-stu-id="8e4b3-103">Overloadable unary operator expected</span></span>  
  
 <span data-ttu-id="8e4b3-104">Zadeklarowano coś, co wygląda jak przeciążony operator bezrektowy, ale brakuje operatora lub znajduje się w niewłaściwej lokalizacji w podpisie.</span><span class="sxs-lookup"><span data-stu-id="8e4b3-104">Something that looks like an overloaded unary operator has been declared, but the operator is missing or is in the wrong location in the signature.</span></span>  
  
 <span data-ttu-id="8e4b3-105">*Operator jednoosobowy* jest operatorem, który działa na jednym operand.</span><span class="sxs-lookup"><span data-stu-id="8e4b3-105">A *unary operator* is an operator that operates on a single operand.</span></span> <span data-ttu-id="8e4b3-106">Na przykład `++` jest operatorem nieujariuszem.</span><span class="sxs-lookup"><span data-stu-id="8e4b3-106">For example, `++` is a unary operator.</span></span> <span data-ttu-id="8e4b3-107">Niektóre operatory jednoargumentowe można `operator` przeciążać za pomocą słowa kluczowego i określając pojedynczy parametr typu, na którym działa operator.</span><span class="sxs-lookup"><span data-stu-id="8e4b3-107">You can overload some unary operators by using the `operator` keyword and specifying a single parameter of the type that the operator operates on.</span></span> <span data-ttu-id="8e4b3-108">Na przykład, jeśli chcesz przeciążać operatora `++` dla `Temperature` klasy zdefiniowanej `Temperature++`przez użytkownika, aby można było pisać, można zdefiniować go w ten sposób:</span><span class="sxs-lookup"><span data-stu-id="8e4b3-108">For example, if you want to overload the operator `++` for a user-defined class `Temperature` so that you can write `Temperature++`, you can define it in this way:</span></span>  
  
```csharp  
public static  Temperature operator ++ (Temperature temp)  
{  
    temp.Degrees++;  
    return temp;  
}  
```  
  
 <span data-ttu-id="8e4b3-109">Po otrzymaniu tego błędu zadeklarowano coś, co wygląda jak przeciążony operator bezrejonowy, z tą różnicą, że brakuje samego operatora lub znajduje się w niewłaściwej lokalizacji w podpisie.</span><span class="sxs-lookup"><span data-stu-id="8e4b3-109">When you receive this error, you have declared something that looks like an overloaded unary operator, except that the operator itself is missing or is in the wrong location in the signature.</span></span> <span data-ttu-id="8e4b3-110">Jeśli usuniesz `++` podpis w poprzednim przykładzie, zostanie wygenerowany CS1019.</span><span class="sxs-lookup"><span data-stu-id="8e4b3-110">If you remove the `++` from the signature in the previous example, you will generate CS1019.</span></span>  
  
 <span data-ttu-id="8e4b3-111">Poniższy kod generuje CS1019:</span><span class="sxs-lookup"><span data-stu-id="8e4b3-111">The following code generates CS1019:</span></span>  
  
```csharp  
// CS1019.cs  
public class ii  
{  
   int i  
   {  
      get  
      {  
         return 0;  
      }  
   }  
}  
  
public class a  
{  
    public int i;  
// Generates CS1019: "ii" is not a unary operator.  
   public static a operator ii(a aa)
  
   // Use the following line instead:  
   //public static a operator ++(a aa)  
   {  
      aa.i++;  
      return aa;
   }  
  
   public static void Main()  
   {  
   }  
}  
```
