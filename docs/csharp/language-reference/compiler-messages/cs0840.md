# <a name="compiler-error-cs0840"></a>Błąd kompilatora CS0840
"Nazwa właściwości" musi zadeklarować treść, ponieważ nie jest ona oznaczona jako abstrakcyjna ani zewnętrzna. Dla automatycznie implementowanych właściwości należy zdefiniować obie metody dostępu; get, set.
 
 O ile regularne właściwości są oznaczone jako `abstract` lub `extern`, lub są elementami członkowskimi `partial` typu, tak w tym przypadku konieczne jest podanie ich treści. Właściwości zaimplementowane automatycznie nie udostępniają treści metod dostępu, ale muszą określać je obie. Aby utworzyć właściwości tylko do odczytu zaimplementowane automatycznie, należy określić metodę dostępu set jako `private`.  
  
## <a name="to-correct-this-error"></a>Aby poprawić ten błąd  
  
1.  Dostarcz brakujące metody dostępu lub użyj jednego ze słów kluczowych [abstract](../../../csharp/language-reference/keywords/abstract.md), [extern](../../../csharp/language-reference/keywords/extern.md), lub [partial (typ)](../../../csharp/language-reference/keywords/partial-type.md).  
  
## <a name="example"></a>Przykład  
 Poniższy przykład generuje CS0840:  
  
```csharp  
// cs0840.cs  
// Skompiluj z /target:library  
using System;  
class Test  
{  
    public int myProp { get; } // CS0840  
  
    // Aby stworzyć właściwośc tylko do odczytu 
    // spróbuj następującego zapisu  
    public int myProp2 { get; private set; }  
  
}  
```
