# <a name="compiler-error-cs0103"></a><span data-ttu-id="351f9-102">Błąd kompilatora CS0103</span><span class="sxs-lookup"><span data-stu-id="351f9-102">Compiler Error CS0103</span></span>

<span data-ttu-id="351f9-103">Nazwa "identyfikator" nie istnieje w bieżącym kontekście</span><span class="sxs-lookup"><span data-stu-id="351f9-103">The name 'identifier' does not exist in the current context</span></span>

 <span data-ttu-id="351f9-104">Podjęto próbę użycia nazwy, która nie istnieje w klasie, [przestrzeni nazw](../keywords/namespace.md)lub zakresie.</span><span class="sxs-lookup"><span data-stu-id="351f9-104">An attempt was made to use a name that does not exist in the class, [namespace](../keywords/namespace.md), or scope.</span></span> <span data-ttu-id="351f9-105">Sprawdź pisownię nazwy i sprawdź za pomocą dyrektyw i odwołań do zestawu, aby upewnić się, że nazwa, której próbujesz użyć, jest dostępna.</span><span class="sxs-lookup"><span data-stu-id="351f9-105">Check the spelling of the name and check your using directives and assembly references to make sure that the name that you are trying to use is available.</span></span>

 <span data-ttu-id="351f9-106">Ten błąd często występuje, jeśli deklarujesz `try` `if` zmienną w pętli lub bloku lub bloku, a następnie próbujesz uzyskać do niej dostęp z otaczającego bloku kodu lub oddzielnego bloku kodu, jak pokazano w poniższym przykładzie:</span><span class="sxs-lookup"><span data-stu-id="351f9-106">This error frequently occurs if you declare a variable in a loop or a `try` or `if` block and then attempt to access it from an enclosing code block or a separate code block, as shown in the following example:</span></span>

```csharp
using System;

class MyClass1
{
    public static void Main()
    {
        try
        {
            // The following declaration is only available inside the try block.
            var conn = new MyClass1();
        }
        catch (Exception e)
        {  
            // The following expression causes error CS0103, because variable
            // conn only exists in the try block.
            if (conn != null)
                Console.WriteLine("{0}", e);
        }
    }
}
```

 <span data-ttu-id="351f9-107">Poniższy przykład rozwiązuje błąd:</span><span class="sxs-lookup"><span data-stu-id="351f9-107">The following example resolves the error:</span></span>

```csharp
using System;

class MyClass2
{
    public static void Main()
    {
        // To resolve the error in the example, the first step is to
        // move the declaration of conn out of the try block. The following
        // declaration is available throughout the Main method.
        MyClass2 conn = null;
        try
        {
            // Inside the try block, use the conn variable that you declared
            // previously.
            conn = new MyClass2();
        }
        catch (Exception e)
        {
            // The following expression no longer causes an error, because
            // the declaration of conn is in scope.
            if (conn != null)
                Console.WriteLine("{0}", e);
        }
    }
}
```
