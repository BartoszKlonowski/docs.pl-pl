# <a name="compiler-error-cs0103"></a>Błąd kompilatora CS0103
Nazwa "identifier" nie istnieje w bieżącym kontekście  
  
 Próbowano użyć nazwy, która nie istnieje w klasie, [przestrzeni nazw](../keywords/namespace.md) lub zakresie. Sprawdź pisownię, a także użycie dyrektyw i odwołań do zestawów, aby upewnić się, że nazwa, której chcesz użyć, jest dostępna.  
  
 Ten błąd występuje często, jeśli zadeklarujesz zmienną w pętli albo bloku `try` lub `if`, a następnie spróbujesz uzyskać do niej dostęp z otaczającego lub osobnego bloku kodu — jak w przykładzie poniżej.  
  
```csharp  
using System;  
  
class MyClass1  
{  
    public static void Main()  
    {  
        try  
        {  
            // The following declaration is only available inside the try block.  
            MyClass1 conn = new MyClass1();  
        }  
        catch (Exception e)  
        {  
            // The following expression causes error CS0103, because variable  
            // conn only exists in the try block.  
            if (conn != null)   
                Console.WriteLine("{0}", e);  
        }  
    }  
}  
```  
  
 Poniższy przykład rozwiązuje błąd.  
  
```csharp  
using System;  
  
class MyClass2  
{  
    public static void Main()  
    {  
        // To resolve the error in the example, the first step is to   
        // move the declaration of conn out of the try block. The following  
        // declaration is available throughout the Main method.  
        MyClass2 conn = null;  
        try  
        {  
            // Inside the try block, use the conn variable that you declared  
            // previously.  
            conn = new MyClass2();  
        }  
        catch (Exception e)  
        {  
            // The following expression no longer causes an error, because  
            // the declaration of conn is in scope.  
            if (conn != null)   
                Console.WriteLine("{0}", e);  
        }  
    }  
}  
```
