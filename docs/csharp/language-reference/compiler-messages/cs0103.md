# <a name="compiler-error-cs0103"></a><span data-ttu-id="8d2fa-103">Błąd kompilatora CS0103</span><span class="sxs-lookup"><span data-stu-id="8d2fa-103">Compiler Error CS0103</span></span>

<span data-ttu-id="8d2fa-104">Nazwa "identifier" nie istnieje w bieżącym kontekście</span><span class="sxs-lookup"><span data-stu-id="8d2fa-104">The name 'identifier' does not exist in the current context</span></span>

 <span data-ttu-id="8d2fa-105">Podjęto próbę użycia nazwy, która nie istnieje w klasie, [przestrzeni nazw](../keywords/namespace.md)lub zakresie.</span><span class="sxs-lookup"><span data-stu-id="8d2fa-105">An attempt was made to use a name that does not exist in the class, [namespace](../keywords/namespace.md), or scope.</span></span> <span data-ttu-id="8d2fa-106">Sprawdź pisownię nazwy i sprawdź swoje dyrektywy using i odwołania do zestawów, aby upewnić się, że nazwa, której próbujesz użyć, jest dostępna.</span><span class="sxs-lookup"><span data-stu-id="8d2fa-106">Check the spelling of the name and check your using directives and assembly references to make sure that the name that you are trying to use is available.</span></span>

 <span data-ttu-id="8d2fa-107">Ten błąd często występuje, Jeśli deklarujesz zmienną w pętli lub `try` bloku lub, `if` a następnie spróbujesz uzyskać do niej dostęp z otaczającego bloku kodu lub z oddzielnym blokiem kodu, jak pokazano w następującym przykładzie:</span><span class="sxs-lookup"><span data-stu-id="8d2fa-107">This error frequently occurs if you declare a variable in a loop or a `try` or `if` block and then attempt to access it from an enclosing code block or a separate code block, as shown in the following example:</span></span>

```csharp
using System;

class MyClass1
{
    public static void Main()
    {
        try
        {
            // The following declaration is only available inside the try block.
            var conn = new MyClass1();
        }
        catch (Exception e)
        {  
            // The following expression causes error CS0103, because variable
            // conn only exists in the try block.
            if (conn != null)
                Console.WriteLine("{0}", e);
        }
    }
}
```

 <span data-ttu-id="8d2fa-108">Poniższy przykład rozwiązuje błąd:</span><span class="sxs-lookup"><span data-stu-id="8d2fa-108">The following example resolves the error:</span></span>

```csharp
using System;

class MyClass2
{
    public static void Main()
    {
        // To resolve the error in the example, the first step is to
        // move the declaration of conn out of the try block. The following
        // declaration is available throughout the Main method.
        MyClass2 conn = null;
        try
        {
            // Inside the try block, use the conn variable that you declared
            // previously.
            conn = new MyClass2();
        }
        catch (Exception e)
        {
            // The following expression no longer causes an error, because
            // the declaration of conn is in scope.
            if (conn != null)
                Console.WriteLine("{0}", e);
        }
    }
}
```
