# <a name="compiler-error-cs1540"></a><span data-ttu-id="f4edd-103">Błąd kompilatora CS1540</span><span class="sxs-lookup"><span data-stu-id="f4edd-103">Compiler Error CS1540</span></span>
<span data-ttu-id="f4edd-104">Nie można uzyskać dostępu do chronionej składowej "member" za pośrednictwem kwalifikatora typu "type1"; kwalifikator musi być typu "type2" (lub pochodzić od niego)</span><span class="sxs-lookup"><span data-stu-id="f4edd-104">Cannot access protected member 'member' via a qualifier of type 'type1'; the qualifier must be of type 'type2' (or derived from it)</span></span>  
  
 <span data-ttu-id="f4edd-105">[Klasa](../keywords/class.md) pochodna nie może uzyskać dostępu do chronionych elementów członkowskich swojej klasy podstawowej za pomocą wystąpienia klasy bazowej.</span><span class="sxs-lookup"><span data-stu-id="f4edd-105">A derived [class](../keywords/class.md) cannot access protected members of its base class through an instance of the base class.</span></span> <span data-ttu-id="f4edd-106">Wystąpienie klasy bazowej zadeklarowanej w klasie pochodnej może w czasie wykonywania być wystąpieniem innego typu, który pochodzi od tego samego elementu bazowego, ale nie jest powiązany z klasą pochodną.</span><span class="sxs-lookup"><span data-stu-id="f4edd-106">An instance of the base class declared in the derived class might, at run time, be an instance of another type that is derived from the same base but is not otherwise related to the derived class.</span></span> <span data-ttu-id="f4edd-107">Ze względu na to, że chronione elementy członkowskie są dostępne tylko dla typów pochodnych, wszelkie próby dostępu do chronionych elementów członkowskich, które mogą być nieprawidłowe w czasie wykonywania, są oznaczane przez kompilator jako nieprawidłowe.</span><span class="sxs-lookup"><span data-stu-id="f4edd-107">Because protected members can be accessed only by derived types, any attempts to access protected members that might not be valid at run time are marked by the compiler as not valid.</span></span>  
  
 <span data-ttu-id="f4edd-108">W `Employee` klasie w poniższym przykładzie `emp2` i `emp3` obie mają typ `Person` w czasie kompilacji, ale `emp2` ma typ `Manager` w czasie wykonywania.</span><span class="sxs-lookup"><span data-stu-id="f4edd-108">In the `Employee` class in the following example, `emp2` and `emp3` both have type `Person` at compile time, but `emp2` has type `Manager` at run time.</span></span> <span data-ttu-id="f4edd-109">Ponieważ `Employee` nie pochodzi od `Manager` , nie można uzyskać dostępu do chronionych elementów członkowskich klasy podstawowej, `Person` za pośrednictwem wystąpienia `Manager` klasy.</span><span class="sxs-lookup"><span data-stu-id="f4edd-109">Because `Employee` is not derived from `Manager`, it cannot access the protected members of the base class, `Person`, through an instance of the `Manager` class.</span></span> <span data-ttu-id="f4edd-110">Kompilator nie może określić, jakiego typu czasu wykonywania `Person` mają być dwa obiekty.</span><span class="sxs-lookup"><span data-stu-id="f4edd-110">The compiler cannot determine what the run-time type of the two `Person` objects will be.</span></span> <span data-ttu-id="f4edd-111">W związku z tym zarówno wywołanie z `emp2` , jak i wywołanie z `emp3` powodu błędu kompilatora CS1540.</span><span class="sxs-lookup"><span data-stu-id="f4edd-111">Therefore, both the call from `emp2` and the call from `emp3` cause compiler error CS1540.</span></span>  
  
```csharp  
using System;  
using System.Text;  
  
namespace CS1540  
{  
    class Program1  
    {  
        static void Main()  
        {  
            Employee.PreparePayroll();  
        }  
    }  
  
    class Person  
    {  
        protected virtual void CalculatePay()
        {  
            Console.WriteLine("CalculatePay in Person class.");  
        }  
    }  
  
    class Manager : Person  
    {  
        protected override void CalculatePay()
        {  
            Console.WriteLine("CalculatePay in Manager class.");
  
        }  
    }  
  
    class Employee : Person  
    {  
        public static void PreparePayroll()  
        {  
            Employee emp1 = new Employee();  
            Person emp2 = new Manager();  
            Person emp3 = new Employee();  
            // The following line calls the method in the Employee base class,  
            // Person.  
            emp1.CalculatePay();
  
            // The following lines cause compiler error CS1540. The compiler
            // cannot determine at compile time what the run-time types of
            // emp2 and emp3 will be.  
            //emp2.CalculatePay();
            //emp3.CalculatePay();  
  
        }  
    }  
}  
```
