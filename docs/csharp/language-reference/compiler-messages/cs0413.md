# <a name="compiler-error-cs0413"></a><span data-ttu-id="51bf6-102">Błąd kompilatora CS0413</span><span class="sxs-lookup"><span data-stu-id="51bf6-102">Compiler Error CS0413</span></span>
<span data-ttu-id="51bf6-103">Parametr typu "parametr typu" nie można używać z operatorem "as", ponieważ nie ma ograniczenia typu klasy ani ograniczenia "class"</span><span class="sxs-lookup"><span data-stu-id="51bf6-103">The type parameter 'type parameter' cannot be used with the 'as' operator because it does not have a class type constraint nor a 'class' constraint</span></span>  
  
 <span data-ttu-id="51bf6-104">Ten błąd występuje, jeśli korzysta z typem ogólnym [jako](../../../csharp/language-reference/keywords/as.md) operatora, ale typ rodzajowy nie ma klasy typu ograniczenia.</span><span class="sxs-lookup"><span data-stu-id="51bf6-104">This error occurs if a generic type uses the [as](../../../csharp/language-reference/keywords/as.md) operator, but that generic type does not have a class type constraint.</span></span> <span data-ttu-id="51bf6-105">`as` Operator jest dozwolona tylko w przypadku typów referencyjnych, więc parametru typu muszą być ograniczone do zagwarantowania, że nie jest typem wartości.</span><span class="sxs-lookup"><span data-stu-id="51bf6-105">The `as` operator is only allowed with reference types, so the type parameter must be constrained to guarantee that it is not a value type.</span></span> <span data-ttu-id="51bf6-106">Aby uniknąć tego błędu, należy użyć ograniczenie typu klasy lub ograniczenie typu odwołania.</span><span class="sxs-lookup"><span data-stu-id="51bf6-106">To avoid this error, use a class type constraint or a reference type constraint.</span></span>  
  
 <span data-ttu-id="51bf6-107">Jest to spowodowane `as` operator może zwrócić `null`, który nie jest możliwa wartość dla typu wartości i parametru typu muszą być traktowane jako typ wartości, chyba że jest to ograniczenie typu klasy lub ograniczenie typu odwołania.</span><span class="sxs-lookup"><span data-stu-id="51bf6-107">This is because the `as` operator could return `null`, which is not a possible value for a value type, and the type parameter must be treated as a value type unless it is a class type constraint or a reference type constraint.</span></span>  
  
## <a name="example"></a><span data-ttu-id="51bf6-108">Przykład</span><span class="sxs-lookup"><span data-stu-id="51bf6-108">Example</span></span>  
 <span data-ttu-id="51bf6-109">Poniższy przykład spowoduje wygenerowanie CS0413.</span><span class="sxs-lookup"><span data-stu-id="51bf6-109">The following sample generates CS0413.</span></span>  
  
```csharp  
// CS0413.cs  
// compile with: /target:library  
class A {}  
class B : A {}  
  
class CMain  
{  
   A a = null;  
   public void G<T>()  
   {  
      a = new A();  
      System.Console.WriteLine (a as T);  // CS0413  
   }  
  
   // OK  
   public void H<T>() where T : A  
   {  
      a = new A();  
      System.Console.WriteLine (a as T);  
   }  
}  
```
