# <a name="compiler-error-cs0826"></a><span data-ttu-id="4c74c-103">Błąd kompilatora CS0826</span><span class="sxs-lookup"><span data-stu-id="4c74c-103">Compiler Error CS0826</span></span>

<span data-ttu-id="4c74c-104">Nie znaleziono najlepszego typu dla niejawnie wpisanej tablicy.</span><span class="sxs-lookup"><span data-stu-id="4c74c-104">No best type found for implicitly typed array.</span></span>  
  
 <span data-ttu-id="4c74c-105">Wszystkie elementy tablicy muszą być tego samego typu lub niejawnie konwertowane na ten sam typ, zgodnie z regułami wnioskowania typów używanymi przez kompilator.</span><span class="sxs-lookup"><span data-stu-id="4c74c-105">Array elements must all be the same type or implicitly convertible to the same type according to the type inference rules used by the compiler.</span></span> <span data-ttu-id="4c74c-106">Najlepszy typ musi być jednym z typów występujących w wyrażeniu tablicy.</span><span class="sxs-lookup"><span data-stu-id="4c74c-106">The best type must be one of the types present in the array expression.</span></span> <span data-ttu-id="4c74c-107">Elementy nie zostaną przekonwertowane na nowy typ, taki jak `object` .</span><span class="sxs-lookup"><span data-stu-id="4c74c-107">Elements will not be converted to a new type such as `object`.</span></span> <span data-ttu-id="4c74c-108">W przypadku niejawnie wpisanej tablicy kompilator musi wywnioskować typ tablicy na podstawie typu elementów przypisanych do niego.</span><span class="sxs-lookup"><span data-stu-id="4c74c-108">For an implicitly typed array, the compiler must infer the array type based on the type of elements assigned to it.</span></span>

## <a name="example"></a><span data-ttu-id="4c74c-113">Przykład</span><span class="sxs-lookup"><span data-stu-id="4c74c-113">Example</span></span>  

 <span data-ttu-id="4c74c-114">Poniższy kod generuje CS0826, ponieważ elementy tablicy nie są tego samego typu, a logika wnioskowania typu kompilatora nie znajduje pojedynczego najlepszego typu:</span><span class="sxs-lookup"><span data-stu-id="4c74c-114">The following code generates CS0826 because the array elements are not all the same type, and the compiler's type inference logic does not find a single best type:</span></span>  
  
```csharp  
// cs0826.cs  
public class C  
{  
    public static int Main()  
    {  
        var x = new[] { 1, "str" }; // CS0826  
  
        char c = 'c';  
        short s1 = 0;  
        short s2 = -0;  
        short s3 = 1;  
        short s4 = -1;  
  
        var array1 = new[] { s1, s2, s3, s4, c, '1' }; // CS0826  
        return 1;  
    }  
}  
```
