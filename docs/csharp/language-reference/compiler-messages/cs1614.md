# <a name="compiler-error-cs1614"></a><span data-ttu-id="8b5ae-102">Błąd kompilatora CS1614</span><span class="sxs-lookup"><span data-stu-id="8b5ae-102">Compiler Error CS1614</span></span>
<span data-ttu-id="8b5ae-103">"Nazwa" jest niejednoznaczny; między "attribute1" i "attribute2".</span><span class="sxs-lookup"><span data-stu-id="8b5ae-103">'name' is ambiguous; between 'attribute1' and 'attribute2'.</span></span> <span data-ttu-id="8b5ae-104">użyć "@attribute" lub "attributeAttribute"</span><span class="sxs-lookup"><span data-stu-id="8b5ae-104">use either '@attribute' or 'attributeAttribute'</span></span>  
  
 <span data-ttu-id="8b5ae-105">Kompilator napotkał specyfikacji atrybutu niejednoznaczne.</span><span class="sxs-lookup"><span data-stu-id="8b5ae-105">The compiler has encountered an ambiguous attribute specification.</span></span>  
  
 <span data-ttu-id="8b5ae-106">Dla wygody kompilatora C# służy do określenia **ExampleAttribute** tylko jako `[Example]`.</span><span class="sxs-lookup"><span data-stu-id="8b5ae-106">For convenience, the C# compiler allows you to specify **ExampleAttribute** as just `[Example]`.</span></span> <span data-ttu-id="8b5ae-107">Jednak niejednoznaczności wystąpi, jeśli klasa atrybutu o nazwie `Example` istnieje wraz z **ExampleAttribute**, ponieważ kompilator nie wiadomo, jeśli `[Example]` odwołuje się do `Example` atrybutu lub  **ExampleAttribute** atrybutu.</span><span class="sxs-lookup"><span data-stu-id="8b5ae-107">However, ambiguity arises if an attribute class named `Example` exists along with **ExampleAttribute**, because the compiler cannot tell if `[Example]` refers to the `Example` attribute or the **ExampleAttribute** attribute.</span></span> <span data-ttu-id="8b5ae-108">Wyjaśnienie, użyj `[@Example]` dla `Example` atrybutu i `[ExampleAttribute]` dla **ExampleAttribute**.</span><span class="sxs-lookup"><span data-stu-id="8b5ae-108">To clarify, use `[@Example]` for the `Example` attribute and `[ExampleAttribute]` for **ExampleAttribute**.</span></span>  
  
 <span data-ttu-id="8b5ae-109">Poniższy przykład generuje CS1614:</span><span class="sxs-lookup"><span data-stu-id="8b5ae-109">The following sample generates CS1614:</span></span>  
  
```csharp  
// CS1614.cs  
using System;  
  
// Both of the following classes are valid attributes with valid  
// names (MySpecial and MySpecialAttribute). However, because the lookup  
// rules for attributes involves auto-appending the 'Attribute' suffix  
// to the identifier, these two attributes become ambiguous; that is,  
// if you specify MySpecial, the compiler can't tell if you want  
// MySpecial or MySpecialAttribute.  
  
public class MySpecial : Attribute {  
   public MySpecial() {}  
}  
  
public class MySpecialAttribute : Attribute {  
   public MySpecialAttribute() {}  
}  
  
class MakeAWarning {  
   [MySpecial()] // CS1614  
                 // Ambiguous: MySpecial or MySpecialAttribute?  
   public static void Main() {  
   }  
  
   [@MySpecial()] // This isn't ambiguous, it binds to the first attribute above.  
   public static void NoWarning() {  
   }  
  
   [MySpecialAttribute()] // This isn't ambiguous, it binds to the second attribute above.  
   public static void NoWarning2() {  
   }  
  
   [@MySpecialAttribute()] // This is also legal.  
   public static void NoWarning3() {  
   }  
}  
```
