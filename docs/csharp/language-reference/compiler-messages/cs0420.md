# <a name="compiler-warning-level-1-cs0420"></a>Ostrzeżenie kompilatora (poziom 1) CS0420
'Identyfikator': odwołanie do pola nietrwałego nie będzie traktowane jako nietrwałe  
  
 Pola nietrwałego nie należy zwykle przekazywać przy użyciu `ref` lub **się** parametru, ponieważ nie będzie traktowane jako nietrwałe w zakresie funkcji. Istnieją wyjątki od tej, takie jak podczas wywołanie blokowanego interfejsu API. Wszelkie ostrzeżenia mogą używać [ostrzeżenie #pragma](../../../csharp/language-reference/preprocessor-directives/preprocessor-pragma-warning.md) Aby wyłączyć to ostrzeżenie w tych rzadkich przypadkach, w których umyślnie używasz pola nietrwałego jako parametr przekazany przez odwołanie.  
  
 Poniższy przykład spowoduje wygenerowanie CS0420:  
  
```csharp  
// CS0420.cs  
// compile with: /W:1  
using System;  
  
class TestClass  
{  
   private volatile int i;  
  
   public void TestVolatile(ref int ii)  
   {  
   }  
  
   public static void Main()  
   {  
      TestClass x = new TestClass();  
      x.TestVolatile(ref x.i);   // CS0420   
   }  
}  
```
