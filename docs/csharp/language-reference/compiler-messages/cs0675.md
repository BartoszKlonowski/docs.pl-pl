# <a name="compiler-warning-level-3-cs0675"></a><span data-ttu-id="2c678-102">Ostrzeżenie kompilatora (poziom 3) CS0675</span><span class="sxs-lookup"><span data-stu-id="2c678-102">Compiler Warning (level 3) CS0675</span></span>
<span data-ttu-id="2c678-103">Bitwise-lub operator używany na znak rozszerzony operand; najpierw rozważyć rzutowanie na mniejszy niepodpisany typ</span><span class="sxs-lookup"><span data-stu-id="2c678-103">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="2c678-104">Kompilator niejawnie poszerzony i znak rozszerzony zmiennej, a następnie używane wynikową wartość w operacji bitowej LUB.</span><span class="sxs-lookup"><span data-stu-id="2c678-104">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="2c678-105">Może to spowodować nieoczekiwane zachowanie.</span><span class="sxs-lookup"><span data-stu-id="2c678-105">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="2c678-106">Poniższy przykład generuje CS0675:</span><span class="sxs-lookup"><span data-stu-id="2c678-106">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
