# <a name="compiler-warning-level-3-cs0675"></a><span data-ttu-id="5ff12-102">Ostrzeżenie kompilatora (poziom 3) CS0675</span><span class="sxs-lookup"><span data-stu-id="5ff12-102">Compiler Warning (level 3) CS0675</span></span>
<span data-ttu-id="5ff12-103">Bitowe or- lub argumentu ze znakiem; użyto operatora należy rozważyć najpierw rzutowania na mniejszy typ bez znaku</span><span class="sxs-lookup"><span data-stu-id="5ff12-103">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="5ff12-104">Kompilator niejawnie poszerzył i rozszerzona o znak zmiennej, a następnie użył wartości wynikowej w operacji bitowej OR.</span><span class="sxs-lookup"><span data-stu-id="5ff12-104">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="5ff12-105">Może to spowodować nieoczekiwane zachowanie.</span><span class="sxs-lookup"><span data-stu-id="5ff12-105">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="5ff12-106">Poniższy przykład spowoduje wygenerowanie CS0675:</span><span class="sxs-lookup"><span data-stu-id="5ff12-106">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
