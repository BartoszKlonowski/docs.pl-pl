<span data-ttu-id="466b8-103">Operator- lub operator używany w rozszerzonym znakiem; Rozważ najpierw rzutowania na mniejszy typ bez znaku</span><span class="sxs-lookup"><span data-stu-id="466b8-103">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="466b8-104">Kompilator niejawnie poszerzył i znakiem zmiennej, a następnie użył wartości wynikowej w operacji bitowej OR.</span><span class="sxs-lookup"><span data-stu-id="466b8-104">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="466b8-105">Może to spowodować nieoczekiwane zachowanie.</span><span class="sxs-lookup"><span data-stu-id="466b8-105">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="466b8-106">Poniższy przykład generuje CS0675:</span><span class="sxs-lookup"><span data-stu-id="466b8-106">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
