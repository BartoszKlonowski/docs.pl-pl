# <a name="compiler-error-cs0545"></a>Błąd kompilatora CS0545
'Funkcja': nie można przesłonić, ponieważ element "właściwość" nie ma metody dostępu get z możliwością przesłonięcia  
  
 Try została wprowadzona do definiowania zastąpienia dla metody dostępu właściwości, gdy klasa bazowa nie ma takich definicji, aby zastąpić. Można naprawić ten błąd, przez:  
  
- Dodawanie `set` metody dostępu w klasie bazowej.  
  
- Usuwanie `set` dostępu z klasy pochodnej.  
  
- Ukrywanie właściwości klasy bazowej, dodając [nowe](../../../csharp/language-reference/keywords/new-modifier.md) — słowo kluczowe do właściwości w klasie pochodnej.  
  
- Tworzenie właściwości klasy bazowej [wirtualnego](../../../csharp/language-reference/keywords/virtual.md).  
  
 Aby uzyskać więcej informacji, zobacz [przy użyciu właściwości](../../../csharp/programming-guide/classes-and-structs/using-properties.md).  
  
## <a name="example"></a>Przykład  
 Poniższy przykład spowoduje wygenerowanie CS0545.  
  
```csharp  
// CS0545.cs  
// compile with: /target:library  
// CS0545  
public class a  
{  
   public virtual int i  
   {  
      set {}  
  
      // Uncomment the following line to resolve.  
      // get { return 0; }  
   }  
}  
  
public class b : a  
{  
   public override int i  
   {  
      get { return 0; }  
      set {}   // OK  
   }  
}  
```
