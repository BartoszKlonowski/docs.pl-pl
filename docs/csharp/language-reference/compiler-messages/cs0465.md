<span data-ttu-id="4ad52-103">Wprowadzenie metody "Finalize" może zakłócać wywołanie destruktora.</span><span class="sxs-lookup"><span data-stu-id="4ad52-103">Introducing a 'Finalize' method can interfere with destructor invocation.</span></span> <span data-ttu-id="4ad52-104">Czy zamierzane było zadeklarowanie destruktora?</span><span class="sxs-lookup"><span data-stu-id="4ad52-104">Did you intend to declare a destructor?</span></span>

 <span data-ttu-id="4ad52-106">Jeśli taka klasa jest używana jako klasa bazowa i jeśli Klasa pochodna definiuje destruktor, ten destruktor przesłoni klasy bazowej `Finalize` metody nie <xref:System.Object.Finalize%2A>.</span><span class="sxs-lookup"><span data-stu-id="4ad52-106">If such a class is used as a base class and if the deriving class defines a destructor, the destructor will override the base class `Finalize` method, not <xref:System.Object.Finalize%2A>.</span></span>  
  
## <a name="example"></a><span data-ttu-id="4ad52-107">Przykład</span><span class="sxs-lookup"><span data-stu-id="4ad52-107">Example</span></span>  
 <span data-ttu-id="4ad52-108">Poniższy przykład spowoduje wygenerowanie CS0465.</span><span class="sxs-lookup"><span data-stu-id="4ad52-108">The following sample generates CS0465.</span></span>  
  
```csharp  
// CS0465.cs  
// compile with: /target:library  
class A  
{  
   public virtual void Finalize() {}   // CS0465  
}  
  
// OK  
class B  
{  
   ~B() {}  
}  
```
