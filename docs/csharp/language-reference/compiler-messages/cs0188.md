 <span data-ttu-id="2d2ec-105">Jeśli zostanie wyświetlony ten błąd podczas próby zainicjowania właściwości w konstruktorze struktury rozwiązaniem jest zmiana parametru konstruktora, aby określić pole zapasowe zamiast samej właściwości.</span><span class="sxs-lookup"><span data-stu-id="2d2ec-105">If you see this error when trying to initialize a property in a struct constructor, the solution is to change the constructor parameter to specify the backing field instead of the property itself.</span></span> <span data-ttu-id="2d2ec-106">Właściwości zaimplementowane automatycznie należy unikać w strukturach, ponieważ mają nie pole zapasowe i nie można zainicjować w żadnym z konstruktora.</span><span class="sxs-lookup"><span data-stu-id="2d2ec-106">Auto-implemented properties should be avoided in structs because they have no backing field and therefore cannot be initialized in any way from the constructor.</span></span>

## <a name="example"></a><span data-ttu-id="2d2ec-108">Przykład</span><span class="sxs-lookup"><span data-stu-id="2d2ec-108">Example</span></span>

 <span data-ttu-id="2d2ec-109">Poniższy przykład spowoduje wygenerowanie CS0188:</span><span class="sxs-lookup"><span data-stu-id="2d2ec-109">The following sample generates CS0188:</span></span>  

```csharp
// CS0188.cs  
// compile with: /t:library  
namespace MyNamespace  
{  
    class MyClass  
    {  
        struct S  
        {  
            public int a;  
  
            void MyMethod()  
            {  
            }  
  
            S(int i)  
            {  
                // a = i;  
                MyMethod();  // CS0188  
            }  
        }  
        public static void Main()  
        { }  
  
    }  
}  
```
