## <a name="designing-class-libraries-for-my"></a><span data-ttu-id="0b0a1-165">Projektowanie bibliotek klas dla `My`</span><span class="sxs-lookup"><span data-stu-id="0b0a1-165">Designing class libraries for `My`</span></span>

<span data-ttu-id="0b0a1-166">Podobnie jak w przypadku większości modeli obiektów, niektóre wzorce projektowania działają prawidłowo w przestrzeni nazw `My` i inne nie.</span><span class="sxs-lookup"><span data-stu-id="0b0a1-166">As is the case with most object models, some design patterns work well in the `My` namespace and others do not.</span></span> <span data-ttu-id="0b0a1-167">Podczas projektowania rozszerzenia do przestrzeni nazw `My` należy wziąć pod uwagę następujące zasady:</span><span class="sxs-lookup"><span data-stu-id="0b0a1-167">When designing an extension to the `My` namespace, consider the following principles:</span></span>

- <span data-ttu-id="0b0a1-168">**Metody bezstanowe.**</span><span class="sxs-lookup"><span data-stu-id="0b0a1-168">**Stateless methods.**</span></span> <span data-ttu-id="0b0a1-169">Metody w przestrzeni nazw `My` powinny dostarczyć kompletne rozwiązanie do określonego zadania.</span><span class="sxs-lookup"><span data-stu-id="0b0a1-169">Methods in the `My` namespace should provide a complete solution to a specific task.</span></span> <span data-ttu-id="0b0a1-170">Upewnij się, że wartości parametrów, które są przesyłane do metody, zapewniają wszystkie dane wejściowe wymagane do ukończenia określonego zadania.</span><span class="sxs-lookup"><span data-stu-id="0b0a1-170">Ensure that the parameter values that are passed to the method provide all the input required to complete the particular task.</span></span> <span data-ttu-id="0b0a1-171">Należy unikać tworzenia metod, które są zależne od wcześniejszego stanu, na przykład otwartych połączeń z zasobami.</span><span class="sxs-lookup"><span data-stu-id="0b0a1-171">Avoid creating methods that rely on prior state, such as open connections to resources.</span></span>

- <span data-ttu-id="0b0a1-178">**Metody fabryki.**</span><span class="sxs-lookup"><span data-stu-id="0b0a1-178">**Factory methods.**</span></span> <span data-ttu-id="0b0a1-179">Niektóre typy muszą być trudne do wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="0b0a1-179">Some types are necessarily difficult to instantiate.</span></span> <span data-ttu-id="0b0a1-180">Dostarczanie metod fabrycznych jako rozszerzeń przestrzeni nazw `My` umożliwia łatwiejsze odnajdywanie i używanie typów, które należą do tej kategorii.</span><span class="sxs-lookup"><span data-stu-id="0b0a1-180">Providing factory methods as extensions to the `My` namespace enables you to more easily discover and consume types that fall into this category.</span></span> <span data-ttu-id="0b0a1-181">Przykładem metody fabryki, która działa prawidłowo, jest `My.Computer.FileSystem.OpenTextFileReader`.</span><span class="sxs-lookup"><span data-stu-id="0b0a1-181">An example of a factory method that works well is `My.Computer.FileSystem.OpenTextFileReader`.</span></span> <span data-ttu-id="0b0a1-182">Istnieje kilka typów strumieni dostępnych w .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="0b0a1-182">There are several stream types available in the .NET Framework.</span></span> <span data-ttu-id="0b0a1-183">Dzięki określeniu plików tekstowych `OpenTextFileReader` pomaga użytkownikowi zrozumieć, którego strumienia użyć.</span><span class="sxs-lookup"><span data-stu-id="0b0a1-183">By specifying text files specifically, the `OpenTextFileReader` helps the user understand which stream to use.</span></span>
