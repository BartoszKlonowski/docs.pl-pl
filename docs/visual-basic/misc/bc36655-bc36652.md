<span data-ttu-id="88240-103">Typów danych parametrów typu w metodzie rozszerzenia "\<methodname >" zdefiniowanej w "\<typename >' nie można wywnioskować na podstawie tych argumentów, ponieważ możliwe jest użycie więcej niż jednego typu.</span><span class="sxs-lookup"><span data-stu-id="88240-103">Data type(s) of the type parameter(s) in extension method '\<methodname>' defined in '\<typename>' cannot be inferred from these arguments because more than one type is possible.</span></span> <span data-ttu-id="88240-104">Określanie danych typów jawnie może rozwiązać ten problem.</span><span class="sxs-lookup"><span data-stu-id="88240-104">Specifying the data type(s) explicitly might correct this error.</span></span>

> [!NOTE]
>  <span data-ttu-id="88240-107">Podczas określania argumentów nie jest opcją (na przykład dla operatorów zapytań w wyrażeniach zapytań), zostanie wyświetlony komunikat o błędzie bez drugie zdanie.</span><span class="sxs-lookup"><span data-stu-id="88240-107">When specifying arguments is not an option (for example, for query operators in query expressions), the error message appears without the second sentence.</span></span>  
  
 <span data-ttu-id="88240-108">Poniższy kod przedstawia błędu.</span><span class="sxs-lookup"><span data-stu-id="88240-108">The following code demonstrates the error.</span></span>  
  
```vb

Imports System.Runtime.CompilerServices  
Module Module1  
    Sub Main()  
  
        Dim caller As New Class1  
        '' Not valid.  
        'caller.targetExtension(1, "2")  
  
    End Sub  
  
    <Extension()> _  
    Sub targetExtension(Of T)(ByVal p0 As Class1, ByVal p1 As T, ByVal p2 As T)  
    End Sub  
  
    Class Class1  
    End Class  
  
End Module
```  
