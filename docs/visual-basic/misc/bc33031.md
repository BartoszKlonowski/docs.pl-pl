<span data-ttu-id="487af-103">Operator konwersji jest zadeklarowany z typem parametru, który pochodzi od typu zwracanego.</span><span class="sxs-lookup"><span data-stu-id="487af-103">A conversion operator is declared with a parameter type that derives from the return type.</span></span>  
  
 <span data-ttu-id="487af-104">W czasie kompilacji, Visual Basic uznaje, że wstępnie zdefiniowana konwersja ma istnieć z dowolnego typu odwołania do dowolnego typu w hierarchii dziedziczenia, czyli dowolnego typu, z którego pochodzi lub który pochodzi od tego elementu.</span><span class="sxs-lookup"><span data-stu-id="487af-104">At compile time, Visual Basic considers a predefined conversion to exist from any reference type to any type in its inheritance hierarchy, that is, any type from which it derives or which derives from it.</span></span> <span data-ttu-id="487af-105">Taka Konwersja może się nie powieść w czasie wykonywania, ale kompilator nie może przewidzieć wyników w czasie wykonywania, dzięki czemu możliwe jest dokonanie takiej konwersji.</span><span class="sxs-lookup"><span data-stu-id="487af-105">Such a conversion might fail at run time, but the compiler cannot predict run-time results, so it allows any such conversion to compile.</span></span>
