# <a name="anonymous-type-property-propertyname-cannot-be-used-in-the-definition-of-a-lambda-expression-within-the-same-initialization-list"></a><span data-ttu-id="f3c31-102">Nie można użyć właściwości typu anonimowego " \<propertyname> " w definicji wyrażenia lambda w obrębie tej samej listy inicjalizacji</span><span class="sxs-lookup"><span data-stu-id="f3c31-102">Anonymous type property '\<propertyname>' cannot be used in the definition of a lambda expression within the same initialization list</span></span>

<span data-ttu-id="f3c31-103">Właściwości zdefiniowane na liście inicjalizacji typu anonimowego nie mogą być częścią definicji wyrażenia lambda na tej samej liście.</span><span class="sxs-lookup"><span data-stu-id="f3c31-103">Properties defined in the initialization list of an anonymous type cannot be part of a lambda expression definition in the same list.</span></span> <span data-ttu-id="f3c31-104">Na przykład, w poniższym kodzie, właściwość `Num` nie może być uwzględniona w definicji `LambdaFun` .</span><span class="sxs-lookup"><span data-stu-id="f3c31-104">For example, in the following code, property `Num` cannot be included in the definition of `LambdaFun`.</span></span>

```vb
' Not valid.
'Dim anon = New With {.Num = 4, .LambdaFun = Function() .Num > 0}
```

<span data-ttu-id="f3c31-105">**Identyfikator błędu:** BC36549</span><span class="sxs-lookup"><span data-stu-id="f3c31-105">**Error ID:** BC36549</span></span>

## <a name="to-correct-this-error"></a><span data-ttu-id="f3c31-106">Aby poprawić ten błąd</span><span class="sxs-lookup"><span data-stu-id="f3c31-106">To correct this error</span></span>

1. <span data-ttu-id="f3c31-107">Rozważ podzielenie typu anonimowego na dwie części:</span><span class="sxs-lookup"><span data-stu-id="f3c31-107">Consider splitting the anonymous type into two parts:</span></span>

    ```vb
    Dim anon1 = New With {.Num = 4}
    Dim anon2 = New With {.LambdaFun = Function() anon1.Num > 0}
    ' - or -
    Dim anon3 = New With {.lambdaFun = Function(n As Integer) n > 0}
    Console.WriteLine((anon2.LambdaFun)())
    Console.WriteLine(anon3.lambdaFun(anon1.Num))
    anon1.Num = -5
    Console.WriteLine((anon2.LambdaFun)())
    Console.WriteLine(anon3.lambdaFun(anon1.Num))
    ```

    <span data-ttu-id="f3c31-108">Należy pamiętać, że jeśli deklarujesz `anon1.Num` jako `Key` Właściwość, jej wartość nie może zostać zmieniona.</span><span class="sxs-lookup"><span data-stu-id="f3c31-108">Note that if you declare `anon1.Num` as a `Key` property, its value cannot be changed.</span></span>

2. <span data-ttu-id="f3c31-109">Alternatywą jest użycie zwykłej instrukcji funkcyjnej w celu uzyskania dostępu do właściwości typu anonimowego:</span><span class="sxs-lookup"><span data-stu-id="f3c31-109">An alternative is to use a regular function statement to access the anonymous type property:</span></span>

    ```vb
    Function testNum(ByVal n As Integer) As Boolean
        Return n > 0
    End Function
    Console.WriteLine(testNum(anon1.Num))
    ```

3. <span data-ttu-id="f3c31-110">Podobnie można użyć funkcji lambda zdefiniowanej poza typem anonimowym:</span><span class="sxs-lookup"><span data-stu-id="f3c31-110">Similarly, you can use a lambda function that is defined outside the anonymous type:</span></span>

    ```vb
    Dim lambdaFun1 = Function() anon1.Num > 0
    Dim lambdaFun2 = Function(n As Integer) n > 0
    ```
