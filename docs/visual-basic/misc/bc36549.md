# <a name="anonymous-type-property-39ltpropertynamegt39-cannot-be-used-in-the-definition-of-a-lambda-expression-within-the-same-initialization-list"></a><span data-ttu-id="47d7a-102">Właściwości typu anonimowego &#39; &lt;propertyname&gt; &#39; nie można użyć w definicji wyrażenia lambda w obrębie tej samej listy inicjowania</span><span class="sxs-lookup"><span data-stu-id="47d7a-102">Anonymous type property &#39;&lt;propertyname&gt;&#39; cannot be used in the definition of a lambda expression within the same initialization list</span></span>
<span data-ttu-id="47d7a-103">Właściwości zdefiniowane na liście inicjowania typu anonimowego nie może być częścią definicji wyrażenia lambda na liście.</span><span class="sxs-lookup"><span data-stu-id="47d7a-103">Properties defined in the initialization list of an anonymous type cannot be part of a lambda expression definition in the same list.</span></span> <span data-ttu-id="47d7a-104">Na przykład w poniższym kodzie, właściwość `Num` nie może występować w definicji `LambdaFun`.</span><span class="sxs-lookup"><span data-stu-id="47d7a-104">For example, in the following code, property `Num` cannot be included in the definition of `LambdaFun`.</span></span>  
  
```vb  
' Not valid.  
'Dim anon = New With {.Num = 4, .LambdaFun = Function() .Num > 0}  
```  
  
 <span data-ttu-id="47d7a-105">**Identyfikator błędu:** BC36549</span><span class="sxs-lookup"><span data-stu-id="47d7a-105">**Error ID:** BC36549</span></span>  

## <a name="to-correct-this-error"></a><span data-ttu-id="47d7a-106">Aby poprawić ten błąd</span><span class="sxs-lookup"><span data-stu-id="47d7a-106">To correct this error</span></span>  
  
1.  <span data-ttu-id="47d7a-107">Należy rozważyć rozdzielenie typu anonimowego na dwie części:</span><span class="sxs-lookup"><span data-stu-id="47d7a-107">Consider splitting the anonymous type into two parts:</span></span>  
  
```vb  
Dim anon1 = New With {.Num = 4}  
Dim anon2 = New With {.LambdaFun = Function() anon1.Num > 0}  
' - or -  
Dim anon3 = New With {.lambdaFun = Function(n As Integer) n > 0}  
Console.WriteLine((anon2.LambdaFun)())  
Console.WriteLine(anon3.lambdaFun(anon1.Num))  
anon1.Num = -5  
Console.WriteLine((anon2.LambdaFun)())  
Console.WriteLine(anon3.lambdaFun(anon1.Num))  
```  
  
<span data-ttu-id="47d7a-108">Należy pamiętać, że w przypadku `anon1.Num` jako `Key` właściwości, nie można zmienić jej wartości.</span><span class="sxs-lookup"><span data-stu-id="47d7a-108">Note that if you declare `anon1.Num` as a `Key` property, its value cannot be changed.</span></span>  
  
2.  <span data-ttu-id="47d7a-109">Alternatywą jest za pomocą instrukcji normalne działanie można uzyskać dostępu do właściwości typu anonimowego:</span><span class="sxs-lookup"><span data-stu-id="47d7a-109">An alternative is to use a regular function statement to access the anonymous type property:</span></span>  
  
```vb  
Function testNum(ByVal n As Integer) As Boolean  
    Return n > 0  
End Function  
Console.WriteLine(testNum(anon1.Num))  
```  
  
3.  <span data-ttu-id="47d7a-110">Podobnie możesz użyć funkcji lambda, który jest zdefiniowany poza typu anonimowego:</span><span class="sxs-lookup"><span data-stu-id="47d7a-110">Similarly, you can use a lambda function that is defined outside the anonymous type:</span></span>  
  
```vb  
Dim lambdaFun1 = Function() anon1.Num > 0  
Dim lambdaFun2 = Function(n As Integer) n > 0  
```
