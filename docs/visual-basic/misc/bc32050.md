# <a name="type-parameter-39lttypeparameternamegt39-for-39ltgenericprocedurenamegt39-cannot-be-inferred"></a><span data-ttu-id="9f711-102">Parametr typu &#39; &lt;typeparametername&gt; &#39; dla &#39; &lt;genericprocedurename&gt; &#39; nie można wywnioskować</span><span class="sxs-lookup"><span data-stu-id="9f711-102">Type parameter &#39;&lt;typeparametername&gt;&#39; for &#39;&lt;genericprocedurename&gt;&#39; cannot be inferred</span></span>
<span data-ttu-id="9f711-103">Ogólny procedura jest wywoływana bez podawania listy argumentów typu, a wnioskowanie o typie nie powiedzie się z jednego z argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="9f711-103">A generic procedure is called without supplying a type argument list, and type inference fails for one of the type arguments.</span></span>  
  
 <span data-ttu-id="9f711-104">Po wywołaniu procedury ogólnego zwykle należy podać typ argumentu dla każdego parametru typ zdefiniowany przez procedurę.</span><span class="sxs-lookup"><span data-stu-id="9f711-104">When you call a generic procedure, you normally supply a type argument for each type parameter defined by the procedure.</span></span> <span data-ttu-id="9f711-105">Jednak masz alternatywnej z pominięciem całkowicie listy argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="9f711-105">However, you have the alternative of omitting the type argument list entirely.</span></span> <span data-ttu-id="9f711-106">Gdy to zrobisz, kompilator próbuje wnioskować o typie argumentu typu z kontekstu wywołania.</span><span class="sxs-lookup"><span data-stu-id="9f711-106">When you do this, the compiler attempts to infer the type of each type argument from the context of your call.</span></span> <span data-ttu-id="9f711-107">Aby uzyskać więcej informacji, zobacz "Wnioskowanie o typie" w [procedury ogólne w Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md).</span><span class="sxs-lookup"><span data-stu-id="9f711-107">For more information, see "Type Inference" in [Generic Procedures in Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md).</span></span>  
  
 <span data-ttu-id="9f711-108">Jedną z możliwych przyczyn niepowodzenia wnioskowania typu niezgodność rangę między parametrem typu i wywoływania typu.</span><span class="sxs-lookup"><span data-stu-id="9f711-108">One possible cause of type inference failure is a mismatch of rank between a type parameter and the calling type.</span></span> <span data-ttu-id="9f711-109">Ilustruje to poniższy kod.</span><span class="sxs-lookup"><span data-stu-id="9f711-109">The following code illustrates this.</span></span>  
  
```  
Public Sub displayLargest(Of t As IComparable)(ByVal arg() As t)  
    ' Insert code to find and display the largest element of arg().  
End Sub  
Public Sub callGenericSub()  
    Dim testValue As Integer  
    findLargest(testValue)  
    Dim testMatrix(,) As Integer  
    findLargest(testMatrix)  
End Sub  
```  
  
 <span data-ttu-id="9f711-110">W powyższym kodzie dwa wywołań `findLargest` zarówno utworzyć ten błąd, ponieważ parametr typu `t` tablicą jednowymiarową wymaga argumentów typu kompilator wnioskuje z wywołania są skalarnej (`testValue`) i tablicą dwuwymiarową (`testMatrix`).</span><span class="sxs-lookup"><span data-stu-id="9f711-110">In the preceding code, the two calls to `findLargest` both produce this error, because the type parameter `t` calls for a one-dimensional array, whereas the type arguments the compiler infers from the calls are a scalar (`testValue`) and a two-dimensional array (`testMatrix`).</span></span>
