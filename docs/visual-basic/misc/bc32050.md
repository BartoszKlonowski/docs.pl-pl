# <a name="type-parameter-typeparametername-for-genericprocedurename-cannot-be-inferred"></a><span data-ttu-id="674af-102">Parametr typu '\<typeparametername >' dla '\<genericprocedurename >' nie można wywnioskować</span><span class="sxs-lookup"><span data-stu-id="674af-102">Type parameter '\<typeparametername>' for '\<genericprocedurename>' cannot be inferred</span></span>
<span data-ttu-id="674af-103">Ogólna procedura jest wywoływana bez podawania lista argumentów typu i wnioskowanie o typie nie powiedzie się z jednego z argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="674af-103">A generic procedure is called without supplying a type argument list, and type inference fails for one of the type arguments.</span></span>  
  
 <span data-ttu-id="674af-104">Po wywołaniu procedury ogólne, zwykle należy podać argument typu dla każdego parametru typ zdefiniowany przez procedurę.</span><span class="sxs-lookup"><span data-stu-id="674af-104">When you call a generic procedure, you normally supply a type argument for each type parameter defined by the procedure.</span></span> <span data-ttu-id="674af-105">Jednak masz alternatywne całkowicie pomijając lista argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="674af-105">However, you have the alternative of omitting the type argument list entirely.</span></span> <span data-ttu-id="674af-106">Gdy to zrobisz, kompilator spróbuje wywnioskować typ każdego argumentu typu z kontekstu wywołania.</span><span class="sxs-lookup"><span data-stu-id="674af-106">When you do this, the compiler attempts to infer the type of each type argument from the context of your call.</span></span> <span data-ttu-id="674af-107">Aby uzyskać więcej informacji, zobacz 'Wnioskowanie o typie' w [procedury ogólne w Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md).</span><span class="sxs-lookup"><span data-stu-id="674af-107">For more information, see "Type Inference" in [Generic Procedures in Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md).</span></span>  
  
 <span data-ttu-id="674af-108">Jedną z możliwych przyczyn niepowodzenia wnioskowanie o typie niezgodność ranga między parametrem typu i wywoływania type.</span><span class="sxs-lookup"><span data-stu-id="674af-108">One possible cause of type inference failure is a mismatch of rank between a type parameter and the calling type.</span></span> <span data-ttu-id="674af-109">Ilustruje to poniższy kod.</span><span class="sxs-lookup"><span data-stu-id="674af-109">The following code illustrates this.</span></span>  
  
```  
Public Sub displayLargest(Of t As IComparable)(ByVal arg() As t)  
    ' Insert code to find and display the largest element of arg().  
End Sub  
Public Sub callGenericSub()  
    Dim testValue As Integer  
    findLargest(testValue)  
    Dim testMatrix(,) As Integer  
    findLargest(testMatrix)  
End Sub  
```  
  
 <span data-ttu-id="674af-110">W poprzednim kodzie dwa wywołania `findLargest` zarówno generuje ten błąd, ponieważ parametr typu `t` wymaga Jednowymiarowa tablica argumentów typu, kompilator wnioskuje wywołań są skalarną (`testValue`) i dwuwymiarowej tablicy (`testMatrix`).</span><span class="sxs-lookup"><span data-stu-id="674af-110">In the preceding code, the two calls to `findLargest` both produce this error, because the type parameter `t` calls for a one-dimensional array, whereas the type arguments the compiler infers from the calls are a scalar (`testValue`) and a two-dimensional array (`testMatrix`).</span></span>  
  
 <span data-ttu-id="674af-111">**Identyfikator błędu:** BC32050</span><span class="sxs-lookup"><span data-stu-id="674af-111">**Error ID:** BC32050</span></span>
