# <a name="type-parameter-typeparametername-for-genericprocedurename-cannot-be-inferred"></a><span data-ttu-id="30ead-102">Parametr typu '\<typeparametername >' dla '\<genericprocedurename >' nie można wywnioskować</span><span class="sxs-lookup"><span data-stu-id="30ead-102">Type parameter '\<typeparametername>' for '\<genericprocedurename>' cannot be inferred</span></span>
<span data-ttu-id="30ead-103">Procedura ogólna jest wywoływana bez dostarczania listy argumentów typu, a wnioskowanie o typie kończy się niepowodzeniem dla jednego z argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="30ead-103">A generic procedure is called without supplying a type argument list, and type inference fails for one of the type arguments.</span></span>  
  
 <span data-ttu-id="30ead-104">Gdy wywoływana jest procedura ogólna, zwykle dostarczany jest argument typu dla każdego parametru typu zdefiniowanego przez procedurę.</span><span class="sxs-lookup"><span data-stu-id="30ead-104">When you call a generic procedure, you normally supply a type argument for each type parameter defined by the procedure.</span></span> <span data-ttu-id="30ead-105">Istnieje jednak alternatywa dla pomijania listy argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="30ead-105">However, you have the alternative of omitting the type argument list entirely.</span></span> <span data-ttu-id="30ead-106">Gdy to zrobisz, kompilator próbuje wywnioskować typ każdego argumentu typu z kontekstu wywołania.</span><span class="sxs-lookup"><span data-stu-id="30ead-106">When you do this, the compiler attempts to infer the type of each type argument from the context of your call.</span></span> <span data-ttu-id="30ead-107">Aby uzyskać więcej informacji, zobacz 'Wnioskowanie o typie' w [procedury ogólne w Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md).</span><span class="sxs-lookup"><span data-stu-id="30ead-107">For more information, see "Type Inference" in [Generic Procedures in Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md).</span></span>  
  
 <span data-ttu-id="30ead-108">Jedną z możliwych przyczyn niepowodzenia wnioskowania typu jest niezgodność rangi między parametrem typu a typem wywołującym.</span><span class="sxs-lookup"><span data-stu-id="30ead-108">One possible cause of type inference failure is a mismatch of rank between a type parameter and the calling type.</span></span> <span data-ttu-id="30ead-109">Ilustruje to poniższy kod.</span><span class="sxs-lookup"><span data-stu-id="30ead-109">The following code illustrates this.</span></span>  
  
```vb  
Public Sub displayLargest(Of t As IComparable)(ByVal arg() As t)  
    ' Insert code to find and display the largest element of arg().  
End Sub  
Public Sub callGenericSub()  
    Dim testValue As Integer  
    findLargest(testValue)  
    Dim testMatrix(,) As Integer  
    findLargest(testMatrix)  
End Sub  
```  
  
 <span data-ttu-id="30ead-110">W powyższym kodzie, dwa wywołania do `findLargest` powodują ten błąd, ponieważ parametr typu `t` wywołuje dla jednowymiarowej tablicy, podczas gdy typem argumentów kompilator wnioskuje z wywołań jest skalarna (`testValue`) i Dwuwymiarowa tablica (@no__ t-3).</span><span class="sxs-lookup"><span data-stu-id="30ead-110">In the preceding code, the two calls to `findLargest` both produce this error, because the type parameter `t` calls for a one-dimensional array, whereas the type arguments the compiler infers from the calls are a scalar (`testValue`) and a two-dimensional array (`testMatrix`).</span></span>  
  
 <span data-ttu-id="30ead-111">**Identyfikator błędu:** BC32050</span><span class="sxs-lookup"><span data-stu-id="30ead-111">**Error ID:** BC32050</span></span>
