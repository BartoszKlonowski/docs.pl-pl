<span data-ttu-id="54351-103">Literał deklaracja XML jest używany w wyrażeniu w lokalizacji, która jest niejednoznaczny dla kompilatora języka Visual Basic.</span><span class="sxs-lookup"><span data-stu-id="54351-103">An XML literal declaration is used in an expression in a location that is ambiguous to the Visual Basic compiler.</span></span> <span data-ttu-id="54351-104">Oznacza to, że kompilator języka Visual Basic nie można określić czy mniej-niż znak (<) ma służyć jako porównawczych operatora lub początek literał XML.</span><span class="sxs-lookup"><span data-stu-id="54351-104">That is, the Visual Basic compiler cannot determine whether the less-than character (<) is intended as a comparative operator or the start of an XML literal.</span></span> <span data-ttu-id="54351-105">Poniższy kod stanowi przykład:</span><span class="sxs-lookup"><span data-stu-id="54351-105">The following code provides an example:</span></span>  

```vb  
' Generates an error.

                  Where <sample>Value</sample> = "Value" _  
                  Select element  
```  
  
 <span data-ttu-id="54351-106">**Identyfikator błędu:** BC31198</span><span class="sxs-lookup"><span data-stu-id="54351-106">**Error ID:** BC31198</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="54351-107">Aby poprawić ten błąd</span><span class="sxs-lookup"><span data-stu-id="54351-107">To correct this error</span></span>

```vb  
                  Select element  
```
