# <a name="type-or-with-expected"></a><span data-ttu-id="ff7c6-102">Oczekiwano typu lub "with"</span><span class="sxs-lookup"><span data-stu-id="ff7c6-102">Type or 'With' expected</span></span>
<span data-ttu-id="ff7c6-103">Po zadeklarowaniu wystąpienia klasy słowo kluczowe `New` musi następować po nazwie typu lub `With`.</span><span class="sxs-lookup"><span data-stu-id="ff7c6-103">When you declare an instance of a class, the `New` keyword must be followed by a type name or by `With`.</span></span> <span data-ttu-id="ff7c6-104">Na przykład następujące instrukcje deklarują `client` jako wystąpienie klasy `Customer`.</span><span class="sxs-lookup"><span data-stu-id="ff7c6-104">For example, the following statements each declare `client` to be an instance of the `Customer` class.</span></span> <span data-ttu-id="ff7c6-105">Nazwa typu `Customer` następuje po `New`.</span><span class="sxs-lookup"><span data-stu-id="ff7c6-105">The type name `Customer` follows `New`.</span></span>  
  
```vb  
' Dim client As New Customer()  
' The next declaration uses an object initializer.  
Dim client As New Customer() With {.Name = "Litware, Inc."}  
```  
  
 <span data-ttu-id="ff7c6-106">Począwszy od Visual Basic 2008, można zadeklarować obiekt jako wystąpienie typu anonimowego, w tym przypadku nie określasz typu danych.</span><span class="sxs-lookup"><span data-stu-id="ff7c6-106">Beginning with Visual Basic 2008, you can declare an object to be an instance of an anonymous type, in which case you do not specify a data type.</span></span> <span data-ttu-id="ff7c6-107">W deklaracjach typu anonimowego słowo kluczowe `With` następuje `New`.</span><span class="sxs-lookup"><span data-stu-id="ff7c6-107">In anonymous type declarations, the keyword `With` follows `New`.</span></span>  
  
```vb  
Dim person = New With {.Name ="Mike Nash", .Age = 27}  
```
