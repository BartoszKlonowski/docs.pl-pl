# <a name="type-or-with-expected"></a><span data-ttu-id="57c3d-102">Oczekiwano typu lub "with"</span><span class="sxs-lookup"><span data-stu-id="57c3d-102">Type or 'With' expected</span></span>
<span data-ttu-id="57c3d-103">Gdy deklarujesz wystąpienie klasy, `New` słowo kluczowe musi następować po nazwie typu `With`lub.</span><span class="sxs-lookup"><span data-stu-id="57c3d-103">When you declare an instance of a class, the `New` keyword must be followed by a type name or by `With`.</span></span> <span data-ttu-id="57c3d-104">Na przykład następujące instrukcje deklarują `client` jako wystąpienie `Customer` klasy.</span><span class="sxs-lookup"><span data-stu-id="57c3d-104">For example, the following statements each declare `client` to be an instance of the `Customer` class.</span></span> <span data-ttu-id="57c3d-105">Nazwa `Customer` typu jest następująca `New`.</span><span class="sxs-lookup"><span data-stu-id="57c3d-105">The type name `Customer` follows `New`.</span></span>  
  
```  
' Dim client As New Customer()  
' The next declaration uses an object initializer.  
Dim client As New Customer() With {.Name = "Litware, Inc."}  
```  
  
 <span data-ttu-id="57c3d-106">Począwszy od Visual Basic 2008, można zadeklarować obiekt jako wystąpienie typu anonimowego, w tym przypadku nie określasz typu danych.</span><span class="sxs-lookup"><span data-stu-id="57c3d-106">Beginning with Visual Basic 2008, you can declare an object to be an instance of an anonymous type, in which case you do not specify a data type.</span></span> <span data-ttu-id="57c3d-107">W deklaracjach typu anonimowego słowo `With` kluczowe `New`jest następujące.</span><span class="sxs-lookup"><span data-stu-id="57c3d-107">In anonymous type declarations, the keyword `With` follows `New`.</span></span>  
  
```  
Dim person = New With {.Name ="Mike Nash", .Age = 27}  
```
