# <a name="type-or-with-expected"></a><span data-ttu-id="77dcb-102">Oczekiwano typu lub "with"</span><span class="sxs-lookup"><span data-stu-id="77dcb-102">Type or 'With' expected</span></span>
<span data-ttu-id="77dcb-103">Gdy deklarujesz wystąpienie klasy, `New` słowo kluczowe musi następować po nazwie typu lub `With` .</span><span class="sxs-lookup"><span data-stu-id="77dcb-103">When you declare an instance of a class, the `New` keyword must be followed by a type name or by `With`.</span></span> <span data-ttu-id="77dcb-104">Na przykład następujące instrukcje deklarują jako `client` wystąpienie `Customer` klasy.</span><span class="sxs-lookup"><span data-stu-id="77dcb-104">For example, the following statements each declare `client` to be an instance of the `Customer` class.</span></span> <span data-ttu-id="77dcb-105">Nazwa typu jest `Customer` następująca `New` .</span><span class="sxs-lookup"><span data-stu-id="77dcb-105">The type name `Customer` follows `New`.</span></span>  
  
```vb  
' Dim client As New Customer()  
' The next declaration uses an object initializer.  
Dim client As New Customer() With {.Name = "Litware, Inc."}  
```  
  
 <span data-ttu-id="77dcb-106">Począwszy od Visual Basic 2008, można zadeklarować obiekt jako wystąpienie typu anonimowego, w tym przypadku nie określasz typu danych.</span><span class="sxs-lookup"><span data-stu-id="77dcb-106">Beginning with Visual Basic 2008, you can declare an object to be an instance of an anonymous type, in which case you do not specify a data type.</span></span> <span data-ttu-id="77dcb-107">W deklaracjach typu anonimowego słowo kluczowe jest `With` następujące `New` .</span><span class="sxs-lookup"><span data-stu-id="77dcb-107">In anonymous type declarations, the keyword `With` follows `New`.</span></span>  
  
```vb  
Dim person = New With {.Name ="Mike Nash", .Age = 27}  
```  
  
 <span data-ttu-id="77dcb-108">**Identyfikator błędu:** BC30988</span><span class="sxs-lookup"><span data-stu-id="77dcb-108">**Error ID:** BC30988</span></span>
