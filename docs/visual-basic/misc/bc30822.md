# <a name="nullconstant-is-not-declared"></a>nie zadeklarowano "\<nullconstant >"
nie zadeklarowano "\<nullconstant >". Stała null nie jest już obsługiwana; Zamiast tego użyj elementu System. DBNull.  
  
 Instrukcja używa słowa kluczowego `Null`, które nie jest już obsługiwane w Visual Basic.  
  
 **Identyfikator błędu:** BC30822  
  
## <a name="to-correct-this-error"></a>Aby poprawić ten błąd  
  
1. Użyj <xref:System.DBNull> zamiast `Null`. Poniższy przykład ilustruje to.  
  
    ```vb  
    Sub TestDBNull()  
        Dim t As DataTable  
        ' Assume the DataGrid is bound to a DataTable.  
        t = CType(DataGrid1.DataSource, DataTable)  
        Dim r As DataRow  
        r = t.Rows(datagrid1.CurrentCell.RowNumber)  
        r.BeginEdit  
        r(1) = System.DBNull.Value ' Assign DBNull to the record.  
        r.EndEdit  
        r.AcceptChanges  
        If r.IsNull(1) Then  
            MsgBox("")  
        End If  
    End Sub  
    ```  
  
2. Użyj słowa kluczowego [Nothing](../../visual-basic/language-reference/nothing.md) do przypisań i porównań w przypadku używania zmiennych obiektu. Poniższy przykład ilustruje to.  
  
    ```vb  
    Sub TestNothing()  
        Dim cls As Object  
        ' cls is Nothing if it has not been assigned using the New keyword.  
        If (cls Is Nothing) Then  
            MsgBox("cls is Nothing")  
        End If  
        cls = Nothing ' Assign Nothing to the class variable cls.  
    End Sub  
    ```
