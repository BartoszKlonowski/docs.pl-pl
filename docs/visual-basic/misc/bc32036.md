<span data-ttu-id="829b5-103">Klasa pochodna nie deklaruje konstruktora, a Visual Basic nie może wygenerować elementu, ponieważ nie może określić, który Konstruktor klasy bazowej ma zostać wywołany.</span><span class="sxs-lookup"><span data-stu-id="829b5-103">A derived class does not declare a constructor, and Visual Basic cannot generate one because it cannot determine which base class constructor to call.</span></span>  
  
 <span data-ttu-id="829b5-104">Gdy Klasa pochodna nie deklaruje konstruktora, Visual Basic próbuje wygenerować niejawnego konstruktora bez parametrów, który wywołuje `MyBase.New()` .</span><span class="sxs-lookup"><span data-stu-id="829b5-104">When a derived class does not declare a constructor, Visual Basic attempts to generate an implicit parameterless constructor that calls `MyBase.New()`.</span></span> <span data-ttu-id="829b5-105">Jeśli w klasie bazowej nie ma dostępnego konstruktora, który można wywołać bez argumentów lub jeśli istnieje więcej niż jeden, Visual Basic nie może wygenerować niejawnego konstruktora.</span><span class="sxs-lookup"><span data-stu-id="829b5-105">If there is no accessible constructor in the base class that can be called without arguments, or if there is more than one, Visual Basic cannot generate an implicit constructor.</span></span>  
  
 <span data-ttu-id="829b5-106">Taka sytuacja może wystąpić, na przykład jeśli jeden Konstruktor klasy bazowej ma jeden argument, `Optional` a drugi ma jeden `ParamArray` argument.</span><span class="sxs-lookup"><span data-stu-id="829b5-106">This situation can arise, for example, if one base class constructor has a single `Optional` argument and another has a single `ParamArray` argument.</span></span> <span data-ttu-id="829b5-107">Każdy z tych elementów można wywołać bez argumentów.</span><span class="sxs-lookup"><span data-stu-id="829b5-107">Each of these can be called with no arguments.</span></span>
