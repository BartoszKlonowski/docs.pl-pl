 <span data-ttu-id="5f369-105">Metody rozszerzenia bez parametrów jest nieprawidłowy, ponieważ pierwszy parametr określa typ danych, które rozszerza metoda.</span><span class="sxs-lookup"><span data-stu-id="5f369-105">An extension method without parameters is not valid because the first parameter specifies which data type the method extends.</span></span> <span data-ttu-id="5f369-106">Pierwszy parametr jest powiązany do wystąpienia typu danych, który wywołuje tę metodę.</span><span class="sxs-lookup"><span data-stu-id="5f369-106">The first parameter is bound to the instance of the data type that invokes the method.</span></span>

## <a name="example"></a><span data-ttu-id="5f369-110">Przykład</span><span class="sxs-lookup"><span data-stu-id="5f369-110">Example</span></span>  
 <span data-ttu-id="5f369-111">Pierwszy parametr w następującym przykładzie wskazuje, że `Print` rozszerza metoda `String` typu danych.</span><span class="sxs-lookup"><span data-stu-id="5f369-111">The first parameter in the following example indicates that the `Print` method extends the `String` data type.</span></span>  
  
```  
<Extension()> _  
Public Sub Print (ByVal str As String)  
    Console.WriteLine(str)  
End Sub  
```  
  
 <span data-ttu-id="5f369-112">Gdy metoda rozszerzająca zostanie w następujący sposób, wywołana parametr `str` w metodzie jest powiązany z `greeting`, wystąpienie `String` wywołująca `Print`.</span><span class="sxs-lookup"><span data-stu-id="5f369-112">When the extension method is called as follows, parameter `str` in the method is bound to `greeting`, the instance of `String` that calls `Print`.</span></span> <span data-ttu-id="5f369-113">Kompilator będzie używał `greeting` jako argument do metody rozszerzenia `Print`.</span><span class="sxs-lookup"><span data-stu-id="5f369-113">The compiler will use `greeting` as the argument to extension method `Print`.</span></span>  
  
```  
Dim greeting As String = "Hello"  
greeting.Print()  
```
