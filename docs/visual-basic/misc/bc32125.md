# <a name="cannot-implement-interfacename1membername-because-its-implementation-could-conflict-with-the-implementation-for-interfacename2membername-for-some-type-arguments"></a><span data-ttu-id="65ceb-102">Nie można zaimplementować elementu " \<interfacename1> . \<membername> ", ponieważ jego implementacja może powodować konflikt z implementacją dla elementu " \<interfacename2> . \<membername> " w przypadku niektórych argumentów typu</span><span class="sxs-lookup"><span data-stu-id="65ceb-102">Cannot implement '\<interfacename1>.\<membername>' because its implementation could conflict with the implementation for '\<interfacename2>.\<membername>' for some type arguments</span></span>
<span data-ttu-id="65ceb-103">Klasa implementuje więcej niż jeden interfejs ogólny, jeden z dziedziczy z innego, a dwie implementacje elementu członkowskiego interfejsu mogą powodować konflikt dla określonych wartości argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="65ceb-103">A class implements more than one generic interface, one of which inherits from another, and two implementations of an interface member could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="65ceb-104">Poniższe instrukcje mogą generować ten błąd.</span><span class="sxs-lookup"><span data-stu-id="65ceb-104">The following statements can generate this error.</span></span>  
  
```vb  
Public Interface iFace1(Of t)  
    Sub testSub()  
End Interface  
Public Interface iFace2(Of u)  
    Inherits iFace1(Of u)  
End Interface  
Public Class testClass(Of y, z)  
    Implements iFace1(Of y), iFace2(Of z)  
    Public Sub testSuby() Implements iFace1(Of y).testSub  
    End Sub  
    Public Sub testSubz() Implements iFace1(Of z).testSub  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="65ceb-105">Ponieważ `iFace2` dziedziczy po `iFace1` użyciu własnego parametru typu ( `u` ), `testClass` implementuje dwie wersje `iFace1.testSub` z identycznymi sygnaturami, jeśli ten sam argument typu został przekazano do `y` i `z` .</span><span class="sxs-lookup"><span data-stu-id="65ceb-105">Because `iFace2` inherits from `iFace1` using its own type parameter (`u`), `testClass` would implement two versions of `iFace1.testSub` with identical signatures if the same type argument were passed to `y` and `z`.</span></span> <span data-ttu-id="65ceb-106">Spowoduje to powstanie niejednoznaczności, do której wersji ma dostęp.</span><span class="sxs-lookup"><span data-stu-id="65ceb-106">This would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="65ceb-107">**Identyfikator błędu:** BC32125</span><span class="sxs-lookup"><span data-stu-id="65ceb-107">**Error ID:** BC32125</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="65ceb-108">Aby poprawić ten błąd</span><span class="sxs-lookup"><span data-stu-id="65ceb-108">To correct this error</span></span>  
  
- <span data-ttu-id="65ceb-109">Zmień strukturę dziedziczenia interfejsów, tak aby nie `iFace1` mogła być implementowana przy użyciu dwóch różnych argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="65ceb-109">Change the inheritance structure of the interfaces so that `iFace1` could not be implemented with two different type arguments.</span></span>  
  
     <span data-ttu-id="65ceb-110">-lub-</span><span class="sxs-lookup"><span data-stu-id="65ceb-110">-or-</span></span>  
  
- <span data-ttu-id="65ceb-111">Usuń z `Implements` instrukcji jeden z interfejsów powodujących konflikt implementacji.</span><span class="sxs-lookup"><span data-stu-id="65ceb-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
