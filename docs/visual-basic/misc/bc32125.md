# <a name="cannot-implement-ltinterfacename1gtltmembernamegt-because-its-implementation-could-conflict-with-the-implementation-for-ltinterfacename2gtltmembernamegt-for-some-type-arguments"></a><span data-ttu-id="ba46b-102">Nie może implementować "&lt;interfacename1&gt;.&lt; membername&gt;"ponieważ jego implementacja może powodować konflikt z implementacją dla"&lt;interfacename2&gt;.&lt; membername&gt;"dla niektórych argumentów typu</span><span class="sxs-lookup"><span data-stu-id="ba46b-102">Cannot implement '&lt;interfacename1&gt;.&lt;membername&gt;' because its implementation could conflict with the implementation for '&lt;interfacename2&gt;.&lt;membername&gt;' for some type arguments</span></span>
<span data-ttu-id="ba46b-103">Klasa implementuje więcej niż jeden interfejs ogólny, jednym z nich dziedziczy od drugiej, a dwie implementacje elementu członkowskiego interfejsu mogą powodować konflikt dla niektórych wartości argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="ba46b-103">A class implements more than one generic interface, one of which inherits from another, and two implementations of an interface member could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="ba46b-104">Poniższe instrukcje może wygenerować tego błędu.</span><span class="sxs-lookup"><span data-stu-id="ba46b-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface iFace1(Of t)  
    Sub testSub()  
End Interface  
Public Interface iFace2(Of u)  
    Inherits iFace1(Of u)  
End Interface  
Public Class testClass(Of y, z)  
    Implements iFace1(Of y), iFace2(Of z)  
    Public Sub testSuby() Implements iFace1(Of y).testSub  
    End Sub  
    Public Sub testSubz() Implements iFace1(Of z).testSub  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="ba46b-105">Ponieważ `iFace2` dziedziczy `iFace1` za pomocą parametru swój własny typ (`u`), `testClass` zaimplementować dwie wersje `iFace1.testSub` o identycznych podpisach, jeśli ten sam argument typu zostały przekazane do `y` i `z`.</span><span class="sxs-lookup"><span data-stu-id="ba46b-105">Because `iFace2` inherits from `iFace1` using its own type parameter (`u`), `testClass` would implement two versions of `iFace1.testSub` with identical signatures if the same type argument were passed to `y` and `z`.</span></span> <span data-ttu-id="ba46b-106">To dałby w efekcie niejednoznaczności dotyczące dostępu do wersji.</span><span class="sxs-lookup"><span data-stu-id="ba46b-106">This would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="ba46b-107">**Identyfikator błędu:** BC32125</span><span class="sxs-lookup"><span data-stu-id="ba46b-107">**Error ID:** BC32125</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="ba46b-108">Aby poprawić ten błąd</span><span class="sxs-lookup"><span data-stu-id="ba46b-108">To correct this error</span></span>  
  
-   <span data-ttu-id="ba46b-109">Zmienić strukturę dziedziczenie, interfejsy, aby `iFace1` nie mogły być wykonane w przypadku dwóch argumentów innego typu.</span><span class="sxs-lookup"><span data-stu-id="ba46b-109">Change the inheritance structure of the interfaces so that `iFace1` could not be implemented with two different type arguments.</span></span>  
  
     <span data-ttu-id="ba46b-110">—lub—</span><span class="sxs-lookup"><span data-stu-id="ba46b-110">-or-</span></span>  
  
-   <span data-ttu-id="ba46b-111">Usuń z `Implements` instrukcji interfejsów powodujące konflikt wdrażania.</span><span class="sxs-lookup"><span data-stu-id="ba46b-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
