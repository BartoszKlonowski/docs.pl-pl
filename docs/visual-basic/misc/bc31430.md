<span data-ttu-id="ef56e-103">A `TypeOf` ... `Is` wyrażenie testuje zmienną odwołania do obiektu dla typu danych, którego nie może przechowywać.</span><span class="sxs-lookup"><span data-stu-id="ef56e-103">A `TypeOf`...`Is` expression tests an object reference variable to a data type that it cannot hold.</span></span>  
  
 <span data-ttu-id="ef56e-104">W niektórych przypadkach kompilator może określić, że `TypeOf` ... `Is` test może się nie powieść, jeśli na przykład nie istnieje relacja dziedziczenia między dwiema klasami.</span><span class="sxs-lookup"><span data-stu-id="ef56e-104">In some cases the compiler can determine that a `TypeOf`...`Is` test can only fail, for example if there is no inheritance relationship between two classes.</span></span>  
  
 <span data-ttu-id="ef56e-105">Poniższy kod może wygenerować ten błąd.</span><span class="sxs-lookup"><span data-stu-id="ef56e-105">The following code can generate this error.</span></span>  
  
 `Dim refVar as System.Windows.Forms.Form`  
  
 `If TypeOf refVar Is System.Array`  
  
 `End If`  
  
 <span data-ttu-id="ef56e-106">Ponieważ <xref:System.Windows.Forms.Form> i <xref:System.Array> są całkowicie niepowiązane typy, kompilator może określić, że `TypeOf` ... `Is` wyrażenie zwraca `False` dla dowolnej wartości `refVar` .</span><span class="sxs-lookup"><span data-stu-id="ef56e-106">Because <xref:System.Windows.Forms.Form> and <xref:System.Array> are totally unrelated types, the compiler can determine that the `TypeOf`...`Is` expression returns `False` for any value of `refVar`.</span></span>
