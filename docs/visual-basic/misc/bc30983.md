# <a name="type-of-variablename-is-ambiguous-because-the-loop-bounds-and-the-step-variable-do-not-widen-to-the-same-type"></a>Typ 'variablename\<>' jest niejednoznaczny, ponieważ granice pętli i zmienna kroku nie rozszerzają się do tego samego typu
Kod zawiera `For...Next` pętlę, w której kompilator nie może wywnioskować typu danych dla zmiennej sterującej pętli, ponieważ spełnione są następujące warunki:  
  
- Typ danych zmiennej sterującej pętli nie `As` jest określony za pomocą klauzuli.  
  
- Granice pętli i zmienna kroku zawierają co najmniej dwa typy danych.  
  
- Istnieje więcej niż jedna możliwa konwersja między typami danych.  
  
- Nie ma najlepszego typu wśród kandydatów, dzięki czemu wybór typu dla zmiennej sterującej pętli jest niejednoznaczny.  
  
 Na przykład następująca pętla ma `Integer` jedną pętlę związaną typu i jedną pętlę związaną typu: `UInteger`  
  
```vb  
Dim m As Integer = 1  
Dim n As UInteger = 10  
' Not valid.  
' For i = m To n  
    ' Loop processing.  
' Next  
```  
  
 Istnieje możliwość konwersji `Integer` `UInteger`między i , `UInteger` i `Integer`możliwe konwersji między i , ale oba są zawężenia konwersji, więc nie jest najlepszym wyborem.  
  
 W następnym przykładzie dodaje się `Double` trzecią zmienną typu. Oba `Integer` `UInteger` i mają standardowe konwersje poszerzające `Double` `Double` do , co sprawia, że konwersja do najlepszego wyboru. Wnioskowanie o typie przypisuje `i` do `Double`zmiennej sterującej pętlą typu danych .  
  
```vb  
Dim stepVar As Double = 1  
' Valid.  
For i = m To n Step stepVar  
    ' Loop processing.  
Next  
```  
  
 **Identyfikator błędu:** Bc30983  
  
## <a name="to-correct-this-error"></a>Aby poprawić ten błąd  
  
- Jawnie przekonwertować jedną ze zmiennych na typ, do który inne mają konwersję rozszerzającą, na przykład:  
  
    ```vb  
    For i = m To CLng(n)  
    ```  
  
- Użyj `As` klauzuli, aby określić typ zmiennej sterującej:  
  
    ```vb  
    For i As Double = m To n
    ```
