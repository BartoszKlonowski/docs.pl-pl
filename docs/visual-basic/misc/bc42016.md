# <a name="implicit-conversion-from-39lttypename1gt39-to-39lttypename2gt39"></a>Niejawna konwersja z &#39; &lt;typename1&gt; &#39; do &#39; &lt;typename2&gt;&#39;
Wyrażenia lub instrukcji przypisania przyjmuje wartość jednego typu danych i konwertuje ją do innego typu. Ponieważ są używane nie — słowo kluczowe konwersji, konwersja jest *niejawne*.  
  
 Domyślnie ten komunikat jest ostrzeżenie. Aby uzyskać informacje na ukrywanie ostrzeżenia lub traktowanie ostrzeżeń jako błędy, zobacz [Konfigurowanie ostrzeżeń w Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).  
  
 **Identyfikator błędu:** BC42016  
  
## <a name="to-correct-this-error"></a>Aby poprawić ten błąd  
  
-   Jeśli to możliwe należy używać wartości tego samego typu danych, dlatego w Visual Basic nie trzeba wykonać żadnej konwersji.  
  
-   Użyj `CType` lub inne słowa kluczowe konwersji tak, aby konwersji *jawne*.
