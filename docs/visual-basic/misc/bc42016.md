# <a name="implicit-conversion-from-typename1-to-typename2"></a>Niejawna konwersja z "\<typename1 >" do "\<typename2 >"
Wyrażenia lub instrukcji przypisania przyjmuje wartość jednego typu danych i konwertuje je do innego typu. Ponieważ są używane nie słowo kluczowe konwersji, konwersja jest *niejawne*.  
  
 Domyślnie ta wiadomość jest ostrzeżenie. Uzyskać informacje o ukrywaniu ostrzeżenia lub traktowanie ostrzeżeń jako błędy, zobacz [Konfigurowanie ostrzeżeń w języku Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).  
  
 **Identyfikator błędu:** BC42016  
  
## <a name="to-correct-this-error"></a>Aby poprawić ten błąd  
  
-   Jeśli to możliwe należy użyć wartości tego samego typu danych, więc języka Visual Basic nie trzeba wykonać żadnych konwersji.  
  
-   Użyj `CType` lub inne słowa kluczowe konwersji tak, że konwersja jest *jawne*.
