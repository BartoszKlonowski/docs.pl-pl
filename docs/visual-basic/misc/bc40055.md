# <a name="casing-of-namespace-name-39ltnamespacename1gt39-does-not-match-casing-of-namespace-name-39ltnamespacename2gt39-in-file-39ltfilepathgt39"></a>Wielkość liter nazwy przestrzeni nazw &#39; &lt;namespacename1&gt;&#39; jest niezgodna wielkość liter nazwy przestrzeni nazw &#39;&lt; namespacename2&gt;&#39; w pliku &#39;&lt; FilePath&gt;&#39;
Obszar nazw występuje więcej niż raz w projekcie, ale z inną osłony.  
  
 *Wielkość liter* odwołuje się do użycia wielkich i małych znaków nazwy elementu programistycznego. Visual Basic nie uwzględnia wielkości liter, ale środowisko uruchomieniowe języka wspólnego (CLR) jest rozróżniana wielkość liter. Aby uzyskać więcej informacji, zobacz "Case liter w nazwach" w [zadeklarowane nazwy elementów](../../visual-basic/programming-guide/language-features/declared-elements/declared-element-names.md).  
  
 Domyślnie ten komunikat jest ostrzeżenie. Aby uzyskać informacje na ukrywanie ostrzeżenia lub traktowanie ostrzeżeń jako błędy, zobacz [Konfigurowanie ostrzeżeń w Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).  
  
 **Identyfikator błędu:** BC40055  
  
## <a name="to-correct-this-error"></a>Aby poprawić ten błąd  
  
-   Ze względów należy zawsze używać identycznych małych i wielkich liter w każde odwołanie do przestrzeni nazw. Może to spowodować błędnej interpretacji przez środowisko uruchomieniowe języka wspólnego.
