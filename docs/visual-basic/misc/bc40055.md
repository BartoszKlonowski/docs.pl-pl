# <a name="casing-of-namespace-name-namespacename1-does-not-match-casing-of-namespace-name-namespacename2-in-file-filepath"></a>Wielkość liter nazwy przestrzeni nazw '\<namespacename1 >' jest niezgodna z wielkością liter nazwy przestrzeni nazw '\<namespacename2 >' w pliku'\<ścieżka_pliku > '
Obszar nazw występuje więcej niż raz w projekcie, ale z inną liter.  
  
 *Wielkość liter* odwołuje się do używania wielkie i małe znaki w nazwie elementu programistycznego. Visual Basic jest rozróżniana wielkość liter, ale środowisko uruchomieniowe języka wspólnego (CLR) jest uwzględniana wielkość liter. Aby uzyskać więcej informacji, zobacz 'Case czułości w nazwy' w [zadeklarowane nazwy elementów](../../visual-basic/programming-guide/language-features/declared-elements/declared-element-names.md).  
  
 Domyślnie ta wiadomość jest ostrzeżenie. Uzyskać informacje o ukrywaniu ostrzeżenia lub traktowanie ostrzeżeń jako błędy, zobacz [Konfigurowanie ostrzeżeń w języku Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).  
  
 **Identyfikator błędu:** BC40055  
  
## <a name="to-correct-this-error"></a>Aby poprawić ten błąd  
  
- Ze względów należy zawsze używać identycznych małych i wielkich liter w każde odwołanie do przestrzeni nazw. Może to spowodować błędnej interpretacji przez środowisko uruchomieniowe języka wspólnego.
