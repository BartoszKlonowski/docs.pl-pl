# <a name="use-command-line-option-option-or-appropriate-project-settings-instead-of-parameter"></a>Użyj opcji wiersza polecenia '\<opcja >' lub odpowiednich ustawień projektu zamiast elementu '\<parametr >'
Preferowanym sposobem określenia pliku zawierającego klucz publiczny dla zestawu, kontenera klucza publicznego dla zestawu lub zestawu podpisanego częściowo jest użycie opcji kompilatora Visual Basic. Nie zaleca się używania <xref:System.Reflection.AssemblyKeyFileAttribute>atrybutów, <xref:System.Reflection.AssemblyKeyNameAttribute>, ani <xref:System.Reflection.AssemblyDelaySignAttribute> w kodzie.  
  
 **Identyfikator błędu:** BC41008  
  
## <a name="to-correct-this-error"></a>Aby poprawić ten błąd  
  
1. Użyj <xref:System.Reflection.AssemblyKeyNameAttribute>opcji [kompilatora/keyfile](../../visual-basic/reference/command-line-compiler/keyfile.md), [/keycontainer](../../visual-basic/reference/command-line-compiler/keycontainer.md)lub Visual Basic [/delaysign](../../visual-basic/reference/command-line-compiler/delaysign.md), zamiast atrybutów <xref:System.Reflection.AssemblyDelaySignAttribute>,lub w kodzie. <xref:System.Reflection.AssemblyKeyFileAttribute>
