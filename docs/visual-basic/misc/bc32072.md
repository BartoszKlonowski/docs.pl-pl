# <a name="cannot-implement-interface-interfacename1-because-its-implementation-could-conflict-with-the-implementation-of-another-implemented-interface-interfacename2-for-some-type-arguments"></a><span data-ttu-id="de7a0-102">Nie można zaimplementować interfejsu '\<interfacename1 >', ponieważ jego implementacja może powodować konflikt z implementacją innego zaimplementowanego interfejsu '\<interfacename2 >' dla niektórych argumentów typu</span><span class="sxs-lookup"><span data-stu-id="de7a0-102">Cannot implement interface '\<interfacename1>' because its implementation could conflict with the implementation of another implemented interface '\<interfacename2>' for some type arguments</span></span>

<span data-ttu-id="de7a0-103">Deklaracja klasy zawiera `Implements` instrukcję, która określa dwa lub więcej interfejsów, ale co najmniej jeden z interfejsów jest ogólny, a dwie implementacje mogą powodować konflikt dla określonych wartości argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="de7a0-103">A class declaration includes an `Implements` statement that specifies two or more interfaces, but at least one of the interfaces is generic and two of the implementations could conflict for certain values of type arguments.</span></span>

<span data-ttu-id="de7a0-104">Poniższe instrukcje mogą generować ten błąd.</span><span class="sxs-lookup"><span data-stu-id="de7a0-104">The following statements can generate this error.</span></span>

```vb
Public Interface iFace1
    Sub testSub(ByVal arg As String)
End Interface
Public Interface iFace2(Of t)
    Sub testSub(ByVal arg As t)
End Interface
Public Class testClass
    Implements iFace1, iFace2(Of String)
End Class
```

<span data-ttu-id="de7a0-105">Ponieważ `iFace2` jest zbudowany przy `String`użyciu `testClass` , należy zaimplementować dwie wersje `testSub` o identycznych sygnaturach.</span><span class="sxs-lookup"><span data-stu-id="de7a0-105">Because `iFace2` is constructed using `String`, `testClass` must implement two versions of `testSub` with identical signatures.</span></span> <span data-ttu-id="de7a0-106">Wykonanie tej operacji spowodowałoby niejednoznaczność, do której wersji ma dostęp.</span><span class="sxs-lookup"><span data-stu-id="de7a0-106">Doing so would produce an ambiguity about which version to access.</span></span>

<span data-ttu-id="de7a0-107">**Identyfikator błędu:** BC32072</span><span class="sxs-lookup"><span data-stu-id="de7a0-107">**Error ID:** BC32072</span></span>

## <a name="to-correct-this-error"></a><span data-ttu-id="de7a0-108">Aby poprawić ten błąd</span><span class="sxs-lookup"><span data-stu-id="de7a0-108">To correct this error</span></span>

- <span data-ttu-id="de7a0-109">Zmień argument typu dostarczony do interfejsu generycznego, aby nie wystąpił konflikt.</span><span class="sxs-lookup"><span data-stu-id="de7a0-109">Change the type argument supplied to the generic interface so that there is no conflict.</span></span>

  <span data-ttu-id="de7a0-110">\-oraz</span><span class="sxs-lookup"><span data-stu-id="de7a0-110">\-or-</span></span>

- <span data-ttu-id="de7a0-111">Usuń z `Implements` instrukcji jeden z interfejsów powodujących konflikt implementacji.</span><span class="sxs-lookup"><span data-stu-id="de7a0-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
