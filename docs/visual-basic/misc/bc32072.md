# <a name="cannot-implement-interface-39ltinterfacename1gt39-because-its-implementation-could-conflict-with-the-implementation-of-another-implemented-interface-39ltinterfacename2gt39-for-some-type-arguments"></a><span data-ttu-id="7133e-102">Nie można zaimplementować interfejsu &#39; &lt;interfacename1&gt;&#39; ponieważ jego implementacja może powodować konflikt z implementacją innego zaimplementowanego interfejsu &#39;&lt; interfacename2&gt;&#39; dla niektórych argumentów typu</span><span class="sxs-lookup"><span data-stu-id="7133e-102">Cannot implement interface &#39;&lt;interfacename1&gt;&#39; because its implementation could conflict with the implementation of another implemented interface &#39;&lt;interfacename2&gt;&#39; for some type arguments</span></span>
<span data-ttu-id="7133e-103">Deklaracja klasy zawiera `Implements` instrukcji, która określa dwa lub więcej interfejsów, ale co najmniej jednego z interfejsów jest rodzajowa i dwa implementacje może powodować konflikt niektórych wartości argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="7133e-103">A class declaration includes an `Implements` statement that specifies two or more interfaces, but at least one of the interfaces is generic and two of the implementations could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="7133e-104">Poniższe instrukcje może spowodować wygenerowanie tego błędu.</span><span class="sxs-lookup"><span data-stu-id="7133e-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface iFace1  
    Sub testSub(ByVal arg As String)  
End Interface  
Public Interface iFace2(Of t)  
    Sub testSub(ByVal arg As t)  
End Interface  
Public Class testClass  
    Implements iFace1, iFace2(Of String)  
End Class  
```  
  
 <span data-ttu-id="7133e-105">Ponieważ `iFace2` jest tworzony przy użyciu `String`, `testClass` musi implementować dwie wersje `testSub` o identycznych podpisach.</span><span class="sxs-lookup"><span data-stu-id="7133e-105">Because `iFace2` is constructed using `String`, `testClass` must implement two versions of `testSub` with identical signatures.</span></span> <span data-ttu-id="7133e-106">Dzięki temu dałby w efekcie niejednoznaczności wersji do uzyskania dostępu.</span><span class="sxs-lookup"><span data-stu-id="7133e-106">Doing so would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="7133e-107">**Identyfikator błędu:** BC32072</span><span class="sxs-lookup"><span data-stu-id="7133e-107">**Error ID:** BC32072</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="7133e-108">Aby poprawić ten błąd</span><span class="sxs-lookup"><span data-stu-id="7133e-108">To correct this error</span></span>  
  
-   <span data-ttu-id="7133e-109">Zmiana argumentu typu dostarczonego do ogólnego interfejsu, tak aby nie było konfliktu.</span><span class="sxs-lookup"><span data-stu-id="7133e-109">Change the type argument supplied to the generic interface so that there is no conflict.</span></span>  
  
     <span data-ttu-id="7133e-110">—lub—</span><span class="sxs-lookup"><span data-stu-id="7133e-110">-or-</span></span>
