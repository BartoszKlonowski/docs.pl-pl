# <a name="cannot-implement-interface-interfacename1-because-its-implementation-could-conflict-with-the-implementation-of-another-implemented-interface-interfacename2-for-some-type-arguments"></a><span data-ttu-id="e62dd-102">Nie można zaimplementować interfejsu "\<interfacename1 >", ponieważ jego implementacja może powodować konflikt z implementacją innego zaimplementowanego interfejsu "\<interfacename2 >" dla niektórych argumentów typu</span><span class="sxs-lookup"><span data-stu-id="e62dd-102">Cannot implement interface '\<interfacename1>' because its implementation could conflict with the implementation of another implemented interface '\<interfacename2>' for some type arguments</span></span>
<span data-ttu-id="e62dd-103">Deklaracja klasy zawiera `Implements` instrukcję, która określa dwie lub więcej interfejsów, ale co najmniej jednego z interfejsów ogólnych i dwie implementacje mogą powodować konflikt dla niektórych wartości argumentów typu.</span><span class="sxs-lookup"><span data-stu-id="e62dd-103">A class declaration includes an `Implements` statement that specifies two or more interfaces, but at least one of the interfaces is generic and two of the implementations could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="e62dd-104">Poniższe instrukcje może wygenerować tego błędu.</span><span class="sxs-lookup"><span data-stu-id="e62dd-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface iFace1  
    Sub testSub(ByVal arg As String)  
End Interface  
Public Interface iFace2(Of t)  
    Sub testSub(ByVal arg As t)  
End Interface  
Public Class testClass  
    Implements iFace1, iFace2(Of String)  
End Class  
```  
  
 <span data-ttu-id="e62dd-105">Ponieważ `iFace2` jest tworzony przy użyciu `String`, `testClass` musi implementować dwie wersje `testSub` o identycznych podpisach.</span><span class="sxs-lookup"><span data-stu-id="e62dd-105">Because `iFace2` is constructed using `String`, `testClass` must implement two versions of `testSub` with identical signatures.</span></span> <span data-ttu-id="e62dd-106">Ten sposób dałby w efekcie niejednoznaczności dotyczące dostępu do wersji.</span><span class="sxs-lookup"><span data-stu-id="e62dd-106">Doing so would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="e62dd-107">**Identyfikator błędu:** BC32072</span><span class="sxs-lookup"><span data-stu-id="e62dd-107">**Error ID:** BC32072</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="e62dd-108">Aby poprawić ten błąd</span><span class="sxs-lookup"><span data-stu-id="e62dd-108">To correct this error</span></span>  
  
-   <span data-ttu-id="e62dd-109">Zmień argument typu dostarczane do ogólny interfejs, tak aby nie występuje konflikt.</span><span class="sxs-lookup"><span data-stu-id="e62dd-109">Change the type argument supplied to the generic interface so that there is no conflict.</span></span>  
  
     <span data-ttu-id="e62dd-110">—lub—</span><span class="sxs-lookup"><span data-stu-id="e62dd-110">-or-</span></span>  
  
-   <span data-ttu-id="e62dd-111">Usuń z `Implements` instrukcji interfejsów powodujące konflikt wdrażania.</span><span class="sxs-lookup"><span data-stu-id="e62dd-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
