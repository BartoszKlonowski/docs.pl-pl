<span data-ttu-id="ff821-103">Wyrażenie uzyskuje dostęp do elementu programowania definicją klasy, struktury, modułu lub interfejsu, który zawiera więcej niż jeden element członkowski o takiej samej nazwie.</span><span class="sxs-lookup"><span data-stu-id="ff821-103">An expression accesses a programming element defined in a class, structure, module, or interface that contains more than one member with the same name.</span></span>

 <span data-ttu-id="ff821-107">Jednak innych języków, takich jak [ C# ](../../csharp/index.md) i [Visual C++](/cpp/index), jest rozróżniana wielkość liter.</span><span class="sxs-lookup"><span data-stu-id="ff821-107">However, other languages, such as [C#](../../csharp/index.md) and [Visual C++](/cpp/index), are case-sensitive.</span></span> <span data-ttu-id="ff821-108">W takich językach `XYZ` i `xyz` nie są uznawane za takiej samej nazwie.</span><span class="sxs-lookup"><span data-stu-id="ff821-108">In such a language, `XYZ` and `xyz` are not considered to be the same name.</span></span> <span data-ttu-id="ff821-109">W związku z tym, klasa napisane w takich językach zdefiniować zmienną o nazwie `XYZ` i właściwość o nazwie `xyz`.</span><span class="sxs-lookup"><span data-stu-id="ff821-109">Therefore, a class written in such a language could define a variable named `XYZ` and a property named `xyz`.</span></span> <span data-ttu-id="ff821-110">Środowisko uruchomieniowe języka wspólnego (CLR) zachowuje rozróżnianie wielkości liter w zestawach.</span><span class="sxs-lookup"><span data-stu-id="ff821-110">The common language runtime (CLR) preserves case sensitivity in assemblies.</span></span> <span data-ttu-id="ff821-111">Jednak jeśli aplikacji Visual Basic uzyskuje dostęp do zestawu z nazwami `XYZ` i `xyz`, pojawiają się jako tej samej nazwie.</span><span class="sxs-lookup"><span data-stu-id="ff821-111">However, if a Visual Basic application accesses an assembly with names `XYZ` and `xyz`, they appear as the same name.</span></span>
