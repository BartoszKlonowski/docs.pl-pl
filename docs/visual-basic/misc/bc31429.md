# <a name="elementname-is-ambiguous-because-multiple-kinds-of-members-with-this-name-exist-in-type-typename"></a>element "\<ElementName >" jest niejednoznaczny, ponieważ wiele rodzajów składowych o tej nazwie istnieje w \<typie > "\<TypeName >"
Wyrażenie uzyskuje dostęp do elementu programowania zdefiniowanego w klasie, strukturze, module lub interfejsie, który zawiera więcej niż jeden element członkowski o tej samej nazwie.  
  
 Najbardziej prawdopodobną przyczyną tego błędu jest *uwzględnianie wielkości liter*. W nazwach Visual Basic nie jest rozróżniana wielkość liter, co oznacza, że można je inaczej różnić w różnych miejscach w kodzie. Na przykład w przypadku zdefiniowania zmiennej o nazwie `XYZ` i późniejszej dostępu do niej jako `xyz`, kompilator traktuje dwie nazwy jako równoważne.  
  
 Jednak w innych językach, takich jak [C#](../../csharp/index.yml) i [wizualizacji C++ ](/cpp/index), jest rozróżniana wielkość liter. W takim języku `XYZ` i `xyz` nie są traktowane jako takie same nazwy. W związku z tym Klasa zapisywana w tym języku może zdefiniować zmienną o nazwie `XYZ` i właściwość o nazwie `xyz`. Środowisko uruchomieniowe języka wspólnego (CLR) zachowuje czułość wielkości liter w zestawach. Jeśli jednak aplikacja Visual Basic uzyskuje dostęp do zestawu z nazwami `XYZ` i `xyz`, pojawiają się one jako o tej samej nazwie.  
  
 **Identyfikator błędu:** BC31429  
  
## <a name="to-correct-this-error"></a>Aby poprawić ten błąd  
  
1. Jeśli masz kontrolę nad kodem źródłowym typu definiującego, Rozważ zmianę nazwy elementów członkowskich, tak aby różnią się tylko wielkością liter. Może to nie być możliwe, jeśli typ definiujący został już opublikowany i jest używany przez inne aplikacje.  
  
2. Jeśli nie można zmienić nazwy elementów członkowskich w typie definiującym, Usuń cytowany element programowania z kodu. Nie można uzyskać dostępu do elementu, który wydaje się Visual Basic, aby miał wiele definicji.
