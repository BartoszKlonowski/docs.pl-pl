# <a name="overload-resolution-visual-basic"></a><span data-ttu-id="412f7-102">Rozpoznanie przeciążenia (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="412f7-102">Overload Resolution (Visual Basic)</span></span>
<span data-ttu-id="412f7-103">Gdy [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] kompilatora napotka wywołaniu procedury, która jest zdefiniowana w kilku wersji przeciążone, kompilator należy zdecydować, które z przeciążeń do wywołania.</span><span class="sxs-lookup"><span data-stu-id="412f7-103">When the [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] compiler encounters a call to a procedure that is defined in several overloaded versions, the compiler must decide which of the overloads to call.</span></span> <span data-ttu-id="412f7-104">Dzieje się tak, wykonując następujące czynności:</span><span class="sxs-lookup"><span data-stu-id="412f7-104">It does this by performing the following steps:</span></span>  
  
1.  <span data-ttu-id="412f7-105">**Ułatwienia dostępu.**</span><span class="sxs-lookup"><span data-stu-id="412f7-105">**Accessibility.**</span></span> <span data-ttu-id="412f7-106">Eliminuje wszystkie przeciążenia z poziomu dostępu, uniemożliwiający wywoływanie jej kod wywołujący.</span><span class="sxs-lookup"><span data-stu-id="412f7-106">It eliminates any overload with an access level that prevents the calling code from calling it.</span></span>  
  
2.  <span data-ttu-id="412f7-107">**Liczba parametrów.**</span><span class="sxs-lookup"><span data-stu-id="412f7-107">**Number of Parameters.**</span></span> <span data-ttu-id="412f7-108">Eliminuje wszystkie przeciążenia, który definiuje różne liczby parametrów niż podano w wywołaniu.</span><span class="sxs-lookup"><span data-stu-id="412f7-108">It eliminates any overload that defines a different number of parameters than are supplied in the call.</span></span>  
  
3.  <span data-ttu-id="412f7-109">**Typ danych parametru.**</span><span class="sxs-lookup"><span data-stu-id="412f7-109">**Parameter Data Types.**</span></span> <span data-ttu-id="412f7-110">Kompilator przekazuje metody rozszerzenia preferencji metody wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="412f7-110">The compiler gives instance methods preference over extension methods.</span></span> <span data-ttu-id="412f7-111">Jeśli zostanie znaleziony dowolnej metody wystąpienia, który wymaga tylko rozszerzanie konwersji w celu dopasowania wywołania procedury, wszystkie metody rozszerzenia są usuwane i kompilator będzie kontynuowane przy użyciu kandydatów metody wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="412f7-111">If any instance method is found that requires only widening conversions to match the procedure call, all extension methods are dropped and the compiler continues with only the instance method candidates.</span></span> <span data-ttu-id="412f7-112">Jeśli zostanie znaleziony taki metody wystąpienia, będzie kontynuowane z wystąpienia i metody rozszerzenia.</span><span class="sxs-lookup"><span data-stu-id="412f7-112">If no such instance method is found, it continues with both instance and extension methods.</span></span>  
  
     <span data-ttu-id="412f7-113">W tym kroku eliminuje wszystkie przeciążenia, dla którego typy danych, wywoływania argumentów nie można przekonwertować na typy parametrów zdefiniowanych w przeciążenia.</span><span class="sxs-lookup"><span data-stu-id="412f7-113">In this step, it eliminates any overload for which the data types of the calling arguments cannot be converted to the parameter types defined in the overload.</span></span>  
  
4.  <span data-ttu-id="412f7-114">**Zawężanie konwersji.**</span><span class="sxs-lookup"><span data-stu-id="412f7-114">**Narrowing Conversions.**</span></span> <span data-ttu-id="412f7-115">Eliminuje wszystkie przeciążenia, które wymaga zwężenia konwersji z wywołania typy argumentów do typów zdefiniowanych parametrów.</span><span class="sxs-lookup"><span data-stu-id="412f7-115">It eliminates any overload that requires a narrowing conversion from the calling argument types to the defined parameter types.</span></span> <span data-ttu-id="412f7-116">Dotyczy to określa, czy sprawdzanie typu przełączać ([Option Strict — instrukcja](../../../../visual-basic/language-reference/statements/option-strict-statement.md)) jest `On` lub `Off`.</span><span class="sxs-lookup"><span data-stu-id="412f7-116">This is true whether the type checking switch ([Option Strict Statement](../../../../visual-basic/language-reference/statements/option-strict-statement.md)) is `On` or `Off`.</span></span>  
  
5.  <span data-ttu-id="412f7-117">**Co najmniej rozszerzanie.**</span><span class="sxs-lookup"><span data-stu-id="412f7-117">**Least Widening.**</span></span> <span data-ttu-id="412f7-118">Kompilator uwzględnia pozostałych przeciążeń w parach.</span><span class="sxs-lookup"><span data-stu-id="412f7-118">The compiler considers the remaining overloads in pairs.</span></span> <span data-ttu-id="412f7-119">Dla każdej pary porównuje typów danych zdefiniowanych parametrów.</span><span class="sxs-lookup"><span data-stu-id="412f7-119">For each pair, it compares the data types of the defined parameters.</span></span> <span data-ttu-id="412f7-120">Jeśli poszerzyć typów w jednym z przeciążeń wszystkie odpowiednie typy w innym, kompilator eliminuje to drugie.</span><span class="sxs-lookup"><span data-stu-id="412f7-120">If the types in one of the overloads all widen to the corresponding types in the other, the compiler eliminates the latter.</span></span> <span data-ttu-id="412f7-121">Oznacza to zachowuje przeciążenia, które wymaga minimalnej liczbie rozszerzanie.</span><span class="sxs-lookup"><span data-stu-id="412f7-121">That is, it retains the overload that requires the least amount of widening.</span></span>  
  
6.  <span data-ttu-id="412f7-122">**Jeden kandydat.**</span><span class="sxs-lookup"><span data-stu-id="412f7-122">**Single Candidate.**</span></span> <span data-ttu-id="412f7-123">Nadal considering przeciążenia parami dopóki tylko jeden przeciążenia pozostaje oraz umożliwia rozwiązanie można wywołać tego przeciążenia.</span><span class="sxs-lookup"><span data-stu-id="412f7-123">It continues considering overloads in pairs until only one overload remains, and it resolves the call to that overload.</span></span> <span data-ttu-id="412f7-124">Czy kompilator nie można zmniejszyć przeciążeń do pojedynczego kandydujących, zostanie wygenerowany błąd.</span><span class="sxs-lookup"><span data-stu-id="412f7-124">If the compiler cannot reduce the overloads to a single candidate, it generates an error.</span></span>
