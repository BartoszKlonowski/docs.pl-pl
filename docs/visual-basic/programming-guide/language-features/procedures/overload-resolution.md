# <a name="overload-resolution-visual-basic"></a><span data-ttu-id="0eeac-102">Rozpoznanie przeciążenia (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="0eeac-102">Overload Resolution (Visual Basic)</span></span>
<span data-ttu-id="0eeac-103">Gdy kompilator języka Visual Basic napotyka wywołanie procedury, która jest zdefiniowana w kilku przeciążone wersje, kompilator należy zdecydować, której przeciążenia do wywołania.</span><span class="sxs-lookup"><span data-stu-id="0eeac-103">When the Visual Basic compiler encounters a call to a procedure that is defined in several overloaded versions, the compiler must decide which of the overloads to call.</span></span> <span data-ttu-id="0eeac-104">Dzieje się tak, wykonując następujące czynności:</span><span class="sxs-lookup"><span data-stu-id="0eeac-104">It does this by performing the following steps:</span></span>  
  
1.  <span data-ttu-id="0eeac-105">**Ułatwienia dostępu.**</span><span class="sxs-lookup"><span data-stu-id="0eeac-105">**Accessibility.**</span></span> <span data-ttu-id="0eeac-106">Eliminuje to żadnego przeciążenia z poziomem dostępu, który uniemożliwia jej wywołanie kodu wywołującego.</span><span class="sxs-lookup"><span data-stu-id="0eeac-106">It eliminates any overload with an access level that prevents the calling code from calling it.</span></span>  
  
2.  <span data-ttu-id="0eeac-107">**Liczba parametrów.**</span><span class="sxs-lookup"><span data-stu-id="0eeac-107">**Number of Parameters.**</span></span> <span data-ttu-id="0eeac-108">Eliminuje to wszystkie przeciążenia, które definiuje szereg różnych parametrów nie są określane w wywołaniu.</span><span class="sxs-lookup"><span data-stu-id="0eeac-108">It eliminates any overload that defines a different number of parameters than are supplied in the call.</span></span>  
  
3.  <span data-ttu-id="0eeac-109">**Typ danych parametru.**</span><span class="sxs-lookup"><span data-stu-id="0eeac-109">**Parameter Data Types.**</span></span> <span data-ttu-id="0eeac-110">Kompilator preferuje wystąpienia metod za pośrednictwem metody rozszerzenia.</span><span class="sxs-lookup"><span data-stu-id="0eeac-110">The compiler gives instance methods preference over extension methods.</span></span> <span data-ttu-id="0eeac-111">Jeśli zostanie znaleziony dowolnej metody wystąpienia, która wymaga tylko poszerzeniem konwersji, w celu dopasowania wywołania procedury, wszystkie metody rozszerzenia są odrzucane i kompilator będzie kontynuowane z użyciem kandydatów metody wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="0eeac-111">If any instance method is found that requires only widening conversions to match the procedure call, all extension methods are dropped and the compiler continues with only the instance method candidates.</span></span> <span data-ttu-id="0eeac-112">Jeśli żadna metoda wystąpienia zostanie znaleziony, jest kontynuowane przy użyciu wystąpienia i metody rozszerzenia.</span><span class="sxs-lookup"><span data-stu-id="0eeac-112">If no such instance method is found, it continues with both instance and extension methods.</span></span>  
  
     <span data-ttu-id="0eeac-113">W tym kroku pozwala wyeliminować żadnego przeciążenia, dla którego typy danych, wywoływania argumentów nie można przekonwertować na typy parametrów zdefiniowanych w przeciążenia.</span><span class="sxs-lookup"><span data-stu-id="0eeac-113">In this step, it eliminates any overload for which the data types of the calling arguments cannot be converted to the parameter types defined in the overload.</span></span>  
  
4.  <span data-ttu-id="0eeac-114">**Konwersje zawężające.**</span><span class="sxs-lookup"><span data-stu-id="0eeac-114">**Narrowing Conversions.**</span></span> <span data-ttu-id="0eeac-115">Eliminuje to żadnego przeciążenia, który wymaga konwersji zawężającej od wywołującego typy argumentów do parametrów zdefiniowanych typów.</span><span class="sxs-lookup"><span data-stu-id="0eeac-115">It eliminates any overload that requires a narrowing conversion from the calling argument types to the defined parameter types.</span></span> <span data-ttu-id="0eeac-116">Jest to istotne, czy przełączyć kontrola typów ([Option Strict — instrukcja](../../../../visual-basic/language-reference/statements/option-strict-statement.md)) jest `On` lub `Off`.</span><span class="sxs-lookup"><span data-stu-id="0eeac-116">This is true whether the type checking switch ([Option Strict Statement](../../../../visual-basic/language-reference/statements/option-strict-statement.md)) is `On` or `Off`.</span></span>  
  
5.  <span data-ttu-id="0eeac-117">**Co najmniej rozszerzanie.**</span><span class="sxs-lookup"><span data-stu-id="0eeac-117">**Least Widening.**</span></span> <span data-ttu-id="0eeac-118">Kompilator traktuje pozostałe przeciążeń w pary.</span><span class="sxs-lookup"><span data-stu-id="0eeac-118">The compiler considers the remaining overloads in pairs.</span></span> <span data-ttu-id="0eeac-119">Dla każdej pary porównuje typy danych zdefiniowanych parametrów.</span><span class="sxs-lookup"><span data-stu-id="0eeac-119">For each pair, it compares the data types of the defined parameters.</span></span> <span data-ttu-id="0eeac-120">Jeśli jedno z przeciążeń wszystkich typów są rozszerzane odpowiadające typy w innym, kompilator eliminuje te ostatnie.</span><span class="sxs-lookup"><span data-stu-id="0eeac-120">If the types in one of the overloads all widen to the corresponding types in the other, the compiler eliminates the latter.</span></span> <span data-ttu-id="0eeac-121">Oznacza to, że zachowuje przeciążenia, które wymaga minimalnej liczbie rozszerzanie.</span><span class="sxs-lookup"><span data-stu-id="0eeac-121">That is, it retains the overload that requires the least amount of widening.</span></span>  
  
6.  <span data-ttu-id="0eeac-122">**Pojedynczy Release Candidate.**</span><span class="sxs-lookup"><span data-stu-id="0eeac-122">**Single Candidate.**</span></span> <span data-ttu-id="0eeac-123">Kontynuuje considering przeciążenia w parach, dopóki tylko jeden przeciążenia pozostaje i jest rozpoznawana jako wywołanie tego przeciążenia.</span><span class="sxs-lookup"><span data-stu-id="0eeac-123">It continues considering overloads in pairs until only one overload remains, and it resolves the call to that overload.</span></span> <span data-ttu-id="0eeac-124">Jeśli kompilator nie może ograniczyć przeciążenia do pojedynczego Release candidate, spowoduje wygenerowanie błędu.</span><span class="sxs-lookup"><span data-stu-id="0eeac-124">If the compiler cannot reduce the overloads to a single candidate, it generates an error.</span></span>
