<span data-ttu-id="98684-103">Gdy przekazujesz jeden lub więcej argumentów do procedury, każdy argument odpowiada bazowemu elementowi programowania w kodzie wywołującym.</span><span class="sxs-lookup"><span data-stu-id="98684-103">When you pass one or more arguments to a procedure, each argument corresponds to an underlying programming element in the calling code.</span></span> <span data-ttu-id="98684-104">Można przekazać wartość tego podstawowego elementu lub odwołanie do niego.</span><span class="sxs-lookup"><span data-stu-id="98684-104">You can pass either the value of this underlying element, or a reference to it.</span></span> <span data-ttu-id="98684-105">Jest to tzw. *mechanizm przekazywania*.</span><span class="sxs-lookup"><span data-stu-id="98684-105">This is known as the *passing mechanism*.</span></span>  
  
## <a name="passing-by-value"></a><span data-ttu-id="98684-106">Przekazywanie przez wartość</span><span class="sxs-lookup"><span data-stu-id="98684-106">Passing by Value</span></span>  

 <span data-ttu-id="98684-107">Argument jest przekazywany *przez wartość* poprzez określenie słowa kluczowego [ByVal](../../../language-reference/modifiers/byval.md) dla odpowiadającego parametru w definicji procedury.</span><span class="sxs-lookup"><span data-stu-id="98684-107">You pass an argument *by value* by specifying the [ByVal](../../../language-reference/modifiers/byval.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="98684-108">Korzystając z tego mechanizmu przekazującego, Visual Basic kopiuje wartość bazowego elementu programowania do zmiennej lokalnej w procedurze.</span><span class="sxs-lookup"><span data-stu-id="98684-108">When you use this passing mechanism, Visual Basic copies the value of the underlying programming element into a local variable in the procedure.</span></span> <span data-ttu-id="98684-109">Kod procedury nie ma dostępu do podstawowego elementu w kodzie wywołującym.</span><span class="sxs-lookup"><span data-stu-id="98684-109">The procedure code does not have any access to the underlying element in the calling code.</span></span>  
  
## <a name="passing-by-reference"></a><span data-ttu-id="98684-110">Przekazywanie przez odwołanie</span><span class="sxs-lookup"><span data-stu-id="98684-110">Passing by Reference</span></span>  

 <span data-ttu-id="98684-111">Argument jest przekazywany *przez odwołanie* poprzez określenie słowa kluczowego [ByRef](../../../language-reference/modifiers/byref.md) dla odpowiadającego parametru w definicji procedury.</span><span class="sxs-lookup"><span data-stu-id="98684-111">You pass an argument *by reference* by specifying the [ByRef](../../../language-reference/modifiers/byref.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="98684-112">W przypadku korzystania z tego mechanizmu przekazującego, Visual Basic daje procedurę bezpośrednio odniesienia do bazowego elementu programowania w kodzie wywołującym.</span><span class="sxs-lookup"><span data-stu-id="98684-112">When you use this passing mechanism, Visual Basic gives the procedure a direct reference to the underlying programming element in the calling code.</span></span>  
  
## <a name="passing-mechanism-and-element-type"></a><span data-ttu-id="98684-113">Mechanizm przekazywania i typ elementu</span><span class="sxs-lookup"><span data-stu-id="98684-113">Passing Mechanism and Element Type</span></span>  

 <span data-ttu-id="98684-114">Wybór mechanizmu przekazywania nie jest taki sam jak Klasyfikacja podstawowego typu elementu.</span><span class="sxs-lookup"><span data-stu-id="98684-114">The choice of passing mechanism is not the same as the classification of the underlying element type.</span></span> <span data-ttu-id="98684-115">Przekazywanie przez wartość lub przez odwołanie dotyczy tego, co Visual Basic dostarcza do kodu procedury.</span><span class="sxs-lookup"><span data-stu-id="98684-115">Passing by value or by reference refers to what Visual Basic supplies to the procedure code.</span></span> <span data-ttu-id="98684-116">Typ wartości lub typ referencyjny odnosi się do sposobu przechowywania elementu programowania w pamięci.</span><span class="sxs-lookup"><span data-stu-id="98684-116">A value type or reference type refers to how a programming element is stored in memory.</span></span>  
  
 <span data-ttu-id="98684-117">Jednak mechanizm przekazywania i typ elementu są wzajemnie powiązane.</span><span class="sxs-lookup"><span data-stu-id="98684-117">However, the passing mechanism and element type are interrelated.</span></span> <span data-ttu-id="98684-118">Wartość typu referencyjnego to wskaźnik do danych znajdujących się w innym miejscu w pamięci.</span><span class="sxs-lookup"><span data-stu-id="98684-118">The value of a reference type is a pointer to the data elsewhere in memory.</span></span> <span data-ttu-id="98684-119">Oznacza to, że w przypadku przekazania typu odwołania przez wartość kod procedury ma wskaźnik do danych elementu podstawowego, nawet jeśli nie może uzyskać dostępu do samego elementu bazowego.</span><span class="sxs-lookup"><span data-stu-id="98684-119">This means that when you pass a reference type by value, the procedure code has a pointer to the underlying element's data, even though it cannot access the underlying element itself.</span></span> <span data-ttu-id="98684-120">Na przykład jeśli element jest zmienną tablicową, kod procedury nie ma dostępu do samej zmiennej, ale może uzyskać dostęp do elementów członkowskich tablicy.</span><span class="sxs-lookup"><span data-stu-id="98684-120">For example, if the element is an array variable, the procedure code does not have access to the variable itself, but it can access the array members.</span></span>
