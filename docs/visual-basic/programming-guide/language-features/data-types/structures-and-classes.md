 <span data-ttu-id="0d761-110">Struktury i klasy są podobne do następujących:</span><span class="sxs-lookup"><span data-stu-id="0d761-110">Structures and classes are similar in the following respects:</span></span>  
  
- <span data-ttu-id="0d761-111">Oba są typami *kontenerów* , co oznacza, że zawierają one inne typy jako elementy członkowskie.</span><span class="sxs-lookup"><span data-stu-id="0d761-111">Both are *container* types, meaning that they contain other types as members.</span></span>  
  
- <span data-ttu-id="0d761-112">Oba mają składowe, które mogą obejmować konstruktory, metody, właściwości, pola, stałe, wyliczenia, zdarzenia i programy obsługi zdarzeń.</span><span class="sxs-lookup"><span data-stu-id="0d761-112">Both have members, which can include constructors, methods, properties, fields, constants, enumerations, events, and event handlers.</span></span> <span data-ttu-id="0d761-113">Jednakże nie należy mylić tych elementów członkowskich z zadeklarowanymi *elementami* struktury.</span><span class="sxs-lookup"><span data-stu-id="0d761-113">However, do not confuse these members with the declared *elements* of a structure.</span></span>  
  
- <span data-ttu-id="0d761-114">Elementy członkowskie obu tych elementów mogą mieć indywidualne poziomy dostępu.</span><span class="sxs-lookup"><span data-stu-id="0d761-114">Members of both can have individualized access levels.</span></span> <span data-ttu-id="0d761-115">Na przykład jeden element członkowski można zadeklarować `Public` i inne `Private`.</span><span class="sxs-lookup"><span data-stu-id="0d761-115">For example, one member can be declared `Public` and another `Private`.</span></span>  
  
- <span data-ttu-id="0d761-116">Oba mogą implementować interfejsy.</span><span class="sxs-lookup"><span data-stu-id="0d761-116">Both can implement interfaces.</span></span>  
  
- <span data-ttu-id="0d761-117">Oba mogą mieć udostępnione konstruktory, z lub bez parametrów.</span><span class="sxs-lookup"><span data-stu-id="0d761-117">Both can have shared constructors, with or without parameters.</span></span>  
  
- <span data-ttu-id="0d761-118">Obie mogą uwidaczniać *Właściwość domyślną*, pod warunkiem, że właściwość przyjmuje co najmniej jeden parametr.</span><span class="sxs-lookup"><span data-stu-id="0d761-118">Both can expose a *default property*, provided that property takes at least one parameter.</span></span>  
  
- <span data-ttu-id="0d761-119">Oba mogą deklarować i wywoływać zdarzenia, a oba mogą deklarować delegatów.</span><span class="sxs-lookup"><span data-stu-id="0d761-119">Both can declare and raise events, and both can declare delegates.</span></span>  
  
## <a name="differences"></a><span data-ttu-id="0d761-120">Wynikających</span><span class="sxs-lookup"><span data-stu-id="0d761-120">Differences</span></span>  
 <span data-ttu-id="0d761-121">Struktury i klasy różnią się w następujących tematach:</span><span class="sxs-lookup"><span data-stu-id="0d761-121">Structures and classes differ in the following particulars:</span></span>  
  
- <span data-ttu-id="0d761-122">Struktury są *typami wartości*; klasy są *typami referencyjnymi*.</span><span class="sxs-lookup"><span data-stu-id="0d761-122">Structures are *value types*; classes are *reference types*.</span></span> <span data-ttu-id="0d761-123">Zmienna typu struktury zawiera dane struktury, a nie zawiera odwołania do danych jako typ klasy.</span><span class="sxs-lookup"><span data-stu-id="0d761-123">A variable of a structure type contains the structure's data, rather than containing a reference to the data as a class type does.</span></span>  
  
- <span data-ttu-id="0d761-124">Struktury używają alokacji stosu; klasy używają alokacji sterty.</span><span class="sxs-lookup"><span data-stu-id="0d761-124">Structures use stack allocation; classes use heap allocation.</span></span>  
  
- <span data-ttu-id="0d761-125">Wszystkie elementy struktury są domyślnie `Public`. zmienne klas i stałe są `Private` domyślnie, podczas gdy inne elementy członkowskie klasy są domyślnie `Public`.</span><span class="sxs-lookup"><span data-stu-id="0d761-125">All structure elements are `Public` by default; class variables and constants are `Private` by default, while other class members are `Public` by default.</span></span> <span data-ttu-id="0d761-126">Takie zachowanie w przypadku elementów członkowskich klasy zapewnia zgodność z systemem Visual Basic 6,0 wartości domyślnych.</span><span class="sxs-lookup"><span data-stu-id="0d761-126">This behavior for class members provides compatibility with the Visual Basic 6.0 system of defaults.</span></span>  
  
- <span data-ttu-id="0d761-127">Struktura musi mieć co najmniej jedną nieudostępnioną zmienną lub nieudostępnianą niestandardową pozycję zdarzenia; Klasa może być całkowicie pusta.</span><span class="sxs-lookup"><span data-stu-id="0d761-127">A structure must have at least one nonshared variable or nonshared, noncustom event element; a class can be completely empty.</span></span>  
  
- <span data-ttu-id="0d761-128">Elementy struktury nie mogą być deklarowane jako `Protected`; elementy członkowskie klasy mogą.</span><span class="sxs-lookup"><span data-stu-id="0d761-128">Structure elements cannot be declared as `Protected`; class members can.</span></span>  
  
- <span data-ttu-id="0d761-129">Procedura struktury może obsługiwać zdarzenia tylko wtedy, gdy jest to [współdzielona](../../../../visual-basic/language-reference/modifiers/shared.md) procedura`Sub` i tylko za pomocą [instrukcji AddHandler](../../../../visual-basic/language-reference/statements/addhandler-statement.md); Każda procedura klasy może obsługiwać zdarzenia za pomocą słowa kluczowego [Handles](../../../../visual-basic/language-reference/statements/handles-clause.md) lub instrukcji `AddHandler`.</span><span class="sxs-lookup"><span data-stu-id="0d761-129">A structure procedure can handle events only if it is a [Shared](../../../../visual-basic/language-reference/modifiers/shared.md)`Sub` procedure, and only by means of the [AddHandler Statement](../../../../visual-basic/language-reference/statements/addhandler-statement.md); any class procedure can handle events, using either the [Handles](../../../../visual-basic/language-reference/statements/handles-clause.md) keyword or the `AddHandler` statement.</span></span> <span data-ttu-id="0d761-130">Aby uzyskać więcej informacji, zobacz [zdarzenia](../../../../visual-basic/programming-guide/language-features/events/index.md).</span><span class="sxs-lookup"><span data-stu-id="0d761-130">For more information, see [Events](../../../../visual-basic/programming-guide/language-features/events/index.md).</span></span>  
  
- <span data-ttu-id="0d761-131">Deklaracje zmiennej struktury nie mogą określać inicjatorów ani początkowych rozmiarów dla tablic; deklaracje zmiennej klasy mogą.</span><span class="sxs-lookup"><span data-stu-id="0d761-131">Structure variable declarations cannot specify initializers or initial sizes for arrays; class variable declarations can.</span></span>  
  
- <span data-ttu-id="0d761-132">Struktury niejawnie dziedziczą z klasy <xref:System.ValueType?displayProperty=nameWithType> i nie mogą dziedziczyć po żadnym innym typie; klasy mogą dziedziczyć z dowolnej klasy lub klas innych niż <xref:System.ValueType?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="0d761-132">Structures implicitly inherit from the <xref:System.ValueType?displayProperty=nameWithType> class and cannot inherit from any other type; classes can inherit from any class or classes other than <xref:System.ValueType?displayProperty=nameWithType>.</span></span>  
  
- <span data-ttu-id="0d761-133">Struktury nie są dziedziczenia; klasy są.</span><span class="sxs-lookup"><span data-stu-id="0d761-133">Structures are not inheritable; classes are.</span></span>  
  
- <span data-ttu-id="0d761-134">Struktury nie są nigdy przerywane, więc środowisko uruchomieniowe języka wspólnego (CLR) nigdy nie wywołuje metody <xref:System.Object.Finalize%2A> w żadnej strukturze; klasy są kończone przez moduł wyrzucania elementów bezużytecznych (GC), który wywołuje <xref:System.Object.Finalize%2A> na klasie, gdy wykryje brak aktywnych odwołań.</span><span class="sxs-lookup"><span data-stu-id="0d761-134">Structures are never terminated, so the common language runtime (CLR) never calls the <xref:System.Object.Finalize%2A> method on any structure; classes are terminated by the garbage collector (GC), which calls <xref:System.Object.Finalize%2A> on a class when it detects there are no active references remaining.</span></span>  
  
- <span data-ttu-id="0d761-135">Struktura nie wymaga konstruktora; Klasa.</span><span class="sxs-lookup"><span data-stu-id="0d761-135">A structure does not require a constructor; a class does.</span></span>  
  
- <span data-ttu-id="0d761-136">Struktury mogą mieć konstruktory nieudostępniane tylko wtedy, gdy przyjmują parametry; klasy mogą mieć parametry z parametrami lub bez nich.</span><span class="sxs-lookup"><span data-stu-id="0d761-136">Structures can have nonshared constructors only if they take parameters; classes can have them with or without parameters.</span></span>  
  
 <span data-ttu-id="0d761-137">Każda struktura ma niejawny Konstruktor publiczny bez parametrów.</span><span class="sxs-lookup"><span data-stu-id="0d761-137">Every structure has an implicit public constructor without parameters.</span></span> <span data-ttu-id="0d761-138">Ten konstruktor inicjuje wszystkie elementy danych struktury do ich wartości domyślnych.</span><span class="sxs-lookup"><span data-stu-id="0d761-138">This constructor initializes all the structure's data elements to their default values.</span></span> <span data-ttu-id="0d761-139">Nie można ponownie zdefiniować tego zachowania.</span><span class="sxs-lookup"><span data-stu-id="0d761-139">You cannot redefine this behavior.</span></span>  
  
## <a name="instances-and-variables"></a><span data-ttu-id="0d761-140">Wystąpienia i zmienne</span><span class="sxs-lookup"><span data-stu-id="0d761-140">Instances and Variables</span></span>  
 <span data-ttu-id="0d761-141">Ponieważ struktury są typami wartości, Każda zmienna struktury jest trwale związana z konkretnym wystąpieniem struktury.</span><span class="sxs-lookup"><span data-stu-id="0d761-141">Because structures are value types, each structure variable is permanently bound to an individual structure instance.</span></span> <span data-ttu-id="0d761-142">Ale klasy są typami odwołań, a zmienna obiektu może odwoływać się do różnych wystąpień klasy w różnych godzinach.</span><span class="sxs-lookup"><span data-stu-id="0d761-142">But classes are reference types, and an object variable can refer to various class instances at different times.</span></span> <span data-ttu-id="0d761-143">Takie rozróżnienie ma wpływ na użycie struktur i klas w następujący sposób:</span><span class="sxs-lookup"><span data-stu-id="0d761-143">This distinction affects your usage of structures and classes in the following ways:</span></span>  
  
- <span data-ttu-id="0d761-144">**Zainicjować.**</span><span class="sxs-lookup"><span data-stu-id="0d761-144">**Initialization.**</span></span> <span data-ttu-id="0d761-145">Zmienna struktury niejawnie obejmuje inicjalizację elementów przy użyciu konstruktora bez parametrów struktury.</span><span class="sxs-lookup"><span data-stu-id="0d761-145">A structure variable implicitly includes an initialization of the elements using the structure's parameterless constructor.</span></span> <span data-ttu-id="0d761-146">W związku z tym `Dim s As struct1` jest równoznaczny z `Dim s As struct1 = New struct1()`.</span><span class="sxs-lookup"><span data-stu-id="0d761-146">Therefore, `Dim s As struct1` is equivalent to `Dim s As struct1 = New struct1()`.</span></span>  
  
- <span data-ttu-id="0d761-147">**Przypisywanie zmiennych.**</span><span class="sxs-lookup"><span data-stu-id="0d761-147">**Assigning Variables.**</span></span> <span data-ttu-id="0d761-148">Po przypisaniu jednej zmiennej struktury do innej lub przekazanie wystąpienia struktury do argumentu procedury, bieżące wartości wszystkich elementów zmiennych są kopiowane do nowej struktury.</span><span class="sxs-lookup"><span data-stu-id="0d761-148">When you assign one structure variable to another, or pass a structure instance to a procedure argument, the current values of all the variable elements are copied to the new structure.</span></span> <span data-ttu-id="0d761-149">Po przypisaniu jednej zmiennej obiektu do innej lub przekazanie zmiennej obiektu do procedury, tylko wskaźnik odwołania jest kopiowany.</span><span class="sxs-lookup"><span data-stu-id="0d761-149">When you assign one object variable to another, or pass an object variable to a procedure, only the reference pointer is copied.</span></span>  
  
- <span data-ttu-id="0d761-150">**Przypisywanie niczego.**</span><span class="sxs-lookup"><span data-stu-id="0d761-150">**Assigning Nothing.**</span></span> <span data-ttu-id="0d761-151">Wartość [Nothing](../../../../visual-basic/language-reference/nothing.md) można przypisać do zmiennej struktury, ale wystąpienie jest nadal skojarzone ze zmienną.</span><span class="sxs-lookup"><span data-stu-id="0d761-151">You can assign the value [Nothing](../../../../visual-basic/language-reference/nothing.md) to a structure variable, but the instance continues to be associated with the variable.</span></span> <span data-ttu-id="0d761-152">Nadal można wywołać metody i uzyskać dostęp do jej elementów danych, chociaż elementy zmiennych są ponownie inicjowane przez przypisanie.</span><span class="sxs-lookup"><span data-stu-id="0d761-152">You can still call its methods and access its data elements, although variable elements are reinitialized by the assignment.</span></span>  
  
     <span data-ttu-id="0d761-153">W przeciwieństwie do ustawienia zmiennej obiektu na `Nothing`, można usunąć skojarzenie jej z dowolnego wystąpienia klasy i nie będzie można uzyskać dostępu do żadnych elementów członkowskich za pośrednictwem zmiennej, dopóki nie przypiszesz do niego kolejnego wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="0d761-153">In contrast, if you set an object variable to `Nothing`, you dissociate it from any class instance, and you cannot access any members through the variable until you assign another instance to it.</span></span>

     <span data-ttu-id="0d761-157">Elementy struktury są jednak izolowane w ich własnym wystąpieniu.</span><span class="sxs-lookup"><span data-stu-id="0d761-157">Structure elements, however, are isolated within their own instance.</span></span> <span data-ttu-id="0d761-158">Zmiany wartości nie są odzwierciedlone w żadnych innych zmiennych struktury, nawet w innych wystąpieniach tej samej deklaracji `Structure`.</span><span class="sxs-lookup"><span data-stu-id="0d761-158">Changes to their values are not reflected in any other structure variables, even in other instances of the same `Structure` declaration.</span></span>  
  
- <span data-ttu-id="0d761-159">**Kryteri.**</span><span class="sxs-lookup"><span data-stu-id="0d761-159">**Equality.**</span></span> <span data-ttu-id="0d761-160">Testy równości dwóch struktur muszą być wykonywane z testem elementu po elemencie.</span><span class="sxs-lookup"><span data-stu-id="0d761-160">Equality testing of two structures must be performed with an element-by-element test.</span></span> <span data-ttu-id="0d761-161">Można porównać dwie zmienne obiektów przy użyciu metody <xref:System.Object.Equals%2A>.</span><span class="sxs-lookup"><span data-stu-id="0d761-161">Two object variables can be compared using the <xref:System.Object.Equals%2A> method.</span></span> <span data-ttu-id="0d761-162"><xref:System.Object.Equals%2A> wskazuje, czy dwie zmienne wskazują na to samo wystąpienie.</span><span class="sxs-lookup"><span data-stu-id="0d761-162"><xref:System.Object.Equals%2A> indicates whether the two variables point to the same instance.</span></span>
