-   <span data-ttu-id="cd55a-116">Jednocześnie można implementować interfejsów.</span><span class="sxs-lookup"><span data-stu-id="cd55a-116">Both can implement interfaces.</span></span>

## <a name="differences"></a><span data-ttu-id="cd55a-120">Różnice</span><span class="sxs-lookup"><span data-stu-id="cd55a-120">Differences</span></span>  
 <span data-ttu-id="cd55a-121">Struktury i klasy różnią się następujące informacje:</span><span class="sxs-lookup"><span data-stu-id="cd55a-121">Structures and classes differ in the following particulars:</span></span>  
  
-   <span data-ttu-id="cd55a-122">Struktury są *typów wartości*; klasy są *typy referencyjne*.</span><span class="sxs-lookup"><span data-stu-id="cd55a-122">Structures are *value types*; classes are *reference types*.</span></span> <span data-ttu-id="cd55a-123">Zmienna typu Struktura zawiera struktury danych, a nie zawierające odwołania do danych jako typ klasy jest.</span><span class="sxs-lookup"><span data-stu-id="cd55a-123">A variable of a structure type contains the structure's data, rather than containing a reference to the data as a class type does.</span></span>  
  
-   <span data-ttu-id="cd55a-124">Alokacja stosu; Użyj struktury klasy używaj alokacji sterty.</span><span class="sxs-lookup"><span data-stu-id="cd55a-124">Structures use stack allocation; classes use heap allocation.</span></span>  
  
-   <span data-ttu-id="cd55a-125">Wszystkie elementy struktury są `Public` domyślnie; klasa zmienne i stałe są `Private` domyślnie, gdy są inne elementy członkowskie klasy `Public` domyślnie.</span><span class="sxs-lookup"><span data-stu-id="cd55a-125">All structure elements are `Public` by default; class variables and constants are `Private` by default, while other class members are `Public` by default.</span></span> <span data-ttu-id="cd55a-126">To zachowanie dla członków klasy zapewnia zgodność z wartości domyślnych systemu Visual Basic 6.0.</span><span class="sxs-lookup"><span data-stu-id="cd55a-126">This behavior for class members provides compatibility with the Visual Basic 6.0 system of defaults.</span></span>  
  
-   <span data-ttu-id="cd55a-127">Struktura musi mieć co najmniej jeden udostępniana noncustom zmiennej lub udostępniana, element zdarzeń; Klasa może być pusty.</span><span class="sxs-lookup"><span data-stu-id="cd55a-127">A structure must have at least one nonshared variable or nonshared, noncustom event element; a class can be completely empty.</span></span>  
  
-   <span data-ttu-id="cd55a-128">Elementy struktury nie można zadeklarować jako `Protected`; można elementów członkowskich klasy.</span><span class="sxs-lookup"><span data-stu-id="cd55a-128">Structure elements cannot be declared as `Protected`; class members can.</span></span>  
  
-   <span data-ttu-id="cd55a-129">Procedury struktury może obsłużyć zdarzenia tylko wtedy, gdy jest [Shared](../../../../visual-basic/language-reference/modifiers/shared.md) `Sub` procedury i tylko za pomocą klasy [AddHandler — instrukcja](../../../../visual-basic/language-reference/statements/addhandler-statement.md); dowolnej klasy procedury może obsłużyć zdarzenia, przy użyciu albo [ Obsługuje](../../../../visual-basic/language-reference/statements/handles-clause.md) — słowo kluczowe lub `AddHandler` instrukcji.</span><span class="sxs-lookup"><span data-stu-id="cd55a-129">A structure procedure can handle events only if it is a [Shared](../../../../visual-basic/language-reference/modifiers/shared.md)`Sub` procedure, and only by means of the [AddHandler Statement](../../../../visual-basic/language-reference/statements/addhandler-statement.md); any class procedure can handle events, using either the [Handles](../../../../visual-basic/language-reference/statements/handles-clause.md) keyword or the `AddHandler` statement.</span></span> <span data-ttu-id="cd55a-130">Aby uzyskać więcej informacji, zobacz [zdarzenia](../../../../visual-basic/programming-guide/language-features/events/index.md).</span><span class="sxs-lookup"><span data-stu-id="cd55a-130">For more information, see [Events](../../../../visual-basic/programming-guide/language-features/events/index.md).</span></span>  
  
-   <span data-ttu-id="cd55a-131">Struktura deklaracji zmiennych nie można określić inicjatory lub początkowe rozmiary tablic; deklaracje zmiennej klasy można.</span><span class="sxs-lookup"><span data-stu-id="cd55a-131">Structure variable declarations cannot specify initializers or initial sizes for arrays; class variable declarations can.</span></span>  
  
-   <span data-ttu-id="cd55a-132">Dziedzicz niejawnie struktury <xref:System.ValueType?displayProperty=nameWithType> klasy i nie może dziedziczyć z żadnym innym typem; klasy mogą dziedziczyć klasy lub klas innych niż <xref:System.ValueType?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="cd55a-132">Structures implicitly inherit from the <xref:System.ValueType?displayProperty=nameWithType> class and cannot inherit from any other type; classes can inherit from any class or classes other than <xref:System.ValueType?displayProperty=nameWithType>.</span></span>  
  
-   <span data-ttu-id="cd55a-133">Struktury nie są dziedziczone; występują następujące klasy.</span><span class="sxs-lookup"><span data-stu-id="cd55a-133">Structures are not inheritable; classes are.</span></span>  
  
-   <span data-ttu-id="cd55a-134">Struktury nigdy nie kończą, więc środowisko uruchomieniowe języka wspólnego (CLR) nigdy nie wywołuje <xref:System.Object.Finalize%2A> metody w dowolnej struktury; klasy kończą się przez moduł garbage collector (GC), która wywołuje <xref:System.Object.Finalize%2A> dla klasy, po wykryciu istnieją żadnych aktywnych odwołań pozostałe.</span><span class="sxs-lookup"><span data-stu-id="cd55a-134">Structures are never terminated, so the common language runtime (CLR) never calls the <xref:System.Object.Finalize%2A> method on any structure; classes are terminated by the garbage collector (GC), which calls <xref:System.Object.Finalize%2A> on a class when it detects there are no active references remaining.</span></span>  
  
-   <span data-ttu-id="cd55a-135">Struktury nie wymaga konstruktora; jest klasą.</span><span class="sxs-lookup"><span data-stu-id="cd55a-135">A structure does not require a constructor; a class does.</span></span>  
  
-   <span data-ttu-id="cd55a-136">Struktury mogą mieć konstruktorów udostępniana tylko wtedy, gdy ich wejściem parametrów; klasy mogą mieć je z lub bez parametrów.</span><span class="sxs-lookup"><span data-stu-id="cd55a-136">Structures can have nonshared constructors only if they take parameters; classes can have them with or without parameters.</span></span>  
  
 <span data-ttu-id="cd55a-137">Struktura co ma niejawne publicznego konstruktora bez parametrów.</span><span class="sxs-lookup"><span data-stu-id="cd55a-137">Every structure has an implicit public constructor without parameters.</span></span> <span data-ttu-id="cd55a-138">Ten konstruktor inicjuje wszystkie struktury elementy danych do wartości domyślnych.</span><span class="sxs-lookup"><span data-stu-id="cd55a-138">This constructor initializes all the structure's data elements to their default values.</span></span> <span data-ttu-id="cd55a-139">Nie można ponownie zdefiniować to zachowanie.</span><span class="sxs-lookup"><span data-stu-id="cd55a-139">You cannot redefine this behavior.</span></span>

 <span data-ttu-id="cd55a-141">Ponieważ struktury są typy wartości, każdej zmiennej struktury trwale jest powiązany z wystąpieniem poszczególnych struktury.</span><span class="sxs-lookup"><span data-stu-id="cd55a-141">Because structures are value types, each structure variable is permanently bound to an individual structure instance.</span></span> <span data-ttu-id="cd55a-142">Jednak klasy są typy referencyjne i zmienna obiektu mogą odwoływać się do różnych wystąpień klasy w różnym czasie.</span><span class="sxs-lookup"><span data-stu-id="cd55a-142">But classes are reference types, and an object variable can refer to various class instances at different times.</span></span> <span data-ttu-id="cd55a-143">Różnica w tej wpływa na użycie struktur i klas w następujący sposób:</span><span class="sxs-lookup"><span data-stu-id="cd55a-143">This distinction affects your usage of structures and classes in the following ways:</span></span>  
  
-   <span data-ttu-id="cd55a-144">**Inicjowanie.**</span><span class="sxs-lookup"><span data-stu-id="cd55a-144">**Initialization.**</span></span> <span data-ttu-id="cd55a-145">Zmienna struktury obejmuje inicjowania elementów przy użyciu konstruktora struktury.</span><span class="sxs-lookup"><span data-stu-id="cd55a-145">A structure variable implicitly includes an initialization of the elements using the structure's parameterless constructor.</span></span> <span data-ttu-id="cd55a-146">W związku z tym `Dim s As struct1` jest odpowiednikiem `Dim s As struct1 = New struct1()`.</span><span class="sxs-lookup"><span data-stu-id="cd55a-146">Therefore, `Dim s As struct1` is equivalent to `Dim s As struct1 = New struct1()`.</span></span>

     <span data-ttu-id="cd55a-157">Elementy struktury, jednak są odizolowane w własne wystąpienie.</span><span class="sxs-lookup"><span data-stu-id="cd55a-157">Structure elements, however, are isolated within their own instance.</span></span> <span data-ttu-id="cd55a-158">Zmiany wartości nie są widoczne w wszelkie inne zmienne struktury, nawet w przypadku innych wystąpień tego samego `Structure` deklaracji.</span><span class="sxs-lookup"><span data-stu-id="cd55a-158">Changes to their values are not reflected in any other structure variables, even in other instances of the same `Structure` declaration.</span></span>  
  
-   <span data-ttu-id="cd55a-159">**Równości.**</span><span class="sxs-lookup"><span data-stu-id="cd55a-159">**Equality.**</span></span> <span data-ttu-id="cd55a-160">Dwa struktur sprawdzaniem równości można wykonać z testem element po elemencie.</span><span class="sxs-lookup"><span data-stu-id="cd55a-160">Equality testing of two structures must be performed with an element-by-element test.</span></span> <span data-ttu-id="cd55a-161">Dwie zmienne obiektu można porównać przy użyciu <xref:System.Object.Equals%2A> metody.</span><span class="sxs-lookup"><span data-stu-id="cd55a-161">Two object variables can be compared using the <xref:System.Object.Equals%2A> method.</span></span> <span data-ttu-id="cd55a-162"><xref:System.Object.Equals%2A>Wskazuje, czy dwie zmienne punktu w tym samym wystąpieniu.</span><span class="sxs-lookup"><span data-stu-id="cd55a-162"><xref:System.Object.Equals%2A> indicates whether the two variables point to the same instance.</span></span>
