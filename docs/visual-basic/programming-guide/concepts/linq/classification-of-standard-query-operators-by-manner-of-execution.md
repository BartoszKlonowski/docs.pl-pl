## <a name="manners-of-execution"></a><span data-ttu-id="d91cf-108">Sposobów wykonania</span><span class="sxs-lookup"><span data-stu-id="d91cf-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="d91cf-109">Natychmiastowe</span><span class="sxs-lookup"><span data-stu-id="d91cf-109">Immediate</span></span>  
 <span data-ttu-id="d91cf-110">Natychmiastowe wykonanie oznacza, że źródło danych jest do odczytu, a operacja jest wykonywana w punkcie w kodzie, którym zadeklarowano zapytania.</span><span class="sxs-lookup"><span data-stu-id="d91cf-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="d91cf-111">Wszystkie standardowe operatory zapytań, które zwracają wynik pojedynczy, wyliczalny wykonywane natychmiast.</span><span class="sxs-lookup"><span data-stu-id="d91cf-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="d91cf-112">Odroczone</span><span class="sxs-lookup"><span data-stu-id="d91cf-112">Deferred</span></span>  
 <span data-ttu-id="d91cf-113">Wykonanie odroczone oznacza to, czy operacja nie jest wykonywane w punkcie w kodzie którym zadeklarowano zapytania.</span><span class="sxs-lookup"><span data-stu-id="d91cf-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="d91cf-114">Operacja jest wykonywana tylko wtedy, gdy zmienna zapytania są wyliczane na przykład za pomocą `For Each` instrukcji.</span><span class="sxs-lookup"><span data-stu-id="d91cf-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="d91cf-115">Oznacza to, że wyników wykonywania zapytania zależy od zawartości źródła danych, gdy zapytanie jest wykonywane, a nie gdy zdefiniowano zapytanie.</span><span class="sxs-lookup"><span data-stu-id="d91cf-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="d91cf-116">Jeśli zmienna zapytania są wyliczane wiele razy, wyniki mogą się różnić z każdym razem, gdy.</span><span class="sxs-lookup"><span data-stu-id="d91cf-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="d91cf-117">Prawie wszystkie standardowe operatory zapytań o zwracanym typie <xref:System.Collections.Generic.IEnumerable%601> lub <xref:System.Linq.IOrderedEnumerable%601> w sposób odroczone.</span><span class="sxs-lookup"><span data-stu-id="d91cf-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="d91cf-118">Operatorów zapytań, które używają odroczonego wykonania mogą być dodatkowo klasyfikowane jako przesyłania strumieniowego lub -streaming.</span><span class="sxs-lookup"><span data-stu-id="d91cf-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="d91cf-119">Przesyłanie strumieniowe</span><span class="sxs-lookup"><span data-stu-id="d91cf-119">Streaming</span></span>  
 <span data-ttu-id="d91cf-120">Operatory przesyłania strumieniowego jest konieczne odczytywać wszystkie dane źródłowego, przed dają one elementy.</span><span class="sxs-lookup"><span data-stu-id="d91cf-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="d91cf-121">W czasie wykonywania przesyłania strumieniowego operator wykonuje jego działania dla każdego elementu źródłowego podczas odczytu i daje w wyniku elementu, jeśli jest to odpowiednie.</span><span class="sxs-lookup"><span data-stu-id="d91cf-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="d91cf-122">Operator przesyłania strumieniowego w dalszym ciągu odczytywać elementy źródłowe, dopóki nie może wygenerować elementu wynik.</span><span class="sxs-lookup"><span data-stu-id="d91cf-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="d91cf-123">Oznacza to, że więcej niż jeden element źródłowy może przeczytać do wyprodukowania jednego elementu wynik.</span><span class="sxs-lookup"><span data-stu-id="d91cf-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="d91cf-124">Obsługiwane strumieniowo</span><span class="sxs-lookup"><span data-stu-id="d91cf-124">Non-Streaming</span></span>  
 <span data-ttu-id="d91cf-125">Operatory nie obsługiwane strumieniowo nie musi odczytywać wszystkie dane źródłowego, przed dają one elementu wynik.</span><span class="sxs-lookup"><span data-stu-id="d91cf-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="d91cf-126">Operacje, takie jak sortowania i grupowania należą do tej kategorii.</span><span class="sxs-lookup"><span data-stu-id="d91cf-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="d91cf-127">W czasie wykonywania operatory obsługiwane strumieniowo zapytań odczytywać wszystkie dane źródłowego, umieść struktury danych, do operacji i uzyskanie wynikowe elementy.</span><span class="sxs-lookup"><span data-stu-id="d91cf-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
