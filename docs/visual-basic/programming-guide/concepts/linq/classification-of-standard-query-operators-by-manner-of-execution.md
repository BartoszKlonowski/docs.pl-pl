<span data-ttu-id="07f03-103">Implementacje LINQ to Objects standardowych metod operatora zapytania są wykonywane na jednym z dwóch głównych sposobów: natychmiastowe lub odroczone.</span><span class="sxs-lookup"><span data-stu-id="07f03-103">The LINQ to Objects implementations of the standard query operator methods execute in one of two main ways: immediate or deferred.</span></span> <span data-ttu-id="07f03-104">Operatory zapytań, które korzystają z odroczonego wykonania, można dodatkowo podzielić na dwie kategorie: streaming i bez przesyłania strumieniowego.</span><span class="sxs-lookup"><span data-stu-id="07f03-104">The query operators that use deferred execution can be additionally divided into two categories: streaming and non-streaming.</span></span> <span data-ttu-id="07f03-105">Jeśli wiesz, jak są wykonywane różne operatory zapytań, może to pomóc w zrozumieniu wyników uzyskanych z danego zapytania.</span><span class="sxs-lookup"><span data-stu-id="07f03-105">If you know how the different query operators execute, it may help you understand the results that you get from a given query.</span></span> <span data-ttu-id="07f03-106">Jest to szczególnie prawdziwe w przypadku zmiany źródła danych lub tworzenia zapytania na podstawie innego zapytania.</span><span class="sxs-lookup"><span data-stu-id="07f03-106">This is especially true if the data source is changing or if you are building a query on top of another query.</span></span> <span data-ttu-id="07f03-107">Ten temat klasyfikuje standardowe operatory zapytań zgodnie z ich sposobem wykonania.</span><span class="sxs-lookup"><span data-stu-id="07f03-107">This topic classifies the standard query operators according to their manner of execution.</span></span>  
  
## <a name="manners-of-execution"></a><span data-ttu-id="07f03-108">Sposób wykonywania</span><span class="sxs-lookup"><span data-stu-id="07f03-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="07f03-109">Natychmiastowe</span><span class="sxs-lookup"><span data-stu-id="07f03-109">Immediate</span></span>  
 <span data-ttu-id="07f03-110">Natychmiastowe wykonanie oznacza, że źródło danych jest odczytywane i operacja jest wykonywana w punkcie w kodzie, w którym jest zadeklarowane zapytanie.</span><span class="sxs-lookup"><span data-stu-id="07f03-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="07f03-111">Wszystkie standardowe operatory zapytań, które zwracają pojedynczy, nieprzeliczalny wynik wykonania.</span><span class="sxs-lookup"><span data-stu-id="07f03-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="07f03-112">Odroczone</span><span class="sxs-lookup"><span data-stu-id="07f03-112">Deferred</span></span>  
 <span data-ttu-id="07f03-113">Wykonanie odroczone oznacza, że operacja nie jest wykonywana w punkcie w kodzie, w którym jest zadeklarowane zapytanie.</span><span class="sxs-lookup"><span data-stu-id="07f03-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="07f03-114">Operacja jest wykonywana tylko wtedy, gdy zmienna zapytania jest wyliczana, na przykład przy użyciu instrukcji `For Each`.</span><span class="sxs-lookup"><span data-stu-id="07f03-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="07f03-115">Oznacza to, że wyniki wykonywania zapytania zależą od zawartości źródła danych, gdy zapytanie jest wykonywane, a nie po zdefiniowaniu zapytania.</span><span class="sxs-lookup"><span data-stu-id="07f03-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="07f03-116">Jeśli zmienna zapytania jest wyliczana wiele razy, wyniki mogą się różnić za każdym razem.</span><span class="sxs-lookup"><span data-stu-id="07f03-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="07f03-117">Prawie wszystkie standardowe operatory zapytań, których typem zwracanym jest <xref:System.Collections.Generic.IEnumerable%601> lub <xref:System.Linq.IOrderedEnumerable%601> wykonywane w sposób odroczony.</span><span class="sxs-lookup"><span data-stu-id="07f03-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="07f03-118">Operatory zapytań, które korzystają z odroczonego wykonania, można dodatkowo klasyfikować jako strumieniowe lub niestrumieniowe.</span><span class="sxs-lookup"><span data-stu-id="07f03-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="07f03-119">Przesyłanie strumieniowe</span><span class="sxs-lookup"><span data-stu-id="07f03-119">Streaming</span></span>  
 <span data-ttu-id="07f03-120">Operatory przesyłania strumieniowego nie muszą odczytywać wszystkich danych źródłowych przed ich zwróceniem.</span><span class="sxs-lookup"><span data-stu-id="07f03-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="07f03-121">Podczas wykonywania operator przesyłania strumieniowego wykonuje swoją operację na każdym elemencie źródłowym w miarę odczytu i dostarcza element, jeśli jest to konieczne.</span><span class="sxs-lookup"><span data-stu-id="07f03-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="07f03-122">Operator przesyłania strumieniowego kontynuuje odczytywanie elementów źródłowych do momentu wygenerowania elementu wynik.</span><span class="sxs-lookup"><span data-stu-id="07f03-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="07f03-123">Oznacza to, że więcej niż jeden element źródłowy może być odczytany w celu utworzenia jednego elementu wynik.</span><span class="sxs-lookup"><span data-stu-id="07f03-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="07f03-124">Bez przesyłania strumieniowego</span><span class="sxs-lookup"><span data-stu-id="07f03-124">Non-Streaming</span></span>  
 <span data-ttu-id="07f03-125">Operatory niestrumieniowe muszą odczytywać wszystkie dane źródłowe, zanim będą mogły dać element wynik.</span><span class="sxs-lookup"><span data-stu-id="07f03-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="07f03-126">Operacje, takie jak sortowanie lub grupowanie, znajdują się w tej kategorii.</span><span class="sxs-lookup"><span data-stu-id="07f03-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="07f03-127">W czasie wykonywania operatory zapytań bez przesyłania strumieniowego odczytują wszystkie dane źródłowe, umieszczają je w strukturze danych, wykonują operację i dają elementy wyników.</span><span class="sxs-lookup"><span data-stu-id="07f03-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
