## <a name="manners-of-execution"></a><span data-ttu-id="fe422-108">Sposobów wykonywania</span><span class="sxs-lookup"><span data-stu-id="fe422-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="fe422-109">Natychmiastowe</span><span class="sxs-lookup"><span data-stu-id="fe422-109">Immediate</span></span>  
 <span data-ttu-id="fe422-110">Wykonanie natychmiastowe oznacza, że źródło danych jest do odczytu, a operacja jest wykonywana w momencie w kodzie, gdzie zapytanie jest zadeklarowany.</span><span class="sxs-lookup"><span data-stu-id="fe422-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="fe422-111">Wszystkie standardowe operatory zapytań zwracających pojedynczą, wyliczalny wynik wykonania natychmiast.</span><span class="sxs-lookup"><span data-stu-id="fe422-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="fe422-112">Odroczone</span><span class="sxs-lookup"><span data-stu-id="fe422-112">Deferred</span></span>  
 <span data-ttu-id="fe422-113">Wykonanie odroczone oznacza to, czy operacja nie jest wykonywana w momencie w kodzie gdzie zapytanie jest zadeklarowany.</span><span class="sxs-lookup"><span data-stu-id="fe422-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="fe422-114">Operacja jest wykonywana tylko wtedy, gdy wyliczone zmienną zapytania, na przykład za pomocą `For Each` instrukcji.</span><span class="sxs-lookup"><span data-stu-id="fe422-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="fe422-115">Oznacza to, że wyniki wykonywania zapytania zależy od zawartości źródła danych podczas wykonywania zapytania zamiast kiedy zdefiniowano zapytania.</span><span class="sxs-lookup"><span data-stu-id="fe422-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="fe422-116">Jeśli do zmiennej zapytania jest wyliczyć wiele razy, wyniki mogą być inne zawsze.</span><span class="sxs-lookup"><span data-stu-id="fe422-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="fe422-117">Prawie wszystkie standardowe operatory zapytań o typie zwracanym <xref:System.Collections.Generic.IEnumerable%601> lub <xref:System.Linq.IOrderedEnumerable%601> w sposób opóźnieniem.</span><span class="sxs-lookup"><span data-stu-id="fe422-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="fe422-118">Operatory zapytań, które używają odroczonego wykonania można ponadto klasyfikowane jako przesyłania strumieniowego lub strumieniowo.</span><span class="sxs-lookup"><span data-stu-id="fe422-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="fe422-119">przesyłanie strumieniowe</span><span class="sxs-lookup"><span data-stu-id="fe422-119">Streaming</span></span>  
 <span data-ttu-id="fe422-120">Operatory przesyłania strumieniowego nie trzeba przed dają one elementy przeczytać źródło danych.</span><span class="sxs-lookup"><span data-stu-id="fe422-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="fe422-121">W czasie wykonywania operator przesyłania strumieniowego wykonuje jego działanie dla każdego elementu źródłowego, ponieważ jest do odczytu i daje w wyniku elementu, jeśli są one.</span><span class="sxs-lookup"><span data-stu-id="fe422-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="fe422-122">Operator przesyłania strumieniowego w dalszym ciągu odczytu elementy źródłowe, dopóki nie może być wywołany element wynik.</span><span class="sxs-lookup"><span data-stu-id="fe422-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="fe422-123">Oznacza to, że więcej niż jeden element źródłowy może przeczytać wygenerowało jeden element wynik.</span><span class="sxs-lookup"><span data-stu-id="fe422-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="fe422-124">Strumieniowo</span><span class="sxs-lookup"><span data-stu-id="fe422-124">Non-Streaming</span></span>  
 <span data-ttu-id="fe422-125">Operatory przesyłania strumieniowego nie musi odczytu źródło danych, przed dają one element wynik.</span><span class="sxs-lookup"><span data-stu-id="fe422-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="fe422-126">Operacje takie jak sortowania lub grupowania należą do tej kategorii.</span><span class="sxs-lookup"><span data-stu-id="fe422-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="fe422-127">W czasie wykonywania strumieniowo operatorów zapytań odczytywać źródło danych, poddane to struktura danych do operacji i uzyskanie wynikowych elementów.</span><span class="sxs-lookup"><span data-stu-id="fe422-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
