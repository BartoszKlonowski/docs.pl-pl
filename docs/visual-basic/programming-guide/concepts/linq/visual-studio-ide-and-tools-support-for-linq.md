# <a name="visual-studio-ide-and-tools-support-for-linq-visual-basic"></a>Visual Studio IDE i obsługę narzędzia LINQ (Visual Basic)
Visual Studio zintegrowane środowisko programistyczne (IDE) zapewnia następujące funkcje obsługujące LINQ projektowanie aplikacji:  
  
## <a name="object-relational-designer"></a>Object Relational Designer  
 Projektant obiektów relacyjnych to narzędzie wizualnego projektu, które można używać w [LINQ do SQL](https://msdn.microsoft.com/library/bb386976) aplikacji można wygenerować klas w języku Visual Basic, które reprezentują danych relacyjnych w podstawowej bazy danych. Aby uzyskać więcej informacji, zobacz [składnika LINQ to SQL narzędzia w programie Visual Studio](/visualstudio/data-tools/linq-to-sql-tools-in-visual-studio2).  
  
## <a name="sqlmetal-command-line-tool"></a>SQLMetal narzędzia wiersza polecenia  
 SQLMetal jest narzędziem wiersza polecenia, które można użyć w procesach kompilacji można wygenerować klas z istniejących baz danych do użycia w składniku LINQ do SQL aplikacji. Aby uzyskać więcej informacji, zobacz [SqlMetal.exe (narzędzie generowania kodu)](http://msdn.microsoft.com/library/819e5a96-7646-4fdb-b14b-fe31221b0614).  
  
## <a name="linq-aware-code-editor"></a>Edytor kodu obsługujący LINQ  
 Edytor kodu języka Visual Basic obsługuje LINQ często z IntelliSense i możliwości formatowania.  
  
## <a name="visual-studio-debugger-support"></a>Obsługa debugera programu Visual Studio  
 Debuger programu Visual Studio obsługuje debugowanie wyrażenia zapytania. Aby uzyskać więcej informacji, zobacz [debugowanie LINQ](/visualstudio/debugger/debugging-linq).
