## <a name="best-practices"></a><span data-ttu-id="e350d-117">Najlepsze rozwiązania</span><span class="sxs-lookup"><span data-stu-id="e350d-117">Best Practices</span></span>

### <a name="enable-security-with-peer-channel-applications"></a><span data-ttu-id="e350d-119">Włącz zabezpieczenia za pomocą aplikacji kanału równorzędnego</span><span class="sxs-lookup"><span data-stu-id="e350d-119">Enable Security with Peer Channel Applications</span></span>  

 <span data-ttu-id="e350d-120">Ze względu na rozproszony charakter protokołów kanałów równorzędnych trudno jest wymusić członkostwo w sieci, poufność i prywatność w niezabezpieczonej sieci.</span><span class="sxs-lookup"><span data-stu-id="e350d-120">Due to the distributed nature of the Peer Channel protocols, it is hard to enforce mesh membership, confidentiality, and privacy in an unsecured mesh.</span></span> <span data-ttu-id="e350d-121">Należy również pamiętać o zabezpieczeniu komunikacji między klientami a usługą rozpoznawania nazw.</span><span class="sxs-lookup"><span data-stu-id="e350d-121">It is also important to remember to secure communication between clients and the resolver service.</span></span> <span data-ttu-id="e350d-122">W obszarze Protokół rozpoznawania nazw równorzędnych (PNRP) używaj bezpiecznych nazw, aby uniknąć fałszowania i innych typowych ataków.</span><span class="sxs-lookup"><span data-stu-id="e350d-122">Under Peer Name Resolution Protocol (PNRP), use secure names to avoid spoofing and other common attacks.</span></span> <span data-ttu-id="e350d-123">Zabezpiecz niestandardową usługę programu rozpoznawania nazw, włączając zabezpieczenia na klientach połączenia używane do kontaktowania się z usługą rozpoznawania nazw, w tym zabezpieczeniami komunikatów i opartych na transportach.</span><span class="sxs-lookup"><span data-stu-id="e350d-123">Secure a custom resolver service by enabling security on the connection clients use to contact the resolver service, including both message- and transport-based security.</span></span>

 <span data-ttu-id="e350d-125">Na przykład, jeśli każdy element członkowski siatki musi być zidentyfikowany indywidualnie, użyj modelu uwierzytelniania opartego na certyfikatach.</span><span class="sxs-lookup"><span data-stu-id="e350d-125">For example, if each member of the mesh needs to be individually identified, use certificate-based authentication model.</span></span> <span data-ttu-id="e350d-126">Jeśli nie jest to możliwe, należy użyć uwierzytelniania opartego na hasłach zgodnie z bieżącymi zaleceniami, aby zapewnić ich bezpieczeństwo.</span><span class="sxs-lookup"><span data-stu-id="e350d-126">If that is not possible, use password-based authentication following current recommendations to keep them secure.</span></span> <span data-ttu-id="e350d-127">Dotyczy to również udostępniania haseł tylko zaufanym podmiotom, przesyłania haseł przy użyciu bezpiecznego nośnika, często zmieniających się haseł i zapewniania silnych haseł (co najmniej osiem znaków).</span><span class="sxs-lookup"><span data-stu-id="e350d-127">This includes sharing passwords only with trusted parties, transmitting passwords using a secure medium, changing passwords frequently, and ensuring that passwords are strong (at least eight characters long, include at least one letter from both cases, a digit, and a special character).</span></span>  
  
### <a name="never-accept-self-signed-certificates"></a><span data-ttu-id="e350d-128">Nigdy nie Akceptuj Self-Signed certyfikatów</span><span class="sxs-lookup"><span data-stu-id="e350d-128">Never Accept Self-Signed Certificates</span></span>

 <span data-ttu-id="e350d-133">Użyj uwierzytelniania wiadomości, aby sprawdzić, czy wiadomość pochodzi z zaufanego źródła i czy nikt nie został naruszony przez komunikat podczas przesyłania.</span><span class="sxs-lookup"><span data-stu-id="e350d-133">Use message authentication to verify that a message originated from a trusted source and that no one has tampered with the message during transmission.</span></span> <span data-ttu-id="e350d-134">Bez uwierzytelniania wiadomości, złośliwy klient może łatwo sfałszować lub naruszać komunikaty w sieci.</span><span class="sxs-lookup"><span data-stu-id="e350d-134">Without message authentication, it is easy for a malicious client to spoof or tamper with messages in the mesh.</span></span>
