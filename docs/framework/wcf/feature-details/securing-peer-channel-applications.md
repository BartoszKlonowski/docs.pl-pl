## <a name="best-practices"></a><span data-ttu-id="2ea01-117">Najlepsze praktyki</span><span class="sxs-lookup"><span data-stu-id="2ea01-117">Best Practices</span></span>

### <a name="enable-security-with-peer-channel-applications"></a><span data-ttu-id="2ea01-119">Włącz zabezpieczenie z aplikacji kanałów równorzędnych</span><span class="sxs-lookup"><span data-stu-id="2ea01-119">Enable Security with Peer Channel Applications</span></span>  
 <span data-ttu-id="2ea01-120">Z powodu Rozproszony charakter protokoły kanału równorzędnego jest trudne do wymuszenia członkostwa siatki, poufność i ochrona prywatności w niezabezpieczonej sieci.</span><span class="sxs-lookup"><span data-stu-id="2ea01-120">Due to the distributed nature of the Peer Channel protocols, it is hard to enforce mesh membership, confidentiality, and privacy in an unsecured mesh.</span></span> <span data-ttu-id="2ea01-121">Jest również pamiętać do zabezpieczania komunikacji między klientami i usługi rozpoznawania nazw.</span><span class="sxs-lookup"><span data-stu-id="2ea01-121">It is also important to remember to secure communication between clients and the resolver service.</span></span> <span data-ttu-id="2ea01-122">W obszarze rozpoznawania protokołu PNRP (Peer Name), za pomocą bezpiecznego nazwy w celu zapobiegania atakom metodą preparowania i inne typowe ataki.</span><span class="sxs-lookup"><span data-stu-id="2ea01-122">Under Peer Name Resolution Protocol (PNRP), use secure names to avoid spoofing and other common attacks.</span></span> <span data-ttu-id="2ea01-123">Zabezpieczanie usługi niestandardowego programu rozpoznawania nazw przez włączenie zabezpieczeń przy użyciu klientów połączenia do kontaktowania się z usługi rozpoznawania nazw, w tym zabezpieczeń zarówno na podstawie komunikatu i transportu.</span><span class="sxs-lookup"><span data-stu-id="2ea01-123">Secure a custom resolver service by enabling security on the connection clients use to contact the resolver service, including both message- and transport-based security.</span></span>  
  
### <a name="use-the-strongest-possible-security-model"></a><span data-ttu-id="2ea01-124">Użyj modelu zabezpieczeń możliwe najwyższego poziomu</span><span class="sxs-lookup"><span data-stu-id="2ea01-124">Use the Strongest Possible Security Model</span></span>  
 <span data-ttu-id="2ea01-125">Na przykład jeśli każdy element członkowski siatki musi się indywidualnie identyfikację, użyj model uwierzytelniania opartego na certyfikatach.</span><span class="sxs-lookup"><span data-stu-id="2ea01-125">For example, if each member of the mesh needs to be individually identified, use certificate-based authentication model.</span></span> <span data-ttu-id="2ea01-126">Jeśli nie jest to możliwe, uwierzytelnianie oparte na hasłach następujących zalecanych rozwiązań w celu zachowania ich zabezpieczeń.</span><span class="sxs-lookup"><span data-stu-id="2ea01-126">If that is not possible, use password-based authentication following current recommendations to keep them secure.</span></span> <span data-ttu-id="2ea01-127">Dotyczy to również udostępnianie haseł, tylko z zaufanych stronami, przesyłania hasła przy użyciu bezpiecznego średniej, zmiana haseł często i sprawdzeniu, czy hasła są silne (co najmniej osiem znaków długości, zawierać co najmniej jedną literę w obu przypadkach cyfrę, i znak specjalny).</span><span class="sxs-lookup"><span data-stu-id="2ea01-127">This includes sharing passwords only with trusted parties, transmitting passwords using a secure medium, changing passwords frequently, and ensuring that passwords are strong (at least eight characters long, include at least one letter from both cases, a digit, and a special character).</span></span>  
  
### <a name="never-accept-self-signed-certificates"></a><span data-ttu-id="2ea01-128">Nie pobieraj certyfikaty z podpisem własnym</span><span class="sxs-lookup"><span data-stu-id="2ea01-128">Never Accept Self-Signed Certificates</span></span>  
 <span data-ttu-id="2ea01-129">Nie pobieraj poświadczenie certyfikatu na podstawie nazw podmiotu.</span><span class="sxs-lookup"><span data-stu-id="2ea01-129">Never accept a certificate credential based on subject names.</span></span> <span data-ttu-id="2ea01-130">Należy pamiętać, że każdy użytkownik może utworzyć certyfikat, a każdy użytkownik może wybrać nazwę, która jest sprawdzana poprawność.</span><span class="sxs-lookup"><span data-stu-id="2ea01-130">Note that anyone can create a certificate, and anyone can choose a name that you are validating.</span></span> <span data-ttu-id="2ea01-131">Aby uniknąć możliwość fałszowania, sprawdzić poprawność certyfikatów opartych na wystawienie poświadczenia urzędu (zaufanych wystawców lub głównego urzędu certyfikacji).</span><span class="sxs-lookup"><span data-stu-id="2ea01-131">To avoid the possibility of spoofing, validate certificates based on issuing authority credentials (either a trusted issuer or a root certification authority).</span></span>  
  
### <a name="use-message-authentication"></a><span data-ttu-id="2ea01-132">Uwierzytelnianie wiadomości</span><span class="sxs-lookup"><span data-stu-id="2ea01-132">Use Message Authentication</span></span>  
 <span data-ttu-id="2ea01-133">Sprawdź, czy komunikat pochodzi z zaufanego źródła i czy nie został zmieniony przez niepowołane komunikat podczas przesyłania za pomocą uwierzytelniania wiadomości.</span><span class="sxs-lookup"><span data-stu-id="2ea01-133">Use message authentication to verify that a message originated from a trusted source and that no one has tampered with the message during transmission.</span></span> <span data-ttu-id="2ea01-134">Bez uwierzytelniania wiadomości jest łatwe złośliwego klienta fałszowania lub naruszyć wiadomości w sieci.</span><span class="sxs-lookup"><span data-stu-id="2ea01-134">Without message authentication, it is easy for a malicious client to spoof or tamper with messages in the mesh.</span></span>
