- <span data-ttu-id="1bc23-112">[Niezawodne sesje](../../../../docs/framework/wcf/feature-details/reliable-sessions.md), które oferują niezawodne przesyłanie bez ponoszenia kosztów duże opóźnienie</span><span class="sxs-lookup"><span data-stu-id="1bc23-112">[Reliable Sessions](../../../../docs/framework/wcf/feature-details/reliable-sessions.md), which offer reliable transfer without the cost of high latency</span></span>  
  
- <span data-ttu-id="1bc23-113">[Kolejki programu WCF](../../../../docs/framework/wcf/feature-details/queues-in-wcf.md), które oferują niezawodne transferu i separacji między źródłową i docelową.</span><span class="sxs-lookup"><span data-stu-id="1bc23-113">[Queues in WCF](../../../../docs/framework/wcf/feature-details/queues-in-wcf.md), which offer both reliable transfers and separation between the source and the destination.</span></span>  
  
## <a name="reliable-sessions"></a><span data-ttu-id="1bc23-114">Niezawodne sesje</span><span class="sxs-lookup"><span data-stu-id="1bc23-114">Reliable Sessions</span></span>  
 <span data-ttu-id="1bc23-115">Niezawodne sesje zapewniają end-to-end niezawodne przesyłanie wiadomości między źródłem i miejsca docelowego przy użyciu protokołu WS-ReliableMessaging niezależnie od liczby i typu pośredników, których punkty końcowe komunikatów (źródłowy i docelowy).</span><span class="sxs-lookup"><span data-stu-id="1bc23-115">Reliable sessions provide end-to-end reliable transfer of messages between a source and a destination using the WS-ReliableMessaging protocol regardless of the number or type of intermediaries that separate the messaging (source and destination) endpoints.</span></span> <span data-ttu-id="1bc23-116">Dotyczy to również wszelkich pośredników transportu, które nie korzystają z protokołu SOAP (na przykład serwera proxy HTTP) lub pośredników, które używają protokołu SOAP (na przykład opartego na protokole SOAP routery lub mostków), które są wymagane dla komunikatów przepływ między punktami końcowymi.</span><span class="sxs-lookup"><span data-stu-id="1bc23-116">This includes any transport intermediaries that do not use SOAP (for example, HTTP proxies) or intermediaries that use SOAP (for example, SOAP-based routers or bridges) that are required for messages to flow between the endpoints.</span></span> <span data-ttu-id="1bc23-117">Niezawodne sesje okno transfer w pamięci na awarie poziom komunikatu protokołu SOAP maski i ponownie ustanowić połączenia w przypadku błędów transportu.</span><span class="sxs-lookup"><span data-stu-id="1bc23-117">Reliable sessions use an in-memory transfer window to mask SOAP message-level failures and re-establish connections in the case of transport failures.</span></span>

 <span data-ttu-id="1bc23-122">Kolejki programu WCF zapewnia zarówno niezawodne transferu wiadomości i separacji między źródeł i miejsc docelowych kosztem duże opóźnienie.</span><span class="sxs-lookup"><span data-stu-id="1bc23-122">Queues in WCF provide both reliable transfers of messages and separation between sources and destinations at the cost of high latency.</span></span> <span data-ttu-id="1bc23-123">Usługi WCF w kolejce komunikacji jest zbudowana na usługi kolejkowania komunikatów (MSMQ).</span><span class="sxs-lookup"><span data-stu-id="1bc23-123">WCF queued communication is built on top of Message Queuing (also known as MSMQ).</span></span>

Soap Reliable Messaging Protocol (SRMP).</span><span class="sxs-lookup"><span data-stu-id="1bc23-128">The protocol can be native or SOAP-based, such as Soap Reliable Messaging Protocol (SRMP).</span></span>  
  
 <span data-ttu-id="1bc23-129">Oddzielenie połączone z przeniesienia niezawodnych komunikatów między kolejek umożliwia aplikacji, które są luźno powiązane niezawodnie komunikować się.</span><span class="sxs-lookup"><span data-stu-id="1bc23-129">The separation, coupled with reliable message transfers between queues, enables applications that are loosely coupled to communicate reliably.</span></span> <span data-ttu-id="1bc23-130">W przeciwieństwie do sesji niezawodnych źródłowym i docelowym nie musi być uruchomiona w tym samym czasie.</span><span class="sxs-lookup"><span data-stu-id="1bc23-130">Unlike reliable sessions, the source and destination do not have to be running at the same time.</span></span> <span data-ttu-id="1bc23-131">Dzięki temu niejawnie scenariuszy, w którym kolejki w efekcie służą jako mechanizm wyrównywania obciążenia w przypadku niezgodności między za produkcji wiadomości według źródeł oraz stawkę za użycie komunikatu przez miejsce docelowe.</span><span class="sxs-lookup"><span data-stu-id="1bc23-131">This implicitly enables scenarios where queues are, in effect, used as a load-leveling mechanism when there is a mismatch between the rate of message production by the source and the rate of the message consumption by the destination.</span></span> <span data-ttu-id="1bc23-132">Aby uzyskać więcej informacji na temat kolejek, zobacz [kolejki programu WCF](../../../../docs/framework/wcf/feature-details/queues-in-wcf.md).</span><span class="sxs-lookup"><span data-stu-id="1bc23-132">For more information about queues, see [Queues in WCF](../../../../docs/framework/wcf/feature-details/queues-in-wcf.md).</span></span>
