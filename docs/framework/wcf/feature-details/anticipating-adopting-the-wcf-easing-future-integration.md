## <a name="protocols"></a><span data-ttu-id="97be1-108">Protokoły</span><span class="sxs-lookup"><span data-stu-id="97be1-108">Protocols</span></span>

 <span data-ttu-id="97be1-120">Przyjmując takie podejście, powinno być możliwe zapewnienie później klas .NET w kontraktach danych dodając <xref:System.Runtime.Serialization.DataContractAttribute> i <xref:System.Runtime.Serialization.DataMemberAttribute> bez zmiany znacznie XML, do którego klasy są serializowane w celu przesłania go.</span><span class="sxs-lookup"><span data-stu-id="97be1-120">By adopting this approach, you should be able to later make the .NET classes into data contracts with the addition of the <xref:System.Runtime.Serialization.DataContractAttribute> and <xref:System.Runtime.Serialization.DataMemberAttribute> without significantly altering the XML into which the classes are serialized for transmission.</span></span> <span data-ttu-id="97be1-121">Typy używane w komunikatach przez usługi sieci Web platformy ASP.NET będzie do przetworzenia jako kontraktów danych przez aplikacje WCF reaguje, oraz inne korzyści, lepszej wydajności w aplikacjach usługi WCF.</span><span class="sxs-lookup"><span data-stu-id="97be1-121">The types used in messages by ASP.NET Web services will be able to be processed as data contracts by WCF applications, yielding, amongst other benefits, better performance in WCF applications.</span></span>
