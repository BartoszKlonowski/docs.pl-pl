 <span data-ttu-id="f464b-108">Scenariusz dystrybucja zawartości wymaga, że każdy odbiorca zawartości zidentyfikować źródła zawartości.</span><span class="sxs-lookup"><span data-stu-id="f464b-108">A content-distribution scenario requires that each content recipient identify the content source.</span></span> <span data-ttu-id="f464b-109">Z powodu Rozproszony charakter tego scenariusza nie zawsze jest możliwe znanych i zaufanych pośredników tego procesu lub przechwycenia wiadomości.</span><span class="sxs-lookup"><span data-stu-id="f464b-109">Due to the distributed nature of the scenario, it is not always possible to know and trust the intermediaries that process or intercept messages.</span></span> <span data-ttu-id="f464b-110">W celu wyeliminowania zagrożenia, które pośrednik niezaufanych może manipulować komunikaty, aplikacje można zabezpieczyć wiadomość na adres nadawcy tak, aby wszelkie prób manipulowania przy łatwo są wykrywane.</span><span class="sxs-lookup"><span data-stu-id="f464b-110">To effectively mitigate the threat that an untrusted intermediary might tamper with the messages, applications can secure the message at the sender so that any tampering attempts are easily detected.</span></span> <span data-ttu-id="f464b-111">W takim przypadku w zależności od poufność zawartość szyfrowania może być konieczne.</span><span class="sxs-lookup"><span data-stu-id="f464b-111">In this case, depending on the confidentiality of the content, encryption can be necessary.</span></span>  
  
 <span data-ttu-id="f464b-112">Współpraca scenariuszy, takich jak grupy współpracy nad dokumentami często wymagają, aby każdy element członkowski uczestniczących w sesji można indywidualnie zidentyfikowane i uwierzytelniony.</span><span class="sxs-lookup"><span data-stu-id="f464b-112">Collaboration scenarios like group document collaboration often require that each member participating in the session be individually identified and authenticated.</span></span> <span data-ttu-id="f464b-113">Oznacza to, że mechanizm Definiowanie grup użytkowników i uwierzytelniania względem tych grup ma konieczne zabezpieczonych sesji.</span><span class="sxs-lookup"><span data-stu-id="f464b-113">This means that a mechanism to define user groups and authenticate against those groups is necessary to have a secured session.</span></span> <span data-ttu-id="f464b-114">Ponadto aplikacje mogą wymagać śledzenie każdy komunikat uwierzytelniania na poziomie wiadomości.</span><span class="sxs-lookup"><span data-stu-id="f464b-114">Moreover, applications might require tracing each message by authentication at the message level.</span></span> <span data-ttu-id="f464b-115">W tych typów aplikacji wydajności mogą zostać zachowane systemu silniejszych zabezpieczeń.</span><span class="sxs-lookup"><span data-stu-id="f464b-115">In these types of applications, performance can be sacrificed for a stronger security scheme.</span></span>  
  
 <span data-ttu-id="f464b-116">Sesja komunikacji między grupy zwykłych użytkowników może wymagać modele nieformalne zabezpieczeń, takich jak wiedzę na temat wspólnego klucza tajnego w grupie.</span><span class="sxs-lookup"><span data-stu-id="f464b-116">A communication session among a group of casual users can require informal security models, like knowledge of a common secret among the group.</span></span> <span data-ttu-id="f464b-117">W przypadku tych typów aplikacji o modelu zabezpieczeń, który jest wygodne i skonfigurować ma większe znaczenie niż posiadające najwyższy formy uwierzytelniania lub zapewnienie środków niemożność.</span><span class="sxs-lookup"><span data-stu-id="f464b-117">For these types of applications, having a security model that is convenient to establish and configure is more important than having the strongest form of authentication or providing nonrepudiation measures.</span></span> <span data-ttu-id="f464b-118">W tych sytuacjach mechanizm uwierzytelniania opartego na hasłach pomaga zabezpieczyć warstwa komunikacji, umożliwiając uwierzytelniania wiadomości.</span><span class="sxs-lookup"><span data-stu-id="f464b-118">For these scenarios, a password-based authentication mechanism helps to secure the communication layer while still allowing for message authentication.</span></span> <span data-ttu-id="f464b-119">Zabezpieczenia oparte na hasłach jest ustawieniem domyślnym dla kanału równorzędnego.</span><span class="sxs-lookup"><span data-stu-id="f464b-119">Password-based security is the default setting for Peer Channel.</span></span>  
  
## <a name="token-types"></a><span data-ttu-id="f464b-120">Typy tokenów</span><span class="sxs-lookup"><span data-stu-id="f464b-120">Token Types</span></span>  
 <span data-ttu-id="f464b-121">Kanał elementu równorzędnego rozpoznaje jeden typ tokenu do identyfikacji silne certyfikatów X.509, które dostarczają oparty na typie uwierzytelniania i autoryzacji, które można wdrożyć modelu silnej tożsamości.</span><span class="sxs-lookup"><span data-stu-id="f464b-121">Peer Channel recognizes a single token type for strong identification, X.509 certificates, which provide a strong identity model based on the type of authentication and authorization that can be implemented.</span></span> <span data-ttu-id="f464b-122">Poufność i integralności łatwo znajdują się za pomocą certyfikatów.</span><span class="sxs-lookup"><span data-stu-id="f464b-122">Confidentiality and integrity are easily provided using certificates.</span></span> <span data-ttu-id="f464b-123">Jednak certyfikaty X.509 może być trudne do użycia i wdrażania.</span><span class="sxs-lookup"><span data-stu-id="f464b-123">However, X.509 certificates can be difficult to use and deploy.</span></span>  
  
 <span data-ttu-id="f464b-124">Kanał elementu równorzędnego także zapewnia obsługę proste aplikacje przy użyciu hasła.</span><span class="sxs-lookup"><span data-stu-id="f464b-124">Peer Channel also provides support for simple applications through the use of passwords.</span></span> <span data-ttu-id="f464b-125">Aplikacje można wybrać do skonfigurowania grup równorzędnych szybkie i proste, w oparciu o podane hasło.</span><span class="sxs-lookup"><span data-stu-id="f464b-125">Applications can choose to set up quick and simple peer groups based on a supplied password.</span></span> <span data-ttu-id="f464b-126">W takim przypadku właściciel grupy decyduje i komunikuje się hasło do elementów członkowskich.</span><span class="sxs-lookup"><span data-stu-id="f464b-126">In this case, a group owner decides and communicates the password to members.</span></span> <span data-ttu-id="f464b-127">Każdy element członkowski musi Zaloguj się przy użyciu tego hasła, zanim one dołączyć do sesji.</span><span class="sxs-lookup"><span data-stu-id="f464b-127">Each member must sign in using this password before they can join the session.</span></span> <span data-ttu-id="f464b-128">Hasła mogą być wykorzystywane tylko w celu wprowadzania z sesją; Nie można ich używany do uwierzytelniania wiadomości.</span><span class="sxs-lookup"><span data-stu-id="f464b-128">Passwords can be used only to allow entry to the session; they cannot be used to perform message authentication.</span></span> <span data-ttu-id="f464b-129">Jest to spowodowane tokenu symetrycznego, który udział grupy elementów równorzędnych jest trudne i nieodpowiednie do użycia na potrzeby uwierzytelniania źródła.</span><span class="sxs-lookup"><span data-stu-id="f464b-129">This is because a symmetric token that a group of peers share is difficult and inappropriate to use for source authentication.</span></span>  
  
## <a name="security-model"></a><span data-ttu-id="f464b-130">Model zabezpieczeń</span><span class="sxs-lookup"><span data-stu-id="f464b-130">Security Model</span></span>  
 <span data-ttu-id="f464b-131">Kanał elementu równorzędnego umożliwia zabezpieczanie poszczególnych łączy między komputerami równorzędnymi.</span><span class="sxs-lookup"><span data-stu-id="f464b-131">Peer Channel provides the ability to secure the individual links between peers.</span></span> <span data-ttu-id="f464b-132">Oznacza to, że komunikat nigdy nie przepływa łącze niezabezpieczona (z perspektywy aplikacji).</span><span class="sxs-lookup"><span data-stu-id="f464b-132">This means that a message never flows on an unsecured link (from the application perspective).</span></span> <span data-ttu-id="f464b-133">Wewnętrznie każdego łącza (kanał transportu między dwoma komputerami) jest zabezpieczona przy użyciu zabezpieczeń TLS (Transport Layer).</span><span class="sxs-lookup"><span data-stu-id="f464b-133">Internally, each link (a transport channel between two peers) is secured using Transport Layer Security (TLS).</span></span> <span data-ttu-id="f464b-134">Oznacza to, że gdy nadawcy Redaguj i wysyła wiadomość jest wysyłana za pośrednictwem bezpiecznego transportu do każdego z jego natychmiastowego elementów równorzędnych, którzy dostępu do wiadomości, a to z kolei wysyła komunikat do natychmiastowego między sobą za pośrednictwem bezpiecznego połączenia.</span><span class="sxs-lookup"><span data-stu-id="f464b-134">This means that when a sender composes and sends a message, it is sent over secure transport to each of its immediate peers, who access the message, and in turn send the message to their immediate peers over secure connections.</span></span> <span data-ttu-id="f464b-135">Zabezpieczeń działa tylko na transport poziomu i jest niezależna od modele zabezpieczeń wiadomości.</span><span class="sxs-lookup"><span data-stu-id="f464b-135">This security only works at the transport level and is independent of the message security models.</span></span>  
  
 <span data-ttu-id="f464b-136">Kanał elementu równorzędnego umożliwia również Zabezpieczanie komunikatów niezależnie od użyć zabezpieczeń transportu.</span><span class="sxs-lookup"><span data-stu-id="f464b-136">Peer Channel also provides a way to secure messages independently of the transport security used.</span></span> <span data-ttu-id="f464b-137">W tym modelu wiadomość jest zabezpieczona w źródle, przy użyciu tokenu zabezpieczającego źródła, chociaż obecnie tylko certyfikaty X.509 są obsługiwane.</span><span class="sxs-lookup"><span data-stu-id="f464b-137">In this model, the message is secured at the source using the source’s security token, although currently only X.509 certificates are supported.</span></span> <span data-ttu-id="f464b-138">Zabezpieczoną wiadomość jest następnie transmisji w sieci równorzędnej.</span><span class="sxs-lookup"><span data-stu-id="f464b-138">The secured message is then transmitted over the peer network.</span></span> <span data-ttu-id="f464b-139">Każdy komputer odbierający można zweryfikować autentyczności źródła.</span><span class="sxs-lookup"><span data-stu-id="f464b-139">Each receiving peer can verify the authenticity of the source.</span></span> <span data-ttu-id="f464b-140">Należy pamiętać, że wiadomość jest zabezpieczona, dzięki czemu pośredników manipulowanie nie można go.</span><span class="sxs-lookup"><span data-stu-id="f464b-140">Note that the message is secured so that intermediaries cannot tamper with it.</span></span>  
  
 <span data-ttu-id="f464b-141">Aby osiągnąć poufność, aplikacje można wdrożyć zabezpieczenia transportu z schematów członkostwa grup silne, aby uniemożliwić nieautoryzowany dostęp do wiadomości.</span><span class="sxs-lookup"><span data-stu-id="f464b-141">To achieve confidentiality, applications can employ transport security with strong group membership schemes to prevent unauthorized access to the message.</span></span>  
  
 <span data-ttu-id="f464b-142">Kanał elementu równorzędnego nie wymaga modelu określonej tożsamości, tak długo, jak aplikacja wybierze jeden z obsługiwanych typów tokenu.</span><span class="sxs-lookup"><span data-stu-id="f464b-142">Peer Channel does not require a specific identity model as long as the application chooses one of the supported token types.</span></span> <span data-ttu-id="f464b-143">Aplikacje całkowicie właścicielem cyklu życia tożsamości i decyzje dotyczące uwierzytelniania.</span><span class="sxs-lookup"><span data-stu-id="f464b-143">Applications completely own the life cycle of these identities and authentication decisions.</span></span>
