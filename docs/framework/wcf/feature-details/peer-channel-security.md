## <a name="security-scenarios"></a><span data-ttu-id="58801-107">Scenariusze zabezpieczeń</span><span class="sxs-lookup"><span data-stu-id="58801-107">Security Scenarios</span></span>  
 <span data-ttu-id="58801-108">Scenariusz dystrybucji zawartości wymaga, aby każdy odbiorca zawartości zidentyfikował źródło zawartości.</span><span class="sxs-lookup"><span data-stu-id="58801-108">A content-distribution scenario requires that each content recipient identify the content source.</span></span> <span data-ttu-id="58801-109">Ze względu na dystrybuowany charakter scenariusza nie zawsze jest możliwe poznanie i ufanie pośrednikom, które przetwarzają lub przechwytuje komunikaty.</span><span class="sxs-lookup"><span data-stu-id="58801-109">Due to the distributed nature of the scenario, it is not always possible to know and trust the intermediaries that process or intercept messages.</span></span> <span data-ttu-id="58801-110">Aby skutecznie ograniczyć zagrożenie, że niezaufanego pośrednika może naruszać komunikaty, aplikacje mogą zabezpieczyć komunikat na nadawcy, aby umożliwić łatwe wykrywanie wszelkich prób naruszenia.</span><span class="sxs-lookup"><span data-stu-id="58801-110">To effectively mitigate the threat that an untrusted intermediary might tamper with the messages, applications can secure the message at the sender so that any tampering attempts are easily detected.</span></span> <span data-ttu-id="58801-111">W takim przypadku, w zależności od poufności zawartości, może być konieczne szyfrowanie.</span><span class="sxs-lookup"><span data-stu-id="58801-111">In this case, depending on the confidentiality of the content, encryption can be necessary.</span></span>  
  
 <span data-ttu-id="58801-112">Scenariusze współpracy, takie jak współpraca dokumentów grup, często wymagają, aby każdy członek uczestniczący w sesji był indywidualnie identyfikowany i uwierzytelniony.</span><span class="sxs-lookup"><span data-stu-id="58801-112">Collaboration scenarios like group document collaboration often require that each member participating in the session be individually identified and authenticated.</span></span> <span data-ttu-id="58801-113">Oznacza to, że mechanizm definiowania grup użytkowników i uwierzytelniania w odniesieniu do tych grup jest niezbędny do posiadania zabezpieczonej sesji.</span><span class="sxs-lookup"><span data-stu-id="58801-113">This means that a mechanism to define user groups and authenticate against those groups is necessary to have a secured session.</span></span> <span data-ttu-id="58801-114">Ponadto aplikacje mogą wymagać śledzenia poszczególnych komunikatów przez uwierzytelnianie na poziomie wiadomości.</span><span class="sxs-lookup"><span data-stu-id="58801-114">Moreover, applications might require tracing each message by authentication at the message level.</span></span> <span data-ttu-id="58801-115">W przypadku tych typów aplikacji można wykluczać wydajność dla silniejszego schematu zabezpieczeń.</span><span class="sxs-lookup"><span data-stu-id="58801-115">In these types of applications, performance can be sacrificed for a stronger security scheme.</span></span>  
  
 <span data-ttu-id="58801-116">Sesja komunikacji między grupą użytkowników niezawodnych może wymagać nieformalnych modeli zabezpieczeń, takich jak znajomość wspólnego wpisu tajnego między grupą.</span><span class="sxs-lookup"><span data-stu-id="58801-116">A communication session among a group of casual users can require informal security models, like knowledge of a common secret among the group.</span></span> <span data-ttu-id="58801-117">W przypadku tych typów aplikacji model zabezpieczeń, który jest wygodny do ustanowienia i skonfigurowania, jest ważniejszy niż najwyższy sposób uwierzytelniania lub zapewniający środki.</span><span class="sxs-lookup"><span data-stu-id="58801-117">For these types of applications, having a security model that is convenient to establish and configure is more important than having the strongest form of authentication or providing nonrepudiation measures.</span></span> <span data-ttu-id="58801-118">W tych scenariuszach mechanizm uwierzytelniania oparty na hasłach pomaga zabezpieczyć warstwę komunikacji i nadal zezwala na uwierzytelnianie komunikatów.</span><span class="sxs-lookup"><span data-stu-id="58801-118">For these scenarios, a password-based authentication mechanism helps to secure the communication layer while still allowing for message authentication.</span></span> <span data-ttu-id="58801-119">Zabezpieczenia oparte na hasłach są ustawieniem domyślnym kanału równorzędnego.</span><span class="sxs-lookup"><span data-stu-id="58801-119">Password-based security is the default setting for Peer Channel.</span></span>  
  
## <a name="token-types"></a><span data-ttu-id="58801-120">Typy tokenów</span><span class="sxs-lookup"><span data-stu-id="58801-120">Token Types</span></span>  
 <span data-ttu-id="58801-121">Kanał równorzędny rozpoznaje typ pojedynczego tokenu dla silnej identyfikacji, certyfikaty X. 509, które zapewniają silny model tożsamości na podstawie typu uwierzytelniania i autoryzacji, które mogą być implementowane.</span><span class="sxs-lookup"><span data-stu-id="58801-121">Peer Channel recognizes a single token type for strong identification, X.509 certificates, which provide a strong identity model based on the type of authentication and authorization that can be implemented.</span></span> <span data-ttu-id="58801-122">Poufność i integralność można łatwo zapewnić przy użyciu certyfikatów.</span><span class="sxs-lookup"><span data-stu-id="58801-122">Confidentiality and integrity are easily provided using certificates.</span></span> <span data-ttu-id="58801-123">Jednak certyfikaty X. 509 mogą być trudne do użycia i wdrożenia.</span><span class="sxs-lookup"><span data-stu-id="58801-123">However, X.509 certificates can be difficult to use and deploy.</span></span>  
  
 <span data-ttu-id="58801-124">Kanał równorzędny zapewnia również obsługę prostych aplikacji przy użyciu haseł.</span><span class="sxs-lookup"><span data-stu-id="58801-124">Peer Channel also provides support for simple applications through the use of passwords.</span></span> <span data-ttu-id="58801-125">Aplikacje mogą zdecydować się na skonfigurowanie szybkich i prostych grup równorzędnych na podstawie podanego hasła.</span><span class="sxs-lookup"><span data-stu-id="58801-125">Applications can choose to set up quick and simple peer groups based on a supplied password.</span></span> <span data-ttu-id="58801-126">W takim przypadku właściciel grupy decyduje o haśle i przekazuje je do członków.</span><span class="sxs-lookup"><span data-stu-id="58801-126">In this case, a group owner decides and communicates the password to members.</span></span> <span data-ttu-id="58801-127">Każdy element członkowski musi zalogować się przy użyciu tego hasła, aby można było dołączyć do sesji.</span><span class="sxs-lookup"><span data-stu-id="58801-127">Each member must sign in using this password before they can join the session.</span></span> <span data-ttu-id="58801-128">Haseł można używać tylko w celu zezwalania na wpis w sesji; nie można ich używać do uwierzytelniania wiadomości.</span><span class="sxs-lookup"><span data-stu-id="58801-128">Passwords can be used only to allow entry to the session; they cannot be used to perform message authentication.</span></span> <span data-ttu-id="58801-129">Jest to spowodowane tym, że symetryczny token, który jest używany przez grupę elementów równorzędnych, jest trudny i nieodpowiedni do uwierzytelniania źródłowego.</span><span class="sxs-lookup"><span data-stu-id="58801-129">This is because a symmetric token that a group of peers share is difficult and inappropriate to use for source authentication.</span></span>  
  
## <a name="security-model"></a><span data-ttu-id="58801-130">Model zabezpieczeń</span><span class="sxs-lookup"><span data-stu-id="58801-130">Security Model</span></span>  
 <span data-ttu-id="58801-131">Kanał równorzędny zapewnia możliwość zabezpieczenia poszczególnych linków między elementami równorzędnymi.</span><span class="sxs-lookup"><span data-stu-id="58801-131">Peer Channel provides the ability to secure the individual links between peers.</span></span> <span data-ttu-id="58801-132">Oznacza to, że komunikat nigdy nie jest przesyłany w niezabezpieczonym łączu (z perspektywy aplikacji).</span><span class="sxs-lookup"><span data-stu-id="58801-132">This means that a message never flows on an unsecured link (from the application perspective).</span></span> <span data-ttu-id="58801-133">Wewnętrznie każde łącze (kanał transportowy między dwoma elementami równorzędnymi) jest zabezpieczone przy użyciu Transport Layer Security (TLS).</span><span class="sxs-lookup"><span data-stu-id="58801-133">Internally, each link (a transport channel between two peers) is secured using Transport Layer Security (TLS).</span></span> <span data-ttu-id="58801-134">Oznacza to, że gdy nadawca redaguje i wyśle wiadomość, jest wysyłana za pośrednictwem bezpiecznego transportu do każdego z jego bezpośrednich elementów równorzędnych, który uzyskuje dostęp do wiadomości i z kolei wysyła komunikat do bezpośrednich elementów równorzędnych przez bezpieczne połączenia.</span><span class="sxs-lookup"><span data-stu-id="58801-134">This means that when a sender composes and sends a message, it is sent over secure transport to each of its immediate peers, who access the message, and in turn send the message to their immediate peers over secure connections.</span></span> <span data-ttu-id="58801-135">Zabezpieczenia te działają tylko na poziomie transportu i są niezależne od modeli zabezpieczeń komunikatów.</span><span class="sxs-lookup"><span data-stu-id="58801-135">This security only works at the transport level and is independent of the message security models.</span></span>  
  
 <span data-ttu-id="58801-136">Kanał równorzędny zapewnia również sposób zabezpieczania komunikatów niezależnie od używanego zabezpieczenia transportu.</span><span class="sxs-lookup"><span data-stu-id="58801-136">Peer Channel also provides a way to secure messages independently of the transport security used.</span></span> <span data-ttu-id="58801-137">W tym modelu komunikat jest zabezpieczony za pomocą tokenu zabezpieczającego źródła, chociaż obsługiwane są tylko certyfikaty X. 509.</span><span class="sxs-lookup"><span data-stu-id="58801-137">In this model, the message is secured at the source using the source’s security token, although currently only X.509 certificates are supported.</span></span> <span data-ttu-id="58801-138">Zabezpieczony komunikat jest następnie przesyłany za pośrednictwem sieci równorzędnej.</span><span class="sxs-lookup"><span data-stu-id="58801-138">The secured message is then transmitted over the peer network.</span></span> <span data-ttu-id="58801-139">Każdy element równorzędny odbiorczej może zweryfikować autentyczność źródła.</span><span class="sxs-lookup"><span data-stu-id="58801-139">Each receiving peer can verify the authenticity of the source.</span></span> <span data-ttu-id="58801-140">Należy pamiętać, że komunikat jest zabezpieczony, dzięki czemu pośrednicy nie może go zmodyfikować.</span><span class="sxs-lookup"><span data-stu-id="58801-140">Note that the message is secured so that intermediaries cannot tamper with it.</span></span>  
  
 <span data-ttu-id="58801-141">Aby zapewnić poufność, aplikacje mogą stosować zabezpieczenia transportu z silnymi schematami członkostwa w grupach, aby zapobiec nieautoryzowanemu dostępowi do wiadomości.</span><span class="sxs-lookup"><span data-stu-id="58801-141">To achieve confidentiality, applications can employ transport security with strong group membership schemes to prevent unauthorized access to the message.</span></span>  
  
 <span data-ttu-id="58801-142">Kanał równorzędny nie wymaga określonego modelu tożsamości, dopóki aplikacja wybierze jeden z obsługiwanych typów tokenów.</span><span class="sxs-lookup"><span data-stu-id="58801-142">Peer Channel does not require a specific identity model as long as the application chooses one of the supported token types.</span></span> <span data-ttu-id="58801-143">Aplikacje pełnią w cyklu życia te tożsamości i decyzje dotyczące uwierzytelniania.</span><span class="sxs-lookup"><span data-stu-id="58801-143">Applications completely own the life cycle of these identities and authentication decisions.</span></span>
