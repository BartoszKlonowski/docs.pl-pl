 <span data-ttu-id="5bac5-119">Poniżej przedstawiono listę invariants w tym trybie:</span><span class="sxs-lookup"><span data-stu-id="5bac5-119">The following is a list of invariants in this mode:</span></span>

 <span data-ttu-id="5bac5-130">Niezależnie od tego, który tryb jest wybierany fabryki kanału klienta obsługuje <xref:System.ServiceModel.Channels.IRequestChannel>, <xref:System.ServiceModel.Channels.IRequestSessionChannel>, i <xref:System.ServiceModel.Channels.IDuplexSessionChannel> komunikatu wzorce programu exchange.</span><span class="sxs-lookup"><span data-stu-id="5bac5-130">No matter which mode is chosen the client channel factory supports <xref:System.ServiceModel.Channels.IRequestChannel>, <xref:System.ServiceModel.Channels.IRequestSessionChannel>, and <xref:System.ServiceModel.Channels.IDuplexSessionChannel> message exchange patterns.</span></span>  
  
 <span data-ttu-id="5bac5-131">W usłudze jest odpowiedzialny za konwertowanie kontekstu dostarczany przez klienta w komunikatach przychodzących do wystąpienia kanału patterns.</span></span>  
  
 <span data-ttu-id="5bac5-136">Protokół wymiany kontekstu wprowadzono nowy `wsc:Context` nagłówek SOAP reprezentujący informacje o kontekście podczas plików cookie protokołu HTTP nie są używane do propagacji kontekstu.</span><span class="sxs-lookup"><span data-stu-id="5bac5-136">The context exchange protocol introduces a new `wsc:Context` SOAP header to represent the context information when HTTP cookies are not used to propagate the context.</span></span> <span data-ttu-id="5bac5-137">Zezwala schemat nagłówka kontekstu dla dowolnej liczby elementów podrzędnych, każda z ciągu klucza i zawartość ciągu.</span><span class="sxs-lookup"><span data-stu-id="5bac5-137">The context header schema allows for any number of child elements, each with a string key and string content.</span></span> <span data-ttu-id="5bac5-138">Oto przykład nagłówka kontekstu.</span><span class="sxs-lookup"><span data-stu-id="5bac5-138">The following is an example of a context header.</span></span>

 `<property name="myContext">context-2</property>`  
  
 `</Context>`
