# <a name="reliable-services"></a><span data-ttu-id="ba61e-102">Niezawodne usługi</span><span class="sxs-lookup"><span data-stu-id="ba61e-102">Reliable Services</span></span>

 <span data-ttu-id="ba61e-105">*Niezawodna obsługa komunikatów* jest jak wiarygodnego źródła wiadomości (o nazwie *źródła*) niezawodnie przesyła komunikaty niezawodnej obsługi komunikatów miejsca docelowego (o nazwie *docelowy*).</span><span class="sxs-lookup"><span data-stu-id="ba61e-105">*Reliable messaging* is how a reliable messaging source (called the *source*) transfers messages reliably to a reliable messaging destination (called the *destination*).</span></span>  
  
 <span data-ttu-id="ba61e-106">Niezawodna obsługa komunikatów wykonuje następujące funkcje:</span><span class="sxs-lookup"><span data-stu-id="ba61e-106">Reliable messaging performs the following functions:</span></span>

- <span data-ttu-id="ba61e-108">Oddziela źródło i miejsce docelowe, od siebie nawzajem.</span><span class="sxs-lookup"><span data-stu-id="ba61e-108">Separates the source and the destination from each other.</span></span> <span data-ttu-id="ba61e-109">Dzięki temu można niezależnie od awarii i odzyskiwania źródło i miejsce docelowe, jak również transferu niezawodne i dostarczanie wiadomości, nawet wtedy, gdy źródłowy lub docelowy jest niedostępny.</span><span class="sxs-lookup"><span data-stu-id="ba61e-109">This provides independent failure and recovery of the source and the destination, as well as reliable transfer and delivery of messages, even when the source or destination is unavailable.</span></span>

- <span data-ttu-id="ba61e-113">[Niezawodne sesje](../../../docs/framework/wcf/feature-details/reliable-sessions.md), która zapewnia niezawodne przesyłanie bez ponoszenia kosztów duże opóźnienie.</span><span class="sxs-lookup"><span data-stu-id="ba61e-113">[Reliable Sessions](../../../docs/framework/wcf/feature-details/reliable-sessions.md), which offers reliable transfer without the cost of high latency.</span></span>  
  
- <span data-ttu-id="ba61e-114">[Kolejki programu WCF](../../../docs/framework/wcf/feature-details/queues-in-wcf.md), które oferuje niezawodny transferu i separacji między źródłową i docelową.</span><span class="sxs-lookup"><span data-stu-id="ba61e-114">[Queues in WCF](../../../docs/framework/wcf/feature-details/queues-in-wcf.md), which offers both reliable transfers and separation between the source and the destination.</span></span>  
  
## <a name="reliable-sessions"></a><span data-ttu-id="ba61e-115">Niezawodne sesje</span><span class="sxs-lookup"><span data-stu-id="ba61e-115">Reliable Sessions</span></span>  
 <span data-ttu-id="ba61e-116">Niezawodne sesje zapewniają end-to-end niezawodne przesyłanie wiadomości między źródłem i miejsca docelowego przy użyciu protokołu elementy usługi WS-Reliable Messaging, niezależnie od liczby i typu pośredników, których punkty końcowe komunikatów (źródłowy i docelowy).</span><span class="sxs-lookup"><span data-stu-id="ba61e-116">Reliable sessions provide end-to-end reliable transfer of messages between a source and a destination using the WS-Reliable Messaging protocol, regardless of the number or type of intermediaries that separate the messaging (source and destination) endpoints.</span></span> <span data-ttu-id="ba61e-117">Dotyczy to również wszelkich pośredników transportu, które nie korzystają z protokołu SOAP (na przykład serwera proxy HTTP) lub pośredników, które używają protokołu SOAP (na przykład opartego na protokole SOAP routery lub mostków), które są wymagane dla komunikatów przepływ między punktami końcowymi.</span><span class="sxs-lookup"><span data-stu-id="ba61e-117">This includes any transport intermediaries that do not use SOAP (for example, HTTP proxies) or intermediaries that use SOAP (for example, SOAP-based routers or bridges) that are required for messages to flow between the endpoints.</span></span> <span data-ttu-id="ba61e-118">Niezawodne sesje okno transfer w pamięci na awarie poziom komunikatu protokołu SOAP maski i ponownie ustanowić połączenia w przypadku błędów transportu.</span><span class="sxs-lookup"><span data-stu-id="ba61e-118">Reliable sessions use an in-memory transfer window to mask SOAP message-level failures and re-establish connections in the case of transport failures.</span></span>  
  
 <span data-ttu-id="ba61e-119">Niezawodne sesje zapewniają transfery wiarygodnych wiadomości o małych opóźnieniach.</span><span class="sxs-lookup"><span data-stu-id="ba61e-119">Reliable sessions provide low-latency reliable message transfers.</span></span> <span data-ttu-id="ba61e-120">Zapewniają one komunikaty protokołu SOAP za pośrednictwem serwerów proxy ani pośredników, równoważna jakiego protokołu TCP zapewnia pakietów przez mostków adresów IP.</span><span class="sxs-lookup"><span data-stu-id="ba61e-120">They provide for SOAP messages over any proxies or intermediaries, equivalent to what TCP provides for packets over IP bridges.</span></span> <span data-ttu-id="ba61e-121">Aby uzyskać więcej informacji o sesjach niezawodnych, zobacz [sesje niezawodnej](../../../docs/framework/wcf/feature-details/reliable-sessions.md).</span><span class="sxs-lookup"><span data-stu-id="ba61e-121">For more information about reliable sessions, see [Reliable Sessions](../../../docs/framework/wcf/feature-details/reliable-sessions.md).</span></span>

 <span data-ttu-id="ba61e-123">Kolejki programu WCF zapewnia zarówno niezawodne transferu wiadomości i separacji między źródeł i miejsc docelowych kosztem duże opóźnienie.</span><span class="sxs-lookup"><span data-stu-id="ba61e-123">Queues in WCF provide both reliable transfers of messages and separation between sources and destinations at the cost of high latency.</span></span> <span data-ttu-id="ba61e-124">Usługi WCF w kolejce komunikacji jest zbudowany na podstawie usługi kolejkowania komunikatów (MSMQ).</span><span class="sxs-lookup"><span data-stu-id="ba61e-124">WCF queued communication is built on top of Message Queuing (MSMQ).</span></span>

 <span data-ttu-id="ba61e-131">Oddzielenie połączone z przeniesienia niezawodnych komunikatów między kolejek umożliwia aplikacji, które są luźno powiązane niezawodnie komunikować się.</span><span class="sxs-lookup"><span data-stu-id="ba61e-131">The separation, coupled with reliable message transfers between queues, enables applications that are loosely coupled to communicate reliably.</span></span> <span data-ttu-id="ba61e-132">W przeciwieństwie do sesji niezawodnych źródłowym i docelowym nie musi być uruchomiona w tym samym czasie.</span><span class="sxs-lookup"><span data-stu-id="ba61e-132">Unlike reliable sessions, the source and destination do not have to be running at the same time.</span></span> <span data-ttu-id="ba61e-133">Dzięki temu niejawnie scenariuszach, gdzie kolejek w praktyce używana jako mechanizm wyrównywania obciążenia, gdy współczynnik źródła wiadomości w środowisku produkcyjnym i miejsca docelowego stopień wykorzystania wiadomości nie są zgodne.</span><span class="sxs-lookup"><span data-stu-id="ba61e-133">This implicitly enables scenarios where queues are, in effect, used as a load-leveling mechanism when the source's rate of message production and the destination's rate of the message consumption do not match.</span></span> <span data-ttu-id="ba61e-134">Aby uzyskać więcej informacji na temat kolejek, zobacz [kolejki programu WCF](../../../docs/framework/wcf/feature-details/queues-in-wcf.md).</span><span class="sxs-lookup"><span data-stu-id="ba61e-134">For more information about queues, see [Queues in WCF](../../../docs/framework/wcf/feature-details/queues-in-wcf.md).</span></span>
