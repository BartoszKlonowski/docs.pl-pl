 <span data-ttu-id="a1554-111">Poniższe wskazówki mogą pomóc w zapobieganiu niezamierzonej insytowaniu zawartości pliku dziennika:</span><span class="sxs-lookup"><span data-stu-id="a1554-111">The following tips can help you to prevent the content of a log file from being exposed unintentionally:</span></span>  
  
- <span data-ttu-id="a1554-112">Upewnij się, że pliki dziennika są chronione przez listy kontroli dostępu (ACL) zarówno w scenariuszach hosta sieci Web, jak i hosta samodzielnego.</span><span class="sxs-lookup"><span data-stu-id="a1554-112">Ensure that the log files are protected by Access Control Lists (ACL) both in Web-host and self-host scenarios.</span></span>  
  
- <span data-ttu-id="a1554-113">Wybierz rozszerzenie pliku, które nie może być łatwo obsługiwane za pomocą żądania sieci Web.</span><span class="sxs-lookup"><span data-stu-id="a1554-113">Choose a file extension that cannot be easily served using a Web request.</span></span> <span data-ttu-id="a1554-114">Na przykład rozszerzenie pliku xml nie jest bezpiecznym wyborem.</span><span class="sxs-lookup"><span data-stu-id="a1554-114">For example, the .xml file extension is not a safe choice.</span></span> <span data-ttu-id="a1554-115">Listę rozszerzeń, które mogą być przekazywane, można znaleźć w podręczniku administratora usług Internet Information Services (IIS).</span><span class="sxs-lookup"><span data-stu-id="a1554-115">You can check the Internet Information Services (IIS) administration guide to see a list of extensions that can be served.</span></span>  
  
- <span data-ttu-id="a1554-116">Określ ścieżkę bezwzględną dla lokalizacji pliku dziennika, która powinna znajdować się poza katalogiem publicznym vroot hosta sieci Web, aby uniemożliwić dostęp do niego przez stronę zewnętrzną za pomocą przeglądarki sieci Web.</span><span class="sxs-lookup"><span data-stu-id="a1554-116">Specify an absolute path for the log file location, which should be outside of the Web host vroot public directory to prevent it from being accessed by an external party using a Web browser.</span></span>  
  
 <span data-ttu-id="a1554-117">Domyślnie klucze i informacje umożliwiające identyfikację użytkownika, takie jak nazwa użytkownika i hasło, nie są rejestrowane w ślady i rejestrowane wiadomości.</span><span class="sxs-lookup"><span data-stu-id="a1554-117">By default, keys and personally identifiable information (PII) such as username and password are not logged in traces and logged messages.</span></span> <span data-ttu-id="a1554-118">Administrator komputera może jednak użyć `enableLoggingKnownPII` atrybutu `machineSettings` w elemencie pliku Machine.config, aby zezwolić aplikacjom działającym na komputerze na rejestrowanie znanych informacji umożliwiających identyfikację użytkownika.</span><span class="sxs-lookup"><span data-stu-id="a1554-118">A machine administrator, however, can use the `enableLoggingKnownPII` attribute in the `machineSettings` element of the Machine.config file to permit applications running on the machine to log known personally identifiable information (PII).</span></span> <span data-ttu-id="a1554-119">Następująca konfiguracja pokazuje, jak to zrobić:</span><span class="sxs-lookup"><span data-stu-id="a1554-119">The following configuration demonstrates how to do this:</span></span>  
  
```xml

   <system.serviceModel>  
      <machineSettings enableLoggingKnownPii="true"/>  
   </system.serviceModel>  
</configuration>
```  
  
 <span data-ttu-id="a1554-120">Wdrażający aplikację może następnie `logKnownPii` użyć atrybutu w pliku App.config lub Web.config, aby włączyć rejestrowanie danych umożliwiających identyfikację użytkowników w następujący sposób:</span><span class="sxs-lookup"><span data-stu-id="a1554-120">An application deployer can then use the `logKnownPii` attribute in either the App.config or Web.config file to enable PII logging as follows:</span></span>

 <span data-ttu-id="a1554-121">Tylko wtedy, `true` gdy oba ustawienia są włączone rejestrowanie pii.</span><span class="sxs-lookup"><span data-stu-id="a1554-121">Only when both settings are `true` is PII logging enabled.</span></span> <span data-ttu-id="a1554-122">Połączenie dwóch przełączników umożliwia elastyczność rejestrowania znanych identyfikatorów umożliwiających identyfikację dla każdej aplikacji.</span><span class="sxs-lookup"><span data-stu-id="a1554-122">The combination of two switches allows the flexibility to log known PII for each application.</span></span>  
  
> [!IMPORTANT]

 <span data-ttu-id="a1554-124">Należy pamiętać, że jeśli określisz dwa lub więcej źródeł niestandardowych w pliku konfiguracyjnym, odczytywane są tylko atrybuty pierwszego źródła.</span><span class="sxs-lookup"><span data-stu-id="a1554-124">You should be aware that if you specify two or more custom sources in a configuration file, only the attributes of the first source are read.</span></span> <span data-ttu-id="a1554-125">Pozostałe są ignorowane.</span><span class="sxs-lookup"><span data-stu-id="a1554-125">The others are ignored.</span></span> <span data-ttu-id="a1554-126">Oznacza to, że dla następującego pliku App.config, informacje umożliwiające identyfikację nie są rejestrowane dla obu źródeł, mimo że rejestrowanie danych umożliwiających identyfikację jest jawnie włączone dla drugiego źródła.</span><span class="sxs-lookup"><span data-stu-id="a1554-126">This means that, for the following App.config, file, PII is not logged for both sources even though PII logging is explicitly enabled for the second source.</span></span>

      <source name="System.ServiceModel.MessageLogging"  
              logKnownPii="false">

                      type="System.Diagnostics.XmlWriterTraceListener"  
                      initializeData="c:\logs\messages.svclog" />

            </source>

              <listeners>  
                 <add name="traces"

              </listeners>

   </sources>  
</system.diagnostics>

 <span data-ttu-id="a1554-127">Jeśli `<machineSettings enableLoggingKnownPii="Boolean"/>` element istnieje poza plikiem Machine.config, system <xref:System.Configuration.ConfigurationErrorsException>zgłasza plik .</span><span class="sxs-lookup"><span data-stu-id="a1554-127">If the `<machineSettings enableLoggingKnownPii="Boolean"/>` element exists outside the Machine.config file, the system throws a <xref:System.Configuration.ConfigurationErrorsException>.</span></span>
