## <a name="using-transfer-to-correlate-activities-within-endpoints"></a><span data-ttu-id="8e076-110">Używanie transferu do skorelowania działań w punktach końcowych</span><span class="sxs-lookup"><span data-stu-id="8e076-110">Using Transfer to Correlate Activities Within Endpoints</span></span>

 <span data-ttu-id="8e076-116">Po przekazaniu od M do N nie można wykonać transferu z powrotem z N do M. ## <a name="activity-transfer-sequence"></a><span data-ttu-id="8e076-132">Sekwencja transferu działań</span><span class="sxs-lookup"><span data-stu-id="8e076-132">Activity Transfer Sequence</span></span>  

 <span data-ttu-id="8e076-133">Poprawnie sformułowana sekwencja transferu działań obejmuje następujące kroki.</span><span class="sxs-lookup"><span data-stu-id="8e076-133">A well-formed activity transfer sequence includes the following steps.</span></span>  
  
1. <span data-ttu-id="8e076-134">Rozpocznij nowe działanie, które składa się z wyboru nowego gAId.</span><span class="sxs-lookup"><span data-stu-id="8e076-134">Begin a new activity, which consists of selecting a new gAId.</span></span>  
  
2. <span data-ttu-id="8e076-135">Emituj ślad transferu do tego nowego gAId z bieżącego identyfikatora działania</span><span class="sxs-lookup"><span data-stu-id="8e076-135">Emit a transfer trace to that new gAId from the current activity ID</span></span>  
  
3. <span data-ttu-id="8e076-136">Ustaw nowy identyfikator w protokole TLS</span><span class="sxs-lookup"><span data-stu-id="8e076-136">Set the new ID in TLS</span></span>  
  
4. <span data-ttu-id="8e076-137">Emituj początkowy ślad, aby wskazać początek nowej aktywności przez.</span><span class="sxs-lookup"><span data-stu-id="8e076-137">Emit a start trace to indicate the beginning of the new activity by.</span></span>  
  
5. <span data-ttu-id="8e076-138">Powrót do oryginalnego działania składa się z następujących elementów:</span><span class="sxs-lookup"><span data-stu-id="8e076-138">Return to the original activity consists of the following:</span></span>  
  
6. <span data-ttu-id="8e076-139">Emituj ślad transferu do oryginalnego gAId</span><span class="sxs-lookup"><span data-stu-id="8e076-139">Emit a transfer trace to the original gAId</span></span>  
  
7. <span data-ttu-id="8e076-140">Emituj śledzenie zatrzymania, aby wskazać koniec nowego działania</span><span class="sxs-lookup"><span data-stu-id="8e076-140">Emit a Stop trace to indicate the end of the new activity</span></span>  
  
8. <span data-ttu-id="8e076-141">Ustaw protokół TLS na stary gAId.</span><span class="sxs-lookup"><span data-stu-id="8e076-141">Set TLS to the old gAId.</span></span>  
  
 <span data-ttu-id="8e076-142">Poniższy przykład kodu demonstruje, jak to zrobić.</span><span class="sxs-lookup"><span data-stu-id="8e076-142">The following code example demonstrates how to do this.</span></span> <span data-ttu-id="8e076-143">W tym przykładzie przyjęto założenie, że jest wywoływane wywołanie podczas przenoszenia do nowego działania i zawiera dane śledzenia wstrzymania/wznowienia.</span><span class="sxs-lookup"><span data-stu-id="8e076-143">This sample assumes a blocking call is made when transferring to the new activity, and includes suspend/resume traces.</span></span>  
  
```csharp
// 0. Create a trace source  
TraceSource ts = new TraceSource("myTS");  

// 1. remember existing ("ambient") activity for clean up  
Guid oldGuid = Trace.CorrelationManager.ActivityId;  
// this will be our new activity  
Guid newGuid = Guid.NewGuid();

// 2. call transfer, indicating that we are switching to the new AID  
ts.TraceTransfer(667, "Transferring.", newGuid);  

// 3. Suspend the current activity.  
ts.TraceEvent(TraceEventType.Suspend, 667, "Suspend: Activity " + i-1);  

// 4. set the new AID in TLS  
Trace.CorrelationManager.ActivityId = newGuid;  

// 5. Emit the start trace  
ts.TraceEvent(TraceEventType.Start, 667, "Boundary: Activity " + i);  

// trace something  
ts.TraceEvent(TraceEventType.Information, 667, "Hello from activity " + i);  

// Perform Work  
// some work.  
// Return  
ts.TraceEvent(TraceEventType.Information, 667, "Work complete on activity " + i);

// 6. Emit the transfer returning to the original activity  
ts.TraceTransfer(667, "Transferring Back.", oldGuid);  

// 7. Emit the End trace  
ts.TraceEvent(TraceEventType.Stop, 667, "Boundary: Activity " + i);  

// 8. Change the tls variable to the original AID  
Trace.CorrelationManager.ActivityId = oldGuid;

// 9. Resume the old activity  
ts.TraceEvent(TraceEventType.Resume, 667, "Resume: Activity " + i-1);  
```
