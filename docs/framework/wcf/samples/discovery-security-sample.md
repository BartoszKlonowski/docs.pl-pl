## <a name="secure-channel-factory"></a><span data-ttu-id="64b09-116">Fabryka kanałów Secure Channel</span><span class="sxs-lookup"><span data-stu-id="64b09-116">Secure Channel Factory</span></span>  

 <span data-ttu-id="64b09-117">Fabryka bezpiecznych kanałów tworzy kanały wyjściowe lub dupleksowe, które dodają skrócony podpis do nagłówków komunikatów.</span><span class="sxs-lookup"><span data-stu-id="64b09-117">The secure channel factory creates output or duplex channels that add a compact signature to message headers.</span></span> <span data-ttu-id="64b09-118">Aby zapewnić, że komunikaty są możliwie małe, jest używany format podpisu kompaktowego.</span><span class="sxs-lookup"><span data-stu-id="64b09-118">To keep messages as small as possible the compact signature format is used.</span></span> <span data-ttu-id="64b09-119">Struktura skróconej sygnatury jest pokazana w poniższym przykładzie.</span><span class="sxs-lookup"><span data-stu-id="64b09-119">The structure of a compact signature is shown in the following example.</span></span>  
  
```xml  
<d:Security ... >
  [<d:Sig Scheme="xs:anyURI"
         [KeyId="xs:base64Binary"]?  
          Refs="..."  
         [PrefixList]="xs:NMTOKENS"
          Sig="xs:base64Binary"
          ... />]?  
  ...
</d:Security>  
```

> <span data-ttu-id="64b09-120">`PrefixList`Dodano do protokołu 2008 odnajdywania wersji.</span><span class="sxs-lookup"><span data-stu-id="64b09-120">The `PrefixList` was added in the 2008 Discovery version protocol.</span></span>  
  
 <span data-ttu-id="64b09-121">Aby obliczyć sygnaturę, przykład określa rozwinięte elementy podpisu.</span><span class="sxs-lookup"><span data-stu-id="64b09-121">To compute the signature, the sample determines the expanded signature items.</span></span> <span data-ttu-id="64b09-122">Sygnatura XML ( `SignedInfo` ) jest tworzona przy użyciu `ds` prefiksu przestrzeni nazw, zgodnie z wymaganiami
