## <a name="discussion"></a><span data-ttu-id="2a7df-108">Dyskusja</span><span class="sxs-lookup"><span data-stu-id="2a7df-108">Discussion</span></span>  
 <span data-ttu-id="2a7df-109">Ten przykład zawiera dwa składniki: projekt aplikacji sieci Web (usługa), który zawiera usługi WCF i aplikacji konsoli (klient), który wywołuje operacje usługi za pomocą powiązań protokołu SOAP i HTTP.</span><span class="sxs-lookup"><span data-stu-id="2a7df-109">This sample consists of two components: a Web Application project (Service) that contains a WCF service and a console application (Client) that invokes service operations using SOAP and HTTP bindings.</span></span>  
  
 <span data-ttu-id="2a7df-110">Usługa WCF przedstawia działanie 2-`GetData` i `PutData` — która echo ciąg, który został przekazany jako dane wejściowe.</span><span class="sxs-lookup"><span data-stu-id="2a7df-110">The WCF service exposes 2 operations –`GetData` and `PutData` – that echo the string that was passed as input.</span></span> <span data-ttu-id="2a7df-111">Operacje usługi jest oznaczony za pomocą <xref:System.ServiceModel.Web.WebGetAttribute> i <xref:System.ServiceModel.Web.WebInvokeAttribute>.</span><span class="sxs-lookup"><span data-stu-id="2a7df-111">The service operations are annotated with <xref:System.ServiceModel.Web.WebGetAttribute> and <xref:System.ServiceModel.Web.WebInvokeAttribute>.</span></span> <span data-ttu-id="2a7df-112">Te atrybuty kontrolować projekcji HTTP tych operacji.</span><span class="sxs-lookup"><span data-stu-id="2a7df-112">These attributes control the HTTP projection of these operations.</span></span> <span data-ttu-id="2a7df-113">Ponadto jest oznaczony za pomocą <xref:System.ServiceModel.OperationContractAttribute>, co pozwala na ujawnianie za pośrednictwem powiązania protokołu SOAP.</span><span class="sxs-lookup"><span data-stu-id="2a7df-113">In addition, they are annotated with <xref:System.ServiceModel.OperationContractAttribute>, which enables them to be exposed over SOAP bindings.</span></span> <span data-ttu-id="2a7df-114">Usługa `PutData` metoda zgłasza wyjątek <xref:System.ServiceModel.Web.WebFaultException>, który zostaje przesłane ponownie przy użyciu protokołu HTTP, przy użyciu kodu stanu HTTP i pobiera wysyłane z powrotem za pośrednictwem protokołu SOAP, jako protokołu SOAP.</span><span class="sxs-lookup"><span data-stu-id="2a7df-114">The service’s `PutData` method throws a <xref:System.ServiceModel.Web.WebFaultException>, which gets sent back over HTTP using the HTTP status code and gets sent back over SOAP as a SOAP fault.</span></span>  
  
 <span data-ttu-id="2a7df-115">Plik Web.config służy do konfigurowania usługi WCF z punktami końcowymi 3:</span><span class="sxs-lookup"><span data-stu-id="2a7df-115">The Web.config file configures the WCF service with 3 endpoints:</span></span>  
  
-   <span data-ttu-id="2a7df-116">Punkt końcowy ~/service.svc/mex, który udostępnia metadane usługi dla dostępu przez klientów opartej na protokole SOAP.</span><span class="sxs-lookup"><span data-stu-id="2a7df-116">The ~/service.svc/mex endpoint that exposes the service metadata for access by SOAP-based clients.</span></span>  
  
-   <span data-ttu-id="2a7df-117">Punkt końcowy ~/service.svc/http, która umożliwia klientom dostęp do usługi za pomocą powiązania protokołu HTTP.</span><span class="sxs-lookup"><span data-stu-id="2a7df-117">The ~/service.svc/http endpoint that enables clients to access the service using the HTTP binding.</span></span>  
  
-   <span data-ttu-id="2a7df-118">Punkt końcowy ~/service.svc/soap, który umożliwia klientom dostęp do usługi za pośrednictwem powiązania protokołu HTTP przy użyciu protokołu SOAP.</span><span class="sxs-lookup"><span data-stu-id="2a7df-118">The ~/service.svc/soap endpoint that allows the clients to access the service using the SOAP over HTTP binding.</span></span>  
  
 <span data-ttu-id="2a7df-119">Punkt końcowy HTTP jest skonfigurowany przy użyciu <`webHttp`> Standardowy punkt końcowy, który ma `helpEnabled` równa `true`.</span><span class="sxs-lookup"><span data-stu-id="2a7df-119">The HTTP endpoint is configured with a <`webHttp`> standard endpoint which has `helpEnabled` set to `true`.</span></span> <span data-ttu-id="2a7df-120">Co w efekcie Usługa udostępnia stronę pomocy XHTML oparte na ~/service.svc/http/help używanego przez klientów opartych na protokole HTTP można uzyskiwać dostęp do usługi.</span><span class="sxs-lookup"><span data-stu-id="2a7df-120">As a result, the service exposes an XHTML based help page at ~/service.svc/http/help that HTTP-based clients can use to access the service.</span></span>  
  
 <span data-ttu-id="2a7df-121">Pokazuje projekt klienta, uzyskiwanie dostępu do usługi przy użyciu serwera proxy protokołu SOAP (wygenerowanych za pośrednictwem **Dodaj odwołanie do usługi**) i uzyskiwania dostępu do usługi za pomocą <xref:System.Net.WebClient>.</span><span class="sxs-lookup"><span data-stu-id="2a7df-121">The client project demonstrates accessing the service using a SOAP proxy (generated through **Add Service Reference**) and accessing the service using <xref:System.Net.WebClient>.</span></span>
