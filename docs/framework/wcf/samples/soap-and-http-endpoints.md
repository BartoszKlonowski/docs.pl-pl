## <a name="discussion"></a><span data-ttu-id="86799-108">Dyskusji</span><span class="sxs-lookup"><span data-stu-id="86799-108">Discussion</span></span>  
 <span data-ttu-id="86799-109">Ten przykład składa się z dwóch składników: projektu aplikacji sieci Web (usługi), który zawiera usługę WCF i aplikację konsoli (klienta), która wywołuje operacje usługi przy użyciu powiązań protokołu SOAP i HTTP.</span><span class="sxs-lookup"><span data-stu-id="86799-109">This sample consists of two components: a Web Application project (Service) that contains a WCF service and a console application (Client) that invokes service operations using SOAP and HTTP bindings.</span></span>  
  
 <span data-ttu-id="86799-110">Usługa WCF uwidacznia 2 operacje —`GetData` i `PutData` — ciąg, który został przesłany jako dane wejściowe.</span><span class="sxs-lookup"><span data-stu-id="86799-110">The WCF service exposes 2 operations –`GetData` and `PutData` – that echo the string that was passed as input.</span></span> <span data-ttu-id="86799-111">Operacje usługi są z adnotacjami <xref:System.ServiceModel.Web.WebGetAttribute> i <xref:System.ServiceModel.Web.WebInvokeAttribute>.</span><span class="sxs-lookup"><span data-stu-id="86799-111">The service operations are annotated with <xref:System.ServiceModel.Web.WebGetAttribute> and <xref:System.ServiceModel.Web.WebInvokeAttribute>.</span></span> <span data-ttu-id="86799-112">Te atrybuty kontrolują projekcję HTTP tych operacji.</span><span class="sxs-lookup"><span data-stu-id="86799-112">These attributes control the HTTP projection of these operations.</span></span> <span data-ttu-id="86799-113">Ponadto są adnotacjami z <xref:System.ServiceModel.OperationContractAttribute>, co umożliwia ich uwidocznienie za pośrednictwem powiązań protokołu SOAP.</span><span class="sxs-lookup"><span data-stu-id="86799-113">In addition, they are annotated with <xref:System.ServiceModel.OperationContractAttribute>, which enables them to be exposed over SOAP bindings.</span></span> <span data-ttu-id="86799-114">Metoda `PutData` usługi zgłasza <xref:System.ServiceModel.Web.WebFaultException>, które są wysyłane z powrotem za pośrednictwem protokołu HTTP przy użyciu kodu stanu HTTP i pobiera z powrotem za pośrednictwem protokołu SOAP jako błąd protokołu SOAP.</span><span class="sxs-lookup"><span data-stu-id="86799-114">The service’s `PutData` method throws a <xref:System.ServiceModel.Web.WebFaultException>, which gets sent back over HTTP using the HTTP status code and gets sent back over SOAP as a SOAP fault.</span></span>  
  
 <span data-ttu-id="86799-115">Plik Web. config konfiguruje usługę WCF z trzema punktami końcowymi:</span><span class="sxs-lookup"><span data-stu-id="86799-115">The Web.config file configures the WCF service with 3 endpoints:</span></span>  
  
- <span data-ttu-id="86799-116">Punkt końcowy/Service.svc/Mex, który uwidacznia metadane usługi dla dostępu przez klientów opartych na protokole SOAP.</span><span class="sxs-lookup"><span data-stu-id="86799-116">The ~/service.svc/mex endpoint that exposes the service metadata for access by SOAP-based clients.</span></span>  
  
- <span data-ttu-id="86799-117">/Service.svc/http punkt końcowy, który umożliwia klientom dostęp do usługi przy użyciu powiązania HTTP.</span><span class="sxs-lookup"><span data-stu-id="86799-117">The ~/service.svc/http endpoint that enables clients to access the service using the HTTP binding.</span></span>  
  
- <span data-ttu-id="86799-118">/Service.svc/SOAP punkt końcowy, który umożliwia klientom dostęp do usługi przy użyciu powiązania protokołu SOAP over HTTP.</span><span class="sxs-lookup"><span data-stu-id="86799-118">The ~/service.svc/soap endpoint that allows the clients to access the service using the SOAP over HTTP binding.</span></span>  
  
 <span data-ttu-id="86799-119">Punkt końcowy HTTP jest skonfigurowany przy użyciu <`webHttp`> standardowego punktu końcowego, który ma `helpEnabled` ustawiony na `true`.</span><span class="sxs-lookup"><span data-stu-id="86799-119">The HTTP endpoint is configured with a <`webHttp`> standard endpoint which has `helpEnabled` set to `true`.</span></span> <span data-ttu-id="86799-120">W związku z tym usługa uwidacznia stronę pomocy opartą na XHTML w lokalizacji ~/Service.svc/http/help, że klienci korzystający z protokołu HTTP mogą uzyskać dostęp do usługi.</span><span class="sxs-lookup"><span data-stu-id="86799-120">As a result, the service exposes an XHTML based help page at ~/service.svc/http/help that HTTP-based clients can use to access the service.</span></span>  
  
 <span data-ttu-id="86799-121">Projekt klienta pokazuje dostęp do usługi przy użyciu serwera proxy SOAP (generowanego za pośrednictwem **Dodaj odwołanie do usługi**) i uzyskując dostęp do usługi przy użyciu <xref:System.Net.WebClient>.</span><span class="sxs-lookup"><span data-stu-id="86799-121">The client project demonstrates accessing the service using a SOAP proxy (generated through **Add Service Reference**) and accessing the service using <xref:System.Net.WebClient>.</span></span>  
  
 <span data-ttu-id="86799-122">Przykład składa się z usługi hostowanej w sieci Web i aplikacji konsolowej.</span><span class="sxs-lookup"><span data-stu-id="86799-122">The sample consists of a Web-hosted service and a console application.</span></span> <span data-ttu-id="86799-123">Gdy aplikacja konsolowa zostanie uruchomiona, klient wysyła żądania do usługi i zapisuje odpowiednie informacje z odpowiedzi do okna konsoli.</span><span class="sxs-lookup"><span data-stu-id="86799-123">As the console application runs, the client makes requests to the service and writes the pertinent information from the responses to the console window.</span></span>
