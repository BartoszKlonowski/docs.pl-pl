 <span data-ttu-id="0efa0-117">Celem używania mechanizmu MTOM jest zoptymalizowanie przesyłania dużych ładunków binarnych.</span><span class="sxs-lookup"><span data-stu-id="0efa0-117">The purpose for using MTOM is to optimize the transmission of large binary payloads.</span></span> <span data-ttu-id="0efa0-118">Wysyłanie komunikatu protokołu SOAP przy użyciu mechanizmu MTOM ma zauważalne obciążenie dla małych ładunków binarnych, ale stają się doskonałym oszczędnościem, gdy rośnie w ciągu kilku tysięcy bajtów.</span><span class="sxs-lookup"><span data-stu-id="0efa0-118">Sending a SOAP message using MTOM has a noticeable overhead for small binary payloads, but becomes a great savings when they grow over a few thousand bytes.</span></span> <span data-ttu-id="0efa0-119">Przyczyną tego jest fakt, że zwykły tekst XML koduje dane binarne przy użyciu algorytmu Base64, który wymaga czterech znaków dla każdego z trzech bajtów i zwiększa rozmiar danych o jeden trzeci.</span><span class="sxs-lookup"><span data-stu-id="0efa0-119">The reason for this is that normal text XML encodes binary data using Base64, which requires four characters for every three bytes, and increases the size of the data by one third.</span></span> <span data-ttu-id="0efa0-120">MTOM jest w stanie przesłać dane binarne jako nieprzetworzone bajty, co umożliwia zapisanie czasu kodowania/dekodowania i wygenerowanie mniejszych komunikatów.</span><span class="sxs-lookup"><span data-stu-id="0efa0-120">MTOM is able to transmit binary data as raw bytes, saving the encoding/decoding time and resulting is smaller messages.</span></span> <span data-ttu-id="0efa0-121">Próg kilku tysięcy bajtów jest mały w porównaniu z współczesnymi dokumentami biznesowymi i fotografiami cyfrowymi.</span><span class="sxs-lookup"><span data-stu-id="0efa0-121">The threshold of a few thousand bytes is small when compared to today's business documents and digital photographs.</span></span>
