# <a name="interoperating-with-asmx-web-services"></a><span data-ttu-id="0c68e-102">Współdziałanie z usługami sieci Web ASMX</span><span class="sxs-lookup"><span data-stu-id="0c68e-102">Interoperating with ASMX Web Services</span></span>
<span data-ttu-id="0c68e-103">W tym przykładzie pokazano, jak zintegrować aplikacja kliencka Windows Communication Foundation (WCF) z istniejącej usługi sieci Web ASMX.</span><span class="sxs-lookup"><span data-stu-id="0c68e-103">This sample demonstrates how to integrate a Windows Communication Foundation (WCF) client application with an existing ASMX Web service.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="0c68e-104">Procedury i kompilacji instrukcje dotyczące instalacji dla tego przykładu znajdują się na końcu tego tematu.</span><span class="sxs-lookup"><span data-stu-id="0c68e-104">The setup procedure and build instructions for this sample are located at the end of this topic.</span></span>  
  
 <span data-ttu-id="0c68e-105">W tym przykładzie składa się z konsoli programu klienckiego (.exe) i usługi biblioteki (.dll), obsługiwane przez Internet Information Services (IIS).</span><span class="sxs-lookup"><span data-stu-id="0c68e-105">This sample consists of a client console program (.exe) and a service library (.dll) hosted by Internet Information Services (IIS).</span></span> <span data-ttu-id="0c68e-106">Usługa jest usługą sieci Web ASMX implementujący kontrakt definiuje wzorzec komunikacji żądanie odpowiedź.</span><span class="sxs-lookup"><span data-stu-id="0c68e-106">The service is an ASMX Web Service that implements a contract that defines a request-reply communication pattern.</span></span> <span data-ttu-id="0c68e-107">Usługa udostępnia operacji matematycznych (`Add`, `Subtract`, `Multiply`, i `Divide`).</span><span class="sxs-lookup"><span data-stu-id="0c68e-107">The service exposes math operations (`Add`, `Subtract`, `Multiply`, and `Divide`).</span></span> <span data-ttu-id="0c68e-108">Klient wysyła żądań synchronicznych operacji matematycznych i odpowiedzi usługi z wynikiem.</span><span class="sxs-lookup"><span data-stu-id="0c68e-108">The client makes synchronous requests to a math operation and the service replies with the result.</span></span> <span data-ttu-id="0c68e-109">Aktywność klienta jest widoczna w oknie konsoli.</span><span class="sxs-lookup"><span data-stu-id="0c68e-109">Client activity is visible in the console window.</span></span>  
  
 <span data-ttu-id="0c68e-110">Implementacja usługi sieci Web ASMX pokazano następujący przykładowy kod oblicza i zwraca wynik odpowiednie.</span><span class="sxs-lookup"><span data-stu-id="0c68e-110">The ASMX Web service implementation shown in the following sample code calculates and returns the appropriate result.</span></span>  
  
```csharp  
[WebService(Namespace="http://Microsoft.ServiceModel.Samples")]  
public class CalculatorService : System.Web.Services.WebService  
    {  
        [WebMethod]  
        public double Add(double n1, double n2)  
        {  
            return n1 + n2;  
        }  
        [WebMethod]  
        public double Subtract(double n1, double n2)  
        {  
            return n1 - n2;  
        }  
        [WebMethod]  
        public double Multiply(double n1, double n2)  
        {  
            return n1 * n2;  
        }  
        [WebMethod]  
        public double Divide(double n1, double n2)  
        {  
            return n1 / n2;  
        }  
    }  
```  
  
 <span data-ttu-id="0c68e-111">Zgodnie z konfiguracją, usługi mogą być wyświetlane w http://localhost/servicemodelsamples/service.asmx przez klienta na tym samym komputerze.</span><span class="sxs-lookup"><span data-stu-id="0c68e-111">As configured, the service can be accessed at http://localhost/servicemodelsamples/service.asmx by a client on the same machine.</span></span> <span data-ttu-id="0c68e-112">Dla klientów na komputerach zdalnych do uzyskania dostępu do usługi należy określić w kwalifikowanej nazwy domeny zamiast nazwy localhost.</span><span class="sxs-lookup"><span data-stu-id="0c68e-112">For clients on remote machines to access the service, a qualified domain name must be specified instead of localhost.</span></span>  
  
 <span data-ttu-id="0c68e-113">Komunikacja odbywa się za pośrednictwem wygenerowanych przez klienta [narzędzie narzędzia metadanych elementu ServiceModel (Svcutil.exe)](../../../../docs/framework/wcf/servicemodel-metadata-utility-tool-svcutil-exe.md).</span><span class="sxs-lookup"><span data-stu-id="0c68e-113">Communication is done through a client generated by the [ServiceModel Metadata Utility Tool (Svcutil.exe)](../../../../docs/framework/wcf/servicemodel-metadata-utility-tool-svcutil-exe.md).</span></span> <span data-ttu-id="0c68e-114">Klient znajduje się w pliku generatedClient.cs.</span><span class="sxs-lookup"><span data-stu-id="0c68e-114">The client is contained in the file generatedClient.cs.</span></span> <span data-ttu-id="0c68e-115">Usługa ASMX musi być dostępny w celu generowania kodu serwera proxy, ponieważ jest używany do pobierania metadanych zaktualizowane.</span><span class="sxs-lookup"><span data-stu-id="0c68e-115">The ASMX service must be available to generate the proxy code, because it is used to retrieve the updated metadata.</span></span> <span data-ttu-id="0c68e-116">Uruchom następujące polecenie w wierszu polecenia w katalogu klienta można wygenerować typu serwera proxy.</span><span class="sxs-lookup"><span data-stu-id="0c68e-116">Run the following command from a command prompt in the client directory to generate the typed proxy.</span></span>  
  
```  
svcutil.exe /n:http://Microsoft.ServiceModel.Samples,Microsoft.ServiceModel.Samples http://localhost/servicemodelsamples/service.svc?wsdl /out:generatedClient.cs  
```  
  
 <span data-ttu-id="0c68e-117">Za pomocą wygenerowanego klienta, można uzyskać dostępu do punktu końcowego usługi, konfigurując odpowiednie adres i powiązanie.</span><span class="sxs-lookup"><span data-stu-id="0c68e-117">By using the generated client, you can access a service endpoint by configuring the appropriate address and binding.</span></span> <span data-ttu-id="0c68e-118">Jak usługa Klient korzysta z pliku konfiguracji (App.config) do określenia punktu końcowego do komunikowania się z.</span><span class="sxs-lookup"><span data-stu-id="0c68e-118">Like the service, the client uses a configuration file (App.config) to specify the endpoint to communicate with.</span></span> <span data-ttu-id="0c68e-119">Konfiguracja punktu końcowego klienta składa się z adres bezwzględny dla punktu końcowego usługi, powiązanie i kontraktu, jak pokazano w poniższych Przykładowa konfiguracja.</span><span class="sxs-lookup"><span data-stu-id="0c68e-119">The client endpoint configuration consists of an absolute address for the service endpoint, the binding, and the contract, as shown in the following sample configuration.</span></span>  
  
```xml  
<client>  
   <endpoint   
      address="http://localhost/ServiceModelSamples/service.asmx"   
      binding="basicHttpBinding"   
      contract="Microsoft.ServiceModel.Samples.CalculatorServiceSoap" />  
</client>  
```  
  
 <span data-ttu-id="0c68e-120">Implementacja klienta tworzy wystąpienie klasy wygenerowanego klienta.</span><span class="sxs-lookup"><span data-stu-id="0c68e-120">The client implementation constructs an instance of the generated client.</span></span> <span data-ttu-id="0c68e-121">Następnie można wygenerowanego klienta do komunikowania się z usługą.</span><span class="sxs-lookup"><span data-stu-id="0c68e-121">The generated client can then be used to communicate with the service.</span></span>  
  
```csharp  
// Create a client.  
CalculatorServiceSoapClient client = new CalculatorServiceSoapClient();  
  
// Call the Add service operation.  
double value1 = 100.00D;  
double value2 = 15.99D;  
double result = client.Add(value1, value2);  
Console.WriteLine("Add({0},{1}) = {2}", value1, value2, result);  
  
// Call the Subtract service operation.  
value1 = 145.00D;  
value2 = 76.54D;  
result = client.Subtract(value1, value2);  
Console.WriteLine("Subtract({0},{1}) = {2}", value1, value2, result);  
  
// Call the Multiply service operation.  
value1 = 9.00D;  
value2 = 81.25D;  
result = client.Multiply(value1, value2);  
Console.WriteLine("Multiply({0},{1}) = {2}", value1, value2, result);  
  
// Call the Divide service operation.  
value1 = 22.00D;  
value2 = 7.00D;  
result = client.Divide(value1, value2);  
Console.WriteLine("Divide({0},{1}) = {2}", value1, value2, result);  
  
//Closing the client gracefully closes the connection and cleans up resources.  
client.Close();  
  
Console.WriteLine();  
Console.WriteLine("Press <ENTER> to terminate client.");  
Console.ReadLine();  
```

