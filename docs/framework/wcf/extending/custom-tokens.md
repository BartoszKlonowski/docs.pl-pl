# <a name="custom-tokens"></a>Tokeny niestandardowe
Windows Communication Foundation (WCF) obsługuje natywnie certyfikatów X.509, tokeny kontekstu zabezpieczeń, bilety protokołu Kerberos i tokeny nazwy użytkownika jako poświadczeń, ale jest wystarczająco elastyczny, aby umożliwić użycie niestandardowych poświadczeń.  
  
## <a name="in-this-section"></a>W tej sekcji  
 [Instrukcje: Tworzenie tokenu niestandardowego](../../../../docs/framework/wcf/extending/how-to-create-a-custom-token.md)  
 Opisuje sposób tworzenia tokenu zabezpieczeń niestandardowych przy użyciu <xref:System.IdentityModel.Tokens.SecurityToken> klasy i jak ją zintegrować z niestandardowego dostawcy tokenów zabezpieczeń i uwierzytelniania.
