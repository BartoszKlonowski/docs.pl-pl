# <a name="custom-tokens"></a>Tokeny niestandardowe
Mimo że Windows Communication Foundation (WCF) natywnie obsługuje certyfikaty X. 509, tokeny kontekstu zabezpieczeń, bilety protokołu Kerberos i tokeny nazw użytkowników jako poświadczenia, jest to wystarczająco elastyczne, aby można było korzystać z własnych poświadczeń niestandardowych.  
  
## <a name="in-this-section"></a>W tej sekcji  
 [Instrukcje: Tworzenie niestandardowego tokenu](how-to-create-a-custom-token.md)  
 Opisuje sposób tworzenia niestandardowego tokenu zabezpieczającego przy użyciu <xref:System.IdentityModel.Tokens.SecurityToken> klasy i sposobu integrowania go z niestandardowym dostawcą tokenu zabezpieczeń i wystawcą uwierzytelnienia.
