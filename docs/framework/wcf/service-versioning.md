 <span data-ttu-id="c1e84-234">Ogólne wskazówki dotyczące przechowywania wersji umowy danych, gdy wymagane jest ścisłe przechowywanie wersji, jest traktowanie kontraktów danych jako niezmiennych i tworzenie nowych, gdy wymagane są zmiany.</span><span class="sxs-lookup"><span data-stu-id="c1e84-234">The general data contract versioning guidance when strict versioning is needed is to treat data contracts as immutable and create new ones when changes are required.</span></span> <span data-ttu-id="c1e84-235">Nowa klasa musi zostać utworzona dla każdej nowej umowy danych, więc mechanizm jest potrzebny, aby uniknąć konieczności podjęcia istniejącego kodu, który został napisany w kategoriach starej klasy kontraktu danych i przepisać go pod względem nowej klasy kontraktu danych.</span><span class="sxs-lookup"><span data-stu-id="c1e84-235">A new class needs to be created for each new data contract, so a mechanism is needed to avoid having to take existing code that was written in terms of the old data contract class and rewrite it in terms of the new data contract class.</span></span>  
  
 <span data-ttu-id="c1e84-236">Jednym z takich mechanizmów jest użycie interfejsów do definiowania elementów członkowskich każdej umowy danych i pisania wewnętrznego kodu implementacji pod względem interfejsów, a nie klas kontraktów danych, które implementują interfejsy.</span><span class="sxs-lookup"><span data-stu-id="c1e84-236">One such mechanism is to use interfaces to define the members of each data contract and write internal implementation code in terms of the interfaces rather than the data contract classes that implement the interfaces.</span></span> <span data-ttu-id="c1e84-237">Poniższy kod dla wersji 1 `IPurchaseOrderV1` usługi pokazuje `PurchaseOrderV1`interfejs i:</span><span class="sxs-lookup"><span data-stu-id="c1e84-237">The following code for version 1 of a service shows an `IPurchaseOrderV1` interface and a `PurchaseOrderV1`:</span></span>

```csharp  
[DataContract(

    [DataMember(...)]

    [DataMember(...)]  
    public string CustomerId {...}  
}  
```  
  
 <span data-ttu-id="c1e84-238">Podczas gdy działalność umowy o świadczenie usług `PurchaseOrderV1`byłaby napisana w kategoriach, rzeczywista logika biznesowa byłaby pod względem `IPurchaseOrderV1`.</span><span class="sxs-lookup"><span data-stu-id="c1e84-238">While the service contract’s operations would be written in terms of `PurchaseOrderV1`, the actual business logic would be in terms of `IPurchaseOrderV1`.</span></span> <span data-ttu-id="c1e84-239">Następnie w wersji 2 będzie nowy `IPurchaseOrderV2` interfejs i `PurchaseOrderV2` nowa klasa, jak pokazano w poniższym kodzie:</span><span class="sxs-lookup"><span data-stu-id="c1e84-239">Then, in version 2, there would be a new `IPurchaseOrderV2` interface and a new `PurchaseOrderV2` class as shown in the following code:</span></span>  
  
```csharp

{  
    DateTime OrderDate { get; set; }

[DataContract(
```  
