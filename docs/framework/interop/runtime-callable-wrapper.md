 <span data-ttu-id="90407-105">Środowisko uruchomieniowe tworzy RCW dokładnie jeden dla każdego obiektu COM, bez względu na liczbę odwołań, które istnieją na tym obiekcie.</span><span class="sxs-lookup"><span data-stu-id="90407-105">The runtime creates exactly one RCW for each COM object, regardless of the number of references that exist on that object.</span></span> <span data-ttu-id="90407-106">Środowisko uruchomieniowe przechowuje pojedynczego RCW na proces dla każdego obiektu.</span><span class="sxs-lookup"><span data-stu-id="90407-106">The runtime maintains a single RCW per process for each object.</span></span>  <span data-ttu-id="90407-107">Jeśli tworzenie RCW w jednej domenie aplikacji lub typu apartment, a następnie przekazać odwołanie do innej domeny aplikacji lub typu apartment, będzie używany serwer proxy, aby pierwszy obiekt.</span><span class="sxs-lookup"><span data-stu-id="90407-107">If you create an RCW in one application domain or apartment, and then pass a reference to another application domain or apartment, a proxy to the first object will be used.</span></span>  <span data-ttu-id="90407-108">Jak pokazano na poniższej ilustracji, dowolną liczbę zarządzanych klientów może zawierać odwołania do obiektów COM, które uwidaczniają INew i INewer interfejsów.</span><span class="sxs-lookup"><span data-stu-id="90407-108">As the following illustration shows, any number of managed clients can hold a reference to the COM objects that expose INew and INewer interfaces.</span></span>

 <span data-ttu-id="90407-111">Przy użyciu metadanych pochodzące z biblioteki typów, środowisko uruchomieniowe tworzy wywoływanego obiektu COM i otoki dla tego obiektu.</span><span class="sxs-lookup"><span data-stu-id="90407-111">Using metadata derived from a type library, the runtime creates both the COM object being called and a wrapper for that object.</span></span> <span data-ttu-id="90407-112">Każdy RCW obsługuje pamięć podręczną wskaźniki interfejsu na obiekt COM, otacza i zwalnia swoje odwołanie do obiektu COM, gdy RCW nie jest już potrzebny.</span><span class="sxs-lookup"><span data-stu-id="90407-112">Each RCW maintains a cache of interface pointers on the COM object it wraps and releases its reference on the COM object when the RCW is no longer needed.</span></span> <span data-ttu-id="90407-113">Środowisko uruchomieniowe wykonuje wyrzucanie elementów bezużytecznych na RCW.</span><span class="sxs-lookup"><span data-stu-id="90407-113">The runtime performs garbage collection on the RCW.</span></span>
