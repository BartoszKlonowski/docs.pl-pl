# <a name="using-a-custom-activity"></a><span data-ttu-id="a11b4-102">Używanie niestandardowego działania</span><span class="sxs-lookup"><span data-stu-id="a11b4-102">Using a custom activity</span></span>

<span data-ttu-id="a11b4-103">Działania pochodzące z <xref:System.Activities.Activity> lub jego podklasy mogą być złożone w większych przepływach pracy lub tworzone bezpośrednio w kodzie.</span><span class="sxs-lookup"><span data-stu-id="a11b4-103">Activities that derive from <xref:System.Activities.Activity> or its subclasses can be composed into larger workflows, or created directly in code.</span></span> <span data-ttu-id="a11b4-104">W tym temacie opisano sposób używania działań niestandardowych w przepływach pracy utworzonych w kodzie lub w projektancie.</span><span class="sxs-lookup"><span data-stu-id="a11b4-104">This topic describes how to use custom activities in workflows created either in code or in the designer.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="a11b4-105">Działań niestandardowych można używać w tym samym projekcie, w którym są zdefiniowane, tak długo jak działanie niestandardowe i działanie, które go używa, są kompilowane (tj. ładowane przez typ tworzenia wystąpienia generowanego przez proces kompilacji), jeśli działanie odwołujące jest ładowane dynamicznie (np. przy użyciu obiekt ActivityXamlServices), zestaw, do którego istnieje odwołanie, powinien być umieszczony w innym projekcie lub kod XAML wygenerowany przez projektanta musi być edytowany ręcznie, aby go włączyć.</span><span class="sxs-lookup"><span data-stu-id="a11b4-105">Custom activities can be used in the same project in which they are defined, as long as both the custom activity and the activity that uses it are compiled (i.e. loaded by an instantiating type generated by the build process) If the referencing activity is loaded dynamically (e.g. using ActivityXAMLServices), then the referenced assembly should be placed in a different project, or the designer-generated XAML needs to be hand-edited to enable this.</span></span>  
  
#### <a name="using-a-custom-activity-to-a-workflow-project"></a><span data-ttu-id="a11b4-106">Używanie niestandardowego działania do projektu przepływu pracy</span><span class="sxs-lookup"><span data-stu-id="a11b4-106">Using a custom activity to a workflow project</span></span>  
  
1. <span data-ttu-id="a11b4-107">Dodaj odwołanie z projektu hosta do projektu biblioteki działań zawierającego działanie niestandardowe.</span><span class="sxs-lookup"><span data-stu-id="a11b4-107">Add a reference from the host project to the activity library project containing the custom activity.</span></span>  
  
2. <span data-ttu-id="a11b4-108">Skompiluj rozwiązanie.</span><span class="sxs-lookup"><span data-stu-id="a11b4-108">Build the solution.</span></span>  
  
3. <span data-ttu-id="a11b4-109">Aby użyć działania niestandardowego w projektancie, zlokalizuj działanie niestandardowe w przyborniku i przeciągnij działanie na powierzchnię projektanta.</span><span class="sxs-lookup"><span data-stu-id="a11b4-109">To use the custom activity in the designer, locate the custom activity in the toolbox, and drag the activity onto the designer surface.</span></span>  
  
4. <span data-ttu-id="a11b4-110">Aby użyć działania niestandardowego w kodzie, Dodaj instrukcję using, która odwołuje się do projektu działania niestandardowego, i przekaż nowe wystąpienie działania do <xref:System.Activities.WorkflowInvoker.Invoke%2A> .</span><span class="sxs-lookup"><span data-stu-id="a11b4-110">To use the custom activity in code, add a Using statement that refers to the custom activity project, and pass a new instance of the activity to <xref:System.Activities.WorkflowInvoker.Invoke%2A>.</span></span>
