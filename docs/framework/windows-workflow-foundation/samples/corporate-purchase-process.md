## <a name="process-definition"></a><span data-ttu-id="b895b-135">Definicja procesu</span><span class="sxs-lookup"><span data-stu-id="b895b-135">Process Definition</span></span>  
 <span data-ttu-id="b895b-136">Podstawowa logika przykładu używa <xref:System.Activities.Statements.ParallelForEach%601> działania, które czeka na oferty od każdego dostawcy (przy użyciu niestandardowego działania, które tworzy zakładkę), i rejestruje propozycję dostawcy jako RFP ( <xref:System.Activities.Statements.InvokeMethod> przy użyciu działania).</span><span class="sxs-lookup"><span data-stu-id="b895b-136">The core logic of the sample uses a <xref:System.Activities.Statements.ParallelForEach%601> activity that waits for the offers from each vendor (using a custom activity that creates a bookmark), and registers the vendor proposal as an RFP (using an <xref:System.Activities.Statements.InvokeMethod> activity).</span></span>  
  
 <span data-ttu-id="b895b-137">Następnie próbka wykonuje iterację we wszystkich odebranych propozycjach przechowywanych `RfpRepository`w, obliczaniu wartości skorygowanej ( <xref:System.Activities.Statements.Assign> przy użyciu <xref:System.Activities.Expressions> działania i działań), a jeśli skorygowana wartość jest lepsza niż poprzednia Najlepsza oferta, przypisuje nową wartość jako najlepszą ofertę (przy użyciu <xref:System.Activities.Statements.If> i <xref:System.Activities.Statements.Assign> działania).</span><span class="sxs-lookup"><span data-stu-id="b895b-137">The sample then iterates through all of the received proposals stored in the `RfpRepository`, calculating the adjusted value (using an <xref:System.Activities.Statements.Assign> activity and <xref:System.Activities.Expressions> activities), and if the adjusted value is better than the previous best offer, assigns the new value as the best offer (using <xref:System.Activities.Statements.If> and <xref:System.Activities.Statements.Assign> activities).</span></span>
