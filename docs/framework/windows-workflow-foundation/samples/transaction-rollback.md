# <a name="transaction-rollback"></a><span data-ttu-id="5a5a1-102">Cofnięcie transakcji</span><span class="sxs-lookup"><span data-stu-id="5a5a1-102">Transaction Rollback</span></span>
<span data-ttu-id="5a5a1-103">W tym przykładzie pokazano, jak utworzyć niestandardową <xref:System.Activities.NativeActivity> uzyskuje dostęp otoczenia <xref:System.Activities.RuntimeTransactionHandle> uzyskać otoczenia transakcji i jawnie wycofać je.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-103">This sample shows how to create a custom <xref:System.Activities.NativeActivity> that accesses the ambient <xref:System.Activities.RuntimeTransactionHandle> to get the ambient transaction and explicitly roll it back.</span></span>  
  
## <a name="sample-details"></a><span data-ttu-id="5a5a1-104">Przykład szczegółów</span><span class="sxs-lookup"><span data-stu-id="5a5a1-104">Sample Details</span></span>  
 <span data-ttu-id="5a5a1-105">W przepływie pracy, transakcja zostanie automatycznie zakończone, kiedy najbardziej zewnętrzna <xref:System.Activities.Statements.TransactionScope> lub <xref:System.ServiceModel.Activities.TransactedReceiveScope> kończy.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-105">In workflow, a transaction is automatically completed when the outermost <xref:System.Activities.Statements.TransactionScope> or <xref:System.ServiceModel.Activities.TransactedReceiveScope> completes.</span></span>  <span data-ttu-id="5a5a1-106">Transakcji przedstawia niejawnie, gdy wystąpił nieobsługiwany wyjątek propagowanie granice zakresu.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-106">A transaction implicitly rolls back when an unhandled exception propagates across the scope boundary.</span></span> <span data-ttu-id="5a5a1-107">Jednak może być czasy, kiedy warto jawnie wycofać transakcji bez konieczności zgłoszenie wyjątku.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-107">However, there may be times when it makes sense to explicitly roll back a transaction without having to throw an exception.</span></span> <span data-ttu-id="5a5a1-108">W takim przypadku służy działanie niestandardowe wycofywania, jak ta, w tym przykładzie jawnie przerwać otoczenia transakcji i zapewnia Przyczyna wyjątku opcjonalne.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-108">In this case, you can use the custom rollback activity like the one in this sample to explicitly abort the ambient transaction and provide an optional exception reason.</span></span>  
  
 <span data-ttu-id="5a5a1-109">`RollbackActivity` Jest <xref:System.Activities.NativeActivity> ponieważ wymaga dostępu do właściwości wykonywania, aby pobrać otoczenia <xref:System.Activities.RuntimeTransactionHandle>.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-109">The `RollbackActivity` is a <xref:System.Activities.NativeActivity> as it requires access to the execution properties to get the ambient <xref:System.Activities.RuntimeTransactionHandle>.</span></span> <span data-ttu-id="5a5a1-110">W `Execute` metody uzyskuje <xref:System.Activities.RuntimeTransactionHandle> i sprawdza, czy jest `null`, wskazuje, czy działanie zostało użyte bez otoczenia transakcji czasu wykonywania.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-110">In the `Execute` method, it obtains the <xref:System.Activities.RuntimeTransactionHandle> and checks whether it is `null`, which indicates that the activity was used without an ambient run-time transaction.</span></span> <span data-ttu-id="5a5a1-111">Następnie uzyskuje transakcja sprawdzanie ponownie czy `null` jest obecny.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-111">It then obtains the transaction, again checking whether `null` is present.</span></span> <span data-ttu-id="5a5a1-112">Istnieje możliwość mają otoczenia <xref:System.Activities.RuntimeTransactionHandle> bez kiedykolwiek inicjowanie transakcji czasu wykonywania.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-112">It is possible to have an ambient <xref:System.Activities.RuntimeTransactionHandle> without ever initializing a run-time transaction.</span></span> <span data-ttu-id="5a5a1-113">Na koniec przerywa transakcję, wywołując <xref:System.Transactions.Transaction.Rollback%2A> i określenie wyjątku dostarczone przez użytkownika lub wyjątek ogólny, z informacją, że działanie przywracana transakcji.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-113">Finally it aborts the transaction by calling <xref:System.Transactions.Transaction.Rollback%2A> and specifying either a user-provided exception or a generic exception that states that the activity rolled back the transaction.</span></span>  
  
 <span data-ttu-id="5a5a1-114">Pokaz przepływu pracy składa się z <xref:System.Activities.Statements.TransactionScope> którego treści Wyświetla stan transakcji przed i po nim `RollbackActivity` wykonuje.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-114">The demonstration workflow consists of a <xref:System.Activities.Statements.TransactionScope> whose body prints the transaction status before and after the `RollbackActivity` executes.</span></span> <span data-ttu-id="5a5a1-115">Należy pamiętać, że nawet jeśli transakcja została wycofana, <xref:System.Activities.Statements.TransactionScope> wykonuje ukończone i nie przerwać przepływu pracy do chwili zakończenia treści.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-115">Note that even though the transaction has been rolled back, the <xref:System.Activities.Statements.TransactionScope> executes to completion and does not abort the workflow until the body completes.</span></span> <span data-ttu-id="5a5a1-116">Przepływ pracy zostało przerwane, ponieważ <xref:System.Activities.Statements.TransactionScope.AbortInstanceOnTransactionFailure%2A> właściwości wartość domyślna to `true`.</span><span class="sxs-lookup"><span data-stu-id="5a5a1-116">The workflow is aborted because the <xref:System.Activities.Statements.TransactionScope.AbortInstanceOnTransactionFailure%2A> property defaults to `true`.</span></span>
