## <a name="wpf-and-assembly-loading"></a><span data-ttu-id="9699f-162">Ładowanie WPF i zestawu</span><span class="sxs-lookup"><span data-stu-id="9699f-162">WPF and Assembly Loading</span></span>  
 <span data-ttu-id="9699f-163">Kontekst schematu XAML dla WPF integruje się z modelem aplikacji WPF, który z kolei używa koncepcji zdefiniowanej przez środowisko CLR <xref:System.AppDomain>.</span><span class="sxs-lookup"><span data-stu-id="9699f-163">The XAML schema context for WPF integrates with the WPF application model, which in turn uses the CLR-defined concept of <xref:System.AppDomain>.</span></span> <span data-ttu-id="9699f-164">Poniższa sekwencja opisuje sposób, w jaki kontekst schematu XAML interpretuje sposób ładowania zestawów lub znajdowania typów w czasie wykonywania lub projektowania, na podstawie użycia platformy WPF <xref:System.AppDomain> i innych czynników.</span><span class="sxs-lookup"><span data-stu-id="9699f-164">The following sequence describes how XAML schema context interprets how to either load assemblies or find types at run time or design time, based on the WPF use of <xref:System.AppDomain> and other factors.</span></span>  
  
1. <span data-ttu-id="9699f-165">Wykonaj iterację <xref:System.AppDomain>, szukając już załadowanego zestawu, który pasuje do wszystkich aspektów nazwy, rozpoczynając od ostatnio załadowanego zestawu.</span><span class="sxs-lookup"><span data-stu-id="9699f-165">Iterate through the <xref:System.AppDomain>, looking for an already-loaded assembly that matches all aspects of the name, starting from the most recently loaded assembly.</span></span>  
  
2. <span data-ttu-id="9699f-166">Jeśli nazwa jest kwalifikowana, wywołaj <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType> na kwalifikowanej nazwie.</span><span class="sxs-lookup"><span data-stu-id="9699f-166">If the name is qualified, call <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType> on the qualified name.</span></span>  
  
3. <span data-ttu-id="9699f-167">Jeśli krótka nazwa i token klucza publicznego kwalifikowanej nazwy są zgodne z zestawem, z którego został załadowany znacznik, zwróć ten zestaw.</span><span class="sxs-lookup"><span data-stu-id="9699f-167">If the short name + public key token of a qualified name matches the assembly that the markup was loaded from, return that assembly.</span></span>  
  
4. <span data-ttu-id="9699f-168">Użyj krótkiej nazwy i tokenu klucza publicznego do wywoływania <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="9699f-168">Use the short name + public key token to call <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType>.</span></span>  
  
5. <span data-ttu-id="9699f-169">Jeśli nazwa jest niekwalifikowana, wywołaj <xref:System.Reflection.Assembly.LoadWithPartialName%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="9699f-169">If the name is unqualified, call <xref:System.Reflection.Assembly.LoadWithPartialName%2A?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="9699f-170">Luźny kod XAML nie używa kroku 3; Brak załadowanego zestawu.</span><span class="sxs-lookup"><span data-stu-id="9699f-170">Loose XAML does not use Step 3; there is no loaded-from assembly.</span></span>  
  
 <span data-ttu-id="9699f-171">Skompilowany kod XAML dla WPF (wygenerowany za pośrednictwem XamlBuildTask) nie używa już załadowanych zestawów z <xref:System.AppDomain> (krok 1).</span><span class="sxs-lookup"><span data-stu-id="9699f-171">Compiled XAML for WPF (generated via XamlBuildTask) does not use the already-loaded assemblies from <xref:System.AppDomain> (Step 1).</span></span> <span data-ttu-id="9699f-172">Ponadto nazwa nigdy nie powinna być niekwalifikowana z danych wyjściowych XamlBuildTask, więc krok 5 nie ma zastosowania.</span><span class="sxs-lookup"><span data-stu-id="9699f-172">Also, the name should never be unqualified from XamlBuildTask output, so Step 5 does not apply.</span></span>  
  
 <span data-ttu-id="9699f-173">Skompilowana BAML (wygenerowana za pośrednictwem PresentationBuildTask) używa wszystkich kroków, chociaż BAML nie powinna również zawierać niekwalifikowanych nazw zestawów.</span><span class="sxs-lookup"><span data-stu-id="9699f-173">Compiled BAML (generated via PresentationBuildTask) uses all steps, although BAML also should not contain unqualified assembly names.</span></span>
