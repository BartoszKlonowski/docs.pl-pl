## <a name="wpf-and-assembly-loading"></a><span data-ttu-id="ebbb6-162">WPF i ładowania zestawu</span><span class="sxs-lookup"><span data-stu-id="ebbb6-162">WPF and Assembly Loading</span></span>  
 <span data-ttu-id="ebbb6-163">Kontekst schematu WPF XAML integruje się z modelem aplikacji WPF, który z kolei używa zdefiniowane CLR koncepcji <xref:System.AppDomain>.</span><span class="sxs-lookup"><span data-stu-id="ebbb6-163">The XAML schema context for WPF integrates with the WPF application model, which in turn uses the CLR-defined concept of <xref:System.AppDomain>.</span></span> <span data-ttu-id="ebbb6-164">Poniższa sekwencja opisuje, jak kontekst schematu XAML interpretuje jak ładować zestawy lub znaleźć typów w czasie wykonywania lub czasie projektowania na podstawie związane z użyciem WPF <xref:System.AppDomain> i inne czynniki.</span><span class="sxs-lookup"><span data-stu-id="ebbb6-164">The following sequence describes how XAML schema context interprets how to either load assemblies or find types at run time or design time, based on the WPF use of <xref:System.AppDomain> and other factors.</span></span>  
  
1.  <span data-ttu-id="ebbb6-165">Iteracyjne przeglądanie <xref:System.AppDomain>, wyszukiwanie już załadowany zestaw, który dopasowuje wszystkie aspekty nazwę, od ostatniego załadowany zestaw.</span><span class="sxs-lookup"><span data-stu-id="ebbb6-165">Iterate through the <xref:System.AppDomain>, looking for an already-loaded assembly that matches all aspects of the name, starting from the most recently loaded assembly.</span></span>  
  
2.  <span data-ttu-id="ebbb6-166">Jeśli nazwa jest kwalifikowana, wywołaj <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType> kwalifikowanej nazwy.</span><span class="sxs-lookup"><span data-stu-id="ebbb6-166">If the name is qualified, call <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType> on the qualified name.</span></span>  
  
3.  <span data-ttu-id="ebbb6-167">Jeśli krótką nazwę i token klucza publicznego kwalifikowana nazwa pasuje do zestawu, który znaczników został załadowany z, zwróć tego zestawu.</span><span class="sxs-lookup"><span data-stu-id="ebbb6-167">If the short name + public key token of a qualified name matches the assembly that the markup was loaded from, return that assembly.</span></span>  
  
4.  <span data-ttu-id="ebbb6-168">Użyj krótkiej nazwy i token klucza publicznego do wywołania <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="ebbb6-168">Use the short name + public key token to call <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType>.</span></span>  
  
5.  <span data-ttu-id="ebbb6-169">Jeśli nazwa jest niekwalifikowana, wywołaj <xref:System.Reflection.Assembly.LoadWithPartialName%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="ebbb6-169">If the name is unqualified, call <xref:System.Reflection.Assembly.LoadWithPartialName%2A?displayProperty=nameWithType>.</span></span>

 <span data-ttu-id="ebbb6-171">Skompilowany XAML dla WPF (generowany za pośrednictwem XamlBuildTask) nie używa zestawów już załadowany z <xref:System.AppDomain> (krok 1).</span><span class="sxs-lookup"><span data-stu-id="ebbb6-171">Compiled XAML for WPF (generated via XamlBuildTask) does not use the already-loaded assemblies from <xref:System.AppDomain> (Step 1).</span></span> <span data-ttu-id="ebbb6-172">Ponadto nazwę nigdy nie należy niekwalifikowanej z danych wyjściowych XamlBuildTask, więc nie ma zastosowania kroku 5.</span><span class="sxs-lookup"><span data-stu-id="ebbb6-172">Also, the name should never be unqualified from XamlBuildTask output, so Step 5 does not apply.</span></span>
