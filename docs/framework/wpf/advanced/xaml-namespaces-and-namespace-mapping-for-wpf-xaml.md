## <a name="wpf-and-assembly-loading"></a><span data-ttu-id="b3c26-162">WPF i ładowania zestawu</span><span class="sxs-lookup"><span data-stu-id="b3c26-162">WPF and Assembly Loading</span></span>  
 <span data-ttu-id="b3c26-163">Kontekst schematu WPF XAML integruje się z modelem aplikacji WPF, który z kolei używa pojęcie zdefiniowane CLR <xref:System.AppDomain>.</span><span class="sxs-lookup"><span data-stu-id="b3c26-163">The XAML schema context for WPF integrates with the WPF application model, which in turn uses the CLR-defined concept of <xref:System.AppDomain>.</span></span> <span data-ttu-id="b3c26-164">Następująca sekwencja opisano, jak kontekst schematu XAML interpretuje jak ładowanie zestawów albo znaleźć typów w czasie wykonywania lub projektu, na podstawie z użyciem WPF <xref:System.AppDomain> i innych czynników.</span><span class="sxs-lookup"><span data-stu-id="b3c26-164">The following sequence describes how XAML schema context interprets how to either load assemblies or find types at run time or design time, based on the WPF use of <xref:System.AppDomain> and other factors.</span></span>  
  
1.  <span data-ttu-id="b3c26-165">Iterowanie za pomocą <xref:System.AppDomain>, wyszukiwanie już załadowany zestaw, który dopasowuje wszystkie aspekty nazwy, zaczynając od ostatniego załadowanego zestawu.</span><span class="sxs-lookup"><span data-stu-id="b3c26-165">Iterate through the <xref:System.AppDomain>, looking for an already-loaded assembly that matches all aspects of the name, starting from the most recently loaded assembly.</span></span>

3.  <span data-ttu-id="b3c26-167">Jeśli krótka nazwa + token klucza publicznego kwalifikowana nazwa pasuje do zestawu, który znaczników został załadowany z, zwróć tego zestawu.</span><span class="sxs-lookup"><span data-stu-id="b3c26-167">If the short name + public key token of a qualified name matches the assembly that the markup was loaded from, return that assembly.</span></span>

 <span data-ttu-id="b3c26-171">Skompilowany XAML dla WPF (wygenerowane za pomocą XamlBuildTask) nie korzysta z zestawów już załadowany z <xref:System.AppDomain> (krok 1).</span><span class="sxs-lookup"><span data-stu-id="b3c26-171">Compiled XAML for WPF (generated via XamlBuildTask) does not use the already-loaded assemblies from <xref:System.AppDomain> (Step 1).</span></span> <span data-ttu-id="b3c26-172">Ponadto nazwa nigdy nie należy niekwalifikowane z danych wyjściowych XamlBuildTask, tak, aby nie stosować krok 5.</span><span class="sxs-lookup"><span data-stu-id="b3c26-172">Also, the name should never be unqualified from XamlBuildTask output, so Step 5 does not apply.</span></span>
