 <span data-ttu-id="7d6bf-105">Dla właściwości zależności w ogóle wzorzec implementacji, który należy wykonać jest zdefiniowanie otoki <xref:System.Windows.DependencyProperty> właściwości CLR, gdzie ta właściwość jest wspierany przez identyfikator, a nie pole lub inne konstrukcje.</span><span class="sxs-lookup"><span data-stu-id="7d6bf-105">For a dependency property in general, the implementation pattern that you follow is that you define a CLR property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="7d6bf-106">Postępuj zgodnie z tym samym wzorcem podczas implementowania właściwości typu kolekcji.</span><span class="sxs-lookup"><span data-stu-id="7d6bf-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="7d6bf-107">Jednak właściwość typu kolekcji wprowadza pewną złożoność do wzorca, gdy typ, który <xref:System.Windows.DependencyObject> <xref:System.Windows.Freezable> jest zawarty w kolekcji jest sama lub klasa pochodna.</span><span class="sxs-lookup"><span data-stu-id="7d6bf-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

zamiast tego jest to wartość domyślna, która ma zastosowanie do wszystkich wystąpień typu.</span><span class="sxs-lookup"><span data-stu-id="7d6bf-111">If your property is a reference type, the default value specified in dependency property metadata is not a default value per instance; ## <a name="reporting-binding-value-changes-from-collection-properties"></a><span data-ttu-id="7d6bf-130">Raportowanie zmian wartości wiązania z właściwości kolekcji</span><span class="sxs-lookup"><span data-stu-id="7d6bf-130">Reporting Binding Value Changes from Collection Properties</span></span>  
 <span data-ttu-id="7d6bf-131">Właściwość kolekcji, która sama jest właściwością zależności, nie zgłasza automatycznie zmian w jego właściwościach podrzędnych.</span><span class="sxs-lookup"><span data-stu-id="7d6bf-131">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="7d6bf-132">Jeśli tworzysz powiązania w kolekcji, może to uniemożliwić powiązanie raportowania zmian, w ten sposób unieważnianie niektórych scenariuszy powiązania danych.</span><span class="sxs-lookup"><span data-stu-id="7d6bf-132">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="7d6bf-133">Jednak jeśli używasz typu <xref:System.Windows.FreezableCollection%601> kolekcji jako typ kolekcji, a następnie zmiany podwładne do zawartych elementów w kolekcji są poprawnie zgłaszane i powiązania działa zgodnie z oczekiwaniami.</span><span class="sxs-lookup"><span data-stu-id="7d6bf-133">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>
