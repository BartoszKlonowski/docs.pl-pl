 <span data-ttu-id="00bf4-105">Dla właściwości zależności ogólnie rzecz biorąc, implementacja wzorzec stosowanej jest zdefiniowanie [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] otoki właściwości, w którym ta właściwość jest wspierana przez <xref:System.Windows.DependencyProperty> identyfikatora, a nie pola lub innej konstrukcji.</span><span class="sxs-lookup"><span data-stu-id="00bf4-105">For a dependency property in general, the implementation pattern that you follow is that you define a [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="00bf4-106">Podczas implementacji właściwości typu kolekcji, stosować tego samego wzorca.</span><span class="sxs-lookup"><span data-stu-id="00bf4-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="00bf4-107">Jednak właściwość typu kolekcji wprowadza złożoność wzorca zawsze wtedy, gdy typ, który znajduje się w tej kolekcji sam <xref:System.Windows.DependencyObject> lub <xref:System.Windows.Freezable> klasy pochodnej.</span><span class="sxs-lookup"><span data-stu-id="00bf4-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

## <a name="reporting-binding-value-changes-from-collection-properties"></a><span data-ttu-id="00bf4-129">Raportowanie zmiany wartości powiązania z właściwościami kolekcji</span><span class="sxs-lookup"><span data-stu-id="00bf4-129">Reporting Binding Value Changes from Collection Properties</span></span>  
 <span data-ttu-id="00bf4-130">Właściwość kolekcji, która sama jest właściwość zależności nie raportuje automatycznie zmiany do właściwości podrzędnych.</span><span class="sxs-lookup"><span data-stu-id="00bf4-130">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="00bf4-131">W przypadku tworzenia powiązań do kolekcji, może to spowodować tworzenie powiązań z raportowania zmian, tym samym unieważniając Niektóre scenariusze powiązania danych.</span><span class="sxs-lookup"><span data-stu-id="00bf4-131">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="00bf4-132">Jednak jeśli używasz typu kolekcji <xref:System.Windows.FreezableCollection%601> jako typ kolekcji, następnie zmiany właściwości podrzędnych elementów zawartych w kolekcji są prawidłowo raportowane, oraz powiązanie działa zgodnie z oczekiwaniami.</span><span class="sxs-lookup"><span data-stu-id="00bf4-132">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>
