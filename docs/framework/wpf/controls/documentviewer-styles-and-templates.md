# <a name="documentviewer-styles-and-templates"></a>DocumentViewer — Style i szablony
W tym temacie opisano, style i szablonów dla <xref:System.Windows.Controls.DocumentViewer> formantu. Można zmodyfikować domyślne <xref:System.Windows.Controls.ControlTemplate> umożliwiają unikatowego wyglądu formantu. Aby uzyskać więcej informacji, zobacz [Dostosowywanie wyglądu formant tworząc ControlTemplate](../../../../docs/framework/wpf/controls/customizing-the-appearance-of-an-existing-control.md).  
  
## <a name="documentviewer-parts"></a>Części DocumentViewer  
 W poniższej tabeli wymieniono nazwanego części dla <xref:System.Windows.Controls.DocumentViewer> formantu.  
  
|Część|Typ|Opis|  
|-|-|-|  
|PART_ContentHost|<xref:System.Windows.Controls.ScrollViewer>|Zawartość i obszar przewijania.|  
|PART_FindToolBarHost|<xref:System.Windows.Controls.ContentControl>|Pole wyszukiwania na dole domyślnie.|  
  
## <a name="documentviewer-states"></a>Stany DocumentViewer  
 W poniższej tabeli wymieniono stany wizualne dla <xref:System.Windows.Controls.DocumentViewer> formantu.  
  
|Nazwa stanu wizualnego|Nazwa VisualStateGroup|Opis|  
|-|-|-|  
|Prawidłowe|ValidationStates|Używa kontrolki <xref:System.Windows.Controls.Validation> klasy i <xref:System.Windows.Controls.Validation.HasError%2A?displayProperty=nameWithType> jest dołączona właściwość `false`.|  
|InvalidFocused|ValidationStates|<xref:System.Windows.Controls.Validation.HasError%2A?displayProperty=nameWithType> Jest dołączona właściwość `true` ma formant ma fokus.|  
|InvalidUnfocused|ValidationStates|<xref:System.Windows.Controls.Validation.HasError%2A?displayProperty=nameWithType> Jest dołączona właściwość `true` ma formant nie ma fokusa.|
