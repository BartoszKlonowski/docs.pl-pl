# <a name="xfieldmodifier-directive"></a><span data-ttu-id="fbd5d-102">x:FieldModifier — dyrektywa</span><span class="sxs-lookup"><span data-stu-id="fbd5d-102">x:FieldModifier Directive</span></span>
<span data-ttu-id="fbd5d-103">Modyfikuje zachowanie kompilacji XAML, tak aby pól dla odwołania do nazwanych obiektów są zdefiniowane z <xref:System.Reflection.TypeAttributes.Public?displayProperty=nameWithType> dostępu zamiast <xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> zachowanie domyślne.</span><span class="sxs-lookup"><span data-stu-id="fbd5d-103">Modifies XAML compilation behavior so that fields for named object references are defined with <xref:System.Reflection.TypeAttributes.Public?displayProperty=nameWithType> access instead of the <xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> default behavior.</span></span>  
  
## <a name="xaml-attribute-usage"></a><span data-ttu-id="fbd5d-104">Użycie atrybutu języka XAML</span><span class="sxs-lookup"><span data-stu-id="fbd5d-104">XAML Attribute Usage</span></span>  
  
```xaml  
<object x:FieldModifier="Public".../>  
```

 <span data-ttu-id="fbd5d-116">Wartość `x:FieldModifier` atrybutu zależy od języka programowania, a jej celem może różnić się w określonej struktury.</span><span class="sxs-lookup"><span data-stu-id="fbd5d-116">The value for the `x:FieldModifier` attribute varies by programming language, and its purpose can vary in specific frameworks.</span></span> <span data-ttu-id="fbd5d-117">Ciąg do użycia zależy od tego, jak implementuje każdego języka jego <xref:System.CodeDom.Compiler.CodeDomProvider> i konwertery typu zwraca do definiowania znaczenie dla <xref:System.Reflection.TypeAttributes.Public?displayProperty=nameWithType> i <xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType>, oraz czy ten język jest uwzględniana wielkość liter.</span><span class="sxs-lookup"><span data-stu-id="fbd5d-117">The string to use depends on how each language implements its <xref:System.CodeDom.Compiler.CodeDomProvider> and the type converters it returns to define the meanings for <xref:System.Reflection.TypeAttributes.Public?displayProperty=nameWithType> and <xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType>, and whether that language is case sensitive.</span></span>

 <span data-ttu-id="fbd5d-121">Można również określić <xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> (`internal` w języku C# `Friend` w języku Visual Basic) ale określania <xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> rzadko ponieważ `NotPublic` jako zachowanie jest już domyślnie.</span><span class="sxs-lookup"><span data-stu-id="fbd5d-121">You can also specify <xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> (`internal` in C#, `Friend` in Visual Basic) but specifying <xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> is unusual because `NotPublic` as the behavior is already the default.</span></span>  
  
 <span data-ttu-id="fbd5d-122"><xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> jest zachowanie domyślne, ponieważ jest rzadko, że kod spoza zestawu, który skompilowany XAML musi mieć dostęp do elementu utworzonych w języku XAML.</span><span class="sxs-lookup"><span data-stu-id="fbd5d-122"><xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> is the default behavior because it is infrequent that code outside the assembly that compiled the XAML needs access to a XAML-created element.</span></span> <span data-ttu-id="fbd5d-123">Architektura zabezpieczeń WPF wraz z zachowaniem kompilacji XAML nie deklaruje pola, które przechowują wystąpienia elementu jako public, chyba że użytkownik `x:FieldModifier` umożliwiających dostęp do publicznego.</span><span class="sxs-lookup"><span data-stu-id="fbd5d-123">WPF security architecture together with XAML compilation behavior will not declare fields that store element instances as public, unless you specifically set the `x:FieldModifier` to allow public access.</span></span>

 <span data-ttu-id="fbd5d-125">Domyślnie jest publiczny; częściowej klasy dla elementu głównego jednak można tworzyć niepubliczne przy użyciu [x: classmodifier — dyrektywa](../../../docs/framework/xaml-services/x-classmodifier-directive.md).</span><span class="sxs-lookup"><span data-stu-id="fbd5d-125">By default, the partial class for the root element is public; however, you can make it nonpublic by using the [x:ClassModifier Directive](../../../docs/framework/xaml-services/x-classmodifier-directive.md).</span></span> <span data-ttu-id="fbd5d-126">[X: classmodifier — dyrektywa](../../../docs/framework/xaml-services/x-classmodifier-directive.md) również wpływa na poziom dostępu do wystąpienia klasy głównym elementu.</span><span class="sxs-lookup"><span data-stu-id="fbd5d-126">The [x:ClassModifier Directive](../../../docs/framework/xaml-services/x-classmodifier-directive.md) also affects the access level of the instance of the root element class.</span></span> <span data-ttu-id="fbd5d-127">Możesz też zaznaczyć oba `x:Name` i `x:FieldModifier` w katalogu głównym elementu, ale tylko sprawia, że pole publiczne kopię element główny z głównego true element klasy poziom dostępu nadal kontrolowane przez [x: classmodifier — dyrektywa](../../../docs/framework/xaml-services/x-classmodifier-directive.md).</span><span class="sxs-lookup"><span data-stu-id="fbd5d-127">You can put both `x:Name` and `x:FieldModifier` on the root element, but this only makes a public field copy of the root element, with the true root element class access level still controlled by [x:ClassModifier Directive](../../../docs/framework/xaml-services/x-classmodifier-directive.md).</span></span>
