# <a name="xshared-attribute"></a><span data-ttu-id="c986f-102">x:Shared — Atrybut</span><span class="sxs-lookup"><span data-stu-id="c986f-102">x:Shared Attribute</span></span>
<span data-ttu-id="c986f-103">Po ustawieniu na `false`modyfikuje zachowanie pobierania zasobów WPF, tak aby żądania dla zasobu z atrybutami utworzyć nowe wystąpienie dla każdego żądania, zamiast udostępniać to samo wystąpienie dla wszystkich żądań.</span><span class="sxs-lookup"><span data-stu-id="c986f-103">When set to `false`, modifies WPF resource-retrieval behavior so that requests for the attributed resource create a new instance for each request instead of sharing the same instance for all requests.</span></span>  
  
## <a name="xaml-attribute-usage"></a><span data-ttu-id="c986f-104">Użycie atrybutu języka XAML</span><span class="sxs-lookup"><span data-stu-id="c986f-104">XAML Attribute Usage</span></span>  
  
```xaml  
<ResourceDictionary>  
  <object x:Shared="false".../>  
</ResourceDictionary>  
```

 <span data-ttu-id="c986f-106">`x:Shared` jest mapowany do przestrzeni nazw XAML języka XAML i jest rozpoznawany jako prawidłowy element języka XAML przez .NET Framework usług XAML i ich czytników XAML.</span><span class="sxs-lookup"><span data-stu-id="c986f-106">`x:Shared` is mapped to the XAML language XAML namespace and is recognized as a valid XAML language element by .NET Framework XAML Services and its XAML readers.</span></span> <span data-ttu-id="c986f-107">Podane możliwości `x:Shared` są jednak odpowiednie dla aplikacji WPF i analizatora XAML WPF.</span><span class="sxs-lookup"><span data-stu-id="c986f-107">However, the stated capabilities of `x:Shared` are only relevant for WPF applications and for the WPF XAML parser.</span></span> <span data-ttu-id="c986f-108">W WPF `x:Shared` jest przydatny tylko jako atrybut, gdy jest stosowany do obiektu, który istnieje w <xref:System.Windows.ResourceDictionary>WPF.</span><span class="sxs-lookup"><span data-stu-id="c986f-108">In WPF, `x:Shared` is only useful as an attribute when it is applied to an object that exists within a WPF <xref:System.Windows.ResourceDictionary>.</span></span> <span data-ttu-id="c986f-109">Inne użycia nie generują wyjątków analizy ani innych błędów, ale nie mają żadnego wpływu.</span><span class="sxs-lookup"><span data-stu-id="c986f-109">Other usages do not throw parse exceptions or other errors, but they have no effect.</span></span>  
  
 <span data-ttu-id="c986f-110">Znaczenia `x:Shared` nie określono w specyfikacji języka XAML.</span><span class="sxs-lookup"><span data-stu-id="c986f-110">The meaning of `x:Shared` is not specified in the XAML language specification.</span></span> <span data-ttu-id="c986f-111">Inne implementacje języka XAML, takie jak te, które kompilują .NET Framework usług XAML, nie muszą zapewniać obsługi udostępniania zasobów.</span><span class="sxs-lookup"><span data-stu-id="c986f-111">Other XAML implementations, such as those that build on .NET Framework XAML Services, do not necessarily provide resource-sharing support.</span></span> <span data-ttu-id="c986f-112">Takie implementacje języka XAML mogą zapewnić podobne zachowanie w strukturze pomocniczej, która również używa wartości `x:Shared`.</span><span class="sxs-lookup"><span data-stu-id="c986f-112">Such XAML implementations could provide similar behavior in the supporting framework that also used `x:Shared` values.</span></span>

 <span data-ttu-id="c986f-115">Modyfikowanie obiektu, który jest zwracany za pomocą interfejsu API zasobów, takiego jak <xref:System.Windows.FrameworkElement.FindResource%2A>, lub modyfikowanie obiektu bezpośrednio w ramach <xref:System.Windows.ResourceDictionary>, powoduje zmianę oryginalnego zasobu.</span><span class="sxs-lookup"><span data-stu-id="c986f-115">Modifying an object that is returned through a resource API, such as <xref:System.Windows.FrameworkElement.FindResource%2A>, or modifying an object directly within a <xref:System.Windows.ResourceDictionary>, changes the original resource.</span></span> <span data-ttu-id="c986f-116">Jeśli odwołania do tego zasobu były odwołaniami do zasobów dynamicznych, odbiorcy tego zasobu otrzymują zmieniony zasób.</span><span class="sxs-lookup"><span data-stu-id="c986f-116">If references to that resource were dynamic resource references, the consumers of that resource get the changed resource.</span></span>  
  
 <span data-ttu-id="c986f-117">Jeśli odwołania do zasobu były odwołaniami zasobów statycznych, zmiany w zasobie po czasie przetwarzania [!INCLUDE[TLA2#tla_xaml](../../../includes/tla2sharptla-xaml-md.md)] są nieistotne.</span><span class="sxs-lookup"><span data-stu-id="c986f-117">If references to the resource were static resource references, changes to the resource after [!INCLUDE[TLA2#tla_xaml](../../../includes/tla2sharptla-xaml-md.md)] processing time are irrelevant.</span></span> <span data-ttu-id="c986f-118">Aby uzyskać więcej informacji na temat odwołań do zasobów statycznych i dynamicznych, zobacz [zasoby XAML](../../desktop-wpf/fundamentals/xaml-resources-define.md).</span><span class="sxs-lookup"><span data-stu-id="c986f-118">For more information about static versus dynamic resource references, see [XAML Resources](../../desktop-wpf/fundamentals/xaml-resources-define.md).</span></span>  
  
 <span data-ttu-id="c986f-119">Jawne określanie `x:Shared="true"` jest rzadko wykonywane, ponieważ jest już wartością domyślną.</span><span class="sxs-lookup"><span data-stu-id="c986f-119">Explicitly specifying `x:Shared="true"` is rarely done, because that is already the default.</span></span> <span data-ttu-id="c986f-120">Nie ma bezpośredniego odpowiednika kodu dla `x:Shared` w modelu obiektów WPF; może być określony tylko w użyciu języka XAML i musi być przetwarzany przy użyciu domyślnego zachowania WPF lub w pośrednim strumieniu węzła XAML na ścieżce ładowania, jeśli jest przetwarzany za pomocą .NET Framework usług XAML i ich czytników XAML.</span><span class="sxs-lookup"><span data-stu-id="c986f-120">There is no direct code equivalent for `x:Shared` in the WPF object model; it can only be specified in a XAML usage and must be processed either by the default WPF behavior or in an intermediate XAML node stream on the load path if processed using .NET Framework XAML Services and its XAML readers.</span></span>  
  
 <span data-ttu-id="c986f-121">Scenariusz dla `x:Shared="false"` polega na zdefiniowaniu <xref:System.Windows.FrameworkElement> lub <xref:System.Windows.FrameworkContentElement> klasie pochodnej jako zasobu, a następnie wprowadzeniu zasobu elementu do modelu zawartości.</span><span class="sxs-lookup"><span data-stu-id="c986f-121">A scenario for `x:Shared="false"` is if you define a <xref:System.Windows.FrameworkElement> or <xref:System.Windows.FrameworkContentElement> derived class as a resource and then you introduce the element resource into a content model.</span></span> <span data-ttu-id="c986f-122">`x:Shared="false"` umożliwia wprowadzanie zasobu elementu wiele razy w tej samej kolekcji (na przykład <xref:System.Windows.Controls.UIElementCollection>).</span><span class="sxs-lookup"><span data-stu-id="c986f-122">`x:Shared="false"` enables an element resource to be introduced multiple times in the same collection (such as a <xref:System.Windows.Controls.UIElementCollection>).</span></span> <span data-ttu-id="c986f-123">Bez `x:Shared="false"` jest to nieprawidłowe, ponieważ kolekcja Wymusza unikatowość jego zawartości.</span><span class="sxs-lookup"><span data-stu-id="c986f-123">Without `x:Shared="false"` this is invalid because the collection enforces uniqueness of its contents.</span></span> <span data-ttu-id="c986f-124">Jednak zachowanie `x:Shared="false"` powoduje utworzenie innego identycznego wystąpienia zasobu, a nie zwrócenie tego samego wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="c986f-124">However, the `x:Shared="false"` behavior creates another identical instance of the resource instead of returning the same instance.</span></span>  
  
 <span data-ttu-id="c986f-125">Innym scenariuszem dla `x:Shared="false"` jest użycie zasobu <xref:System.Windows.Freezable> do wartości animacji, ale chcesz zmodyfikować zasób dla każdej animacji.</span><span class="sxs-lookup"><span data-stu-id="c986f-125">Another scenario for `x:Shared="false"` is if you use a <xref:System.Windows.Freezable> resource for animation values but want to modify the resource on a per animation basis.</span></span>  
  
 <span data-ttu-id="c986f-126">W obsłudze ciągów `false` nie jest rozróżniana wielkość liter.</span><span class="sxs-lookup"><span data-stu-id="c986f-126">The string handling of `false` is not case sensitive.</span></span>  
  
 <span data-ttu-id="c986f-127">W WPF `x:Shared` są prawidłowe tylko w następujących warunkach:</span><span class="sxs-lookup"><span data-stu-id="c986f-127">In WPF, `x:Shared` is only valid under the following conditions:</span></span>
