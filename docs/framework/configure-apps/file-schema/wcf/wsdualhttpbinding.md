## <a name="remarks"></a><span data-ttu-id="a690d-200">Uwagi</span><span class="sxs-lookup"><span data-stu-id="a690d-200">Remarks</span></span>  
 <span data-ttu-id="a690d-201">`WSDualHttpBinding` Obsługę tych samych protokołów usług sieci Web jako `WSHttpBinding`, ale do użytku z programem kontrakty dwukierunkowe.</span><span class="sxs-lookup"><span data-stu-id="a690d-201">The `WSDualHttpBinding` provides the same support for Web Service protocols as the `WSHttpBinding`, but for use with duplex contracts.</span></span> <span data-ttu-id="a690d-202">`WSDualHttpBinding` obsługuje zabezpieczenia protokołu SOAP i tylko wymaga niezawodną obsługę komunikatów.</span><span class="sxs-lookup"><span data-stu-id="a690d-202">`WSDualHttpBinding` only supports SOAP security and requires reliable messaging.</span></span> <span data-ttu-id="a690d-203">To powiązanie wymaga, że klient ma publiczny identyfikator URI, który udostępnia punkt końcowy wywołania zwrotnego dla usługi.</span><span class="sxs-lookup"><span data-stu-id="a690d-203">This binding requires that the client has a public URI that provides a callback endpoint for the service.</span></span> <span data-ttu-id="a690d-204">To jest świadczona przez `clientBaseAddress` atrybutu.</span><span class="sxs-lookup"><span data-stu-id="a690d-204">This is provided by the `clientBaseAddress` attribute.</span></span> <span data-ttu-id="a690d-205">Podwójna powiązania udostępnia adres IP klienta do usługi.</span><span class="sxs-lookup"><span data-stu-id="a690d-205">A dual binding exposes the IP address of the client to the service.</span></span> <span data-ttu-id="a690d-206">Klienta należy użyć zabezpieczeń, aby upewnić się, że tylko nawiązuje połączenie z usługi ona relacje zaufania.</span><span class="sxs-lookup"><span data-stu-id="a690d-206">The client should use security to ensure that it only connects to services it trusts.</span></span>  
  
 <span data-ttu-id="a690d-207">To powiązanie może służyć do niezawodnie komunikować się za pośrednictwem pośredników SOAP jeden lub więcej.</span><span class="sxs-lookup"><span data-stu-id="a690d-207">This binding can be used to communicate reliably through one or more SOAP intermediaries.</span></span>  
  
 <span data-ttu-id="a690d-208">Domyślnie to powiązanie generuje stosu środowiska uruchomieniowego za pomocą WS-ReliableMessaging zapewniający niezawodność, WS-Security dla zabezpieczenia wiadomości i każde uwierzytelnienie, HTTP do dostarczania wiadomości i kodowanie komunikatu Text/XML.</span><span class="sxs-lookup"><span data-stu-id="a690d-208">By default, this binding generates a runtime stack with WS-ReliableMessaging for reliability, WS-Security for message security and authentication, HTTP for message delivery, and a Text/XML message encoding.</span></span>  
  
## <a name="example"></a><span data-ttu-id="a690d-209">Przykład</span><span class="sxs-lookup"><span data-stu-id="a690d-209">Example</span></span>  
  
```xml  
<configuration>
  <system.ServiceModel>
    <bindings>
      <wsDualHttpBinding>
        <binding closeTimeout="00:00:10"
                 openTimeout="00:00:20"
                 receiveTimeout="00:00:30"
                 sendTimeout="00:00:40"
                 bypassProxyOnLocal="false"
                 clientBaseAddress="http://localhost:8001/client/"
                 transactionFlow="true"
                 hostNameComparisonMode="WeakWildcard"
                 maxReceivedMessageSize="1000"
                 messageEncoding="Mtom"
                 proxyAddress="http://foo/bar"
                 textEncoding="utf-16"
                 useDefaultWebProxy="false">
          <reliableSession ordered="false"
                           inactivityTimeout="00:02:00" />
          <security mode="None">
            <message clientCredentialType="None"
                     negotiateServiceCredential="false"
                     algorithmSuite="Aes128" />
          </security>
        </binding>
      </wsDualHttpBinding>
    </bindings>
  </system.ServiceModel>
</configuration>
```
