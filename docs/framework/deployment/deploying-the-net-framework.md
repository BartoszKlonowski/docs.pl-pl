# <a name="deploying-the-net-framework"></a>Wdrażanie programu .NET Framework
Ta sekcja dokumentacji programu .NET Framework zawiera informacje dla deweloperów, którzy chcą zainstalować program .NET Framework ze swoimi aplikacjami, oraz administratorów, którzy chcą wdrożyć program .NET Framework w sieci. Omówiono również problemy z aktywacją i ponownym uruchomieniem związane z wdrażaniem oraz sposób monitorowania postępu instalacji programu .NET Framework.  
  
## <a name="in-this-section"></a>W tej sekcji  
 [Przewodnik wdrażania dla deweloperów](deployment-guide-for-developers.md)  
 W tym artykule wyjaśniono, jak deweloperzy mogą instalować program .NET Framework na komputerach swoich użytkowników za pomocą aplikacji.  
  
 [Przewodnik wdrażania dla administratorów](guide-for-administrators.md)  
 W tym artykule wyjaśniono, jak administrator systemu może wdrażać program .NET Framework i jego zależności systemowe w sieci przy użyciu programu Microsoft Endpoint Configuration Manager.  
  
 [Zmniejszenie liczby ponownych uruchomień systemu podczas instalowania programu .NET Framework 4.5](reducing-system-restarts.md)  
 W tym artykule opisano Menedżera ponownego uruchamiania, który zapobiega ponownemu uruchomieniu, gdy jest to możliwe, i wyjaśniono, jak aplikacje, które instalują program .NET Framework mogą z niego korzystać.  
  
 [Porady: pobieranie danych o postępie z Instalatora .NET Framework 4.5](how-to-get-progress-from-the-dotnet-installer.md)  
 W tym artykule opisano sposób dyskretnego uruchamiania i śledzenia procesu instalacji programu .NET Framework, podczas gdy wyświetlany jest własny widok postępu instalacji.  
  
 [Błędy inicjowania programu .NET Framework: zarządzanie wrażeniami użytkownika](initialization-errors-managing-the-user-experience.md)  
 Wyjaśniono, co się dzieje, gdy aplikacja .NET Framework wymaga wersji CLR, która jest nieprawidłowa lub nie zainstalowana na komputerze użytkownika, jak rozwiązać te błędy i jak kontrolować komunikat o błędzie wyświetlany użytkownikowi.  
  
 [Instrukcje: Debugowanie problemów aktywacji środowiska CLR](how-to-debug-clr-activation-issues.md)  
 W tym artykule wyjaśniono, jak można wyświetlać i debugować dzienniki aktywacji CLR, aby rozwiązać problemy, które mogą wystąpić podczas uruchamiania aplikacji z poprawną wersją środowiska CLR.
