is used to connect to the data source.</span></span>  
  
 <span data-ttu-id="68fdc-109">Podobnie, gdy `DataReader` zwraca wartość ze źródła danych, wynikowa wartość jest przechowywana w zmiennej lokalnej, która ma typ .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="68fdc-109">Likewise, when a `DataReader` returns a value from a data source, the resulting value is stored in a local variable that has a .NET Framework type.</span></span> <span data-ttu-id="68fdc-110">W przypadku `Fill` operacji `DataAdapter` i `Get` metod `DataReader`typ .NET Framework jest wywnioskowany na podstawie wartości zwróconej przez dostawcę danych .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="68fdc-110">For both the `Fill` operations of the `DataAdapter` and the `Get` methods of the `DataReader`, the .NET Framework type is inferred from the value returned from the .NET Framework data provider.</span></span>  
  
 <span data-ttu-id="68fdc-111">Zamiast polegać na wywnioskowanym typie danych, można użyć metod metody dostępu typu w `DataReader`, gdy jest znany konkretny typ zwracanej wartości.</span><span class="sxs-lookup"><span data-stu-id="68fdc-111">Instead of relying on the inferred data type, you can use the typed accessor methods of the `DataReader` when you know the specific type of the value being returned.</span></span> <span data-ttu-id="68fdc-112">Wpisywane metody dostępu zapewniają lepszą wydajność, zwracając wartość jako określony typ .NET Framework, co eliminuje konieczność stosowania dodatkowej konwersji typu.</span><span class="sxs-lookup"><span data-stu-id="68fdc-112">Typed accessor methods give you better performance by returning a value as a specific .NET Framework type, which eliminates the need for additional type conversion.</span></span>
