<span data-ttu-id="03603-103">SQL Server udostępnia funkcje do szyfrowania i odszyfrowywania danych przy użyciu certyfikatu, klucza asymetrycznego lub klucza symetrycznego.</span><span class="sxs-lookup"><span data-stu-id="03603-103">SQL Server provides functions to encrypt and decrypt data using a certificate, asymmetric key, or symmetric key.</span></span> <span data-ttu-id="03603-104">Wszystkie te elementy są zarządzane w wewnętrznym magazynie certyfikatów.</span><span class="sxs-lookup"><span data-stu-id="03603-104">It manages all of these in an internal certificate store.</span></span> <span data-ttu-id="03603-105">Magazyn używa hierarchii szyfrowania, która zabezpiecza certyfikaty i klucze na jednym poziomie z warstwą powyżej tej warstwy w hierarchii.</span><span class="sxs-lookup"><span data-stu-id="03603-105">The store uses an encryption hierarchy that secures certificates and keys at one level with the layer above it in the hierarchy.</span></span> <span data-ttu-id="03603-106">Ten obszar funkcji SQL Server nosi nazwę magazyn tajny.</span><span class="sxs-lookup"><span data-stu-id="03603-106">This feature area of SQL Server is called Secret Storage.</span></span>

## <a name="keys-and-algorithms"></a><span data-ttu-id="03603-110">Klucze i algorytmy</span><span class="sxs-lookup"><span data-stu-id="03603-110">Keys and Algorithms</span></span>  
 <span data-ttu-id="03603-111">SQL Server obsługuje kilka algorytmów szyfrowania kluczy symetrycznych, w tym DES, Triple DES, RC2, RC4, 128-bitowy RC4, DESX, 128-bit AES, 192-bit AES i 256-bitowy AES.</span><span class="sxs-lookup"><span data-stu-id="03603-111">SQL Server supports several symmetric key encryption algorithms, including DES, Triple DES, RC2, RC4, 128-bit RC4, DESX, 128-bit AES, 192-bit AES, and 256-bit AES.</span></span> <span data-ttu-id="03603-112">Algorytmy są implementowane przy użyciu interfejsu API kryptografii systemu Windows.</span><span class="sxs-lookup"><span data-stu-id="03603-112">The algorithms are implemented using the Windows Crypto API.</span></span>  
  
 <span data-ttu-id="03603-113">W zakresie połączenia z bazą danych SQL Server może obsługiwać wiele otwartych kluczy symetrycznych.</span><span class="sxs-lookup"><span data-stu-id="03603-113">Within the scope of a database connection, SQL Server can maintain multiple open symmetric keys.</span></span> <span data-ttu-id="03603-114">Otwarty klucz zostanie pobrany ze sklepu i jest dostępny do odszyfrowania danych.</span><span class="sxs-lookup"><span data-stu-id="03603-114">An open key is retrieved from the store and is available for decrypting data.</span></span> <span data-ttu-id="03603-115">Gdy dane są odszyfrowywane, nie ma potrzeby określania klucza symetrycznego do użycia.</span><span class="sxs-lookup"><span data-stu-id="03603-115">When a piece of data is decrypted, there is no need to specify the symmetric key to use.</span></span> <span data-ttu-id="03603-116">Każda zaszyfrowana wartość zawiera identyfikator klucza (identyfikator GUID klucza) klucza używany do szyfrowania.</span><span class="sxs-lookup"><span data-stu-id="03603-116">Each encrypted value contains the key identifier (key GUID) of the key used to encrypt it.</span></span> <span data-ttu-id="03603-117">Aparat dopasowuje strumień szyfrowanych bajtów do otwartego klucza symetrycznego, jeśli prawidłowy klucz został odszyfrowany i jest otwarty.</span><span class="sxs-lookup"><span data-stu-id="03603-117">The engine matches the encrypted byte stream to an open symmetric key, if the correct key has been decrypted and is open.</span></span> <span data-ttu-id="03603-118">Ten klucz jest następnie używany do odszyfrowywania i zwracania danych.</span><span class="sxs-lookup"><span data-stu-id="03603-118">This key is then used to perform decryption and return the data.</span></span> <span data-ttu-id="03603-119">Jeśli prawidłowy klucz nie jest otwarty, zwracana jest wartość NULL.</span><span class="sxs-lookup"><span data-stu-id="03603-119">If the correct key is not open, NULL is returned.</span></span>  
  
 <span data-ttu-id="03603-120">Aby zapoznać się z przykładem, który pokazuje, jak korzystać z szyfrowanych danych w bazie danych, zobacz artykuł [szyfrowanie kolumny danych](/sql/relational-databases/security/encryption/encrypt-a-column-of-data).</span><span class="sxs-lookup"><span data-stu-id="03603-120">For an example that shows how to work with encrypted data in a database, see [Encrypt a Column of Data](/sql/relational-databases/security/encryption/encrypt-a-column-of-data).</span></span>
