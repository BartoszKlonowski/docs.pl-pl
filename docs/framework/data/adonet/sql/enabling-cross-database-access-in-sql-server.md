# <a name="enabling-cross-database-access-in-sql-server"></a><span data-ttu-id="6dc18-102">Włączanie dostępu między bazami danych w programie SQL Server</span><span class="sxs-lookup"><span data-stu-id="6dc18-102">Enabling Cross-Database Access in SQL Server</span></span>
<span data-ttu-id="6dc18-103">Tworzenie łańcucha własności między bazami danych występuje, gdy procedura w jednej bazie danych jest zależna od obiektów w innej bazie danych.</span><span class="sxs-lookup"><span data-stu-id="6dc18-103">Cross-database ownership chaining occurs when a procedure in one database depends on objects in another database.</span></span> <span data-ttu-id="6dc18-104">Łańcucha własności między bazami danych działa w taki sam sposób jak łańcucha własności, w ramach pojedynczej bazy danych, z tą różnicą, że łańcucha własności nieprzerwany wymaga, że wszyscy właściciele obiektu są mapowane do tego samego konta logowania.</span><span class="sxs-lookup"><span data-stu-id="6dc18-104">A cross-database ownership chain works in the same way as ownership chaining within a single database, except that an unbroken ownership chain requires that all the object owners are mapped to the same login account.</span></span> <span data-ttu-id="6dc18-105">Jeśli obiekt źródłowy w źródłowej bazie danych i obiektów docelowych w docelowych baz danych są własnością tego samego konta logowania, programu SQL Server nie sprawdza uprawnienia do obiektów docelowych.</span><span class="sxs-lookup"><span data-stu-id="6dc18-105">If the source object in the source database and the target objects in the target databases are owned by the same login account, SQL Server does not check permissions on the target objects.</span></span>  
  
## <a name="off-by-default"></a><span data-ttu-id="6dc18-106">Funkcja domyślnie wyłączona</span><span class="sxs-lookup"><span data-stu-id="6dc18-106">Off By Default</span></span>

-   <span data-ttu-id="6dc18-109">Właściciele i członkowie bazy danych `db_ddladmin` lub `db_owners` ról bazy danych można tworzyć obiekty, które są własnością innych użytkowników.</span><span class="sxs-lookup"><span data-stu-id="6dc18-109">Database owners and members of the `db_ddladmin` or the `db_owners` database roles can create objects that are owned by other users.</span></span> <span data-ttu-id="6dc18-110">Te obiekty mogą potencjalnie kierować obiektów w innych bazach danych.</span><span class="sxs-lookup"><span data-stu-id="6dc18-110">These objects can potentially target objects in other databases.</span></span> <span data-ttu-id="6dc18-111">Oznacza to, że jeśli włączysz tworzenie łańcucha własności między bazami danych, należy pełnego zaufania, tych użytkowników, danych we wszystkich bazach danych.</span><span class="sxs-lookup"><span data-stu-id="6dc18-111">This means that if you enable cross-database ownership chaining, you must fully trust these users with data in all databases.</span></span>  
  
-   <span data-ttu-id="6dc18-112">Użytkownicy mający uprawnienie Tworzenie bazy danych można tworzyć nowe bazy danych i dołączanie istniejących baz danych.</span><span class="sxs-lookup"><span data-stu-id="6dc18-112">Users with CREATE DATABASE permission can create new databases and attach existing databases.</span></span> <span data-ttu-id="6dc18-113">Jeśli włączono tworzenie łańcucha własności między bazami danych tych użytkowników może uzyskać dostęp do obiektów w innych bazach danych, które mogą nie mieć uprawnień z bazy danych nowo utworzone lub dołączone, które tworzą.</span><span class="sxs-lookup"><span data-stu-id="6dc18-113">If cross-database ownership chaining is enabled, these users can access objects in other databases that they might not have privileges in from the newly created or attached databases that they create.</span></span>  
  
## <a name="enabling-cross-database-ownership-chaining"></a><span data-ttu-id="6dc18-114">Włączanie łańcucha własności między bazami danych</span><span class="sxs-lookup"><span data-stu-id="6dc18-114">Enabling Cross-database Ownership Chaining</span></span>  
 <span data-ttu-id="6dc18-115">Tworzenie łańcucha własności między bazami danych należy włączyć tylko w środowiskach, w którym można w pełni zaufane wysoce uprzywilejowanych użytkowników.</span><span class="sxs-lookup"><span data-stu-id="6dc18-115">Cross-database ownership chaining should only be enabled in environments where you can fully trust highly-privileged users.</span></span> <span data-ttu-id="6dc18-116">Można skonfigurować podczas instalacji dla wszystkich baz danych lub selektywnie dla określonych baz danych przy użyciu [!INCLUDE[tsql](../../../../../includes/tsql-md.md)] polecenia `sp_configure` i `ALTER DATABASE`.</span><span class="sxs-lookup"><span data-stu-id="6dc18-116">It can be configured during setup for all databases, or selectively for specific databases using the [!INCLUDE[tsql](../../../../../includes/tsql-md.md)] commands `sp_configure` and `ALTER DATABASE`.</span></span>  
  
 <span data-ttu-id="6dc18-117">Aby selektywnie skonfigurować tworzenie łańcucha własności między bazami danych, użyj `sp_configure` wyłączenia serwera.</span><span class="sxs-lookup"><span data-stu-id="6dc18-117">To selectively configure cross-database ownership chaining, use `sp_configure` to turn it off for the server.</span></span> <span data-ttu-id="6dc18-118">Następnie użyj polecenia ALTER DATABASE za pomocą zestawu ON DB_CHAINING do skonfigurowania własności między bazami danych, tworzenie łańcucha tylko bazy danych, które tego wymagają.</span><span class="sxs-lookup"><span data-stu-id="6dc18-118">Then use the ALTER DATABASE command with SET DB_CHAINING ON to configure cross-database ownership chaining for only the databases that require it.</span></span>  
  
 <span data-ttu-id="6dc18-119">Poniższy przykład powoduje włączenie łańcucha własności między bazami danych, dla wszystkich baz danych:</span><span class="sxs-lookup"><span data-stu-id="6dc18-119">The following sample turns on cross-database ownership chaining for all databases:</span></span>  
  
```  
EXECUTE sp_configure 'show advanced', 1;  
RECONFIGURE;  
EXECUTE sp_configure 'cross db ownership chaining', 1;  
RECONFIGURE;  
```  
  
 <span data-ttu-id="6dc18-120">Poniższy przykład powoduje włączenie łańcucha własności między bazami danych, do konkretnych baz danych:</span><span class="sxs-lookup"><span data-stu-id="6dc18-120">The following sample turns on cross-database ownership chaining for specific databases:</span></span>  
  
```  
ALTER DATABASE Database1 SET DB_CHAINING ON;  
ALTER DATABASE Database2 SET DB_CHAINING ON;  
```  
  
### <a name="dynamic-sql"></a><span data-ttu-id="6dc18-121">Dynamiczny język SQL</span><span class="sxs-lookup"><span data-stu-id="6dc18-121">Dynamic SQL</span></span>  
 <span data-ttu-id="6dc18-122">Tworzenie łańcucha własności między bazami danych nie działa w przypadku których dynamicznie utworzoną instrukcje SQL są wykonywane, chyba że ten sam użytkownik istnieje w obu bazach danych.</span><span class="sxs-lookup"><span data-stu-id="6dc18-122">Cross-database ownership chaining does not work in cases where dynamically created SQL statements are executed unless the same user exists in both databases.</span></span> <span data-ttu-id="6dc18-123">Można obejść to w programie SQL Server, tworząc procedury przechowywanej, która uzyskuje dostęp do danych w innej bazie danych i podpisywania procedury przy użyciu certyfikatu, który znajduje się w obu bazach danych.</span><span class="sxs-lookup"><span data-stu-id="6dc18-123">You can work around this in SQL Server by creating a stored procedure that accesses data in another database and signing the procedure with a certificate that exists in both databases.</span></span> <span data-ttu-id="6dc18-124">To zapewnia użytkownikom dostęp do zasobów bazy danych używane przez tę procedurę, bez nadawania im dostępu do bazy danych lub uprawnienia.</span><span class="sxs-lookup"><span data-stu-id="6dc18-124">This gives users access to the database resources used by the procedure without granting them database access or permissions.</span></span>
