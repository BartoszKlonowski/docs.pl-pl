## <a name="stored-procedure-benefits"></a><span data-ttu-id="f2d21-106">Procedura składowana korzyści</span><span class="sxs-lookup"><span data-stu-id="f2d21-106">Stored Procedure Benefits</span></span>  
 <span data-ttu-id="f2d21-107">Procedury składowane zapewnia następujące korzyści:</span><span class="sxs-lookup"><span data-stu-id="f2d21-107">Stored procedures have the following benefits:</span></span>  
  
-   <span data-ttu-id="f2d21-108">Można hermetyzować reguł biznesowych i logiki danych, dzięki czemu użytkownicy mogą uzyskiwać dostęp do danych i obiektów tylko w sposób, który ma deweloperów i administratorów baz danych.</span><span class="sxs-lookup"><span data-stu-id="f2d21-108">Data logic and business rules can be encapsulated so that users can access data and objects only in ways that developers and database administrators intend.</span></span>

-   <span data-ttu-id="f2d21-116">Procedury składowane można zapisywane jeden raz i używane przez wiele aplikacji.</span><span class="sxs-lookup"><span data-stu-id="f2d21-116">Stored procedures can be written once, and accessed by many applications.</span></span>  
  
-   <span data-ttu-id="f2d21-117">Aplikacje klienckie nie trzeba nic wiedzieć o podstawowej struktury danych.</span><span class="sxs-lookup"><span data-stu-id="f2d21-117">Client applications do not need to know anything about the underlying data structures.</span></span> <span data-ttu-id="f2d21-118">Procedura składowana kod, można zmienić bez konieczności wprowadzania zmian w aplikacjach klienckich, tak długo, jak zmiany nie wpływają na listy parametrów lub zwracane typy danych.</span><span class="sxs-lookup"><span data-stu-id="f2d21-118">Stored procedure code can be changed without requiring changes in client applications as long as the changes do not affect parameter lists or returned data types.</span></span>  
  
-   <span data-ttu-id="f2d21-119">Procedury składowane można zmniejszyć ruch sieciowy, łącząc wiele operacji do wywołania jednej procedury.</span><span class="sxs-lookup"><span data-stu-id="f2d21-119">Stored procedures can reduce network traffic by combining multiple operations into one procedure call.</span></span>  
  
## <a name="stored-procedure-execution"></a><span data-ttu-id="f2d21-120">Wykonywanie procedury składowanej</span><span class="sxs-lookup"><span data-stu-id="f2d21-120">Stored Procedure Execution</span></span>  
 <span data-ttu-id="f2d21-121">Przechowywane procedury Wykorzystaj własności łańcucha zapewnienie dostępu do danych, dzięki czemu użytkownicy nie muszą mieć jawne uprawnienia do dostępu do obiektów bazy danych.</span><span class="sxs-lookup"><span data-stu-id="f2d21-121">Stored procedures take advantage of ownership chaining to provide access to data so that users do not need to have explicit permission to access database objects.</span></span> <span data-ttu-id="f2d21-122">Łańcucha własności istnieje, gdy obiektów mających dostęp do sekwencyjnego sobą należą do tego samego użytkownika.</span><span class="sxs-lookup"><span data-stu-id="f2d21-122">An ownership chain exists when objects that access each other sequentially are owned by the same user.</span></span> <span data-ttu-id="f2d21-123">Na przykład wywołać procedurę składowaną innych procedur składowanych lub procedury składowanej mogą uzyskiwać dostęp do wielu tabel.</span><span class="sxs-lookup"><span data-stu-id="f2d21-123">For example, a stored procedure can call other stored procedures, or a stored procedure can access multiple tables.</span></span> <span data-ttu-id="f2d21-124">Jeśli wszystkie obiekty w łańcuchu wykonywania mają tego samego właściciela, a następnie programu SQL Server sprawdza tylko uprawnień wykonywanie do obiektu wywołującego, nie uprawnienia obiektu wywołującego od innych obiektów.</span><span class="sxs-lookup"><span data-stu-id="f2d21-124">If all objects in the chain of execution have the same owner, then SQL Server only checks the EXECUTE permission for the caller, not the caller's permissions on other objects.</span></span> <span data-ttu-id="f2d21-125">W związku z tym należy udzielić tylko uprawnienia wykonywania na procedurach przechowywanych; można odwołać lub odmówić wszystkie uprawnienia wobec tabel podstawowych.</span><span class="sxs-lookup"><span data-stu-id="f2d21-125">Therefore you need to grant only EXECUTE permissions on stored procedures; you can revoke or deny all permissions on the underlying tables.</span></span>  
  
## <a name="best-practices"></a><span data-ttu-id="f2d21-126">Najlepsze praktyki</span><span class="sxs-lookup"><span data-stu-id="f2d21-126">Best Practices</span></span>  
 <span data-ttu-id="f2d21-127">Po prostu pisania procedur składowanych nie jest wystarczająco dużo, aby odpowiednio zabezpieczenia aplikacji.</span><span class="sxs-lookup"><span data-stu-id="f2d21-127">Simply writing stored procedures isn't enough to adequately secure your application.</span></span> <span data-ttu-id="f2d21-128">Należy również rozważyć następujące potencjalnych luk w zabezpieczeniach.</span><span class="sxs-lookup"><span data-stu-id="f2d21-128">You should also consider the following potential security holes.</span></span>  
  
-   <span data-ttu-id="f2d21-129">Udziel uprawnień wykonywanie procedur składowanych do ról bazy danych, które chcesz mieć możliwość dostępu do danych.</span><span class="sxs-lookup"><span data-stu-id="f2d21-129">Grant EXECUTE permissions on the stored procedures for database roles you want to be able to access the data.</span></span>  
  
-   <span data-ttu-id="f2d21-130">Odwoływanie lub Odrzuć wszystkie uprawnienia w tabelach dla wszystkich ról i użytkowników w bazie danych, w tym `public` roli.</span><span class="sxs-lookup"><span data-stu-id="f2d21-130">Revoke or deny all permissions to the underlying tables for all roles and users in the database, including the `public` role.</span></span> <span data-ttu-id="f2d21-131">Wszyscy użytkownicy dziedziczą uprawnienia z publicznej.</span><span class="sxs-lookup"><span data-stu-id="f2d21-131">All users inherit permissions from public.</span></span> <span data-ttu-id="f2d21-132">W związku z tym odmawia uprawnienia do `public` oznacza, że tylko właściciele i `sysadmin` członkowie mają dostęp; wszyscy inni użytkownicy będą mogli dziedziczą uprawnienia z członkostwa w innych ról.</span><span class="sxs-lookup"><span data-stu-id="f2d21-132">Therefore denying permissions to `public` means that only owners and `sysadmin` members have access; all other users will be unable to inherit permissions from membership in other roles.</span></span>  
  
-   <span data-ttu-id="f2d21-133">Nie należy dodawać użytkowników lub ról, aby `sysadmin` lub `db_owner` ról.</span><span class="sxs-lookup"><span data-stu-id="f2d21-133">Do not add users or roles to the `sysadmin` or `db_owner` roles.</span></span> <span data-ttu-id="f2d21-134">Administratorzy systemu i właścicieli bazy danych można uzyskać dostęp do wszystkich obiektów bazy danych.</span><span class="sxs-lookup"><span data-stu-id="f2d21-134">System administrators and database owners can access all database objects.</span></span>  
  
-   <span data-ttu-id="f2d21-135">Wyłącz `guest` konta.</span><span class="sxs-lookup"><span data-stu-id="f2d21-135">Disable the `guest` account.</span></span> <span data-ttu-id="f2d21-136">Uniemożliwi to użytkownicy anonimowi połączenie z bazą danych.</span><span class="sxs-lookup"><span data-stu-id="f2d21-136">This will prevent anonymous users from connecting to the database.</span></span> <span data-ttu-id="f2d21-137">Konto gościa jest domyślnie wyłączona, w nowych baz danych.</span><span class="sxs-lookup"><span data-stu-id="f2d21-137">The guest account is disabled by default in new databases.</span></span>  
  
-   <span data-ttu-id="f2d21-138">Implementowanie błędu dziennika i obsługi błędów.</span><span class="sxs-lookup"><span data-stu-id="f2d21-138">Implement error handling and log errors.</span></span>  
  
-   <span data-ttu-id="f2d21-139">Utwórz sparametryzowanej procedury składowane, które sprawdzają poprawność wszystkich danych wejściowych użytkownika.</span><span class="sxs-lookup"><span data-stu-id="f2d21-139">Create parameterized stored procedures that validate all user input.</span></span> <span data-ttu-id="f2d21-140">Wszystkie dane wejściowe użytkownika należy traktować jako niezaufane.</span><span class="sxs-lookup"><span data-stu-id="f2d21-140">Treat all user input as untrusted.</span></span>  
  
-   <span data-ttu-id="f2d21-141">Należy unikać dynamiczny język SQL, chyba że jest to absolutnie konieczne.</span><span class="sxs-lookup"><span data-stu-id="f2d21-141">Avoid dynamic SQL unless absolutely necessary.</span></span> <span data-ttu-id="f2d21-142">Funkcja QUOTENAME() języka Transact-SQL do ograniczania wartości ciągu i wprowadzić dowolne wystąpienie ogranicznika w ciągu wejściowym.</span><span class="sxs-lookup"><span data-stu-id="f2d21-142">Use the Transact-SQL QUOTENAME() function to delimit a string value and escape any occurrence of the delimiter in the input string.</span></span>
