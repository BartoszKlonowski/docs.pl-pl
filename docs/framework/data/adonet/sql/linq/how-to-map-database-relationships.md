- <span data-ttu-id="cb6b2-114">Jeden do jednego: Reprezentuje tego rodzaju relacji, umieszczając <xref:System.Data.Linq.EntitySet%601> po obu stronach.</span><span class="sxs-lookup"><span data-stu-id="cb6b2-114">One-to-one: Represent this kind of relationship by including <xref:System.Data.Linq.EntitySet%601> on both sides.</span></span>  
  
     <span data-ttu-id="cb6b2-115">Na przykład, rozważmy `Customer` - `SecurityCode` relacji, utworzony kod zabezpieczeń klienta nie zostanie znaleziony w `Customer` tabeli i może mieć dostęp tylko upoważnione osoby.</span><span class="sxs-lookup"><span data-stu-id="cb6b2-115">For example, consider a `Customer`-`SecurityCode` relationship, created so that the customer's security code will not be found in the `Customer` table and can be accessed only by authorized persons.</span></span>

     <span data-ttu-id="cb6b2-117">Na przykład, rozważmy `Employee` - `Project` relacji wiele do wielu utworzone za pomocą tabela łączy `EmployeeProject`.</span><span class="sxs-lookup"><span data-stu-id="cb6b2-117">For example, consider an `Employee`-`Project` many-to-many relationship formed by using link table `EmployeeProject`.</span></span> [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] <span data-ttu-id="cb6b2-118">wymaga, że takiej relacji jest modelowane przy użyciu trzech klas: `Employee`, `Project`, i `EmployeeProject`.</span><span class="sxs-lookup"><span data-stu-id="cb6b2-118">requires that such a relationship be modeled by using three classes: `Employee`, `Project`, and `EmployeeProject`.</span></span> <span data-ttu-id="cb6b2-119">W tym przypadku zmiana relacji między `Employee` i `Project` może okazać się wymagają aktualizacji klucza podstawowego `EmployeeProject`.</span><span class="sxs-lookup"><span data-stu-id="cb6b2-119">In this case, changing the relationship between an `Employee` and a `Project` can appear to require an update of the primary key `EmployeeProject`.</span></span> <span data-ttu-id="cb6b2-120">Jednak ta sytuacja jest najlepiej formę usunięcie istniejącej `EmployeeProject` i utworzenie nowego `EmployeeProject`.</span><span class="sxs-lookup"><span data-stu-id="cb6b2-120">However, this situation is best modeled as deleting an existing `EmployeeProject` and the creating a new `EmployeeProject`.</span></span>  
  
    > [!NOTE]
    >  <span data-ttu-id="cb6b2-121">Relacji w relacyjnych baz danych zwykle są modelowane jako wartości klucza obcego, które odwołują się do kluczy podstawowych w innych tabelach.</span><span class="sxs-lookup"><span data-stu-id="cb6b2-121">Relationships in relational databases are typically modeled as foreign key values that refer to primary keys in other tables.</span></span> <span data-ttu-id="cb6b2-122">Przechodzenie między nimi przy użyciu relacyjną skojarzysz jest jawnie dwie tabele *sprzężenia* operacji.</span><span class="sxs-lookup"><span data-stu-id="cb6b2-122">To navigate between them you explicitly associate the two tables by using a relational *join* operation.</span></span>  
    >   
    >  <span data-ttu-id="cb6b2-123">Obiekty w [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)], z drugiej strony, zapoznaj się ze sobą za pomocą odwołania do właściwości lub kolekcji odwołań, które można nawigować przy użyciu *kropka* notacji.</span><span class="sxs-lookup"><span data-stu-id="cb6b2-123">Objects in [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)], on the other hand, refer to each other by using property references or collections of references that you navigate by using *dot* notation.</span></span>
