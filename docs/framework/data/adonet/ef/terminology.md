# <a name="entity-framework-terminology"></a>Terminologia programu Entity Framework
W tym temacie definiuje warunki często w [!INCLUDE[adonet_ef](../../../../../includes/adonet-ef-md.md)] dokumentacji. Podano linki do powiązanych tematów, w którym dostępne są dodatkowe informacje.  
  
|Termin|Definicja|  
|----------|----------------|  
|skojarzenie|Definicja relacji między typami encji.<br /><br /> Aby uzyskać więcej informacji, zobacz [skojarzenia elementu (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#association-element-csdl) i [typ skojarzenia](../../../../../docs/framework/data/adonet/association-type.md).|  
|zestaw skojarzeń|Kontener logiczny dla wystąpień skojarzeń tego samego typu.<br /><br /> Aby uzyskać więcej informacji, zobacz [elementu AssociationSet (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#associationset-element-csdl) i [zestaw skojarzeń](../../../../../docs/framework/data/adonet/association-set.md).|  
|Najpierw kod|Uruchamianie 4.1 Framework jednostki można utworzyć modelu programowo przy użyciu rozwiązania deweloperskiego Code First. Istnieją dwa różne scenariusze dla rozwiązania deweloperskiego Code First. W obu przypadkach deweloper definiuje model zarządzanego, tworząc definicje klas .NET Framework, a następnie opcjonalnie określa dodatkowego mapowania lub konfiguracji przy użyciu adnotacji danych lub interfejsu API fluent.<br /><br /> Należy pamiętać, że rozwiązania deweloperskiego Code First jest częścią [Entity Framework 5.0](https://go.microsoft.com/fwlink/?LinkId=234900). Entity Framework 5.0 nie jest częścią programu .NET Framework, ale jest oparta na programie .NET Framework 4.5. Entity Framework 5.0 jest dostępna jako ["Entity Framework"](https://go.microsoft.com/fwlink/?LinkID=215714)[NuGet](https://go.microsoft.com/fwlink/?LinkId=232488) pakietu. Aby uzyskać więcej informacji, zobacz [Entity Framework w wersji oraz zarządzanie ich wersjami](https://go.microsoft.com/fwlink/?LinkId=234899).|  
|drzewo poleceń|Wspólny, programowe reprezentacja wszystkich [!INCLUDE[adonet_ef](../../../../../includes/adonet-ef-md.md)] zapytań, które składają się z co najmniej jednego wyrażenia.<br /><br /> Aby uzyskać więcej informacji, zobacz [Omówienie programu Entity Framework](../../../../../docs/framework/data/adonet/ef/overview.md).|  
|typ złożony|A [!INCLUDE[dnprdnshort](../../../../../includes/dnprdnshort-md.md)] klasę, która reprezentuje właściwości złożonej, zgodnie z definicją w modelu koncepcyjnym. Typy złożone włączyć właściwości skalarne, które mają być organizowane w ramach jednostek. Złożone obiekty są wystąpieniami typów złożonych. Aby uzyskać więcej informacji, zobacz [Element ComplexType (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#complextype-element-csdl) i [typu złożonego](../../../../../docs/framework/data/adonet/complex-type.md).|  
|ComplexType|Specyfikacja typu danych, który reprezentuje właściwość nieskalarnego typu jednostki, która nie ma właściwości klucza.<br /><br /> Aby uzyskać więcej informacji, zobacz [Element ComplexType (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#complextype-element-csdl) i [typu złożonego](../../../../../docs/framework/data/adonet/complex-type.md).|  
|model koncepcyjny|Streszczenie specyfikacji dla typów jednostek, typy złożone, skojarzenia, kontenerów encji, zestawów encji i skojarzenia ustawia się w domenie aplikacji w [!INCLUDE[adonet_ef](../../../../../includes/adonet-ef-md.md)]. Model koncepcyjny jest zdefiniowany w CSDL w pliku .csdl.<br /><br /> Aby uzyskać więcej informacji, zobacz [modelowanie i mapowanie](../../../../../docs/framework/data/adonet/ef/modeling-and-mapping.md).|  
|plik .csdl|Plik XML, który zawiera model koncepcyjny, wyrażone w CSDL.|  
|Conceptual Schema Definition Language (CSDL)|Model koncepcyjny zestawów opartych na języku XML język, który jest używany do definiowania typów jednostek, skojarzeń, kontenerów encji, zestawów encji i skojarzeń.<br /><br /> Aby uzyskać więcej informacji, zobacz [Specyfikacja CSDL](../../../../../docs/framework/data/adonet/ef/language-reference/csdl-specification.md).|  
|Kontener|Logiczne grupowanie encji i skojarzeń ustawia.<br /><br /> Aby uzyskać więcej informacji, zobacz [Element EntityContainer (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitycontainer-element-csdl) i [kontener jednostek](../../../../../docs/framework/data/adonet/entity-container.md).|  
|współbieżność|Proces, który umożliwia wielu użytkownikom dostępu i zmiany udostępnionych danych w tym samym czasie. Domyślnie [!INCLUDE[adonet_ef](../../../../../includes/adonet-ef-md.md)] wdraża model optymistycznej współbieżności.|  
|kierunek|Odnosi się do asymetryczne charakter niektóre skojarzenia. Kierunek jest określony za pomocą `FromRole` i `ToRole` atrybuty `NavigationProperty` lub `ReferentialConstraint` elementu w schemacie.<br /><br /> Aby uzyskać więcej informacji, zobacz [element NavigationProperty — Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#navigationproperty-element-csdl) i [właściwość nawigacji](../../../../../docs/framework/data/adonet/navigation-property.md).|  
|Wczesne ładowanie|Proces ładowania określony zbiór powiązanych obiektów wraz z obiektami, które były wyraźnie wymagane w kwerendzie.|  
|Plik edmx|Plik XML, który zawiera model koncepcyjny (w CSDL), modelu magazynu (w SSDL) i mapowania między nimi (w pliku MSL). Plik edmx jest tworzony przez [!INCLUDE[adonet_edm](../../../../../includes/adonet-edm-md.md)] narzędzia. Aby uzyskać więcej informacji, zobacz [Omówienie pliku edmx](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/cc982042(v=vs.100)).|  
|end|Podmiot uczestniczący w skojarzeniu.<br /><br /> Aby uzyskać więcej informacji, zobacz [elementu End (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#end-element-csdl) i [end skojarzenia](../../../../../docs/framework/data/adonet/association-end.md).|  
|jednostka|Koncepcja w domenie aplikacji, w którym zdefiniowano typ danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [Element EntityType (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitytype-element-csdl) i [typu jednostki](../../../../../docs/framework/data/adonet/entity-type.md).|  
|EntityClient|Magazyn niezależne od [!INCLUDE[vstecado](../../../../../includes/vstecado-md.md)] dostawcy danych, który zawiera klasy, takie jak `EntityConnection`, `EntityCommand`, i `EntityDataReader`. W programach [!INCLUDE[esql](../../../../../includes/esql-md.md)] połączyć z usługą storage określone [!INCLUDE[vstecado](../../../../../includes/vstecado-md.md)] dostawcy danych, takich jak `SqlClient`.<br /><br /> Aby uzyskać więcej informacji, zobacz [dostawca EntityClient dla programu Entity Framework](../../../../../docs/framework/data/adonet/ef/entityclient-provider-for-the-entity-framework.md).|  
|kontener jednostek|Określa, zestawów encji i zestawów skojarzeń, które będą realizowane w określonej przestrzeni nazw.<br /><br /> Aby uzyskać więcej informacji, zobacz [Element EntityContainer (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitycontainer-element-csdl) i [kontener jednostek](../../../../../docs/framework/data/adonet/entity-container.md).|  
|Entity Data Model (EDM)|Zestaw pojęcia, które opisują struktury danych, jako jednostek i relacji, niezależnie od tego, w postaci przechowywane.<br /><br /> Aby uzyskać więcej informacji, zobacz [modelu Entity Data Model](../../../../../docs/framework/data/adonet/entity-data-model.md).|  
|Entity Framework|Zestaw technologii, który obsługuje opracowywanie aplikacji zorientowanych na dane, umożliwiając deweloperom pracę z modelami koncepcyjnymi, które są mapowane do schematów logicznych w źródłach danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [Omówienie programu Entity Framework](../../../../../docs/framework/data/adonet/ef/overview.md).|  
|zestaw jednostek|Kontener logiczny dla jednostki danego typu i jego podtypy. Zestawy jednostek są mapowane do tabel w bazie danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [elementu obiektu EntitySet (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entityset-element-csdl) i [zestaw jednostek](../../../../../docs/framework/data/adonet/entity-set.md).|  
|Entity SQL|Dialekt niezależny od magazynu programu SQL Server, współpracuje bezpośrednio ze schematami koncepcyjnymi encji i który obsługuje model koncepcyjny pojęć, takich jak dziedziczenie i relacje.<br /><br /> Aby uzyskać więcej informacji, zobacz [jednostki języka SQL](../../../../../docs/framework/data/adonet/ef/language-reference/entity-sql-language.md).|  
|typ jednostki|A [!INCLUDE[dnprdnshort](../../../../../includes/dnprdnshort-md.md)] klasa, która reprezentuje jednostkę, jak jest zdefiniowany w modelu koncepcyjnym. Typy jednostek może mieć właściwości scalar, złożone i nawigacji. Obiekty są wystąpieniami typów jednostek. Aby uzyskać więcej informacji, zobacz [Praca z obiektami](../../../../../docs/framework/data/adonet/ef/working-with-objects.md).|  
|Typ entityType|Specyfikacja typu danych, który zawiera klucz i nazwany zestaw właściwości i stanowi element najwyższego poziomu w modelu koncepcyjnym i modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [Element EntityType (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitytype-element-csdl) i [typu jednostki](../../../../../docs/framework/data/adonet/entity-type.md).|  
|jawne ładowanie|Gdy obiekty są zwracane przez zapytanie, obiekty powiązane nie są ładowane w tym samym czasie. Domyślnie, ich nie są ładowane aż do wyraźnego żądania przy użyciu `Load` metoda we właściwości nawigacji.|  
|Skojarzenie klucza obcego|Skojarzenia między jednostkami, które odbywa się za pośrednictwem właściwości klucza obcego.|  
|Identyfikowanie relacji|Relacja, w którym klucz podstawowy jednostki głównej jest częścią klucza podstawowego jednostki zależne. W tego rodzaju relacji jednostki zależne nie może istnieć bez jednostki głównej.|  
|Niezależnie od skojarzenia|Skojarzenia między jednostkami, które są reprezentowane i śledzone przez tworzenie niezależnych obiektów.|  
|klawisz|Atrybut typu jednostki, która określa, które właściwości lub zestaw właściwości używany do identyfikowania unikatowe wystąpień typu jednostki. Reprezentowane w warstwie obiektów przez <xref:System.Data.EntityKey> klasy.<br /><br /> Aby uzyskać więcej informacji, zobacz [klucz elementu (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#key-element-csdl) i [klucz jednostki](../../../../../docs/framework/data/adonet/entity-key.md).|  
|Ładowanie z opóźnieniem|Gdy obiekty są zwracane przez zapytanie, obiekty powiązane nie są ładowane w tym samym czasie. Zamiast tego są one ładowane automatycznie podczas uzyskiwania dostępu do właściwości nawigacji.|  
|[!INCLUDE[linq_entities](../../../../../includes/linq-entities-md.md)]|Składnia zapytania definiuje jego zestaw operatorów, które umożliwiają przechodzenie, filtrowanie i operacje być wyrażone w sposób bezpośredni i deklaracyjny w Visual C# i Visual Basic.<br /><br /> Aby uzyskać więcej informacji, zobacz [składnik LINQ to Entities](../../../../../docs/framework/data/adonet/ef/language-reference/linq-to-entities.md).|  
|mapowanie|Specyfikacja korelacji między elementów w modelu koncepcyjnego i elementów w modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [Specyfikacja MSL](../../../../../docs/framework/data/adonet/ef/language-reference/msl-specification.md).|  
|pliku MSL albo identyfikatorem|Plik XML, który zawiera mapowania między modelu koncepcyjnego i modelem w pamięci masowej w pliku MSL.|  
|mapping specification language (MSL)|Język oparty na formacie XML, który jest używany do mapowania elementy zdefiniowane w model koncepcyjny do elementów w modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [Specyfikacja MSL](../../../../../docs/framework/data/adonet/ef/language-reference/msl-specification.md).|  
|Modyfikowanie funkcji|Procedury składowane, które są używane do wstawiania, aktualizacji i usuwania danych, który znajduje się w źródle danych. Funkcje te są używane zamiast [!INCLUDE[adonet_ef](../../../../../includes/adonet-ef-md.md)] generowane poleceń. Modyfikowanie funkcji są definiowane przez `Function` elementu w modelu magazynu. [ModificationFunctionMapping](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/cc716778(v=vs.100)) element mapuje te funkcje modyfikacji, aby Wstawianie, aktualizowanie i usuwanie operacji wykonywanych względem jednostek, które są zdefiniowane w modelu koncepcyjnym.|  
|Liczebność|Liczba jednostek, które nie mogą występować na każdej stronie relacji, zgodnie z definicją skojarzenia. Nazywane również kardynalność.<br /><br /> Aby uzyskać więcej informacji, zobacz [elementu End (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#end-element-csdl) i [end skojarzenia](../../../../../docs/framework/data/adonet/association-end.md).|  
|wiele zestawów jednostek według typu|Ustaw możliwości dla typu jednostki zdefiniowanych w więcej niż jedną jednostkę.<br /><br /> Aby uzyskać więcej informacji, zobacz [elementu obiektu EntitySet (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entityset-element-csdl) i [jak: Zdefiniowanie modelu z wieloma zestawami jednostki według typu](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb738537(v=vs.100)).|  
|właściwość nawigacji|Właściwość typu jednostki, która reprezentuje relację do innego typu jednostki, zgodnie z definicją skojarzenia. Właściwości nawigacji są używane do zwrócenia powiązanych obiektów jako <xref:System.Data.Objects.DataClasses.EntityCollection%601> lub <xref:System.Data.Objects.DataClasses.EntityReference%601>, w zależności od liczebność na drugim końcu skojarzenia.<br /><br /> Aby uzyskać więcej informacji, zobacz [element NavigationProperty — Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#navigationproperty-element-csdl) i [właściwość nawigacji](../../../../../docs/framework/data/adonet/navigation-property.md).|  
|Ścieżka kwerendy|Ciąg reprezentujący ścieżki określający, którego powiązane obiekty do zwrócenia, jeśli obiekt zapytanie jest wykonywane. Ścieżka kwerendy jest zdefiniowany przez wywołanie metody <xref:System.Data.Objects.ObjectQuery%601.Include%2A> metody <xref:System.Data.Objects.ObjectQuery%601>.<br /><br /> Aby uzyskać więcej informacji, zobacz [ładowanie powiązanych obiektów](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb896272(v=vs.100)).|  
|Obiekt kontekstu|Reprezentuje kontener jednostek zdefiniowane w modelu koncepcyjnym. Zawiera połączenia do bazowego źródła danych i zapewnia usługi, takie jak rozwiązania change tracking and rozwiązanie tożsamości. Ext CD obiekt jest reprezentowany przez wystąpienie <xref:System.Data.Objects.ObjectContext> lub `DbContext` klasy.<br /><br /> `DbContext` jest częścią [Entity Framework 5.0](https://go.microsoft.com/fwlink/?LinkId=234900). Entity Framework 5.0 nie jest częścią programu .NET Framework, ale jest oparta na programie .NET Framework 4.5. Entity Framework 5.0 jest dostępna jako ["Entity Framework"](https://go.microsoft.com/fwlink/?LinkID=215714)[NuGet](https://go.microsoft.com/fwlink/?LinkId=232488) pakietu. Aby uzyskać więcej informacji, zobacz [Entity Framework w wersji oraz zarządzanie ich wersjami](https://go.microsoft.com/fwlink/?LinkId=234899).|  
|warstwy obiektu|Typy jednostek i definicji kontekstu obiektów, które są używane przez program Entity Framework.|  
|Zapytanie obiektów|Zapytanie wykonana w obrębie kontekstu obiektów względem modelu koncepcyjnego, która zwraca dane jako obiekty.<br /><br /> Aby uzyskać więcej informacji, zobacz [zapytań dotyczących obiektów](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb896241(v=vs.100)).|  
|mapowania obiektowo relacyjny|Technika przekształcaniu danych pochodzących z relacyjnej bazy danych do typów danych, które mogą być używane w aplikacji zorientowanych na obiekt.<br /><br /> [!INCLUDE[adonet_ef](../../../../../includes/adonet-ef-md.md)] Udostępnia usługi mapowania obiektowo relacyjny przez mapowanie danych relacyjnych, zgodnie z definicją w modelu magazynu do typów danych, zgodnie z definicją w modelu koncepcyjnym.<br /><br /> Aby uzyskać więcej informacji, zobacz [modelowanie i mapowanie](../../../../../docs/framework/data/adonet/ef/modeling-and-mapping.md).|  
|Usługi obiektów|Usługi świadczone przez [!INCLUDE[adonet_ef](../../../../../includes/adonet-ef-md.md)] , dzięki którym kod aplikacji może działać w jednostki, takie jak [!INCLUDE[dnprdnshort](../../../../../includes/dnprdnshort-md.md)] obiektów.|  
|obiekt zakresu trwałości|Obiekt, który nie zawiera wszelka logika, która jest powiązana z magazynu danych. Znany także jako jednostki POCO.|  
|POCO|Zwykły stary obiekt CLR. Obiekt, który dziedziczył z innej klasy lub nie implementuje interfejsu.|  
|Jednostki POCO|Jednostki w usłudze [!INCLUDE[adonet_ef](../../../../../includes/adonet-ef-md.md)] który nie dziedziczy <xref:System.Data.Objects.DataClasses.EntityObject> lub <xref:System.Data.Objects.DataClasses.ComplexObject> implementuje [!INCLUDE[adonet_ef](../../../../../includes/adonet-ef-md.md)] interfejsów. Często jednostki POCO czy istniejące obiektów domeny, które używają w [!INCLUDE[adonet_ef](../../../../../includes/adonet-ef-md.md)] aplikacji. Te jednostki obsługuje nieznajomości trwałości. Aby uzyskać więcej informacji, zobacz [Praca z jednostkami obiektów POCO](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/dd456853(v=vs.100)).|  
|proxy — obiekt|Obiekt, który pochodzi od klasy POCO i jest generowany przez [!INCLUDE[adonet_ef](../../../../../includes/adonet-ef-md.md)] do obsługi śledzenia zmian i ładowania z opóźnieniem. Aby uzyskać więcej informacji, zobacz [wymagania dotyczące tworzenia obiektów POCO proxy](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/dd468057(v=vs.100)).|  
|ograniczenie więzów|Ograniczenie, które jest zdefiniowane w modelu koncepcyjnym, która wskazuje, że określona jednostka ma relację elementu zależnego do innej jednostki. To ograniczenie oznacza, że wystąpienie jednostki zależne nie może istnieć bez odpowiedniego wystąpienia jednostki zasady<br /><br /> Aby uzyskać więcej informacji, zobacz [elemencie ReferentialConstraint (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#referentialconstraint-element-csdl) i [ograniczenia integralności referencyjnej](../../../../../docs/framework/data/adonet/referential-integrity-constraint.md).|  
|Relacja|Połączenie logiczne między jednostkami.|  
|rola|Nazwa nadana każdego `End` skojarzenia wyjaśnienie semantyki relacji.<br /><br /> Aby uzyskać więcej informacji, zobacz [elementu End (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#end-element-csdl) i [end skojarzenia](../../../../../docs/framework/data/adonet/association-end.md).|  
|Właściwość skalarną|Właściwość obiektu, który jest mapowany do jednego pola w modelu magazynu.|  
|Samodzielnie śledzenia jednostki|Utworzona na podstawie tekstu szablonu przekształcania pakiet narzędzi (T4), który ma możliwość rejestrowania zmian skalarną skomplikowane, a właściwości nawigacji jednostki.|  
|Typ prosty|Pierwotny typ, który służy do definiowania właściwości w modelu koncepcyjnym.<br /><br /> Aby uzyskać więcej informacji, zobacz [koncepcyjny modelu typy (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#conceptual-model-types-csdl) i [modelu danych jednostki: Pierwotne typy danych](../../../../../docs/framework/data/adonet/entity-data-model-primitive-data-types.md).|  
|Podziel jednostki|Typ jednostki, który jest mapowany na dwa oddzielne typy w modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [jak: Zdefiniuj Model przy użyciu pojedynczej jednostki mapowane na dwie tabele](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb896233(v=vs.100)).|  
|model magazynu|Definicja modelu logicznego danych z obsługiwanego źródła danych, takich jak relacyjne bazy danych. Model magazynu jest zdefiniowany w SSDL souboru ssdl.<br /><br /> Aby uzyskać więcej informacji, zobacz [modelowanie i mapowanie](../../../../../docs/framework/data/adonet/ef/modeling-and-mapping.md) i [Specyfikacja SSDL](../../../../../docs/framework/data/adonet/ef/language-reference/ssdl-specification.md).|  
|Plik ssdl|Plik XML, który zawiera model magazynu, wyrażone w SSDL.|  
|store schema definition language (SSDL)|Oparte na języku XML język, który jest używany do definiowania typów jednostek, skojarzeń, kontenerów encji, zestawów encji i skojarzeń ustawia modelu magazynu, który często odnosi się do schematu bazy danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [Specyfikacja SSDL](../../../../../docs/framework/data/adonet/ef/language-reference/ssdl-specification.md).|  
|Tabela wg hierarchii|Metoda modelowania hierarchii typów w bazie danych, która zawiera atrybuty wszystkich typów w hierarchii w jednej tabeli.|  
|Tabela wg typu|Metoda modelowania hierarchii typów w bazie danych, która korzysta z wielu tabel z relacji jeden do jednego do modelu różnych typów.|
