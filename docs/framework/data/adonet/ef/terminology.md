# <a name="entity-framework-terminology"></a>Terminologia programu Entity Framework
W tym temacie zdefiniowano terminy często odwoływane w dokumentacji programu Entity Framework. Dostępne są linki do odpowiednich tematów, w których dostępne są dodatkowe informacje.  
  
|Termin|Definicja|  
|----------|----------------|  
|skojarzenie|Definicja relacji między typami jednostek.<br /><br /> Aby uzyskać więcej informacji, zobacz [Element skojarzenia (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#association-element-csdl) i [typ skojarzenia](../association-type.md).|  
|zestaw skojarzeń|Kontener logiczny dla wystąpień skojarzeń tego samego typu.<br /><br /> Aby uzyskać więcej informacji, zobacz [AssociationSet Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#associationset-element-csdl) i [zestaw skojarzeń](../association-set.md).|  
|Najpierw kod|Począwszy od entity framework 4.1 można utworzyć model programowo przy użyciu kodu pierwszego rozwoju. Istnieją dwa różne scenariusze rozwoju code first. W obu przypadkach deweloper definiuje model przez kodowanie definicji klas .NET Framework, a następnie opcjonalnie określa dodatkowe mapowanie lub konfigurację przy użyciu adnotacji danych lub płynnego interfejsu API.<br /><br /> Należy zauważyć, że code first rozwoju jest częścią entity framework 5.0. Entity Framework 5.0 nie jest częścią programu .NET Framework, ale jest zbudowany na .NET Framework 4.5. Entity Framework 5.0 jest dostępny jako pakiet NuGet [framework jednostki.](https://www.nuget.org/packages/EntityFramework) Aby uzyskać więcej informacji, zobacz [Poprzednie wersje entity framework](/ef/ef6/what-is-new/past-releases).|  
|drzewo poleceń|Wspólna, programowa reprezentacja wszystkich zapytań frameworka jednostek, które składają się z jednego lub więcej wyrażeń.<br /><br /> Aby uzyskać więcej informacji, zobacz [Omówienie struktury encji](overview.md).|  
|typ złożony|Klasa .NET Framework reprezentująca złożoną właściwość zgodnie z definicją w modelu koncepcyjnym. Typy złożone umożliwiają organizowanie właściwości skalarnych w obrębie encji. Obiekty złożone są wystąpieniami typów złożonych. Aby uzyskać więcej informacji, zobacz [ComplexType Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#complextype-element-csdl) i [typ złożony](../complex-type.md).|  
|Complextype|Specyfikacja typu danych, który reprezentuje właściwość nieskawarkową typu jednostki, która nie ma właściwości klucza.<br /><br /> Aby uzyskać więcej informacji, zobacz [ComplexType Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#complextype-element-csdl) i [typ złożony](../complex-type.md).|  
|model koncepcyjny|Abstrakcyjna specyfikacja typów jednostek, typów złożonych, skojarzeń, kontenerów jednostek, zestawów jednostek i zestawów skojarzeń w domenie aplikacji w ramach encji. Model koncepcyjny jest zdefiniowany w CSDL w pliku csdl.<br /><br /> Aby uzyskać więcej informacji, zobacz [Modelowanie i mapowanie](modeling-and-mapping.md).|  
|.csdl plik|Plik XML zawierający model koncepcyjny, wyrażony w csdl.|  
|Conceptual Schema Definition Language (CSDL)|Język oparty na języku XML, który jest używany do definiowania typów jednostek, skojarzeń, kontenerów jednostek, zestawów jednostek i zestawów skojarzeń modelu koncepcyjnego.<br /><br /> Aby uzyskać więcej informacji, zobacz [specyfikacja CSDL](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec).|  
|kontener|Logiczne grupowanie zestawów jednostek i skojarzeń.<br /><br /> Aby uzyskać więcej informacji, zobacz [EntityContainer Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitycontainer-element-csdl) i [kontener encji](../entity-container.md).|  
|współbieżność|Proces, który umożliwia wielu użytkownikom dostęp i zmianę udostępnionych danych w tym samym czasie. Domyślnie entity framework implementuje modelu współbieżności optymistyczne.|  
|kierunek|Odnosi się do asymetrycznego charakteru niektórych skojarzeń. Kierunek jest `FromRole` określony `ToRole` z `NavigationProperty` i `ReferentialConstraint` atrybuty lub elementu w schemacie.<br /><br /> Aby uzyskać więcej informacji, zobacz [NavigationProperty Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#navigationproperty-element-csdl) i [właściwość nawigacji](../navigation-property.md).|  
|gorliwy załadunek|Proces ładowania określonego zestawu powiązanych obiektów wraz z obiektami, które zostały jawnie wymagane w kwerendzie.|  
|.edmx plik|Plik XML zawierający model koncepcyjny (w CSDL), model magazynu (w SSDL) i mapowania między nimi (w MSL). Plik .edmx jest tworzony przez narzędzia modelu danych jednostki. Aby uzyskać więcej informacji, zobacz [.edmx File Overview](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/cc982042(v=vs.100)).|  
|end|Jednostka uczestnicząca w stowarzyszeniu.<br /><br /> Aby uzyskać więcej informacji, zobacz [Element końcowy (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#end-element-csdl) i [koniec skojarzenia](../association-end.md).|  
|jednostka|Koncepcja w domenie aplikacji, z której zdefiniowano typ danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [EntityType Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitytype-element-csdl) i [typ jednostki](../entity-type.md).|  
|Entityclient|Niezależny od magazynu dostawca danych ADO.NET zawierający `EntityConnection`takie `EntityCommand`klasy, jak , i `EntityDataReader`. Współpracuje [!INCLUDE[esql](../../../../../includes/esql-md.md)] z dostawcami danych ADO.NET specyficznych dla magazynu `SqlClient`i łączy się z nią, takimi jak .<br /><br /> Aby uzyskać więcej informacji, zobacz [EntityClient Provider for the Entity Framework](entityclient-provider-for-the-entity-framework.md).|  
|kontener jednostek|Określa zestawy jednostek i zestawy skojarzeń, które będą implementowane w określonym obszarze nazw.<br /><br /> Aby uzyskać więcej informacji, zobacz [EntityContainer Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitycontainer-element-csdl) i [kontener encji](../entity-container.md).|  
|Entity Data Model (EDM)|Zestaw pojęć, które opisują strukturę danych jako jednostki i relacje, niezależnie od jego postaci przechowywanej.<br /><br /> Aby uzyskać więcej informacji, zobacz [Model danych jednostki](../entity-data-model.md).|  
|Entity Framework|Zestaw technologii, które obsługuje rozwój aplikacji zorientowanych na dane, umożliwiając deweloperom pracę z modelami koncepcyjnymi, które są mapowane na schematy logiczne w źródłach danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [Omówienie struktury encji](overview.md).|  
|zestaw jednostek|Kontener logiczny dla jednostek danego typu i jego podtypów. Zestawy jednostek są mapowane do tabel w bazie danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [EntitySet Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entityset-element-csdl) i [zestaw jednostek](../entity-set.md).|  
|Entity SQL|Niezależny od magazynu dialekt JĘZYKA SQL, który działa bezpośrednio z schematami jednostek koncepcyjnych i który obsługuje pojęcia modelu koncepcyjnego, takie jak dziedziczenie i relacje.<br /><br /> Aby uzyskać więcej informacji, zobacz [Język SQL jednostki](./language-reference/entity-sql-language.md).|  
|typ jednostki|Klasa .NET Framework, która reprezentuje jednostkę zdefiniowaną w modelu koncepcyjnym. Typy jednostek mogą mieć właściwości skalarne, złożone i nawigacji. Obiekty są wystąpieniami typów jednostek. Aby uzyskać więcej informacji, zobacz [Praca z obiektami](working-with-objects.md).|  
|Typ obiektu|Specyfikacja typu danych, który zawiera klucz i nazwany zestaw właściwości i reprezentuje element najwyższego poziomu w modelu koncepcyjnym lub modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [EntityType Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitytype-element-csdl) i [typ jednostki](../entity-type.md).|  
|jawne ładowanie|Gdy obiekty są zwracane przez kwerendę, powiązane obiekty nie są ładowane w tym samym czasie. Domyślnie nie są one ładowane, `Load` dopóki jawnie wymagane przy użyciu metody na właściwości nawigacji.|  
|stowarzyszenie kluczy zagranicznych|Skojarzenie między jednostkami, które jest zarządzane za pośrednictwem właściwości klucza obcego.|  
|identyfikacja relacji|Relacja, w której klucz podstawowy jednostki głównej jest częścią klucza podstawowego jednostki zależnej. W tego rodzaju relacji jednostka zależna nie może istnieć bez jednostki głównej.|  
|niezależne stowarzyszenie|Skojarzenie między jednostkami, które są reprezentowane i śledzone przez niezależny obiekt.|  
|key|Atrybut typu jednostki, który określa, która właściwość lub zestaw właściwości jest używana do identyfikowania unikatowych wystąpień typu jednostki. Reprezentowane w warstwie <xref:System.Data.EntityKey> obiektu przez klasę.<br /><br /> Aby uzyskać więcej informacji, zobacz [Element klucza (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#key-element-csdl) i [klucz jednostki](../entity-key.md).|  
|leniwy załadunek|Gdy obiekty są zwracane przez kwerendę, powiązane obiekty nie są ładowane w tym samym czasie. Zamiast tego są one ładowane automatycznie, gdy dostęp do właściwości nawigacji.|  
|LINQ do Jednostek|Składnia kwerendy, która definiuje zestaw operatorów zapytań, które umożliwiają przechodzenie, filtrowanie i operacje rzutowania, które mają być wyrażone w sposób bezpośredni, deklaratywny w języku Visual C# i Visual Basic.<br /><br /> Aby uzyskać więcej informacji, zobacz [LINQ do jednostek](./language-reference/linq-to-entities.md).|  
|mapowanie|Specyfikacja korespondencji między elementami w modelu koncepcyjnym i elementów w modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [specyfikacja MSL](/ef/ef6/modeling/designer/advanced/edmx/msl-spec).|  
|msl|Plik XML, który zawiera mapowanie między modelem koncepcyjnym a modelem magazynu, wyrażony w pliku MSL.|  
|mapping specification language (MSL)|Język oparty na języku XML, który jest używany do mapowania elementów zdefiniowanych w modelu koncepcyjnym na elementy w modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [specyfikacja MSL](/ef/ef6/modeling/designer/advanced/edmx/msl-spec).|  
|funkcje modyfikacji|Procedury przechowywane, które są używane do wstawiania, aktualizowania i usuwania danych, które znajdują się w źródle danych. Te funkcje są używane zamiast poleceń generowanych przez platformę Entity Framework. Funkcje modyfikacji `Function` są definiowane przez element w modelu magazynu. [Element ModificationFunctionMapping](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/cc716778(v=vs.100)) mapuje te funkcje modyfikacji, aby wstawiać, aktualizować i usuwać operacje względem jednostek zdefiniowanych w modelu koncepcyjnym.|  
|Liczebność|Liczba jednostek, które mogą istnieć po każdej stronie relacji, zgodnie z definicją skojarzenia. Znany również jako kardynalność.<br /><br /> Aby uzyskać więcej informacji, zobacz [Element końcowy (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#end-element-csdl) i [koniec skojarzenia](../association-end.md).|  
|wiele zestawów jednostek dla typu|Możliwość zdefiniowania typu jednostki w więcej niż jednym zestawie jednostek.<br /><br /> Aby uzyskać więcej informacji, zobacz [EntitySet Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entityset-element-csdl) i [Jak: Definiowanie modelu z wieloma zestawami jednostek na typ](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb738537(v=vs.100)).|  
|właściwość nawigacji|Właściwość typu jednostki, która reprezentuje relację z innym typem jednostki, zgodnie z definicją skojarzenia. Właściwości nawigacji są używane do zwracania powiązanych obiektów jako <xref:System.Data.Objects.DataClasses.EntityCollection%601> lub <xref:System.Data.Objects.DataClasses.EntityReference%601>, w zależności od liczebności na drugim końcu skojarzenia.<br /><br /> Aby uzyskać więcej informacji, zobacz [NavigationProperty Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#navigationproperty-element-csdl) i [właściwość nawigacji](../navigation-property.md).|  
|ścieżka zapytania|Reprezentacja ciągu ścieżki określająca, które powiązane obiekty mają zostać zwracane podczas wykonywania kwerendy obiektowej. Ścieżka kwerendy jest definiowana przez wywołanie <xref:System.Data.Objects.ObjectQuery%601.Include%2A> metody w pliku <xref:System.Data.Objects.ObjectQuery%601>.<br /><br /> Aby uzyskać więcej informacji, zobacz [Ładowanie powiązanych obiektów](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb896272(v=vs.100)).|  
|kontekst obiektu|Reprezentuje kontener jednostki zdefiniowany w modelu koncepcyjnym. Zawiera połączenie z źródłem danych i zapewnia usługi, takie jak śledzenie zmian i rozpoznawanie tożsamości. Obiekt cont ext jest reprezentowany przez <xref:System.Data.Objects.ObjectContext> `DbContext` wystąpienie lub klasy.<br /><br /> `DbContext`jest częścią entity framework 5.0. Entity Framework 5.0 nie jest częścią programu .NET Framework, ale jest zbudowany na .NET Framework 4.5. Entity Framework 5.0 jest dostępny jako pakiet NuGet [framework jednostki.](https://www.nuget.org/packages/EntityFramework) Aby uzyskać więcej informacji, zobacz [Poprzednie wersje entity framework](/ef/ef6/what-is-new/past-releases).|  
|warstwa obiektu|Typy jednostek i definicje kontekstu obiektu, które są używane przez entity framework.|  
|kwerenda obiektu|Kwerenda wykonywana w kontekście obiektu względem modelu koncepcyjnego, który zwraca dane jako obiekty.<br /><br /> Aby uzyskać więcej informacji, zobacz [Kwerendy obiektów](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb896241(v=vs.100)).|  
|mapowanie obiektowe i relacyjne|Technika przekształcania danych z relacyjnej bazy danych w typy danych, które mogą być używane w aplikacjach obiektowych.<br /><br /> Entity Framework zapewnia usługi mapowania obiektowo-relacyjne przez mapowanie danych relacyjnych, zgodnie z definicją w modelu magazynu, do typów danych, zgodnie z definicją w modelu koncepcyjnym.<br /><br /> Aby uzyskać więcej informacji, zobacz [Modelowanie i mapowanie](modeling-and-mapping.md).|  
|Usługi obiektowe|Usługi świadczone przez entity framework, które umożliwiają kod aplikacji do pracy na jednostkach, takich jak .NET Framework obiektów.|  
|obiekt ignorantów trwałości|Obiekt, który nie zawiera żadnej logiki, która jest związana z magazynowaniem danych. Znany również jako jednostka POCO.|  
|Poco|Zwykły stary obiekt CLR. Obiekt, który nie dziedziczy z innej klasy lub implementuje interfejs.|  
|Jednostka POCO|Jednostka w ramach jednostki, która <xref:System.Data.Objects.DataClasses.EntityObject> <xref:System.Data.Objects.DataClasses.ComplexObject> nie dziedziczy po interfejsach entity Framework lub nie implementuje interfejsów entity framework. Często jednostki POCO są istniejącymi obiektami domeny, których używasz w aplikacji Entity Framework. Te jednostki obsługują trwałość ignorancji. Aby uzyskać więcej informacji, zobacz [Praca z jednostkami POCO](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/dd456853(v=vs.100)).|  
|obiekt proxy|Obiekt, który pochodzi z klasy POCO i jest generowany przez Entity Framework do obsługi śledzenia zmian i ładowania z opóźnieniem. Aby uzyskać więcej informacji, zobacz [Wymagania dotyczące tworzenia serwerów proxy POCO](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/dd468057(v=vs.100)).|  
|ograniczenie referencyjne|Ograniczenie zdefiniowane w modelu koncepcyjnym, które wskazuje, że jednostka ma zależną relację z inną encją. Ograniczenie to oznacza, że wystąpienie jednostki zależnej nie może istnieć bez odpowiedniego wystąpienia jednostki<br /><br /> Aby uzyskać więcej informacji, zobacz [ReferentialConstraint Element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#referentialconstraint-element-csdl) i [więzy integralności referencyjnej](../referential-integrity-constraint.md).|  
|Relacji|Logiczne połączenie między encjami.|  
|role (rola)|Nazwa nadana `End` każdemu skojarzeniu w celu wyjaśnienia semantyki relacji.<br /><br /> Aby uzyskać więcej informacji, zobacz [Element końcowy (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#end-element-csdl) i [koniec skojarzenia](../association-end.md).|  
|właściwość skalarna|Właściwość jednostki, która mapuje do jednego pola w modelu magazynu.|  
|jednostka samoobsługowa|Jednostka zbudowana na podstawie zestawu narzędzi przekształcania szablonu tekstu (T4), który ma możliwość rejestrowania zmian we właściwościach skalarnych, złożonych i nawigacyjnych.|  
|prosty typ|Typ pierwotny, który jest używany do definiowania właściwości w modelu koncepcyjnym.<br /><br /> Aby uzyskać więcej informacji, zobacz [Typy modeli koncepcyjnych (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#conceptual-model-types-csdl) i [Model danych jednostki: Typy danych pierwotnych](../entity-data-model-primitive-data-types.md).|  
|jednostka podzielona|Typ jednostki, który jest mapowany na dwa oddzielne typy w modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [Jak: Definiowanie modelu z pojedynczą jednostką mapowane na dwie tabele](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb896233(v=vs.100)).|  
|model przechowywania|Definicja modelu logicznego danych w obsługiwanym źródle danych, takim jak relacyjnej bazy danych. Model pamięci masowej jest zdefiniowany w SSDL w pliku .ssdl.<br /><br /> Aby uzyskać więcej informacji, zobacz [Modelowanie i mapowanie](modeling-and-mapping.md) oraz [Specyfikacja SSDL](/ef/ef6/modeling/designer/advanced/edmx/ssdl-spec).|  
|.ssdl plik|Plik XML zawierający model magazynu wyrażony w SSDL.|  
|store schema definition language (SSDL)|Język oparty na języku XML, który jest używany do definiowania typów jednostek, skojarzeń, kontenerów jednostek, zestawów jednostek i zestawów skojarzeń modelu magazynu, który często odpowiada schematowi bazy danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [Specyfikacja SSDL](/ef/ef6/modeling/designer/advanced/edmx/ssdl-spec).|  
|tabela na hierarchię|Metoda modelowania hierarchii typów w bazie danych, która zawiera atrybuty wszystkich typów w hierarchii w jednej tabeli.|  
|tabela na typ|Metoda modelowania hierarchii typów w bazie danych, która używa wielu tabel z relacjami jeden do jednego do modelowania różnych typów.|
