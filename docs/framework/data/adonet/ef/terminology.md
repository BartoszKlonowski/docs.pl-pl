# <a name="entity-framework-terminology"></a>Terminologia programu Entity Framework
Ten temat zawiera definicje terminów często występujących w dokumentacji Entity Framework. Linki są udostępniane odpowiednim tematom, w których są dostępne dodatkowe informacje.  
  
|Termin|Definicja|  
|----------|----------------|  
|skojarzenie|Definicja relacji między typami jednostek.<br /><br /> Aby uzyskać więcej informacji, zobacz [element Association (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#association-element-csdl) i [Typ powiązania](../association-type.md).|  
|zestaw skojarzeń|Logiczny kontener dla wystąpień skojarzeń tego samego typu.<br /><br /> Aby uzyskać więcej informacji, zobacz [element AssociationSet (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#associationset-element-csdl) i [zestaw skojarzeń](../association-set.md).|  
|Code First|Począwszy od Entity Framework 4,1 można programistycznie utworzyć model przy użyciu Code First programowanie. Istnieją dwa różne scenariusze tworzenia Code First. W obu przypadkach deweloper definiuje model poprzez kodowanie .NET Framework definicji klas, a następnie opcjonalnie określa dodatkowe mapowanie lub konfigurację przy użyciu adnotacji danych lub interfejsu API Fluent.<br /><br /> Należy pamiętać, że programowanie Code First jest częścią [Entity Framework 5,0](https://go.microsoft.com/fwlink/?LinkId=234900). Entity Framework 5,0 nie jest częścią .NET Framework, ale jest oparta na .NET Framework 4,5. Entity Framework 5,0 jest dostępny jako pakiet NuGet ["Entity Framework"](https://go.microsoft.com/fwlink/?LinkID=215714)[](https://go.microsoft.com/fwlink/?LinkId=232488) . Aby uzyskać więcej informacji, zobacz [Entity Framework wydań i przechowywania wersji](https://go.microsoft.com/fwlink/?LinkId=234899).|  
|drzewo poleceń|Wspólna, programowa reprezentacja wszystkich Entity Framework zapytań, które składają się z co najmniej jednego wyrażenia.<br /><br /> Aby uzyskać więcej informacji, zobacz [Entity Framework przegląd](overview.md).|  
|typ złożony|Klasa .NET Framework, która reprezentuje właściwość złożoną zdefiniowaną w modelu koncepcyjnym. Typy złożone umożliwiają organizowanie właściwości skalarnych w obrębie jednostek. Obiekty złożone są wystąpieniami typów złożonych. Aby uzyskać więcej informacji, zobacz [elementu complexType](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#complextype-element-csdl) i [typu złożonego](../complex-type.md).|  
|ComplexType|Specyfikacja typu danych, który reprezentuje nieskalarną właściwość typu jednostki, która nie ma właściwości Key.<br /><br /> Aby uzyskać więcej informacji, zobacz [elementu complexType](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#complextype-element-csdl) i [typu złożonego](../complex-type.md).|  
|model koncepcyjny|Specyfikacja abstrakcyjna dla typów jednostek, typów złożonych, skojarzeń, kontenerów jednostek, zestawów jednostek i zestawów skojarzeń w domenie aplikacji w Entity Framework. Model koncepcyjny jest zdefiniowany w pliku CSDL.<br /><br /> Aby uzyskać więcej informacji, zobacz [modelowanie i mapowanie](modeling-and-mapping.md).|  
|plik CSDL|Plik XML, który zawiera model koncepcyjny, wyrażony w CSDL.|  
|Conceptual Schema Definition Language (CSDL)|Język oparty na języku XML, który jest używany do definiowania typów jednostek, skojarzeń, kontenerów jednostek, zestawów jednostek i zestawów skojarzeń modelu koncepcyjnego.<br /><br /> Aby uzyskać więcej informacji, zobacz [Specyfikacja CSDL](./language-reference/csdl-specification.md).|  
|container|Logiczne grupowanie zestawów jednostek i skojarzeń.<br /><br /> Aby uzyskać więcej informacji, zobacz [element EntityContainer (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitycontainer-element-csdl) i [kontener jednostek](../entity-container.md).|  
|współbieżność|Proces umożliwiający wielu użytkownikom dostęp do danych udostępnionych i ich zmienianie jednocześnie. Domyślnie Entity Framework implementuje optymistyczny model współbieżności.|  
|kierunek|Odnosi się do asymetrycznego charakteru niektórych skojarzeń. Kierunek jest określony przy `FromRole` użyciu `ToRole` atrybutów `NavigationProperty` elementu lub lub `ReferentialConstraint` w schemacie.<br /><br /> Aby uzyskać więcej informacji, zobacz [element NavigationProperty (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#navigationproperty-element-csdl) i [Właściwość nawigacji](../navigation-property.md).|  
|Ładowanie eager|Proces ładowania określonego zestawu powiązanych obiektów wraz z obiektami, które zostały jawnie zażądane w zapytaniu.|  
|plik. edmx|Plik XML, który zawiera model koncepcyjny (w CSDL), model magazynu (w plikach SSDL) i mapowania między nimi (w pliku MSL). Plik EDMX jest tworzony przez narzędzia Entity Data Model. Aby uzyskać więcej informacji, zobacz [plik. edmx — Omówienie](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/cc982042(v=vs.100)).|  
|end|Jednostka uczestnicząca w skojarzeniu.<br /><br /> Aby uzyskać więcej informacji, zobacz [końcowy element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#end-element-csdl) i [koniec skojarzenia](../association-end.md).|  
|jednostka|Koncepcja w domenie aplikacji, z której jest zdefiniowany typ danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [element EntityType (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitytype-element-csdl) i [Typ jednostki](../entity-type.md).|  
|EntityClient|Niezależny od magazynu dostawca danych ADO.NET, który zawiera klasy `EntityConnection`takie `EntityCommand`jak, `EntityDataReader`, i. Współpracuje z [!INCLUDE[esql](../../../../../includes/esql-md.md)] i nawiązuje połączenie z dostawcami danych ADO.NET specyficznymi `SqlClient`dla magazynu, takimi jak.<br /><br /> Aby uzyskać więcej informacji, zobacz [EntityClient Provider for the Entity Framework](entityclient-provider-for-the-entity-framework.md).|  
|kontener jednostek|Określa zestawy jednostek i zestawy skojarzeń, które zostaną zaimplementowane w określonym obszarze nazw.<br /><br /> Aby uzyskać więcej informacji, zobacz [element EntityContainer (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitycontainer-element-csdl) i [kontener jednostek](../entity-container.md).|  
|Entity Data Model (EDM)|Zestaw koncepcji opisujący strukturę danych jako jednostki i relacje, niezależnie od ich przechowywanego formularza.<br /><br /> Aby uzyskać więcej informacji, zobacz [Entity Data Model](../entity-data-model.md).|  
|Entity Framework|Zestaw technologii, który obsługuje opracowywanie aplikacji zorientowanych na dane przez umożliwienie deweloperom pracy z modelami koncepcyjnymi, które są mapowane na schematy logiczne w źródłach danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [Entity Framework przegląd](overview.md).|  
|zestaw jednostek|Kontener logiczny dla jednostek danego typu i jego podtypów. Zestawy jednostek są mapowane na tabele w bazie danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [element EntitySet (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entityset-element-csdl) i [zestaw jednostek](../entity-set.md).|  
|Entity SQL|Dialekt niezależny od magazynu SQL, który działa bezpośrednio ze schematami jednostki koncepcyjnej i obsługuje koncepcje modelu koncepcyjnego, takie jak dziedziczenie i relacje.<br /><br /> Aby uzyskać więcej informacji, zobacz [Entity SQL Language](./language-reference/entity-sql-language.md).|  
|typ jednostki|Klasa .NET Framework, która reprezentuje jednostkę, która jest zdefiniowana w modelu koncepcyjnym. Typy jednostek mogą mieć właściwości skalarne, złożone i nawigacji. Obiekty są wystąpieniami typów jednostek. Aby uzyskać więcej informacji, zobacz [Praca z obiektami](working-with-objects.md).|  
|Typ entityType|Specyfikacja typu danych, który zawiera klucz i nazwany zestaw właściwości i reprezentuje element najwyższego poziomu w modelu koncepcyjnym lub modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [element EntityType (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entitytype-element-csdl) i [Typ jednostki](../entity-type.md).|  
|jawne ładowanie|Gdy obiekty są zwracane przez zapytanie, powiązane obiekty nie są ładowane w tym samym czasie. Domyślnie nie są one ładowane do momentu jawnego żądania przy `Load` użyciu metody dla właściwości nawigacji.|  
|Skojarzenie klucza obcego|Skojarzenie między jednostkami zarządzanymi przy użyciu właściwości klucza obcego.|  
|Identyfikowanie relacji|Relacja, w której klucz podstawowy jednostki głównej jest częścią klucza podstawowego jednostki zależnej. W tym rodzaju relacji jednostka zależna nie może istnieć bez jednostki podmiotu zabezpieczeń.|  
|niezależne skojarzenie|Skojarzenie między jednostkami, które są reprezentowane i śledzone przez niezależny obiekt.|  
|klawisz|Atrybut typu jednostki, który określa, która właściwość lub zestaw właściwości jest używany do identyfikowania unikatowych wystąpień typu jednostki. Reprezentowane w warstwie obiektów przez <xref:System.Data.EntityKey> klasę.<br /><br /> Aby uzyskać więcej informacji, zobacz [element Key (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#key-element-csdl) i [klucz jednostki](../entity-key.md).|  
|ładowanie z opóźnieniem|Gdy obiekty są zwracane przez zapytanie, powiązane obiekty nie są ładowane w tym samym czasie. Zamiast tego są ładowane automatycznie podczas uzyskiwania dostępu do właściwości nawigacji.|  
|LINQ do Jednostek|Składnia zapytania, która definiuje zestaw operatorów zapytań, które umożliwiają przechodzenie, filtrowanie i projekcje, które mają być wyrażone w bezpośrednim, deklaratywnym sposobie C# w wizualizacji i Visual Basic.<br /><br /> Aby uzyskać więcej informacji, zobacz [LINQ to Entities](./language-reference/linq-to-entities.md).|  
|mapowanie|Specyfikacja zgodności między elementami w modelu koncepcyjnym i elementami w modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [Specyfikacja MSL](./language-reference/msl-specification.md).|  
|plik. MSL|Plik XML, który zawiera mapowanie między modelem koncepcyjnym i modelem magazynu, wyrażony w pliku MSL.|  
|mapping specification language (MSL)|Język oparty na języku XML, który jest używany do mapowania elementów zdefiniowanych w modelu koncepcyjnym do elementów w modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [Specyfikacja MSL](./language-reference/msl-specification.md).|  
|funkcje modyfikacji|Procedury składowane służące do wstawiania, aktualizowania i usuwania danych znajdujących się w źródle danych. Te funkcje są używane zamiast Entity Framework wygenerowanych poleceń. Funkcje modyfikacji są definiowane przez `Function` element w modelu magazynu. Element [ModificationFunctionMapping](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/cc716778(v=vs.100)) mapuje te funkcje modyfikacji, aby wstawiać, aktualizować i usuwać operacje do jednostek, które są zdefiniowane w modelu koncepcyjnym.|  
|liczebność|Liczba jednostek, które mogą istnieć na każdej stronie relacji, zgodnie z definicją przez skojarzenie. Znana także jako kardynalność.<br /><br /> Aby uzyskać więcej informacji, zobacz [końcowy element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#end-element-csdl) i [koniec skojarzenia](../association-end.md).|  
|wiele zestawów jednostek na typ|Możliwość zdefiniowania typu jednostki w więcej niż jednym zestawie jednostek.<br /><br /> Aby uzyskać więcej informacji, zobacz [element EntitySet](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#entityset-element-csdl) i [instrukcje: Zdefiniuj model z wieloma zestawami jednostek na typ](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb738537(v=vs.100)).|  
|właściwość nawigacji|Właściwość typu jednostki, która reprezentuje relację z innym typem jednostki zdefiniowanym przez skojarzenie. Właściwości nawigacji służą do zwracania powiązanych obiektów jako <xref:System.Data.Objects.DataClasses.EntityCollection%601> <xref:System.Data.Objects.DataClasses.EntityReference%601>lub, w zależności od liczebności na drugim końcu skojarzenia.<br /><br /> Aby uzyskać więcej informacji, zobacz [element NavigationProperty (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#navigationproperty-element-csdl) i [Właściwość nawigacji](../navigation-property.md).|  
|Ścieżka zapytania|Ciąg reprezentujący ścieżkę, która określa, które obiekty będą zwracane po wykonaniu zapytania obiektu. Ścieżka zapytania jest definiowana przez wywołanie <xref:System.Data.Objects.ObjectQuery%601.Include%2A> metody <xref:System.Data.Objects.ObjectQuery%601>na.<br /><br /> Aby uzyskać więcej informacji, zobacz [ładowanie powiązanych obiektów](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb896272(v=vs.100)).|  
|kontekst obiektu|Reprezentuje kontener jednostek zdefiniowany w modelu koncepcyjnym. Zawiera połączenie z podstawowym źródłem danych i zapewnia usługi, takie jak śledzenie zmian i rozpoznawanie tożsamości. Element EXT kontynuacji obiektu jest reprezentowany przez wystąpienie <xref:System.Data.Objects.ObjectContext> klasy or. `DbContext`<br /><br /> `DbContext`jest częścią [Entity Framework 5,0](https://go.microsoft.com/fwlink/?LinkId=234900). Entity Framework 5,0 nie jest częścią .NET Framework, ale jest oparta na .NET Framework 4,5. Entity Framework 5,0 jest dostępny jako pakiet NuGet ["Entity Framework"](https://go.microsoft.com/fwlink/?LinkID=215714)[](https://go.microsoft.com/fwlink/?LinkId=232488) . Aby uzyskać więcej informacji, zobacz [Entity Framework wydań i przechowywania wersji](https://go.microsoft.com/fwlink/?LinkId=234899).|  
|warstwa obiektów|Typy jednostek i definicje kontekstu obiektów, które są używane przez Entity Framework.|  
|zapytanie obiektu|Zapytanie zostało wykonane w kontekście obiektu względem modelu koncepcyjnego, który zwraca dane jako obiekty.<br /><br /> Aby uzyskać więcej informacji, zobacz [zapytania dotyczące obiektów](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb896241(v=vs.100)).|  
|Mapowanie relacyjne obiektu|Technika przekształcania danych z relacyjnej bazy danych na typy danych, które mogą być używane w aplikacjach zorientowanych obiektowo.<br /><br /> Entity Framework zapewnia usługi mapowania relacyjnego obiektów, mapując relacyjne dane, zgodnie z definicją w modelu magazynu, na typy danych, zgodnie z definicją w modelu koncepcyjnym.<br /><br /> Aby uzyskać więcej informacji, zobacz [modelowanie i mapowanie](modeling-and-mapping.md).|  
|Usługi obiektów|Usługi zapewniane przez Entity Framework, które umożliwiają działanie kodu aplikacji na jednostkach, takich jak obiekty .NET Framework.|  
|trwałość — obiekt ignorujących|Obiekt, który nie zawiera żadnych logiki związanych z magazynem danych. Znana także jako jednostka POCO.|  
|POCO|Zwykły stary obiekt CLR. Obiekt, który nie dziedziczy z innej klasy ani nie implementuje interfejsu.|  
|Jednostka POCO|Jednostka w Entity Framework, która nie dziedziczy <xref:System.Data.Objects.DataClasses.EntityObject> po lub <xref:System.Data.Objects.DataClasses.ComplexObject> i nie implementuje interfejsów Entity Framework. Często jednostki POCO są istniejącymi obiektami domeny, które są używane w aplikacji Entity Framework. Te jednostki obsługują ignorujących trwałości. Aby uzyskać więcej informacji, zobacz [Praca z jednostkami poco](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/dd456853(v=vs.100)).|  
|obiekt serwera proxy|Obiekt, który pochodzi z klasy POCO i jest generowany przez Entity Framework do obsługi śledzenia zmian i ładowania z opóźnieniem. Aby uzyskać więcej informacji, zobacz [wymagania dotyczące tworzenia serwerów proxy poco](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/dd468057(v=vs.100)).|  
|ograniczenie referencyjne|Ograniczenie zdefiniowane w modelu koncepcyjnym, które wskazuje, że jednostka ma zależną relację z inną jednostką. To ograniczenie oznacza, że wystąpienie jednostki zależnej nie może istnieć bez odpowiadającego mu wystąpienia obiektu zasad<br /><br /> Aby uzyskać więcej informacji, zobacz [ReferentialConstraint element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#referentialconstraint-element-csdl) i [ograniczenie integralności referencyjnej](../referential-integrity-constraint.md).|  
|relacje|Połączenie logiczne między jednostkami.|  
|rola|Nazwa nadana każdemu `End` skojarzeniu w celu wyjaśnienia semantyki relacji.<br /><br /> Aby uzyskać więcej informacji, zobacz [końcowy element (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#end-element-csdl) i [koniec skojarzenia](../association-end.md).|  
|Właściwość skalarna|Właściwość jednostki, która mapuje do jednego pola w modelu magazynu.|  
|Jednostka samośledzenia|Jednostka skompilowana za pomocą zestawu narzędzi do przekształcania szablonów tekstowych (T4), który ma możliwość rejestrowania zmian właściwości skalarnej, złożonej i nawigacji.|  
|typ prosty|Typ pierwotny, który jest używany do definiowania właściwości w modelu koncepcyjnym.<br /><br /> Aby uzyskać więcej informacji, zobacz [Typy modeli koncepcyjnych (CSDL)](/ef/ef6/modeling/designer/advanced/edmx/csdl-spec#conceptual-model-types-csdl) i [Entity Data Model: Typy](../entity-data-model-primitive-data-types.md)danych pierwotnych.|  
|Podziel jednostkę|Typ jednostki, który jest mapowany na dwa oddzielne typy w modelu magazynu.<br /><br /> Aby uzyskać więcej informacji, zobacz [jak: Zdefiniuj model z pojedynczą jednostką zamapowana na dwie](https://docs.microsoft.com/previous-versions/dotnet/netframework-4.0/bb896233(v=vs.100))tabele.|  
|Model magazynu|Definicja modelu logicznego danych w obsługiwanym źródle danych, na przykład relacyjna baza danych. Model magazynu jest definiowany w plikach SSDL w pliku SSDL.<br /><br /> Aby uzyskać więcej informacji, zobacz [modelowanie i mapowanie](modeling-and-mapping.md) i [Specyfikacja SSDL](./language-reference/ssdl-specification.md).|  
|plik SSDL|Plik XML, który zawiera model magazynu wyrażony w plikach SSDL.|  
|store schema definition language (SSDL)|Język oparty na języku XML, który jest używany do definiowania typów jednostek, skojarzeń, kontenerów jednostek, zestawów jednostek i zestawów skojarzeń modelu magazynu, który często odpowiada schematowi bazy danych.<br /><br /> Aby uzyskać więcej informacji, zobacz [Specyfikacja SSDL](./language-reference/ssdl-specification.md).|  
|tabela na hierarchię|Metoda modelowania hierarchii typów w bazie danych, która zawiera atrybuty wszystkich typów w hierarchii w jednej tabeli.|  
|typ tabeli|Metoda modelowania hierarchii typów w bazie danych, która korzysta z wielu tabel z relacjami jeden-do-jednego, aby modelować różne typy.|
