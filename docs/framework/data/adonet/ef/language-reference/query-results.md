<span data-ttu-id="1d19c-103">Po przekonwertowaniu LINQ to Entities zapytania do drzew poleceń i wykonaniu wyniki zapytania są zwykle zwracane jako jeden z następujących elementów:</span><span class="sxs-lookup"><span data-stu-id="1d19c-103">After a LINQ to Entities query is converted to command trees and executed, the query results are usually returned as one of the following:</span></span>  
  
- <span data-ttu-id="1d19c-104">Kolekcja obiektów jednostek typu "zero" lub "rzutowanie typów złożonych" w modelu koncepcyjnym.</span><span class="sxs-lookup"><span data-stu-id="1d19c-104">A collection of zero or more typed entity objects or a projection of complex types in the conceptual model.</span></span>  
  
- <span data-ttu-id="1d19c-105">Typy CLR obsługiwane przez model koncepcyjny.</span><span class="sxs-lookup"><span data-stu-id="1d19c-105">CLR types supported by the conceptual model.</span></span>  
  
- <span data-ttu-id="1d19c-106">Kolekcje wbudowane.</span><span class="sxs-lookup"><span data-stu-id="1d19c-106">Inline collections.</span></span>  
  
- <span data-ttu-id="1d19c-107">Typy anonimowe.</span><span class="sxs-lookup"><span data-stu-id="1d19c-107">Anonymous types.</span></span>  
  
 <span data-ttu-id="1d19c-108">Gdy zapytanie zostało wykonane względem źródła danych, wyniki są uwzględniane w typach CLR i zwracane do klienta.</span><span class="sxs-lookup"><span data-stu-id="1d19c-108">When the query has executed against the data source, the results are materialized into CLR types and returned to the client.</span></span> <span data-ttu-id="1d19c-109">Wszystkie materializację obiektów są wykonywane przez Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="1d19c-109">All object materialization is performed by the Entity Framework.</span></span> <span data-ttu-id="1d19c-110">Wszelkie błędy, które wynikają z braku możliwości mapowania między Entity Framework i CLR spowodują, że wyjątki będą zgłaszane podczas materializację obiektów.</span><span class="sxs-lookup"><span data-stu-id="1d19c-110">Any errors that result from an inability to map between the Entity Framework and the CLR will cause exceptions to be thrown during object materialization.</span></span>
  
 <span data-ttu-id="1d19c-111">Jeśli wykonanie zapytania zwróci podstawowe typy modelu koncepcyjnego, wyniki składają się z typów CLR, które są autonomiczne i odłączone od Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="1d19c-111">If the query execution returns primitive conceptual model types, the results consist of CLR types that are stand-alone and disconnected from the Entity Framework.</span></span> <span data-ttu-id="1d19c-112">Jeśli jednak zapytanie zwraca kolekcję obiektów jednostek z określonym typem, reprezentowane przez <xref:System.Data.Objects.ObjectQuery%601> , te typy są śledzone przez kontekst obiektu.</span><span class="sxs-lookup"><span data-stu-id="1d19c-112">However, if the query returns a collection of typed entity objects, represented by <xref:System.Data.Objects.ObjectQuery%601>, those types are tracked by the object context.</span></span> <span data-ttu-id="1d19c-113">Wszystkie zachowania obiektów (takie jak kolekcje podrzędne/nadrzędne, śledzenie zmian, polimorfizm i tak dalej) są zdefiniowane w Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="1d19c-113">All object behavior (such as child/parent collections, change tracking, polymorphism, and so on) are as defined in the Entity Framework.</span></span> <span data-ttu-id="1d19c-114">Tej funkcji można użyć w swojej pojemności, zgodnie z definicją w Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="1d19c-114">This functionality can be used in its capacity, as defined in the Entity Framework.</span></span> <span data-ttu-id="1d19c-115">Aby uzyskać więcej informacji, zobacz [Praca z obiektami](../working-with-objects.md).</span><span class="sxs-lookup"><span data-stu-id="1d19c-115">For more information, see [Working with Objects](../working-with-objects.md).</span></span>
  
 <span data-ttu-id="1d19c-116">Typy struktur zwracane z zapytań (takich jak typy anonimowe i typy złożone o wartości null) mogą być `null` wartościami.</span><span class="sxs-lookup"><span data-stu-id="1d19c-116">Struct types returned from queries (such as anonymous types and nullable complex types) can be of `null` value.</span></span> <span data-ttu-id="1d19c-117"><xref:System.Data.Objects.DataClasses.EntityCollection%601>Właściwość zwróconego obiektu może być również `null` wartością.</span><span class="sxs-lookup"><span data-stu-id="1d19c-117">An <xref:System.Data.Objects.DataClasses.EntityCollection%601> property of a returned entity can also be of `null` value.</span></span> <span data-ttu-id="1d19c-118">Może to wynikać z projekcji właściwości kolekcji jednostki, która jest `null` wartością, taką jak wywołanie elementu, <xref:System.Linq.Queryable.FirstOrDefault%2A> <xref:System.Data.Objects.ObjectQuery%601> który nie ma żadnych elementów.</span><span class="sxs-lookup"><span data-stu-id="1d19c-118">This can result from projecting the collection property of an entity that is of `null` value, such as calling <xref:System.Linq.Queryable.FirstOrDefault%2A> on an <xref:System.Data.Objects.ObjectQuery%601> that has no elements.</span></span>  
  
 <span data-ttu-id="1d19c-119">W niektórych sytuacjach może wydawać się, że zapytanie generuje materiałowy wynik w trakcie jego wykonywania, ale zapytanie zostanie wykonane na serwerze, a obiekt Entity nie będzie nigdy materiałowy w środowisku CLR.</span><span class="sxs-lookup"><span data-stu-id="1d19c-119">In certain situations, a query might appear to generate a materialized result during its execution, but the query will be executed on the server and the entity object will never be materialized in the CLR.</span></span> <span data-ttu-id="1d19c-120">Może to spowodować problemy, jeśli są zależne od efektów ubocznych obiektu materializację.</span><span class="sxs-lookup"><span data-stu-id="1d19c-120">This can cause problems if you are depending on side effects of object materialization.</span></span>  
  
 <span data-ttu-id="1d19c-121">Poniższy przykład zawiera klasę niestandardową, `MyContact` z `LastName` właściwością.</span><span class="sxs-lookup"><span data-stu-id="1d19c-121">The following example contains a custom class, `MyContact`, with a `LastName` property.</span></span> <span data-ttu-id="1d19c-122">Gdy `LastName` Właściwość jest ustawiona, `count` zmienna jest zwiększana.</span><span class="sxs-lookup"><span data-stu-id="1d19c-122">When the `LastName` property is set, a `count` variable is incremented.</span></span> <span data-ttu-id="1d19c-123">Jeśli wykonasz dwa następujące zapytania, pierwsze zapytanie zostanie rozrastane, `count` gdy drugie zapytanie nie zostanie wykonane.</span><span class="sxs-lookup"><span data-stu-id="1d19c-123">If you execute the two following queries, the first query will increment `count` while the second query will not.</span></span> <span data-ttu-id="1d19c-124">Wynika to z faktu, że w drugim zapytaniu `LastName` Właściwość jest rzutowana z wyników, a `MyContact` Klasa nigdy nie jest tworzona, ponieważ nie jest wymagana do wykonania zapytania w sklepie.</span><span class="sxs-lookup"><span data-stu-id="1d19c-124">This is because in the second query the `LastName` property is projected from the results and the `MyContact` class is never created, because it is not required to execute the query on the store.</span></span>
