# <a name="canonical-functions"></a><span data-ttu-id="7b656-102">Funkcje Canonical</span><span class="sxs-lookup"><span data-stu-id="7b656-102">Canonical Functions</span></span>
<span data-ttu-id="7b656-103">W tej sekcji omówiono funkcje canonical, które są obsługiwane przez wszystkich dostawców danych i może być używany przez wszystkie technologie zapytań.</span><span class="sxs-lookup"><span data-stu-id="7b656-103">This section discusses canonical functions that are supported by all data providers, and can be used by all querying technologies.</span></span> <span data-ttu-id="7b656-104">Nie można rozszerzyć funkcje Canonical przez dostawcę.</span><span class="sxs-lookup"><span data-stu-id="7b656-104">Canonical functions cannot be extended by a provider.</span></span>  
  
 <span data-ttu-id="7b656-105">Te funkcje canonical będzie tłumaczona na odpowiednie funkcje źródła danych dla dostawcy.</span><span class="sxs-lookup"><span data-stu-id="7b656-105">These canonical functions will be translated to the corresponding data source functionality for the provider.</span></span> <span data-ttu-id="7b656-106">Dzięki temu wywołania funkcji wyrażony w postaci typowych źródeł danych.</span><span class="sxs-lookup"><span data-stu-id="7b656-106">This allows for function invocations expressed in a common form across data sources.</span></span>  
  
 <span data-ttu-id="7b656-107">Ponieważ te funkcje canonical są niezależne od źródeł danych, argumentów i typy kanonicznej funkcji są definiowane pod względem typów w modelu koncepcyjnym.</span><span class="sxs-lookup"><span data-stu-id="7b656-107">Because these canonical functions are independent of data sources, argument and return types of canonical functions are defined in terms of types in the conceptual model.</span></span> <span data-ttu-id="7b656-108">Jednak niektóre źródła danych mogą nie obsługiwać wszystkich typów w modelu koncepcyjnym.</span><span class="sxs-lookup"><span data-stu-id="7b656-108">However, some data sources might not support all types in the conceptual model.</span></span>  
  
 <span data-ttu-id="7b656-109">Gdy funkcje canonical są używane w [!INCLUDE[esql](../../../../../../includes/esql-md.md)] zapytania, odpowiednią funkcję zostanie wywołana w źródle danych.</span><span class="sxs-lookup"><span data-stu-id="7b656-109">When canonical functions are used in an [!INCLUDE[esql](../../../../../../includes/esql-md.md)] query, the appropriate function will be called at the data source.</span></span>  
  
 <span data-ttu-id="7b656-110">Wszystkie funkcje canonical mają zachowanie danych wejściowych o wartości null i warunki błędów, które zostały jawnie określone.</span><span class="sxs-lookup"><span data-stu-id="7b656-110">All canonical functions have both null-input behavior and error conditions explicitly specified.</span></span> <span data-ttu-id="7b656-111">Dostawców Store powinny być zgodne z tego zachowania, ale [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] nie wymusza to zachowanie.</span><span class="sxs-lookup"><span data-stu-id="7b656-111">Store providers should comply with that behavior, but [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] does not enforce this behavior.</span></span>  
  
 <span data-ttu-id="7b656-112">W przypadku scenariuszy LINQ zapytania względem [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] obejmują metody CLR mapowania do metod w źródle danych.</span><span class="sxs-lookup"><span data-stu-id="7b656-112">For LINQ scenarios, queries against the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] involve mapping CLR methods to methods in the underlying data source.</span></span> <span data-ttu-id="7b656-113">Mapę metodach CLR, aby funkcje canonical tak, aby zestaw konkretnych metod poprawnie zmapuje, niezależnie od tego źródła danych.</span><span class="sxs-lookup"><span data-stu-id="7b656-113">The CLR methods map to canonical functions, so that a specific set of methods will correctly map, regardless of the data source.</span></span>
