## <a name="sample-diffgram"></a><span data-ttu-id="31016-177">Przykładowy format DiffGram</span><span class="sxs-lookup"><span data-stu-id="31016-177">Sample DiffGram</span></span>  
 <span data-ttu-id="31016-178">Poniżej przedstawiono przykład formatu w formacie DiffGram.</span><span class="sxs-lookup"><span data-stu-id="31016-178">An example of the DiffGram format is shown below.</span></span> <span data-ttu-id="31016-179">Ten przykład przedstawia wynik aktualizacji do wiersza w tabeli, aby zmiany zostały zatwierdzone.</span><span class="sxs-lookup"><span data-stu-id="31016-179">This example shows the result of an update to a row in a table before the changes have been committed.</span></span> <span data-ttu-id="31016-180">Wiersz z CustomerID "ALFKI" ma został zmodyfikowany, ale nie zaktualizowano.</span><span class="sxs-lookup"><span data-stu-id="31016-180">The row with a CustomerID of "ALFKI" has been modified, but not updated.</span></span> <span data-ttu-id="31016-181">Co w efekcie występuje **bieżącego** wiersz z **diffgr:id** z "Customers1" w **\<** ***DataInstance*** **>** bloku, a **oryginalnego** wiersz z **diffgr:id** z "Customers1" w  **\<diffgr: przed >** bloku.</span><span class="sxs-lookup"><span data-stu-id="31016-181">As a result, there is a **Current** row with a **diffgr:id** of "Customers1" in the **\<** ***DataInstance*** **>** block, and an **Original** row with a **diffgr:id** of "Customers1" in the **\<diffgr:before>** block.</span></span> <span data-ttu-id="31016-182">Zawiera wiersz z CustomerID "ANATR" **RowError**, dlatego jest oznaczona za pomocą `diffgr:hasErrors="true"` i powiązane elementu w  **\<diffgr:errors >** bloku.</span><span class="sxs-lookup"><span data-stu-id="31016-182">The row with a CustomerID of "ANATR" includes a **RowError**, so it is annotated with `diffgr:hasErrors="true"` and there is a related element in the **\<diffgr:errors>** block.</span></span>  
  
```xml  
<diffgr:diffgram xmlns:msdata="urn:schemas-microsoft-com:xml-msdata" xmlns:diffgr="urn:schemas-microsoft-com:xml-diffgram-v1">  
  <CustomerDataSet>  
    <Customers diffgr:id="Customers1" msdata:rowOrder="0" diffgr:hasChanges="modified">  
      <CustomerID>ALFKI</CustomerID>  
      <CompanyName>New Company</CompanyName>  
    </Customers>  
    <Customers diffgr:id="Customers2" msdata:rowOrder="1" diffgram:hasErrors="true">  
      <CustomerID>ANATR</CustomerID>  
      <CompanyName>Ana Trujillo Emparedados y Helados</CompanyName>  
    </Customers>  
    <Customers diffgr:id="Customers3" msdata:rowOrder="2">  
      <CustomerID>ANTON</CustomerID>  
      <CompanyName>Antonio Moreno Taquera</CompanyName>  
    </Customers>  
    <Customers diffgr:id="Customers4" msdata:rowOrder="3">  
      <CustomerID>AROUT</CustomerID>  
      <CompanyName>Around the Horn</CompanyName>  
    </Customers>  
  </CustomerDataSet>  
  <diffgr:before>  
    <Customers diffgr:id="Customers1" msdata:rowOrder="0">  
      <CustomerID>ALFKI</CustomerID>  
      <CompanyName>Alfreds Futterkiste</CompanyName>  
    </Customers>  
  </diffgr:before>  
  <diffgr:errors>  
    <Customers diffgr:id="Customers2" diffgr:Error="An optimistic concurrency violation has occurred for this row."/>  
  </diffgr:errors>  
</diffgr:diffgram>  
```
