 <span data-ttu-id="380cc-108">W modelu koncepcyjnym można tworzyć hierarchie dziedziczenia, w których typ pochodny dziedziczy z innego typu pochodnego.</span><span class="sxs-lookup"><span data-stu-id="380cc-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="380cc-109">Typ w górnej części hierarchii (jeden typ w hierarchii, która nie jest typem pochodnym) jest nazywany *typem głównym*.</span><span class="sxs-lookup"><span data-stu-id="380cc-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="380cc-110">W hierarchii dziedziczenia [klucz jednostki](entity-key.md) musi być zdefiniowany w typie głównym.</span><span class="sxs-lookup"><span data-stu-id="380cc-110">In an inheritance hierarchy, the [entity key](entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="380cc-111">Nie można skompilować hierarchii dziedziczenia, w których typ pochodny dziedziczy z więcej niż jednego typu.</span><span class="sxs-lookup"><span data-stu-id="380cc-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="380cc-112">Na przykład w modelu `Book` koncepcyjnym z typem jednostki można zdefiniować typy `FictionBook` pochodne i `NonFictionBook` każdy z `Book`nich dziedziczy.</span><span class="sxs-lookup"><span data-stu-id="380cc-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="380cc-113">Nie można jednak zdefiniować typu, który dziedziczy z obu `FictionBook` typów i. `NonFictionBook`</span><span class="sxs-lookup"><span data-stu-id="380cc-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
