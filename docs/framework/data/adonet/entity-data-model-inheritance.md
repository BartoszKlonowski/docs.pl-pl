<span data-ttu-id="4054a-103">Modelu danych jednostki (EDM) obsługuje dziedziczenia dla [typów jednostek](../../../../docs/framework/data/adonet/entity-type.md).</span><span class="sxs-lookup"><span data-ttu-id="4054a-104">Dziedziczenie w modelu EDM jest podobny do dziedziczenia klas w zorientowane obiektowo języków programowania.</span><span class="sxs-lookup"><span data-stu-id="4054a-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="4054a-105">Tak jak w przypadku klas w językach zorientowane obiektowo w modelu koncepcyjnym można zdefiniować typu jednostki ( *typu pochodnego*) dziedziczący z innego typu jednostki ( *typ podstawowy*).</span><span class="sxs-lookup"><span data-stu-id="4054a-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="4054a-106">Jednak w przeciwieństwie do klas w programowanie zorientowane obiektowo, w modelu koncepcyjnym Typ pochodny zawsze dziedziczy wszystkie [właściwości](../../../../docs/framework/data/adonet/property.md) i [właściwości nawigacji](../../../../docs/framework/data/adonet/navigation-property.md) typu podstawowego.</span><span class="sxs-lookup"><span data-stu-id="4054a-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](../../../../docs/framework/data/adonet/property.md) and [navigation properties](../../../../docs/framework/data/adonet/navigation-property.md) of the base type.</span></span> <span data-ttu-id="4054a-107">Nie można przesłonić dziedziczonego właściwości w typie pochodnym.</span><span class="sxs-lookup"><span data-stu-id="4054a-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="4054a-108">W modelu koncepcyjnym można utworzyć hierarchii dziedziczenia, w których typem pochodnym dziedziczy z typu pochodnego.</span><span class="sxs-lookup"><span data-stu-id="4054a-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="4054a-109">Typ w hierarchii (jeden typ w hierarchii, która nie jest typem pochodnym) jest nazywany *Typ główny*.</span><span class="sxs-lookup"><span data-stu-id="4054a-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="4054a-110">W hierarchii dziedziczenia [klucz jednostki](../../../../docs/framework/data/adonet/entity-key.md) musi być zdefiniowana dla typu głównego.</span><span class="sxs-lookup"><span data-stu-id="4054a-110">In an inheritance hierarchy, the [entity key](../../../../docs/framework/data/adonet/entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="4054a-111">Nie można utworzyć hierarchii dziedziczenia, w których typem pochodnym dziedziczy z więcej niż jednego typu.</span><span class="sxs-lookup"><span data-stu-id="4054a-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="4054a-112">Na przykład w modelu koncepcyjnym z `Book` typu jednostki, można zdefiniować typy pochodne `FictionBook` i `NonFictionBook` czy każdy dziedziczyć `Book`.</span><span class="sxs-lookup"><span data-stu-id="4054a-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="4054a-113">Jednak nie następnie można zdefiniować typu, który dziedziczy z obu `FictionBook` i `NonFictionBook` typów.</span><span class="sxs-lookup"><span data-stu-id="4054a-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
