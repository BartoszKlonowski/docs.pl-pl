# <a name="entity-data-model-inheritance"></a><span data-ttu-id="c8288-102">Entity Data Model: dziedziczenie</span><span class="sxs-lookup"><span data-stu-id="c8288-102">Entity Data Model: Inheritance</span></span>
<span data-ttu-id="c8288-103">Entity Data Model (EDM) obsługuje dziedziczenie dla [typów jednostek](entity-type.md).</span><span class="sxs-lookup"><span data-stu-id="c8288-103">The Entity Data Model (EDM) supports inheritance for [entity types](entity-type.md).</span></span> <span data-ttu-id="c8288-104">Dziedziczenie w modelu EDM jest podobne do dziedziczenia klas w językach programowania zorientowanego obiektowo.</span><span class="sxs-lookup"><span data-stu-id="c8288-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="c8288-105">Podobnie jak w przypadku klas w językach zorientowanych obiektowo, w modelu koncepcyjnym można zdefiniować typ jednostki ( *Typ pochodny*), który dziedziczy z innego typu jednostki ( *Typ podstawowy*).</span><span class="sxs-lookup"><span data-stu-id="c8288-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="c8288-106">Jednak w przeciwieństwie do klas w programowaniu zorientowanym na obiektach, w modelu koncepcyjnym typ pochodny zawsze dziedziczy wszystkie [Właściwości](property.md) i [właściwości nawigacji](navigation-property.md) typu podstawowego.</span><span class="sxs-lookup"><span data-stu-id="c8288-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](property.md) and [navigation properties](navigation-property.md) of the base type.</span></span> <span data-ttu-id="c8288-107">Nie można przesłonić dziedziczonych właściwości w typie pochodnym.</span><span class="sxs-lookup"><span data-stu-id="c8288-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="c8288-108">W modelu koncepcyjnym można tworzyć hierarchie dziedziczenia, w których typ pochodny dziedziczy z innego typu pochodnego.</span><span class="sxs-lookup"><span data-stu-id="c8288-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="c8288-109">Typ w górnej części hierarchii (jeden typ w hierarchii, która nie jest typem pochodnym) jest nazywany *typem głównym*.</span><span class="sxs-lookup"><span data-stu-id="c8288-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="c8288-110">W hierarchii dziedziczenia [klucz jednostki](entity-key.md) musi być zdefiniowany w typie głównym.</span><span class="sxs-lookup"><span data-stu-id="c8288-110">In an inheritance hierarchy, the [entity key](entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="c8288-111">Nie można skompilować hierarchii dziedziczenia, w których typ pochodny dziedziczy z więcej niż jednego typu.</span><span class="sxs-lookup"><span data-stu-id="c8288-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="c8288-112">Na przykład w modelu koncepcyjnym z typem jednostki `Book` można zdefiniować typy pochodne `FictionBook` i `NonFictionBook`, które dziedziczą z `Book`.</span><span class="sxs-lookup"><span data-stu-id="c8288-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="c8288-113">Nie można jednak zdefiniować typu, który dziedziczy z obu typów `FictionBook` i `NonFictionBook`.</span><span class="sxs-lookup"><span data-stu-id="c8288-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
