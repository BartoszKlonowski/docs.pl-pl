# <a name="client-application-services-overview"></a>Przegląd usług aplikacji klienta
Usługi aplikacji klienta zapewniają uproszczony dostęp do [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] logowania, ról i usług profilu z aplikacji Windows Forms i Windows Presentation Foundation (WPF). [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] usługi aplikacji znajdują się w firmy Microsoft w wersji 2.0 rozszerzenia AJAX programu ASP.NET, który jest dołączony do programu Visual Studio 2008 i [!INCLUDE[net_v35_long](../../../includes/net-v35-long-md.md)]. Te usługi umożliwiają wielu sieci Web i aplikacji opartych na Windows, aby udostępnić funkcje zarządzania użytkownikami z jednego serwera i informacji o użytkowniku.  
  
 Usługi aplikacji klienta obejmują klienta dostawców usług, które Podłącz do modelu rozszerzalności usługi sieci Web, aby umożliwić następujące funkcje dla aplikacji systemu Windows:  
  
-   Prostej konfiguracji klienta. Można włączyć i skonfigurować logowania, ról i usług profilu za pomocą projektanta projektu programu Visual Studio lub za pośrednictwem dostawców usług klienta w pliku App.config projektu. Aby uzyskać więcej informacji, zobacz [porady: Konfigurowanie usług aplikacji klienta](../../../docs/framework/common-client-technologies/how-to-configure-client-application-services.md).  
  
-   Prosty sposób programowania. Po włączeniu i skonfigurowaniu usługi aplikacji klienta dostawcy usług są dostępne pośrednio za pośrednictwem istniejących [!INCLUDE[dnprdnlong](../../../includes/dnprdnlong-md.md)] członkostwo, role i klasy ustawień aplikacji. Można również uzyskać dostęp do [!INCLUDE[net_v35_long](../../../includes/net-v35-long-md.md)] klasy, które implementują usług aplikacji klienta. Jednak w większości przypadków bezpośredniego dostępu jest niepotrzebne. Aby uzyskać więcej informacji na temat klas usługi aplikacji klienta Zobacz sekcję "Klasy usług aplikacji klienta" w tym temacie.  
  
-   Obsługa trybu offline. Aplikacje systemu Windows często muszą działać w środowiskach sporadycznie łączone. Gdy aplikacja jest w trybie online, dostawców usług klienta będzie buforować wartościami pobranymi z serwera do użycia, gdy aplikacja jest w trybie offline.  
  
-   Integracja za pomocą projektanta ustawień aplikacji programu Visual Studio. Po dodaniu ustawienia do projektu w programie Visual Studio, można określić ustawienia, które mają być dostępne za pośrednictwem dostawcy usług w ustawieniach klienta.  
  
 W poniższych sekcjach opisano te funkcje, które bardziej szczegółowo. Aby uzyskać więcej informacji na temat [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] usługi aplikacji, zobacz [przegląd usług aplikacji ASP.NET](https://msdn.microsoft.com/library/1162e529-0d70-44b2-b3ab-83e60c695013).  
  
## <a name="authentication"></a>Uwierzytelnianie  
 Usługi aplikacji klienta służy do sprawdzania poprawności użytkownika przez istniejące [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] usługi uwierzytelniania. Użytkownik może sprawdzić przy użyciu uwierzytelniania Windows lub formularzy. Uwierzytelnianie Windows oznacza, że tożsamość użytkownika dostarczonego przez system operacyjny, gdy użytkownik loguje się komputera lub domeny. Zazwyczaj użyje uwierzytelniania Windows za pomocą aplikacji wdrożonej w firmowej sieci intranet. Uwierzytelnianie formularzy oznacza, że należy umieścić kontrolek logowania w aplikacji i poświadczenia pozyskana, są przekazywane do dostawcy uwierzytelniania. Będą zwykle używać uwierzytelniania formularzy z aplikacji wdrożonej w Internecie.  
  
 Aby sprawdzić poprawność użytkownika, należy wywołać `static` <xref:System.Web.Security.Membership.ValidateUser%2A?displayProperty=nameWithType> metody. Ta metoda uzyskuje dostęp do dostawcy usług klienta, które są skonfigurowane dla twojej aplikacji i zwraca <xref:System.Boolean> wartość wskazującą, czy użytkownik jest prawidłowa. Aby uzyskać więcej informacji, zobacz [porady: Implementowanie logowania użytkownika przy użyciu usług aplikacji klienta](../../../docs/framework/common-client-technologies/how-to-implement-user-login-with-client-application-services.md).  
  
 Korzystając z uwierzytelniania Windows, musisz przekazać puste ciągi lub `null` jako parametry <xref:System.Web.Security.Membership.ValidateUser%2A> metody. Podczas korzystania z uwierzytelniania Windows należy zawsze zwraca wywołanie tej metody `true`.  
  
 Za pomocą uwierzytelniania formularzy <xref:System.Web.Security.Membership.ValidateUser%2A> metoda zwraca wartość wskazującą, czy usługi zdalnej został uwierzytelniony użytkownik. Jeśli weryfikacja zakończy się pomyślnie, pliku cookie uwierzytelniania są przechowywane na lokalnym dysku twardym. Ten plik cookie jest używany w celu potwierdzenia weryfikacji podczas uzyskiwania dostępu do ról i ustawienia usług.  
  
 Korzystając z uwierzytelniania formularzy, można przekazać nazwę użytkownika i hasło, aby <xref:System.Web.Security.Membership.ValidateUser%2A> metody. Można również przekazać puste ciągi lub `null` jako parametry do używania dostawcy poświadczeń. Dostawca poświadczeń jest klasa, która zapewnia i określ w konfiguracji aplikacji. Klasa dostawcy poświadczeń musi implementować <xref:System.Web.ClientServices.Providers.IClientFormsAuthenticationCredentialsProvider> interfejs, który zawiera jedną metodę o nazwie <xref:System.Web.ClientServices.Providers.IClientFormsAuthenticationCredentialsProvider.GetCredentials%2A>. Za pomocą dostawcy poświadczeń umożliwia udostępnianie jednej nazwy logowania, okno dialogowe wśród wielu aplikacji. Aby uzyskać więcej informacji, zobacz [porady: Konfigurowanie usług aplikacji klienta](../../../docs/framework/common-client-technologies/how-to-configure-client-application-services.md).  
  
 Po skonfigurowaniu aplikacji do używania dostawcy poświadczeń za pomocą uwierzytelniania formularzy, należy przekazać puste ciągi lub `null` jako parametry <xref:System.Web.Security.Membership.ValidateUser%2A> metody. Dostawcy usług będą wywoływać usługi <xref:System.Web.ClientServices.Providers.IClientFormsAuthenticationCredentialsProvider.GetCredentials%2A?displayProperty=nameWithType> implementacji metody. Zazwyczaj będzie zaimplementować tę metodę, aby wyświetlić okno dialogowe i wrócić wypełnione <xref:System.Web.ClientServices.Providers.ClientFormsAuthenticationCredentials> obiektu.  
  
 Aby uzyskać więcej informacji na temat uwierzytelniania, zobacz [Uwierzytelnianie ASP.NET](https://msdn.microsoft.com/library/fc10b0ef-4ce4-4a7f-9174-886325221ee1). Aby uzyskać informacje dotyczące sposobu konfigurowania [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] usługi uwierzytelniania, zobacz [za pomocą uwierzytelniania formularzy z Microsoft Ajax](https://msdn.microsoft.com/library/c50f7dc5-323c-4c63-b4f3-96edfc1e815e).  
  
## <a name="roles"></a>Role  
 Usługi aplikacji klienta można użyć do pobierania informacji o roli z istniejącego [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] usługi ról. Aby ustalić, czy obecnie uwierzytelniony użytkownik ma określoną rolę, należy wywołać <xref:System.Security.Principal.IPrincipal.IsInRole%2A> metody <xref:System.Security.Principal.IPrincipal> odwołania są pobierane z `static` <xref:System.Threading.Thread.CurrentPrincipal%2A?displayProperty=nameWithType> właściwości. <xref:System.Security.Principal.IPrincipal.IsInRole%2A> Metoda przyjmuje nazwę roli, jako parametr i zwraca <xref:System.Boolean> wartość wskazującą, czy bieżący użytkownik ma określoną rolę. Ta metoda zwróci `false` Jeśli użytkownik nie jest uwierzytelniony, lub nie znajduje się w określonej roli.  
  
 Aby uzyskać informacje dotyczące sposobu konfigurowania [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] usługi ról, zobacz [przy użyciu informacje o rolach przy użyciu Microsoft Ajax](https://msdn.microsoft.com/library/280f6ad9-ba1a-4fc9-b0cc-22e39e54a82d).  
  
## <a name="settings"></a>Ustawienia  
 Usługi aplikacji klienta można użyć do pobierania ustawień aplikacji użytkownika z istniejącego [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] profilu usługi. Usługi aplikacji klienta funkcja ustawień sieci Web integruje się z funkcji ustawień aplikacji, które są udostępniane w [!INCLUDE[dnprdnlong](../../../includes/dnprdnlong-md.md)]. Można pobrać ustawień sieci Web, należy najpierw wygenerować `Settings` klasy (dostępne jako `Properties.Settings.Default` w języku C# i `My.Settings` w języku Visual Basic) w projekcie za pomocą **ustawienia** kartę projektanta projektu programu Visual Studio. Na **ustawienia** karcie, można użyć **ustawień sieci Web obciążenia** przycisk, aby pobrać ustawień sieci Web i dodać je do wygenerowany `Settings` klasy. Można użyć ustawień sieci Web skonfigurowana do użycia przez wszystkich uwierzytelnionych użytkowników lub wszyscy użytkownicy anonimowi.  
  
 Aby uzyskać więcej informacji na temat ustawień aplikacji, zobacz [Przegląd ustawień aplikacji](../../../docs/framework/winforms/advanced/application-settings-overview.md). Aby uzyskać informacje o tym, jak można implementować własne klasy ustawienia zamiast generując w programie Visual Studio, zobacz [porady: Tworzenie ustawień aplikacji](../../../docs/framework/winforms/advanced/how-to-create-application-settings.md). Aby uzyskać informacje dotyczące sposobu konfigurowania [!INCLUDE[ajax_current_short](../../../includes/ajax-current-short-md.md)] profilu usługi, zobacz [przy użyciu informacji o profilu z Microsoft Ajax](https://msdn.microsoft.com/library/91239ae6-d01c-4f4e-a433-eb9040dbed61).  
  
## <a name="client-application-services-classes"></a>Klasy usługi aplikacji klienta  
 W poniższej tabeli opisano klasy, które implementują funkcji usług aplikacji klienta.  
  
 Aplikacje, które używają tylko uwierzytelnianie podstawowe, ról i funkcji ustawienia nie trzeba bezpośrednio dostęp do tych klas. Zamiast tego takich aplikacji będzie dostęp do dostawców usług aplikacji klienta, pośrednio przy użyciu konfiguracji aplikacji i interfejsów API, opisanych w poprzednich sekcjach. Będzie dostęp do tych klas bezpośrednio, aby zaimplementować dodatkowe funkcje, takie jak wylogowania użytkownika i możliwości w trybie offline.  
  
> [!NOTE]
>  Wszystkie usługi aplikacji klienta interfejsy API są synchroniczne. Usługi aplikacji klienta nie obsługują bezpośrednio zachowanie asynchroniczne.  
  
 Dostawców usług aplikacji klienta wdrożenia lub rozszerzyć pewne standardowe [!INCLUDE[dnprdnlong](../../../includes/dnprdnlong-md.md)] typów, ale nie należy implementować każdy element członkowski i funkcji zdefiniowanych przez te typy. Na przykład nie można użyć dostawców usług aplikacji klienta do wdrożenia aplikacji do zarządzania użytkownika do tworzenia nowych użytkowników i zarządzania członkostwa w roli. Aby zaimplementować tę funkcję, należy użyć aktualnie kodu po stronie serwera i aplikacji sieci Web. Aby określić, które elementy członkowskie nie są zaimplementowane, zobacz dokumentacją referencyjną, który jest dostępny z poziomu linków w tej tabeli.

|-----------|-----------------|

|<xref:System.Web.ClientServices.ClientRolePrincipal>|Ta klasa zarządza rolami użytkowników.<br /><br /> Ta klasa nie ma żadnych elementów członkowskich, które nie są dostępne bezpośrednio. |<xref:System.Web.ClientServices.ConnectivityStatus>|Ta klasa udostępnia `static` <xref:System.Web.ClientServices.ConnectivityStatus.IsOffline%2A> właściwości, który można wykorzystać do przełączania aplikacji do trybu offline.|
