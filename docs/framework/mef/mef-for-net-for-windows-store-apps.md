# <a name="mef-for-net-for-windows-store-apps"></a>MEF dla platformy .NET dla aplikacji do Sklepu Windows
<xref:System.Composition?displayProperty=nameWithType> i jej podrzędne przestrzenie nazw zawierają typy służące do opracowywania aplikacji rozszerzalnych [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] za pomocą Managed Extensibility Framework (MEF). Te przestrzenie nazw są częścią podzestawu [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] dla [!INCLUDE[win8](../../../includes/win8-md.md)] systemu operacyjnego.  
  
 Te przestrzenie nazw nie są częścią podstawowej biblioteki klas dystrybuowanej z .NET Framework. Aby zainstalować te przestrzenie nazw, Otwórz projekt w programie Visual Studio, wybierz polecenie **Zarządzaj pakietami NuGet** z menu **projekt** i Wyszukaj w trybie online pakiet Microsoft. kompozycji.  
  
- <xref:System.Composition?displayProperty=nameWithType> zawiera klasy, które stanowią podstawowe MEF dla [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] aplikacji.  
  
- <xref:System.Composition.Convention?displayProperty=nameWithType> udostępnia typy obsługujące używanie MEF z modelem konfiguracji opartym na Konwencji.  
  
- <xref:System.Composition.Hosting?displayProperty=nameWithType> udostępnia typy MEF, które są przydatne dla deweloperów aplikacji hosta.  
  
- <xref:System.Composition.Hosting.Core?displayProperty=nameWithType> zapewnia typy MEF używane wewnętrznie przez aparat kompozycji.  
  
 Aby uzyskać więcej informacji na temat [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] i listę przestrzeni nazw i typów, które zawiera, zobacz artykuł [.NET dla aplikacji do sklepu Windows — omówienie](https://go.microsoft.com/fwlink/p/?LinkID=238312) w centrum deweloperów systemu Windows.
