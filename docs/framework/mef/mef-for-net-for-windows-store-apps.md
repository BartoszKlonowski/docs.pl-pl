# <a name="mef-for-net-for-windows-store-apps"></a>MEF dla platformy .NET dla aplikacji ze Sklepu Windows
<xref:System.Composition?displayProperty=nameWithType> i jej podrzędne przestrzenie nazw zawiera typy związane z opracowywaniem extensible [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] aplikacji z Framework Managed Extensibility (MEF). Te przestrzenie nazw są częścią [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] podzestawu dla [!INCLUDE[win8](../../../includes/win8-md.md)] systemu operacyjnego.  
  
 Te obszary nazw nie są częścią podstawowej biblioteki klas rozpowszechnianej za pomocą programu .NET Framework. Aby zainstalować te obszary nazw, otwórz projekt w [!INCLUDE[vs_dev11_long](../../../includes/vs-dev11-long-md.md)], wybierz **Zarządzaj pakietami NuGet** z **projektu** menu, a następnie wyszukaj w trybie online pakiet Microsoft.Composition.  
  
-   <xref:System.Composition?displayProperty=nameWithType> udostępnia klasy, które stanowią podstawę MEF dla [!INCLUDE[win8_appname_long](../../../includes/win8-appname-long-md.md)] aplikacji.  
  
-   <xref:System.Composition.Convention?displayProperty=nameWithType> zawiera typy, które obsługuje używania MEF z modelu opartych na konwencjach konfiguracji.  
  
-   <xref:System.Composition.Hosting?displayProperty=nameWithType> zawiera typy MEF, które są przydatne dla deweloperów aplikacji hosta.  
  
-   <xref:System.Composition.Hosting.Core?displayProperty=nameWithType> zawiera typy MEF używana wewnętrznie przez aparat kompozycji.  
  
 Aby uzyskać więcej informacji na temat [!INCLUDE[net_win8_profile](../../../includes/net-win8-profile-md.md)] i listę obszary nazw i typy, które zawiera, zobacz [Omówienie aplikacji .NET dla Sklepu Windows](http://go.microsoft.com/fwlink/p/?LinkID=238312) w Centrum deweloperów systemu Windows.
