## <a name="implementation-guidelines-and-conventions"></a><span data-ttu-id="d8141-110">Wytyczne i konwencje dotyczące implementacji</span><span class="sxs-lookup"><span data-stu-id="d8141-110">Implementation Guidelines and Conventions</span></span>

<span data-ttu-id="d8141-111">Podczas implementowania wzorca kontrolki Invoke należy zwrócić uwagę na następujące wytyczne i konwencje:</span><span class="sxs-lookup"><span data-stu-id="d8141-111">When implementing the Invoke control pattern, note the following guidelines and conventions:</span></span>

- <span data-ttu-id="d8141-112">Formanty implementują <xref:System.Windows.Automation.Provider.IInvokeProvider> , jeśli takie samo zachowanie nie jest ujawniane za pomocą innego dostawcy wzorca kontroli.</span><span class="sxs-lookup"><span data-stu-id="d8141-112">Controls implement <xref:System.Windows.Automation.Provider.IInvokeProvider> if the same behavior is not exposed through another control pattern provider.</span></span> <span data-ttu-id="d8141-113">Na przykład jeśli <xref:System.Windows.Automation.InvokePattern.Invoke%2A> Metoda kontrolki wykonuje tę samą akcję <xref:System.Windows.Automation.ExpandCollapsePattern.Expand%2A> co metoda lub <xref:System.Windows.Automation.ExpandCollapsePattern.Collapse%2A> , formant nie powinien implementować <xref:System.Windows.Automation.Provider.IInvokeProvider>.</span><span class="sxs-lookup"><span data-stu-id="d8141-113">For example, if the <xref:System.Windows.Automation.InvokePattern.Invoke%2A> method on a control performs the same action as the <xref:System.Windows.Automation.ExpandCollapsePattern.Expand%2A> or <xref:System.Windows.Automation.ExpandCollapsePattern.Collapse%2A> method, the control should not implement <xref:System.Windows.Automation.Provider.IInvokeProvider>.</span></span>

- <span data-ttu-id="d8141-114">Wywoływanie kontrolki jest zazwyczaj wykonywane przez kliknięcie lub dwukrotne kliknięcie lub naciśnięcie klawisza ENTER, wstępnie zdefiniowanego skrótu klawiaturowego lub alternatywnej kombinacji naciśnięć klawiszy.</span><span class="sxs-lookup"><span data-stu-id="d8141-114">Invoking a control is generally performed by clicking or double-clicking or pressing ENTER, a predefined keyboard shortcut, or some alternate combination of keystrokes.</span></span>

  - <span data-ttu-id="d8141-118">Nie jest to możliwe ani praktyczne, aby poczekać na zakończenie działania.</span><span class="sxs-lookup"><span data-stu-id="d8141-118">It is not possible or practical to wait until the action is complete.</span></span>

  - <span data-ttu-id="d8141-119">Akcja wymaga interakcji z użytkownikiem.</span><span class="sxs-lookup"><span data-stu-id="d8141-119">The action requires user interaction.</span></span>

  - <span data-ttu-id="d8141-120">Akcja jest czasochłonna i spowoduje, że klient wywołujący będzie blokował przez znaczący czas.</span><span class="sxs-lookup"><span data-stu-id="d8141-120">The action is time-consuming and will cause the calling client to block for a significant amount of time.</span></span>

- <span data-ttu-id="d8141-121">Jeśli wywołanie formantu ma znaczące efekty uboczne, te efekty uboczne powinny być uwidaczniane przez <xref:System.Windows.Automation.AutomationElement.AutomationElementInformation.HelpText%2A> właściwość.</span><span class="sxs-lookup"><span data-stu-id="d8141-121">If invoking the control has significant side-effects, those side-effects should be exposed through the <xref:System.Windows.Automation.AutomationElement.AutomationElementInformation.HelpText%2A> property.</span></span> <span data-ttu-id="d8141-122">Na przykład mimo że <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> nie jest skojarzony z zaznaczeniem, może to spowodować, <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> że zostanie zaznaczona inna kontrolka.</span><span class="sxs-lookup"><span data-stu-id="d8141-122">For example, even though <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> is not associated with selection, <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> may cause another control to become selected.</span></span>

- <span data-ttu-id="d8141-123">Efekty przemieszczania (lub przełączenia do trybu failover) zwykle nie stanowią wywoływanego zdarzenia.</span><span class="sxs-lookup"><span data-stu-id="d8141-123">Hover (or mouse-over) effects generally do not constitute an Invoked event.</span></span> <span data-ttu-id="d8141-124">Jednak kontrolki, które wykonują akcję (w przeciwieństwie do efektów wizualnych), na podstawie stanu aktywowania powinny obsługiwać <xref:System.Windows.Automation.InvokePattern> wzorzec kontrolki.</span><span class="sxs-lookup"><span data-stu-id="d8141-124">However, controls that perform an action (as opposed to cause a visual effect) based on the hover state should support the <xref:System.Windows.Automation.InvokePattern> control pattern.</span></span>

- <span data-ttu-id="d8141-129">Element może zniknąć od [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] drzewa natychmiast po wywołaniu.</span><span class="sxs-lookup"><span data-stu-id="d8141-129">An element can disappear from the [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] tree immediately upon being invoked.</span></span> <span data-ttu-id="d8141-130">Żądanie informacji z elementu dostarczonego przez wywołanie zwrotne zdarzenia może zakończyć się niepowodzeniem w wyniku.</span><span class="sxs-lookup"><span data-stu-id="d8141-130">Requesting information from the element provided by the event callback may fail as a result.</span></span> <span data-ttu-id="d8141-131">Zalecanym obejściem jest wstępne pobranie informacji w pamięci podręcznej.</span><span class="sxs-lookup"><span data-stu-id="d8141-131">Pre-fetching cached information is the recommended workaround.</span></span>

- <span data-ttu-id="d8141-132">Kontrolki mogą zaimplementować wiele wzorców kontrolek.</span><span class="sxs-lookup"><span data-stu-id="d8141-132">Controls can implement multiple control patterns.</span></span> <span data-ttu-id="d8141-133">Na przykład kontrolka kolor wypełnienia na [!INCLUDE[TLA#tla_xl](../../../includes/tlasharptla-xl-md.md)] pasku narzędzi implementuje <xref:System.Windows.Automation.InvokePattern> zarówno wzorce, jak i <xref:System.Windows.Automation.ExpandCollapsePattern> elementy sterujące.</span><span class="sxs-lookup"><span data-stu-id="d8141-133">For example, the Fill Color control on the [!INCLUDE[TLA#tla_xl](../../../includes/tlasharptla-xl-md.md)] toolbar implements both the <xref:System.Windows.Automation.InvokePattern> and the <xref:System.Windows.Automation.ExpandCollapsePattern> control patterns.</span></span> <span data-ttu-id="d8141-134"><xref:System.Windows.Automation.ExpandCollapsePattern>uwidacznia menu i <xref:System.Windows.Automation.InvokePattern> wypełnia aktywne zaznaczenie wybranym kolorem.</span><span class="sxs-lookup"><span data-stu-id="d8141-134"><xref:System.Windows.Automation.ExpandCollapsePattern> exposes the menu and the <xref:System.Windows.Automation.InvokePattern> fills the active selection with the chosen color.</span></span>
