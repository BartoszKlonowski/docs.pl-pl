 <span data-ttu-id="b34a3-160">Profilowanie usługi systemu Windows jest podobne do profilowania typowych aplikacji środowiska wykonawczego języka.</span><span class="sxs-lookup"><span data-stu-id="b34a3-160">Profiling a Windows Service is like profiling a common language runtime application.</span></span> <span data-ttu-id="b34a3-161">Obie profilowania operacje są włączone w zmiennych środowiskowych.</span><span class="sxs-lookup"><span data-stu-id="b34a3-161">Both profiling operations are enabled through environment variables.</span></span> <span data-ttu-id="b34a3-162">Ponieważ usługa systemu Windows jest uruchomiony, gdy zostanie uruchomiony system operacyjny, zmienne środowiskowe omówionych wcześniej w tym temacie musi już być istnieje i ustaw wartości wymaganych przed uruchomieniem systemu.</span><span class="sxs-lookup"><span data-stu-id="b34a3-162">Because a Windows Service is started when the operating system starts, the environment variables discussed previously in this topic must already be present and set to the required values before the system starts.</span></span> <span data-ttu-id="b34a3-163">Ponadto DLL profilowania musi już być zarejestrowany w systemie.</span><span class="sxs-lookup"><span data-stu-id="b34a3-163">In addition, the profiling DLL must already be registered on the system.</span></span>  
  
 <span data-ttu-id="b34a3-164">Po ustawieniu zmiennych środowiskowych COR_ENABLE_PROFILING i COR_PROFILER i zarejestrować biblioteki DLL profilera, dzięki czemu usługa systemu Windows wykrywa te zmiany należy ponownie uruchomić komputer docelowy.</span><span class="sxs-lookup"><span data-stu-id="b34a3-164">After you set the COR_ENABLE_PROFILING and COR_PROFILER environment variables and register the profiler DLL, you should restart the target computer so that the Windows Service can detect those changes.</span></span>  
  
 <span data-ttu-id="b34a3-165">Należy pamiętać, że te zmiany spowoduje włączenie profilowania na poziomie systemu.</span><span class="sxs-lookup"><span data-stu-id="b34a3-165">Note that these changes will enable profiling on a system-wide basis.</span></span> <span data-ttu-id="b34a3-166">Aby zapobiec co zarządzanych aplikację, która następnie działa z profilowanego, należy usunąć zmiennych środowiskowych systemu po ponownym uruchomieniu komputera docelowego.</span><span class="sxs-lookup"><span data-stu-id="b34a3-166">To prevent every managed application that subsequently runs from being profiled, you should delete the system environment variables after you restart the target computer.</span></span>  
  
 <span data-ttu-id="b34a3-167">Ta technika również prowadzi do każdego procesu CLR pobierania profilowaniu.</span><span class="sxs-lookup"><span data-stu-id="b34a3-167">This technique also leads to every CLR process getting profiled.</span></span> <span data-ttu-id="b34a3-168">Profiler należy dodać logikę do swojej [ICorProfilerCallback::Initialize](../../../../docs/framework/unmanaged-api/profiling/icorprofilercallback-initialize-method.md) wywołanie zwrotne, aby wykryć, czy bieżący proces jest przydatne.</span><span class="sxs-lookup"><span data-stu-id="b34a3-168">The profiler should add logic to its [ICorProfilerCallback::Initialize](../../../../docs/framework/unmanaged-api/profiling/icorprofilercallback-initialize-method.md) callback to detect whether the current process is of interest.</span></span> <span data-ttu-id="b34a3-169">Jeśli nie, profiler może zakończyć się niepowodzeniem wywołania zwrotnego bez wykonywania inicjowania.</span><span class="sxs-lookup"><span data-stu-id="b34a3-169">If it is not, the profiler can fail the callback without performing the initialization.</span></span>
