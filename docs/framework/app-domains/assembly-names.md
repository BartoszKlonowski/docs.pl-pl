# <a name="assembly-names"></a><span data-ttu-id="54356-102">Nazwy zestawów</span><span class="sxs-lookup"><span data-stu-id="54356-102">Assembly Names</span></span>
<span data-ttu-id="54356-103">Nazwa zestawu jest przechowywany w metadanych i ma znaczny wpływ na zakres zestawu i użycia przez aplikację.</span><span class="sxs-lookup"><span data-stu-id="54356-103">An assembly's name is stored in metadata and has a significant impact on the assembly's scope and use by an application.</span></span> <span data-ttu-id="54356-104">Zestawu z silną nazwą ma w pełni kwalifikowana nazwa, która zawiera nazwę zestawu, kultury, klucz publiczny i numer wersji.</span><span class="sxs-lookup"><span data-stu-id="54356-104">A strong-named assembly has a fully qualified name that includes the assembly's name, culture, public key, and version number.</span></span> <span data-ttu-id="54356-105">Jest to często określane jako wyświetlaną nazwę i dla zestawów załadowanych można uzyskać za pomocą <xref:System.Reflection.Assembly.FullName%2A> właściwości.</span><span class="sxs-lookup"><span data-stu-id="54356-105">This is frequently referred to as the display name, and for loaded assemblies can be obtained by using the <xref:System.Reflection.Assembly.FullName%2A> property.</span></span>  
  
 <span data-ttu-id="54356-106">Środowisko wykonawcze używa tych informacji, aby zlokalizować zestawu i odróżnić go od innych zestawów o takiej samej nazwie.</span><span class="sxs-lookup"><span data-stu-id="54356-106">The runtime uses this information to locate the assembly and differentiate it from other assemblies with the same name.</span></span> <span data-ttu-id="54356-107">Na przykład o nazwie zestawu z silną nazwą `myTypes` może mieć następującej nazwy FQDN:</span><span class="sxs-lookup"><span data-stu-id="54356-107">For example, a strong-named assembly called `myTypes` could have the following fully qualified name:</span></span>  
  
```  
myTypes, Version=1.0.1234.0, Culture=en-US, PublicKeyToken=b77a5c561934e089c, ProcessorArchitecture=msil  
```  
  
> [!NOTE]
>  <span data-ttu-id="54356-108">Architektura procesora jest dodawana do tożsamości zestawu w .NET Framework w wersji 2.0, aby umożliwić specyficznych dla procesora wersje zestawów.</span><span class="sxs-lookup"><span data-stu-id="54356-108">Processor architecture is added to the assembly identity in the .NET Framework version 2.0, to allow processor-specific versions of assemblies.</span></span> <span data-ttu-id="54356-109">Można utworzyć wersji zestawu, którego tożsamość różni się tylko architektury procesora, na przykład 32-bitowych i 64-bitowych wersjach specyficznych dla procesora.</span><span class="sxs-lookup"><span data-stu-id="54356-109">You can create versions of an assembly whose identity differs only by processor architecture, for example 32-bit and 64-bit processor-specific versions.</span></span> <span data-ttu-id="54356-110">Architektura procesora nie jest wymagane silne nazwy.</span><span class="sxs-lookup"><span data-stu-id="54356-110">Processor architecture is not required for strong names.</span></span> <span data-ttu-id="54356-111">Aby uzyskać więcej informacji, zobacz <xref:System.Reflection.AssemblyName.ProcessorArchitecture%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="54356-111">For more information, see <xref:System.Reflection.AssemblyName.ProcessorArchitecture%2A?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="54356-112">W tym przykładzie w pełni kwalifikowana nazwa wskazuje, że `myTypes` zestaw z silną nazwą przy użyciu tokenu klucza publicznego, ma wartość kultury dla tego języka i numerem wersji 1.0.1234.0.</span><span class="sxs-lookup"><span data-stu-id="54356-112">In this example, the fully qualified name indicates that the `myTypes` assembly has a strong name with a public key token, has the culture value for US English, and has a version number of 1.0.1234.0.</span></span> <span data-ttu-id="54356-113">Jego architektura procesora jest "msil", co oznacza, że będzie on just-in-time (JIT)-kompilowane do kodu 32-bitowego lub 64-bitowego kodu w zależności od systemu operacyjnego i procesora.</span><span class="sxs-lookup"><span data-stu-id="54356-113">Its processor architecture is "msil", which means that it will be just-in-time (JIT)-compiled to 32-bit code or 64-bit code depending on the operating system and processor.</span></span>  
  
 <span data-ttu-id="54356-114">Kod, który żąda typy w zestawie, należy użyć w pełni kwalifikowanej nazwy zestawu.</span><span class="sxs-lookup"><span data-stu-id="54356-114">Code that requests types in an assembly must use a fully qualified assembly name.</span></span> <span data-ttu-id="54356-115">Jest to powiązanie w pełni kwalifikowana.</span><span class="sxs-lookup"><span data-stu-id="54356-115">This is called fully qualified binding.</span></span> <span data-ttu-id="54356-116">Częściowe powiązania, który określa nazwę zestawu, nie jest dozwolone podczas odwoływania się do zestawów .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="54356-116">Partial binding, which specifies only an assembly name, is not permitted when referencing assemblies in the .NET Framework.</span></span>  
  
 <span data-ttu-id="54356-117">Wszystkie odwołania do zestawów do zestawów, które składają się na programie .NET Framework również musi zawierać w pełni kwalifikowana nazwa zestawu.</span><span class="sxs-lookup"><span data-stu-id="54356-117">All assembly references to assemblies that make up the .NET Framework also must contain a fully qualified name of the assembly.</span></span> <span data-ttu-id="54356-118">Na przykład aby odwoływać się do zestawu System.Data .NET Framework w wersji 1.0 zadania obejmują:</span><span class="sxs-lookup"><span data-stu-id="54356-118">For example, to reference the System.Data .NET Framework assembly for version 1.0 would include:</span></span>  
  
```  
System.data, version=1.0.3300.0, Culture=neutral, PublicKeyToken=b77a5c561934e089  
```  
  
 <span data-ttu-id="54356-119">Należy pamiętać, że wersja odpowiada numer wersji wszystkich zestawów .NET Framework, które są dostarczane z programem .NET Framework w wersji 1.0.</span><span class="sxs-lookup"><span data-stu-id="54356-119">Note that the version corresponds to the version number of all .NET Framework assemblies that shipped with .NET Framework version 1.0.</span></span> <span data-ttu-id="54356-120">Dla zestawów .NET Framework wartość kultury jest zawsze neutralne i klucz publiczny jest taki sam, jak pokazano w powyższym przykładzie.</span><span class="sxs-lookup"><span data-stu-id="54356-120">For .NET Framework assemblies, the culture value is always neutral, and the public key is the same as shown in the above example.</span></span>  
  
 <span data-ttu-id="54356-121">Na przykład aby dodać odwołanie do zestawu w pliku konfiguracji, aby skonfigurować odbiornik śledzenia, należy dołączyć w pełni kwalifikowaną nazwę systemu zestawu .NET Framework:</span><span class="sxs-lookup"><span data-stu-id="54356-121">For example, to add an assembly reference in a configuration file to set up a trace listener, you would include the fully qualified name of the system .NET Framework assembly:</span></span>  
  
```xml  
<add name="myListener" type="System.Diagnostics.TextWriterTraceListener, System, Version=1.0.3300.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" initializeData="c:\myListener.log" />  
```  
  
> [!NOTE]
>  <span data-ttu-id="54356-122">Środowisko uruchomieniowe traktuje nazwy zestawu jako bez uwzględniania wielkości liter, podczas tworzenia wiązania do zestawu, ale zachowuje, niezależnie od przypadku jest używany w nazwie zestawu.</span><span class="sxs-lookup"><span data-stu-id="54356-122">The runtime treats assembly names as case-insensitive when binding to an assembly, but preserves whatever case is used in an assembly name.</span></span> <span data-ttu-id="54356-123">Niektóre narzędzia [!INCLUDE[winsdklong](../../../includes/winsdklong-md.md)] obsługi nazw zestawów jako wielkość liter.</span><span class="sxs-lookup"><span data-stu-id="54356-123">Several tools in the [!INCLUDE[winsdklong](../../../includes/winsdklong-md.md)] handle assembly names as case-sensitive.</span></span> <span data-ttu-id="54356-124">Aby uzyskać najlepsze wyniki Zarządzanie nazw zestawów tak, jakby były uwzględniana wielkość liter.</span><span class="sxs-lookup"><span data-stu-id="54356-124">For best results, manage assembly names as though they were case-sensitive.</span></span>  
  
## <a name="naming-application-components"></a><span data-ttu-id="54356-125">Nadawanie nazw składników aplikacji</span><span class="sxs-lookup"><span data-stu-id="54356-125">Naming Application Components</span></span>  
 <span data-ttu-id="54356-126">Środowisko wykonawcze nie należy wziąć pod uwagę nazwę pliku podczas ustalania tożsamości zestawu.</span><span class="sxs-lookup"><span data-stu-id="54356-126">The runtime does not consider the file name when determining an assembly's identity.</span></span> <span data-ttu-id="54356-127">Tożsamość zestawu, który składa się z nazwy zestawu, wersji, kultury i silnej nazwy, musi być czysty do środowiska uruchomieniowego.</span><span class="sxs-lookup"><span data-stu-id="54356-127">The assembly identity, which consists of the assembly name, version, culture, and strong name, must be clear to the runtime.</span></span>  
  
 <span data-ttu-id="54356-128">Na przykład jeśli zestaw o nazwie myAssembly.exe odwołujący się do zestawu o nazwie myAssembly.dll, powiązanie występuje poprawnie jeśli wykonywane myAssembly.exe.</span><span class="sxs-lookup"><span data-stu-id="54356-128">For example, if you have an assembly called myAssembly.exe that references an assembly called myAssembly.dll, binding occurs correctly if you execute myAssembly.exe.</span></span> <span data-ttu-id="54356-129">Jednakże jeśli inna aplikacja wykonuje myAssembly.exe przy użyciu metody <xref:System.AppDomain.ExecuteAssembly%2A?displayProperty=nameWithType>, środowisko wykonawcze określa, że "myAssembly" jest już załadowana w momencie myAssembly.exe żądania powiązania "myAssembly."</span><span class="sxs-lookup"><span data-stu-id="54356-129">However, if another application executes myAssembly.exe using the method <xref:System.AppDomain.ExecuteAssembly%2A?displayProperty=nameWithType>, the runtime determines that "myAssembly" is already loaded when myAssembly.exe requests binding to "myAssembly."</span></span> <span data-ttu-id="54356-130">W tym przypadku myAssembly.dll nigdy nie został załadowany.</span><span class="sxs-lookup"><span data-stu-id="54356-130">In this case, myAssembly.dll is never loaded.</span></span> <span data-ttu-id="54356-131">Ponieważ myAssembly.exe nie zawiera żądanego typu <xref:System.TypeLoadException> występuje.</span><span class="sxs-lookup"><span data-stu-id="54356-131">Because myAssembly.exe does not contain the requested type, a <xref:System.TypeLoadException> occurs.</span></span>  
  
 <span data-ttu-id="54356-132">Aby uniknąć tego problemu, upewnij się, nie mają taką samą nazwę zestawu lub umieść zestawów o takiej samej nazwie w różnych katalogach zestawów, które składają się na aplikację.</span><span class="sxs-lookup"><span data-stu-id="54356-132">To avoid this problem, make sure the assemblies that make up your application do not have the same assembly name or place assemblies with the same name in different directories.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="54356-133">Jeśli zestaw o silnej nazwie zostanie umieszczone w globalnej pamięci podręcznej, nazwa pliku zestawu musi być zgodna z nazwy zestawu (nie w tym rozszerzenie nazwy pliku, na przykład .exe lub .dll).</span><span class="sxs-lookup"><span data-stu-id="54356-133">If you put a strong-named assembly in the global assembly cache, the assembly's file name must match the assembly name (not including the file name extension, such as .exe or .dll).</span></span> <span data-ttu-id="54356-134">Na przykład jeśli nazwa pliku zestawu jest myAssembly.dll, nazwa zestawu musi być myAssembly.</span><span class="sxs-lookup"><span data-stu-id="54356-134">For example, if the file name of an assembly is myAssembly.dll, the assembly name must be myAssembly.</span></span> <span data-ttu-id="54356-135">Zestawy prywatne wdrożone tylko w katalogu głównym aplikacji może mieć nazwy zestawu, który różni się od nazwy pliku.</span><span class="sxs-lookup"><span data-stu-id="54356-135">Private assemblies deployed only in the root application directory can have an assembly name that is different from the file name.</span></span>
