<span data-ttu-id="b6220-103">Zestawy są blokami konstrukcyjnymi aplikacji .NET Framework; tworzą one podstawową jednostką wdrażania, kontroli wersji, ponownego użycia, określania zakresu aktywacji i uprawnień zabezpieczeń.</span><span class="sxs-lookup"><span data-stu-id="b6220-103">Assemblies are the building blocks of .NET Framework applications; they form the fundamental unit of deployment, version control, reuse, activation scoping, and security permissions.</span></span> <span data-ttu-id="b6220-104">Zestaw jest kolekcją typów i zasobów, które zostały opracowane w celu współpracują ze sobą i tworzą jednostkę logiczną funkcji.</span><span class="sxs-lookup"><span data-stu-id="b6220-104">An assembly is a collection of types and resources that are built to work together and form a logical unit of functionality.</span></span> <span data-ttu-id="b6220-105">Zestaw zawiera środowisko uruchomieniowe języka wspólnego informacje potrzebne do należy pamiętać o implementacji typu.</span><span class="sxs-lookup"><span data-stu-id="b6220-105">An assembly provides the common language runtime with the information it needs to be aware of type implementations.</span></span> <span data-ttu-id="b6220-106">Do środowiska uruchomieniowego typem nie istnieje poza kontekstem zestawu.</span><span class="sxs-lookup"><span data-stu-id="b6220-106">To the runtime, a type does not exist outside the context of an assembly.</span></span>  
  
 <span data-ttu-id="b6220-107">Zespół wykonuje następujące funkcje:</span><span class="sxs-lookup"><span data-stu-id="b6220-107">An assembly performs the following functions:</span></span>

- <span data-ttu-id="b6220-117">Wchodzi w skład granice zakresu odwołań.</span><span class="sxs-lookup"><span data-stu-id="b6220-117">It forms a reference scope boundary.</span></span> <span data-ttu-id="b6220-118">Manifest zestawu zawiera metadane zestawu jest używana do rozpoznawania typów i spełniającej żądania zasobu.</span><span class="sxs-lookup"><span data-stu-id="b6220-118">The assembly's manifest contains assembly metadata that is used for resolving types and satisfying resource requests.</span></span> <span data-ttu-id="b6220-119">Określa typy i zasoby, które są udostępniane poza zestawem.</span><span class="sxs-lookup"><span data-stu-id="b6220-119">It specifies the types and resources that are exposed outside the assembly.</span></span> <span data-ttu-id="b6220-120">Manifest również wylicza inne zestawy, od których zależy.</span><span class="sxs-lookup"><span data-stu-id="b6220-120">The manifest also enumerates other assemblies on which it depends.</span></span>  
  
- <span data-ttu-id="b6220-121">Stanowi on granicy wersji.</span><span class="sxs-lookup"><span data-stu-id="b6220-121">It forms a version boundary.</span></span> <span data-ttu-id="b6220-122">Zestaw jest najmniejszego obsługą wersji środowiska uruchomieniowego języka wspólnego; wszystkie typy i zasoby z tego samego zestawu są wersjonowane jako jednostka.</span><span class="sxs-lookup"><span data-stu-id="b6220-122">The assembly is the smallest versionable unit in the common language runtime; all types and resources in the same assembly are versioned as a unit.</span></span> <span data-ttu-id="b6220-123">Manifest zestawu zawiera opis wersji zależności, o których należy określić wszystkie zależne zestawy.</span><span class="sxs-lookup"><span data-stu-id="b6220-123">The assembly's manifest describes the version dependencies you specify for any dependent assemblies.</span></span> <span data-ttu-id="b6220-124">Aby uzyskać więcej informacji dotyczących wersjonowania, zobacz [przechowywanie wersji zestawu](../../../docs/framework/app-domains/assembly-versioning.md).</span><span class="sxs-lookup"><span data-stu-id="b6220-124">For more information about versioning, see [Assembly Versioning](../../../docs/framework/app-domains/assembly-versioning.md).</span></span>  
  
- <span data-ttu-id="b6220-125">Wchodzi w skład jednostki wdrożenia.</span><span class="sxs-lookup"><span data-stu-id="b6220-125">It forms a deployment unit.</span></span> <span data-ttu-id="b6220-126">Po uruchomieniu aplikacji tylko zestawy, które początkowo wywołuje aplikacji musi być obecny.</span><span class="sxs-lookup"><span data-stu-id="b6220-126">When an application starts, only the assemblies that the application initially calls must be present.</span></span> <span data-ttu-id="b6220-127">Inne zestawy, takie jak zasoby lokalizacyjne lub zestawów zawierających klasy narzędzi mogą być pobierane na żądanie.</span><span class="sxs-lookup"><span data-stu-id="b6220-127">Other assemblies, such as localization resources or assemblies containing utility classes, can be retrieved on demand.</span></span> <span data-ttu-id="b6220-128">Dzięki temu aplikacje były proste i alokowania elastycznego, gdy pobrane po raz pierwszy.</span><span class="sxs-lookup"><span data-stu-id="b6220-128">This allows applications to be kept simple and thin when first downloaded.</span></span> <span data-ttu-id="b6220-129">Aby uzyskać więcej informacji o wdrażaniu zestawów, zobacz [wdrażanie aplikacji](../../../docs/framework/deployment/index.md).</span><span class="sxs-lookup"><span data-stu-id="b6220-129">For more information about deploying assemblies, see [Deploying Applications](../../../docs/framework/deployment/index.md).</span></span>
