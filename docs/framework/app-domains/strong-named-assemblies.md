## <a name="why-strong-name-your-assemblies"></a><span data-ttu-id="bec0c-113">Dlaczego silnej nazwy zestawów?</span><span class="sxs-lookup"><span data-stu-id="bec0c-113">Why strong-name your assemblies?</span></span>  
 <span data-ttu-id="bec0c-114">Gdy odwołujesz się zestawu z silną nazwą, można oczekiwać, że niektóre korzyści, takie jak przechowywanie wersji i nazewnictwa ochrony.</span><span class="sxs-lookup"><span data-stu-id="bec0c-114">When you reference a strong-named assembly, you can expect certain benefits, such as versioning and naming protection.</span></span> <span data-ttu-id="bec0c-115">Zestawy o silnych nazwach można zainstalować w globalnej pamięci podręcznej zestawów, który jest wymagany do obsługi niektórych scenariuszy.</span><span class="sxs-lookup"><span data-stu-id="bec0c-115">Strong-named assemblies can be installed in the Global Assembly Cache, which is required to enable some scenarios.</span></span>  
  
 <span data-ttu-id="bec0c-116">Zestawy o silnych nazwach są przydatne w następujących scenariuszach:</span><span class="sxs-lookup"><span data-stu-id="bec0c-116">Strong-named assemblies are useful in the following scenarios:</span></span>  
  
-   <span data-ttu-id="bec0c-117">Aby włączyć swoje zestawy odwoływać się zestawów o silnych nazwach lub chcesz nadać `friend` dostęp do Twoich zestawów od innych zestawów o silnych nazwach.</span><span class="sxs-lookup"><span data-stu-id="bec0c-117">You want to enable your assemblies to be referenced by strong-named assemblies, or you want to give `friend` access to your assemblies from other strong-named assemblies.</span></span>  
  
-   <span data-ttu-id="bec0c-118">Aplikacja potrzebuje dostępu do różnych wersji tego samego zestawu.</span><span class="sxs-lookup"><span data-stu-id="bec0c-118">An app needs access to different versions of the same assembly.</span></span> <span data-ttu-id="bec0c-119">Oznacza to, że potrzebujesz różnych wersji zestawu do załadowania obok siebie w tej samej domenie aplikacji, bez powodowania konfliktów.</span><span class="sxs-lookup"><span data-stu-id="bec0c-119">This means  you need different versions of an assembly to load side by side in the same app domain without conflict.</span></span> <span data-ttu-id="bec0c-120">Na przykład jeśli istnieją inne rozszerzenia interfejsu API w zestawach, które mają tej samej prostej nazwie, silne nazwy zapewnia unikatową tożsamość dla każdej wersji zestawu.</span><span class="sxs-lookup"><span data-stu-id="bec0c-120">For example, if different extensions of an API exist in assemblies that have the same simple name, strong-naming provides a unique identity for each version of the assembly.</span></span>  
  
-   <span data-ttu-id="bec0c-121">Czy chcesz mieć negatywny wpływ na wydajność aplikacji przy użyciu swoim zestawie, więc chcesz zestawie, który ma być niezależne od domeny.</span><span class="sxs-lookup"><span data-stu-id="bec0c-121">You do not want to negatively affect performance of apps using your assembly, so you want the assembly to be domain neutral.</span></span> <span data-ttu-id="bec0c-122">Wymaga to silne nazwy, ponieważ zestaw niezależne od domeny musi być zainstalowany w globalnej pamięci podręcznej.</span><span class="sxs-lookup"><span data-stu-id="bec0c-122">This requires strong-naming because a domain-neutral assembly must be installed in the global assembly cache.</span></span>  
  
-   <span data-ttu-id="bec0c-123">Jeśli chcesz scentralizować obsługi dla aplikacji, stosując zasady wydawcy, oznacza to zestaw musi być zainstalowany w globalnej pamięci podręcznej.</span><span class="sxs-lookup"><span data-stu-id="bec0c-123">When you want to centralize servicing for your app by applying publisher policy, which means the assembly must be installed in the  global assembly cache.</span></span>  
  
 <span data-ttu-id="bec0c-124">Jeśli jesteś deweloperem typu open source i korzystać z zalet tożsamości zestawu z silną nazwą, należy wziąć pod uwagę ewidencjonowanie klucz prywatny skojarzone z zestawem w systemie kontroli źródła.</span><span class="sxs-lookup"><span data-stu-id="bec0c-124">If you are an open-source developer and you want the identity benefits of a strong-named assembly, consider checking in the private key associated with an assembly into your source control system.</span></span>
