## <a name="what-makes-a-strong-named-assembly"></a><span data-ttu-id="aaff5-104">Dzięki czemu zestawu z silną nazwą?</span><span class="sxs-lookup"><span data-stu-id="aaff5-104">What makes a strong-named assembly?</span></span>  
 <span data-ttu-id="aaff5-105">Zestaw jest generowany przy użyciu klucza prywatnego, który odpowiada kluczowi publicznemu rozpowszechnianej za pomocą zestawu i zestawu.</span><span class="sxs-lookup"><span data-stu-id="aaff5-105">A strong named assembly is generated by using the private key that corresponds to the public key distributed with the assembly, and the assembly itself.</span></span> <span data-ttu-id="aaff5-106">Zestaw zawiera manifest zestawu, który zawiera nazwy i wartości skrótu wszystkie pliki wchodzące w skład zestawu.</span><span class="sxs-lookup"><span data-stu-id="aaff5-106">The assembly includes the assembly manifest, which contains the names and hashes of all the files that make up the assembly.</span></span> <span data-ttu-id="aaff5-107">Zestawy, które mają taką samą nazwę silne powinny być identyczne.</span><span class="sxs-lookup"><span data-stu-id="aaff5-107">Assemblies that have the same strong name should be identical.</span></span>

 <span data-ttu-id="aaff5-110">Po utworzeniu zestawu z silną nazwą zawiera zwykły tekst nazwę zestawu, numer wersji informacji o kulturze opcjonalne, podpisu cyfrowego i klucz publiczny, który odpowiada klucz prywatny używany do podpisywania.</span><span class="sxs-lookup"><span data-stu-id="aaff5-110">When a strong-named assembly is created, it contains the simple text name of the assembly, the version number, optional culture information, a digital signature, and the public key that corresponds to the private key used for signing.</span></span>

## <a name="why-strong-name-your-assemblies"></a><span data-ttu-id="aaff5-113">Dlaczego silnej nazwy zestawów użytkownika?</span><span class="sxs-lookup"><span data-stu-id="aaff5-113">Why strong-name your assemblies?</span></span>  
 <span data-ttu-id="aaff5-114">W przypadku odwołania zestawu z silną nazwą, można oczekiwać, że niektóre korzyści, takich jak przechowywanie wersji i nazewnictwa ochrony.</span><span class="sxs-lookup"><span data-stu-id="aaff5-114">When you reference a strong-named assembly, you can expect certain benefits, such as versioning and naming protection.</span></span> <span data-ttu-id="aaff5-115">Zestawy o silnych nazwach można zainstalować w globalnej pamięci podręcznej zestawów, który jest wymagany na potrzeby niektórych scenariuszy.</span><span class="sxs-lookup"><span data-stu-id="aaff5-115">Strong-named assemblies can be installed in the Global Assembly Cache, which is required to enable some scenarios.</span></span>  
  
 <span data-ttu-id="aaff5-116">Zestawy o silnych nazwach są przydatne w następujących scenariuszach:</span><span class="sxs-lookup"><span data-stu-id="aaff5-116">Strong-named assemblies are useful in the following scenarios:</span></span>  
  
-   <span data-ttu-id="aaff5-117">Aby włączyć ponownie zestawy można odwoływać się zestawy o silnych nazwach lub chcesz nadać `friend` dostęp do Twojego zestawy od innych zestawów o silnych nazwach.</span><span class="sxs-lookup"><span data-stu-id="aaff5-117">You want to enable your assemblies to be referenced by strong-named assemblies, or you want to give `friend` access to your assemblies from other strong-named assemblies.</span></span>  
  
-   <span data-ttu-id="aaff5-118">Aplikacja potrzebuje dostępu do różnych wersji tego samego zestawu.</span><span class="sxs-lookup"><span data-stu-id="aaff5-118">An app needs access to different versions of the same assembly.</span></span> <span data-ttu-id="aaff5-119">Oznacza to, że należy różnych wersji zestawu do załadowania obok siebie w tej samej domenie aplikacji bez konfliktu.</span><span class="sxs-lookup"><span data-stu-id="aaff5-119">This means  you need different versions of an assembly to load side by side in the same app domain without conflict.</span></span> <span data-ttu-id="aaff5-120">Na przykład jeśli istnieją inne rozszerzenia interfejsu API w zestawy, które mają taką samą prostą nazwę, silne nazwy zapewnia unikatową tożsamość dla każdej wersji zestawu.</span><span class="sxs-lookup"><span data-stu-id="aaff5-120">For example, if different extensions of an API exist in assemblies that have the same simple name, strong-naming provides a unique identity for each version of the assembly.</span></span>  
  
-   <span data-ttu-id="aaff5-121">Czy chcesz mieć negatywny wpływ na wydajność aplikacji, korzystając z zestawu tak ma zestaw jako neutralny dla domen.</span><span class="sxs-lookup"><span data-stu-id="aaff5-121">You do not want to negatively affect performance of apps using your assembly, so you want the assembly to be domain neutral.</span></span> <span data-ttu-id="aaff5-122">Wymaga to silne nazwy zestawu neutralne dla domen musi zostać zainstalowany w globalnej pamięci podręcznej zestawów.</span><span class="sxs-lookup"><span data-stu-id="aaff5-122">This requires strong-naming because a domain-neutral assembly must be installed in the global assembly cache.</span></span>  
  
-   <span data-ttu-id="aaff5-123">Umożliwia scentralizowane obsługi dla aplikacji, stosując zasady wydawcy, co oznacza zestawu musi być zainstalowany w globalnej pamięci podręcznej zestawów.</span><span class="sxs-lookup"><span data-stu-id="aaff5-123">When you want to centralize servicing for your app by applying publisher policy, which means the assembly must be installed in the  global assembly cache.</span></span>  
  
 <span data-ttu-id="aaff5-124">Jeśli jesteś deweloperem open source i korzystać z zalet tożsamości zestawu z silną nazwą, należy wziąć pod uwagę ewidencjonowanie klucz prywatny skojarzony z zestawem do systemu kontroli źródła.</span><span class="sxs-lookup"><span data-stu-id="aaff5-124">If you are an open-source developer and you want the identity benefits of a strong-named assembly, consider checking in the private key associated with an assembly into your source control system.</span></span>
