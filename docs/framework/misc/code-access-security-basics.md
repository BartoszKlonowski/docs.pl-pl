## <a name="using-managed-wrapper-classes"></a><span data-ttu-id="901b9-178">Używanie zarządzanych klas otoki</span><span class="sxs-lookup"><span data-stu-id="901b9-178">Using Managed Wrapper Classes</span></span>

<span data-ttu-id="901b9-179">Większość aplikacji i składników (z wyjątkiem bezpiecznych bibliotek) nie należy bezpośrednio wywoływać kod niezarządzany.</span><span class="sxs-lookup"><span data-stu-id="901b9-179">Most applications and components (except secure libraries) should not directly call unmanaged code.</span></span> <span data-ttu-id="901b9-180">Jest kilka powodów.</span><span class="sxs-lookup"><span data-stu-id="901b9-180">There are several reasons for this.</span></span> <span data-ttu-id="901b9-181">Jeśli kod wywołuje kod niezarządzany bezpośrednio, nie będzie można uruchomić w wielu okolicznościach, ponieważ kod musi być przyznawany wysoki poziom zaufania do wywołania kodu macierzystego.</span><span class="sxs-lookup"><span data-stu-id="901b9-181">If code calls unmanaged code directly, it will not be allowed to run in many circumstances because code must be granted a high level of trust to call native code.</span></span> <span data-ttu-id="901b9-182">Jeśli zasady są modyfikowane w celu umożliwienia uruchamiania takiej aplikacji, może znacznie osłabić bezpieczeństwo systemu, pozostawiając aplikacji swobodę wykonywania prawie każdej operacji.</span><span class="sxs-lookup"><span data-stu-id="901b9-182">If policy is modified to allow such an application to run, it can significantly weaken the security of the system, leaving the application free to perform almost any operation.</span></span>
