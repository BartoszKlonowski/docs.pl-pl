- <span data-ttu-id="c91dd-113">Domyślnie biblioteki udostępnione [poziomu 1](security-transparent-code-level-1.md) o silnej nazwie mają niejawny [LinkDemand](link-demands.md) dla pełnego zaufania, bez konieczności wykonywania jakichkolwiek czynności przez moduł zapisujący biblioteki.</span><span class="sxs-lookup"><span data-stu-id="c91dd-113">By default, strong-named [Level 1](security-transparent-code-level-1.md) shared libraries perform an implicit [LinkDemand](link-demands.md) for full trust automatically, without the library writer having to do anything.</span></span>  
  
- <span data-ttu-id="c91dd-114">Jeśli obiekt wywołujący nie ma pełnego zaufania, ale nadal próbuje wywołać taką bibliotekę, środowisko uruchomieniowe zgłosi <xref:System.Security.SecurityException> i obiekt wywołujący nie będzie mógł połączyć się z biblioteką.</span><span class="sxs-lookup"><span data-stu-id="c91dd-114">If a caller does not have full trust but still tries to call such a library, the runtime throws a <xref:System.Security.SecurityException> and the caller is not allowed to link to the library.</span></span>  
  
- <span data-ttu-id="c91dd-115">Aby wyłączyć automatyczne **LinkDemand** i zapobiec zgłaszaniu wyjątku, można umieścić atrybut **AllowPartiallyTrustedCallersAttribute** w zakresie zestawu biblioteki udostępnionej.</span><span class="sxs-lookup"><span data-stu-id="c91dd-115">In order to disable the automatic **LinkDemand** and prevent the exception from being thrown, you can place the **AllowPartiallyTrustedCallersAttribute** attribute on the assembly scope of a shared library.</span></span> <span data-ttu-id="c91dd-116">Ten atrybut umożliwia wywoływanie bibliotek z częściowo zaufanego kodu zarządzanego.</span><span class="sxs-lookup"><span data-stu-id="c91dd-116">This attribute allows your libraries to be called from partially trusted managed code.</span></span>  
  
- <span data-ttu-id="c91dd-117">Częściowo zaufany kod, który ma przyznane dostęp do biblioteki z tym atrybutem nadal podlega dalszym ograniczeniom zdefiniowanym przez <xref:System.AppDomain>.</span><span class="sxs-lookup"><span data-stu-id="c91dd-117">Partially trusted code that is granted access to a library with this attribute is still subject to further restrictions defined by the <xref:System.AppDomain>.</span></span>  
  
- <span data-ttu-id="c91dd-118">Nie istnieje programistyczny sposób, aby częściowo zaufany kod wywoływał bibliotekę, która nie ma atrybutu **AllowPartiallyTrustedCallersAttribute** .</span><span class="sxs-lookup"><span data-stu-id="c91dd-118">There is no programmatic way for partially trusted code to call a library that does not have the **AllowPartiallyTrustedCallersAttribute** attribute.</span></span>  
  
 <span data-ttu-id="c91dd-119">Biblioteki, które są prywatne dla określonej aplikacji, nie wymagają silnej nazwy lub atrybutu **AllowPartiallyTrustedCallersAttribute** i nie mogą odwoływać się do potencjalnie złośliwego kodu poza aplikacją.</span><span class="sxs-lookup"><span data-stu-id="c91dd-119">Libraries that are private to a specific application do not require a strong name or the **AllowPartiallyTrustedCallersAttribute** attribute and cannot be referenced by potentially malicious code outside the application.</span></span> <span data-ttu-id="c91dd-120">Taki kod jest chroniony przed zamierzonym lub przypadkowym nadużyciem przez częściowo zaufany kod mobilny bez konieczności wykonywania dodatkowych czynności przez dewelopera.</span><span class="sxs-lookup"><span data-stu-id="c91dd-120">Such code is protected against intentional or unintentional misuse by partially trusted mobile code without the developer having to do anything extra.</span></span>  
  
 <span data-ttu-id="c91dd-121">Należy rozważyć jawne włączenie korzystania przez częściowo zaufany kod dla następujących typów kodu:</span><span class="sxs-lookup"><span data-stu-id="c91dd-121">You should consider explicitly enabling use by partially trusted code for the following types of code:</span></span>  
  
- <span data-ttu-id="c91dd-122">Kod, który został pracujemy testowany pod kątem luk w zabezpieczeniach i jest zgodny z wytycznymi opisanymi w temacie [zasady bezpiecznego kodowania](../../standard/security/secure-coding-guidelines.md).</span><span class="sxs-lookup"><span data-stu-id="c91dd-122">Code that has been diligently tested for security vulnerabilities and is in compliance with the guidelines described in [Secure Coding Guidelines](../../standard/security/secure-coding-guidelines.md).</span></span>  
  
- <span data-ttu-id="c91dd-123">Biblioteki kodu o silnej nazwie, które są przeznaczone dla częściowo zaufanych scenariuszy.</span><span class="sxs-lookup"><span data-stu-id="c91dd-123">Strong-named code libraries that are specifically written for partially trusted scenarios.</span></span>  
  
- <span data-ttu-id="c91dd-124">Wszystkie składniki (częściowo lub w pełni zaufane) podpisane przy użyciu silnej nazwy, które będą wywoływane przez kod pobrany z Internetu.</span><span class="sxs-lookup"><span data-stu-id="c91dd-124">Any components (whether partially or fully trusted) signed with a strong name that will be called by code that is downloaded from the Internet.</span></span>
