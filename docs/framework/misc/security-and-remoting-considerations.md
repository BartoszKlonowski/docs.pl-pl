## <a name="protected-objects"></a><span data-ttu-id="4f150-110">Chronionych obiektów</span><span class="sxs-lookup"><span data-stu-id="4f150-110">Protected objects</span></span>

 <span data-ttu-id="4f150-118">Aby wyizolować kodu w zarządzanych środowiskach hostingu, jest często stosowanym rozwiązaniem generowanie wielu domen aplikacji podrzędnych z zasad jawnych zmniejszenie poziomy uprawnień dla różnych zestawów.</span><span class="sxs-lookup"><span data-stu-id="4f150-118">To isolate code in managed hosting environments, it is common to generate multiple child application domains with explicit policy reducing the permission levels for various assemblies.</span></span> <span data-ttu-id="4f150-119">Jednak zasady dla tych zestawów pozostaje bez zmian w domyślnej domeny aplikacji.</span><span class="sxs-lookup"><span data-stu-id="4f150-119">However, the policy for those assemblies remains unchanged in the default application domain.</span></span> <span data-ttu-id="4f150-120">Jeśli jeden z domen podrzędnych aplikacji można wymusić domyślnej domeny aplikacji do załadowania zestawu, izolacji kodu powoduje utratę i typów w zestawie wymuszanie załadować będzie można uruchomić kod na wyższym poziomie zaufania.</span><span class="sxs-lookup"><span data-stu-id="4f150-120">If one of the child application domains can force the default application domain to load an assembly, the effect of code isolation is lost and types in the forcibly loaded assembly will be able to run code at a higher level of trust.</span></span>

data-stu-id="4f150-127">Occasionally, the default application domain calls its proxy to the control object.</span></span> <span data-ttu-id="4f150-128">Jednak może być przypadki, w których jest
