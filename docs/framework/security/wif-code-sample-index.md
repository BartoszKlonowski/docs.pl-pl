# <a name="wif-code-sample-index"></a>Indeks przykładów kodu programu WIF

Poniżej przedstawiono przykłady kodu dla programu Windows Identity Foundation 4,5:

- [ClaimsAwareWebApp](https://go.microsoft.com/fwlink/?LinkID=248405) — ten przykład demonstruje podstawowe użycie uwierzytelniania externalization (do lokalnej usługi tokenów zabezpieczeń testów z narzędzia do obsługi tożsamości i dostępu dla programu Visual Studio 11) w klasycznej aplikacji ASP.NET (w przeciwieństwie do witryny sieci Web).

- [ClaimsAwareWebService](https://go.microsoft.com/fwlink/?LinkID=248406) — w tym przykładzie przedstawiono podstawowe użycie uwierzytelniania externalization w klasycznej usłudze WCF.

- [ClaimsAwareMvcApplication](https://go.microsoft.com/fwlink/?LinkID=248407) — ten przykład pokazuje, jak zintegrować WIF z MVC, łącznie z niezbiorczą ochroną i kodem, który jest zgodny z przekierowaniami uwierzytelniania formularzy poza kontroler logowania.

- [ClaimsAwareWebFarm](https://go.microsoft.com/fwlink/?LinkID=248408) — ten przykład pokazuje pamięć podręczną sesji w farmie (w przeciwieństwie do tokenreplycache), dzięki czemu można używać sesji przez odwołanie zamiast wymieniać duże pliki cookie. Przedstawiono w nim również łatwiejszy sposób zabezpieczania plików cookie w farmie.

- [ClaimsAwareFormsAuthentication](https://go.microsoft.com/fwlink/?LinkID=248409) — ten bardzo prosty przykład pokazuje, że w programie .NET 4,5 otrzymujesz oświadczenia w podmiotach zabezpieczeń, niezależnie od sposobu uwierzytelniania użytkowników.

- [ClaimsBasedAuthorization](https://go.microsoft.com/fwlink/?LinkID=248410)— ten przykład pokazuje, jak używać klasy składnika ClaimsAuthorizationManager i ClaimsAuthorizationModule do stosowania własnych zasad autoryzacji.

- [Federationmetadata](https://go.microsoft.com/fwlink/?LinkID=248411) — ten przykład pokazuje dynamiczną generację (na NIESTANDARDOWEJ usłudze STS) i dynamiczne użycie (w aplikacji jednostki uzależnionej) dokumentów metadanych.

- [CustomToken](https://go.microsoft.com/fwlink/?LinkID=248412) — ten przykład pokazuje, jak utworzyć niestandardowy typ tokenu prostego tokenu sieci Web (SWT).
