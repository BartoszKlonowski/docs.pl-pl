# <a name="install-the-net-framework-for-developers"></a>Instalowanie programu .NET Framework dla deweloperów

.NET jest integralną częścią wielu aplikacji uruchomionych w systemie Windows i zapewnia typowe funkcje dla tych aplikacji do uruchomienia. Dla deweloperów .NET Framework zapewnia kompleksowy i spójny model programowania do tworzenia aplikacji, które mają wizualnie oszałamiające środowisko użytkownika i bezproblemową i bezpieczną komunikację.

> [!NOTE]
> Ten temat jest przeznaczony dla **deweloperów,** którzy chcą zainstalować program .NET Framework w swoim systemie lub którzy chcą zainstalować go z ich aplikacji. W przypadku **użytkowników** zainteresowanych instalacją programu .NET Framework zobacz poszczególne tematy dotyczące instalowania programu .NET Framework w określonych systemach operacyjnych, takich jak [Instalowanie programu .NET Framework w systemie Windows 10 i Windows Server 2016](on-windows-10.md).

Ten artykuł zawiera łącza do instalowania wszystkich wersji programu .NET Framework z programu .NET Framework 4.5 do .NET Framework 4.8 na komputerze. Jeśli jesteś programistą, możesz również użyć tych łączy do pobierania i rozpowszechnianie programu .NET Framework z aplikacjami. Aby uzyskać informacje na temat wdrażania wersji programu .NET Framework za pomocą aplikacji, zobacz [przewodnik po wdrażaniu programu .NET Framework dla deweloperów](../deployment/deployment-guide-for-developers.md).

## <a name="installation-choices"></a>Wybór instalacji

Zainstaluj pakiet kierowania dla deweloperów, aby opracować dla najnowszej wersji programu .NET Framework w programie Visual Studio lub innym środowisku programistycznym, lub pobierz program .NET Framework redystrybucyjny do dystrybucji za pomocą aplikacji lub formantu.

### <a name="to-install-the-net-framework-developer-pack-or-targeting-pack"></a>Aby zainstalować pakiet dla deweloperów programu .NET Framework lub pakiet targetowania

*Pakiet kierowania* umożliwia aplikacji docelowych określonej wersji programu .NET Framework podczas tworzenia w programie Visual Studio i niektórych innych środowisk programistów. *Pakiet dla deweloperów* zawiera określoną wersję programu .NET Framework i towarzyszący mu zestaw SDK wraz z odpowiednim pakietem docelowym.

Pakiet dla deweloperów programu .NET Framework 4.5.1 lub 4.5.2, pakiet kierowania dla platformy .NET Framework 4.6 oraz pakiet dla deweloperów dla platformy .NET Framework 4.6.1, 4.6.2, 4.7, 4.7.1, 4.7.2 lub 4.8 zawiera określoną wersję zestawów referencyjnych programu .NET Framework, język pakiety i pliki IntelliSense do użycia w zintegrowanym środowisku programistycznym, takim jak Visual Studio.  Jeśli używasz programu Visual Studio, pakiet dewelopera lub pakiet docelowy również dodaje zainstalowaną wersję programu .NET Framework do opcji docelowych podczas tworzenia nowego projektu.  Wybierz jedną z następujących opcji:

- [.NET Framework 4.8](https://dotnet.microsoft.com/download/dotnet-framework/net48)
- [.NET Framework 4.7.2](https://dotnet.microsoft.com/download/dotnet-framework/net472)
- [.NET Framework 4.7.1](https://dotnet.microsoft.com/download/dotnet-framework/net471)
- [.NET Framework 4.7](https://dotnet.microsoft.com/download/dotnet-framework/net47)
- [.NET Framework 4.6.2](https://dotnet.microsoft.com/download/dotnet-framework/net462)
- [.NET Framework 4.6.1](https://dotnet.microsoft.com/download/dotnet-framework/net461)
- [Program .NET Framework 4.6](https://dotnet.microsoft.com/download/dotnet-framework/net46)
- [Program .NET Framework 4.5.2,](https://dotnet.microsoft.com/download/dotnet-framework/net452) aby zainstalować wersję 4.5.2 w systemie Windows 8.1 lub starszym, Visual Studio 2013, Visual Studio 2012 lub inne środowiska ied.
- [.NET Framework 4.5.1](https://dotnet.microsoft.com/download/dotnet-framework/net451) zainstalować wersję 4.5.1 w programie Visual Studio 2012 lub innych programach IE.

Na stronie pobierania pakietu deweloperskiego wybierz pozycję **Pobierz**. Następnie wybierz pozycję **Uruchom** lub **Zapisz**i postępuj zgodnie z instrukcjami po wyświetleniu monitu. Można również zainstalować pakiet dewelopera lub pakiet docelowy dla określonej wersji programu .NET Framework, wybierając go z opcjonalnych składników w obciążeniu **deweloperskim pulpitu platformy .NET** w Instalatorze programu Visual Studio, jak pokazano na poniższym rysunku.

   ![Instalator programu Visual Studio z obciążeniem dewelopera na pulpicie platformy .NET](./media/visual-studio-installer.jpg)

Gdy jest kierowana na określoną wersję programu .NET Framework, aplikacja jest komplikowana przy użyciu zestawów odwołań, które są dołączone do pakietu deweloperów tej wersji. W czasie wykonywania zestawy są rozpoznawane z globalnej pamięci podręcznej zestawów, a zestawy odwołań nie są używane.

Podczas tworzenia aplikacji z programu Visual Studio lub przy użyciu MSBuild z wiersza polecenia, MSBuild może wyświetlać błąd MSB3644, "Zestawy odwołań dla framework "*framework-version*" nie zostały znalezione." Aby rozwiązać ten problem, pobierz pakiet deweloperski lub pakiet docelowy dla tej wersji programu .NET Framework.

### <a name="to-install-or-download-the-net-framework-redistributable"></a>Aby zainstalować lub pobrać redystrybucyjny program .NET Framework

Instalatorzy pobierają składniki programu .NET Framework dla aplikacji lub formantu przeznaczone dla tych wersji programu .NET Framework. Te składniki muszą być zainstalowane na każdym komputerze, na którym działa aplikacja lub formant. Te instalatory są redystrybucyjne, dzięki czemu można je uwzględnić w programie instalacyjnym aplikacji.

Strona pobierania jest dostępna w kilku językach, ale większość pobrań jest dostępna tylko w języku angielskim. Aby uzyskać dodatkową obsługę języka, należy zainstalować pakiet językowy.

Dostępne są dwa typy redystrybucyjnych instalatorów:

- **Instalator sieci Web** (web bootstrapper) pobiera wymagane składniki i pakiet językowy, który pasuje do systemu operacyjnego komputera instalacyjnego z sieci Web. Ten pakiet jest znacznie mniejszy niż instalator w trybie offline, ale wymaga spójnego połączenia z Internetem. Możesz pobrać [autonomiczne pakiety językowe,](#to-install-language-packs) aby zainstalować dodatkową obsługę języka.

- **Instalator w trybie offline** (autonomiczny redystrybucyjny) zawiera wszystkie składniki wymagane do zainstalowania programu .NET Framework, ale nie zawiera pakietów językowych. Ten plik do pobrania jest większy niż instalator internetowy. Instalator w trybie offline nie wymaga połączenia z Internetem. Po uruchomieniu instalatora w trybie offline można pobrać [autonomiczne pakiety językowe,](#to-install-language-packs) aby zainstalować obsługę języka. Użyj instalatora w trybie offline, jeśli nie możesz polegać na stałym połączeniu internetowym.

Zarówno instalatory internetowe, jak i offline są przeznaczone dla komputerów z systemem x86 i x64 (patrz [wymagania systemowe),](../get-started/system-requirements.md)ale nie obsługują komputerów z systemem Itanium.

1. Otwórz stronę pobierania wersji programu .NET Framework, którą chcesz zainstalować:

   - [.NET Framework 4.8](https://dotnet.microsoft.com/download/dotnet-framework/net48)
   - [.NET Framework 4.7.2](https://dotnet.microsoft.com/download/dotnet-framework/net472)
   - [.NET Framework 4.7.1](https://dotnet.microsoft.com/download/dotnet-framework/net471)
   - [.NET Framework 4.7](https://dotnet.microsoft.com/download/dotnet-framework/net47)
   - [.NET Framework 4.6.2](https://dotnet.microsoft.com/download/dotnet-framework/net462)
   - [.NET Framework 4.6.1](https://dotnet.microsoft.com/download/dotnet-framework/net461)
   - [Program .NET Framework 4.6](https://dotnet.microsoft.com/download/dotnet-framework/net46)
   - [.NET Framework 4.5.2](https://dotnet.microsoft.com/download/dotnet-framework/net452)
   - [.NET Framework 4.5.1](https://dotnet.microsoft.com/download/dotnet-framework/net451)
   - [.NET Framework 4.5](https://dotnet.microsoft.com/download/dotnet-framework/net45)

1. Wybierz język strony pobierania. Ta opcja nie pobiera zlokalizowanych zasobów programu .NET Framework; wpływa tylko na tekst wyświetlany na stronie pobierania.

1. Wybierz **pozycję Pobierz**.

1. Jeśli zostanie wyświetlony monit, wybierz pobieranie zgodne z architekturą systemu, a następnie wybierz pozycję **Dalej**.

1. Gdy pojawi się monit o pobranie, wykonaj **jedną z** następujących czynności:

   - Jeśli chcesz zainstalować program .NET Framework na komputerze, wybierz pozycję **Uruchom**, a następnie postępuj zgodnie z instrukcjami wyświetlanymi na ekranie.

   - Jeśli chcesz pobrać program .NET Framework do redystrybucji, wybierz pozycję **Zapisz**, a następnie postępuj zgodnie z instrukcjami wyświetlanymi na ekranie.

1. Jeśli chcesz pobrać zasoby dotyczące dodatkowych języków, postępuj zgodnie z instrukcjami w następnej sekcji, aby zainstalować jeden lub więcej pakietów językowych.

> [!NOTE]
> Jeśli podczas instalacji wystąpią jakiekolwiek problemy, zobacz [Rozwiązywanie problemów](troubleshoot-blocked-installations-and-uninstallations.md).

**Uwagi dotyczące instalacji:**

- .NET Framework 4.5 i nowsze wersje zastępują .NET Framework 4.0. Po zainstalowaniu tych wersji w systemie, który ma zainstalowany program .NET Framework 4, zestawy są zastępowane.

- Odinstalowanie wersji programu .NET Framework 4.5 lub nowszych powoduje również usunięcie wcześniej istniejących plików programu .NET Framework 4. Jeśli chcesz wrócić do programu .NET Framework 4, należy ponownie zainstalować go i wszelkie aktualizacje do niego. Zobacz [Instalowanie programu .NET Framework 4](https://go.microsoft.com/fwlink/p/?LinkId=230665).

- Aby zainstalować program .NET Framework 4.5 lub nowszych, musisz mieć poświadczenia administracyjne.

- Redystrybucyjne programu .NET Framework 4.5 zostało zaktualizowane 9 października 2012 r., aby rozwiązać problem związany z niewłaściwą sygnaturą czasową certyfikatu cyfrowego, która spowodowała przedwczesne wygaśnięcie podpisu cyfrowego na plikach wyprodukowanych i podpisanych przez firmę Microsoft. Jeśli wcześniej zainstalowano redystrybucyjny pakiet .NET Framework 4.5 z dnia 16 sierpnia 2012 r., zalecamy zaktualizowanie kopii najnowszą redystrybucjową [ze strony pobierania programu .NET Framework](https://dotnet.microsoft.com/download/dotnet-framework/net45). Aby uzyskać więcej informacji na temat tego problemu, zobacz [Poradnik zabezpieczeń firmy Microsoft 2749655](https://docs.microsoft.com/security-updates/SecurityAdvisories/2012/2749655) i [artykuł 2770445 z bazy wiedzy .](https://support.microsoft.com/kb/2770445)

## <a name="to-install-language-packs"></a>Aby zainstalować pakiety językowe

Pakiety językowe to pliki wykonywalne zawierające zlokalizowane zasoby (takie jak przetłumaczone komunikaty o błędach i tekst interfejsu użytkownika) dla obsługiwanych języków. Jeśli nie zainstalujesz pakietu językowego, komunikaty o błędach programu .NET Framework i inny tekst są wyświetlane w języku angielskim.  Należy pamiętać, że instalator internetowy automatycznie instaluje pakiet językowy zgodny z systemem operacyjnym, ale można pobrać dodatkowe pakiety językowe na komputer. Instalatorzy w trybie offline nie zawierają żadnych pakietów językowych.

> [!IMPORTANT]
> Pakiety językowe nie zawierają składników .NET Framework, które są wymagane do uruchomienia aplikacji, dlatego należy uruchomić instalator sieci Web lub offline przed zainstalowaniem pakietu językowego. Jeśli masz już zainstalowany pakiet językowy, odinstaluj go, zainstaluj program .NET Framework, a następnie zainstaluj ponownie pakiet językowy.

1. Otwórz stronę pobierania pakietu językowego dla zainstalowanej wersji programu .NET Framework:

   - [.NET Framework 4.7.1](https://dotnet.microsoft.com/download/dotnet-framework/net471)

   - [.NET Framework 4.6.2](https://dotnet.microsoft.com/download/dotnet-framework/net462)
   - [.NET Framework 4.6.1](https://dotnet.microsoft.com/download/dotnet-framework/net461)
