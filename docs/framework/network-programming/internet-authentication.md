# <a name="internet-authentication"></a><span data-ttu-id="33273-103">Uwierzytelnianie internetowe</span><span class="sxs-lookup"><span data-stu-id="33273-103">Internet Authentication</span></span>
<span data-ttu-id="33273-104"><xref:System.Net>Klasy obsługują różne mechanizmy uwierzytelniania klientów, w tym standardowe metody uwierzytelniania internetowego Basic, Digest, Negotiate, NTLM i Kerberos, a także metody niestandardowe, które można utworzyć.</span><span class="sxs-lookup"><span data-stu-id="33273-104">The <xref:System.Net> classes support a variety of client authentication mechanisms, including the standard Internet authentication methods basic, digest, negotiate, NTLM, and Kerberos authentication, as well as custom methods that you can create.</span></span>  
  
 <span data-ttu-id="33273-105">Poświadczenia uwierzytelniania są przechowywane w <xref:System.Net.NetworkCredential> <xref:System.Net.CredentialCache> klasach i, które implementują <xref:System.Net.ICredentials> interfejs.</span><span class="sxs-lookup"><span data-stu-id="33273-105">Authentication credentials are stored in the <xref:System.Net.NetworkCredential> and <xref:System.Net.CredentialCache> classes, which implement the <xref:System.Net.ICredentials> interface.</span></span> <span data-ttu-id="33273-106">Gdy jedna z tych klas jest kwerendą dla poświadczeń, zwraca wystąpienie klasy **NetworkCredential** .</span><span class="sxs-lookup"><span data-stu-id="33273-106">When one of these classes is queried for credentials, it returns an instance of the **NetworkCredential** class.</span></span> <span data-ttu-id="33273-107">Proces uwierzytelniania jest zarządzany przez <xref:System.Net.AuthenticationManager> klasę, a rzeczywisty proces uwierzytelniania jest wykonywany przez klasę modułu uwierzytelniania, która implementuje <xref:System.Net.IAuthenticationModule> interfejs.</span><span class="sxs-lookup"><span data-stu-id="33273-107">The authentication process is managed by the <xref:System.Net.AuthenticationManager> class, and the actual authentication process is performed by an authentication module class that implements the <xref:System.Net.IAuthenticationModule> interface.</span></span> <span data-ttu-id="33273-108">Aby można było użyć niestandardowego **modułu uwierzytelniania,** należy zarejestrować go. moduły dla metod uwierzytelniania podstawowa, Digest, Negotiate, NTLM i Kerberos są domyślnie zarejestrowane.</span><span class="sxs-lookup"><span data-stu-id="33273-108">You must register a custom authentication module with the **AuthenticationManager** before it can be used; modules for the basic, digest, negotiate, NTLM, and Kerberos authentication methods are registered by default.</span></span>  
  
 <span data-ttu-id="33273-109">**NetworkCredential** przechowuje zestaw poświadczeń skojarzonych z pojedynczym zasobem internetowym identyfikowanym przez identyfikator URI i zwraca je w odpowiedzi na każde wywołanie <xref:System.Net.NetworkCredential.GetCredential%2A> metody.</span><span class="sxs-lookup"><span data-stu-id="33273-109">**NetworkCredential** stores a set of credentials associated with a single Internet resource identified by a URI and returns them in response to any call to the <xref:System.Net.NetworkCredential.GetCredential%2A> method.</span></span> <span data-ttu-id="33273-110">Klasa **NetworkCredential** jest zwykle używana przez aplikacje, które uzyskują dostęp do ograniczonej liczby zasobów internetowych lub przez aplikacje korzystające z tego samego zestawu poświadczeń we wszystkich przypadkach.</span><span class="sxs-lookup"><span data-stu-id="33273-110">The **NetworkCredential** class is typically used by applications that access a limited number of Internet resources or by applications that use the same set of credentials in all cases.</span></span>  
  
 <span data-ttu-id="33273-111">Klasa **CredentialCache** przechowuje kolekcję poświadczeń dla różnych zasobów sieci Web.</span><span class="sxs-lookup"><span data-stu-id="33273-111">The **CredentialCache** class stores a collection of credentials for various Web resources.</span></span> <span data-ttu-id="33273-112">Gdy <xref:System.Net.CredentialCache.GetCredential%2A> Metoda jest wywoływana, **CredentialCache** zwraca właściwy zestaw poświadczeń, określony przez identyfikator URI zasobu sieci Web i żądany schemat uwierzytelniania.</span><span class="sxs-lookup"><span data-stu-id="33273-112">When the <xref:System.Net.CredentialCache.GetCredential%2A> method is called, **CredentialCache** returns the proper set of credentials, as determined by the URI of the Web resource and the requested authentication scheme.</span></span> <span data-ttu-id="33273-113">Aplikacje korzystające z różnych zasobów internetowych z różnymi schematami uwierzytelniania korzystają z klasy **CredentialCache** , ponieważ przechowuje wszystkie poświadczenia i udostępnia je zgodnie z żądaniem.</span><span class="sxs-lookup"><span data-stu-id="33273-113">Applications that use a variety of Internet resources with different authentication schemes benefit from using the **CredentialCache** class, since it stores all the credentials and provides them as requested.</span></span>  
  
 <span data-ttu-id="33273-114">Gdy zasób internetowy żąda uwierzytelnienia, <xref:System.Net.WebRequest.GetResponse%2A?displayProperty=nameWithType> Metoda wysyła <xref:System.Net.WebRequest> do elementu **AuthenticationManager** wraz z żądaniem poświadczeń.</span><span class="sxs-lookup"><span data-stu-id="33273-114">When an Internet resource requests authentication, the <xref:System.Net.WebRequest.GetResponse%2A?displayProperty=nameWithType> method sends the <xref:System.Net.WebRequest> to the **AuthenticationManager** along with the request for credentials.</span></span> <span data-ttu-id="33273-115">Żądanie jest następnie uwierzytelniane zgodnie z następującym procesem:</span><span class="sxs-lookup"><span data-stu-id="33273-115">The request is then authenticated according to the following process:</span></span>  
  
1. <span data-ttu-id="33273-116">Metoda **AuthenticationManager** wywołuje <xref:System.Net.IAuthenticationModule.Authenticate%2A> metodę dla każdego z zarejestrowanych modułów uwierzytelniania w kolejności, w której zostały zarejestrowane.</span><span class="sxs-lookup"><span data-stu-id="33273-116">The **AuthenticationManager** calls the <xref:System.Net.IAuthenticationModule.Authenticate%2A> method on each of the registered authentication modules in the order they were registered.</span></span> <span data-ttu-id="33273-117">Program **AuthenticationManager** używa pierwszego modułu, który nie zwraca **wartości null** w celu przeprowadzenia procesu uwierzytelniania.</span><span class="sxs-lookup"><span data-stu-id="33273-117">The **AuthenticationManager** uses the first module that does not return **null** to carry out the authentication process.</span></span> <span data-ttu-id="33273-118">Szczegóły procesu różnią się w zależności od typu używanego modułu uwierzytelniania.</span><span class="sxs-lookup"><span data-stu-id="33273-118">The details of the process vary depending on the type of authentication module involved.</span></span>  
  
2. <span data-ttu-id="33273-119">Po zakończeniu procesu uwierzytelniania moduł uwierzytelniania zwraca <xref:System.Net.Authorization> do **żądania webżądanie** , które zawiera informacje konieczne do uzyskania dostępu do zasobu internetowego.</span><span class="sxs-lookup"><span data-stu-id="33273-119">When the authentication process is complete, the authentication module returns an <xref:System.Net.Authorization> to the **WebRequest** that contains the information needed to access the Internet resource.</span></span>  
  
 <span data-ttu-id="33273-120">Niektóre schematy uwierzytelniania mogą uwierzytelniać użytkownika bez uprzedniego utworzenia żądania dla zasobu.</span><span class="sxs-lookup"><span data-stu-id="33273-120">Some authentication schemes can authenticate a user without first making a request for a resource.</span></span> <span data-ttu-id="33273-121">Aplikacja może zaoszczędzić czas poprzez preuwierzytelnianie użytkownika przy użyciu zasobu, co eliminuje co najmniej jedną rundę na serwerze.</span><span class="sxs-lookup"><span data-stu-id="33273-121">An application can save time by preauthenticating the user with the resource, thus eliminating at least one round trip to the server.</span></span> <span data-ttu-id="33273-122">Można też przeprowadzić uwierzytelnianie podczas uruchamiania programu w celu późniejszego reagowania na użytkownika.</span><span class="sxs-lookup"><span data-stu-id="33273-122">Or, it can perform authentication during program startup in order to be more responsive to the user later.</span></span> <span data-ttu-id="33273-123">Schematy uwierzytelniania, które mogą używać wstępnego uwierzytelniania, mają ustawioną <xref:System.Net.IAuthenticationModule.PreAuthenticate%2A> Właściwość na **wartość true**.</span><span class="sxs-lookup"><span data-stu-id="33273-123">Authentication schemes that can use preauthentication set the <xref:System.Net.IAuthenticationModule.PreAuthenticate%2A> property to **true**.</span></span>
