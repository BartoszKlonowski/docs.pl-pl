<span data-ttu-id="e1b1a-103">Zasady pamięci podręcznej definiuje reguły, które są używane do określenia, czy żądanie może być spełnione przy użyciu buforowanej kopii żądanego zasobu.</span><span class="sxs-lookup"><span data-stu-id="e1b1a-103">A cache policy defines rules that are used to determine whether a request can be satisfied using a cached copy of the requested resource.</span></span> <span data-ttu-id="e1b1a-104">Aplikacje określają wymagania pamięci podręcznej klienta dla świeżości, ale zasady skutecznej pamięci podręcznej zależy od wymagań pamięci podręcznej klienta, wymagań wygaśnięcia zawartości serwera i wymagań ponownego licencjonowania serwera.</span><span class="sxs-lookup"><span data-stu-id="e1b1a-104">Applications specify client cache requirements for freshness, but the effective cache policy is determined by the client cache requirements, the server's content expiration requirements, and the server's revalidation requirements.</span></span> <span data-ttu-id="e1b1a-105">Interakcja zasad pamięci podręcznej klienta i wymagania serwera zawsze powoduje najbardziej konserwatywne zasady pamięci podręcznej, aby zapewnić, że najświeższa zawartość jest zwracana do aplikacji klienckiej.</span><span class="sxs-lookup"><span data-stu-id="e1b1a-105">The interaction of client cache policy and server requirements always results in the most conservative cache policy, to help ensure that the freshest content is returned to the client application.</span></span>  
  
 <span data-ttu-id="e1b1a-106">Zasady pamięci podręcznej są oparte na lokalizacji lub na podstawie czasu.</span><span class="sxs-lookup"><span data-stu-id="e1b1a-106">Cache policies are either location-based or time-based.</span></span> <span data-ttu-id="e1b1a-107">Zasady pamięci podręcznej oparte na lokalizacji definiują świeżość buforowanych wpisów na podstawie tego, skąd można zabrać żądany zasób.</span><span class="sxs-lookup"><span data-stu-id="e1b1a-107">A location-based cache policy defines the freshness of cached entries based on where the requested resource can be taken from.</span></span> <span data-ttu-id="e1b1a-108">Zasady pamięci podręcznej oparte na czasie definiuje świeżość wpisów w pamięci podręcznej przy użyciu czasu, w której zasób został pobrany, nagłówków zwróconych z zasobem i bieżącej.</span><span class="sxs-lookup"><span data-stu-id="e1b1a-108">A time-based cache policy defines the freshness of cached entries using the time the resource was retrieved, headers returned with the resource, and the current time.</span></span> <span data-ttu-id="e1b1a-109">Większość aplikacji może używać domyślnych zasad pamięci podręcznej opartych na czasie, która implementuje zasady buforowania określone w RFC 2616, dostępne w witrynie internetowej [Internet Engineering Task Force (IETF).](https://www.ietf.org/)</span><span class="sxs-lookup"><span data-stu-id="e1b1a-109">Most applications can use the default time-based cache policy, which implements the caching policy specified in RFC 2616, available at [Internet Engineering Task Force (IETF)](https://www.ietf.org/) website.</span></span>
