-   <span data-ttu-id="0ec55-107">Określ miejsce docelowe danych wyjściowych śledzenia.</span><span class="sxs-lookup"><span data-stu-id="0ec55-107">Specify a destination for tracing output.</span></span>  
  
-   <span data-ttu-id="0ec55-108">Skonfiguruj zachowanie funkcji śledzenia sieci.</span><span class="sxs-lookup"><span data-stu-id="0ec55-108">Configure the behavior of network tracing.</span></span> <span data-ttu-id="0ec55-109">Zobacz [porady: Konfigurowanie śledzenia sieci](../../../docs/framework/network-programming/how-to-configure-network-tracing.md) Aby uzyskać szczegółowe informacje.</span><span class="sxs-lookup"><span data-stu-id="0ec55-109">See [How to: Configure Network Tracing](../../../docs/framework/network-programming/how-to-configure-network-tracing.md) for detailed information.</span></span>  
  
 <span data-ttu-id="0ec55-110">Najpopularniejsze miejsca docelowe śledzenia, nazywane również detektorów śledzenia są odbiornika domyślne, a plik dziennika.</span><span class="sxs-lookup"><span data-stu-id="0ec55-110">The most common trace destinations, also referred to as trace listeners, are the default listener and the log file.</span></span>  
  
 <span data-ttu-id="0ec55-111">Śledzenie używa odbiornika domyślne, jeśli nie określisz odbiornik śledzenia.</span><span class="sxs-lookup"><span data-stu-id="0ec55-111">Tracing uses the default listener if you do not specify a trace listener.</span></span> <span data-ttu-id="0ec55-112">Można wyświetlić komunikaty wysyłane do odbiornika domyślne przez wykonywanie kodu zarządzanego debugera kodu włączone, takich jak debuger CLR są dostarczane z zestawu SDK programu .NET Framework lub DBwin32.exe są dostarczane z zestawu Windows SDK.</span><span class="sxs-lookup"><span data-stu-id="0ec55-112">You can view messages sent to the default listener by executing your code in a managed code-enabled debugger such as the CLR Debugger shipped with the .NET Framework SDK, or DBwin32.exe shipped with the Windows SDK.</span></span> <span data-ttu-id="0ec55-113">Korzystanie z debugera CLR, komunikaty śledzenia są wyświetlane w **dane wyjściowe** okna.</span><span class="sxs-lookup"><span data-stu-id="0ec55-113">Using the CLR Debugger, the trace messages appear in the **Output** window.</span></span>  
  
 <span data-ttu-id="0ec55-114">Jeśli chcesz otrzymywać danych śledzenia przy użyciu pliku można określić plik dziennika przy użyciu ustawień konfiguracji, jak pokazano w poniższym przykładzie.</span><span class="sxs-lookup"><span data-stu-id="0ec55-114">If you prefer to use a file to receive traces, you can specify a log file by using configuration settings, as shown in the following example.</span></span> <span data-ttu-id="0ec55-115">(Ogólne omówienie plików konfiguracyjnych, zobacz [pliki konfiguracyjne](../../../docs/framework/configure-apps/index.md).)</span><span class="sxs-lookup"><span data-stu-id="0ec55-115">(For a general discussion of configuration files, see [Configuration Files](../../../docs/framework/configure-apps/index.md).)</span></span>  
  
 <span data-ttu-id="0ec55-116">Aby wysłać dane śledzenia w pliku dziennika, należy dodać następujący węzeł do `<system.diagnostics>` węzeł odpowiedniego pliku konfiguracyjnego (aplikacji lub komputera).</span><span class="sxs-lookup"><span data-stu-id="0ec55-116">To send traces to a log file, add the following node to the `<system.diagnostics>` node of the appropriate configuration file (application or machine).</span></span> <span data-ttu-id="0ec55-117">Można zmienić nazwę pliku (trace.log) do własnych potrzeb.</span><span class="sxs-lookup"><span data-stu-id="0ec55-117">You can change the name of the file (trace.log) to suit your needs.</span></span>  
  
```xml  
<system.diagnostics>  
  <trace autoflush="true" indentsize="4">  
    <listeners>  
      <add name="file" type="System.Diagnostics.TextWriterTraceListener" initializeData="trace.log"/>  
    </listeners>   
  </trace>  
</system.diagnostics>  
```
