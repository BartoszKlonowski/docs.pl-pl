# <a name="pnrp-clouds"></a><span data-ttu-id="df666-102">Usługa PNRP chmury</span><span class="sxs-lookup"><span data-stu-id="df666-102">PNRP Clouds</span></span>

 <span data-ttu-id="df666-105">Komunikacja między węzłami nigdy nie powinien przechodzą z jednej chmury do innej.</span><span class="sxs-lookup"><span data-stu-id="df666-105">Communication between nodes should never cross from one cloud to another.</span></span> <span data-ttu-id="df666-106">A <xref:System.Net.PeerToPeer.Cloud> wystąpienia jest unikatowo identyfikowana przez jego nazwę, która jest uwzględniana wielkość liter.</span><span class="sxs-lookup"><span data-stu-id="df666-106">A <xref:System.Net.PeerToPeer.Cloud> instance is uniquely identified by its name, which is case-sensitive.</span></span> <span data-ttu-id="df666-107">Pojedynczego elementu równorzędnego lub węzeł może być połączone do więcej niż jednej chmury.</span><span class="sxs-lookup"><span data-stu-id="df666-107">A single peer or node may be connected to more than one cloud.</span></span>  
  
 <span data-ttu-id="df666-108">Chmury są bardzo ściśle powiązane z interfejsów sieciowych.</span><span class="sxs-lookup"><span data-stu-id="df666-108">Clouds are tied very closely to network interfaces.</span></span>  <span data-ttu-id="df666-109">Na maszynie wieloadresowego z dwóch kart sieciowych, dołączone do różnych podsieci zostanie zwrócony trzy chmury: jeden dla poszczególnych adresów lokalnych łącza na interfejs i w chmurze pojedynczy zakres globalny.</span><span class="sxs-lookup"><span data-stu-id="df666-109">On a multi-homed machine with two network cards attached to different subnets, three clouds will be returned: one for each of the link local addresses per interface, and a single global scope cloud.</span></span>  
  
 <span data-ttu-id="df666-110">Usługa PNRP używa trzech chmury "zakresy", w których zakres to grupa komputerów, które są w stanie odnajdować:</span><span class="sxs-lookup"><span data-stu-id="df666-110">PNRP uses three cloud "scopes", in which a scope is a grouping of computers that are able to find each other:</span></span>  
  
-   <span data-ttu-id="df666-111">Globalne chmury odpowiada zasięg globalny adres IPv6 i adresy globalne i reprezentuje wszystkie komputery w całej Internet IPv6.</span><span class="sxs-lookup"><span data-stu-id="df666-111">The global cloud corresponds to the global IPv6 address scope and global addresses and represents all the computers on the entire IPv6 Internet.</span></span> <span data-ttu-id="df666-112">Istnieje tylko jeden chmury globalnej.</span><span class="sxs-lookup"><span data-stu-id="df666-112">There is only a single global cloud.</span></span>  
  
-   <span data-ttu-id="df666-113">Link-local chmury odpowiada zakres adresów IPv6 połączenia lokalnego i adresów połączenia lokalnego.</span><span class="sxs-lookup"><span data-stu-id="df666-113">The link-local cloud corresponds to the link-local IPv6 address scope and link-local addresses.</span></span> <span data-ttu-id="df666-114">Chmury połączenia lokalnego jest przeznaczony dla określonego łącza, które zwykle jest taka sama jak lokalnie dołączone podsieci.</span><span class="sxs-lookup"><span data-stu-id="df666-114">A link-local cloud is for a specific link, which is typically the same as the locally attached subnet.</span></span> <span data-ttu-id="df666-115">Może istnieć wiele chmur połączenia lokalnego.</span><span class="sxs-lookup"><span data-stu-id="df666-115">There can be multiple link-local clouds.</span></span>  
  
 <span data-ttu-id="df666-116">Trzeci chmury, chmury specyficzne dla lokacji, odnosi się do witryny IPv6 zakres adresów i adresy lokacji lokalnej.</span><span class="sxs-lookup"><span data-stu-id="df666-116">A third cloud, the site-specific cloud, corresponds to the site IPv6 address scope and site-local addresses.</span></span> <span data-ttu-id="df666-117">Ta chmura jest przestarzałe, mimo że nadal jest obsługiwany w protokole PNRP.</span><span class="sxs-lookup"><span data-stu-id="df666-117">This cloud has been deprecated, although it is still supported in PNRP.</span></span>  
  
## <a name="clouds"></a><span data-ttu-id="df666-118">Chmury</span><span class="sxs-lookup"><span data-stu-id="df666-118">Clouds</span></span>

 <span data-ttu-id="df666-122">Poszczególne chmury mają unikatowe nazwy, reprezentowany jako ciąg Unicode 256 znaków.</span><span class="sxs-lookup"><span data-stu-id="df666-122">Individual clouds have unique names, represented as a 256 character Unicode string.</span></span> <span data-ttu-id="df666-123">Te nazwy, wraz z wyżej zakresu, są używane do utworzenia unikatowych wystąpień klasy chmury.</span><span class="sxs-lookup"><span data-stu-id="df666-123">These names, along with the above-mentioned scope, are used to construct unique instances of the Cloud class.</span></span> <span data-ttu-id="df666-124">Te wystąpienia można serializować i odtworzyć trwałe użycia.</span><span class="sxs-lookup"><span data-stu-id="df666-124">These instances can be serialized and reconstructed for persistent usage.</span></span>  
  
 <span data-ttu-id="df666-125">Po wystąpienia chmury jest utworzyć lub uzyskać, można zarejestrować nazwy elementów równorzędnych z go w celu utworzenia siatkę znanych elementów równorzędnych.</span><span class="sxs-lookup"><span data-stu-id="df666-125">Once a Cloud instance is created or obtained, peer names can be registered with it to create a mesh of known peers.</span></span>
