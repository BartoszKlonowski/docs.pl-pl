# <a name="pnrp-clouds"></a><span data-ttu-id="e7c8e-102">Chmury PNRP</span><span class="sxs-lookup"><span data-stu-id="e7c8e-102">PNRP Clouds</span></span>
<span data-ttu-id="e7c8e-103">PNRP "chmura" reprezentuje zestaw węzłów, które mogą komunikować się ze sobą za pośrednictwem sieci.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-103">A PNRP "cloud" represents a set of nodes that can communicate with each other through the network.</span></span> <span data-ttu-id="e7c8e-104">Termin "chmura" jest synonimem "siatki elementów równorzędnych" i "peer-to-peer grafu".</span><span class="sxs-lookup"><span data-stu-id="e7c8e-104">The term "cloud" is synonymous with "peer mesh" and "peer-to-peer graph".</span></span>  
  
 <span data-ttu-id="e7c8e-105">Komunikacji między węzłami powinno nigdy nie przechodzą z chmury do innego.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-105">Communication between nodes should never cross from one cloud to another.</span></span> <span data-ttu-id="e7c8e-106">A <xref:System.Net.PeerToPeer.Cloud> wystąpienia jest unikatowo identyfikowane przez jego nazwę, która jest uwzględniana wielkość liter.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-106">A <xref:System.Net.PeerToPeer.Cloud> instance is uniquely identified by its name, which is case-sensitive.</span></span> <span data-ttu-id="e7c8e-107">Pojedynczego elementu równorzędnego lub węzeł może być podłączony do więcej niż jednej chmury.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-107">A single peer or node may be connected to more than one cloud.</span></span>  
  
 <span data-ttu-id="e7c8e-108">Chmury są ściśle powiązane interfejsy sieciowe.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-108">Clouds are tied very closely to network interfaces.</span></span>  <span data-ttu-id="e7c8e-109">Na maszynie wieloadresowych z dwóch kart sieciowych dołączonych do różnych podsieci, zostaną zwrócone trzy chmury: jeden dla każdego z adresów lokalnych łącza na interfejsu i w chmurze pojedynczy zakres globalny.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-109">On a multi-homed machine with two network cards attached to different subnets, three clouds will be returned: one for each of the link local addresses per interface, and a single global scope cloud.</span></span>  
  
 <span data-ttu-id="e7c8e-110">Protokół PNRP używa trzech chmury "zakresów", w których zakresie to grupa komputerów, które są w stanie odnajdować:</span><span class="sxs-lookup"><span data-stu-id="e7c8e-110">PNRP uses three cloud "scopes", in which a scope is a grouping of computers that are able to find each other:</span></span>  
  
- <span data-ttu-id="e7c8e-111">Chmury globalnej odnosi się do globalnego zakresu adresów IPv6 i adresy globalne i przedstawia wszystkie komputery w cały Internet IPv6.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-111">The global cloud corresponds to the global IPv6 address scope and global addresses and represents all the computers on the entire IPv6 Internet.</span></span> <span data-ttu-id="e7c8e-112">Istnieje tylko jednej chmury globalnej.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-112">There is only a single global cloud.</span></span>  
  
- <span data-ttu-id="e7c8e-113">Chmura link-local odpowiada zakres adresów IPv6 połączenia lokalnego i adresów połączenia lokalnego.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-113">The link-local cloud corresponds to the link-local IPv6 address scope and link-local addresses.</span></span> <span data-ttu-id="e7c8e-114">Chmura połączenia lokalnego jest jedno łącze jest zwykle taka sama, jak podsieć, podłączonych lokalnie.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-114">A link-local cloud is for a specific link, which is typically the same as the locally attached subnet.</span></span> <span data-ttu-id="e7c8e-115">Może istnieć wiele chmur połączenia lokalnego.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-115">There can be multiple link-local clouds.</span></span>  
  
 <span data-ttu-id="e7c8e-116">Trzeci chmury, chmury specyficzne dla lokacji, odnosi się do witryny IPv6 zakres adresów i adresy lokacji lokalnej.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-116">A third cloud, the site-specific cloud, corresponds to the site IPv6 address scope and site-local addresses.</span></span> <span data-ttu-id="e7c8e-117">Ta chmura została wycofana, mimo że nadal jest obsługiwana w PNRP.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-117">This cloud has been deprecated, although it is still supported in PNRP.</span></span>  
  
## <a name="clouds"></a><span data-ttu-id="e7c8e-118">Chmury</span><span class="sxs-lookup"><span data-stu-id="e7c8e-118">Clouds</span></span>

 <span data-ttu-id="e7c8e-122">Poszczególne chmury mają unikatowych nazw, reprezentowany jako ciąg Unicode 256 znaków.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-122">Individual clouds have unique names, represented as a 256 character Unicode string.</span></span> <span data-ttu-id="e7c8e-123">Nazwy te, wraz z wyżej zakresu, są używane do konstruowania unikatowe wystąpień klasy chmury.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-123">These names, along with the above-mentioned scope, are used to construct unique instances of the Cloud class.</span></span> <span data-ttu-id="e7c8e-124">Te wystąpienia mogą serializacji i odtworzone za użycie trwałego.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-124">These instances can be serialized and reconstructed for persistent usage.</span></span>  
  
 <span data-ttu-id="e7c8e-125">Po utworzeniu wystąpienia w chmurze lub uzyskane, nazwy elementów równorzędnych może być zarejestrowany go, aby utworzyć siatki elementów równorzędnych znane.</span><span class="sxs-lookup"><span data-stu-id="e7c8e-125">Once a Cloud instance is created or obtained, peer names can be registered with it to create a mesh of known peers.</span></span>
