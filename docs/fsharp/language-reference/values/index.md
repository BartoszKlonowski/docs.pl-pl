<span data-ttu-id="a9dac-120">W czystym języków funkcjonalności istnieją żadnych zmiennych i funkcje działają wyłącznie jako funkcje matematyczne.</span><span class="sxs-lookup"><span data-stu-id="a9dac-120">In pure functional languages, there are no variables, and functions behave strictly as mathematical functions.</span></span> <span data-ttu-id="a9dac-121">W przypadku, gdy kodu w języku procedurach używa przypisanie zmiennej, aby zmienić wartość, odpowiednik kodu w języku funkcjonalności ma niezmienne wartość danych wejściowych, funkcja niezmienne i różne wartości niezmienne jako dane wyjściowe.</span><span class="sxs-lookup"><span data-stu-id="a9dac-121">Where code in a procedural language uses a variable assignment to alter a value, the equivalent code in a functional language has an immutable value that is the input, an immutable function, and different immutable values as the output.</span></span> <span data-ttu-id="a9dac-122">Ta matematyczne poziom ścisłości umożliwia większego wnioskowania o zachowaniu program.</span><span class="sxs-lookup"><span data-stu-id="a9dac-122">This mathematical strictness allows for tighter reasoning about the behavior of the program.</span></span> <span data-ttu-id="a9dac-123">Zwiększenie poziomu uzasadnienie jest temu kompilatory dokładniej Sprawdź kod i zoptymalizować efektywniej i podnosi poziom stało się łatwiejsze do zrozumienia i zapisać prawidłowego kodu.</span><span class="sxs-lookup"><span data-stu-id="a9dac-123">This tighter reasoning is what enables compilers to check code more stringently and to optimize more effectively, and helps make it easier for developers to understand and write correct code.</span></span> <span data-ttu-id="a9dac-124">Funkcjonalny kod jest w związku z tym mogą być łatwiejsze do debugowania niż zwykłe kod procedurach.</span><span class="sxs-lookup"><span data-stu-id="a9dac-124">Functional code is therefore likely to be easier to debug than ordinary procedural code.</span></span>
