# <a name="formatting-rules"></a>Reguły formatowania

Reguły formatowania wpływają na to, jak wcięcia, spacje i nowe wiersze są wyrównane do konstrukcji języka programowania .NET. Reguły są uwzględniane w następujących kategoriach:

- [Reguły formatowania platformy .NET](#net-formatting-rules): reguły, które mają zastosowanie do języka C# i Visual Basic. Nazwy opcji EditorConfig dla tych reguł zaczynają się od `dotnet_` prefiksu.
- [Reguły formatowania języka c#](#c-formatting-rules): reguły, które są specyficzne tylko dla języka C#. Nazwy opcji EditorConfig dla tych reguł zaczynają się od `csharp_` prefiksu.

## <a name="rule-id-ide0055-fix-formatting"></a>Identyfikator reguły: "IDE0055" (Popraw formatowanie)

Wszystkie opcje formatowania mają identyfikator `IDE0055` i tytuł reguły `Fix formatting` . Ustaw ważność naruszenia formatowania w pliku EditorConfig przy użyciu następującego wiersza konfiguracji.

```ini
dotnet_diagnostic.IDE0055.severity = <severity value>
```

Wartość ważności musi być `warning` `error` [wymuszana w kompilacji](../overview.md#code-style-analysis). Dla wszystkich możliwych wartości ważności zobacz [poziom ważności](../configuration-options.md#severity-level).

## <a name="option-format"></a>Format opcji

Opcje formatowania reguł można określić w pliku EditorConfig o następującym formacie:

`rule_name = value`

W przypadku wielu reguł należy określić jeden `true` (Preferuj ten styl) lub `false` (nie Preferuj tego stylu) dla `value` . W przypadku innych reguł należy określić wartość taką jak `flush_left` lub, `before_and_after` aby opisać, kiedy i gdzie należy zastosować regułę. Nie określisz ważności.

#### <a name="csharp_new_line_before_open_brace"></a>CSharp \_ Nowy \_ wiersz \_ przed \_ open_brace

Ta zasada ma wpływ na to, czy otwierający nawias klamrowy `{` powinien być umieszczony w tym samym wierszu co poprzedni kod, czy w nowym wierszu. Dla tej reguły należy określić **wszystkie**, **Brak** lub co najmniej jeden element kodu, taki jak **metody** lub **Właściwości**, aby określić, kiedy ta reguła ma być stosowana. Aby określić wiele elementów kodu, rozdziel je przecinkami (,).

|Właściwość|Wartość|
|-|-|
| **Nazwa opcji** | csharp_new_line_before_open_brace |
| **Odpowiednie języki** | C# |
| **Wprowadzona wersja** | Visual Studio 2017, wersja 15.3 |
| **Wartości opcji** | `all` -Wymagaj nawiasów klamrowych w nowym wierszu dla wszystkich wyrażeń (styl "Allman").<br /><br />`none` -Wymagaj nawiasów klamrowych w tym samym wierszu dla wszystkich wyrażeń ("K&R").<br /><br />`accessors`,,,,,,,,,, `anonymous_methods` `anonymous_types` `control_blocks` `events` `indexers` `lambdas` `local_functions` `methods` `object_collection_array_initializers` `properties` , `types` — Wymagaj nawiasów klamrowych, aby znajdować się w nowym wierszu dla określonego elementu kodu (styl "Allman"). |

Przykłady kodu:

```csharp

{
    if (...)
    {
        ...
    }
}

// csharp_new_line_before_open_brace = none

    if (...) {
        ...
    }
}
```
