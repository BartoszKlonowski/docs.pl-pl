## <a name="overview"></a><span data-ttu-id="4411a-116">Omówienie</span><span class="sxs-lookup"><span data-stu-id="4411a-116">Overview</span></span>

<span data-ttu-id="4411a-117">Ta reguła wykrywa struktury, które zawierają jedno dodatkowe `readonly` pola i zawiera przypisanie do `this` spoza konstruktora.</span><span class="sxs-lookup"><span data-stu-id="4411a-117">This rule detects structs that contain one ore more `readonly` fields and also contains an assignment to `this` outside of the constructor.</span></span> <span data-ttu-id="4411a-118">Reguła zaleca konwersję `readonly` pól do nie tylko do odczytu, czyli do zapisu.</span><span class="sxs-lookup"><span data-stu-id="4411a-118">The rule recommends converting `readonly` fields to non-readonly, i.e. writable.</span></span> <span data-ttu-id="4411a-119">Oznaczenie takich pól struktury jako tylko do odczytu może prowadzić do nieoczekiwanego zachowania, ponieważ wartość przypisana do pola może ulec zmianie, gdy `this` jest przypisana poza konstruktorem.</span><span class="sxs-lookup"><span data-stu-id="4411a-119">Marking such struct fields as readonly can lead to unexpected behavior, as the value assigned to the field can change when `this` is assigned outside the constructor.</span></span>

## <a name="example"></a><span data-ttu-id="4411a-120">Przykład</span><span class="sxs-lookup"><span data-stu-id="4411a-120">Example</span></span>

```csharp
// Code with violations
struct MyStruct
{
    public readonly int Value;

    public MyStruct(int value)
    {
        Value = value;
    }

    public void Test()
    {
        this = new MyStruct(5);
    }
}

// Fixed code
struct MyStruct
{
    public int Value;

    public MyStruct(int value)
    {
        Value = value;
    }

    public void Test()
    {
        this = new MyStruct(5);
    }
}
```
