## <a name="rule-description"></a><span data-ttu-id="f8b80-113">Opis reguły</span><span class="sxs-lookup"><span data-stu-id="f8b80-113">Rule description</span></span>

<span data-ttu-id="f8b80-114">Reguła CA1507 flaguje użycie `string` literału jako argumentu metody lub konstruktora, gdzie wyrażenie [nameof](../../../csharp/language-reference/operators/nameof.md) ( `NameOf` w Visual Basic) mogłoby zwiększyć łatwość utrzymania.</span><span class="sxs-lookup"><span data-stu-id="f8b80-114">Rule CA1507 flags the use of a `string` literal as an argument to a method or constructor where a [nameof](../../../csharp/language-reference/operators/nameof.md) (`NameOf` in Visual Basic) expression would add maintainability.</span></span> <span data-ttu-id="f8b80-115">Reguła jest wyzwalana po spełnieniu wszystkich następujących warunków:</span><span class="sxs-lookup"><span data-stu-id="f8b80-115">The rule fires if all of the following conditions are met:</span></span>

- <span data-ttu-id="f8b80-116">Argument jest `string` literałem lub stałą.</span><span class="sxs-lookup"><span data-stu-id="f8b80-116">The argument is a `string` literal or constant.</span></span>

- <span data-ttu-id="f8b80-117">Argument odnosi się do `string` parametru-type metody lub konstruktora, który jest wywoływany (oznacza to, że nie ma żadnej konwersji w miejscu wywołania).</span><span class="sxs-lookup"><span data-stu-id="f8b80-117">The argument corresponds to a `string`-typed parameter of the method or the constructor that's being invoked (that is, there is no conversion involved at the call site).</span></span>

  - <span data-ttu-id="f8b80-119">Zadeklarowana Nazwa parametru jest `paramName` i stała wartość `string` literału jest zgodna z nazwą parametru metody, lambda lub funkcji lokalnej, w której wywoływana jest metoda lub Konstruktor.</span><span class="sxs-lookup"><span data-stu-id="f8b80-119">The declared name of the parameter is `paramName` and the constant value of the `string` literal matches the name of a parameter of the method, lambda, or local function within which the method or constructor is being invoked.</span></span>

  - <span data-ttu-id="f8b80-120">Zadeklarowana Nazwa parametru jest `propertyName` i stała wartość `string` literału pasuje do nazwy właściwości typu, w której wywoływana jest metoda lub Konstruktor.</span><span class="sxs-lookup"><span data-stu-id="f8b80-120">The declared name of the parameter is `propertyName` and the constant value of the `string` literal matches the name of a property of the type within which the method or constructor is being invoked.</span></span>

<span data-ttu-id="f8b80-121">Reguła CA1507 zwiększa łatwość utrzymania kodu w przypadkach, w których można zmienić nazwę parametru w przyszłości, ale `string` nie zmieniono nazwy tego literału.</span><span class="sxs-lookup"><span data-stu-id="f8b80-121">Rule CA1507 improves code maintainability in cases where the parameter may be renamed in the future, but the `string` literal is mistakenly not renamed.</span></span> <span data-ttu-id="f8b80-122">Przy użyciu `nameof` , symbol zostanie zmieniony w przypadku zmiany nazwy parametru przez operację refaktoryzacji.</span><span class="sxs-lookup"><span data-stu-id="f8b80-122">By using `nameof`, the symbol will be renamed when the parameter is renamed through a refactoring operation.</span></span> <span data-ttu-id="f8b80-123">Ponadto wszystkie błędy pisowni w nazwie parametru są przechwytywane przez kompilator.</span><span class="sxs-lookup"><span data-stu-id="f8b80-123">In addition, any spelling mistakes in the name of the parameter are caught by the compiler.</span></span>

## <a name="how-to-fix-violations"></a><span data-ttu-id="f8b80-124">Jak naprawić naruszenia</span><span class="sxs-lookup"><span data-stu-id="f8b80-124">How to fix violations</span></span>

<span data-ttu-id="f8b80-125">Aby naprawić naruszenie, Zastąp `string` literał wyrażeniem [nameof](../../../csharp/language-reference/operators/nameof.md) ( `NameOf` w Visual Basic).</span><span class="sxs-lookup"><span data-stu-id="f8b80-125">To fix a violation, replace the `string` literal with a [nameof](../../../csharp/language-reference/operators/nameof.md) (`NameOf` in Visual Basic) expression.</span></span> <span data-ttu-id="f8b80-126">Na przykład następujące dwa fragmenty kodu pokazują naruszenie reguły i sposób jej naprawy:</span><span class="sxs-lookup"><span data-stu-id="f8b80-126">For example, the following two code snippets show a violation of the rule and how to fix it:</span></span>

```csharp
public Book(string title)
{
    // Violates rule CA1507
    Title = title ?? throw new ArgumentNullException("title", "All books must have a title.");
}
```

```csharp
public Book(string title)
{
    // Resolves rule CA1507 violation
    Title = title ?? throw new ArgumentNullException(nameof(title), "All books must have a title.");
}
```
