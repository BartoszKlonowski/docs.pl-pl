## <a name="rule-description"></a><span data-ttu-id="8bc07-113">Opis reguły</span><span class="sxs-lookup"><span data-stu-id="8bc07-113">Rule description</span></span>

<span data-ttu-id="8bc07-114">Kierowanie programu `StringBuilder` zawsze tworzy natywną kopię buforu, powodującą wiele alokacji dla jednego wywołania P/Invoke.</span><span class="sxs-lookup"><span data-stu-id="8bc07-114">Marshaling of `StringBuilder` always creates a native buffer copy, resulting in multiple allocations for one P/Invoke call.</span></span> <span data-ttu-id="8bc07-115">W celu zorganizowania `StringBuilder` jako parametru P/Invoke środowisko uruchomieniowe wykona następujące działania:</span><span class="sxs-lookup"><span data-stu-id="8bc07-115">To marshal a `StringBuilder` as a P/Invoke parameter, the runtime will:</span></span>

- <span data-ttu-id="8bc07-116">Przydziel bufor macierzysty.</span><span class="sxs-lookup"><span data-stu-id="8bc07-116">Allocate a native buffer.</span></span>
- <span data-ttu-id="8bc07-117">Jeśli jest `In` parametrem, skopiuj zawartość `StringBuilder` do bufora macierzystego.</span><span class="sxs-lookup"><span data-stu-id="8bc07-117">If it is an `In` parameter, copy the contents of the `StringBuilder` to the native buffer.</span></span>
- <span data-ttu-id="8bc07-118">Jeśli jest `Out` parametrem, skopiuj bufor macierzysty do nowo przydzielonej zarządzanej tablicy.</span><span class="sxs-lookup"><span data-stu-id="8bc07-118">If it is an `Out` parameter, copy the native buffer into a newly allocated managed array.</span></span>

<span data-ttu-id="8bc07-119">Domyślnie `StringBuilder` jest `In` i `Out` .</span><span class="sxs-lookup"><span data-stu-id="8bc07-119">By default, `StringBuilder` is `In` and `Out`.</span></span>

<span data-ttu-id="8bc07-120">Ta reguła jest domyślnie wyłączona, ponieważ może wymagać analizy wielkości liter w przypadku, gdy naruszenie jest istotne i potencjalnie nieprosta refaktoryzacji, aby rozwiązać naruszenie.</span><span class="sxs-lookup"><span data-stu-id="8bc07-120">This rule is disabled by default, because it can require case-by-case analysis of whether the violation is of interest and potentially non-trivial refactoring to address the violation.</span></span> <span data-ttu-id="8bc07-121">Użytkownicy mogą jawnie włączyć tę regułę przez [skonfigurowanie jej ważności](../configuration-options.md#severity-level).</span><span class="sxs-lookup"><span data-stu-id="8bc07-121">Users can explicitly enable this rule by [configuring its severity](../configuration-options.md#severity-level).</span></span>

## <a name="how-to-fix-violations"></a><span data-ttu-id="8bc07-122">Jak naprawić naruszenia</span><span class="sxs-lookup"><span data-stu-id="8bc07-122">How to fix violations</span></span>

<span data-ttu-id="8bc07-123">Ogólnie rzecz biorąc, odnoszące się do naruszenia obejmuje przeznaczenie elementu P/Invoke i jego obiektów wywołujących w celu użycia buforu zamiast `StringBuilder` .</span><span class="sxs-lookup"><span data-stu-id="8bc07-123">In general, addressing a violation involves reworking the P/Invoke and its callers to use a buffer instead of `StringBuilder`.</span></span> <span data-ttu-id="8bc07-124">Te szczegóły zależą od przypadków użycia dla elementu P/Invoke.</span><span class="sxs-lookup"><span data-stu-id="8bc07-124">The specifics would depend on the use cases for the P/Invoke.</span></span>

<span data-ttu-id="8bc07-125">Oto przykład typowego scenariusza używania `StringBuilder` jako bufor wyjściowy do wypełnienia przez funkcję natywną:</span><span class="sxs-lookup"><span data-stu-id="8bc07-125">Here is an example for the common scenario of using `StringBuilder` as an output buffer to be filled by the native function:</span></span>

```csharp
// Violation
[DllImport("MyLibrary", CharSet = CharSet.Unicode)]
private static extern void Foo(StringBuilder sb, ref int length);

public void Bar()
{
    int BufferSize = ...
    StringBuilder sb = new StringBuilder(BufferSize);
    int len = sb.Capacity;
    Foo(sb, ref len);
    string result = sb.ToString();
}
```

<span data-ttu-id="8bc07-126">W przypadku użycia, gdy bufor jest mały `unsafe` , a kod jest akceptowalny, [stackalloc](../../../csharp/language-reference/operators/stackalloc.md) może być użyty do przydzielenia buforu na stosie:</span><span class="sxs-lookup"><span data-stu-id="8bc07-126">For use cases where the buffer is small and `unsafe` code is acceptable, [stackalloc](../../../csharp/language-reference/operators/stackalloc.md) can be used to allocate the buffer on the stack:</span></span>

```csharp
[DllImport("MyLibrary", CharSet = CharSet.Unicode)]
private static extern unsafe void Foo(char* buffer, ref int length);

public void Bar()
{
    int BufferSize = ...
    unsafe
    {
        char* buffer = stackalloc char[BufferSize];
        int len = BufferSize;
        Foo(buffer, ref len);
        string result = new string(buffer);
    }
}
```

<span data-ttu-id="8bc07-127">W przypadku większych buforów Nowa tablica może być przypisana jako bufor:</span><span class="sxs-lookup"><span data-stu-id="8bc07-127">For larger buffers, a new array can be allocated as the buffer:</span></span>

```csharp
[DllImport("MyLibrary", CharSet = CharSet.Unicode)]
private static extern void Foo([Out] char[] buffer, ref int length);

public void Bar()
{
    int BufferSize = ...
    char[] buffer = new char[BufferSize];
    int len = buffer.Length;
    Foo(buffer, ref len);
    string result = new string(buffer);
}
```

<span data-ttu-id="8bc07-128">Gdy P/Invoke jest często wywoływana dla większych buforów, <xref:System.Buffers.ArrayPool%601> można użyć, aby uniknąć powtarzających się alokacji i niedostatecznego wykorzystania pamięci:</span><span class="sxs-lookup"><span data-stu-id="8bc07-128">When the P/Invoke is frequently called for larger buffers, <xref:System.Buffers.ArrayPool%601> can be used to avoid the repeated allocations and memory pressure that comes with them:</span></span>

```csharp
[DllImport("MyLibrary", CharSet = CharSet.Unicode)]
private static extern unsafe void Foo([Out] char[] buffer, ref int length);

public void Bar()
{
    int BufferSize = ...
    char[] buffer = ArrayPool<char>.Shared.Rent(BufferSize);
    try
    {
        int len = buffer.Length;
        Foo(buffer, ref len);
        string result = new string(buffer);
    }
    finally
    {
        ArrayPool<char>.Shared.Return(buffer);
    }
}
```

<span data-ttu-id="8bc07-129">Jeśli rozmiar buforu nie jest znany do czasu wykonania, może być konieczne utworzenie buforu w różny sposób w zależności od rozmiaru, aby uniknąć przydziału dużych buforów z `stackalloc` .</span><span class="sxs-lookup"><span data-stu-id="8bc07-129">If the buffer size is not known until runtime, the buffer may need to be created differently based on the size to avoid allocating large buffers with `stackalloc`.</span></span>

<span data-ttu-id="8bc07-130">W powyższych przykładach użyto znaków 2-bajtowych ( `CharSet.Unicode` ).</span><span class="sxs-lookup"><span data-stu-id="8bc07-130">The preceding examples use 2-byte wide characters (`CharSet.Unicode`).</span></span> <span data-ttu-id="8bc07-131">Jeśli funkcja natywna używa znaków 1-bajtowych ( `CharSet.Ansi` ), `byte` można użyć buforu zamiast `char` buforu.</span><span class="sxs-lookup"><span data-stu-id="8bc07-131">If the native function uses 1-byte characters (`CharSet.Ansi`), a `byte` buffer can be used instead of a `char` buffer.</span></span> <span data-ttu-id="8bc07-132">Na przykład:</span><span class="sxs-lookup"><span data-stu-id="8bc07-132">For example:</span></span>

```csharp
[DllImport("MyLibrary", CharSet = CharSet.Ansi)]
private static extern unsafe void Foo(byte* buffer, ref int length);

public void Bar()
{
    int BufferSize = ...
    unsafe
    {
        byte* buffer = stackalloc byte[BufferSize];
        int len = BufferSize;
        Foo(buffer, ref len);
        string result = Marshal.PtrToStringAnsi((IntPtr)buffer);
    }
}
```

<span data-ttu-id="8bc07-133">Jeśli parametr jest również używany jako dane wejściowe, bufory muszą być wypełnione danymi ciągu z dodanym terminatorem o wartości null.</span><span class="sxs-lookup"><span data-stu-id="8bc07-133">If the parameter is also used as input, the buffers need to be populated with the string data with any null terminator explicitly added.</span></span>
