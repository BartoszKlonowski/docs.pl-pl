## <a name="rule-description"></a><span data-ttu-id="f6f4d-115">Opis reguły</span><span class="sxs-lookup"><span data-stu-id="f6f4d-115">Rule description</span></span>

<span data-ttu-id="f6f4d-116">Metody mogą mieć kod warunkowy, taki jak instrukcje if, wyrażenia binarne ( `==` , `!=` ,, `<` `>` ), sprawdzenia wartości null itd. Rozważmy na przykład następujący kod:</span><span class="sxs-lookup"><span data-stu-id="f6f4d-116">Methods can have conditional code, such as if statements, binary expressions (`==`, `!=`, `<`, `>`), null checks, etc. For example, consider the following code:</span></span>

```csharp
public void M(int i, int j)
{
    if (i != 0)
    {
        return;
    }

    if (j != 0)
    {
        return;
    }

    // Below condition will always evaluate to 'false' as 'i' and 'j' are both '0' here.
    if (i != j)
    {
        // Code in this 'if' branch is dead code.
        // It can either be removed or refactored.
        ...
    }
}
```

<span data-ttu-id="f6f4d-117">Kompilatory C# i VB wykonują analizę kontroli warunkowej obejmującej _wartości stałe_ czasu kompilacji, które zawsze są oceniane do `true` lub `false` .</span><span class="sxs-lookup"><span data-stu-id="f6f4d-117">C# and VB compilers perform analysis of conditional checks involving compile-time _constant values_ that always evaluate to `true` or `false`.</span></span> <span data-ttu-id="f6f4d-118">Ten Analizator wykonuje analizę przepływu danych zmiennych niestałych, aby określić nadmiarowe kontrole warunkowe obejmujące _wartości niestałe_.</span><span class="sxs-lookup"><span data-stu-id="f6f4d-118">This analyzer performs data flow analysis of non-constant variables to determine redundant conditional checks involving _non-constant values_.</span></span> <span data-ttu-id="f6f4d-119">W poprzednim kodzie Analizator określa, że `i` i `j` są zarówno `0` dla wszystkich ścieżek kodu, które docierają do `i != j` sprawdzenia.</span><span class="sxs-lookup"><span data-stu-id="f6f4d-119">In the preceding code, the analyzer determines that `i` and `j` are both `0` for all code paths that reach `i != j` check.</span></span> <span data-ttu-id="f6f4d-120">W związku z tym sprawdzanie będzie zawsze oceniane `false` w czasie wykonywania.</span><span class="sxs-lookup"><span data-stu-id="f6f4d-120">Hence, this check will always evaluate to `false` at runtime.</span></span> <span data-ttu-id="f6f4d-121">Kod wewnątrz instrukcji if to martwy kod i można go usunąć lub refaktoryzacji.</span><span class="sxs-lookup"><span data-stu-id="f6f4d-121">The code inside the if statement is dead code and can be removed or refactored.</span></span> <span data-ttu-id="f6f4d-122">Podobnie Analizator śledzi wartości null zmiennych i raportów nadmiarowe sprawdzenia wartości null.</span><span class="sxs-lookup"><span data-stu-id="f6f4d-122">Similarly, the analyzer tracks nullness of variables and reports redundant null checks.</span></span>
