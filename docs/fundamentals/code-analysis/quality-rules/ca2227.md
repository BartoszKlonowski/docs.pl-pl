<span data-ttu-id="7098b-115">Właściwość kolekcji zapisywalnej umożliwia użytkownikowi zastąpienie kolekcji zupełnie inną kolekcją.</span><span class="sxs-lookup"><span data-stu-id="7098b-115">A writable collection property allows a user to replace the collection with a completely different collection.</span></span> <span data-ttu-id="7098b-116">Właściwość tylko do odczytu uniemożliwia zamienienie kolekcji, ale nadal zezwala na ustawianie poszczególnych członków.</span><span class="sxs-lookup"><span data-stu-id="7098b-116">A read-only property stops the collection from being replaced, but still allows the individual members to be set.</span></span> <span data-ttu-id="7098b-117">Jeśli zastępowanie kolekcji jest celem, preferowanym wzorcem projektu jest dołączenie metody usuwania wszystkich elementów z kolekcji i metody ponownego wypełnienia kolekcji.</span><span class="sxs-lookup"><span data-stu-id="7098b-117">If replacing the collection is a goal, the preferred design pattern is to include a method to remove all the elements from the collection, and a method to repopulate the collection.</span></span> <span data-ttu-id="7098b-118">Zobacz <xref:System.Collections.ArrayList.Clear%2A> i <xref:System.Collections.ArrayList.AddRange%2A> metody klasy, <xref:System.Collections.ArrayList?displayProperty=fullName> Aby zapoznać się z przykładem tego wzorca.</span><span class="sxs-lookup"><span data-stu-id="7098b-118">See the <xref:System.Collections.ArrayList.Clear%2A> and <xref:System.Collections.ArrayList.AddRange%2A> methods of the <xref:System.Collections.ArrayList?displayProperty=fullName> class for an example of this pattern.</span></span>
