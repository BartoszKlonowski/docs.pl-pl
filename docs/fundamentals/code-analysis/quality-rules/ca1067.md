## <a name="how-to-fix-violations"></a><span data-ttu-id="2e645-118">Jak naprawić naruszenia</span><span class="sxs-lookup"><span data-stu-id="2e645-118">How to fix violations</span></span>

<span data-ttu-id="2e645-119">Aby naprawić naruszenie, Przesłoń <xref:System.Object.Equals%2A> i zaimplementuj je przez wywołanie <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> implementacji.</span><span class="sxs-lookup"><span data-stu-id="2e645-119">To fix a violation, override <xref:System.Object.Equals%2A> and implement it by invoking the <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> implementation.</span></span> <span data-ttu-id="2e645-120">Na przykład następujące dwa fragmenty kodu pokazują naruszenie reguły i sposób jej naprawy:</span><span class="sxs-lookup"><span data-stu-id="2e645-120">For example, the following two code snippets show a violation of the rule and how to fix it:</span></span>

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;
}
```

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;

    public override bool Equals(object obj)
        => obj is S objS && Equals(objS);

    public override int GetHashCode()
        => _value.GetHashCode();
}
```
