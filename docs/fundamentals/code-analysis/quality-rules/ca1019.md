## <a name="rule-description"></a><span data-ttu-id="42cd9-114">Opis reguły</span><span class="sxs-lookup"><span data-stu-id="42cd9-114">Rule description</span></span>

<span data-ttu-id="42cd9-115">Atrybuty mogą definiować obowiązkowe argumenty, które trzeba określić, aby móc zastosować atrybut do obiektu docelowego.</span><span class="sxs-lookup"><span data-stu-id="42cd9-115">Attributes can define mandatory arguments that must be specified when you apply the attribute to a target.</span></span> <span data-ttu-id="42cd9-116">Znane są również jako argumenty pozycyjne, ponieważ są one dostarczane do konstruktorów atrybutu jako parametry pozycyjne.</span><span class="sxs-lookup"><span data-stu-id="42cd9-116">These are also known as positional arguments because they are supplied to attribute constructors as positional parameters.</span></span> <span data-ttu-id="42cd9-117">Dla każdego obowiązkowego argumentu atrybut powinien również dostarczyć odpowiadającą właściwość tylko do odczytu, dzięki której można pobrać wartość argumentu w czasie wykonywania.</span><span class="sxs-lookup"><span data-stu-id="42cd9-117">For every mandatory argument, the attribute should also provide a corresponding read-only property so that the value of the argument can be retrieved at execution time.</span></span> <span data-ttu-id="42cd9-118">Ta reguła sprawdza, czy dla każdego parametru konstruktora zdefiniowano odpowiednią właściwość.</span><span class="sxs-lookup"><span data-stu-id="42cd9-118">This rule checks that for each constructor parameter, you have defined the corresponding property.</span></span>

<span data-ttu-id="42cd9-121">W przypadku argumentów obowiązkowych i opcjonalnych odpowiednie właściwości i konstruktory powinny mieć taką samą nazwę, ale różne wielkości liter.</span><span class="sxs-lookup"><span data-stu-id="42cd9-121">For mandatory and optional arguments, the corresponding properties and constructor parameters should use the same name but different casing.</span></span> <span data-ttu-id="42cd9-122">Właściwości używają wielkości liter w języku Pascal i parametry używają notacji CamelCase wielkości liter.</span><span class="sxs-lookup"><span data-stu-id="42cd9-122">Properties use Pascal casing, and parameters use camel casing.</span></span>
