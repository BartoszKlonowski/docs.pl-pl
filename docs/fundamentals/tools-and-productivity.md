# <a name="tools-and-diagnostics-in-net"></a>Narzędzia i Diagnostyka w programie .NET

Ten artykuł zawiera informacje o różnych narzędziach dostępnych dla deweloperów platformy .NET. W przypadku platformy .NET masz niezawodny zestaw SDK (Software Development Kit), który zawiera interfejs wiersza polecenia (CLI). Interfejs wiersza polecenia platformy .NET umożliwia korzystanie z wielu funkcji w ramach programu. Gotowe do zaprojektowania środowiska deweloperskie (środowisk IDE). Ten artykuł zawiera również zasoby wydajnościowe, takie jak narzędzia interfejsu wiersza polecenia platformy .NET służące do diagnozowania problemów z wydajnością, przecieków pamięci, dużego procesora, zakleszczenii i obsługi narzędzi do analizy kodu.

## <a name="net-sdk"></a>Zestaw SDK .NET

Zestaw SDK platformy .NET obejmuje środowisko uruchomieniowe .NET i interfejs wiersza polecenia platformy .NET. [Zestaw .NET SDK](https://dotnet.microsoft.com/download) dla systemu Windows, Linux, MacOS lub Docker można pobrać. Aby uzyskać więcej informacji, zobacz [Omówienie zestawu SDK platformy .NET](../core/sdk.md).

## <a name="net-cli"></a>INTERFEJS WIERSZA POLECENIA PLATFORMY .NET

Interfejs wiersza polecenia platformy .NET to Międzyplatformowy łańcucha narzędzi służący do tworzenia, tworzenia, uruchamiania i publikowania aplikacji platformy .NET. Interfejs wiersza polecenia platformy .NET jest dołączony do zestawu .NET SDK. Aby uzyskać więcej informacji, zobacz [Omówienie interfejsu wiersza polecenia platformy .NET](../core/tools/index.md).

## <a name="ides"></a>Środowiska IDE

Aplikacje .NET można pisać w [Visual Studio Code](https://code.visualstudio.com/docs), [Visual Studio](/visualstudio/windows)lub [Visual Studio dla komputerów Mac](/visualstudio/mac). Aby uzyskać informacje na temat środowisk programistycznych opartych na chmurze, zobacz [Visual Studio Codespaces](/visualstudio/codespaces/overview/what-is-vsonline).

## <a name="additional-tools"></a>Dodatkowe narzędzia

Oprócz bardziej popularnych narzędzi środowisko .NET oferuje także narzędzia dla konkretnych scenariuszy. Niektóre przypadki użycia obejmują Odinstalowywanie zestawu .NET SDK lub środowisko uruchomieniowe platformy .NET, Pobieranie metadanych Windows Communication Foundation (WCF), generowanie kodu źródłowego serwera proxy i Serializowanie kodu XML. Aby uzyskać więcej informacji, zobacz temat [dodatkowe narzędzia platformy .NET — Omówienie](../core/additional-tools/index.md).

## <a name="diagnostics-and-instrumentation"></a>Diagnostyka i instrumentacja

Jako deweloper platformy .NET możesz używać popularnych narzędzi diagnostycznych wydajności do monitorowania wydajności aplikacji, profilowania aplikacji, śledzenia, zbierania metryk wydajności i analizowania plików zrzutów. Można zbierać metryki wydajności z licznikami zdarzeń i używać narzędzi profilowania do uzyskiwania szczegółowych informacji o sposobie wykonywania aplikacji. Aby uzyskać więcej informacji, zobacz [Narzędzia diagnostyczne programu .NET](../core/diagnostics/index.md).

## <a name="code-analysis"></a>Analiza kodu

Analizatory platformy kompilatora .NET (Roslyn) sprawdzają kod w języku C# lub Visual Basic w przypadku problemów z jakością kodu i stylem kodu. Aby uzyskać więcej informacji, zobacz [Omówienie analizy kodu źródłowego platformy .NET](code-analysis/overview.md).
