<span data-ttu-id="aba33-104">Funkcja WCF ma wiele różnych wbudowanych *powiązań* , które określają różne protokoły sieciowe, formaty przewodowe i inne szczegóły implementacji.</span><span class="sxs-lookup"><span data-stu-id="aba33-104">WCF has lots of different built-in *bindings* that specify different network protocols, wire formats, and other implementation details.</span></span> <span data-ttu-id="aba33-105">gRPC wydajnie ma tylko jeden protokół sieciowy i jeden format sieci (technicznie można *dostosować format przewodu* , ale jest on poza zakresem tej książki).</span><span class="sxs-lookup"><span data-stu-id="aba33-105">gRPC effectively has just one network protocol and one wire format (technically the wire format *may* be customized, but that is beyond the scope of this book).</span></span> <span data-ttu-id="aba33-106">Prawdopodobnie odkryjesz, że gRPC oferuje najlepsze rozwiązanie w większości przypadków.</span><span class="sxs-lookup"><span data-stu-id="aba33-106">You are likely to discover that gRPC offers the best solution in most cases.</span></span> <span data-ttu-id="aba33-107">Poniżej przedstawiono krótką dyskusję na temat najbardziej odpowiednich powiązań WCF oraz sposób ich porównania z ich odpowiednikiem w gRPC.</span><span class="sxs-lookup"><span data-stu-id="aba33-107">What follows is a short discussion about the most relevant WCF bindings and how they compare to their equivalent in gRPC.</span></span>

## <a name="nettcp"></a><span data-ttu-id="aba33-108">NetTCP</span><span class="sxs-lookup"><span data-stu-id="aba33-108">NetTCP</span></span>

<span data-ttu-id="aba33-109">Powiązanie NetTCP programu WCF umożliwia tworzenie trwałych połączeń, małych komunikatów i komunikatów dwukierunkowych, ale tylko między klientami i serwerami platformy .NET.</span><span class="sxs-lookup"><span data-stu-id="aba33-109">WCF's NetTCP binding allows for persistent connections, small messages, and two-way messaging but only works between .NET clients and servers.</span></span> <span data-ttu-id="aba33-110">gRPC umożliwia korzystanie z tych samych funkcji, ale jest obsługiwana w wielu językach programowania i platformach.</span><span class="sxs-lookup"><span data-stu-id="aba33-110">gRPC allows the same functionality but is supported across multiple programming languages and platforms.</span></span> <span data-ttu-id="aba33-111">gRPC ma wiele funkcji powiązania NetTCP WCF, chociaż nie zawsze są zaimplementowane w ten sam sposób.</span><span class="sxs-lookup"><span data-stu-id="aba33-111">gRPC has many of the features of WCF NetTCP binding, although not always implemented in the same way.</span></span> <span data-ttu-id="aba33-112">Na przykład w programie WCF szyfrowanie jest kontrolowane przez konfigurację i obsługiwane w strukturze.</span><span class="sxs-lookup"><span data-stu-id="aba33-112">For example, in WCF, encryption is controlled through configuration and handled in the framework.</span></span> <span data-ttu-id="aba33-113">W programie gRPC szyfrowanie jest uzyskiwane na poziomie połączenia przy użyciu protokołu HTTP/2 za pośrednictwem protokołu TLS.</span><span class="sxs-lookup"><span data-stu-id="aba33-113">In gRPC, encryption is achieved at the connection level using HTTP/2 over TLS.</span></span>

## <a name="http"></a><span data-ttu-id="aba33-114">HTTP</span><span class="sxs-lookup"><span data-stu-id="aba33-114">HTTP</span></span>

<span data-ttu-id="aba33-115">BasicHttpBinding WCF jest zwykle oparty na tekście, przy użyciu protokołu SOAP jako formatu sieci i jest powolne w porównaniu do powiązania NetTCP.</span><span class="sxs-lookup"><span data-stu-id="aba33-115">WCF's BasicHttpBinding is usually text based, using SOAP as the wire format, and is slow compared to the NetTCP binding.</span></span> <span data-ttu-id="aba33-116">Jest on zazwyczaj używany do zapewnienia współdziałania międzyplatformowego lub połączenia przez infrastrukturę internetową.</span><span class="sxs-lookup"><span data-stu-id="aba33-116">It's generally used to provide cross-platform interoperability, or connection over internet infrastructure.</span></span> <span data-ttu-id="aba33-117">Odpowiednik w gRPC — ponieważ używa protokołu HTTP/2 jako podstawowej warstwy transportu z binarnym formatem sieci protobuf dla komunikatów — może oferować NetTCP wydajność na poziomie usług, ale z pełną współdziałaniem międzyplatformowym ze wszystkimi nowoczesnymi językami programowania i platform.</span><span class="sxs-lookup"><span data-stu-id="aba33-117">The equivalent in gRPC—because it uses HTTP/2 as the underlying transport layer with the binary Protobuf wire format for messages—can offer NetTCP service level performance but with full cross-platform interoperability with all modern programming languages and frameworks.</span></span>

## <a name="named-pipes"></a><span data-ttu-id="aba33-118">Nazwane potoki</span><span class="sxs-lookup"><span data-stu-id="aba33-118">Named Pipes</span></span>

<span data-ttu-id="aba33-119">Funkcja WCF dostarczyła powiązanie nazwanych potoków na potrzeby komunikacji między procesami na tym samym komputerze fizycznym.</span><span class="sxs-lookup"><span data-stu-id="aba33-119">WCF provided a Named Pipes binding for communication between processes on the same physical machine.</span></span> <span data-ttu-id="aba33-120">Nazwane potoki nie są obsługiwane przez pierwszą wersję ASP.NET Core gRPC.</span><span class="sxs-lookup"><span data-stu-id="aba33-120">Named pipes are not supported by the first release of ASP.NET Core gRPC.</span></span> <span data-ttu-id="aba33-121">Dodawanie obsługi klienta i serwera dla potoków nazwanych (i gniazd domen systemu UNIX) jest celem przyszłej wersji.</span><span class="sxs-lookup"><span data-stu-id="aba33-121">Adding client and server support for named pipes (and Unix domain sockets) is a goal for a future release.</span></span>

## <a name="msmq"></a><span data-ttu-id="aba33-122">Usługa MSMQ</span><span class="sxs-lookup"><span data-stu-id="aba33-122">MSMQ</span></span>

<span data-ttu-id="aba33-123">Usługa MSMQ jest zastrzeżoną kolejką komunikatów systemu Windows.</span><span class="sxs-lookup"><span data-stu-id="aba33-123">MSMQ is a proprietary Windows message queue.</span></span> <span data-ttu-id="aba33-124">Powiązanie WCF z usługą MSMQ umożliwia "pożar i zapomnij" żądań od klientów, które mogą być przetwarzane w dowolnym momencie w przyszłości.</span><span class="sxs-lookup"><span data-stu-id="aba33-124">WCF's binding to MSMQ enables "fire and forget" requests from clients that may be processed at any time in the future.</span></span> <span data-ttu-id="aba33-125">gRPC nie zapewnia natywnej żadnej funkcjonalności kolejki komunikatów.</span><span class="sxs-lookup"><span data-stu-id="aba33-125">gRPC doesn't natively provide any message queue functionality.</span></span> <span data-ttu-id="aba33-126">Najlepszą alternatywą byłoby bezpośrednie użycie systemu obsługi komunikatów, takiego jak Azure Service Bus, RabbitMQ lub Kafka.</span><span class="sxs-lookup"><span data-stu-id="aba33-126">The best alternative would be to directly use a messaging system like Azure Service Bus, RabbitMQ, or Kafka.</span></span> <span data-ttu-id="aba33-127">Może to zostać zaimplementowane przy użyciu klienta umieszczania komunikatów bezpośrednio w kolejce lub gRPC usługi przesyłania strumieniowego klienta, która enqueues komunikaty.</span><span class="sxs-lookup"><span data-stu-id="aba33-127">This could be implemented with the client placing messages directly onto the queue, or a gRPC client streaming service that enqueues the messages.</span></span>

<span data-ttu-id="aba33-129">Element WebHttpBinding (znany również jako ReST WCF), z atrybutami `WebGet` i `WebInvoke`, umożliwia opracowywanie interfejsów API ReSTful, które mogą wypowiadać dane JSON w czasie, gdy były one mniej powszechne niż teraz.</span><span class="sxs-lookup"><span data-stu-id="aba33-129">The WebHttpBinding (also known as WCF ReST), with the `WebGet` and `WebInvoke` attributes, enabled you to develop ReSTful APIs that could speak JSON at a time when this was less common than now.</span></span> <span data-ttu-id="aba33-130">W związku z tym, jeśli masz interfejs API RESTful utworzony za pomocą usługi WCF REST, rozważ migrację go do zwykłej aplikacji interfejsu API sieci Web MVC ASP.NET Core, która zapewni funkcję podobną, a nie konwertowanie na gRPC.</span><span class="sxs-lookup"><span data-stu-id="aba33-130">Therefore, if you have a RESTful API built with WCF REST, consider migrating it to a regular ASP.NET Core MVC Web API application, which would provide equivalent functionality, rather than converting to gRPC.</span></span>
