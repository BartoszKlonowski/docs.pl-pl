<span data-ttu-id="8e1f6-127">Kontrolki na stronie są zwykle umieszczane na tej samej stronie, która przedstawiła formant i przenoszą razem z nimi ładunek z pola ukrytego formularza o nazwie `ViewState`.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-127">Controls on a Page typically post-back to the same page that presented the control, and carry along with them a payload from a hidden form field called `ViewState`.</span></span> <span data-ttu-id="8e1f6-128">`ViewState` Pole zawiera informacje o stanie kontrolek w czasie, gdy zostały renderowane i przedstawione na stronie, co umożliwia środowisko uruchomieniowe ASP.NET w celu porównania i identyfikowania zmian zawartości przesłanej do serwera.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-128">The `ViewState` field contains information about the state of the controls at the time they were rendered and presented on the page, allowing the ASP.NET runtime to compare and identify changes in the content submitted to the server.</span></span>

## <a name="blazor"></a><span data-ttu-id="8e1f6-129">Blazor</span><span class="sxs-lookup"><span data-stu-id="8e1f6-129">Blazor</span></span>

<span data-ttu-id="8e1f6-130">Blazor to struktura interfejsu użytkownika sieci Web po stronie klienta podobna do platformy języka JavaScript, w której znajdują się środowiska frontonu, takie jak elementy kątowe lub reagowanie.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-130">Blazor is a client-side web UI framework similar in nature to JavaScript front-end frameworks like Angular or React.</span></span> <span data-ttu-id="8e1f6-131">Blazor obsługuje interakcje użytkowników i renderuje niezbędne aktualizacje interfejsu użytkownika.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-131">Blazor handles user interactions and renders the necessary UI updates.</span></span> <span data-ttu-id="8e1f6-132">Blazor *nie jest* oparta na modelu żądanie-odpowiedź.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-132">Blazor *isn't* based on a request-reply model.</span></span> <span data-ttu-id="8e1f6-133">Interakcje użytkownika są obsługiwane jako zdarzenia, które nie znajdują się w kontekście określonego żądania HTTP.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-133">User interactions are handled as events that aren't in the context of any particular HTTP request.</span></span>

<span data-ttu-id="8e1f6-134">Aplikacje Blazor składają się z jednego lub większej liczby składników głównych, które są renderowane na stronie HTML.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-134">Blazor apps consist of one or more root components that are rendered on an HTML page.</span></span>

<span data-ttu-id="8e1f6-136">Sposób, w jaki użytkownik określa, gdzie mają być renderowane składniki, oraz jak są rozłączone składniki, aby Interakcje użytkownika były [hostem](hosting-models.md) specyficznym dla modelu.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-136">How the user specifies where components should render and how the components are then wired up for user interactions is [hosting model](hosting-models.md) specific.</span></span>

<span data-ttu-id="8e1f6-137">[Składniki](components.md) Blazor są klasami .NET, które reprezentują element interfejsu użytkownika wielokrotnego użytku.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-137">Blazor [components](components.md) are .NET classes that represent a reusable piece of UI.</span></span> <span data-ttu-id="8e1f6-138">Każdy składnik utrzymuje własny stan i określa własną logikę renderowania, która może obejmować renderowanie innych składników.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-138">Each component maintains its own state and specifies its own rendering logic, which can include rendering other components.</span></span> <span data-ttu-id="8e1f6-139">Składniki określają programy obsługi zdarzeń dla konkretnych interakcji użytkowników w celu zaktualizowania stanu składnika.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-139">Components specify event handlers for specific user interactions to update the component's state.</span></span>

<span data-ttu-id="8e1f6-140">Gdy składnik obsługuje zdarzenie, Blazor renderuje składnik i śledzi zmiany w renderowanych danych wyjściowych.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-140">After a component handles an event, Blazor renders the component and keeps track of what changed in the rendered output.</span></span> <span data-ttu-id="8e1f6-141">Składniki nie są renderowane bezpośrednio do Document Object Model (DOM).</span><span class="sxs-lookup"><span data-stu-id="8e1f6-141">Components don't render directly to the Document Object Model (DOM).</span></span> <span data-ttu-id="8e1f6-142">Zamiast tego są one renderowane do reprezentacji w pamięci modelu dom o nazwie `RenderTree` a, tak aby Blazor można było śledzić zmiany.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-142">They instead render to an in-memory representation of the DOM called a `RenderTree` so that Blazor can track the changes.</span></span> <span data-ttu-id="8e1f6-143">Blazor porównuje nowo renderowane dane wyjściowe z poprzednimi danymi wyjściowymi, aby obliczyć różnice między interfejsem użytkownika, który następnie jest wydajnie stosowany do modelu DOM.</span><span class="sxs-lookup"><span data-stu-id="8e1f6-143">Blazor compares the newly rendered output with the previous output to calculate a UI diff that it then applies efficiently to the DOM.</span></span>
