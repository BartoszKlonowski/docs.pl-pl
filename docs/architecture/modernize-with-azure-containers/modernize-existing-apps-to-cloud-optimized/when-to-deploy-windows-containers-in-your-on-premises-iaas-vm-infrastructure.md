# <a name="when-to-deploy-windows-containers-in-your-on-premises-iaas-vm-infrastructure"></a>Kiedy należy wdrażać kontenery systemu Windows w lokalnej infrastrukturze maszyn wirtualnych IaaS

- Twoja organizacja może nie być gotowa do przejścia do chmury lub może nie być w stanie przejść do chmury w celu uzyskania przyczyny biznesowej. Jednak nadal możesz korzystać z kontenerów systemu Windows w swoich centrach danych.

- Mogą istnieć inne artefakty, które są używane lokalnie, i które mogą spowalniać pracę podczas próby przejścia do chmury. Na przykład zależności zabezpieczeń lub uwierzytelniania związane z lokalnym systemem Windows Server Active Directory lub dowolnym innym zasobem lokalnym.

- Jeśli już dziś zaczynasz korzystać z kontenerów systemu Windows, możesz dokonać migracji etapowej do chmury jutro od dużo lepszego stanowiska. Kontenery systemu Windows stają się jednostką wdrożenia dla każdej chmury bez żadnej blokady.
