# <a name="when-to-deploy-windows-containers-in-your-on-premises-iaas-vm-infrastructure"></a>Kiedy należy wdrażać kontenery systemu Windows w lokalnej infrastrukturze maszyn wirtualnych IaaS

- Organizacja może nie być gotowa do przejścia do chmury lub może nie być w stanie przenieść się do chmury z przyczyn biznesowych. Jednak nadal można uzyskać korzyści z używania kontenerów systemu Windows we własnych centrach danych.

- Może mieć inne artefakty, które są używane lokalnie i które mogą spowolnić podczas próby przejścia do chmury. Na przykład zależności zabezpieczeń lub uwierzytelniania z lokalną usługą Active Directory systemu Windows Server lub innym zasobem lokalnym.

- Jeśli zaczniesz używać kontenerów systemu Windows już dziś, możesz jutro wprowadzić migrację etapową do chmury z dużo lepszej pozycji. Kontenery systemu Windows staje się jednostką wdrażania dla każdej chmury, bez blokady.
