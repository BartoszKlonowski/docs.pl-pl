### <a name="encapsulation"></a><span data-ttu-id="f9973-119">Hermetyzacja</span><span class="sxs-lookup"><span data-stu-id="f9973-119">Encapsulation</span></span>

### <a name="explicit-dependencies"></a><span data-ttu-id="f9973-143">Jawne zależności</span><span class="sxs-lookup"><span data-stu-id="f9973-143">Explicit dependencies</span></span>

> <span data-ttu-id="f9973-168">Należy unikać wiązania razem zachowanie, które jest tylko przypadkowo powtarzalne.</span><span class="sxs-lookup"><span data-stu-id="f9973-168">Avoid binding together behavior that is only coincidentally repetitive.</span></span> <span data-ttu-id="f9973-169">Na przykład tylko dlatego, że dwie różne stałe mają tę samą wartość, nie oznacza to, że powinieneś mieć tylko jedną stałą, jeśli koncepcyjnie odnoszą się do różnych rzeczy.</span><span class="sxs-lookup"><span data-stu-id="f9973-169">For example, just because two different constants both have the same value, that doesn't mean you should have only one constant, if conceptually they're referring to different things.</span></span>

### <a name="persistence-ignorance"></a><span data-ttu-id="f9973-170">Niewiedza o trwałości</span><span class="sxs-lookup"><span data-stu-id="f9973-170">Persistence ignorance</span></span>

<span data-ttu-id="f9973-171">**Nieznajomość trwałości** (PI) odnosi się do typów, które muszą być utrwaliłe, ale których kod nie ma wpływu na wybór technologii trwałości.</span><span class="sxs-lookup"><span data-stu-id="f9973-171">**Persistence ignorance** (PI) refers to types that need to be persisted, but whose code is unaffected by the choice of persistence technology.</span></span> <span data-ttu-id="f9973-172">Takie typy w .NET są czasami określane jako zwykły starych obiektów CLR (POC), ponieważ nie trzeba dziedziczyć z określonej klasy podstawowej lub implementować określonego interfejsu.</span><span class="sxs-lookup"><span data-stu-id="f9973-172">Such types in .NET are sometimes referred to as Plain Old CLR Objects (POCOs), because they do not need to inherit from a particular base class or implement a particular interface.</span></span> <span data-ttu-id="f9973-173">Nieznajomość trwałości jest cenne, ponieważ pozwala na ten sam model biznesowy, które mają być utrwalić na wiele sposobów, oferując dodatkową elastyczność do aplikacji.</span><span class="sxs-lookup"><span data-stu-id="f9973-173">Persistence ignorance is valuable because it allows the same business model to be persisted in multiple ways, offering additional flexibility to the application.</span></span> <span data-ttu-id="f9973-174">Opcje trwałości może się zmieniać wraz z uchwycenia, z jednej technologii bazy danych do innej lub dodatkowe formy trwałości może być wymagane oprócz niezależnie od aplikacji rozpoczęte z (na przykład przy użyciu pamięci podręcznej Redis lub usługi Azure Cosmos DB oprócz relacyjnej bazy danych).</span><span class="sxs-lookup"><span data-stu-id="f9973-174">Persistence choices might change over time, from one database technology to another, or additional forms of persistence might be required in addition to whatever the application started with (for example, using a Redis cache or Azure Cosmos DB in addition to a relational database).</span></span>

<span data-ttu-id="f9973-175">Oto kilka przykładów naruszeń tej zasady:</span><span class="sxs-lookup"><span data-stu-id="f9973-175">Some examples of violations of this principle include:</span></span>

- <span data-ttu-id="f9973-176">Wymagana klasa podstawowa.</span><span class="sxs-lookup"><span data-stu-id="f9973-176">A required base class.</span></span>

- <span data-ttu-id="f9973-177">Wymagana implementacja interfejsu.</span><span class="sxs-lookup"><span data-stu-id="f9973-177">A required interface implementation.</span></span>

- <span data-ttu-id="f9973-178">Klasy odpowiedzialne za zapisywanie się (takie jak wzorzec rekordu aktywnego).</span><span class="sxs-lookup"><span data-stu-id="f9973-178">Classes responsible for saving themselves (such as the Active Record pattern).</span></span>

- <span data-ttu-id="f9973-179">Wymagany konstruktor bezparametrów.</span><span class="sxs-lookup"><span data-stu-id="f9973-179">Required parameterless constructor.</span></span>

- <span data-ttu-id="f9973-180">Właściwości wymagające wirtualnego słowa kluczowego.</span><span class="sxs-lookup"><span data-stu-id="f9973-180">Properties requiring virtual keyword.</span></span>

- <span data-ttu-id="f9973-181">Atrybuty wymagane do swoistego dla trwałości.</span><span class="sxs-lookup"><span data-stu-id="f9973-181">Persistence-specific required attributes.</span></span>

<span data-ttu-id="f9973-182">Wymóg, że klasy mają dowolną z powyższych funkcji lub zachowań dodaje sprzężenia między typami, które mają być utrwalić i wybór technologii trwałości, co utrudnia przyjęcie nowych strategii dostępu do danych w przyszłości.</span><span class="sxs-lookup"><span data-stu-id="f9973-182">The requirement that classes have any of the above features or behaviors adds coupling between the types to be persisted and the choice of persistence technology, making it more difficult to adopt new data access strategies in the future.</span></span>

### <a name="bounded-contexts"></a><span data-ttu-id="f9973-183">Ograniczone konteksty</span><span class="sxs-lookup"><span data-stu-id="f9973-183">Bounded contexts</span></span>

<span data-ttu-id="f9973-184">**Ograniczone konteksty** są centralnym wzorcem w projekcie opartym na domenie.</span><span class="sxs-lookup"><span data-stu-id="f9973-184">**Bounded contexts** are a central pattern in Domain-Driven Design.</span></span> <span data-ttu-id="f9973-185">Zapewniają one sposób radzenia sobie ze złożonością w dużych aplikacjach lub organizacjach, rozbijając ją na oddzielne moduły koncepcyjne.</span><span class="sxs-lookup"><span data-stu-id="f9973-185">They provide a way of tackling complexity in large applications or organizations by breaking it up into separate conceptual modules.</span></span> <span data-ttu-id="f9973-186">Każdy moduł koncepcyjny reprezentuje kontekst, który jest oddzielony od innych kontekstów (stąd, ograniczone) i może ewoluować niezależnie.</span><span class="sxs-lookup"><span data-stu-id="f9973-186">Each conceptual module then represents a context that is separated from other contexts (hence, bounded), and can evolve independently.</span></span> <span data-ttu-id="f9973-187">Każdy ograniczony kontekst powinien idealnie mieć swobodę wyboru własnych nazw dla pojęć w nim i powinien mieć wyłączny dostęp do własnego magazynu trwałości.</span><span class="sxs-lookup"><span data-stu-id="f9973-187">Each bounded context should ideally be free to choose its own names for concepts within it, and should have exclusive access to its own persistence store.</span></span>

<span data-ttu-id="f9973-188">Co najmniej poszczególnych aplikacji sieci web należy dążyć do własnego kontekstu ograniczone, z własnego magazynu trwałości dla ich modelu biznesowego, a nie udostępnianie bazy danych z innymi aplikacjami.</span><span class="sxs-lookup"><span data-stu-id="f9973-188">At a minimum, individual web applications should strive to be their own bounded context, with their own persistence store for their business model, rather than sharing a database with other applications.</span></span> <span data-ttu-id="f9973-189">Komunikacja między kontekstami ograniczonymi odbywa się za pośrednictwem interfejsów programowych, a nie za pośrednictwem udostępnionej bazy danych, co pozwala na logikę biznesową i zdarzenia, które mają miejsce w odpowiedzi na zmiany, które mają miejsce.</span><span class="sxs-lookup"><span data-stu-id="f9973-189">Communication between bounded contexts occurs through programmatic interfaces, rather than through a shared database, which allows for business logic and events to take place in response to changes that take place.</span></span> <span data-ttu-id="f9973-190">Ograniczone konteksty map ściśle do mikrousług, które również są idealnie zaimplementowane jako własne indywidualne ograniczone konteksty.</span><span class="sxs-lookup"><span data-stu-id="f9973-190">Bounded contexts map closely to microservices, which also are ideally implemented as their own individual bounded contexts.</span></span>
