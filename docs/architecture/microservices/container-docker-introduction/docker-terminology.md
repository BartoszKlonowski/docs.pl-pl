<span data-ttu-id="2d4ac-106">**Obraz kontenera:** pakiet ze wszystkimi zależnościami i informacjami potrzebnymi do utworzenia kontenera.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-106">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="2d4ac-107">Obraz zawiera wszystkie zależności (takie jak struktury) oraz konfiguracji wdrażania i wykonywania do użycia przez środowisko uruchomieniowe kontenera.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-107">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="2d4ac-108">Zazwyczaj obraz pochodzi z wielu obrazów podstawowych, które są warstwami ułożonymi jeden na drugim w celu utworzenia systemu plików kontenera.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-108">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container's filesystem.</span></span> <span data-ttu-id="2d4ac-109">Obraz jest niezmienny po jego utworzeniu.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-109">An image is immutable once it has been created.</span></span>

<span data-ttu-id="2d4ac-110">**Dockerfile**: Plik tekstowy zawierający instrukcje dotyczące tworzenia obrazu platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-110">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span> <span data-ttu-id="2d4ac-111">Jest jak skrypt wsadowy, pierwszy wiersz stwierdza obraz podstawowy na początek, a następnie postępuj zgodnie z instrukcjami, aby zainstalować wymagane programy, skopiować pliki i tak dalej, aż pojawi się środowisko pracy, którego potrzebujesz.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-111">It's like a batch script, the first line states the base image to begin with and then follow the instructions to install required programs, copy files and so on, until you get the working environment you need.</span></span>

<span data-ttu-id="2d4ac-112">**Kompilacja:** Akcja tworzenia obrazu kontenera na podstawie informacji i kontekstu dostarczonych przez jego Dockerfile, plus dodatkowe pliki w folderze, w którym jest zbudowany obraz.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-112">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="2d4ac-113">Obrazy można tworzyć za pomocą polecenia Docker:</span><span class="sxs-lookup"><span data-stu-id="2d4ac-113">You can build images with the Docker command:</span></span>

> `docker build`

<span data-ttu-id="2d4ac-114">**Kontener:** Wystąpienie obrazu platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-114">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="2d4ac-115">Kontener reprezentuje wykonanie pojedynczej aplikacji, procesu lub usługi.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-115">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="2d4ac-116">Składa się z zawartości obrazu platformy Docker, środowiska wykonywania i standardowego zestawu instrukcji.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-116">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="2d4ac-117">Podczas skalowania usługi, można utworzyć wiele wystąpień kontenera z tego samego obrazu.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-117">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="2d4ac-118">Lub zadanie wsadowe można utworzyć wiele kontenerów z tego samego obrazu, przekazując różne parametry do każdego wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-118">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="2d4ac-119">**Woluminy:** Zaoferuj zapisywalny system plików, którego kontener może używać.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-119">**Volumes**: Offer a writable filesystem that the container can use.</span></span> <span data-ttu-id="2d4ac-120">Ponieważ obrazy są tylko do odczytu, ale większość programów musi zapisywać do systemu plików, woluminy dodać zapisywalną warstwę, na górze obrazu kontenera, więc programy mają dostęp do zapisywalny system plików.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-120">Since images are read-only but most programs need to write to the filesystem, volumes add a writable layer, on top of the container image, so the programs have access to a writable filesystem.</span></span> <span data-ttu-id="2d4ac-121">Program nie wie, że uzyskuje dostęp do warstwowego systemu plików, to tylko system plików jak zwykle.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-121">The program doesn't know it is accessing a layered filesystem, it is just the filesystem as usual.</span></span> <span data-ttu-id="2d4ac-122">Woluminy są aktywne w systemie hosta i są zarządzane przez docker.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-122">Volumes live in the host system and are managed by Docker.</span></span>

<span data-ttu-id="2d4ac-123">**Tag:** Znacznik lub etykieta, które można zastosować do obrazów, tak aby można było zidentyfikować różne obrazy lub wersje tego samego obrazu (w zależności od numeru wersji lub środowiska docelowego).</span><span class="sxs-lookup"><span data-stu-id="2d4ac-123">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="2d4ac-124">**Kompilacja wieloetapowa:** Jest to funkcja, ponieważ docker 17.05 lub nowsze, który pomaga zmniejszyć rozmiar obrazów końcowych.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-124">**Multi-stage Build**: Is a feature, since Docker 17.05 or higher, that helps to reduce the size of the final images.</span></span> <span data-ttu-id="2d4ac-125">W kilku zdaniach, z kompilacji wieloetapowej można użyć, na przykład, duży obraz podstawowy, zawierający zestaw SDK, do kompilowania i publikowania aplikacji, a następnie za pomocą folderu publikowania z małym obrazem podstawowym tylko do wykonywania, do uzyskania znacznie mniejszy obraz końcowy.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-125">In a few sentences, with multi-stage build you can use, for example, a large base image, containing the SDK, for compiling and publishing the application and then using the publishing folder with a small runtime-only base image, to produce a much smaller final image.</span></span>

<span data-ttu-id="2d4ac-126">**Repozytorium (repozytorium)**: Zbiór powiązanych obrazów platformy Docker, oznaczony tagiem wskazującym wersję obrazu.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-126">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="2d4ac-127">Niektóre repozytoria zawierają wiele wariantów określonego obrazu, takich jak obraz zawierający zestaw SDK (cięższy), obraz zawierający tylko środowiska wykonawcze (lżejsze) itp. Te warianty mogą być oznaczone tagami.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-127">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="2d4ac-128">Pojedyncze repozytorium może zawierać warianty platformy, takie jak obraz systemu Linux i obraz systemu Windows.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-128">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="2d4ac-129">**Rejestr**: Usługa, która zapewnia dostęp do repozytoriów.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-129">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="2d4ac-130">Domyślnym rejestrem większości obrazów publicznych jest [Centrum platformy Docker](https://hub.docker.com/) (należące do platformy Docker jako organizacji).</span><span class="sxs-lookup"><span data-stu-id="2d4ac-130">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="2d4ac-131">Rejestr zwykle zawiera repozytoria z wielu zespołów.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-131">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="2d4ac-132">Firmy często mają prywatne rejestry do przechowywania i zarządzania obrazami, które stworzyli.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-132">Companies often have private registries to store and manage images they've created.</span></span> <span data-ttu-id="2d4ac-133">Innym przykładem jest usługa Azure Container Registry.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-133">Azure Container Registry is another example.</span></span>

<span data-ttu-id="2d4ac-134">**Obraz wielocherzekowy:** W przypadku wielu architektur jest to funkcja, która upraszcza wybór odpowiedniego obrazu, zgodnie z platformą, na której działa docker.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-134">**Multi-arch image**: For multi-architecture, it's a feature that simplifies the selection of the appropriate image, according to the platform where Docker is running.</span></span> <span data-ttu-id="2d4ac-135">Na przykład, gdy Dockerfile żąda obrazu podstawowego **z mcr.microsoft.com/dotnet/core/sdk:3.1** z rejestru, to rzeczywiście pobiera **3.1-sdk-nanoserver-1909**, **3.1-sdk-nanoserver-1809** lub **3.1-sdk-buster-slim**, w zależności od systemu operacyjnego i wersji, w której jest uruchomiony docker.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-135">For example, when a Dockerfile requests a base image **FROM mcr.microsoft.com/dotnet/core/sdk:3.1** from the registry, it actually gets **3.1-sdk-nanoserver-1909**, **3.1-sdk-nanoserver-1809** or **3.1-sdk-buster-slim**, depending on the operating system and version where Docker is running.</span></span>

<span data-ttu-id="2d4ac-136">**Docker Hub:** rejestr publiczny do przekazywania obrazów i pracy z nimi.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-136">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="2d4ac-137">Docker Hub zapewnia hosting obrazów platformy Docker, rejestry publiczne lub prywatne, wyzwalacze kompilacji i haki sieci web oraz integrację z GitHub i Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-137">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="2d4ac-138">**Usługa Azure Container Registry:** zasób publiczny do pracy z obrazami platformy Docker i jego składnikami na platformie Azure.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-138">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="2d4ac-139">Zapewnia to rejestr, który jest blisko wdrożeń na platformie Azure i który daje kontrolę nad dostępem, dzięki czemu można używać grup i uprawnień usługi Azure Active Directory.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-139">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="2d4ac-140">**Docker Trusted Registry (DTR)**: Usługa rejestru platformy Docker (z platformy Docker), która może być zainstalowana lokalnie, dzięki czemu działa w centrum danych i sieci organizacji.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-140">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization's datacenter and network.</span></span> <span data-ttu-id="2d4ac-141">Jest to wygodne dla obrazów prywatnych, które powinny być zarządzane w przedsiębiorstwie.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-141">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="2d4ac-142">Zaufany rejestr platformy Docker jest dołączony jako część produktu centrum danych platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-142">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="2d4ac-143">Aby uzyskać więcej informacji, zobacz [Zaufany rejestr platformy Docker (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="2d4ac-143">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="2d4ac-144">**Docker Community Edition (CE)**: Narzędzia programistyczne dla systemów Windows i macOS do tworzenia, uruchamiania i testowania kontenerów lokalnie.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-144">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="2d4ac-145">Docker CE dla systemu Windows udostępnia środowiska programistyczne dla kontenerów systemu Linux i Windows.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-145">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="2d4ac-146">Host platformy Docker systemu Linux w systemie Windows jest oparty na maszynie wirtualnej [funkcji Hyper-V.](https://www.microsoft.com/cloud-platform/server-virtualization)</span><span class="sxs-lookup"><span data-stu-id="2d4ac-146">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) virtual machine.</span></span> <span data-ttu-id="2d4ac-147">Host kontenerów systemu Windows jest oparty bezpośrednio na systemie Windows.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-147">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="2d4ac-148">Docker CE dla komputerów Mac opiera się na platformie Apple Hypervisor i [hypervisor xhyve](https://github.com/mist64/xhyve), który zapewnia linux docker hosta maszyny wirtualnej na Mac OS X. Docker CE dla Windows i dla Komputerów Mac zastępuje Docker Toolbox, który został oparty na Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-148">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="2d4ac-149">**Docker Enterprise Edition (EE)**: wersja docker w skali przedsiębiorstwa dla tworzenia systemów Linux i Windows.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-149">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="2d4ac-150">**Redagowanie:** Narzędzie wiersza polecenia i format pliku YAML z metadanymi do definiowania i uruchamiania aplikacji z wieloma kontenerami.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-150">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="2d4ac-151">Definiujesz pojedynczą aplikację na podstawie wielu obrazów z jednym lub kilkoma plikami .yml, które mogą zastępować wartości w zależności od środowiska.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-151">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="2d4ac-152">Po utworzeniu definicji można wdrożyć całą aplikację wielu kontenerów za pomocą jednego polecenia (docker-compose up), który tworzy kontener na obraz na hoście platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-152">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="2d4ac-153">**Klaster:** Kolekcja hostów platformy Docker jest widoczna tak, jakby była pojedynczym wirtualnym hostem platformy Docker, dzięki czemu aplikacja może skalować do wielu wystąpień usług rozłożonych na wiele hostów w klastrze.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-153">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="2d4ac-154">Klastry platformy Docker można tworzyć za pomocą platformy Kubernetes, usługi Azure Service Fabric, Docker Swarm i Mesosphere DC/OS.</span><span class="sxs-lookup"><span data-stu-id="2d4ac-154">Docker clusters can be created with Kubernetes, Azure Service Fabric, Docker Swarm and Mesosphere DC/OS.</span></span>
