# <a name="microservices-architecture"></a><span data-ttu-id="262e2-103">Architektura mikrousług</span><span class="sxs-lookup"><span data-stu-id="262e2-103">Microservices architecture</span></span>

<span data-ttu-id="262e2-104">Jak sama nazwa wskazuje architektura mikrousług jest podejście do tworzenia aplikacji serwera jako zestaw małych usług.</span><span class="sxs-lookup"><span data-stu-id="262e2-104">As the name implies, a microservices architecture is an approach to building a server application as a set of small services.</span></span> <span data-ttu-id="262e2-105">Oznacza to, że architektura mikrousług jest zorientowana głównie na zapleczu, mimo że podejście jest również używane dla frontonu.</span><span class="sxs-lookup"><span data-stu-id="262e2-105">That means a microservices architecture is mainly oriented to the back-end, although the approach is also being used for the front end.</span></span> <span data-ttu-id="262e2-106">Każda usługa jest uruchamiana we własnym procesie i komunikuje się z innymi procesami przy użyciu protokołów, takich jak HTTP/HTTPS, WebSockets lub [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span><span class="sxs-lookup"><span data-stu-id="262e2-106">Each service runs in its own process and communicates with other processes using protocols such as HTTP/HTTPS, WebSockets, or [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span></span> <span data-ttu-id="262e2-107">Każda mikrousługa implementuje określonej end-to-end domeny lub możliwości biznesowych w ramach określonej granicy kontekstu i każdy musi być opracowany autonomicznie i być wdrażalne niezależnie.</span><span class="sxs-lookup"><span data-stu-id="262e2-107">Each microservice implements a specific end-to-end domain or business capability within a certain context boundary, and each must be developed autonomously and be deployable independently.</span></span> <span data-ttu-id="262e2-108">Na koniec każda mikrousługa powinna być właścicielem powiązanego modelu danych domeny i logiki domeny (suwerenność i zdecentralizowane zarządzanie danymi) i może być oparta na różnych technologiach przechowywania danych (SQL, NoSQL) i różnych językach programowania.</span><span class="sxs-lookup"><span data-stu-id="262e2-108">Finally, each microservice should own its related domain data model and domain logic (sovereignty and decentralized data management) and could be based on different data storage technologies (SQL, NoSQL) and different programming languages.</span></span>

<span data-ttu-id="262e2-109">Jaki rozmiar powinien mieć mikrousługi?</span><span class="sxs-lookup"><span data-stu-id="262e2-109">What size should a microservice be?</span></span> <span data-ttu-id="262e2-110">Podczas opracowywania mikrousługi rozmiar nie powinien być ważnym punktem.</span><span class="sxs-lookup"><span data-stu-id="262e2-110">When developing a microservice, size shouldn't be the important point.</span></span> <span data-ttu-id="262e2-111">Zamiast tego ważnym punktem powinno być tworzenie luźno powiązanych usług, dzięki czemu masz autonomię rozwoju, wdrażania i skalowania dla każdej usługi.</span><span class="sxs-lookup"><span data-stu-id="262e2-111">Instead, the important point should be to create loosely coupled services so you have autonomy of development, deployment, and scale, for each service.</span></span> <span data-ttu-id="262e2-112">Oczywiście podczas identyfikowania i projektowania mikrousług, należy spróbować uczynić je tak małe, jak to możliwe, tak długo, jak nie masz zbyt wiele bezpośrednich zależności z innymi mikrousługami.</span><span class="sxs-lookup"><span data-stu-id="262e2-112">Of course, when identifying and designing microservices, you should try to make them as small as possible as long as you don't have too many direct dependencies with other microservices.</span></span> <span data-ttu-id="262e2-113">Ważniejsze niż rozmiar mikrousługi jest spójność wewnętrzna musi mieć i jego niezależności od innych usług.</span><span class="sxs-lookup"><span data-stu-id="262e2-113">More important than the size of the microservice is the internal cohesion it must have and its independence from other services.</span></span>

<span data-ttu-id="262e2-114">Dlaczego architektura mikrousług?</span><span class="sxs-lookup"><span data-stu-id="262e2-114">Why a microservices architecture?</span></span> <span data-ttu-id="262e2-115">Krótko mówiąc, zapewnia długoterminową zwinność.</span><span class="sxs-lookup"><span data-stu-id="262e2-115">In short, it provides long-term agility.</span></span> <span data-ttu-id="262e2-116">Mikrousługi umożliwiają lepszą łatwość konserwacji w złożonych, dużych i wysoce skalowalnych systemach, umożliwiając tworzenie aplikacji na podstawie wielu niezależnie wdrażanych usług, z których każdy ma szczegółowe i autonomiczne cykle życia.</span><span class="sxs-lookup"><span data-stu-id="262e2-116">Microservices enable better maintainability in complex, large, and highly-scalable systems by letting you create applications based on many independently deployable services that each have granular and autonomous lifecycles.</span></span>

<span data-ttu-id="262e2-117">Jako dodatkową korzyść mikrousług można skalować w sposób niezależny.</span><span class="sxs-lookup"><span data-stu-id="262e2-117">As an additional benefit, microservices can scale out independently.</span></span> <span data-ttu-id="262e2-118">Zamiast jednej aplikacji monolityczne, które należy skalować w sposób skalowany w sposób skalowany jako jednostka, zamiast tego można skalować w sposób określony mikrousług.</span><span class="sxs-lookup"><span data-stu-id="262e2-118">Instead of having a single monolithic application that you must scale out as a unit, you can instead scale out specific microservices.</span></span> <span data-ttu-id="262e2-119">W ten sposób można skalować tylko obszar funkcjonalny, który wymaga większej mocy obliczeniowej lub przepustowości sieci do obsługi popytu, zamiast skalować inne obszary aplikacji, które nie muszą być skalowane.</span><span class="sxs-lookup"><span data-stu-id="262e2-119">That way, you can scale just the functional area that needs more processing power or network bandwidth to support demand, rather than scaling out other areas of the application that don't need to be scaled.</span></span> <span data-ttu-id="262e2-120">Oznacza to oszczędność kosztów, ponieważ potrzebujesz mniej sprzętu.</span><span class="sxs-lookup"><span data-stu-id="262e2-120">That means cost savings because you need less hardware.</span></span>

<span data-ttu-id="262e2-124">Jak pokazuje rysunek 4-6, w tradycyjnym podejściu monolitycznym aplikacja skaluje się, klonując całą aplikację na kilku serwerach/maszynach wirtualnych.</span><span class="sxs-lookup"><span data-stu-id="262e2-124">As Figure 4-6 shows, in the traditional monolithic approach, the application scales by cloning the whole app in several servers/VM.</span></span> <span data-ttu-id="262e2-125">W podejściu mikrousług funkcjonalność jest segregowana w mniejszych usługach, dzięki czemu każda usługa może skalować niezależnie.</span><span class="sxs-lookup"><span data-stu-id="262e2-125">In the microservices approach, functionality is segregated in smaller services, so each service can scale independently.</span></span> <span data-ttu-id="262e2-126">Podejście mikrousług umożliwia elastyczne zmiany i szybką iterację każdej mikrousługi, ponieważ można zmienić konkretne, małe obszary złożonych, dużych i skalowalnych aplikacji.</span><span class="sxs-lookup"><span data-stu-id="262e2-126">The microservices approach allows agile changes and rapid iteration of each microservice, because you can change specific, small areas of complex, large, and scalable applications.</span></span>

<span data-ttu-id="262e2-127">Projektowanie precyzyjnych aplikacji opartych na mikrousługach umożliwia ciągłą integrację i ciągłe dostarczanie.</span><span class="sxs-lookup"><span data-stu-id="262e2-127">Architecting fine-grained microservices-based applications enables continuous integration and continuous delivery practices.</span></span> <span data-ttu-id="262e2-128">Przyspiesza również dostarczanie nowych funkcji do aplikacji.</span><span class="sxs-lookup"><span data-stu-id="262e2-128">It also accelerates delivery of new functions into the application.</span></span> <span data-ttu-id="262e2-129">Szczegółowy skład aplikacji umożliwia również uruchamianie i testowanie mikrousług w izolacji oraz rozwijanie ich autonomicznie przy zachowaniu wyraźnych kontraktów między nimi.</span><span class="sxs-lookup"><span data-stu-id="262e2-129">Fine-grained composition of applications also allows you to run and test microservices in isolation, and to evolve them autonomously while maintaining clear contracts between them.</span></span> <span data-ttu-id="262e2-130">Tak długo, jak nie zmienisz interfejsów lub kontraktów, można zmienić wewnętrzną implementację dowolnej mikrousługi lub dodać nowe funkcje bez przerywania innych mikrousług.</span><span class="sxs-lookup"><span data-stu-id="262e2-130">As long as you don't change the interfaces or contracts, you can change the internal implementation of any microservice or add new functionality without breaking other microservices.</span></span>

<span data-ttu-id="262e2-131">Oto ważne aspekty, aby umożliwić powodzenie w przechodzeniu do produkcji z systemem opartym na mikrousługach:</span><span class="sxs-lookup"><span data-stu-id="262e2-131">The following are important aspects to enable success in going into production with a microservices-based system:</span></span>

- <span data-ttu-id="262e2-132">Monitorowanie i kontrole stanu usług i infrastruktury.</span><span class="sxs-lookup"><span data-stu-id="262e2-132">Monitoring and health checks of the services and infrastructure.</span></span>

- <span data-ttu-id="262e2-133">Skalowalna infrastruktura dla usług (czyli chmury i koordynatorów).</span><span class="sxs-lookup"><span data-stu-id="262e2-133">Scalable infrastructure for the services (that is, cloud and orchestrators).</span></span>

- <span data-ttu-id="262e2-134">Projektowanie i wdrażanie zabezpieczeń na wielu poziomach: uwierzytelnianie, autoryzacja, zarządzanie wtajemnicami, bezpieczna komunikacja itp.</span><span class="sxs-lookup"><span data-stu-id="262e2-134">Security design and implementation at multiple levels: authentication, authorization, secrets management, secure communication, etc.</span></span>

- <span data-ttu-id="262e2-135">Szybkie dostarczanie aplikacji, zwykle z różnych zespołów koncentrujących się na różnych mikrousług.</span><span class="sxs-lookup"><span data-stu-id="262e2-135">Rapid application delivery, usually with different teams focusing on different microservices.</span></span>

- <span data-ttu-id="262e2-136">DevOps i CI/CD praktyk i infrastruktury.</span><span class="sxs-lookup"><span data-stu-id="262e2-136">DevOps and CI/CD practices and infrastructure.</span></span>
