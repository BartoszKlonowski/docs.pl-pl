# <a name="service-oriented-architecture"></a><span data-ttu-id="c8132-103">Architektura zorientowana na usługi</span><span class="sxs-lookup"><span data-stu-id="c8132-103">Service-oriented architecture</span></span>

<span data-ttu-id="c8132-104">Architektura zorientowana na usługi (SOA) była nadstosowana i ma różne znaczenie dla różnych osób.</span><span class="sxs-lookup"><span data-stu-id="c8132-104">Service-oriented architecture (SOA) was an overused term and has meant different things to different people.</span></span> <span data-ttu-id="c8132-105">Jednak jako typowy mianownik, SOA oznacza, że tworzysz strukturę aplikacji, tworząc ją w wielu usługach (najczęściej jako usługi HTTP), które mogą być klasyfikowane jako różne typy, takie jak podsystemy lub warstwy.</span><span class="sxs-lookup"><span data-stu-id="c8132-105">But as a common denominator, SOA means that you structure your application by decomposing it into multiple services (most commonly as HTTP services) that can be classified as different types like subsystems or tiers.</span></span>

<span data-ttu-id="c8132-106">Te usługi można teraz wdrożyć jako kontenery platformy Docker, co rozwiązuje problemy z wdrażaniem, ponieważ wszystkie zależności są zawarte w obrazie kontenera.</span><span class="sxs-lookup"><span data-stu-id="c8132-106">Those services can now be deployed as Docker containers, which solves deployment issues, because all the dependencies are included in the container image.</span></span> <span data-ttu-id="c8132-107">Jeśli jednak chcesz skalować w górę aplikacje SOA, możesz mieć problemy ze skalowalnością i dostępnością w przypadku wdrażania na podstawie jednego hosta platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="c8132-107">However, when you need to scale up SOA applications, you might have scalability and availability challenges if you're deploying based on single Docker hosts.</span></span> <span data-ttu-id="c8132-108">Jest to miejsce, w którym można znaleźć oprogramowanie do klastrowania platformy Docker lub programu Orchestrator, jak wyjaśniono w kolejnych sekcjach, w których opisano podejścia do wdrożenia dla mikrousług.</span><span class="sxs-lookup"><span data-stu-id="c8132-108">This is where Docker clustering software or an orchestrator can help you, as explained in later sections where deployment approaches for microservices are described.</span></span>

<span data-ttu-id="c8132-109">Kontenery platformy Docker są przydatne (ale nie są wymagane) dla tradycyjnych architektur zorientowanych na usługę i bardziej zaawansowanych architektur mikrousług.</span><span class="sxs-lookup"><span data-stu-id="c8132-109">Docker containers are useful (but not required) for both traditional service-oriented architectures and the more advanced microservices architectures.</span></span>

<span data-ttu-id="c8132-110">Mikrousługi pochodzą z SOA, ale SOA różni się od architektury mikrousług.</span><span class="sxs-lookup"><span data-stu-id="c8132-110">Microservices derive from SOA, but SOA is different from microservices architecture.</span></span> <span data-ttu-id="c8132-111">Funkcje takie jak duże centralne brokerzy, centralne Koordynatory na poziomie organizacji, a [Service Bus Enterprise (ESB)](https://en.wikipedia.org/wiki/Enterprise_service_bus) są typowe w SOA.</span><span class="sxs-lookup"><span data-stu-id="c8132-111">Features like large central brokers, central orchestrators at the organization level, and the [Enterprise Service Bus (ESB)](https://en.wikipedia.org/wiki/Enterprise_service_bus) are typical in SOA.</span></span> <span data-ttu-id="c8132-112">Jednak w większości przypadków są to antywzorce w społeczności mikrousług.</span><span class="sxs-lookup"><span data-stu-id="c8132-112">But in most cases, these are anti-patterns in the microservice community.</span></span> <span data-ttu-id="c8132-113">W rzeczywistości niektórzy użytkownicy twierdzili, że "architektura mikrousług jest w pełni gotowa."</span><span class="sxs-lookup"><span data-stu-id="c8132-113">In fact, some people argue that "The microservice architecture is SOA done right."</span></span>

<span data-ttu-id="c8132-114">Ten przewodnik koncentruje się na mikrousługach, ponieważ podejście SOA jest mniej opisowe niż wymagania i techniki używane w architekturze mikrousług.</span><span class="sxs-lookup"><span data-stu-id="c8132-114">This guide focuses on microservices, because a SOA approach is less prescriptive than the requirements and techniques used in a microservice architecture.</span></span> <span data-ttu-id="c8132-115">Jeśli wiesz, jak utworzyć aplikację opartą na mikrousługach, wiesz również, jak utworzyć prostsze aplikacje zorientowane na usługę.</span><span class="sxs-lookup"><span data-stu-id="c8132-115">If you know how to build a microservice-based application, you also know how to build a simpler service-oriented application.</span></span>
