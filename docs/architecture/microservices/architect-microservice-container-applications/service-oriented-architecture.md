# <a name="service-oriented-architecture"></a><span data-ttu-id="ca392-103">Architektura zorientowana na usługi</span><span class="sxs-lookup"><span data-stu-id="ca392-103">Service-oriented architecture</span></span>

<span data-ttu-id="ca392-104">Architektura zorientowana na usługi (SOA) była nadużywanym terminem i oznaczała różne rzeczy dla różnych ludzi.</span><span class="sxs-lookup"><span data-stu-id="ca392-104">Service-oriented architecture (SOA) was an overused term and has meant different things to different people.</span></span> <span data-ttu-id="ca392-105">Ale jako wspólny mianownik SOA oznacza, że struktura aplikacji przez rozkładanie go na wiele usług (najczęściej jako usługi HTTP), które mogą być klasyfikowane jako różne typy, takie jak podsystemy lub warstwy.</span><span class="sxs-lookup"><span data-stu-id="ca392-105">But as a common denominator, SOA means that you structure your application by decomposing it into multiple services (most commonly as HTTP services) that can be classified as different types like subsystems or tiers.</span></span>

<span data-ttu-id="ca392-106">Te usługi można teraz wdrożyć jako kontenery platformy Docker, co rozwiązuje problemy z wdrażaniem, ponieważ wszystkie zależności są uwzględnione w obrazie kontenera.</span><span class="sxs-lookup"><span data-stu-id="ca392-106">Those services can now be deployed as Docker containers, which solves deployment issues, because all the dependencies are included in the container image.</span></span> <span data-ttu-id="ca392-107">Jednak gdy trzeba skalować w górę aplikacji SOA, może być skalowalność i dostępność wyzwania, jeśli wdrażasz na podstawie pojedynczych hostów platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="ca392-107">However, when you need to scale up SOA applications, you might have scalability and availability challenges if you're deploying based on single Docker hosts.</span></span> <span data-ttu-id="ca392-108">W tym miejscu oprogramowanie do klastrowania platformy Docker lub koordynator może ci pomóc, jak wyjaśniono w kolejnych sekcjach, w których opisano metody wdrażania mikrousług.</span><span class="sxs-lookup"><span data-stu-id="ca392-108">This is where Docker clustering software or an orchestrator can help you, as explained in later sections where deployment approaches for microservices are described.</span></span>

<span data-ttu-id="ca392-109">Kontenery platformy Docker są przydatne (ale nie są wymagane) zarówno dla tradycyjnych architektur zorientowanych na usługi, jak i bardziej zaawansowanych architektur mikrousług.</span><span class="sxs-lookup"><span data-stu-id="ca392-109">Docker containers are useful (but not required) for both traditional service-oriented architectures and the more advanced microservices architectures.</span></span>

<span data-ttu-id="ca392-110">Mikrousługi pochodzą z SOA, ale SOA różni się od architektury mikrousług.</span><span class="sxs-lookup"><span data-stu-id="ca392-110">Microservices derive from SOA, but SOA is different from microservices architecture.</span></span> <span data-ttu-id="ca392-111">Funkcje takie jak duzi centralni brokerzy, centralni koordynatorzy na poziomie organizacji i [Enterprise Service Bus (ESB)](https://en.wikipedia.org/wiki/Enterprise_service_bus) są typowe w SOA.</span><span class="sxs-lookup"><span data-stu-id="ca392-111">Features like large central brokers, central orchestrators at the organization level, and the [Enterprise Service Bus (ESB)](https://en.wikipedia.org/wiki/Enterprise_service_bus) are typical in SOA.</span></span> <span data-ttu-id="ca392-112">Ale w większości przypadków są to anty-wzorce w społeczności mikrousług.</span><span class="sxs-lookup"><span data-stu-id="ca392-112">But in most cases, these are anti-patterns in the microservice community.</span></span> <span data-ttu-id="ca392-113">W rzeczywistości niektórzy ludzie twierdzą, że "architektura mikrousług jest SOA zrobić dobrze."</span><span class="sxs-lookup"><span data-stu-id="ca392-113">In fact, some people argue that "The microservice architecture is SOA done right."</span></span>

<span data-ttu-id="ca392-114">Ten przewodnik koncentruje się na mikrousług, ponieważ podejście SOA jest mniej nakazowe niż wymagania i techniki używane w architekturze mikrousług.</span><span class="sxs-lookup"><span data-stu-id="ca392-114">This guide focuses on microservices, because a SOA approach is less prescriptive than the requirements and techniques used in a microservice architecture.</span></span> <span data-ttu-id="ca392-115">Jeśli wiesz, jak utworzyć aplikację opartą na mikrousługach, wiesz również, jak utworzyć prostszą aplikację zorientowaną na usługi.</span><span class="sxs-lookup"><span data-stu-id="ca392-115">If you know how to build a microservice-based application, you also know how to build a simpler service-oriented application.</span></span>
