<span data-ttu-id="0620c-105">Architektura mikrousług często zaczyna się od danych i logiki obsługi po stronie serwera, ale w wielu przypadkach interfejsu jest nadal obsługiwane jako monolit.</span><span class="sxs-lookup"><span data-stu-id="0620c-105">Microservices architecture often starts with the server-side handling data and logic, but, in many cases, the UI is still handled as a monolith.</span></span> <span data-ttu-id="0620c-106">Jednak bardziej zaawansowane podejście, o nazwie [mikro frontonów](https://martinfowler.com/articles/micro-frontends.html), jest zaprojektowanie interfejsu użytkownika aplikacji na podstawie mikrousług, jak również.</span><span class="sxs-lookup"><span data-stu-id="0620c-106">However, a more advanced approach, called [micro frontends](https://martinfowler.com/articles/micro-frontends.html), is to design your application UI based on microservices as well.</span></span> <span data-ttu-id="0620c-107">Oznacza to, że złożony interfejsu opracowanego przez mikrousług, zamiast mikrousług na serwerze i tylko monolityczne aplikacji klienckiej korzystających z mikrousług.</span><span class="sxs-lookup"><span data-stu-id="0620c-107">That means having a composite UI produced by the microservices, instead of having microservices on the server and just a monolithic client app consuming the microservices.</span></span> <span data-ttu-id="0620c-108">Dzięki takiemu podejściu mikrousług, które można utworzyć można uzupełnić zarówno logiki i reprezentacji wizualnej.</span><span class="sxs-lookup"><span data-stu-id="0620c-108">With this approach, the microservices you build can be complete with both logic and visual representation.</span></span>

<span data-ttu-id="0620c-118">W czasie uruchamiania aplikacji klienckiej każdy ze składników interfejsu interfejsu klienta (na przykład klasy TypeScript) rejestruje się za pomocą mikrousługi infrastruktury zdolnej do dostarczania ViewModels dla danego scenariusza.</span><span class="sxs-lookup"><span data-stu-id="0620c-118">At client application start-up time, each of the client UI components (TypeScript classes, for example) registers itself with an infrastructure microservice capable of providing ViewModels for a given scenario.</span></span> <span data-ttu-id="0620c-119">Jeśli mikrousługi zmienia kształt, interfejsu użytkownika zmienia się również.</span><span class="sxs-lookup"><span data-stu-id="0620c-119">If the microservice changes the shape, the UI changes also.</span></span>
