<span data-ttu-id="3e07c-109">Jednostki reprezentują obiekty domeny i są definiowane głównie przez ich tożsamość, ciągłość i trwałość w czasie, a nie tylko przez atrybuty, które je tworzą.</span><span class="sxs-lookup"><span data-stu-id="3e07c-109">Entities represent domain objects and are primarily defined by their identity, continuity, and persistence over time, and not only by the attributes that comprise them.</span></span> <span data-ttu-id="3e07c-110">Jak mówi Eric Evans, "obiekt definiowany głównie przez jego tożsamość jest nazywany jednostką".</span><span class="sxs-lookup"><span data-stu-id="3e07c-110">As Eric Evans says, “an object primarily defined by its identity is called an Entity.”</span></span> <span data-ttu-id="3e07c-111">Jednostki są bardzo ważne w modelu domeny, ponieważ są one podstawą dla modelu.</span><span class="sxs-lookup"><span data-stu-id="3e07c-111">Entities are very important in the domain model, since they are the base for a model.</span></span> <span data-ttu-id="3e07c-112">Dlatego należy je dokładnie zidentyfikować i zaprojektować.</span><span class="sxs-lookup"><span data-stu-id="3e07c-112">Therefore, you should identify and design them carefully.</span></span>

<span data-ttu-id="3e07c-113">*Tożsamości jednostki można przekreślić wiele mikrousług lub ograniczone konteksty.*</span><span class="sxs-lookup"><span data-stu-id="3e07c-113">*An entity’s identity can cross multiple microservices or Bounded Contexts.*</span></span>

### <a name="the-aggregate-pattern"></a><span data-ttu-id="3e07c-179">Wzorzec agregacji</span><span class="sxs-lookup"><span data-stu-id="3e07c-179">The Aggregate pattern</span></span>

<span data-ttu-id="3e07c-180">Model domeny zawiera klastry różnych jednostek danych i procesów, które mogą kontrolować znaczący obszar funkcji, takich jak realizacja zamówień lub spis.</span><span class="sxs-lookup"><span data-stu-id="3e07c-180">A domain model contains clusters of different data entities and processes that can control a significant area of functionality, such as order fulfillment or inventory.</span></span> <span data-ttu-id="3e07c-181">Bardziej drobnoziarnistą jednostką DDD jest agregacja, która opisuje klaster lub grupę jednostek i zachowań, które mogą być traktowane jako jednostka spoista.</span><span class="sxs-lookup"><span data-stu-id="3e07c-181">A more fine-grained DDD unit is the aggregate, which describes a cluster or group of entities and behaviors that can be treated as a cohesive unit.</span></span>

<span data-ttu-id="3e07c-182">Zazwyczaj definiuje się agregację na podstawie potrzebnych transakcji.</span><span class="sxs-lookup"><span data-stu-id="3e07c-182">You usually define an aggregate based on the transactions that you need.</span></span> <span data-ttu-id="3e07c-183">Klasycznym przykładem jest kolejność, która zawiera również listę elementów zamówienia.</span><span class="sxs-lookup"><span data-stu-id="3e07c-183">A classic example is an order that also contains a list of order items.</span></span> <span data-ttu-id="3e07c-184">Element zamówienia będzie zwykle jednostką.</span><span class="sxs-lookup"><span data-stu-id="3e07c-184">An order item will usually be an entity.</span></span> <span data-ttu-id="3e07c-185">Ale będzie to jednostka podrzędna w agregacji zamówienia, która będzie również zawierać jednostkę zamówienia jako jednostkę główną, zwykle nazywaną agregowanym katalogiem głównym.</span><span class="sxs-lookup"><span data-stu-id="3e07c-185">But it will be a child entity within the order aggregate, which will also contain the order entity as its root entity, typically called an aggregate root.</span></span>

<span data-ttu-id="3e07c-186">Identyfikacja agregatów może być trudna.</span><span class="sxs-lookup"><span data-stu-id="3e07c-186">Identifying aggregates can be hard.</span></span> <span data-ttu-id="3e07c-187">Agregacja jest grupą obiektów, które muszą być spójne ze sobą, ale nie można po prostu wybrać grupę obiektów i oznaczyć je agregacji.</span><span class="sxs-lookup"><span data-stu-id="3e07c-187">An aggregate is a group of objects that must be consistent together, but you cannot just pick a group of objects and label them an aggregate.</span></span> <span data-ttu-id="3e07c-188">Należy rozpocząć od koncepcji domeny i myśleć o jednostkach, które są używane w najbardziej typowych transakcji związanych z tą koncepcją.</span><span class="sxs-lookup"><span data-stu-id="3e07c-188">You must start with a domain concept and think about the entities that are used in the most common transactions related to that concept.</span></span> <span data-ttu-id="3e07c-189">Te jednostki, które muszą być transakcyjnie spójne są tym, co stanowi agregat.</span><span class="sxs-lookup"><span data-stu-id="3e07c-189">Those entities that need to be transactionally consistent are what forms an aggregate.</span></span> <span data-ttu-id="3e07c-190">Myślenie o operacjach transakcji jest prawdopodobnie najlepszym sposobem identyfikowania agregatów.</span><span class="sxs-lookup"><span data-stu-id="3e07c-190">Thinking about transaction operations is probably the best way to identify aggregates.</span></span>

<span data-ttu-id="3e07c-192">Agregat składa się z co najmniej jednej jednostki: agregacji głównej, nazywanej również jednostką główną lub jednostką podstawową.</span><span class="sxs-lookup"><span data-stu-id="3e07c-192">An aggregate is composed of at least one entity: the aggregate root, also called root entity or primary entity.</span></span> <span data-ttu-id="3e07c-193">Ponadto może mieć wiele jednostek podrzędnych i obiektów wartości, ze wszystkimi jednostkami i obiektami współpracującymi ze sobą w celu zaimplementowania wymaganego zachowania i transakcji.</span><span class="sxs-lookup"><span data-stu-id="3e07c-193">Additionally, it can have multiple child entities and value objects, with all entities and objects working together to implement required behavior and transactions.</span></span>

<span data-ttu-id="3e07c-194">Celem zagregowanego katalogu głównego jest zapewnienie spójności agregatu; powinien być jedynym punktem wejścia dla aktualizacji agregacji za pomocą metod lub operacji w klasie głównej agregacji.</span><span class="sxs-lookup"><span data-stu-id="3e07c-194">The purpose of an aggregate root is to ensure the consistency of the aggregate;
