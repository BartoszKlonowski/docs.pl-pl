# <a name="introduction-to-containers-and-docker"></a><span data-ttu-id="55c4d-103">Wprowadzenie do kontenerów i platformy Docker</span><span class="sxs-lookup"><span data-stu-id="55c4d-103">Introduction to containers and Docker</span></span>

<span data-ttu-id="55c4d-104">*Konteneryzacja jest podejście do tworzenia oprogramowania, w którym aplikacji lub usługi, jego zależności i jego konfiguracji (abstrakcjonowane jako pliki manifestu wdrożenia) są pakowane razem jako obraz kontenera. Następnie można przetestować konteneryzowaną aplikację jako jednostkę i wdrożyć ją jako wystąpienie obrazu kontenera w systemie operacyjnym hosta (OS).*</span><span class="sxs-lookup"><span data-stu-id="55c4d-104">*Containerization is an approach to software development in which an application or service, its dependencies, and its configuration (abstracted as deployment manifest files) are packaged together as a container image. You then can test the containerized application as a unit and deploy it as a container image instance to the host operating system (OS).*</span></span>

<span data-ttu-id="55c4d-105">Podobnie jak kontenery wysyłkowe umożliwiają transport towarów statkiem, pociągiem lub ciężarówką niezależnie od ładunku w środku, kontenery oprogramowania działają jako standardowa jednostka wdrażania oprogramowania, która może zawierać inny kod i zależności.</span><span class="sxs-lookup"><span data-stu-id="55c4d-105">Just as shipping containers allow goods to be transported by ship, train, or truck regardless of the cargo inside, software containers act as a standard unit of software deployment that can contain different code and dependencies.</span></span> <span data-ttu-id="55c4d-106">Konteneryzowanie oprogramowania w ten sposób umożliwia programistom i specjalistom IT wdrażanie ich w różnych środowiskach z niewielkimi lub żadnymi modyfikacjami.</span><span class="sxs-lookup"><span data-stu-id="55c4d-106">Containerizing software this way enables developers and IT professionals to deploy them across environments with little or no modification.</span></span>

<span data-ttu-id="55c4d-107">Kontenery również izolować aplikacje od siebie na wspólnym os.</span><span class="sxs-lookup"><span data-stu-id="55c4d-107">Containers also isolate applications from each other on a shared OS.</span></span> <span data-ttu-id="55c4d-108">Konteneryzowane aplikacje działają na serwerze kontenera, który z kolei działa w systemie operacyjnym (Linux lub Windows).</span><span class="sxs-lookup"><span data-stu-id="55c4d-108">Containerized applications run on top of a container host that in turn runs on the OS (Linux or Windows).</span></span> <span data-ttu-id="55c4d-109">Kontenery mają zatem znacznie mniejszy rozmiar niż obrazy maszyny wirtualnej (VM).</span><span class="sxs-lookup"><span data-stu-id="55c4d-109">Containers therefore have a much smaller footprint than virtual machine (VM) images.</span></span>

<span data-ttu-id="55c4d-110">Każdy kontener można uruchomić całą aplikację sieci web lub usługi, jak pokazano na rysunku 1-1.</span><span class="sxs-lookup"><span data-stu-id="55c4d-110">Each container can run a whole web application or a service, as shown in Figure 1-1.</span></span> <span data-ttu-id="55c4d-111">W tym przykładzie host platformy Docker jest hostem kontenerów, a App1, App2, Svc1 i Svc2 są konteneryzowanymi aplikacjami lub usługami.</span><span class="sxs-lookup"><span data-stu-id="55c4d-111">In this example, Docker host is a container host, and App1, App2, Svc1, and Svc2 are containerized applications or services.</span></span>
