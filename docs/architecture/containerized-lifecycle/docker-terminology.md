<span data-ttu-id="dee8e-106">**Obraz kontenera**: Pakiet zawierający wszystkie zależności i informacje, które są konieczne do utworzenia kontenera.</span><span class="sxs-lookup"><span data-stu-id="dee8e-106">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="dee8e-107">Obraz zawiera wszystkie zależności (takie jak struktury) oraz konfigurację wdrażania i wykonywania, które mają być używane przez środowisko uruchomieniowe kontenera.</span><span class="sxs-lookup"><span data-stu-id="dee8e-107">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="dee8e-108">Zazwyczaj obraz pochodzi z wielu obrazów podstawowych, które są warstwowo ułożone na siebie, aby utworzyć system plików kontenera.</span><span class="sxs-lookup"><span data-stu-id="dee8e-108">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container's filesystem.</span></span> <span data-ttu-id="dee8e-109">Obraz jest niezmienny, gdy zostanie utworzony.</span><span class="sxs-lookup"><span data-stu-id="dee8e-109">An image is immutable once it has been created.</span></span>

<span data-ttu-id="dee8e-110">**Pliku dockerfile**: Plik tekstowy, który zawiera instrukcje dotyczące sposobu tworzenia obrazu platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="dee8e-110">**Dockerfile**: A text file that contains instructions on how to build a Docker image.</span></span> <span data-ttu-id="dee8e-111">Podobnie jak skrypt wsadowy, pierwszy wiersz określa podstawowy obraz, a następnie postępuje zgodnie z instrukcjami, aby zainstalować wymagane programy, skopiować pliki i tak dalej, aż do momentu uzyskania potrzebnego środowiska roboczego.</span><span class="sxs-lookup"><span data-stu-id="dee8e-111">It's like a batch script, the first line states the base image to begin with and then follow the instructions to install required programs, copy files, and so on, until you get the working environment you need.</span></span>

<span data-ttu-id="dee8e-112">**Kompilacja**: Akcja tworzenia obrazu kontenera na podstawie informacji i kontekstu dostarczonych przez jego pliku dockerfile oraz dodatkowych plików w folderze, w którym utworzono obraz.</span><span class="sxs-lookup"><span data-stu-id="dee8e-112">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="dee8e-113">Możesz tworzyć obrazy za pomocą **`docker build`** polecenia Docker.</span><span class="sxs-lookup"><span data-stu-id="dee8e-113">You can build images with the Docker **`docker build`** command.</span></span>

<span data-ttu-id="dee8e-114">**Kontener**: Wystąpienie obrazu platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="dee8e-114">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="dee8e-115">Kontener reprezentuje wykonywanie pojedynczej aplikacji, procesu lub usługi.</span><span class="sxs-lookup"><span data-stu-id="dee8e-115">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="dee8e-116">Składa się z zawartości obrazu platformy Docker, środowiska wykonawczego i standardowego zestawu instrukcji.</span><span class="sxs-lookup"><span data-stu-id="dee8e-116">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="dee8e-117">W przypadku skalowania usługi należy utworzyć wiele wystąpień kontenera z tego samego obrazu.</span><span class="sxs-lookup"><span data-stu-id="dee8e-117">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="dee8e-118">Lub zadanie usługi Batch może utworzyć wiele kontenerów z tego samego obrazu, przekazując różne parametry do każdego wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="dee8e-118">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="dee8e-119">**Woluminy**: Oferuje zapisywalny system plików, który może być używany przez kontener.</span><span class="sxs-lookup"><span data-stu-id="dee8e-119">**Volumes**: Offer a writable filesystem that the container can use.</span></span> <span data-ttu-id="dee8e-120">Ponieważ obrazy są dostępne tylko do odczytu, ale większość programów musi zapisywać w systemie plików, woluminy dodają warstwę zapisywalną na podstawie obrazu kontenera, dzięki czemu programy mają dostęp do systemu plików zapisywalnych.</span><span class="sxs-lookup"><span data-stu-id="dee8e-120">Since images are read-only but most programs need to write to the filesystem, volumes add a writable layer, on top of the container image, so the programs have access to a writable filesystem.</span></span> <span data-ttu-id="dee8e-121">Program nie wie, że uzyskuje dostęp do warstwowego systemu plików, jest to tylko system plików w zwykły sposób.</span><span class="sxs-lookup"><span data-stu-id="dee8e-121">The program doesn't know it's accessing a layered filesystem, it's just the filesystem as usual.</span></span> <span data-ttu-id="dee8e-122">Woluminy na żywo w systemie hosta i zarządzane przez platformę Docker.</span><span class="sxs-lookup"><span data-stu-id="dee8e-122">Volumes live in the host system and are managed by Docker.</span></span>

<span data-ttu-id="dee8e-123">**Tag**: Znacznik lub etykieta, które można zastosować do obrazów w taki sposób, aby można było zidentyfikować różne obrazy lub wersje tego samego obrazu (w zależności od numeru wersji lub środowiska docelowego).</span><span class="sxs-lookup"><span data-stu-id="dee8e-123">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="dee8e-124">**Kompilacja wieloetapowa**: Jest funkcją, ponieważ platforma Docker 17,05 lub nowsza, która pomaga zmniejszyć rozmiar obrazów końcowych.</span><span class="sxs-lookup"><span data-stu-id="dee8e-124">**Multi-stage Build**: Is a feature, since Docker 17.05 or higher, that helps to reduce the size of the final images.</span></span> <span data-ttu-id="dee8e-125">W kilku zdaniach, z Wieloetapową kompilacją można użyć, na przykład, dużego obrazu podstawowego, zawierającego zestaw SDK, do kompilowania i publikowania aplikacji, a następnie przy użyciu folderu publikacji z małym obrazem podstawowym tylko środowiska uruchomieniowego, w celu utworzenia znacznie mniejszego obrazu końcowego</span><span class="sxs-lookup"><span data-stu-id="dee8e-125">In a few sentences, with multi-stage build you can use, for example, a large base image, containing the SDK, for compiling and publishing the application and then using the publishing folder with a small runtime-only base image, to produce a much smaller final image</span></span>

<span data-ttu-id="dee8e-126">**Repozytorium (repozytorium)** : Kolekcja powiązanych obrazów platformy Docker oznaczona tagami z tagiem wskazującym wersję obrazu.</span><span class="sxs-lookup"><span data-stu-id="dee8e-126">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="dee8e-127">Niektóre repozytoria zawierają wiele wariantów określonego obrazu, takich jak obraz zawierający zestawy SDK (grubszy), obraz zawierający tylko środowiska uruchomieniowe (jaśniejszy) itp. Te warianty mogą być oznaczone tagami.</span><span class="sxs-lookup"><span data-stu-id="dee8e-127">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="dee8e-128">Pojedyncze repozytorium może zawierać warianty platformy, takie jak obraz systemu Linux i obraz Windows.</span><span class="sxs-lookup"><span data-stu-id="dee8e-128">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="dee8e-129">**Rejestr**: Usługa, która zapewnia dostęp do repozytoriów.</span><span class="sxs-lookup"><span data-stu-id="dee8e-129">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="dee8e-130">Domyślnym rejestrem większości obrazów publicznych jest [centrum platformy Docker](https://hub.docker.com/) (należące do platformy Docker jako organizacja).</span><span class="sxs-lookup"><span data-stu-id="dee8e-130">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="dee8e-131">Rejestr zawiera zwykle repozytoria z wielu zespołów.</span><span class="sxs-lookup"><span data-stu-id="dee8e-131">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="dee8e-132">Firmy często mają prywatne rejestry do przechowywania utworzonych przez siebie obrazów i zarządzania nimi.</span><span class="sxs-lookup"><span data-stu-id="dee8e-132">Companies often have private registries to store and manage images they've created.</span></span> <span data-ttu-id="dee8e-133">Azure Container Registry jest kolejnym przykładem.</span><span class="sxs-lookup"><span data-stu-id="dee8e-133">Azure Container Registry is another example.</span></span>

<span data-ttu-id="dee8e-134">**Obraz z obsługą wielodostępności**: W przypadku wielu architektur program to funkcja, która upraszcza wybór odpowiedniego obrazu, zgodnie z platformą, w której działa system Docker, na przykład gdy pliku dockerfile żąda obrazu **`FROM mcr.microsoft.com/dotnet/core/sdk:2.2`** podstawowego z rejestru, który faktycznie otrzymuje, **`2.2-nanoserver-1709`** **`2.2-nanoserver-1803`** , **lub,`2.2-stretch`** w zależności od systemu operacyjnego i wersji, w której uruchomiono platformę Docker. **`2.2-nanoserver-1809`**</span><span class="sxs-lookup"><span data-stu-id="dee8e-134">**Multi-arch image**: For multi-architecture, is a feature that simplifies the selection of the appropriate image, according to the platform where Docker is running, for example, when a Dockerfile requests a base image **`FROM mcr.microsoft.com/dotnet/core/sdk:2.2`** from the registry it actually gets **`2.2-nanoserver-1709`**, **`2.2-nanoserver-1803`**, **`2.2-nanoserver-1809`** or **`2.2-stretch`**, depending on the operating system and version where Docker is running.</span></span>

<span data-ttu-id="dee8e-135">**Centrum platformy Docker**: Rejestr publiczny służący do przekazywania obrazów i pracy z nimi.</span><span class="sxs-lookup"><span data-stu-id="dee8e-135">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="dee8e-136">Centrum platformy Docker zapewnia hosting obrazów platformy Docker, rejestry publiczne lub prywatne, wyzwalacze kompilacji i elementy webhook oraz integrację z usługą GitHub i Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="dee8e-136">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="dee8e-137">**Azure Container Registry**: Zasób publiczny do pracy z obrazami platformy Docker i jej składnikami na platformie Azure.</span><span class="sxs-lookup"><span data-stu-id="dee8e-137">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="dee8e-138">Dzięki temu rejestr znajduje się blisko wdrożeń na platformie Azure i zapewnia kontrolę nad dostępem, dzięki czemu można korzystać z Azure Active Directory grup i uprawnień.</span><span class="sxs-lookup"><span data-stu-id="dee8e-138">This provides a registry that's close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="dee8e-139">**Zaufany rejestr platformy Docker (DTR)** : Usługa rejestru platformy Docker (z platformy Docker), którą można zainstalować lokalnie, tak aby była w centrum danych i sieci organizacji.</span><span class="sxs-lookup"><span data-stu-id="dee8e-139">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization's datacenter and network.</span></span> <span data-ttu-id="dee8e-140">Jest to wygodne w przypadku obrazów prywatnych, które powinny być zarządzane w przedsiębiorstwie.</span><span class="sxs-lookup"><span data-stu-id="dee8e-140">It's convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="dee8e-141">Zaufany rejestr platformy Docker jest dołączony jako część produktu centrum danych platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="dee8e-141">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="dee8e-142">Aby uzyskać więcej informacji, zobacz artykuł [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="dee8e-142">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="dee8e-143">**Docker Community Edition (CE)** : Narzędzia programistyczne dla systemu Windows i macOS do tworzenia, uruchamiania i testowania kontenerów lokalnie.</span><span class="sxs-lookup"><span data-stu-id="dee8e-143">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="dee8e-144">Docker CE for Windows zapewnia środowiska deweloperskie dla kontenerów systemu Linux i Windows.</span><span class="sxs-lookup"><span data-stu-id="dee8e-144">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="dee8e-145">Host platformy Docker systemu Linux w systemie Windows jest oparty na maszynie wirtualnej [funkcji Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) .</span><span class="sxs-lookup"><span data-stu-id="dee8e-145">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) virtual machine.</span></span> <span data-ttu-id="dee8e-146">Host dla kontenerów systemu Windows jest bezpośrednio oparty na systemie Windows.</span><span class="sxs-lookup"><span data-stu-id="dee8e-146">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="dee8e-147">Program Docker CE dla komputerów Mac jest oparty na strukturze funkcji hypervisor firmy Apple i [funkcji hypervisor xhyve](https://github.com/mist64/xhyve), która udostępnia maszynę wirtualną hosta platformy Docker systemu Linux na Mac OS X. Docker CE for Windows i dla komputerów Mac zastępuje Przybornik platformy Docker, który został oparty na bazie danych Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="dee8e-147">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. <span data-ttu-id="dee8e-148">**Docker Enterprise Edition (EE)** : Skalowalna w poziomie korporacyjnym narzędzia platformy Docker dla systemów Linux i Windows Development.</span><span class="sxs-lookup"><span data-stu-id="dee8e-148">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="dee8e-149">**Redaguj**: Narzędzie wiersza polecenia i format pliku YAML z metadanymi do definiowania i uruchamiania aplikacji wielokontenerowych.</span><span class="sxs-lookup"><span data-stu-id="dee8e-149">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="dee8e-150">Należy zdefiniować pojedynczą aplikację na podstawie wielu obrazów z co najmniej jednym plikami. yml, która może zastąpić wartości w zależności od środowiska.</span><span class="sxs-lookup"><span data-stu-id="dee8e-150">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="dee8e-151">Po utworzeniu definicji można wdrożyć całą aplikację obejmującą wiele kontenerów za pomocą jednego polecenia (Docker-Zredaguj), które tworzy kontener na obraz na hoście platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="dee8e-151">After you've created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>
