<span data-ttu-id="b5ece-119">Po utworzeniu projektu będzie on zawierał pliku dockerfile i środowisko uruchomieniowe procesu roboczego skonfigurowane do `dotnet`.</span><span class="sxs-lookup"><span data-stu-id="b5ece-119">When the project is created, it will include a Dockerfile and the worker runtime configured to `dotnet`.</span></span> <span data-ttu-id="b5ece-120">Teraz można tworzyć i testować funkcję lokalnie.</span><span class="sxs-lookup"><span data-stu-id="b5ece-120">Now, you can create and test your function locally.</span></span> <span data-ttu-id="b5ece-121">Kompiluj i uruchamiaj przy użyciu `docker build` poleceń `docker run` i.</span><span class="sxs-lookup"><span data-stu-id="b5ece-121">Build and run it using the  `docker build` and `docker run` commands.</span></span> <span data-ttu-id="b5ece-122">Aby uzyskać szczegółowe instrukcje dotyczące rozpoczynania tworzenia Azure Functions z obsługą platformy Docker, zobacz samouczek [Tworzenie funkcji w systemie Linux przy użyciu niestandardowego obrazu](https://docs.microsoft.com/azure/azure-functions/functions-create-function-linux-custom-image) .</span><span class="sxs-lookup"><span data-stu-id="b5ece-122">For detailed steps to get started building Azure Functions with Docker support, see the [Create a function on Linux using a custom image](https://docs.microsoft.com/azure/azure-functions/functions-create-function-linux-custom-image) tutorial.</span></span>
