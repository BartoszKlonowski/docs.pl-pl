<span data-ttu-id="a1202-109">W aplikacji monolitycznej komunikacja jest prosta.</span><span class="sxs-lookup"><span data-stu-id="a1202-109">In a monolithic application, communication is straightforward.</span></span> <span data-ttu-id="a1202-110">Moduły kodu są wykonywane razem w tym samym obszarze plików wykonywalnych (proces) na serwerze.</span><span class="sxs-lookup"><span data-stu-id="a1202-110">The code modules execute together in the same executable space (process) on a server.</span></span> <span data-ttu-id="a1202-111">Takie podejście może mieć zalety wydajności, ponieważ wszystkie elementy działają razem w pamięci współdzielonej, ale w przypadku ściśle sprzężonego kodu trudno zachować, rozwijać i skalować.</span><span class="sxs-lookup"><span data-stu-id="a1202-111">This approach can have performance advantages as everything runs together in shared memory, but results in tightly coupled code that becomes difficult to maintain, evolve, and scale.</span></span>

<span data-ttu-id="a1202-112">Systemy natywne w chmurze implementują architekturę opartą na mikrousługach z wieloma małymi, niezależnymi mikrousługami.</span><span class="sxs-lookup"><span data-stu-id="a1202-112">Cloud-native systems implement a microservice-based architecture with many small, independent microservices.</span></span> <span data-ttu-id="a1202-113">Każda mikrousługa jest wykonywana w osobnym procesie i zazwyczaj jest uruchamiana wewnątrz kontenera wdrożonego w *klastrze*.</span><span class="sxs-lookup"><span data-stu-id="a1202-113">Each microservice executes in a separate process and typically runs inside a container that is deployed to a *cluster*.</span></span>
