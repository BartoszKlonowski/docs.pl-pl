# <a name="lift-and-shift-existing-apps-devops"></a>Podnieś i przesunięcie istniejące aplikacje opracowywania oprogramowania
> Podnieś wizji: I przesunięcia istniejących aplikacji .NET Framework do aplikacji chmury DevOps gotowe do znacząco zwiększyć aktywności wdrożenia, aby można było wysłać szybciej i obniżyć koszty dostarczania aplikacji.

Aby skorzystać z zalet chmury i nowych technologii, takich jak kontenery, należy co najmniej częściowo modernizacji istniejących aplikacji .NET. Ostatecznie modernizacji aplikacje przedsiębiorstwa będzie obniżyć całkowity koszt posiadania.

Częściowo modernizacji aplikacji nie musi oznaczać pełnej migracji i architektura ponownie. Można początkowo modernizacji istniejących aplikacji przy użyciu procesu przyrostu i shift, który jest łatwo i szybko. Możesz zachować bieżącego kodu podstawowej, napisana istniejące wersje .NET Framework, wszelkie zależności systemu Windows i usług IIS. Rysunek 4-1 prezentuje położenie aplikacje gotowe DevOps chmury w modelach dojrzałości modernizacji aplikacji Azure.
