# <a name="weak-references"></a><span data-ttu-id="461ef-102">Słabe odwołania</span><span class="sxs-lookup"><span data-stu-id="461ef-102">Weak References</span></span>

 <span data-ttu-id="461ef-109">Słabe odwołania są przydatne dla obiektów, które używa dużej ilości pamięci, ale może być odtworzona łatwo, jeśli są odzyskiwane przez wyrzucanie elementów bezużytecznych.</span><span class="sxs-lookup"><span data-stu-id="461ef-109">Weak references are useful for objects that use a lot of memory, but can be recreated easily if they are reclaimed by garbage collection.</span></span>  
  
 <span data-ttu-id="461ef-110">Załóżmy, że widok drzewa w aplikacji Windows Forms Wyświetla złożonych hierarchiczne opcje do wyboru dla użytkownika.</span><span class="sxs-lookup"><span data-stu-id="461ef-110">Suppose a tree view in a Windows Forms application displays a complex hierarchical choice of options to the user.</span></span> <span data-ttu-id="461ef-111">W przypadku dużych danych bazowych, utrzymywanie drzewa w pamięci jest nieefektywne po użytkownik jest związany z innych obiektów w aplikacji.</span><span class="sxs-lookup"><span data-stu-id="461ef-111">If the underlying data is large, keeping the tree in memory is inefficient when the user is involved with something else in the application.</span></span>

 <span data-ttu-id="461ef-114">Aby ustanowić słabe odwołanie z obiektu, należy utworzyć <xref:System.WeakReference> mają być śledzone za pomocą wystąpienia obiektu.</span><span class="sxs-lookup"><span data-stu-id="461ef-114">To establish a weak reference with an object, you create a <xref:System.WeakReference> using the instance of the object to be tracked.</span></span> <span data-ttu-id="461ef-115">Następnie ustawiamy <xref:System.WeakReference.Target%2A> właściwości tego obiektu i zestawu, oryginalnym odwoływać się do obiektu, aby `null`.</span><span class="sxs-lookup"><span data-stu-id="461ef-115">You then set the <xref:System.WeakReference.Target%2A> property to that object and set the original reference to the object to `null`.</span></span> <span data-ttu-id="461ef-116">Dla przykładu kodu zobacz <xref:System.WeakReference> w bibliotece klas.</span><span class="sxs-lookup"><span data-stu-id="461ef-116">For a code example, see <xref:System.WeakReference> in the class library.</span></span>  
  
## <a name="short-and-long-weak-references"></a><span data-ttu-id="461ef-117">Krótko- i długo słabe odwołania</span><span class="sxs-lookup"><span data-stu-id="461ef-117">Short and Long Weak References</span></span>  
 <span data-ttu-id="461ef-118">Można utworzyć krótkiej słabe odwołanie lub długo słabe odwołanie:</span><span class="sxs-lookup"><span data-stu-id="461ef-118">You can create a short weak reference or a long weak reference:</span></span>  
  
-   <span data-ttu-id="461ef-119">Krótka</span><span class="sxs-lookup"><span data-stu-id="461ef-119">Short</span></span>  
  
     <span data-ttu-id="461ef-120">Staje się celem krótki słabe odwołanie `null` gdy obiekt jest odzyskiwane przez wyrzucanie elementów bezużytecznych.</span><span class="sxs-lookup"><span data-stu-id="461ef-120">The target of a short weak reference becomes `null` when the object is reclaimed by garbage collection.</span></span> <span data-ttu-id="461ef-121">Słabe odwołanie jest sam obiekt zarządzany i podlega wyrzucania elementów bezużytecznych, podobnie jak inne zarządzanego obiektu.</span><span class="sxs-lookup"><span data-stu-id="461ef-121">The weak reference is itself a managed object, and is subject to garbage collection just like any other managed object.</span></span>  <span data-ttu-id="461ef-122">Krótkie słabe odwołanie jest domyślny konstruktor dla <xref:System.WeakReference>.</span><span class="sxs-lookup"><span data-stu-id="461ef-122">A short weak reference is the default constructor for <xref:System.WeakReference>.</span></span>  
  
-   <span data-ttu-id="461ef-123">długi</span><span class="sxs-lookup"><span data-stu-id="461ef-123">Long</span></span>  
  
     <span data-ttu-id="461ef-124">Długi słabe odwołanie jest zachowywane po obiektu <xref:System.Object.Finalize%2A> została wywołana metoda.</span><span class="sxs-lookup"><span data-stu-id="461ef-124">A long weak reference is retained after the object's <xref:System.Object.Finalize%2A> method has been called.</span></span> <span data-ttu-id="461ef-125">Dzięki temu obiekt do odtworzenia, ale stan obiektu jest nieprzewidywalne.</span><span class="sxs-lookup"><span data-stu-id="461ef-125">This allows the object to be recreated, but the state of the object remains unpredictable.</span></span> <span data-ttu-id="461ef-126">Aby użyć długie odwołania, określ `true` w <xref:System.WeakReference> konstruktora.</span><span class="sxs-lookup"><span data-stu-id="461ef-126">To use a long reference, specify `true` in the <xref:System.WeakReference> constructor.</span></span>  
  
     <span data-ttu-id="461ef-127">Jeśli nie ma typu obiektu <xref:System.Object.Finalize%2A> metody, funkcja krótki słabe odwołanie ma zastosowanie i słabe odwołanie jest prawidłowy tylko do momentu docelowej są zbierane, co może nastąpić w dowolnej chwili po finalizator uruchomieniu.</span><span class="sxs-lookup"><span data-stu-id="461ef-127">If the object's type does not have a <xref:System.Object.Finalize%2A> method, the short weak reference functionality applies and the weak reference is valid only until the target is collected, which can occur anytime after the finalizer is run.</span></span>
