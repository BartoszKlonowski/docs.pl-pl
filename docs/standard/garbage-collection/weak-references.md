# <a name="weak-references"></a><span data-ttu-id="7e95b-102">Słabe odwołania</span><span class="sxs-lookup"><span data-stu-id="7e95b-102">Weak References</span></span>
<span data-ttu-id="7e95b-103">Moduł wyrzucania elementów bezużytecznych nie może zebrać obiektu, który jest używany przez aplikację, gdy kod aplikacji może dotrzeć do tego obiektu.</span><span class="sxs-lookup"><span data-stu-id="7e95b-103">The garbage collector cannot collect an object in use by an application while the application's code can reach that object.</span></span> <span data-ttu-id="7e95b-104">Aplikacja ma silne odwołanie do obiektu.</span><span class="sxs-lookup"><span data-stu-id="7e95b-104">The application is said to have a strong reference to the object.</span></span>  
  
 <span data-ttu-id="7e95b-105">Słabe odwołanie umożliwia modułowi wyrzucania elementów bezużytecznych zbieranie obiektu, a jednocześnie pozwala aplikacji na dostęp do obiektu.</span><span class="sxs-lookup"><span data-stu-id="7e95b-105">A weak reference permits the garbage collector to collect the object while still allowing the application to access the object.</span></span> <span data-ttu-id="7e95b-106">Słabe odwołanie jest prawidłowe tylko w nieokreślonym czasie do momentu zebrania obiektu, gdy nie istnieją silne odwołania.</span><span class="sxs-lookup"><span data-stu-id="7e95b-106">A weak reference is valid only during the indeterminate amount of time until the object is collected when no strong references exist.</span></span> <span data-ttu-id="7e95b-107">W przypadku użycia słabego odwołania aplikacja może nadal uzyskać silne odwołanie do obiektu, co uniemożliwia jego zebranie.</span><span class="sxs-lookup"><span data-stu-id="7e95b-107">When you use a weak reference, the application can still obtain a strong reference to the object, which prevents it from being collected.</span></span> <span data-ttu-id="7e95b-108">Jednak zawsze istnieje ryzyko, że moduł zbierający elementy bezużyteczne uzyska najpierw do obiektu przed ponownym ustanowieniem silnego odwołania.</span><span class="sxs-lookup"><span data-stu-id="7e95b-108">However, there is always the risk that the garbage collector will get to the object first before a strong reference is reestablished.</span></span>  
  
 <span data-ttu-id="7e95b-109">Słabe odwołania są przydatne w przypadku obiektów, które korzystają z dużej ilości pamięci, ale można je odtworzyć, jeśli są odzyskiwane przez wyrzucanie elementów bezużytecznych.</span><span class="sxs-lookup"><span data-stu-id="7e95b-109">Weak references are useful for objects that use a lot of memory, but can be recreated easily if they are reclaimed by garbage collection.</span></span>  
  
 <span data-ttu-id="7e95b-110">Załóżmy, że widok drzewa w aplikacji Windows Forms wyświetla skomplikowaną hierarchiczną opcję opcji dla użytkownika.</span><span class="sxs-lookup"><span data-stu-id="7e95b-110">Suppose a tree view in a Windows Forms application displays a complex hierarchical choice of options to the user.</span></span> <span data-ttu-id="7e95b-111">Jeśli dane podstawowe są duże, utrzymywanie drzewa w pamięci jest nieefektywne, gdy użytkownik korzysta z innego elementu w aplikacji.</span><span class="sxs-lookup"><span data-stu-id="7e95b-111">If the underlying data is large, keeping the tree in memory is inefficient when the user is involved with something else in the application.</span></span>  
  
 <span data-ttu-id="7e95b-112">Gdy użytkownik przełączy się do innej części aplikacji, można użyć klasy <xref:System.WeakReference>, aby utworzyć słabe odwołanie do drzewa i zniszczyć wszystkie silne odwołania.</span><span class="sxs-lookup"><span data-stu-id="7e95b-112">When the user switches away to another part of the application, you can use the <xref:System.WeakReference> class to create a weak reference to the tree and destroy all strong references.</span></span> <span data-ttu-id="7e95b-113">Gdy użytkownik przełącza się z powrotem do drzewa, aplikacja próbuje uzyskać silne odwołanie do drzewa i, jeśli to się powiedzie, unika odbudowy drzewa.</span><span class="sxs-lookup"><span data-stu-id="7e95b-113">When the user switches back to the tree, the application attempts to obtain a strong reference to the tree and, if successful, avoids reconstructing the tree.</span></span>  
  
 <span data-ttu-id="7e95b-114">Aby ustanowić słabe odwołanie z obiektem, należy utworzyć <xref:System.WeakReference> przy użyciu wystąpienia obiektu, który ma być śledzony.</span><span class="sxs-lookup"><span data-stu-id="7e95b-114">To establish a weak reference with an object, you create a <xref:System.WeakReference> using the instance of the object to be tracked.</span></span> <span data-ttu-id="7e95b-115">Następnie ustawisz dla właściwości <xref:System.WeakReference.Target%2A> ten obiekt i ustawisz oryginalne odwołanie do obiektu, aby `null`.</span><span class="sxs-lookup"><span data-stu-id="7e95b-115">You then set the <xref:System.WeakReference.Target%2A> property to that object and set the original reference to the object to `null`.</span></span> <span data-ttu-id="7e95b-116">Aby zapoznać się z przykładem kodu, zobacz <xref:System.WeakReference> w bibliotece klas.</span><span class="sxs-lookup"><span data-stu-id="7e95b-116">For a code example, see <xref:System.WeakReference> in the class library.</span></span>  
  
## <a name="short-and-long-weak-references"></a><span data-ttu-id="7e95b-117">Krótkie i długie słabe odwołania</span><span class="sxs-lookup"><span data-stu-id="7e95b-117">Short and Long Weak References</span></span>  
 <span data-ttu-id="7e95b-118">Można utworzyć krótkie odwołanie słabe lub długie, słabe odwołanie:</span><span class="sxs-lookup"><span data-stu-id="7e95b-118">You can create a short weak reference or a long weak reference:</span></span>  
  
- <span data-ttu-id="7e95b-119">Wybierak</span><span class="sxs-lookup"><span data-stu-id="7e95b-119">Short</span></span>  
  
     <span data-ttu-id="7e95b-120">Obiekt docelowy krótkiego odwołania zostanie `null`, gdy obiekt zostanie odrzucony przez wyrzucanie elementów bezużytecznych.</span><span class="sxs-lookup"><span data-stu-id="7e95b-120">The target of a short weak reference becomes `null` when the object is reclaimed by garbage collection.</span></span> <span data-ttu-id="7e95b-121">Słabe odwołanie jest samym obiektem zarządzanym i podlega wyrzucaniu elementów bezużytecznych, podobnie jak każdy inny obiekt zarządzany.</span><span class="sxs-lookup"><span data-stu-id="7e95b-121">The weak reference is itself a managed object, and is subject to garbage collection just like any other managed object.</span></span>  <span data-ttu-id="7e95b-122">Krótkie niesłabe odwołanie jest konstruktorem bez parametrów dla <xref:System.WeakReference>.</span><span class="sxs-lookup"><span data-stu-id="7e95b-122">A short weak reference is the parameterless constructor for <xref:System.WeakReference>.</span></span>  
  
- <span data-ttu-id="7e95b-123">Długo</span><span class="sxs-lookup"><span data-stu-id="7e95b-123">Long</span></span>
