# <a name="weak-references"></a><span data-ttu-id="911f1-102">Słabe odwołania</span><span class="sxs-lookup"><span data-stu-id="911f1-102">Weak References</span></span>
<span data-ttu-id="911f1-103">Moduł zbierający elementy bezużyteczne nie może zbierać obiektu używanego przez aplikację, podczas gdy kod aplikacji może dotrzeć do tego obiektu.</span><span class="sxs-lookup"><span data-stu-id="911f1-103">The garbage collector cannot collect an object in use by an application while the application's code can reach that object.</span></span> <span data-ttu-id="911f1-104">Mówi się, że aplikacja ma silne odniesienie do obiektu.</span><span class="sxs-lookup"><span data-stu-id="911f1-104">The application is said to have a strong reference to the object.</span></span>  
  
 <span data-ttu-id="911f1-105">Słabe odwołanie zezwala modułowi odśmiecania pamięci do zbierania obiektu, a jednocześnie umożliwia aplikacji dostęp do obiektu.</span><span class="sxs-lookup"><span data-stu-id="911f1-105">A weak reference permits the garbage collector to collect the object while still allowing the application to access the object.</span></span> <span data-ttu-id="911f1-106">Słabe odwołanie jest prawidłowy tylko w czasie nieokreślony czasu, dopóki obiekt jest zbierane, gdy nie istnieją silne odwołania.</span><span class="sxs-lookup"><span data-stu-id="911f1-106">A weak reference is valid only during the indeterminate amount of time until the object is collected when no strong references exist.</span></span> <span data-ttu-id="911f1-107">Korzystając z słabeodwołanie, aplikacja nadal można uzyskać silne odwołanie do obiektu, który uniemożliwia jego zbierania.</span><span class="sxs-lookup"><span data-stu-id="911f1-107">When you use a weak reference, the application can still obtain a strong reference to the object, which prevents it from being collected.</span></span> <span data-ttu-id="911f1-108">Jednak zawsze istnieje ryzyko, że moduł zbierający elementy bezużyteczne przejdzie do obiektu najpierw przed ponownym ustanowieniem silnego odwołania.</span><span class="sxs-lookup"><span data-stu-id="911f1-108">However, there is always the risk that the garbage collector will get to the object first before a strong reference is reestablished.</span></span>  
  
 <span data-ttu-id="911f1-109">Słabe odwołania są przydatne dla obiektów, które używają dużo pamięci, ale można je łatwo odtworzyć, jeśli są one odzyskane przez wyrzucanie elementów bezużytecznych.</span><span class="sxs-lookup"><span data-stu-id="911f1-109">Weak references are useful for objects that use a lot of memory, but can be recreated easily if they are reclaimed by garbage collection.</span></span>  
  
 <span data-ttu-id="911f1-110">Załóżmy, że widok drzewa w aplikacji Formularze systemu Windows wyświetla złożony hierarchiczny wybór opcji dla użytkownika.</span><span class="sxs-lookup"><span data-stu-id="911f1-110">Suppose a tree view in a Windows Forms application displays a complex hierarchical choice of options to the user.</span></span> <span data-ttu-id="911f1-111">Jeśli dane źródłowe są duże, utrzymywanie drzewa w pamięci jest nieefektywne, gdy użytkownik jest zaangażowany w coś innego w aplikacji.</span><span class="sxs-lookup"><span data-stu-id="911f1-111">If the underlying data is large, keeping the tree in memory is inefficient when the user is involved with something else in the application.</span></span>  
  
 <span data-ttu-id="911f1-112">Gdy użytkownik przełącza się do innej części aplikacji, <xref:System.WeakReference> można użyć klasy, aby utworzyć słabe odwołanie do drzewa i zniszczyć wszystkie silne odwołania.</span><span class="sxs-lookup"><span data-stu-id="911f1-112">When the user switches away to another part of the application, you can use the <xref:System.WeakReference> class to create a weak reference to the tree and destroy all strong references.</span></span> <span data-ttu-id="911f1-113">Gdy użytkownik przełączy się z powrotem do drzewa, aplikacja próbuje uzyskać silne odwołanie do drzewa i, jeśli się powiedzie, unika rekonstrukcji drzewa.</span><span class="sxs-lookup"><span data-stu-id="911f1-113">When the user switches back to the tree, the application attempts to obtain a strong reference to the tree and, if successful, avoids reconstructing the tree.</span></span>  
  
 <span data-ttu-id="911f1-114">Aby ustanowić słabe odwołanie z obiektem, należy utworzyć <xref:System.WeakReference> przy użyciu wystąpienia obiektu, który ma być śledzony.</span><span class="sxs-lookup"><span data-stu-id="911f1-114">To establish a weak reference with an object, you create a <xref:System.WeakReference> using the instance of the object to be tracked.</span></span> <span data-ttu-id="911f1-115">Następnie należy <xref:System.WeakReference.Target%2A> ustawić właściwość na ten obiekt i ustawić `null`oryginalne odwołanie do obiektu .</span><span class="sxs-lookup"><span data-stu-id="911f1-115">You then set the <xref:System.WeakReference.Target%2A> property to that object and set the original reference to the object to `null`.</span></span> <span data-ttu-id="911f1-116">Na przykład kodu <xref:System.WeakReference> zobacz w bibliotece klas.</span><span class="sxs-lookup"><span data-stu-id="911f1-116">For a code example, see <xref:System.WeakReference> in the class library.</span></span>  
  
## <a name="short-and-long-weak-references"></a><span data-ttu-id="911f1-117">Krótkie i długie słabe odniesienia</span><span class="sxs-lookup"><span data-stu-id="911f1-117">Short and Long Weak References</span></span>  
 <span data-ttu-id="911f1-118">Można utworzyć krótkie słabe odniesienie lub długie słabe odniesienie:</span><span class="sxs-lookup"><span data-stu-id="911f1-118">You can create a short weak reference or a long weak reference:</span></span>  
  
- <span data-ttu-id="911f1-119">Krótki</span><span class="sxs-lookup"><span data-stu-id="911f1-119">Short</span></span>  
  
     <span data-ttu-id="911f1-120">Obiekt docelowy krótkiego słabe `null` odwołanie staje się, gdy obiekt jest odzyskiwany przez wyrzucanie elementów bezużytecznych.</span><span class="sxs-lookup"><span data-stu-id="911f1-120">The target of a short weak reference becomes `null` when the object is reclaimed by garbage collection.</span></span> <span data-ttu-id="911f1-121">Słabe odwołanie jest sam obiekt zarządzany i podlega wyrzucania elementów bezużytecznych, podobnie jak każdy inny obiekt zarządzany.</span><span class="sxs-lookup"><span data-stu-id="911f1-121">The weak reference is itself a managed object, and is subject to garbage collection just like any other managed object.</span></span>  <span data-ttu-id="911f1-122">Krótkie słabe odwołanie jest konstruktorem bezparametrów dla <xref:System.WeakReference>.</span><span class="sxs-lookup"><span data-stu-id="911f1-122">A short weak reference is the parameterless constructor for <xref:System.WeakReference>.</span></span>  
  
- <span data-ttu-id="911f1-123">Długi</span><span class="sxs-lookup"><span data-stu-id="911f1-123">Long</span></span>  
  
     <span data-ttu-id="911f1-124">Długie słabe odwołanie jest zachowywane po <xref:System.Object.Finalize%2A> wywołaniu metody obiektu.</span><span class="sxs-lookup"><span data-stu-id="911f1-124">A long weak reference is retained after the object's <xref:System.Object.Finalize%2A> method has been called.</span></span> <span data-ttu-id="911f1-125">Dzięki temu obiekt ma zostać odtworzony, ale stan obiektu pozostaje nieprzewidywalny.</span><span class="sxs-lookup"><span data-stu-id="911f1-125">This allows the object to be recreated, but the state of the object remains unpredictable.</span></span> <span data-ttu-id="911f1-126">Aby użyć odwołania długiego, należy określić `true` w <xref:System.WeakReference> konstruktorze.</span><span class="sxs-lookup"><span data-stu-id="911f1-126">To use a long reference, specify `true` in the <xref:System.WeakReference> constructor.</span></span>  
  
     <span data-ttu-id="911f1-127">Jeśli typ obiektu nie ma <xref:System.Object.Finalize%2A> metody, stosuje się funkcję krótkiego słabego odwołania, a słabe odwołanie jest prawidłowe tylko do momentu zebrania obiektu docelowego, co może wystąpić w dowolnym momencie po uruchomieniu finalizatora.</span><span class="sxs-lookup"><span data-stu-id="911f1-127">If the object's type does not have a <xref:System.Object.Finalize%2A> method, the short weak reference functionality applies and the weak reference is valid only until the target is collected, which can occur anytime after the finalizer is run.</span></span>  
  
 <span data-ttu-id="911f1-128">Aby ustanowić silne odwołanie i ponownie <xref:System.WeakReference.Target%2A> użyć obiektu, rzutować właściwość typu <xref:System.WeakReference> do typu obiektu.</span><span class="sxs-lookup"><span data-stu-id="911f1-128">To establish a strong reference and use the object again, cast the <xref:System.WeakReference.Target%2A> property of a <xref:System.WeakReference> to the type of the object.</span></span> <span data-ttu-id="911f1-129">Jeśli <xref:System.WeakReference.Target%2A> właściwość `null`zwraca , obiekt został zebrany; w przeciwnym razie można nadal używać obiektu, ponieważ aplikacja odzyskał silne odwołanie do niego.</span><span class="sxs-lookup"><span data-stu-id="911f1-129">If the <xref:System.WeakReference.Target%2A> property returns `null`, the object was collected; otherwise, you can continue to use the object because the application has regained a strong reference to it.</span></span>
