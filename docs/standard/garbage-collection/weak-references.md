# <a name="weak-references"></a><span data-ttu-id="26600-102">Słabe odwołania</span><span class="sxs-lookup"><span data-stu-id="26600-102">Weak References</span></span>

 <span data-ttu-id="26600-105">Słabe odwołanie umożliwia modułowi wyrzucania elementów bezużytecznych zbieranie obiektu, a jednocześnie pozwala aplikacji na dostęp do obiektu.</span><span class="sxs-lookup"><span data-stu-id="26600-105">A weak reference permits the garbage collector to collect the object while still allowing the application to access the object.</span></span> <span data-ttu-id="26600-106">Słabe odwołanie jest prawidłowe tylko w nieokreślonym czasie do momentu zebrania obiektu, gdy nie istnieją silne odwołania.</span><span class="sxs-lookup"><span data-stu-id="26600-106">A weak reference is valid only during the indeterminate amount of time until the object is collected when no strong references exist.</span></span> <span data-ttu-id="26600-107">W przypadku użycia słabego odwołania aplikacja może nadal uzyskać silne odwołanie do obiektu, co uniemożliwia jego zebranie.</span><span class="sxs-lookup"><span data-stu-id="26600-107">When you use a weak reference, the application can still obtain a strong reference to the object, which prevents it from being collected.</span></span> <span data-ttu-id="26600-108">Jednak zawsze istnieje ryzyko, że moduł zbierający elementy bezużyteczne uzyska najpierw do obiektu przed ponownym ustanowieniem silnego odwołania.</span><span class="sxs-lookup"><span data-stu-id="26600-108">However, there is always the risk that the garbage collector will get to the object first before a strong reference is reestablished.</span></span>  
  
 <span data-ttu-id="26600-109">Słabe odwołania są przydatne w przypadku obiektów, które korzystają z dużej ilości pamięci, ale można je odtworzyć, jeśli są odzyskiwane przez wyrzucanie elementów bezużytecznych.</span><span class="sxs-lookup"><span data-stu-id="26600-109">Weak references are useful for objects that use a lot of memory, but can be recreated easily if they are reclaimed by garbage collection.</span></span>  
  
 <span data-ttu-id="26600-110">Załóżmy, że widok drzewa w aplikacji Windows Forms wyświetla skomplikowaną hierarchiczną opcję opcji dla użytkownika.</span><span class="sxs-lookup"><span data-stu-id="26600-110">Suppose a tree view in a Windows Forms application displays a complex hierarchical choice of options to the user.</span></span> <span data-ttu-id="26600-111">Jeśli dane podstawowe są duże, utrzymywanie drzewa w pamięci jest nieefektywne, gdy użytkownik korzysta z innego elementu w aplikacji.</span><span class="sxs-lookup"><span data-stu-id="26600-111">If the underlying data is large, keeping the tree in memory is inefficient when the user is involved with something else in the application.</span></span>

 <span data-ttu-id="26600-114">Aby ustanowić słabe odwołanie do obiektu, należy utworzyć <xref:System.WeakReference> wystąpienie obiektu, które ma być śledzone.</span><span class="sxs-lookup"><span data-stu-id="26600-114">To establish a weak reference with an object, you create a <xref:System.WeakReference> using the instance of the object to be tracked.</span></span> <span data-ttu-id="26600-115">Następnie ustawisz <xref:System.WeakReference.Target%2A> Właściwość na ten obiekt i ustawisz oryginalne odwołanie do obiektu `null` .</span><span class="sxs-lookup"><span data-stu-id="26600-115">You then set the <xref:System.WeakReference.Target%2A> property to that object and set the original reference to the object to `null`.</span></span> <span data-ttu-id="26600-116">Aby zapoznać się z przykładem kodu, zobacz <xref:System.WeakReference> w bibliotece klas.</span><span class="sxs-lookup"><span data-stu-id="26600-116">For a code example, see <xref:System.WeakReference> in the class library.</span></span>

 <span data-ttu-id="26600-118">Można utworzyć krótkie odwołanie słabe lub długie, słabe odwołanie:</span><span class="sxs-lookup"><span data-stu-id="26600-118">You can create a short weak reference or a long weak reference:</span></span>  
  
- <span data-ttu-id="26600-119">Wybierak</span><span class="sxs-lookup"><span data-stu-id="26600-119">Short</span></span>  
  
     <span data-ttu-id="26600-120">Obiekt docelowy krótkiego słabego odwołania zostaje przerzucony `null` po odbraniu obiektu przez wyrzucanie elementów bezużytecznych.</span><span class="sxs-lookup"><span data-stu-id="26600-120">The target of a short weak reference becomes `null` when the object is reclaimed by garbage collection.</span></span> <span data-ttu-id="26600-121">Słabe odwołanie jest samym obiektem zarządzanym i podlega wyrzucaniu elementów bezużytecznych, podobnie jak każdy inny obiekt zarządzany.</span><span class="sxs-lookup"><span data-stu-id="26600-121">The weak reference is itself a managed object, and is subject to garbage collection just like any other managed object.</span></span>  <span data-ttu-id="26600-122">Krótkie, słabe odwołanie jest konstruktorem bez parametrów dla <xref:System.WeakReference> .</span><span class="sxs-lookup"><span data-stu-id="26600-122">A short weak reference is the parameterless constructor for <xref:System.WeakReference>.</span></span>  
  
- <span data-ttu-id="26600-123">Długo</span><span class="sxs-lookup"><span data-stu-id="26600-123">Long</span></span>  
  
     <span data-ttu-id="26600-124">Po wywołaniu metody obiektu jest zachowywane długie odwołanie <xref:System.Object.Finalize%2A> .</span><span class="sxs-lookup"><span data-stu-id="26600-124">A long weak reference is retained after the object's <xref:System.Object.Finalize%2A> method has been called.</span></span> <span data-ttu-id="26600-125">Pozwala to na odtworzenie obiektu, ale stan obiektu pozostaje nieprzewidywalny.</span><span class="sxs-lookup"><span data-stu-id="26600-125">This allows the object to be recreated, but the state of the object remains unpredictable.</span></span> <span data-ttu-id="26600-126">Aby użyć długich odwołań, należy określić `true` w <xref:System.WeakReference> konstruktorze.</span><span class="sxs-lookup"><span data-stu-id="26600-126">To use a long reference, specify `true` in the <xref:System.WeakReference> constructor.</span></span>  
  
     <span data-ttu-id="26600-127">Jeśli typ obiektu nie ma <xref:System.Object.Finalize%2A> metody, stosowana jest krótka funkcja referencyjna odniesienia, a słabe odwołanie jest prawidłowe tylko do momentu zebrania elementu docelowego, co może wystąpić w dowolnym momencie po uruchomieniu finalizatora.</span><span class="sxs-lookup"><span data-stu-id="26600-127">If the object's type does not have a <xref:System.Object.Finalize%2A> method, the short weak reference functionality applies and the weak reference is valid only until the target is collected, which can occur anytime after the finalizer is run.</span></span>
