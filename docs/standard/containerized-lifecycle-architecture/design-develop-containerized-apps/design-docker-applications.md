# <a name="design-docker-applications"></a>Projektowanie aplikacji Docker

Rozdział 1 wprowadzono podstawowe pojęcia dotyczące kontenerów i Docker. Te informacje jest podstawowy poziom informacje wymagane do rozpoczęcia pracy. Jednak aplikacje przedsiębiorstwa mogą być złożone i składać się z wielu usług zamiast jednej usługi lub kontenera. W tych przypadkach opcjonalnym użyciem musisz wiedzieć dodatkowe podejścia do projektu, takie jak architektura Service-Oriented (SOA) oraz bardziej zaawansowanych mikrousług pojęcia i kontener pojęcia aranżacji. Zakres tego dokumentu nie jest ograniczone do mikrousług, ale do dowolnego Docker cyklu życia aplikacji, w związku z tym go nie Eksploruj architektura mikrousług szczegółowo ponieważ możesz również użyć kontenery i Docker z zadań, zadania w tle i regularnie Wyspy, lub nawet z metod wdrażania wbudowanymi aplikacji.

Przed uzyskujemy w cyklu życia aplikacji i opracowywania oprogramowania, należy wiedzieć, jak zamierzasz projektowania i tworzenia aplikacji i jakie są opcje projektu.
