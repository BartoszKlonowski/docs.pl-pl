<span data-ttu-id="a2a03-110">Aplikacje są skalowanych w poziomie w wielu systemach hosta, możliwość zarządzania każdym systemie hosta i streszczenie natychmiast złożoność podstawowej platformy staje się atrakcyjne.</span><span class="sxs-lookup"><span data-stu-id="a2a03-110">When applications are scaled-out across multiple host systems, the ability to manage each host system and abstract away the complexity of the underlying platform becomes attractive.</span></span> <span data-ttu-id="a2a03-111">To dokładnie co zapewniają koordynatorów i transfery danych.</span><span class="sxs-lookup"><span data-stu-id="a2a03-111">That's precisely what orchestrators and schedulers provide.</span></span> <span data-ttu-id="a2a03-112">Przyjrzyjmy się krótki opis ich tutaj:</span><span class="sxs-lookup"><span data-stu-id="a2a03-112">Let's take a brief look at them here:</span></span>

- <span data-ttu-id="a2a03-113">**Planiści**.</span><span class="sxs-lookup"><span data-stu-id="a2a03-113">**Schedulers**.</span></span><span data-ttu-id="a2a03-114"> "Planowanie" odnosi się do możliwości, aby administrator mógł załadować pliku service w systemie hosta, który określa sposób uruchamiania kontenera dla.</span><span class="sxs-lookup"><span data-stu-id="a2a03-114"> "Scheduling" refers to the ability for an administrator to load a service file onto a host system that establishes how to run a specific container.</span></span> <span data-ttu-id="a2a03-115">Uruchamianie kontenerów w klastrze Docker zwykle nazywane planowania.</span><span class="sxs-lookup"><span data-stu-id="a2a03-115">Launching containers in a Docker cluster tends to be known as scheduling.</span></span> <span data-ttu-id="a2a03-116">Chociaż planowanie odwołuje się do określonych act ładowania definicji usługi, w tym sensie bardziej ogólnych transfery danych są odpowiedzialne za przechwytywanie systemu init hosta do zarządzania usługami w każdej pojemności potrzebne.</span><span class="sxs-lookup"><span data-stu-id="a2a03-116">Although scheduling refers to the specific act of loading the service definition, in a more general sense, schedulers are responsible for hooking into a host's init system to manage services in whatever capacity needed.</span></span>

   <span data-ttu-id="a2a03-117">Harmonogram klastra ma wiele cele: efektywne korzystanie z zasobów klastra, Praca z ograniczeniami dotyczącymi umieszczania dostarczone przez użytkownika, w planowanie aplikacjom szybko nie pozostawić je w stanie oczekiwania, o stopniu "sprawiedliwe", są niezawodne, błędy, a zawsze być dostępna.</span><span class="sxs-lookup"><span data-stu-id="a2a03-117">A cluster scheduler has multiple goals: using the cluster's resources efficiently, working with user-supplied placement constraints, scheduling applications rapidly to not leave them in a pending state, having a degree of "fairness," being robust to errors, and always be available.</span></span>

- <span data-ttu-id="a2a03-118">**Koordynatorów**.</span><span class="sxs-lookup"><span data-stu-id="a2a03-118">**Orchestrators**.</span></span><span data-ttu-id="a2a03-119"> Platform rozszerzyć możliwości zarządzania cyklem życia złożonych, wielokontenerowych obciążenia wdrożone w klastrze hostów.</span><span class="sxs-lookup"><span data-stu-id="a2a03-119"> Platforms extend life-cycle management capabilities to complex, multi-container workloads deployed on a cluster of hosts.</span></span> <span data-ttu-id="a2a03-120">Zapewniając abstrakcyjność infrastruktury hostowania, narzędziami koordynowania zapewniają sposób traktować cały klaster jako obiekt docelowy pojedynczego wdrożenia.</span><span class="sxs-lookup"><span data-stu-id="a2a03-120">By abstracting the host infrastructure, orchestration tools give users a way to treat the entire cluster as a single deployment target.</span></span>

   <span data-ttu-id="a2a03-122">Aranżacja to szeroka termin, który odwołuje się do kontenera planowania, zarządzania klastrem i prawdopodobnie udostępniania dodatkowych hostów.</span><span class="sxs-lookup"><span data-stu-id="a2a03-122">Orchestration is a broad term that refers to container scheduling, cluster management, and possibly the provisioning of additional hosts.</span></span>
