<span data-ttu-id="59774-106">**Obraz kontenera**: Pakiet ze wszystkimi zależnościami i informacje potrzebne do utworzenia kontenera.</span><span class="sxs-lookup"><span data-stu-id="59774-106">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="59774-107">Obraz obejmuje wszystkie zależności (np. Platform) oraz wdrażaniem i uruchamianiem konfiguracji ma być używany przez kontener środowiska uruchomieniowego.</span><span class="sxs-lookup"><span data-stu-id="59774-107">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="59774-108">Zwykle obraz pochodzi z wielu obrazy podstawowe, znajdujących się na siebie w celu utworzenia kontenera systemu plików, stos warstwy.</span><span class="sxs-lookup"><span data-stu-id="59774-108">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container's filesystem.</span></span> <span data-ttu-id="59774-109">Obraz jest niezmienny, po jego utworzeniu.</span><span class="sxs-lookup"><span data-stu-id="59774-109">An image is immutable once it has been created.</span></span>

<span data-ttu-id="59774-110">**Dockerfile**: Plik tekstowy, który zawiera instrukcje dotyczące sposobu tworzenia obrazu platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="59774-110">**Dockerfile**: A text file that contains instructions on how to build a Docker image.</span></span> <span data-ttu-id="59774-111">Jest on podobny do skryptu wsadowego, pierwszy wiersz stany obrazu podstawowego zacząć od, a następnie postępuj zgodnie z instrukcjami, aby zainstalować wymagane programy i pliki kopii i potrzebujesz itd., dopóki nie zostanie wyświetlony środowiska pracy.</span><span class="sxs-lookup"><span data-stu-id="59774-111">It's like a batch script, the first line states the base image to begin with and then follow the instructions to install required programs, copy files, and so on, until you get the working environment you need.</span></span>

<span data-ttu-id="59774-112">**Tworzenie**: Akcja kompilowania obrazu kontenera, na podstawie informacji i kontekst dostarczonych przez jego pliku Dockerfile, a także dodatkowe pliki w folderze, w którym obraz jest kompilowany.</span><span class="sxs-lookup"><span data-stu-id="59774-112">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="59774-113">Możesz tworzyć obrazy platformy Docker **`docker build`** polecenia.</span><span class="sxs-lookup"><span data-stu-id="59774-113">You can build images with the Docker **`docker build`** command.</span></span>

<span data-ttu-id="59774-114">**kontener**: Wystąpienie obrazu platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="59774-114">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="59774-115">Kontener reprezentuje wykonanie jednej aplikacji, proces lub usługi.</span><span class="sxs-lookup"><span data-stu-id="59774-115">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="59774-116">Składa się z zawartości obrazu platformy Docker, środowisko wykonawcze i standardowy zestaw instrukcji.</span><span class="sxs-lookup"><span data-stu-id="59774-116">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="59774-117">Podczas skalowania usługi, tworzysz wiele wystąpień kontenera za pomocą tego samego obrazu.</span><span class="sxs-lookup"><span data-stu-id="59774-117">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="59774-118">Zadania usługi batch można też tworzyć wiele kontenerów za pomocą tego samego obrazu, przekazując różnych parametrów do każdego wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="59774-118">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="59774-119">**Woluminy**: Oferują zapisywalny system plików, można użyć kontenera.</span><span class="sxs-lookup"><span data-stu-id="59774-119">**Volumes**: Offer a writable filesystem that the container can use.</span></span> <span data-ttu-id="59774-120">Ponieważ obrazy są przeznaczone tylko do odczytu, ale w większości programów trzeba było pisać w systemie plików, woluminy Dodaj warstwę zapisywalny, dodaną do obrazu kontenera, dzięki czemu programy mają dostęp do zapisu plików.</span><span class="sxs-lookup"><span data-stu-id="59774-120">Since images are read-only but most programs need to write to the filesystem, volumes add a writable layer, on top of the container image, so the programs have access to a writable filesystem.</span></span> <span data-ttu-id="59774-121">Program nie może ustalić uzyskuje dostęp do plików warstwowych, jest po prostu systemu plików w zwykły sposób.</span><span class="sxs-lookup"><span data-stu-id="59774-121">The program doesn't know it's accessing a layered filesystem, it's just the filesystem as usual.</span></span> <span data-ttu-id="59774-122">Woluminy live w systemie hosta, a także są zarządzane przez platformę Docker.</span><span class="sxs-lookup"><span data-stu-id="59774-122">Volumes live in the host system and are managed by Docker.</span></span>

<span data-ttu-id="59774-123">**Tag**: Znak lub etykiety, które można stosować do obrazów, dzięki czemu można zidentyfikować różne obrazy lub wersji tego samego obrazu (w zależności od numeru wersji lub środowiska docelowego).</span><span class="sxs-lookup"><span data-stu-id="59774-123">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="59774-124">**Kompilacja wieloetapowych**: Jest funkcją, od 17.05 platformy Docker lub wyższym, która pomaga zredukować rozmiar końcowy obrazów.</span><span class="sxs-lookup"><span data-stu-id="59774-124">**Multi-stage Build**: Is a feature, since Docker 17.05 or higher, that helps to reduce the size of the final images.</span></span> <span data-ttu-id="59774-125">W kilku zdaniach wieloetapowych kompilacji można użyć, na przykład duży obraz podstawowy, zawierające zestaw SDK do kompilowania i publikowania aplikacji i następnie przy użyciu folderu publikowania za pomocą małych tylko do środowiska wykonawczego obrazu podstawowego, dają znacznie mniejszy obraz końcowe</span><span class="sxs-lookup"><span data-stu-id="59774-125">In a few sentences, with multi-stage build you can use, for example, a large base image, containing the SDK, for compiling and publishing the application and then using the publishing folder with a small runtime-only base image, to produce a much smaller final image</span></span>

<span data-ttu-id="59774-126">**Repozytorium (repozytorium)**: Zbiór powiązanych obrazów platformy Docker, oznaczone znacznikiem, który wskazuje wersję obrazu.</span><span class="sxs-lookup"><span data-stu-id="59774-126">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="59774-127">Niektóre repozytoria zawierają wiele wariantów określonego obrazu, takich jak obrazy, które zawierają zestawy SDK (większe), obrazy, które zawierają tylko środowisk uruchomieniowych (jaśniejszy) itp. Warianty te mogą być oznaczone tagami.</span><span class="sxs-lookup"><span data-stu-id="59774-127">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="59774-128">Jednym repozytorium może zawierać warianty platformy, takich jak obraz systemu Linux i Windows obraz.</span><span class="sxs-lookup"><span data-stu-id="59774-128">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="59774-129">**Rejestr**: To usługa, która zapewnia dostęp do repozytoriów.</span><span class="sxs-lookup"><span data-stu-id="59774-129">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="59774-130">Rejestr domyślne obrazy najbardziej publiczny jest [usługi Docker Hub](https://hub.docker.com/) (właściciel platformy Docker jako organizacja).</span><span class="sxs-lookup"><span data-stu-id="59774-130">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="59774-131">Rejestr zawiera zazwyczaj repozytoriów z wielu zespołów.</span><span class="sxs-lookup"><span data-stu-id="59774-131">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="59774-132">Firmy często mają prywatnych rejestrów do przechowywania i zarządzania obrazami, którym zostały utworzone.</span><span class="sxs-lookup"><span data-stu-id="59774-132">Companies often have private registries to store and manage images they've created.</span></span> <span data-ttu-id="59774-133">Usługa Azure Container Registry jest inny przykład.</span><span class="sxs-lookup"><span data-stu-id="59774-133">Azure Container Registry is another example.</span></span>

<span data-ttu-id="59774-134">**Obraz architektury wielu**: W przypadku wielu architektury jest funkcja, która ułatwia wybór odpowiedniej obrazu, zależnie od platformy, której platformy Docker jest uruchomiona, na przykład, gdy plik Dockerfile żąda obrazu podstawowego **`FROM microsoft/dotnet:2.1-sdk`** z rejestru faktycznie pobiera **`2.1-sdk-nanoserver-1709`**, **`2.1-sdk-nanoserver-1803`** lub **`2.1-sdk-alpine`**, w zależności od systemu operacyjnego i wersji, w którym platformy Docker jest uruchomiona.</span><span class="sxs-lookup"><span data-stu-id="59774-134">**Multi-arch image**: For multi-architecture, is a feature that simplifies the selection of the appropriate image, according to the platform where Docker is running, for example, when a Dockerfile requests a base image **`FROM microsoft/dotnet:2.1-sdk`** from the registry it actually gets **`2.1-sdk-nanoserver-1709`**, **`2.1-sdk-nanoserver-1803`** or **`2.1-sdk-alpine`**, depending on the operating system and version where Docker is running.</span></span>

<span data-ttu-id="59774-135">**Docker Hub**: Publicznego rejestru przekazywania obrazów i pracować z nimi.</span><span class="sxs-lookup"><span data-stu-id="59774-135">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="59774-136">Usługi docker Hub udostępnia platformy Docker hostingu obrazu, publicznych lub prywatnych rejestrów, wyzwalaczy kompilacji oraz elementów web hook i integracja z usługą GitHub i Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="59774-136">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="59774-137">**Usługa Azure Container Registry**: Zasób publicznego do pracy z obrazów platformy Docker i jego składniki na platformie Azure.</span><span class="sxs-lookup"><span data-stu-id="59774-137">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="59774-138">Dzięki temu rejestru, który znajduje się w pobliżu wdrożeń na platformie Azure i zapewnia kontrolę dostępu, dzięki czemu można użyć usługi Azure Active Directory, grup i uprawnień.</span><span class="sxs-lookup"><span data-stu-id="59774-138">This provides a registry that's close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="59774-139">**Docker Trusted Registry (DTR)**: Rejestru usługi Docker (z platformy Docker), może być zainstalowany w środowisku lokalnym, więc znajduje się w obrębie centrum danych organizacji i sieć.</span><span class="sxs-lookup"><span data-stu-id="59774-139">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization's datacenter and network.</span></span> <span data-ttu-id="59774-140">Jest to wygodne dla prywatnych obrazów, które powinny być zarządzane w ramach organizacji.</span><span class="sxs-lookup"><span data-stu-id="59774-140">It's convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="59774-141">Docker Trusted Registry wchodzi w skład produktów Docker Datacenter.</span><span class="sxs-lookup"><span data-stu-id="59774-141">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="59774-142">Aby uzyskać więcej informacji, zobacz [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="59774-142">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="59774-143">**Platformę docker Community Edition (CE)**: Narzędzia programistyczne dla Windows i macOS, tworzenia, uruchamiania i testowania kontenery lokalnie.</span><span class="sxs-lookup"><span data-stu-id="59774-143">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="59774-144">Docker CE dla Windows zapewnia środowisk deweloperskich dla systemu Linux i Windows kontenery.</span><span class="sxs-lookup"><span data-stu-id="59774-144">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="59774-145">Hosta platformy Docker w systemie Linux na Windows opiera się na [funkcji Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) maszyny wirtualnej.</span><span class="sxs-lookup"><span data-stu-id="59774-145">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) virtual machine.</span></span> <span data-ttu-id="59774-146">Host dla kontenerów Windows jest bezpośrednio oparty na Windows.</span><span class="sxs-lookup"><span data-stu-id="59774-146">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="59774-147">Docker CE dla komputerów Mac opiera się w ramach funkcji Hypervisor firmy Apple i [funkcji hypervisor xhyve](https://github.com/mist64/xhyve), zapewniającą maszyny wirtualnej hosta platformy Docker w systemie Linux na systemu operacyjnego Mac Docker X. CE for Windows i dla komputerów Mac zastępuje przybornika Docker zależała od bazy danych Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="59774-147">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="59774-148">**Rozwiązania docker Enterprise Edition (EE)**: Wersja skali korporacyjnej narzędzia platformy Docker do tworzenia aplikacji systemu Linux i Windows.</span><span class="sxs-lookup"><span data-stu-id="59774-148">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="59774-149">**Redagowanie**: Narzędzie wiersza polecenia i YAML pliku formatu metadanych do definiowania i uruchamiania aplikacji obsługującej wiele kontenerów.</span><span class="sxs-lookup"><span data-stu-id="59774-149">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="59774-150">Należy zdefiniować pojedynczej aplikacji opartych na obrazach wielu z co najmniej jeden plik yml, które mogą zastępować wartości w zależności od środowiska.</span><span class="sxs-lookup"><span data-stu-id="59774-150">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="59774-151">Po utworzeniu definicjami można wdrożyć aplikacji wielokontenerowych zupełnie za pomocą jednego polecenia (docker-compose się), tworzy kontener na obraz na hoście platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="59774-151">After you've created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="59774-152">**Klaster**: Kolekcja hostów platformy Docker, udostępniane tak, jakby była pojedynczego wirtualnego hosta platformy Docker, tak aby aplikację można skalować do wielu wystąpień usługi rozkładają się na wielu hostach w klastrze.</span><span class="sxs-lookup"><span data-stu-id="59774-152">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="59774-153">Klastry platformy docker można tworzyć przy użyciu rozwiązania Kubernetes, Azure Service Fabric, Docker Swarm i Mesosphere DC/OS.</span><span class="sxs-lookup"><span data-stu-id="59774-153">Docker clusters can be created with Kubernetes, Azure Service Fabric, Docker Swarm and Mesosphere DC/OS.</span></span>
