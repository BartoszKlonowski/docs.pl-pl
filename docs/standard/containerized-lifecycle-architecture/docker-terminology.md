# <a name="docker-terminology"></a><span data-ttu-id="67f01-103">Terminologia platformy docker</span><span class="sxs-lookup"><span data-stu-id="67f01-103">Docker terminology</span></span>

<span data-ttu-id="67f01-104">W tej sekcji przedstawiono terminów i definicje, z którymi należy zapoznać się z przed zagłębieniem bardziej platformy docker (uzyskać dalsze definicje, zobacz rozbudowane [słownik](https://docs.docker.com/glossary/) dostarczane przez platformę Docker na <https://docs.docker.com/glossary/>:</span><span class="sxs-lookup"><span data-stu-id="67f01-104">This section lists terms and definitions with which you should become familiar with before delving deeper into Docker (for further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker at <https://docs.docker.com/glossary/>:</span></span>

-   <span data-ttu-id="67f01-105">**Obraz kontenera** pakietu ze wszystkimi zależnościami i informacje potrzebne do tworzenia kontenera.</span><span class="sxs-lookup"><span data-stu-id="67f01-105">**Container image** A package with all of the dependencies and information needed to create a container.</span></span> <span data-ttu-id="67f01-106">Obraz obejmuje wszystkie zależności (np. Platform) oraz wdrażania i konfiguracji, który będzie używany przez kontener środowiska uruchomieniowego.</span><span class="sxs-lookup"><span data-stu-id="67f01-106">An image includes all of the dependencies (such as frameworks) plus deployment and configuration to be used by a container runtime.</span></span> <span data-ttu-id="67f01-107">Zwykle obraz pochodzi z wielu obrazy podstawowe, są warstwy ułożone jeden na jego podstawie innych w celu utworzenia kontenera systemu plików.</span><span class="sxs-lookup"><span data-stu-id="67f01-107">Usually, an image derives from multiple base images that are layers stacked one atop the other to form the container's file system.</span></span> <span data-ttu-id="67f01-108">Obraz jest niezmienny, po jego utworzeniu.</span><span class="sxs-lookup"><span data-stu-id="67f01-108">An image is immutable after it has been created.</span></span>

-   <span data-ttu-id="67f01-109">**Kontener** wystąpienia obrazu platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="67f01-109">**Container** An instance of a Docker image.</span></span> <span data-ttu-id="67f01-110">Kontener reprezentuje środowiska uruchomieniowego dla pojedynczej aplikacji, proces lub usługi.</span><span class="sxs-lookup"><span data-stu-id="67f01-110">A container represents a runtime for a single application, process, or service.</span></span> <span data-ttu-id="67f01-111">Składa się z zawartości obrazu platformy Docker, środowisko uruchomieniowe i standardowy zestaw instrukcji.</span><span class="sxs-lookup"><span data-stu-id="67f01-111">It consists of the contents of a Docker image, a runtime environment, and a standard set of instructions.</span></span> <span data-ttu-id="67f01-112">Podczas skalowania usługi, tworzysz wiele wystąpień kontenera za pomocą tego samego obrazu.</span><span class="sxs-lookup"><span data-stu-id="67f01-112">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="67f01-113">Lub zadania usługi batch można tworzyć wiele kontenerów za pomocą tego samego obrazu, przekazując różnych parametrów do każdego wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="67f01-113">Or, a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

-   <span data-ttu-id="67f01-114">**Tag** znaku lub etykiety, które można zastosować do obrazów, dzięki czemu można zidentyfikować różne obrazy lub wersji tego samego obrazu (w zależności od numeru wersji lub środowisko docelowe).</span><span class="sxs-lookup"><span data-stu-id="67f01-114">**Tag** A mark or label that you can apply to images so that different images or versions of the same image (depending on the version number or the destination environment) can be identified.</span></span>

-   <span data-ttu-id="67f01-115">**Plik Dockerfile** plik tekstowy, który zawiera instrukcje dotyczące sposobu tworzenia obrazu platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="67f01-115">**Dockerfile** A text file that contains instructions for how to build a Docker image.</span></span>

-   <span data-ttu-id="67f01-116">**Tworzenie** Akcja kompilowania obrazu kontenera na podstawie informacji i kontekst dostarczonych przez jego pliku Dockerfile, a także dodatkowe pliki w folderze, w którym obraz jest kompilowany.</span><span class="sxs-lookup"><span data-stu-id="67f01-116">**Build** The action of building a container image based on the information and context provided by its Dockerfile as well as additional files in the folder where the image is built.</span></span> <span data-ttu-id="67f01-117">Możesz tworzyć obrazy za pomocą polecenia kompilacji platformy docker platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="67f01-117">You can build images by using the Docker docker build command.</span></span>

-   <span data-ttu-id="67f01-118">**Repozytorium (znany także jako repozytorium)** zbiór powiązanych obrazów platformy Docker z etykietą znacznik, który wskazuje wersję obrazu.</span><span class="sxs-lookup"><span data-stu-id="67f01-118">**Repository (also known as repo)** A collection of related Docker images labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="67f01-119">Niektóre repozytoria zawierają wiele wariantów określonego obrazu, takich jak obrazy, które zawierają zestawy SDK (większe), obrazy, które zawierają tylko środowisk uruchomieniowych (jaśniejszy), i tak dalej.</span><span class="sxs-lookup"><span data-stu-id="67f01-119">Some repositories contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), and so on.</span></span> <span data-ttu-id="67f01-120">Warianty te mogą być oznaczone tagami.</span><span class="sxs-lookup"><span data-stu-id="67f01-120">Those variants can be marked with tags.</span></span> <span data-ttu-id="67f01-121">Pojedyncze repozytorium może zawierać warianty platformy, takich jak obraz systemu Linux i Windows obraz.</span><span class="sxs-lookup"><span data-stu-id="67f01-121">A single repository can contain platform variants, such as a Linux image and a Windows image.</span></span>

-   <span data-ttu-id="67f01-122">**Rejestr** to usługa, która zapewnia dostęp do repozytoriów.</span><span class="sxs-lookup"><span data-stu-id="67f01-122">**Registry** A service that provides access to repositories.</span></span> <span data-ttu-id="67f01-123">Rejestr domyślne obrazy najbardziej publiczny jest [usługi Docker Hub](https://hub.docker.com/) (właściciel platformy Docker jako organizacja).</span><span class="sxs-lookup"><span data-stu-id="67f01-123">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="67f01-124">Rejestr zawiera zazwyczaj repozytoriów z wielu zespołów.</span><span class="sxs-lookup"><span data-stu-id="67f01-124">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="67f01-125">Firmy często mają prywatnych rejestrów do przechowywania obrazów i zarządzanie nimi, które zostały przez nich utworzone.</span><span class="sxs-lookup"><span data-stu-id="67f01-125">Companies often have private registries to store and manage images that they've created.</span></span> <span data-ttu-id="67f01-126">*Usługa Azure Container Registry* jest inny przykład.</span><span class="sxs-lookup"><span data-stu-id="67f01-126">*Azure Container Registry* is another example.</span></span>

-   <span data-ttu-id="67f01-127">**Usługi docker Hub** publicznego rejestru przekazywania obrazów i pracować z nimi.</span><span class="sxs-lookup"><span data-stu-id="67f01-127">**Docker Hub** A public registry to upload images and work with them.</span></span> <span data-ttu-id="67f01-128">Usługi docker Hub udostępnia platformy Docker hostingu obrazu, publicznych lub prywatnych rejestrów, wyzwalaczy kompilacji oraz elementów web hook i integracja z usługą GitHub i Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="67f01-128">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

-   <span data-ttu-id="67f01-129">**Usługa Azure Container Registry** zasób publicznego do pracy z obrazów platformy Docker i jego składniki na platformie Azure.</span><span class="sxs-lookup"><span data-stu-id="67f01-129">**Azure Container Registry** A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="67f01-130">Dzięki temu rejestru, który znajduje się w pobliżu wdrożeń na platformie Azure i zapewnia kontrolę dostępu, dzięki czemu można użyć usługi Azure Active Directory, grup i uprawnień.</span><span class="sxs-lookup"><span data-stu-id="67f01-130">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

-   <span data-ttu-id="67f01-131">**Docker Trusted Registry (DTR)** usługi rejestru aparatu Docker (z platformy Docker), który można zainstalować w środowisku lokalnym, aby znajduje się w obrębie centrum danych i sieci w organizacji.</span><span class="sxs-lookup"><span data-stu-id="67f01-131">**Docker Trusted Registry (DTR)** A Docker registry service (from Docker) that you can install on-premises so that it resides within the organization's datacenter and network.</span></span> <span data-ttu-id="67f01-132">Jest to wygodne dla prywatnych obrazów, które powinny być zarządzane w ramach organizacji.</span><span class="sxs-lookup"><span data-stu-id="67f01-132">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="67f01-133">Docker Trusted Registry wchodzi w skład produktów Docker Datacenter.</span><span class="sxs-lookup"><span data-stu-id="67f01-133">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="67f01-134">Aby uzyskać więcej informacji, przejdź do [ https://docs.docker.com/docker-trusted-registry/overview/ ](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="67f01-134">For more information, go to [https://docs.docker.com/docker-trusted-registry/overview/](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

-   <span data-ttu-id="67f01-135">**Docker Community Edition (CE)** narzędzia programistyczne dla Windows i macOS, tworzenia, uruchamiania i testowania kontenery lokalnie.</span><span class="sxs-lookup"><span data-stu-id="67f01-135">**Docker Community Edition (CE)** Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="67f01-136">Docker CE dla Windows zapewnia środowisk deweloperskich dla systemu Linux i Windows kontenery.</span><span class="sxs-lookup"><span data-stu-id="67f01-136">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="67f01-137">Hosta platformy Docker w systemie Linux na Windows opiera się na [funkcji Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) maszyny Wirtualnej.</span><span class="sxs-lookup"><span data-stu-id="67f01-137">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) VM.</span></span> <span data-ttu-id="67f01-138">Host dla kontenerów Windows jest bezpośrednio oparty na Windows.</span><span class="sxs-lookup"><span data-stu-id="67f01-138">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="67f01-139">Docker CE dla komputerów Mac opiera się w ramach funkcji Hypervisor firmy Apple i [funkcji hypervisor xhyve](https://github.com/mist64/xhyve), zapewniającą hosta platformy Docker w systemie Linux maszyny Wirtualnej na systemu operacyjnego Mac Docker X. CE for Windows i dla komputerów Mac zastępuje przybornika platformy Docker został oparty na Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="67f01-139">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host VM on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

-   <span data-ttu-id="67f01-140">**Docker Enterprise Edition (EE)** skali korporacyjnej wersję narzędzia platformy Docker do tworzenia aplikacji systemu Linux i Windows.</span><span class="sxs-lookup"><span data-stu-id="67f01-140">**Docker Enterprise Edition (EE)** An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

-   <span data-ttu-id="67f01-141">**Redagowanie** narzędzie wiersza polecenia i YAML pliku w formacie za pomocą metadanych do definiowania i uruchamiania aplikacji z wieloma kontenerami w celu.</span><span class="sxs-lookup"><span data-stu-id="67f01-141">**Compose** A command-line tool and YAML file format with metadata for defining and running multicontainer applications.</span></span> <span data-ttu-id="67f01-142">Należy zdefiniować pojedynczej aplikacji opartych na obrazach wielu z co najmniej jeden plik yml, które mogą zastępować wartości w zależności od środowiska.</span><span class="sxs-lookup"><span data-stu-id="67f01-142">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="67f01-143">Po utworzeniu definicjami wieloma kontenerami w celu całej aplikacji można wdrożyć za pomocą jednego polecenia (docker-compose się), tworzy kontener na obraz na hoście platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="67f01-143">After you have created the definitions, you can deploy the entire multicontainer application by using a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

-   <span data-ttu-id="67f01-144">**Klaster** zbiór hostów platformy Docker, udostępniane tak, jakby były one jednego wirtualnego hosta platformy Docker tak, aby aplikację można skalować do wielu wystąpień usługi rozkładają się na wielu hostach w klastrze.</span><span class="sxs-lookup"><span data-stu-id="67f01-144">**Cluster** A collection of Docker hosts exposed as if they were a single virtual Docker host so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="67f01-145">Można utworzyć klastry platformy Docker przy użyciu rozwiązania Docker Swarm, Mesosphere DC/OS, Kubernetes i usługi Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="67f01-145">You can create Docker clusters by using Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="67f01-146">(Jeśli używasz rozwiązania Docker Swarm dla zarządzania klastrem, zwykle odwołasz się do klastra jako *swarm* zamiast klastra.)</span><span class="sxs-lookup"><span data-stu-id="67f01-146">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

-   <span data-ttu-id="67f01-147">**Program orchestrator** narzędzia, która upraszcza zarządzanie klastrów i hostów platformy Docker.</span><span class="sxs-lookup"><span data-stu-id="67f01-147">**Orchestrator** A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="67f01-148">Przy użyciu koordynatorów, można zarządzać ich obrazów kontenerów i hostami za pomocą interfejsu wiersza polecenia lub graficznego interfejsu użytkownika.</span><span class="sxs-lookup"><span data-stu-id="67f01-148">Using orchestrators, you can manage their images, containers, and hosts through a CLI or a graphical user interface.</span></span> <span data-ttu-id="67f01-149">Można zarządzać kontenerowa, konfiguracje, równoważenia obciążenia, odnajdowania usług, wysoką dostępność, Konfiguracja hosta platformy Docker i więcej.</span><span class="sxs-lookup"><span data-stu-id="67f01-149">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="67f01-150">Program orchestrator jest odpowiedzialny za systemem, dystrybucja, skalowanie i naprawiania obciążeń między kolekcję węzłów.</span><span class="sxs-lookup"><span data-stu-id="67f01-150">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="67f01-151">Zazwyczaj produkty programu orchestrator są tego samego produktów, które udostępniają infrastrukturę klastra, takie jak Mesosphere DC/OS, Kubernetes, Docker Swarm i usługi Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="67f01-151">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
