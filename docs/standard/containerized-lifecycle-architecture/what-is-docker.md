<span data-ttu-id="2d053-126">Ponieważ kontenery wymaga znacznie mniej zasobów (na przykład, nie ma potrzeby pełnej wersji systemu operacyjnego), są one łatwe do wdrożenia i szybkiego uruchamiania.</span><span class="sxs-lookup"><span data-stu-id="2d053-126">Because containers require far fewer resources (for example, they do not need a full OS), they are easy to deploy and they start fast.</span></span> <span data-ttu-id="2d053-127">Dzięki temu umożliwiają zwiększenie gęstości, co oznacza, czy można uruchomić więcej usług w tej samej jednostce sprzętu, zmniejszając w ten sposób kosztów.</span><span class="sxs-lookup"><span data-stu-id="2d053-127">This makes it possible for you to have higher density, meaning that you can run more services on the same hardware unit, thereby reducing costs.</span></span>

<span data-ttu-id="2d053-128">Jako efekt uboczny uruchomionych na tym samym jądra osiągasz izolacji jest mniejszy niż maszyny wirtualne.</span><span class="sxs-lookup"><span data-stu-id="2d053-128">As a side effect of running on the same kernel, you achieve less isolation than VMs.</span></span>

<span data-ttu-id="2d053-129">Głównym celem obrazu jest fakt, że środowisko (zależności) takie same w różnych wdrożeniach.</span><span class="sxs-lookup"><span data-stu-id="2d053-129">The main goal of an image is that it makes the environment (dependencies) the same across different deployments.</span></span> <span data-ttu-id="2d053-130">Oznacza to, można to debugować na maszynie i następnie wdrożyć go na inny komputer, z tym samym środowisku, gwarantowana.</span><span class="sxs-lookup"><span data-stu-id="2d053-130">This means that you can debug it on your machine and then deploy it to another machine with the same environment guaranteed.</span></span>

<span data-ttu-id="2d053-131">Obraz kontenera jest sposobem pakietu aplikacji lub usługi oraz wdrożyć ją w sposób niezawodny i odtworzenia.</span><span class="sxs-lookup"><span data-stu-id="2d053-131">A container image is a way to package an app or service and deploy it in a reliable and reproducible manner.</span></span> <span data-ttu-id="2d053-132">W związku z tym Docker nie jest tylko technologii, również jest filozofii i procesu.</span><span class="sxs-lookup"><span data-stu-id="2d053-132">In this respect, Docker is not only a technology, it's also a philosophy and a process.</span></span>
