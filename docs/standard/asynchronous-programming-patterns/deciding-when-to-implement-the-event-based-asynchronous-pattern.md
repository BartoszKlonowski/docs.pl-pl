 <span data-ttu-id="ba4e0-108">Ogólnie rzecz biorąc należy udostępnić funkcje asynchroniczne przy użyciu opartego na zdarzeniach wzorca asynchronicznego zawsze, gdy jest to możliwe.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-108">In general, you should expose asynchronous features using the Event-based Asynchronous Pattern whenever possible.</span></span> <span data-ttu-id="ba4e0-109">Istnieją jednak pewne wymagania, które nie spełniają wzorzec oparty na zdarzeniach.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-109">However, there are some requirements that the event-based pattern cannot meet.</span></span> <span data-ttu-id="ba4e0-110">W takich przypadkach użytkownik może być konieczne wdrożenie <xref:System.IAsyncResult> wzorzec oprócz wzorzec oparty na zdarzeniach.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-110">In those cases, you may need to implement the <xref:System.IAsyncResult> pattern in addition to the event-based pattern.</span></span>

>  <span data-ttu-id="ba4e0-111">Rzadko <xref:System.IAsyncResult> wzorzec do zaimplementowania bez wzorzec oparty na zdarzeniach, również implementowany.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-111">It is rare for the <xref:System.IAsyncResult> pattern to be implemented without the event-based pattern also being implemented.</span></span>  
  
## <a name="guidelines"></a><span data-ttu-id="ba4e0-112">Wytyczne dotyczące</span><span class="sxs-lookup"><span data-stu-id="ba4e0-112">Guidelines</span></span>  
 <span data-ttu-id="ba4e0-113">Na poniższej liście opisano wytyczne podczas należy zaimplementować wzorzec asynchroniczny oparty na zdarzeniach:</span><span class="sxs-lookup"><span data-stu-id="ba4e0-113">The following list describes the guidelines for when you should implement Event-based Asynchronous Pattern:</span></span>

-   <span data-ttu-id="ba4e0-118">Nie ujawniaj <xref:System.IAsyncResult> wzorzec bez narażania również wzorzec oparty na zdarzeniach.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-118">Do not expose the <xref:System.IAsyncResult> pattern without also exposing the event-based pattern.</span></span>  
  
-   <span data-ttu-id="ba4e0-119">Jeśli musisz ujawnić <xref:System.IAsyncResult> wzorca, w tym celu jako zaawansowana opcja.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-119">If you must expose the <xref:System.IAsyncResult> pattern, do so as an advanced option.</span></span> <span data-ttu-id="ba4e0-120">Na przykład, jeśli użytkownik generuje obiekt serwera proxy, generowanie wzorzec oparty na zdarzeniach domyślnie z opcją do generowania <xref:System.IAsyncResult> wzorca.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-120">For example, if you generate a proxy object, generate the event-based pattern by default, with an option to generate the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="ba4e0-121">Tworzenie implementacji wzorca opartego na zdarzeniach w swojej <xref:System.IAsyncResult> implementacji wzorca.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-121">Build your event-based pattern implementation on your <xref:System.IAsyncResult> pattern implementation.</span></span>  
  
-   <span data-ttu-id="ba4e0-122">Należy unikać udostępnianie wzorzec oparty na zdarzeniach i <xref:System.IAsyncResult> wzorca na tej samej klasy.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-122">Avoid exposing both the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class.</span></span> <span data-ttu-id="ba4e0-123">Udostępnić wzorzec oparty na zdarzeniach w klasach "na wyższym poziomie" i <xref:System.IAsyncResult> wzorca na "obniżyć poziom" klasy.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-123">Expose the event-based pattern on "higher level" classes and the <xref:System.IAsyncResult> pattern on "lower level" classes.</span></span> <span data-ttu-id="ba4e0-124">Na przykład porównać wzorzec oparty na zdarzeniach na <xref:System.Net.WebClient> składnika za pomocą <xref:System.IAsyncResult> wzorca na <xref:System.Web.HttpRequest> klasy.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-124">For example, compare the event-based pattern on the <xref:System.Net.WebClient> component with the <xref:System.IAsyncResult> pattern on the <xref:System.Web.HttpRequest> class.</span></span>  
  
    -   <span data-ttu-id="ba4e0-125">Udostępnić wzorzec oparty na zdarzeniach i <xref:System.IAsyncResult> wzorca na tej samej klasy, gdy wymaga zgodności.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-125">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class when compatibility requires it.</span></span> <span data-ttu-id="ba4e0-126">Na przykład, jeśli już zostały wydane interfejsu API, który używa <xref:System.IAsyncResult> wzorzec, należy zachować <xref:System.IAsyncResult> wzorca dla zgodności z poprzednimi wersjami.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-126">For example, if you have already released an API that uses the <xref:System.IAsyncResult> pattern, you would need to retain the <xref:System.IAsyncResult> pattern for backward compatibility.</span></span>  
  
    -   <span data-ttu-id="ba4e0-127">Udostępnić wzorzec oparty na zdarzeniach i <xref:System.IAsyncResult> wzorca na tej samej klasy, jeśli wynikowy złożoność modelu obiektu przewyższa korzyści oddzielenie implementacji.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-127">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class if the resulting object model complexity outweighs the benefit of separating the implementations.</span></span> <span data-ttu-id="ba4e0-128">Zaleca się uwidocznić obu wzorców dotyczących jednej klasy niż Aby uniknąć ujawnienia wzorzec oparty na zdarzeniach.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-128">It is better to expose both patterns on a single class than to avoid exposing the event-based pattern.</span></span>  
  
    -   <span data-ttu-id="ba4e0-129">Jeśli musisz ujawnić wzorzec oparty na zdarzeniach i <xref:System.IAsyncResult> wzorzec dotyczących jednej klasy, użyj <xref:System.ComponentModel.EditorBrowsableAttribute> równa <xref:System.ComponentModel.EditorBrowsableState.Advanced> do oznaczania <xref:System.IAsyncResult> implementacji wzorca jako funkcja zaawansowana.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-129">If you must expose both the event-based pattern and <xref:System.IAsyncResult> pattern on a single class, use <xref:System.ComponentModel.EditorBrowsableAttribute> set to <xref:System.ComponentModel.EditorBrowsableState.Advanced> to mark the <xref:System.IAsyncResult> pattern implementation as an advanced feature.</span></span> <span data-ttu-id="ba4e0-130">Oznacza to, aby środowiska projektowe, takie jak Visual Studio technologii IntelliSense, aby nie wyświetlać <xref:System.IAsyncResult> właściwości i metody.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-130">This indicates to design environments, such as Visual Studio IntelliSense, not to display the <xref:System.IAsyncResult> properties and methods.</span></span> <span data-ttu-id="ba4e0-131">Te właściwości i metody są nadal w pełni użyteczne, ale Deweloper pracy za pomocą funkcji IntelliSense zawiera bardziej zrozumiały widok interfejsu API.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-131">These properties and methods are still fully usable, but the developer working through IntelliSense has a clearer view of the API.</span></span>  
  
## <a name="criteria-for-exposing-the-iasyncresult-pattern-in-addition-to-the-event-based-pattern"></a><span data-ttu-id="ba4e0-132">Kryteria udostępnianie wzorzec IAsyncResult oprócz wzorzec oparty na zdarzeniach</span><span class="sxs-lookup"><span data-stu-id="ba4e0-132">Criteria for Exposing the IAsyncResult Pattern in Addition to the Event-based Pattern</span></span>  
 <span data-ttu-id="ba4e0-133">Gdy wzorzec asynchroniczny oparty na zdarzeniach ma wiele zalet w scenariuszach wymienionych wcześniej, ma pewne wady, które powinny być świadome Jeśli wydajność jest z najważniejszych wymagań.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-133">While the Event-based Asynchronous Pattern has many benefits under the previously mentioned scenarios, it does have some drawbacks, which you should be aware of if performance is your most important requirement.</span></span>  
  
 <span data-ttu-id="ba4e0-134">Istnieją trzy scenariusze, w których nie opisano kwestii wzorzec oparty na zdarzeniach, jak również <xref:System.IAsyncResult> wzorca:</span><span class="sxs-lookup"><span data-stu-id="ba4e0-134">There are three scenarios that the event-based pattern does not address as well as the <xref:System.IAsyncResult> pattern:</span></span>  
  
-   <span data-ttu-id="ba4e0-135">Poczekaj na jednym blokowania <xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="ba4e0-135">Blocking wait on one <xref:System.IAsyncResult></span></span>  
  
-   <span data-ttu-id="ba4e0-136">Blokowanie oczekiwania na wielu <xref:System.IAsyncResult> obiektów</span><span class="sxs-lookup"><span data-stu-id="ba4e0-136">Blocking wait on many <xref:System.IAsyncResult> objects</span></span>  
  
-   <span data-ttu-id="ba4e0-137">Sondowanie na ukończenie <xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="ba4e0-137">Polling for completion on the <xref:System.IAsyncResult></span></span>  
  
 <span data-ttu-id="ba4e0-138">Można scenariuszy za pomocą wzorca opartego na zdarzeniach, ale jest bardziej skomplikowane niż użycie <xref:System.IAsyncResult> wzorca.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-138">You can address these scenarios by using the event-based pattern, but doing so is more cumbersome than using the <xref:System.IAsyncResult> pattern.</span></span>  
  
 <span data-ttu-id="ba4e0-139">Deweloperzy często używają <xref:System.IAsyncResult> wzorca dla usług, które zwykle mają wymagania bardzo wysokiej wydajności.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-139">Developers often use the <xref:System.IAsyncResult> pattern for services that typically have very high performance requirements.</span></span> <span data-ttu-id="ba4e0-140">Na przykład sondowania ukończenia scenariusza jest to technika o wysokiej wydajności serwera.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-140">For example, the polling for completion scenario is a high-performance server technique.</span></span>  
  
 <span data-ttu-id="ba4e0-141">Ponadto jest mniej wydajne niż wzorzec oparty na zdarzeniach <xref:System.IAsyncResult> wzorca, ponieważ tworzy on więcej obiektów, szczególnie <xref:System.EventArgs>, a ponieważ synchronizuje między wątkami.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-141">Additionally, the event-based pattern is less efficient than the <xref:System.IAsyncResult> pattern because it creates more objects, especially <xref:System.EventArgs>, and because it synchronizes across threads.</span></span>  
  
 <span data-ttu-id="ba4e0-142">Na poniższej liście przedstawiono kilka zaleceń, które należy wykonać, jeśli użytkownik zdecyduje się użyć <xref:System.IAsyncResult> wzorca:</span><span class="sxs-lookup"><span data-stu-id="ba4e0-142">The following list shows some recommendations to follow if you decide to use the <xref:System.IAsyncResult> pattern:</span></span>  
  
-   <span data-ttu-id="ba4e0-143">Tylko udostępnianie <xref:System.IAsyncResult> wzorca, gdy potrzebujesz specjalnie obsługę <xref:System.Threading.WaitHandle> lub <xref:System.IAsyncResult> obiektów.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-143">Only expose the <xref:System.IAsyncResult> pattern when you specifically require support for <xref:System.Threading.WaitHandle> or <xref:System.IAsyncResult> objects.</span></span>  
  
-   <span data-ttu-id="ba4e0-144">Tylko udostępnianie <xref:System.IAsyncResult> wzorca, gdy masz istniejących interfejsów API, który używa <xref:System.IAsyncResult> wzorca.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-144">Only expose the <xref:System.IAsyncResult> pattern when you have an existing API that uses the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="ba4e0-145">W przypadku istniejącego interfejsu API na podstawie <xref:System.IAsyncResult> wzorca, należy wziąć pod uwagę także udostępnianie wzorzec oparty na zdarzeniach w swojej następnej wersji.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-145">If you have an existing API based on the <xref:System.IAsyncResult> pattern, consider also exposing the event-based pattern in your next release.</span></span>  
  
-   <span data-ttu-id="ba4e0-146">Tylko udostępnianie <xref:System.IAsyncResult> wzorca, jeśli masz wymagania o wysokiej wydajności, które upewnieniu się, nie mogą zostać spełnione przez wzorzec oparty na zdarzeniach, ale mogą zostać spełnione przez <xref:System.IAsyncResult> wzorca.</span><span class="sxs-lookup"><span data-stu-id="ba4e0-146">Only expose <xref:System.IAsyncResult> pattern if you have high performance requirements which you have verified cannot be met by the event-based pattern but can be met by the <xref:System.IAsyncResult> pattern.</span></span>
