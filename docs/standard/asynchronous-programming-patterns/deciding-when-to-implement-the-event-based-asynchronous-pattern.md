## <a name="general-principles"></a><span data-ttu-id="4e889-107">Zasady ogólne</span><span class="sxs-lookup"><span data-stu-id="4e889-107">General Principles</span></span>

## <a name="guidelines"></a><span data-ttu-id="4e889-112">Wytyczne</span><span class="sxs-lookup"><span data-stu-id="4e889-112">Guidelines</span></span>

<span data-ttu-id="4e889-113">Na poniższej liście opisano wskazówki dotyczące tego, kiedy należy zaimplementować wzorzec asynchroniczny oparty na zdarzeniach:</span><span class="sxs-lookup"><span data-stu-id="4e889-113">The following list describes the guidelines for when you should implement Event-based Asynchronous Pattern:</span></span>

- <span data-ttu-id="4e889-115">Nie uwidaczniawyszy <xref:System.IAsyncResult> wzorzec, gdy klasa jest używana głównie w aplikacji klienckiej, na przykład formularze systemu Windows.</span><span class="sxs-lookup"><span data-stu-id="4e889-115">Do not expose the <xref:System.IAsyncResult> pattern when your class is primarily used in a client application, for example Windows Forms.</span></span>

- <span data-ttu-id="4e889-118">Nie należy <xref:System.IAsyncResult> ujawniać wzorzec bez również odsłaniając wzorzec oparty na zdarzeniach.</span><span class="sxs-lookup"><span data-stu-id="4e889-118">Do not expose the <xref:System.IAsyncResult> pattern without also exposing the event-based pattern.</span></span>

- <span data-ttu-id="4e889-121">Skompiluj implementację <xref:System.IAsyncResult> wzorca opartego na zdarzeniach na implementacji wzorca.</span><span class="sxs-lookup"><span data-stu-id="4e889-121">Build your event-based pattern implementation on your <xref:System.IAsyncResult> pattern implementation.</span></span>

- <span data-ttu-id="4e889-122">Należy unikać ujawniania zarówno wzorca <xref:System.IAsyncResult> opartego na zdarzeniach, jak i wzorca w tej samej klasie.</span><span class="sxs-lookup"><span data-stu-id="4e889-122">Avoid exposing both the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class.</span></span> <span data-ttu-id="4e889-123">Uwidaczniawy wzorzec <xref:System.IAsyncResult> oparty na zdarzeniach na klasach "wyższego poziomu" i wzorzec na klasach "niższego poziomu".</span><span class="sxs-lookup"><span data-stu-id="4e889-123">Expose the event-based pattern on "higher level" classes and the <xref:System.IAsyncResult> pattern on "lower level" classes.</span></span> <span data-ttu-id="4e889-124">Na przykład porównaj wzorzec <xref:System.Net.WebClient> oparty <xref:System.IAsyncResult> na zdarzeniu na składniku ze szykiem na <xref:System.Web.HttpRequest> klasie.</span><span class="sxs-lookup"><span data-stu-id="4e889-124">For example, compare the event-based pattern on the <xref:System.Net.WebClient> component with the <xref:System.IAsyncResult> pattern on the <xref:System.Web.HttpRequest> class.</span></span>

  - <span data-ttu-id="4e889-125">Uwidacz nia <xref:System.IAsyncResult> wzorca opartego na zdarzeniach i wzorzec w tej samej klasie, gdy wymaga tego zgodność.</span><span class="sxs-lookup"><span data-stu-id="4e889-125">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class when compatibility requires it.</span></span> <span data-ttu-id="4e889-126">Na przykład jeśli już zwolniono interfejs API, który używa wzorca, <xref:System.IAsyncResult> należy zachować <xref:System.IAsyncResult> wzorzec dla zgodności z poprzednimi wersjami.</span><span class="sxs-lookup"><span data-stu-id="4e889-126">For example, if you have already released an API that uses the <xref:System.IAsyncResult> pattern, you would need to retain the <xref:System.IAsyncResult> pattern for backward compatibility.</span></span>

  - <span data-ttu-id="4e889-127">Uwidaczniaszy <xref:System.IAsyncResult> wzorzec oparty na zdarzeniach i wzorzec w tej samej klasie, jeśli wynikowa złożoność modelu obiektu przeważa nad korzyścią z oddzielenia implementacji.</span><span class="sxs-lookup"><span data-stu-id="4e889-127">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class if the resulting object model complexity outweighs the benefit of separating the implementations.</span></span> <span data-ttu-id="4e889-128">Lepiej jest udostępnić oba wzorce na jednej klasy, niż uniknąć ujawniania wzorca opartego na zdarzeniach.</span><span class="sxs-lookup"><span data-stu-id="4e889-128">It is better to expose both patterns on a single class than to avoid exposing the event-based pattern.</span></span>

  - <span data-ttu-id="4e889-129">Jeśli należy udostępnić zarówno wzorzec oparty na zdarzeniach, jak <xref:System.IAsyncResult> i wzorzec w jednej klasie, należy użyć <xref:System.ComponentModel.EditorBrowsableAttribute> zestawu do <xref:System.ComponentModel.EditorBrowsableState.Advanced> oznaczenia implementacji <xref:System.IAsyncResult> wzorca jako zaawansowanej funkcji.</span><span class="sxs-lookup"><span data-stu-id="4e889-129">If you must expose both the event-based pattern and <xref:System.IAsyncResult> pattern on a single class, use <xref:System.ComponentModel.EditorBrowsableAttribute> set to <xref:System.ComponentModel.EditorBrowsableState.Advanced> to mark the <xref:System.IAsyncResult> pattern implementation as an advanced feature.</span></span> <span data-ttu-id="4e889-130">Oznacza to, że środowiska projektowe, takie jak Visual <xref:System.IAsyncResult> Studio IntelliSense, nie wyświetlają właściwości i metod.</span><span class="sxs-lookup"><span data-stu-id="4e889-130">This indicates to design environments, such as Visual Studio IntelliSense, not to display the <xref:System.IAsyncResult> properties and methods.</span></span> <span data-ttu-id="4e889-131">Te właściwości i metody są nadal w pełni użyteczne, ale deweloper pracujący za pośrednictwem IntelliSense ma jaśniejszy widok interfejsu API.</span><span class="sxs-lookup"><span data-stu-id="4e889-131">These properties and methods are still fully usable, but the developer working through IntelliSense has a clearer view of the API.</span></span>

## <a name="criteria-for-exposing-the-iasyncresult-pattern-in-addition-to-the-event-based-pattern"></a><span data-ttu-id="4e889-132">Kryteria dotyczące uwzidłania wzorca Wyniku IAsync w uzupełnieniu do wzorca opartego na zdarzeniach</span><span class="sxs-lookup"><span data-stu-id="4e889-132">Criteria for Exposing the IAsyncResult Pattern in Addition to the Event-based Pattern</span></span>

<span data-ttu-id="4e889-133">Podczas gdy wzorzec asynchroniczny oparty na zdarzeniach ma wiele korzyści w ramach wcześniej wymienionych scenariuszy, ma pewne wady, o których należy pamiętać, jeśli wydajność jest najważniejszym wymaganiem.</span><span class="sxs-lookup"><span data-stu-id="4e889-133">While the Event-based Asynchronous Pattern has many benefits under the previously mentioned scenarios, it does have some drawbacks, which you should be aware of if performance is your most important requirement.</span></span>

<span data-ttu-id="4e889-134">Istnieją trzy scenariusze, które wzorzec oparty na <xref:System.IAsyncResult> zdarzeniach nie odnosi się, jak również wzorzec:</span><span class="sxs-lookup"><span data-stu-id="4e889-134">There are three scenarios that the event-based pattern does not address as well as the <xref:System.IAsyncResult> pattern:</span></span>

<span data-ttu-id="4e889-139">Deweloperzy często <xref:System.IAsyncResult> używają wzorca dla usług, które zazwyczaj mają bardzo wysokie wymagania dotyczące wydajności.</span><span class="sxs-lookup"><span data-stu-id="4e889-139">Developers often use the <xref:System.IAsyncResult> pattern for services that typically have very high performance requirements.</span></span> <span data-ttu-id="4e889-140">Na przykład sondowanie dla scenariusza ukończenia jest techniką serwera o wysokiej wydajności.</span><span class="sxs-lookup"><span data-stu-id="4e889-140">For example, the polling for completion scenario is a high-performance server technique.</span></span>

- <span data-ttu-id="4e889-144">Tylko uwidaczniaszy <xref:System.IAsyncResult> wzorzec, gdy masz istniejący interfejs API, który używa wzorca. <xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="4e889-144">Only expose the <xref:System.IAsyncResult> pattern when you have an existing API that uses the <xref:System.IAsyncResult> pattern.</span></span>
