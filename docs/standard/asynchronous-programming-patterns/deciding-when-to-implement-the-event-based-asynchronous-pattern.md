## <a name="general-principles"></a><span data-ttu-id="14c48-107">Zasady ogólne</span><span class="sxs-lookup"><span data-stu-id="14c48-107">General Principles</span></span>

<span data-ttu-id="14c48-108">Ogólnie rzecz biorąc, należy uwidocznić funkcje asynchroniczne za pomocą wzorca asynchronicznego opartego na zdarzeniach, gdy jest to możliwe.</span><span class="sxs-lookup"><span data-stu-id="14c48-108">In general, you should expose asynchronous features using the Event-based Asynchronous Pattern whenever possible.</span></span> <span data-ttu-id="14c48-109">Jednak istnieją pewne wymagania, których wzorzec oparty na zdarzeniach nie może spełnić.</span><span class="sxs-lookup"><span data-stu-id="14c48-109">However, there are some requirements that the event-based pattern cannot meet.</span></span> <span data-ttu-id="14c48-110">W takich przypadkach może być konieczne zaimplementowanie <xref:System.IAsyncResult> wzorca oprócz wzorca opartego na zdarzeniach.</span><span class="sxs-lookup"><span data-stu-id="14c48-110">In those cases, you may need to implement the <xref:System.IAsyncResult> pattern in addition to the event-based pattern.</span></span>

## <a name="guidelines"></a><span data-ttu-id="14c48-112">Wytyczne</span><span class="sxs-lookup"><span data-stu-id="14c48-112">Guidelines</span></span>

<span data-ttu-id="14c48-113">Na poniższej liście przedstawiono wskazówki dotyczące sytuacji, w których należy zaimplementować wzorzec asynchroniczny oparty na zdarzeniach:</span><span class="sxs-lookup"><span data-stu-id="14c48-113">The following list describes the guidelines for when you should implement Event-based Asynchronous Pattern:</span></span>

- <span data-ttu-id="14c48-115">Nie ujawniaj <xref:System.IAsyncResult> wzorca, gdy Klasa jest używana głównie w aplikacji klienckiej, na przykład Windows Forms.</span><span class="sxs-lookup"><span data-stu-id="14c48-115">Do not expose the <xref:System.IAsyncResult> pattern when your class is primarily used in a client application, for example Windows Forms.</span></span>

- <span data-ttu-id="14c48-116">Uwidaczniaj <xref:System.IAsyncResult> wzorzec tylko wtedy, gdy jest to niezbędne do spełnienia wymagań.</span><span class="sxs-lookup"><span data-stu-id="14c48-116">Only expose the <xref:System.IAsyncResult> pattern when it is necessary for meeting your requirements.</span></span> <span data-ttu-id="14c48-117">Na przykład zgodność z istniejącym interfejsem API może wymagać uwidocznienia <xref:System.IAsyncResult> wzorca.</span><span class="sxs-lookup"><span data-stu-id="14c48-117">For example, compatibility with an existing API may require you to expose the <xref:System.IAsyncResult> pattern.</span></span>

- <span data-ttu-id="14c48-118">Nie ujawniaj <xref:System.IAsyncResult> wzorca bez również ujawniania wzorca opartego na zdarzeniach.</span><span class="sxs-lookup"><span data-stu-id="14c48-118">Do not expose the <xref:System.IAsyncResult> pattern without also exposing the event-based pattern.</span></span>

- <span data-ttu-id="14c48-121">Kompiluj implementację wzorca opartego na zdarzeniach w <xref:System.IAsyncResult> implementacji wzorca.</span><span class="sxs-lookup"><span data-stu-id="14c48-121">Build your event-based pattern implementation on your <xref:System.IAsyncResult> pattern implementation.</span></span>

- <span data-ttu-id="14c48-122">Należy unikać ujawniania wzorca opartego na zdarzeniach i <xref:System.IAsyncResult> wzorca w tej samej klasie.</span><span class="sxs-lookup"><span data-stu-id="14c48-122">Avoid exposing both the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class.</span></span> <span data-ttu-id="14c48-123">Uwidocznić wzorzec oparty na zdarzeniach dla klas "wyższego poziomu" i <xref:System.IAsyncResult> wzorzec dla klas "niższego poziomu".</span><span class="sxs-lookup"><span data-stu-id="14c48-123">Expose the event-based pattern on "higher level" classes and the <xref:System.IAsyncResult> pattern on "lower level" classes.</span></span> <span data-ttu-id="14c48-124">Na przykład Porównaj wzorzec oparty na zdarzeniach w <xref:System.Net.WebClient> składniku ze <xref:System.IAsyncResult> wzorcem w <xref:System.Web.HttpRequest> klasie.</span><span class="sxs-lookup"><span data-stu-id="14c48-124">For example, compare the event-based pattern on the <xref:System.Net.WebClient> component with the <xref:System.IAsyncResult> pattern on the <xref:System.Web.HttpRequest> class.</span></span>

  - <span data-ttu-id="14c48-125">Uwidocznić wzorzec oparty na zdarzeniach i <xref:System.IAsyncResult> wzorzec dla tej samej klasy, gdy wymaga tego zgodności.</span><span class="sxs-lookup"><span data-stu-id="14c48-125">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class when compatibility requires it.</span></span> <span data-ttu-id="14c48-126">Jeśli na przykład już wydano interfejs API, który używa <xref:System.IAsyncResult> wzorca, należy zachować <xref:System.IAsyncResult> wzorzec zgodności z poprzednimi wersjami.</span><span class="sxs-lookup"><span data-stu-id="14c48-126">For example, if you have already released an API that uses the <xref:System.IAsyncResult> pattern, you would need to retain the <xref:System.IAsyncResult> pattern for backward compatibility.</span></span>

  - <span data-ttu-id="14c48-127">Uwidocznij wzorzec oparty na zdarzeniach i <xref:System.IAsyncResult> wzorzec dla tej samej klasy, jeśli złożoność modelu obiektów powstała w efekcie przewagi korzyści wynikające z oddzielenia implementacji.</span><span class="sxs-lookup"><span data-stu-id="14c48-127">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class if the resulting object model complexity outweighs the benefit of separating the implementations.</span></span> <span data-ttu-id="14c48-128">Lepiej jest uwidocznić oba wzorce na pojedynczej klasie, aby uniknąć ujawniania wzorca opartego na zdarzeniach.</span><span class="sxs-lookup"><span data-stu-id="14c48-128">It is better to expose both patterns on a single class than to avoid exposing the event-based pattern.</span></span>

  - <span data-ttu-id="14c48-129">Jeśli musisz uwidocznić wzorzec oparty na zdarzeniach i <xref:System.IAsyncResult> wzorzec dla pojedynczej klasy, użyj opcji <xref:System.ComponentModel.EditorBrowsableAttribute> Ustaw, aby <xref:System.ComponentModel.EditorBrowsableState.Advanced> oznaczyć <xref:System.IAsyncResult> implementację wzorca jako funkcję zaawansowaną.</span><span class="sxs-lookup"><span data-stu-id="14c48-129">If you must expose both the event-based pattern and <xref:System.IAsyncResult> pattern on a single class, use <xref:System.ComponentModel.EditorBrowsableAttribute> set to <xref:System.ComponentModel.EditorBrowsableState.Advanced> to mark the <xref:System.IAsyncResult> pattern implementation as an advanced feature.</span></span> <span data-ttu-id="14c48-130">Wskazuje to na środowiska projektowania, takie jak Visual Studio IntelliSense, nie wyświetla <xref:System.IAsyncResult> właściwości i metod.</span><span class="sxs-lookup"><span data-stu-id="14c48-130">This indicates to design environments, such as Visual Studio IntelliSense, not to display the <xref:System.IAsyncResult> properties and methods.</span></span> <span data-ttu-id="14c48-131">Te właściwości i metody są nadal w pełni użyteczne, ale deweloper pracujący za pomocą technologii IntelliSense ma wyraźniejszy widok interfejsu API.</span><span class="sxs-lookup"><span data-stu-id="14c48-131">These properties and methods are still fully usable, but the developer working through IntelliSense has a clearer view of the API.</span></span>

## <a name="criteria-for-exposing-the-iasyncresult-pattern-in-addition-to-the-event-based-pattern"></a><span data-ttu-id="14c48-132">Kryteria udostępniania wzorca IAsyncResult oprócz wzorca opartego na zdarzeniach</span><span class="sxs-lookup"><span data-stu-id="14c48-132">Criteria for Exposing the IAsyncResult Pattern in Addition to the Event-based Pattern</span></span>

<span data-ttu-id="14c48-133">Chociaż wzorzec asynchroniczny oparty na zdarzeniach ma wiele korzyści w ramach wyżej wymienionych scenariuszy, ma pewne wady, których należy wiedzieć, jeśli wydajność jest najważniejszym wymaganiem.</span><span class="sxs-lookup"><span data-stu-id="14c48-133">While the Event-based Asynchronous Pattern has many benefits under the previously mentioned scenarios, it does have some drawbacks, which you should be aware of if performance is your most important requirement.</span></span>

<span data-ttu-id="14c48-134">Istnieją trzy scenariusze, w których wzorzec oparty na zdarzeniach nie dotyczy i <xref:System.IAsyncResult> wzorca:</span><span class="sxs-lookup"><span data-stu-id="14c48-134">There are three scenarios that the event-based pattern does not address as well as the <xref:System.IAsyncResult> pattern:</span></span>

- <span data-ttu-id="14c48-135">Blokowanie oczekiwania na jeden<xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="14c48-135">Blocking wait on one <xref:System.IAsyncResult></span></span>

- <span data-ttu-id="14c48-136">Blokowanie oczekiwania na wiele <xref:System.IAsyncResult> obiektów</span><span class="sxs-lookup"><span data-stu-id="14c48-136">Blocking wait on many <xref:System.IAsyncResult> objects</span></span>

- <span data-ttu-id="14c48-137">Sondowanie w celu ukończenia<xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="14c48-137">Polling for completion on the <xref:System.IAsyncResult></span></span>

<span data-ttu-id="14c48-138">Te scenariusze można rozwiązać przy użyciu wzorca zdarzeń, ale robi to bardziej skomplikowany niż używanie <xref:System.IAsyncResult> wzorca.</span><span class="sxs-lookup"><span data-stu-id="14c48-138">You can address these scenarios by using the event-based pattern, but doing so is more cumbersome than using the <xref:System.IAsyncResult> pattern.</span></span>

<span data-ttu-id="14c48-139">Deweloperzy często używają <xref:System.IAsyncResult> wzorca dla usług, które zwykle mają bardzo wysokie wymagania dotyczące wydajności.</span><span class="sxs-lookup"><span data-stu-id="14c48-139">Developers often use the <xref:System.IAsyncResult> pattern for services that typically have very high performance requirements.</span></span> <span data-ttu-id="14c48-140">Na przykład scenariusz sondowania dla ukończenia jest techniką serwera o wysokiej wydajności.</span><span class="sxs-lookup"><span data-stu-id="14c48-140">For example, the polling for completion scenario is a high-performance server technique.</span></span>

<span data-ttu-id="14c48-141">Ponadto wzorzec oparty na zdarzeniach jest mniej wydajny niż <xref:System.IAsyncResult> wzorzec, ponieważ tworzy więcej obiektów, szczególnie <xref:System.EventArgs> i ponieważ synchronizuje się między wątkami.</span><span class="sxs-lookup"><span data-stu-id="14c48-141">Additionally, the event-based pattern is less efficient than the <xref:System.IAsyncResult> pattern because it creates more objects, especially <xref:System.EventArgs>, and because it synchronizes across threads.</span></span>

<span data-ttu-id="14c48-142">Na poniższej liście przedstawiono zalecenia, które należy wykonać, jeśli zdecydujesz się użyć <xref:System.IAsyncResult> wzorca:</span><span class="sxs-lookup"><span data-stu-id="14c48-142">The following list shows some recommendations to follow if you decide to use the <xref:System.IAsyncResult> pattern:</span></span>

- <span data-ttu-id="14c48-143">Uwidocznić <xref:System.IAsyncResult> wzorzec tylko wtedy, gdy użytkownik wymaga obsługi <xref:System.Threading.WaitHandle> <xref:System.IAsyncResult> obiektów lub.</span><span class="sxs-lookup"><span data-stu-id="14c48-143">Only expose the <xref:System.IAsyncResult> pattern when you specifically require support for <xref:System.Threading.WaitHandle> or <xref:System.IAsyncResult> objects.</span></span>

- <span data-ttu-id="14c48-144">Uwidaczniaj <xref:System.IAsyncResult> wzorzec tylko wtedy, gdy masz istniejący interfejs API, który używa <xref:System.IAsyncResult> wzorca.</span><span class="sxs-lookup"><span data-stu-id="14c48-144">Only expose the <xref:System.IAsyncResult> pattern when you have an existing API that uses the <xref:System.IAsyncResult> pattern.</span></span>

- <span data-ttu-id="14c48-145">Jeśli masz istniejący interfejs API oparty na <xref:System.IAsyncResult> wzorcu, rozważ także udostępnienie wzorca opartego na zdarzeniach w następnej wersji.</span><span class="sxs-lookup"><span data-stu-id="14c48-145">If you have an existing API based on the <xref:System.IAsyncResult> pattern, consider also exposing the event-based pattern in your next release.</span></span>

- <span data-ttu-id="14c48-146">Uwidaczniaj tylko <xref:System.IAsyncResult> wzorzec, jeśli masz wymagania o wysokiej wydajności, które nie mogą być spełnione przez wzorzec oparty na zdarzeniach, ale mogą być spełnione przez wzorzec <xref:System.IAsyncResult> .</span><span class="sxs-lookup"><span data-stu-id="14c48-146">Only expose <xref:System.IAsyncResult> pattern if you have high performance requirements which you have verified cannot be met by the event-based pattern but can be met by the <xref:System.IAsyncResult> pattern.</span></span>
