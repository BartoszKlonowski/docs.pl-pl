# <a name="interop-with-other-asynchronous-patterns-and-types"></a>Współdziałanie z innymi wzorcami asynchronicznymi i typami

Krótka historia asynchronicznych wzorców w programie .NET:

- W .NET Framework 1,0 wprowadzono <xref:System.IAsyncResult> wzorzec, w przeciwnym razie znany jako [asynchroniczny model programowania (APM)](asynchronous-programming-model-apm.md)lub `Begin/End` wzorzec.
- .NET Framework 2,0 dodano [wzorzec asynchroniczny oparty na zdarzeniach (EAP)](event-based-asynchronous-pattern-eap.md).
- .NET Framework 4 wprowadził [wzorzec asynchroniczny oparty na zadaniach (TAP)](task-based-asynchronous-pattern-tap.md), który zastępuje zarówno APM, jak i EAP, i umożliwia łatwe tworzenie procedur migracji z wcześniejszych wzorców.
