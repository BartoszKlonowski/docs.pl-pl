# <a name="strong-named-assemblies"></a><span data-ttu-id="19a33-102">Zestawy o silnych nazwach</span><span class="sxs-lookup"><span data-stu-id="19a33-102">Strong-named assemblies</span></span>

<span data-ttu-id="19a33-103">Silne nazewnictwo zestawu tworzy unikatową tożsamość dla zestawu i może zapobiec konfliktom zestawów.</span><span class="sxs-lookup"><span data-stu-id="19a33-103">Strong-naming an assembly creates a unique identity for the assembly, and can prevent assembly conflicts.</span></span>

## <a name="what-makes-a-strong-named-assembly"></a><span data-ttu-id="19a33-104">Co sprawia, że zestaw o silnej nazwie?</span><span class="sxs-lookup"><span data-stu-id="19a33-104">What makes a strong-named assembly?</span></span>

<span data-ttu-id="19a33-105">Zestaw o silnej nazwie jest generowany przy użyciu klucza prywatnego, który odpowiada kluczpubliczny dystrybuowany z zestawu i samego zestawu.</span><span class="sxs-lookup"><span data-stu-id="19a33-105">A strong named assembly is generated by using the private key that corresponds to the public key distributed with the assembly, and the assembly itself.</span></span> <span data-ttu-id="19a33-106">Zestaw zawiera manifest zestawu, który zawiera nazwy i haszysze wszystkich plików, które tworzą zestaw.</span><span class="sxs-lookup"><span data-stu-id="19a33-106">The assembly includes the assembly manifest, which contains the names and hashes of all the files that make up the assembly.</span></span> <span data-ttu-id="19a33-107">Zestawy, które mają taką samą silną nazwę powinny być identyczne.</span><span class="sxs-lookup"><span data-stu-id="19a33-107">Assemblies that have the same strong name should be identical.</span></span>

<span data-ttu-id="19a33-108">Zestawy o silnej nazwie można używać za pomocą programu Visual Studio lub narzędzia wiersza polecenia.</span><span class="sxs-lookup"><span data-stu-id="19a33-108">You can strong-name assemblies by using Visual Studio or a command-line tool.</span></span> <span data-ttu-id="19a33-109">Aby uzyskać więcej informacji, zobacz [Jak: Podpisywać zestaw o silnej nazwie](sign-strong-name.md) lub [Sn.exe (narzędzie Silna nazwa)](../../framework/tools/sn-exe-strong-name-tool.md).</span><span class="sxs-lookup"><span data-stu-id="19a33-109">For more information, see [How to: Sign an assembly with a strong name](sign-strong-name.md) or [Sn.exe (Strong Name tool)](../../framework/tools/sn-exe-strong-name-tool.md).</span></span>

<span data-ttu-id="19a33-110">Po utworzeniu zestawu o silnej nazwie zawiera prostą nazwę tekstową zestawu, numer wersji, opcjonalne informacje o kulturze, podpis cyfrowy i klucz publiczny, który odpowiada kluczowi prywatnemu używanemu do podpisywania.</span><span class="sxs-lookup"><span data-stu-id="19a33-110">When a strong-named assembly is created, it contains the simple text name of the assembly, the version number, optional culture information, a digital signature, and the public key that corresponds to the private key used for signing.</span></span>

> <span data-ttu-id="19a33-111">Nie należy polegać na silne nazwy dla bezpieczeństwa.</span><span class="sxs-lookup"><span data-stu-id="19a33-111">Do not rely on strong names for security.</span></span> <span data-ttu-id="19a33-112">Zapewniają one tylko unikatową tożsamość.</span><span class="sxs-lookup"><span data-stu-id="19a33-112">They provide a unique identity only.</span></span>

## <a name="why-strong-name-your-assemblies"></a><span data-ttu-id="19a33-113">Dlaczego silne nazwy zespołów?</span><span class="sxs-lookup"><span data-stu-id="19a33-113">Why strong-name your assemblies?</span></span>

<span data-ttu-id="19a33-114">Podczas odwoływania się do zestawu o silnej nazwie, można oczekiwać pewnych korzyści, takich jak przechowywanie wersji i nazewnictwa ochrony.</span><span class="sxs-lookup"><span data-stu-id="19a33-114">When you reference a strong-named assembly, you can expect certain benefits, such as versioning and naming protection.</span></span> <span data-ttu-id="19a33-115">W platformie .NET Framework zestawy o silnych nazwach można zainstalować w globalnej pamięci podręcznej zestawów, co jest wymagane do włączenia niektórych scenariuszy.</span><span class="sxs-lookup"><span data-stu-id="19a33-115">In the .NET Framework, strong-named assemblies can be installed in the global assembly cache, which is required to enable some scenarios.</span></span>

<span data-ttu-id="19a33-116">Zestawy o silnych nazwach są przydatne w następujących scenariuszach:</span><span class="sxs-lookup"><span data-stu-id="19a33-116">Strong-named assemblies are useful in the following scenarios:</span></span>

- <span data-ttu-id="19a33-117">Chcesz włączyć zestawy, do których mają odwoływać się zestawy o `friend` silnej nazwie lub chcesz udzielić dostępu do zestawów z innych zestawów o silnych nazwach.</span><span class="sxs-lookup"><span data-stu-id="19a33-117">You want to enable your assemblies to be referenced by strong-named assemblies, or you want to give `friend` access to your assemblies from other strong-named assemblies.</span></span>

- <span data-ttu-id="19a33-118">Aplikacja wymaga dostępu do różnych wersji tego samego zestawu.</span><span class="sxs-lookup"><span data-stu-id="19a33-118">An app needs access to different versions of the same assembly.</span></span> <span data-ttu-id="19a33-119">Oznacza to, że potrzebujesz różnych wersji zestawu, aby załadować obok siebie w tej samej domenie aplikacji bez konfliktów.</span><span class="sxs-lookup"><span data-stu-id="19a33-119">This means  you need different versions of an assembly to load side by side in the same app domain without conflict.</span></span> <span data-ttu-id="19a33-120">Na przykład jeśli istnieją różne rozszerzenia interfejsu API w zestawach, które mają tę samą prostą nazwę, silne nazewnictwo zapewnia unikatową tożsamość dla każdej wersji zestawu.</span><span class="sxs-lookup"><span data-stu-id="19a33-120">For example, if different extensions of an API exist in assemblies that have the same simple name, strong-naming provides a unique identity for each version of the assembly.</span></span>

- <span data-ttu-id="19a33-121">Nie chcesz negatywnie wpływać na wydajność aplikacji przy użyciu zestawu, więc chcesz, aby zestaw był neutralny dla domeny.</span><span class="sxs-lookup"><span data-stu-id="19a33-121">You do not want to negatively affect performance of apps using your assembly, so you want the assembly to be domain neutral.</span></span> <span data-ttu-id="19a33-122">Wymaga to silnego nazewnictwa, ponieważ zestaw neutralny dla domeny musi być zainstalowany w globalnej pamięci podręcznej zestawów.</span><span class="sxs-lookup"><span data-stu-id="19a33-122">This requires strong-naming because a domain-neutral assembly must be installed in the global assembly cache.</span></span>

- <span data-ttu-id="19a33-123">Chcesz scentralizować obsługę aplikacji, stosując zasady wydawcy, co oznacza, że zestaw musi być zainstalowany w globalnej pamięci podręcznej zestawów.</span><span class="sxs-lookup"><span data-stu-id="19a33-123">You want to centralize servicing for your app by applying publisher policy, which means the assembly must be installed in the global assembly cache.</span></span>

<span data-ttu-id="19a33-124">Jeśli jesteś deweloperem typu open source i chcesz korzyści tożsamości zestawu o silnej nazwie, należy rozważyć zaewidencjonowanie w kluczprywatny skojarzony z zestawem do systemu kontroli źródła.</span><span class="sxs-lookup"><span data-stu-id="19a33-124">If you are an open-source developer and you want the identity benefits of a strong-named assembly, consider checking in the private key associated with an assembly to your source control system.</span></span>
