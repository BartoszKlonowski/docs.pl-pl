# <a name="assembly-names"></a><span data-ttu-id="416f0-102">Nazwy zestawów</span><span class="sxs-lookup"><span data-stu-id="416f0-102">Assembly names</span></span>
<span data-ttu-id="416f0-103">Nazwa zestawu jest przechowywana w metadanych i ma znaczący wpływ na zakres zestawu i wykorzystania przez aplikację.</span><span class="sxs-lookup"><span data-stu-id="416f0-103">An assembly's name is stored in metadata and has a significant impact on the assembly's scope and use by an application.</span></span> <span data-ttu-id="416f0-104">Zestaw o silnej nazwie ma w pełni kwalifikowaną nazwę, która zawiera nazwę zestawu, kulturę, klucz publiczny i numer wersji.</span><span class="sxs-lookup"><span data-stu-id="416f0-104">A strong-named assembly has a fully qualified name that includes the assembly's name, culture, public key, and version number.</span></span> <span data-ttu-id="416f0-105">Jest to często określane jako nazwa wyświetlana, a dla załadowanych zestawów można uzyskać za pomocą <xref:System.Reflection.Assembly.FullName%2A> właściwości.</span><span class="sxs-lookup"><span data-stu-id="416f0-105">This is frequently referred to as the display name, and for loaded assemblies can be obtained by using the <xref:System.Reflection.Assembly.FullName%2A> property.</span></span>

 <span data-ttu-id="416f0-106">Czas wykonywania używa tych informacji, aby zlokalizować zestaw i odróżnić go od innych zestawów o tej samej nazwie.</span><span class="sxs-lookup"><span data-stu-id="416f0-106">The runtime uses this information to locate the assembly and differentiate it from other assemblies with the same name.</span></span> <span data-ttu-id="416f0-107">Na przykład zestaw o silnej nazwie może `myTypes` mieć następującą w pełni kwalifikowaną nazwę:</span><span class="sxs-lookup"><span data-stu-id="416f0-107">For example, a strong-named assembly called `myTypes` could have the following fully qualified name:</span></span>

> [!NOTE]
> <span data-ttu-id="416f0-108">Architektura procesora jest dodawana do tożsamości zestawu w wersji .NET Framework 2.0, aby umożliwić wersje zestawów specyficzne dla procesora.</span><span class="sxs-lookup"><span data-stu-id="416f0-108">Processor architecture is added to the assembly identity in the .NET Framework version 2.0, to allow processor-specific versions of assemblies.</span></span> <span data-ttu-id="416f0-109">Można utworzyć wersje zestawu, którego tożsamość różni się tylko architekturą procesora, na przykład wersjami 32-bitowymi i 64-bitowymi specyficznymi dla procesora.</span><span class="sxs-lookup"><span data-stu-id="416f0-109">You can create versions of an assembly whose identity differs only by processor architecture, for example 32-bit and 64-bit processor-specific versions.</span></span> <span data-ttu-id="416f0-110">Architektura procesora nie jest wymagana dla silnych nazw.</span><span class="sxs-lookup"><span data-stu-id="416f0-110">Processor architecture is not required for strong names.</span></span> <span data-ttu-id="416f0-111">Aby uzyskać więcej informacji, zobacz <xref:System.Reflection.AssemblyName.ProcessorArchitecture%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="416f0-111">For more information, see <xref:System.Reflection.AssemblyName.ProcessorArchitecture%2A?displayProperty=nameWithType>.</span></span>

 <span data-ttu-id="416f0-112">W tym przykładzie w pełni kwalifikowana `myTypes` nazwa wskazuje, że zestaw ma silną nazwę z tokenem klucza publicznego, ma wartość kultury dla języka angielskiego amerykańskiego i ma numer wersji 1.0.1234.0.</span><span class="sxs-lookup"><span data-stu-id="416f0-112">In this example, the fully qualified name indicates that the `myTypes` assembly has a strong name with a public key token, has the culture value for US English, and has a version number of 1.0.1234.0.</span></span> <span data-ttu-id="416f0-113">Jego architektura procesora jest "msil", co oznacza, że będzie just-in-time (JIT) skompilowany do kodu 32-bitowego lub 64-bitowego w zależności od systemu operacyjnego i procesora.</span><span class="sxs-lookup"><span data-stu-id="416f0-113">Its processor architecture is "msil", which means that it will be just-in-time (JIT)-compiled to 32-bit code or 64-bit code depending on the operating system and processor.</span></span>

 <span data-ttu-id="416f0-114">Kod, który żąda typów w zestawie musi używać w pełni kwalifikowanej nazwy zestawu.</span><span class="sxs-lookup"><span data-stu-id="416f0-114">Code that requests types in an assembly must use a fully qualified assembly name.</span></span> <span data-ttu-id="416f0-115">Jest to nazywane w pełni kwalifikowane wiązanie.</span><span class="sxs-lookup"><span data-stu-id="416f0-115">This is called fully qualified binding.</span></span> <span data-ttu-id="416f0-116">Częściowe powiązanie, które określa tylko nazwę zestawu, nie jest dozwolone podczas odwoływania się do zestawów w .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="416f0-116">Partial binding, which specifies only an assembly name, is not permitted when referencing assemblies in the .NET Framework.</span></span>

 <span data-ttu-id="416f0-117">Wszystkie odwołania do zestawu, które tworzą .NET Framework musi również zawierać w pełni kwalifikowaną nazwę zestawu.</span><span class="sxs-lookup"><span data-stu-id="416f0-117">All assembly references to assemblies that make up the .NET Framework must also contain the fully qualified name of the assembly.</span></span> <span data-ttu-id="416f0-118">Na przykład odwołanie do zestawu System.Data .NET Framework dla wersji 1.0 będzie zawierać:</span><span class="sxs-lookup"><span data-stu-id="416f0-118">For example, a reference to the System.Data .NET Framework assembly for version 1.0 would include:</span></span>

```
System.data, version=1.0.3300.0, Culture=neutral, PublicKeyToken=b77a5c561934e089
```

 <span data-ttu-id="416f0-119">Należy zauważyć, że wersja odpowiada numer wersji wszystkich zestawów .NET Framework, które dostarczane z .NET Framework w wersji 1.0.</span><span class="sxs-lookup"><span data-stu-id="416f0-119">Note that the version corresponds to the version number of all .NET Framework assemblies that shipped with .NET Framework version 1.0.</span></span> <span data-ttu-id="416f0-120">W przypadku zestawów .NET Framework wartość kultury jest zawsze neutralna, a klucz publiczny jest taki sam, jak pokazano w powyższym przykładzie.</span><span class="sxs-lookup"><span data-stu-id="416f0-120">For .NET Framework assemblies, the culture value is always neutral, and the public key is the same as shown in the above example.</span></span>

 <span data-ttu-id="416f0-121">Na przykład, aby dodać odwołanie do zestawu w pliku konfiguracji w celu skonfigurowania odbiornika śledzenia, należy dołączyć w pełni kwalifikowaną nazwę zestawu systemu .NET Framework:</span><span class="sxs-lookup"><span data-stu-id="416f0-121">For example, to add an assembly reference in a configuration file to set up a trace listener, you would include the fully qualified name of the system .NET Framework assembly:</span></span>

```xml
<add name="myListener" type="System.Diagnostics.TextWriterTraceListener, System, Version=1.0.3300.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" initializeData="c:\myListener.log" />
```

> [!NOTE]
> <span data-ttu-id="416f0-122">Środowiska uruchomieniowego traktuje nazwy zestawów jako wielkość liter podczas wiązania z zestawem, ale zachowuje niezależnie od przypadku jest używany w nazwie zestawu.</span><span class="sxs-lookup"><span data-stu-id="416f0-122">The runtime treats assembly names as case-insensitive when binding to an assembly, but preserves whatever case is used in an assembly name.</span></span> <span data-ttu-id="416f0-123">Kilka narzędzi w zestawie Windows SDK obsługuje nazwy zestawów jako rozróżniające wielkość liter.</span><span class="sxs-lookup"><span data-stu-id="416f0-123">Several tools in the Windows SDK handle assembly names as case-sensitive.</span></span> <span data-ttu-id="416f0-124">Aby uzyskać najlepsze wyniki, zarządzaj nazwami zestawów tak, jakby były uwzględniane wielkości liter.</span><span class="sxs-lookup"><span data-stu-id="416f0-124">For best results, manage assembly names as though they were case-sensitive.</span></span>

 <span data-ttu-id="416f0-132">Aby uniknąć tego problemu, upewnij się, że zestawy, które tworzą aplikację nie mają tej samej nazwy zestawu lub zestawy o tej samej nazwie w różnych katalogach.</span><span class="sxs-lookup"><span data-stu-id="416f0-132">To avoid this problem, make sure the assemblies that make up your application do not have the same assembly name or place assemblies with the same name in different directories.</span></span>
