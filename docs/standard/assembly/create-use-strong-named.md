## <a name="strong-name-scenario"></a><span data-ttu-id="cba06-112">Scenariusz silnej nazwy</span><span class="sxs-lookup"><span data-stu-id="cba06-112">Strong name scenario</span></span>

<span data-ttu-id="cba06-113">Poniższy scenariusz przedstawia proces podpisywania zestawu o silnej nazwie, a następnie odwoływania się do niego o tej nazwie.</span><span class="sxs-lookup"><span data-stu-id="cba06-113">The following scenario outlines the process of signing an assembly with a strong name and later referencing it by that name.</span></span>

1. <span data-ttu-id="cba06-114">Zestaw A jest tworzony z silną nazwą przy użyciu jednej z następujących metod:</span><span class="sxs-lookup"><span data-stu-id="cba06-114">Assembly A is created with a strong name using one of the following methods:</span></span>

    - <span data-ttu-id="cba06-115">Przy użyciu środowiska programistycznego, który obsługuje tworzenie silnych nazw, takich jak Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="cba06-115">Using a development environment that supports creating strong names, such as Visual Studio.</span></span>

    - <span data-ttu-id="cba06-116">Tworzenie pary kluczy kryptograficznych przy użyciu [narzędzia Silna nazwa (Sn.exe)](../../framework/tools/sn-exe-strong-name-tool.md) i przypisywanie tej pary kluczy do zestawu przy użyciu kompilatora wiersza polecenia lub [konsolidatora zestawu (Al.exe)](../../framework/tools/al-exe-assembly-linker.md).</span><span class="sxs-lookup"><span data-stu-id="cba06-116">Creating a cryptographic key pair using the [Strong Name tool (Sn.exe)](../../framework/tools/sn-exe-strong-name-tool.md) and assigning that key pair to the assembly using either a command-line compiler or the [Assembly Linker (Al.exe)](../../framework/tools/al-exe-assembly-linker.md).</span></span> <span data-ttu-id="cba06-117">Zestaw Windows SDK zapewnia zarówno program Sn.exe, jak i program Al.exe.</span><span class="sxs-lookup"><span data-stu-id="cba06-117">The Windows SDK provides both Sn.exe and Al.exe.</span></span>

2. <span data-ttu-id="cba06-118">Środowisko programistyczne lub narzędzie podpisuje skrót pliku zawierający manifest zestawu kluczem prywatnym dewelopera.</span><span class="sxs-lookup"><span data-stu-id="cba06-118">The development environment or tool signs the hash of the file containing the assembly's manifest with the developer's private key.</span></span> <span data-ttu-id="cba06-119">Ten podpis cyfrowy jest przechowywany w przenośnym pliku wykonywalnym (PE), który zawiera manifest zestawu A.</span><span class="sxs-lookup"><span data-stu-id="cba06-119">This digital signature is stored in the portable executable (PE) file that contains Assembly A's manifest.</span></span>

3. <span data-ttu-id="cba06-120">Zespół B jest konsumentem zespołu A. Sekcja referencyjna manifestu zestawu B zawiera token, który reprezentuje klucz publiczny zestawu A.</span><span class="sxs-lookup"><span data-stu-id="cba06-120">Assembly B is a consumer of Assembly A. The reference section of Assembly B's manifest includes a token that represents Assembly A's public key.</span></span> <span data-ttu-id="cba06-121">Token jest częścią pełnego klucza publicznego i jest używany, a nie sam klucz, aby zaoszczędzić miejsce.</span><span class="sxs-lookup"><span data-stu-id="cba06-121">A token is a portion of the full public key and is used rather than the key itself to save space.</span></span>

4. <span data-ttu-id="cba06-122">Czas wykonywania języka wspólnego weryfikuje podpis silnej nazwy, gdy zestaw jest umieszczony w globalnej pamięci podręcznej zestawów.</span><span class="sxs-lookup"><span data-stu-id="cba06-122">The common language runtime verifies the strong name signature when the assembly is placed in the global assembly cache.</span></span> <span data-ttu-id="cba06-123">Podczas wiązania przez silną nazwę w czasie wykonywania, czas wykonywania języka wspólnego porównuje klucz przechowywany w manifeście zestawu B z kluczem używanym do generowania silnej nazwy dla zestawu A. Jeśli sprawdza zabezpieczeń .NET Framework przekazać i powiązanie powiedzie się, zestaw B ma gwarancję, że zestaw a bity nie zostały naruszone i że te bity faktycznie pochodzą od twórców zestawu A.</span><span class="sxs-lookup"><span data-stu-id="cba06-123">When binding by strong name at run time, the common language runtime compares the key stored in Assembly B's manifest with the key used to generate the strong name for Assembly A. If the .NET Framework security checks pass and the bind succeeds, Assembly B has a guarantee that Assembly A's bits have not been tampered with and that these bits actually come from the developers of Assembly A.</span></span>

> [!NOTE]
> <span data-ttu-id="cba06-124">W tym scenariuszu nie rozwiązuje problemów z zaufaniem.</span><span class="sxs-lookup"><span data-stu-id="cba06-124">This scenario doesn't address trust issues.</span></span> <span data-ttu-id="cba06-125">Zestawy mogą przenosić pełne podpisy Microsoft Authenticode oprócz silnej nazwy.</span><span class="sxs-lookup"><span data-stu-id="cba06-125">Assemblies can carry full Microsoft Authenticode signatures in addition to a strong name.</span></span> <span data-ttu-id="cba06-126">Podpisy uwierzytelnione zawierają certyfikat ustanawiający zaufanie.</span><span class="sxs-lookup"><span data-stu-id="cba06-126">Authenticode signatures include a certificate that establishes trust.</span></span> <span data-ttu-id="cba06-127">Należy pamiętać, że silne nazwy nie wymagają kodu do podpisania w ten sposób.</span><span class="sxs-lookup"><span data-stu-id="cba06-127">It's important to note that strong names don't require code to be signed in this way.</span></span> <span data-ttu-id="cba06-128">Silne nazwy zapewniają tylko unikatową tożsamość.</span><span class="sxs-lookup"><span data-stu-id="cba06-128">Strong names only provide a unique identity.</span></span>

- <span data-ttu-id="cba06-136">Załadowany do <xref:System.AppDomain>w pełni zaufanego .</span><span class="sxs-lookup"><span data-stu-id="cba06-136">Loaded into a fully trusted <xref:System.AppDomain>.</span></span>

- <span data-ttu-id="cba06-137">Załadowany z lokalizacji <xref:System.AppDomainSetup.ApplicationBase%2A> pod <xref:System.AppDomain>własnością tego .</span><span class="sxs-lookup"><span data-stu-id="cba06-137">Loaded from a location under the <xref:System.AppDomainSetup.ApplicationBase%2A> property of that <xref:System.AppDomain>.</span></span>
