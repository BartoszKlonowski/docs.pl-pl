<span data-ttu-id="f09f6-103">Kreska ułamkowa odwrócona (\\) w wyrażeniu regularnym wskazuje jedną z następujących czynności:</span><span class="sxs-lookup"><span data-stu-id="f09f6-103">The backslash (\\) in a regular expression indicates one of the following:</span></span>

-   <span data-ttu-id="f09f6-106">Znak, które w przeciwnym razie będzie interpretowana jako konstrukcji języka niezmienionym znaczeniu, które powinny być rozumiane jako literału.</span><span class="sxs-lookup"><span data-stu-id="f09f6-106">A character that otherwise would be interpreted as an unescaped language construct should be interpreted literally.</span></span> <span data-ttu-id="f09f6-107">Na przykład nawiasu klamrowego (`{`) rozpoczyna się definicji kwantyfikator, ale ukośnik odwrotny, a następnie nawiasu klamrowego (`\{`) wskazuje, że nawiasem powinna być zgodna z aparatem wyrażeń regularnych.</span><span class="sxs-lookup"><span data-stu-id="f09f6-107">For example, a brace (`{`) begins the definition of a quantifier, but a backslash followed by a brace (`\{`) indicates that the regular expression engine should match the brace.</span></span> <span data-ttu-id="f09f6-108">Podobnie, pojedynczy ukośnik odwrotny oznacza początek konstrukcji języka zmieniony, ale dwa razy (`\\`) wskazuje, że aparat wyrażeń regularnych powinna być zgodna ukośniku odwrotnym.</span><span class="sxs-lookup"><span data-stu-id="f09f6-108">Similarly, a single backslash marks the beginning of an escaped language construct, but two backslashes (`\\`) indicate that the regular expression engine should match the backslash.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="f09f6-109">Znaki specjalne są rozpoznawane w wzorców wyrażeń regularnych, ale nie w wzorce zamiany.</span><span class="sxs-lookup"><span data-stu-id="f09f6-109">Character escapes are recognized in regular expression patterns but not in replacement patterns.</span></span>
