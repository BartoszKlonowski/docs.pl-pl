<span data-ttu-id="a4526-110">Jednak wygenerowane MSIL nie może być zwolniony.</span><span class="sxs-lookup"><span data-stu-id="a4526-110">However, generated MSIL cannot be unloaded.</span></span> <span data-ttu-id="a4526-111">Jedynym sposobem, aby zwolnić kodu jest wyładować domeny aplikacji (oznacza to, aby zwolnienie wszystkich aplikacji do kodu.).</span><span class="sxs-lookup"><span data-stu-id="a4526-111">The only way to unload code is to unload an entire application domain (that is, to unload all of your application's code.).</span></span> <span data-ttu-id="a4526-112">Efektywnie gdy wyrażenie regularne jest skompilowana przy użyciu <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> opcji, nigdy nie zwalnia zasoby używane przez wyrażenie skompilowane, nawet jeśli wyrażenie regularne został utworzony przez <xref:System.Text.RegularExpressions.Regex> obiekt, który jest elementem wydane wyrzucania elementów bezużytecznych do.</span><span class="sxs-lookup"><span data-stu-id="a4526-112">Effectively, once a regular expression is compiled with the <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option,   never releases the resources used by the compiled expression, even if the regular expression was created by a <xref:System.Text.RegularExpressions.Regex> object that is itself released to garbage collection.</span></span>

## <a name="the-regular-expressions-cache"></a><span data-ttu-id="a4526-119">Pamięć podręczna wyrażeń regularnych</span><span class="sxs-lookup"><span data-stu-id="a4526-119">The Regular Expressions Cache</span></span>  
 <span data-ttu-id="a4526-120">Aby zwiększyć wydajność, aparat wyrażeń regularnych przechowuje podręcznej całej aplikacji skompilowanych wyrażeń regularnych.</span><span class="sxs-lookup"><span data-stu-id="a4526-120">To improve performance, the regular expression engine maintains an application-wide cache of compiled regular expressions.</span></span> <span data-ttu-id="a4526-121">Pamięć podręczna przechowuje wzorce wyrażenie regularne, które są używane tylko w przypadku wywołania metody statycznej.</span><span class="sxs-lookup"><span data-stu-id="a4526-121">The cache stores regular expression patterns that are used only in static method calls.</span></span> <span data-ttu-id="a4526-122">(Wyrażenie regularne wzorce dostarczony do metody wystąpienia nie są buforowane.) Dzięki temu można uniknąć konieczności ponownej analizy wyrażenia do kodu bajtowego wysokiego poziomu każdym razem, gdy jest używany.</span><span class="sxs-lookup"><span data-stu-id="a4526-122">(Regular expression patterns supplied to instance methods are not cached.) This avoids the need to reparse an expression into high-level byte code each time it is used.</span></span>  
  
 <span data-ttu-id="a4526-123">Maksymalna liczba buforowanych wyrażeń regularnych jest określana przez wartość `static` (`Shared` w języku Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> właściwości.</span><span class="sxs-lookup"><span data-stu-id="a4526-123">The maximum number of cached regular expressions is determined by the value of the `static` (`Shared` in Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="a4526-124">Domyślnie aparat wyrażeń regularnych buforuje maksymalnie 15 skompilowane wyrażenia regularnego.</span><span class="sxs-lookup"><span data-stu-id="a4526-124">By default, the regular expression engine caches up to 15 compiled regular expressions.</span></span> <span data-ttu-id="a4526-125">Jeśli liczba skompilowanych wyrażeń regularnych przekracza rozmiar pamięci podręcznej, jest odrzucany najdawniej używaną wyrażenia regularnego i nowe wyrażenie regularne jest buforowana.</span><span class="sxs-lookup"><span data-stu-id="a4526-125">If the number of compiled regular expressions exceeds the cache size, the least recently used regular expression is discarded and the new regular expression is cached.</span></span>  
  
 <span data-ttu-id="a4526-126">Aplikację można korzystać z wstępnie skompilowanym wyrażeń regularnych w jednym z dwóch sposobów:</span><span class="sxs-lookup"><span data-stu-id="a4526-126">Your application can take advantage of precompiled regular expressions in one of the following two ways:</span></span>  
  
-   <span data-ttu-id="a4526-127">Za pomocą metody statycznej z <xref:System.Text.RegularExpressions.Regex> obiekt do definiowania wyrażenia regularnego.</span><span class="sxs-lookup"><span data-stu-id="a4526-127">By using a static method of the <xref:System.Text.RegularExpressions.Regex> object to define the regular expression.</span></span> <span data-ttu-id="a4526-128">Jeśli używasz wzorzec wyrażenia regularnego, który został już zdefiniowany w innym wywołaniu metody statycznej, aparat wyrażeń regularnych pobierze go z pamięci podręcznej.</span><span class="sxs-lookup"><span data-stu-id="a4526-128">If you are using a regular expression pattern that has already been defined in another static method call, the regular expression engine will retrieve it from the cache.</span></span> <span data-ttu-id="a4526-129">Jeśli nie, aparat będzie skompilować wyrażenia regularnego i dodaj go do pamięci podręcznej.</span><span class="sxs-lookup"><span data-stu-id="a4526-129">If not, the engine will compile the regular expression and add it to the cache.</span></span>  
  
-   <span data-ttu-id="a4526-130">Na podstawie istniejącej <xref:System.Text.RegularExpressions.Regex> obiekt tak długo, jak jego wzorzec wyrażenia regularnego jest wymagana.</span><span class="sxs-lookup"><span data-stu-id="a4526-130">By reusing an existing <xref:System.Text.RegularExpressions.Regex> object as long as its regular expression pattern is needed.</span></span>  
  
 <span data-ttu-id="a4526-131">Ze względu na obciążenie podczas tworzenia wystąpienia obiektu i kompilacji wyrażeń regularnych, tworzenie i szybko niszczenie wiele <xref:System.Text.RegularExpressions.Regex> obiektów jest bardzo kosztowna procesu.</span><span class="sxs-lookup"><span data-stu-id="a4526-131">Because of the overhead of object instantiation and regular expression compilation, creating and rapidly destroying numerous <xref:System.Text.RegularExpressions.Regex> objects is a very expensive process.</span></span> <span data-ttu-id="a4526-132">Dla aplikacji, które korzystają z wielu różnych wyrażeń regularnych, można zoptymalizować wydajność przy użyciu wywołań statyczne `Regex` metody i prawdopodobnie przez zwiększenie rozmiaru pamięci podręcznej wyrażenia regularnego.</span><span class="sxs-lookup"><span data-stu-id="a4526-132">For applications that use a large number of different regular expressions, you can optimize performance by using calls to static `Regex` methods and possibly by increasing the size of the regular expression cache.</span></span>
