## <a name="the-regular-expressions-cache"></a><span data-ttu-id="269c9-119">Pamięć podręczna wyrażeń regularnych</span><span class="sxs-lookup"><span data-stu-id="269c9-119">The Regular Expressions Cache</span></span>  
 <span data-ttu-id="269c9-120">Aby zwiększyć wydajność, aparat wyrażeń regularnych zachowuje pamięć podręczną całej aplikacji skompilowanych wyrażeń regularnych.</span><span class="sxs-lookup"><span data-stu-id="269c9-120">To improve performance, the regular expression engine maintains an application-wide cache of compiled regular expressions.</span></span> <span data-ttu-id="269c9-121">Pamięć podręczna przechowuje wzorce wyrażeń regularnych, które są używane tylko w wywołaniach metody statycznej.</span><span class="sxs-lookup"><span data-stu-id="269c9-121">The cache stores regular expression patterns that are used only in static method calls.</span></span> <span data-ttu-id="269c9-122">(Wzorce wyrażeń regularnych, przekazana do metody wystąpienia nie są buforowane.) Umożliwia to uniknięcie konieczności ponownej analizy wyrażenia do kodu bajtowego wysokiego poziomu każdorazowo, gdy jest używany.</span><span class="sxs-lookup"><span data-stu-id="269c9-122">(Regular expression patterns supplied to instance methods are not cached.) This avoids the need to reparse an expression into high-level byte code each time it is used.</span></span>  
  
 <span data-ttu-id="269c9-123">Maksymalna liczba buforowanych wyrażeń regularnych jest określana przez wartość `static` (`Shared` w języku Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> właściwości.</span><span class="sxs-lookup"><span data-stu-id="269c9-123">The maximum number of cached regular expressions is determined by the value of the `static` (`Shared` in Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="269c9-124">Domyślnie aparat wyrażenia regularnego buforuje maksymalnie 15 skompilowanych wyrażeń regularnych.</span><span class="sxs-lookup"><span data-stu-id="269c9-124">By default, the regular expression engine caches up to 15 compiled regular expressions.</span></span> <span data-ttu-id="269c9-125">Jeśli liczba skompilowanych wyrażeń regularnych przekracza rozmiar pamięci podręcznej, najdawniej używaną wyrażenia regularnego jest odrzucana i nowe wyrażenie regularne jest buforowany.</span><span class="sxs-lookup"><span data-stu-id="269c9-125">If the number of compiled regular expressions exceeds the cache size, the least recently used regular expression is discarded and the new regular expression is cached.</span></span>  
  
 <span data-ttu-id="269c9-126">Aplikację można korzystać ze wstępnie skompilowanych wyrażeń regularnych w jednym z dwóch sposobów:</span><span class="sxs-lookup"><span data-stu-id="269c9-126">Your application can take advantage of precompiled regular expressions in one of the following two ways:</span></span>  
  
- <span data-ttu-id="269c9-127">Za pomocą metody statycznej <xref:System.Text.RegularExpressions.Regex> obiekt do definiowania wyrażeń regularnych.</span><span class="sxs-lookup"><span data-stu-id="269c9-127">By using a static method of the <xref:System.Text.RegularExpressions.Regex> object to define the regular expression.</span></span> <span data-ttu-id="269c9-128">Jeśli korzystasz z wzorcem wyrażenia regularnego, który został już zdefiniowany w innym wywołaniu metody statyczne, aparat wyrażeń regularnych pobierze go z pamięci podręcznej.</span><span class="sxs-lookup"><span data-stu-id="269c9-128">If you are using a regular expression pattern that has already been defined in another static method call, the regular expression engine will retrieve it from the cache.</span></span> <span data-ttu-id="269c9-129">Jeśli nie, aparat będzie kompilowania wyrażeń regularnych i dodaj go do pamięci podręcznej.</span><span class="sxs-lookup"><span data-stu-id="269c9-129">If not, the engine will compile the regular expression and add it to the cache.</span></span>  
  
- <span data-ttu-id="269c9-130">Dzięki ponownemu wykorzystaniu istniejące <xref:System.Text.RegularExpressions.Regex> obiektu tak długo, jak jej wzorca wyrażenia regularnego jest wymagana.</span><span class="sxs-lookup"><span data-stu-id="269c9-130">By reusing an existing <xref:System.Text.RegularExpressions.Regex> object as long as its regular expression pattern is needed.</span></span>  
  
 <span data-ttu-id="269c9-131">Ze względu na obciążenie podczas tworzenia wystąpienia obiektu i kompilowania wyrażeń regularnych, tworzenie i szybko niszczenie liczne <xref:System.Text.RegularExpressions.Regex> obiektów jest procesem bardzo kosztowna.</span><span class="sxs-lookup"><span data-stu-id="269c9-131">Because of the overhead of object instantiation and regular expression compilation, creating and rapidly destroying numerous <xref:System.Text.RegularExpressions.Regex> objects is a very expensive process.</span></span> <span data-ttu-id="269c9-132">W przypadku aplikacji korzystających z dużej liczby różnych wyrażeń regularnych można zoptymalizować wydajność przy użyciu wywołania statycznej `Regex` metody i prawdopodobnie przez zwiększenie rozmiaru pamięci podręcznej wyrażenia regularnego.</span><span class="sxs-lookup"><span data-stu-id="269c9-132">For applications that use a large number of different regular expressions, you can optimize performance by using calls to static `Regex` methods and possibly by increasing the size of the regular expression cache.</span></span>
