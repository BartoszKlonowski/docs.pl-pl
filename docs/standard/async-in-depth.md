<span data-ttu-id="a4477-127">Drugi przykład dodaje użycie `async` i `await` słowa kluczowe do działania na zadanie.</span><span class="sxs-lookup"><span data-stu-id="a4477-127">The second example adds the use of the `async` and `await` keywords to operate on the task.</span></span>

```csharp

{
    // Execution is synchronous here
    var client = new HttpClient();

    // Execution of GetFirstCharactersCountAsync() is yielded to the caller here
    // GetStringAsync returns a Task<string>, which is *awaited*

    // Execution resumes when the client.GetStringAsync task completes,
    // becoming synchronous again.

        return page;
    }
    else
    {
        return page.Substring(0, count);
    }
}
```

## <a name="deeper-dive-into-task-and-taskt-for-a-cpu-bound-operation"></a><span data-ttu-id="a4477-176">Głębiej zanurz się\<w> zadania i zadania T dla operacji związanej z procesorem</span><span class="sxs-lookup"><span data-stu-id="a4477-176">Deeper Dive into Task and Task\<T> for a CPU-Bound Operation</span></span>

<span data-ttu-id="a4477-177">Kod związany `async` z procesorem CPU jest nieco `async` inny niż kod związany z we/wy.</span><span class="sxs-lookup"><span data-stu-id="a4477-177">CPU-bound `async` code is a bit different than I/O-bound `async` code.</span></span>  <span data-ttu-id="a4477-178">Ponieważ praca jest wykonywana na procesorze CPU, nie ma sposobu, aby obejść poświęcając wątek do obliczeń.</span><span class="sxs-lookup"><span data-stu-id="a4477-178">Because the work is done on the CPU, there's no way to get around dedicating a thread to the computation.</span></span>  <span data-ttu-id="a4477-179">Korzystanie z `async` `await` i zapewnia czysty sposób interakcji z wątku w tle i zachować obiekt wywołujący metody asynchronicznej elastyczne.</span><span class="sxs-lookup"><span data-stu-id="a4477-179">The use of `async` and `await` provides you with a clean way to interact with a background thread and keep the caller of the async method responsive.</span></span>  <span data-ttu-id="a4477-180">Należy pamiętać, że nie zapewnia to żadnej ochrony udostępnionych danych.</span><span class="sxs-lookup"><span data-stu-id="a4477-180">Note that this does not provide any protection for shared data.</span></span>  <span data-ttu-id="a4477-181">Jeśli używasz udostępnionych danych, nadal trzeba będzie zastosować odpowiednią strategię synchronizacji.</span><span class="sxs-lookup"><span data-stu-id="a4477-181">If you are using shared data, you will still need to apply an appropriate synchronization strategy.</span></span>
