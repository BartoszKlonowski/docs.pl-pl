## <a name="deeper-dive-into-task-and-taskt-for-a-cpu-bound-operation"></a><span data-ttu-id="ce4dd-177">Bardziej zgłębić temat do zadań i zadań<T> operacji procesora</span><span class="sxs-lookup"><span data-stu-id="ce4dd-177">Deeper Dive into Task and Task<T> for a CPU-Bound Operation</span></span>

<span data-ttu-id="ce4dd-183">W tym miejscu jest 10 000 stopy widok wywołania asynchronicznego procesora:</span><span class="sxs-lookup"><span data-stu-id="ce4dd-183">Here's a 10,000 foot view of a CPU-bound async call:</span></span>

```csharp

{
    // This queues up the work on the threadpool.
    var expensiveResultTask = Task.Run(() => DoExpensiveCalculation(data));
    
    // Note that at this point, you can do some other work concurrently,
    // as CalculateResult() is still executing!
    
    // Execution of CalculateResult is yielded here!
    var result = await expensiveResultTask;
    
    return result;
}
```

<span data-ttu-id="ce4dd-184">`CalculateResult()`wykonuje, na który została wywołana w wątku.</span><span class="sxs-lookup"><span data-stu-id="ce4dd-184">`CalculateResult()` executes on the thread it was called on.</span></span>  <span data-ttu-id="ce4dd-185">Gdy wywołuje `Task.Run`, jego kolejki kosztowna operacja procesora `DoExpensiveCalculation()`, w puli wątków i odbiera `Task<int>` obsługi.</span><span class="sxs-lookup"><span data-stu-id="ce4dd-185">When it calls `Task.Run`, it queues the expensive CPU-bound operation, `DoExpensiveCalculation()`, on the thread pool and receives a `Task<int>` handle.</span></span>  <span data-ttu-id="ce4dd-186">`DoExpensiveCalculation()`Po pewnym czasie działa jednocześnie na dalej dostępne wątku, prawdopodobnie w innym rdzeń procesora CPU.</span><span class="sxs-lookup"><span data-stu-id="ce4dd-186">`DoExpensiveCalculation()` is eventually run concurrently on the next available thread, likely on another CPU core.</span></span>  <span data-ttu-id="ce4dd-187">Istnieje możliwość pracy równoczesnych podczas `DoExpensiveCalculation()` jest zajęty w innym wątku, ponieważ wątek, który wywołuje `CalculateResult()` jest nadal wykonywane.</span><span class="sxs-lookup"><span data-stu-id="ce4dd-187">It's possible to do concurrent work while `DoExpensiveCalculation()` is busy on another thread, because the thread which called `CalculateResult()` is still executing.</span></span>
