## <a name="handling-exceptions"></a><span data-ttu-id="4b743-116">Obsługa wyjątków</span><span class="sxs-lookup"><span data-stu-id="4b743-116">Handling Exceptions</span></span>

 <span data-ttu-id="4b743-120"><xref:System.IObserver%601.OnError%2A> Metoda ma pełnić komunikat informacyjny do obserwatorów, znacznie takich jak <xref:System.IObserver%601.OnNext%2A?displayProperty=nameWithType> metody.</span><span class="sxs-lookup"><span data-stu-id="4b743-120">The <xref:System.IObserver%601.OnError%2A> method is intended as an informational message to observers, much like the <xref:System.IObserver%601.OnNext%2A?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="4b743-121">Jednak <xref:System.IObserver%601.OnNext%2A> metoda umożliwia obserwatora przy użyciu bieżących lub zaktualizowanych danych, natomiast <xref:System.IObserver%601.OnError%2A> metodę zaprojektowano w celu wskazania, że dostawca jest nie można podać prawidłowe dane.</span><span class="sxs-lookup"><span data-stu-id="4b743-121">However, the <xref:System.IObserver%601.OnNext%2A> method is designed to provide an observer with current or updated data, whereas the <xref:System.IObserver%601.OnError%2A> method is designed to indicate that the provider is unable to provide valid data.</span></span>  
  
 <span data-ttu-id="4b743-122">Dostawca należy stosować następujące najlepsze rozwiązania, podczas obsługi wyjątków i wywoływać metodę <xref:System.IObserver%601.OnError%2A> metody:</span><span class="sxs-lookup"><span data-stu-id="4b743-122">The provider should follow these best practices when handling exceptions and calling the <xref:System.IObserver%601.OnError%2A> method:</span></span>

- <span data-ttu-id="4b743-125">Dostawca powinien wywoływać <xref:System.IObserver%601.OnError%2A> metodą podczas obsługi wyjątku, który obniża możliwości na potrzeby udostępniania aktualizacji.</span><span class="sxs-lookup"><span data-stu-id="4b743-125">The provider should call the <xref:System.IObserver%601.OnError%2A> method when it handles an exception that compromises its ability to provide updates.</span></span> <span data-ttu-id="4b743-126">Informacji na temat wyjątków może być przekazywany do obserwatora.</span><span class="sxs-lookup"><span data-stu-id="4b743-126">Information on such exceptions can be passed to the observer.</span></span> <span data-ttu-id="4b743-127">W innych przypadkach nie ma potrzeby powiadomić obserwatorów wyjątek.</span><span class="sxs-lookup"><span data-stu-id="4b743-127">In other cases, there is no need to notify observers of an exception.</span></span>  
  
 <span data-ttu-id="4b743-128">Raz wywołania dostawcy <xref:System.IObserver%601.OnError%2A> lub <xref:System.IObserver%601.OnCompleted%2A?displayProperty=nameWithType> metody powinny istnieć żadne dalsze powiadomienia, a dostawca swoją subskrypcję możesz cofnąć jego obserwatorów.</span><span class="sxs-lookup"><span data-stu-id="4b743-128">Once the provider calls the <xref:System.IObserver%601.OnError%2A> or <xref:System.IObserver%601.OnCompleted%2A?displayProperty=nameWithType> method, there should be no further notifications, and the provider can unsubscribe its observers.</span></span> <span data-ttu-id="4b743-129">Jednak obserwatorzy również anulować subskrypcję samodzielnie w dowolnym momencie, w tym przed i po otrzymaniu <xref:System.IObserver%601.OnError%2A> lub <xref:System.IObserver%601.OnCompleted%2A?displayProperty=nameWithType> powiadomień.</span><span class="sxs-lookup"><span data-stu-id="4b743-129">However, the observers can also unsubscribe themselves at any time, including both before and after they receive an <xref:System.IObserver%601.OnError%2A> or <xref:System.IObserver%601.OnCompleted%2A?displayProperty=nameWithType> notification.</span></span> <span data-ttu-id="4b743-130">Wzorzec projektowy obserwatora nie określa, czy przez dostawcę lub obserwatora jest odpowiedzialny za anulowania subskrypcji;
