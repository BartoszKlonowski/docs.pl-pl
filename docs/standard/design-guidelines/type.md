# <a name="type-design-guidelines"></a><span data-ttu-id="8d9a8-102">Typy — zalecenia dotyczące projektowania</span><span class="sxs-lookup"><span data-stu-id="8d9a8-102">Type Design Guidelines</span></span>
<span data-ttu-id="8d9a8-103">Z perspektywy środowiska CLR istnieją tylko dwie kategorie typów — typy referencyjne i typy wartości — ale na potrzeby dyskusji na temat projektu struktury dzielą typy na więcej grup logicznych, z których każda ma własne specyficzne reguły projektowania.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-103">From the CLR perspective, there are only two categories of types—reference types and value types—but for the purpose of a discussion about framework design, we divide types into more logical groups, each with its own specific design rules.</span></span>

 <span data-ttu-id="8d9a8-104">Klasy są ogólną wielkością liter typów referencyjnych.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-104">Classes are the general case of reference types.</span></span> <span data-ttu-id="8d9a8-105">Składają się na siebie zbiory typów w większości platform.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-105">They make up the bulk of types in the majority of frameworks.</span></span> <span data-ttu-id="8d9a8-106">Klasy są długoterminowe w rozbudowanym zestawie funkcji opartych na obiektach, które obsługują i do ich ogólnego zastosowania.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-106">Classes owe their popularity to the rich set of object-oriented features they support and to their general applicability.</span></span> <span data-ttu-id="8d9a8-107">Klasy bazowe i klasy abstrakcyjne są specjalnymi grupami logicznymi związanymi z rozszerzalnością.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-107">Base classes and abstract classes are special logical groups related to extensibility.</span></span>

 <span data-ttu-id="8d9a8-108">Interfejsy to typy, które mogą być implementowane przez oba typy odwołań i typy wartości.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-108">Interfaces are types that can be implemented by both reference types and value types.</span></span> <span data-ttu-id="8d9a8-109">Mogą to być jako elementy główne hierarchii polimorficznych typów referencyjnych i typów wartości.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-109">They can thus serve as roots of polymorphic hierarchies of reference types and value types.</span></span> <span data-ttu-id="8d9a8-110">Ponadto interfejsy mogą służyć do symulowania wielokrotnego dziedziczenia, które nie jest natywnie obsługiwane przez środowisko CLR.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-110">In addition, interfaces can be used to simulate multiple inheritance, which is not natively supported by the CLR.</span></span>

 <span data-ttu-id="8d9a8-111">Struktury są ogólną wielkością liter typów wartości i powinny być zarezerwowane dla małych, prostych typów, podobnie jak w przypadku elementów podstawowych języka.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-111">Structs are the general case of value types and should be reserved for small, simple types, similar to language primitives.</span></span>

 <span data-ttu-id="8d9a8-112">Wyliczenia są szczególnym przypadkiem typów wartości używanych do definiowania krótkich zestawów wartości, takich jak dni tygodnia, kolory konsoli i tak dalej.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-112">Enums are a special case of value types used to define short sets of values, such as days of the week, console colors, and so on.</span></span>

 <span data-ttu-id="8d9a8-113">Klasy statyczne są typami przeznaczonymi do kontenerów dla statycznych elementów członkowskich.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-113">Static classes are types intended to be containers for static members.</span></span> <span data-ttu-id="8d9a8-114">Są one często używane do udostępniania skrótów do innych operacji.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-114">They are commonly used to provide shortcuts to other operations.</span></span>

 <span data-ttu-id="8d9a8-115">Delegaty, wyjątki, atrybuty, tablice i kolekcje są szczególnymi przypadkami typów referencyjnych, które są przeznaczone do określonych zastosowań, oraz wskazówki dotyczące ich projektowania i użycia zostały omówione w innym miejscu w tej książce.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-115">Delegates, exceptions, attributes, arrays, and collections are all special cases of reference types intended for specific uses, and guidelines for their design and usage are discussed elsewhere in this book.</span></span>

 <span data-ttu-id="8d9a8-116">✔️ Upewnij się, że każdy typ jest dobrze zdefiniowanym zestawem powiązanych członków, a nie tylko losową kolekcją niepowiązanych funkcji.</span><span class="sxs-lookup"><span data-stu-id="8d9a8-116">✔️ DO ensure that each type is a well-defined set of related members, not just a random collection of unrelated functionality.</span></span>
