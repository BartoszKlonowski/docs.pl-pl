 <span data-ttu-id="8c467-168">**X DO NOT** Użycie tablic dla właściwości, jeśli właściwość musiałaby zwracać nowej tablicy (np. kopia tablicy wewnętrznej) zawsze jest wywoływana metoda pobierająca właściwości.</span><span class="sxs-lookup"><span data-stu-id="8c467-168">**X DO NOT** use arrays for properties if the property would have to return a new array (e.g., a copy of an internal array) every time the property getter is called.</span></span>  
  
## <a name="implementing-custom-collections"></a><span data-ttu-id="8c467-169">Implementowanie kolekcji niestandardowych</span><span class="sxs-lookup"><span data-stu-id="8c467-169">Implementing Custom Collections</span></span>  
 <span data-ttu-id="8c467-170">**✓ CONSIDER** dziedziczących `Collection<T>`, `ReadOnlyCollection<T>`, lub `KeyedCollection<TKey,TItem>` podczas projektowania nowej kolekcji.</span><span class="sxs-lookup"><span data-stu-id="8c467-170">**✓ CONSIDER** inheriting from `Collection<T>`, `ReadOnlyCollection<T>`, or `KeyedCollection<TKey,TItem>` when designing new collections.</span></span>  
  
 <span data-ttu-id="8c467-171">**✓ DO** zaimplementować `IEnumerable<T>` podczas projektowania nowej kolekcji.</span><span class="sxs-lookup"><span data-stu-id="8c467-171">**✓ DO** implement `IEnumerable<T>` when designing new collections.</span></span> <span data-ttu-id="8c467-172">Rozważ zaimplementowanie `ICollection<T>` lub nawet `IList<T>` gdzie dobrym pomysłem.</span><span class="sxs-lookup"><span data-stu-id="8c467-172">Consider implementing `ICollection<T>` or even `IList<T>` where it makes sense.</span></span>  
  
 <span data-ttu-id="8c467-173">Podczas implementowania takich niestandardowej kolekcji, oparte na wzorcu interfejsu API ustanowione przez `Collection<T>` i `ReadOnlyCollection<T>` możliwie najlepszy sposób.</span><span class="sxs-lookup"><span data-stu-id="8c467-173">When implementing such custom collection, follow the API pattern established by `Collection<T>` and `ReadOnlyCollection<T>` as closely as possible.</span></span> <span data-ttu-id="8c467-174">Oznacza to należy zaimplementować te same elementy członkowskie jawnie, nazwy parametrów, takich jak te dwie kolekcje, nazwij je i tak dalej.</span><span class="sxs-lookup"><span data-stu-id="8c467-174">That is, implement the same members explicitly, name the parameters like these two collections name them, and so on.</span></span>  
  
 <span data-ttu-id="8c467-175">**✓ CONSIDER** implementowanie interfejsów kolekcji nierodzajowe (`IList` i `ICollection`) Jeśli kolekcji będą często przekazywane do interfejsów API biorąc te interfejsy jako dane wejściowe.</span><span class="sxs-lookup"><span data-stu-id="8c467-175">**✓ CONSIDER** implementing nongeneric collection interfaces (`IList` and `ICollection`) if the collection will often be passed to APIs taking these interfaces as input.</span></span>  
  
 <span data-ttu-id="8c467-176">**X AVOID** implementowanie interfejsów kolekcji na typach z złożonych interfejsów API niezwiązanych ze sobą koncepcji kolekcji.</span><span class="sxs-lookup"><span data-stu-id="8c467-176">**X AVOID** implementing collection interfaces on types with complex APIs unrelated to the concept of a collection.</span></span>  
  
 <span data-ttu-id="8c467-177">**X DO NOT** dziedziczyć nierodzajowe kolekcje podstawowej takich jak `CollectionBase`.</span><span class="sxs-lookup"><span data-stu-id="8c467-177">**X DO NOT** inherit from nongeneric base collections such as `CollectionBase`.</span></span> <span data-ttu-id="8c467-178">Użyj `Collection<T>`, `ReadOnlyCollection<T>`, i `KeyedCollection<TKey,TItem>` zamiast tego.</span><span class="sxs-lookup"><span data-stu-id="8c467-178">Use `Collection<T>`, `ReadOnlyCollection<T>`, and `KeyedCollection<TKey,TItem>` instead.</span></span>  
  
### <a name="naming-custom-collections"></a><span data-ttu-id="8c467-179">Nazewnictwo kolekcje niestandardowe</span><span class="sxs-lookup"><span data-stu-id="8c467-179">Naming Custom Collections</span></span>  
 <span data-ttu-id="8c467-180">Kolekcje (typami, które implementują `IEnumerable`) są tworzone głównie dwóch powodów: (1), aby utworzyć nową strukturę danych za pomocą operacji specyficznych dla struktury i często różną charakterystykę wydajności niż istniejącymi strukturami danych (np. <xref:System.Collections.Generic.List%601>, <xref:System.Collections.Generic.LinkedList%601>, <xref:System.Collections.Generic.Stack%601>), a (2), aby utworzyć kolekcję specjalne do przechowywania określony zbiór elementów (np. <xref:System.Collections.Specialized.StringCollection>).</span><span class="sxs-lookup"><span data-stu-id="8c467-180">Collections (types that implement `IEnumerable`) are created mainly for two reasons: (1) to create a new data structure with structure-specific operations and often different performance characteristics than existing data structures (e.g.,  <xref:System.Collections.Generic.List%601>, <xref:System.Collections.Generic.LinkedList%601>, <xref:System.Collections.Generic.Stack%601>), and (2) to create a specialized collection for holding a specific set of items (e.g.,  <xref:System.Collections.Specialized.StringCollection>).</span></span> <span data-ttu-id="8c467-181">Struktury danych są najczęściej używane w wewnętrznej implementacji aplikacji i bibliotek.</span><span class="sxs-lookup"><span data-stu-id="8c467-181">Data structures are most often used in the internal implementation of applications and libraries.</span></span> <span data-ttu-id="8c467-182">Specjalne kolekcje są głównie ujawnianie w interfejsach API (jako typy właściwości i parametrów).</span><span class="sxs-lookup"><span data-stu-id="8c467-182">Specialized collections are mainly to be exposed in APIs (as property and parameter types).</span></span>  
  
 <span data-ttu-id="8c467-183">**✓ DO** Użyj sufiksu "Słownik" w nazwach obiektów abstrakcyjnych implementacja `IDictionary` lub `IDictionary<TKey,TValue>`.</span><span class="sxs-lookup"><span data-stu-id="8c467-183">**✓ DO** use the "Dictionary" suffix in names of abstractions implementing `IDictionary` or `IDictionary<TKey,TValue>`.</span></span>  
  
 <span data-ttu-id="8c467-184">**✓ DO** Użyj sufiksu "Kolekcji" w nazwach typów implementujących `IEnumerable` (lub któregokolwiek z jej obiektów podrzędnych) i reprezentujący listę elementów.</span><span class="sxs-lookup"><span data-stu-id="8c467-184">**✓ DO** use the "Collection" suffix in names of types implementing `IEnumerable` (or any of its descendants) and representing a list of items.</span></span>  
  
 <span data-ttu-id="8c467-185">**✓ DO** Użyj nazwy struktury danych w strukturach danych niestandardowych.</span><span class="sxs-lookup"><span data-stu-id="8c467-185">**✓ DO** use the appropriate data structure name for custom data structures.</span></span>  
  
 <span data-ttu-id="8c467-186">**X AVOID** przy użyciu wszelkie sufiksy podejrzeń konkretnej implementacji, takie jak "LinkedList" lub "Hashtable," w nazwach obiektów abstrakcyjnych kolekcji.</span><span class="sxs-lookup"><span data-stu-id="8c467-186">**X AVOID** using any suffixes implying particular implementation, such as "LinkedList" or "Hashtable," in names of collection abstractions.</span></span>  
  
 <span data-ttu-id="8c467-187">**✓ CONSIDER** prefiksu nazwy kolekcji o nazwie typu elementu.</span><span class="sxs-lookup"><span data-stu-id="8c467-187">**✓ CONSIDER** prefixing collection names with the name of the item type.</span></span> <span data-ttu-id="8c467-188">Na przykład przechowywanie elementów tego typu kolekcji `Address` (Implementowanie `IEnumerable<Address>`) powinno się nazywać `AddressCollection`.</span><span class="sxs-lookup"><span data-stu-id="8c467-188">For example, a collection storing items of type `Address` (implementing `IEnumerable<Address>`) should be named `AddressCollection`.</span></span> <span data-ttu-id="8c467-189">Jeśli typ elementu to interfejs, prefiks "I" elementu typu można pominąć.</span><span class="sxs-lookup"><span data-stu-id="8c467-189">If the item type is an interface, the "I" prefix of the item type can be omitted.</span></span> <span data-ttu-id="8c467-190">W związku z tym, zbiór <xref:System.IDisposable> elementy mogą być wywoływane `DisposableCollection`.</span><span class="sxs-lookup"><span data-stu-id="8c467-190">Thus, a collection of <xref:System.IDisposable> items can be called `DisposableCollection`.</span></span>  
  
 <span data-ttu-id="8c467-191">**✓ CONSIDER** przy użyciu prefiksu "ReadOnly" w nazwach kolekcji tylko do odczytu, jeśli odpowiedniej kolekcji zapisu mogą być dodane lub już istnieje w ramach.</span><span class="sxs-lookup"><span data-stu-id="8c467-191">**✓ CONSIDER** using the "ReadOnly" prefix in names of read-only collections if a corresponding writeable collection might be added or already exists in the framework.</span></span>  
  
 <span data-ttu-id="8c467-192">Na przykład, można wywołać tylko do odczytu kolekcji ciągów `ReadOnlyStringCollection`.</span><span class="sxs-lookup"><span data-stu-id="8c467-192">For example, a read-only collection of strings should be called `ReadOnlyStringCollection`.</span></span>
