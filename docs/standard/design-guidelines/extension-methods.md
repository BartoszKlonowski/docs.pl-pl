# <a name="extension-methods"></a><span data-ttu-id="80098-102">Metody rozszerzeń</span><span class="sxs-lookup"><span data-stu-id="80098-102">Extension Methods</span></span>
<span data-ttu-id="80098-103">Metody rozszerzenia są funkcją języka, umożliwiająca metody statyczne, które można wywoływać za pomocą składni wywołania metody wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="80098-103">Extension methods are a language feature that allows static methods to be called using instance method call syntax.</span></span> <span data-ttu-id="80098-104">Te metody musi mieć co najmniej jeden parametr, który reprezentuje wystąpienie, którego metoda jest używana w ramach operacji.</span><span class="sxs-lookup"><span data-stu-id="80098-104">These methods must take at least one parameter, which represents the instance the method is to operate on.</span></span>  
  
 <span data-ttu-id="80098-105">Klasa, która definiuje takie metody rozszerzenia jest określany jako klasy "sponsora", a musi być zadeklarowana jako statyczna.</span><span class="sxs-lookup"><span data-stu-id="80098-105">The class that defines such extension methods is referred to as the "sponsor" class, and it must be declared as static.</span></span> <span data-ttu-id="80098-106">Można użyć metod rozszerzających, jeden zaimportować Definiowanie klasy sponsora przestrzeni nazw.</span><span class="sxs-lookup"><span data-stu-id="80098-106">To use extension methods, one must import the namespace defining the sponsor class.</span></span>  
  
 <span data-ttu-id="80098-107">**X AVOID** frivolously definiowania metod rozszerzenia, szczególnie w przypadku typów nie jesteś właścicielem.</span><span class="sxs-lookup"><span data-stu-id="80098-107">**X AVOID** frivolously defining extension methods, especially on types you don’t own.</span></span>  
  
 <span data-ttu-id="80098-108">Jeśli jesteś właścicielem kodu źródłowego typu, metody regularnego wystąpienia zamiast tego Rozważ użycie.</span><span class="sxs-lookup"><span data-stu-id="80098-108">If you do own source code of a type, consider using regular instance methods instead.</span></span> <span data-ttu-id="80098-109">Jeśli nie jesteś właścicielem, i chcesz dodać metodę, należy zwrócić szczególną uwagę.</span><span class="sxs-lookup"><span data-stu-id="80098-109">If you don’t own, and you want to add a method, be very careful.</span></span> <span data-ttu-id="80098-110">Liberalne stosowanie metod rozszerzenia stwarza możliwość potencjalnego zaśmiecania interfejsów API, typów, które nie zostały zaprojektowane do mieć tych metod.</span><span class="sxs-lookup"><span data-stu-id="80098-110">Liberal use of extension methods has the potential of cluttering APIs of types that were not designed to have these methods.</span></span>  
  
 <span data-ttu-id="80098-111">**✓ CONSIDER** przy użyciu metod rozszerzenia w jednym z następujących scenariuszy:</span><span class="sxs-lookup"><span data-stu-id="80098-111">**✓ CONSIDER** using extension methods in any of the following scenarios:</span></span>  
  
-   <span data-ttu-id="80098-112">Zapewnienie pomocy funkcji, które dotyczą każdej implementacji interfejsu, w przypadku funkcji mogą być napisane pod kątem interfejsu core.</span><span class="sxs-lookup"><span data-stu-id="80098-112">To provide helper functionality relevant to every implementation of an interface, if said functionality can be written in terms of the core interface.</span></span> <span data-ttu-id="80098-113">Jest to spowodowane konkretnych implementacji, w przeciwnym razie nie można przypisać do interfejsów.</span><span class="sxs-lookup"><span data-stu-id="80098-113">This is because concrete implementations cannot otherwise be assigned to interfaces.</span></span> <span data-ttu-id="80098-114">Na przykład `LINQ to Objects` operatory są implementowane jako metody rozszerzenia dla wszystkich <xref:System.Collections.Generic.IEnumerable%601> typów.</span><span class="sxs-lookup"><span data-stu-id="80098-114">For example, the `LINQ to Objects` operators are implemented as extension methods for all <xref:System.Collections.Generic.IEnumerable%601> types.</span></span> <span data-ttu-id="80098-115">W związku z dowolnego `IEnumerable<>` implementacja jest włączane automatycznie LINQ.</span><span class="sxs-lookup"><span data-stu-id="80098-115">Thus, any `IEnumerable<>` implementation is automatically LINQ-enabled.</span></span>  
  
-   <span data-ttu-id="80098-116">Gdy metoda wystąpienia spowodowałoby to powstanie zależności pewnego typu, ale takie zależności zaburzyłaby reguły zarządzania zależności.</span><span class="sxs-lookup"><span data-stu-id="80098-116">When an instance method would introduce a dependency on some type, but such a dependency would break dependency management rules.</span></span> <span data-ttu-id="80098-117">Na przykład zależność od <xref:System.String> do <xref:System.Uri?displayProperty=nameWithType> prawdopodobnie nie jest pożądane, a tym samym `String.ToUri()` metodę wystąpienia, zwracając `System.Uri` będzie niewłaściwy projekt z punktu widzenia zarządzania zależności.</span><span class="sxs-lookup"><span data-stu-id="80098-117">For example, a dependency from <xref:System.String> to <xref:System.Uri?displayProperty=nameWithType> is probably not desirable, and so `String.ToUri()` instance method returning `System.Uri` would be the wrong design from a dependency management perspective.</span></span> <span data-ttu-id="80098-118">Metody statyczne rozszerzenie `Uri.ToUri(this string str)` zwracanie `System.Uri` będzie znacznie lepiej projektu.</span><span class="sxs-lookup"><span data-stu-id="80098-118">A static extension method `Uri.ToUri(this string str)` returning `System.Uri` would be a much better design.</span></span>  
  
 <span data-ttu-id="80098-119">**X AVOID** definiowanie metody rozszerzenia na <xref:System.Object?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="80098-119">**X AVOID** defining extension methods on <xref:System.Object?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="80098-120">Użytkownicy VB nie będzie wywoływać tych metod w odwołania do obiektu za pomocą składni metody rozszerzenia.</span><span class="sxs-lookup"><span data-stu-id="80098-120">VB users will not be able to call such methods on object references using the extension method syntax.</span></span> <span data-ttu-id="80098-121">VB nie obsługuje wywoływania takich metod, ponieważ w języku Visual Basic, deklaruje odwołanie, jako obiekt wymusza wszystkich wywołań metod w celu Spóźnię się powiązany (rzeczywistego elementu członkowskiego, wywoływana jest określana w czasie wykonywania), podczas gdy powiązania z metody rozszerzenia są określane w czasie kompilacji (wcześnie powiązany).</span><span class="sxs-lookup"><span data-stu-id="80098-121">VB does not support calling such methods because, in VB, declaring a reference as Object forces all method invocations on it to be late bound (actual member called is determined at runtime), while bindings to extension methods are determined at compile-time (early bound).</span></span>  
  
 <span data-ttu-id="80098-122">Należy pamiętać, że wytyczna ma zastosowanie do innych języków, w którym znajduje się takie samo zachowanie powiązania, lub gdy metody rozszerzenia nie są obsługiwane.</span><span class="sxs-lookup"><span data-stu-id="80098-122">Note that the guideline applies to other languages where the same binding behavior is present, or where extension methods are not supported.</span></span>  
  
 <span data-ttu-id="80098-123">**X DO NOT** umieszczenia metod rozszerzenia w tej samej przestrzeni nazw jako typ rozszerzony, chyba że jest dodanie metody interfejsów lub zarządzania zależności.</span><span class="sxs-lookup"><span data-stu-id="80098-123">**X DO NOT** put extension methods in the same namespace as the extended type unless it is for adding methods to interfaces or for dependency management.</span></span>  
  
 <span data-ttu-id="80098-124">**X AVOID** zdefiniowania co najmniej dwie metody rozszerzenia o tej samej sygnaturze, nawet jeśli są one przechowywane w różnych przestrzeniach nazw.</span><span class="sxs-lookup"><span data-stu-id="80098-124">**X AVOID** defining two or more extension methods with the same signature, even if they reside in different namespaces.</span></span>  
  
 <span data-ttu-id="80098-125">**✓ CONSIDER** definiowanie metody rozszerzenia w tej samej przestrzeni nazw jako typ rozszerzonej, jeśli typ jest interfejsem i metody rozszerzenia są przeznaczone do użycia w przypadku większości lub wszystkim.</span><span class="sxs-lookup"><span data-stu-id="80098-125">**✓ CONSIDER** defining extension methods in the same namespace as the extended type if the type is an interface and if the extension methods are meant to be used in most or all cases.</span></span>  
  
 <span data-ttu-id="80098-126">**X DO NOT** definiować metody rozszerzenia wdrożenie funkcją w przestrzeniach nazw zwykle skojarzone z innymi funkcjami.</span><span class="sxs-lookup"><span data-stu-id="80098-126">**X DO NOT** define extension methods implementing a feature in namespaces normally associated with other features.</span></span> <span data-ttu-id="80098-127">Zamiast tego zdefiniuj je w przestrzeni nazw skojarzonego z funkcją, które należą do.</span><span class="sxs-lookup"><span data-stu-id="80098-127">Instead, define them in the namespace associated with the feature they belong to.</span></span>  
  
 <span data-ttu-id="80098-128">**X AVOID** ogólnego nazewnictwa przestrzeni nazw przeznaczona do metody rozszerzenia (np. "rozszerzenia").</span><span class="sxs-lookup"><span data-stu-id="80098-128">**X AVOID** generic naming of namespaces dedicated to extension methods (e.g., "Extensions").</span></span> <span data-ttu-id="80098-129">Użyj nazwy opisowej (np. "Routing") zamiast tego.</span><span class="sxs-lookup"><span data-stu-id="80098-129">Use a descriptive name (e.g., "Routing") instead.</span></span>
