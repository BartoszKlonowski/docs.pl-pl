# <a name="events-and-callbacks"></a><span data-ttu-id="d3c62-102">Zdarzenia i wywołania zwrotne</span><span class="sxs-lookup"><span data-stu-id="d3c62-102">Events and Callbacks</span></span>
<span data-ttu-id="d3c62-103">Wywołania zwrotne to punkty rozszerzalności, które umożliwiają platformie wywołanie z powrotem do kodu użytkownika przez delegata.</span><span class="sxs-lookup"><span data-stu-id="d3c62-103">Callbacks are extensibility points that allow a framework to call back into user code through a delegate.</span></span> <span data-ttu-id="d3c62-104">Te Delegaty są zwykle przesyłane do struktury za pomocą parametru metody.</span><span class="sxs-lookup"><span data-stu-id="d3c62-104">These delegates are usually passed to the framework through a parameter of a method.</span></span>

 <span data-ttu-id="d3c62-105">Zdarzenia to specjalne przypadki wywołania zwrotnego, które obsługują wygodną i spójną składnię do dostarczania delegata (program obsługi zdarzeń).</span><span class="sxs-lookup"><span data-stu-id="d3c62-105">Events are a special case of callbacks that supports convenient and consistent syntax for supplying the delegate (an event handler).</span></span> <span data-ttu-id="d3c62-106">Ponadto uzupełnianie instrukcji i projektantów programu Visual Studio zapewnia pomoc przy korzystaniu z interfejsów API opartych na zdarzeniach.</span><span class="sxs-lookup"><span data-stu-id="d3c62-106">In addition, Visual Studio's statement completion and designers provide help in using event-based APIs.</span></span> <span data-ttu-id="d3c62-107">(Zobacz [projekt zdarzenia](event.md)).</span><span class="sxs-lookup"><span data-stu-id="d3c62-107">(See [Event Design](event.md).)</span></span>

 <span data-ttu-id="d3c62-108">✔️ ROZWAŻYĆ użycie wywołania zwrotnego, aby umożliwić użytkownikom udostępnianie niestandardowego kodu do wykonania przez platformę.</span><span class="sxs-lookup"><span data-stu-id="d3c62-108">✔️ CONSIDER using callbacks to allow users to provide custom code to be executed by the framework.</span></span>

 <span data-ttu-id="d3c62-109">✔️ ROZWAŻYĆ użycie zdarzeń, aby umożliwić użytkownikom dostosowanie zachowania struktury bez konieczności poznania projektu zorientowanego obiektowo.</span><span class="sxs-lookup"><span data-stu-id="d3c62-109">✔️ CONSIDER using events to allow users to customize the behavior of a framework without the need for understanding object-oriented design.</span></span>

 <span data-ttu-id="d3c62-110">✔️ Preferuj zdarzenia przez zwykłe wywołania zwrotne, ponieważ są one bardziej znane dla szerszego zakresu deweloperów i są zintegrowane z uzupełnianiem instrukcji programu Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="d3c62-110">✔️ DO prefer events over plain callbacks, because they are more familiar to a broader range of developers and are integrated with Visual Studio statement completion.</span></span>

 <span data-ttu-id="d3c62-111">❌ UNIKAj używania wywołań zwrotnych w interfejsach API z uwzględnieniem wydajności.</span><span class="sxs-lookup"><span data-stu-id="d3c62-111">❌ AVOID using callbacks in performance-sensitive APIs.</span></span>

 <span data-ttu-id="d3c62-112">`Func<...>` `Action<...>` `Expression<...>` podczas definiowania interfejsów API z wywołaniami zwrotnymi ✔️ używać nowych typów zamiast niestandardowych delegatów.</span><span class="sxs-lookup"><span data-stu-id="d3c62-112">✔️ DO use the new `Func<...>`, `Action<...>`, or `Expression<...>` types instead of custom delegates, when defining APIs with callbacks.</span></span>

 <span data-ttu-id="d3c62-113">`Func<...>` i `Action<...>` reprezentuje delegatów ogólnych.</span><span class="sxs-lookup"><span data-stu-id="d3c62-113">`Func<...>` and `Action<...>` represent generic delegates.</span></span> <span data-ttu-id="d3c62-114">`Expression<...>` reprezentuje definicje funkcji, które mogą być kompilowane i następnie wywoływane w czasie wykonywania, ale mogą być również serializowane i przesyłane do procesów zdalnych.</span><span class="sxs-lookup"><span data-stu-id="d3c62-114">`Expression<...>` represents function definitions that can be compiled and subsequently invoked at runtime but can also be serialized and passed to remote processes.</span></span>

 <span data-ttu-id="d3c62-115">✔️ DO mierzenia i zrozumienia implikacji wydajności przy użyciu `Expression<...>` , zamiast używać `Func<...>` i `Action<...>` delegatów.</span><span class="sxs-lookup"><span data-stu-id="d3c62-115">✔️ DO measure and understand performance implications of using `Expression<...>`, instead of using `Func<...>` and `Action<...>` delegates.</span></span>

 <span data-ttu-id="d3c62-116">`Expression<...>` typy są w większości przypadków logicznie równoważne z `Func<...>` i `Action<...>` delegatów.</span><span class="sxs-lookup"><span data-stu-id="d3c62-116">`Expression<...>` types are in most cases logically equivalent to `Func<...>` and `Action<...>` delegates.</span></span> <span data-ttu-id="d3c62-117">Główna różnica polega na tym, że Delegaty mają być używane w scenariuszach procesu lokalnego; wyrażenia są przeznaczone do przypadków, gdy są korzystne i możliwe do obliczenia wyrażenia w procesie lub na komputerze zdalnym.</span><span class="sxs-lookup"><span data-stu-id="d3c62-117">The main difference between them is that the delegates are intended to be used in local process scenarios; expressions are intended for cases where it's beneficial and possible to evaluate the expression in a remote process or machine.</span></span>

 <span data-ttu-id="d3c62-118">✔️ zrozumieć, że wywołując delegata, wykonujesz dowolny kod, który może mieć wpływ na bezpieczeństwo, prawidłowość i zgodność.</span><span class="sxs-lookup"><span data-stu-id="d3c62-118">✔️ DO understand that by calling a delegate, you are executing arbitrary code and that could have security, correctness, and compatibility repercussions.</span></span>
