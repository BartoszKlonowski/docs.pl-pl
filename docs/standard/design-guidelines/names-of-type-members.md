## <a name="names-of-methods"></a><span data-ttu-id="8ccda-106">Nazwy metod</span><span class="sxs-lookup"><span data-stu-id="8ccda-106">Names of Methods</span></span>
 <span data-ttu-id="8ccda-107">Ponieważ metody są metodą podjęcia działania, wskazówki dotyczące projektowania wymagają, aby nazwy metod były czasownikami lub wyrażeniami czasownikowymi.</span><span class="sxs-lookup"><span data-stu-id="8ccda-107">Because methods are the means of taking action, the design guidelines require that method names be verbs or verb phrases.</span></span> <span data-ttu-id="8ccda-108">Poniższe wytyczne umożliwiają również odróżnianie nazw metod od nazw właściwości i typów, które są frazami rzeczownikmi lub przymiotnikami.</span><span class="sxs-lookup"><span data-stu-id="8ccda-108">Following this guideline also serves to distinguish method names from property and type names, which are noun or adjective phrases.</span></span>

 <span data-ttu-id="8ccda-109">✔️ NALEŻY nadać metodom nazwy, które są czasownikami lub czasownikami zleceń.</span><span class="sxs-lookup"><span data-stu-id="8ccda-109">✔️ DO give methods names that are verbs or verb phrases.</span></span>

```csharp
public class String {
    public int CompareTo(...);
    public string[] Split(...);
    public string Trim();
}
```

## <a name="names-of-properties"></a><span data-ttu-id="8ccda-110">Nazwy właściwości</span><span class="sxs-lookup"><span data-stu-id="8ccda-110">Names of Properties</span></span>
 <span data-ttu-id="8ccda-111">W przeciwieństwie do innych elementów członkowskich, właściwości powinny zawierać frazę rzeczowników lub nazwy przymiotników.</span><span class="sxs-lookup"><span data-stu-id="8ccda-111">Unlike other members, properties should be given noun phrase or adjective names.</span></span> <span data-ttu-id="8ccda-112">Oznacza to, że Właściwość odwołuje się do danych, a nazwa właściwości odzwierciedla to.</span><span class="sxs-lookup"><span data-stu-id="8ccda-112">That is because a property refers to data, and the name of the property reflects that.</span></span> <span data-ttu-id="8ccda-113">PascalCasing jest zawsze używana dla nazw właściwości.</span><span class="sxs-lookup"><span data-stu-id="8ccda-113">PascalCasing is always used for property names.</span></span>

 <span data-ttu-id="8ccda-114">✔️ Właściwości nazwy przy użyciu rzeczownika, frazy rzeczowników lub przymiotniku.</span><span class="sxs-lookup"><span data-stu-id="8ccda-114">✔️ DO name properties using a noun, noun phrase, or adjective.</span></span>

 <span data-ttu-id="8ccda-115">❌ NIE mają właściwości, które pasują do nazwy metod "Get", tak jak w poniższym przykładzie:</span><span class="sxs-lookup"><span data-stu-id="8ccda-115">❌ DO NOT have properties that match the name of "Get" methods as in the following example:</span></span>

 <span data-ttu-id="8ccda-116">`public string TextWriter { get {...} set {...} }` `public string GetTextWriter(int value) { ... }`</span><span class="sxs-lookup"><span data-stu-id="8ccda-116">`public string TextWriter { get {...} set {...} }` `public string GetTextWriter(int value) { ... }`</span></span>

 <span data-ttu-id="8ccda-117">Ten wzorzec zazwyczaj wskazuje, że właściwość powinna być w rzeczywistości metodą.</span><span class="sxs-lookup"><span data-stu-id="8ccda-117">This pattern typically indicates that the property should really be a method.</span></span>

 <span data-ttu-id="8ccda-118">✔️ DO właściwości kolekcji nazw z frazą w liczbie mnogiej opisującą elementy w kolekcji, zamiast używać pojedynczej frazy, a po niej "list" lub "Collection".</span><span class="sxs-lookup"><span data-stu-id="8ccda-118">✔️ DO name collection properties with a plural phrase describing the items in the collection instead of using a singular phrase followed by "List" or "Collection".</span></span>

 <span data-ttu-id="8ccda-119">✔️ NALEŻY nazwać właściwości logiczne z wyrażeniem potwierdzającym ( `CanSeek` zamiast `CantSeek` ).</span><span class="sxs-lookup"><span data-stu-id="8ccda-119">✔️ DO name Boolean properties with an affirmative phrase (`CanSeek` instead of `CantSeek`).</span></span> <span data-ttu-id="8ccda-120">Opcjonalnie można również prefiksować właściwości logiczne z "is", "może" lub "ma", ale tylko wtedy, gdy dodaje wartość.</span><span class="sxs-lookup"><span data-stu-id="8ccda-120">Optionally, you can also prefix Boolean properties with "Is", "Can", or "Has", but only where it adds value.</span></span>

 <span data-ttu-id="8ccda-121">✔️ ROZWAŻYĆ nadanie właściwości takiej samej nazwy jak jej typ.</span><span class="sxs-lookup"><span data-stu-id="8ccda-121">✔️ CONSIDER giving a property the same name as its type.</span></span>

 <span data-ttu-id="8ccda-122">Na przykład następująca Właściwość prawidłowo pobiera i ustawia wartość wyliczenia o nazwie `Color` , więc właściwość ma nazwę `Color` :</span><span class="sxs-lookup"><span data-stu-id="8ccda-122">For example, the following property correctly gets and sets an enum value named `Color`, so the property is named `Color`:</span></span>

```csharp
public enum Color {...}
public class Control {
    public Color Color { get {...} set {...} }
}
```

## <a name="names-of-events"></a><span data-ttu-id="8ccda-123">Nazwy zdarzeń</span><span class="sxs-lookup"><span data-stu-id="8ccda-123">Names of Events</span></span>
 <span data-ttu-id="8ccda-124">Zdarzenia zawsze odwołują się do niektórych akcji, takich, które są wykonywane lub takie, które wystąpiły.</span><span class="sxs-lookup"><span data-stu-id="8ccda-124">Events always refer to some action, either one that is happening or one that has occurred.</span></span> <span data-ttu-id="8ccda-125">W związku z tym, podobnie jak w przypadku metod, zdarzenia są nazwane przy użyciu czasowników, a w celu wskazania czasu, gdy zdarzenie jest zgłaszane, używana jest wartość zlecenia.</span><span class="sxs-lookup"><span data-stu-id="8ccda-125">Therefore, as with methods, events are named with verbs, and verb tense is used to indicate the time when the event is raised.</span></span>

 <span data-ttu-id="8ccda-126">✔️ NALEŻY nazwać zdarzenia za pomocą czasownika lub wyrażenia orzeczenia.</span><span class="sxs-lookup"><span data-stu-id="8ccda-126">✔️ DO name events with a verb or a verb phrase.</span></span>

 <span data-ttu-id="8ccda-127">Przykłady obejmują `Clicked` , `Painting` , `DroppedDown` , i tak dalej.</span><span class="sxs-lookup"><span data-stu-id="8ccda-127">Examples include `Clicked`, `Painting`, `DroppedDown`, and so on.</span></span>

 <span data-ttu-id="8ccda-128">✔️ NALEŻY udzielić nazw zdarzeń z koncepcją przed i po, przy użyciu obecnych i ostatnich dziesiątek.</span><span class="sxs-lookup"><span data-stu-id="8ccda-128">✔️ DO give events names with a concept of before and after, using the present and past tenses.</span></span>

 <span data-ttu-id="8ccda-129">Na przykład zdarzenie zamknięcia, które jest wywoływane przed zamknięciem okna, zostanie wywołane `Closing` , a jeden, który jest wywoływany po zamknięciu okna, zostanie wywołany `Closed` .</span><span class="sxs-lookup"><span data-stu-id="8ccda-129">For example, a close event that is raised before a window is closed would be called `Closing`, and one that is raised after the window is closed would be called `Closed`.</span></span>

 <span data-ttu-id="8ccda-130">❌ NIE używaj "Before" lub "After" prefiksów ani przyrostków, aby wskazać zdarzenia poprzedzające i końcowe.</span><span class="sxs-lookup"><span data-stu-id="8ccda-130">❌ DO NOT use "Before" or "After" prefixes or postfixes to indicate pre- and post-events.</span></span> <span data-ttu-id="8ccda-131">Użyj obecnych i ostatnich dziesiątek tak samo, jak zostało to opisane.</span><span class="sxs-lookup"><span data-stu-id="8ccda-131">Use present and past tenses as just described.</span></span>

 <span data-ttu-id="8ccda-132">✔️ obsługi zdarzeń nazw (delegatów używanych jako typy zdarzeń) przy użyciu sufiksu "EventHandler", jak pokazano w następującym przykładzie:</span><span class="sxs-lookup"><span data-stu-id="8ccda-132">✔️ DO name event handlers (delegates used as types of events) with the "EventHandler" suffix, as shown in the following example:</span></span>

 `public delegate void ClickedEventHandler(object sender, ClickedEventArgs e);`

 <span data-ttu-id="8ccda-133">✔️ używać dwóch parametrów o nazwach `sender` i `e` w obsłudze zdarzeń.</span><span class="sxs-lookup"><span data-stu-id="8ccda-133">✔️ DO use two parameters named `sender` and `e` in event handlers.</span></span>

 <span data-ttu-id="8ccda-134">Parametr Sender reprezentuje obiekt, który spowodował zdarzenie.</span><span class="sxs-lookup"><span data-stu-id="8ccda-134">The sender parameter represents the object that raised the event.</span></span> <span data-ttu-id="8ccda-135">Parametr nadawcy jest zazwyczaj typu `object` , nawet jeśli można użyć bardziej określonego typu.</span><span class="sxs-lookup"><span data-stu-id="8ccda-135">The sender parameter is typically of type `object`, even if it is possible to employ a more specific type.</span></span>

 <span data-ttu-id="8ccda-136">✔️ NALEŻY nazwać klasy argumentów zdarzeń z sufiksem "EventArgs".</span><span class="sxs-lookup"><span data-stu-id="8ccda-136">✔️ DO name event argument classes with the "EventArgs" suffix.</span></span>
