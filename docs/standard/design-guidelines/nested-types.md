# <a name="nested-types"></a><span data-ttu-id="53e21-102">Zagnieżdżone typy</span><span class="sxs-lookup"><span data-stu-id="53e21-102">Nested Types</span></span>
<span data-ttu-id="53e21-103">Typ zagnieżdżony jest typem zdefiniowanym w zakresie innego typu, który jest nazywany typem otaczającym.</span><span class="sxs-lookup"><span data-stu-id="53e21-103">A nested type is a type defined within the scope of another type, which is called the enclosing type.</span></span> <span data-ttu-id="53e21-104">Typ zagnieżdżony ma dostęp do wszystkich elementów członkowskich jego typu otaczającego.</span><span class="sxs-lookup"><span data-stu-id="53e21-104">A nested type has access to all members of its enclosing type.</span></span> <span data-ttu-id="53e21-105">Na przykład ma dostęp do prywatnych pól zdefiniowanych w typie otaczającym i do chronionych pól zdefiniowanych we wszystkich elementów nadrzędnych typu otaczającego.</span><span class="sxs-lookup"><span data-stu-id="53e21-105">For example, it has access to private fields defined in the enclosing type and to protected fields defined in all ascendants of the enclosing type.</span></span>  
  
 <span data-ttu-id="53e21-106">Ogólnie rzecz biorąc, zagnieżdżone typy powinny być używane oszczędnie.</span><span class="sxs-lookup"><span data-stu-id="53e21-106">In general, nested types should be used sparingly.</span></span> <span data-ttu-id="53e21-107">Istnieje kilka przyczyn tego.</span><span class="sxs-lookup"><span data-stu-id="53e21-107">There are several reasons for this.</span></span> <span data-ttu-id="53e21-108">Niektórzy deweloperzy nie są w pełni zaznajomieni z koncepcją.</span><span class="sxs-lookup"><span data-stu-id="53e21-108">Some developers are not fully familiar with the concept.</span></span> <span data-ttu-id="53e21-109">Tacy deweloperzy mogą na przykład mieć problemy z składnią deklarowania zmiennych typów zagnieżdżonych.</span><span class="sxs-lookup"><span data-stu-id="53e21-109">These developers might, for example, have problems with the syntax of declaring variables of nested types.</span></span> <span data-ttu-id="53e21-110">Zagnieżdżone typy są również bardzo ściśle sprzężone z ich otaczającymi typami i nie są odpowiednie do typów ogólnego przeznaczenia.</span><span class="sxs-lookup"><span data-stu-id="53e21-110">Nested types are also very tightly coupled with their enclosing types, and as such are not suited to be general-purpose types.</span></span>  
  
 <span data-ttu-id="53e21-111">Typy zagnieżdżone są najlepiej dopasowane do szczegółów implementacji modelowania ich typów otaczających.</span><span class="sxs-lookup"><span data-stu-id="53e21-111">Nested types are best suited for modeling implementation details of their enclosing types.</span></span> <span data-ttu-id="53e21-112">Użytkownik końcowy powinien rzadko musieli zadeklarować zmienne typu zagnieżdżonego i prawie nigdy nie powinien mieć jawnie utworzyć wystąpienia zagnieżdżonych typów.</span><span class="sxs-lookup"><span data-stu-id="53e21-112">The end user should rarely have to declare variables of a nested type and almost never should have to explicitly instantiate nested types.</span></span> <span data-ttu-id="53e21-113">Na przykład moduł wyliczający kolekcji może być typem zagnieżdżonym tej kolekcji.</span><span class="sxs-lookup"><span data-stu-id="53e21-113">For example, the enumerator of a collection can be a nested type of that collection.</span></span> <span data-ttu-id="53e21-114">Moduły wyliczające są zwykle tworzone przez ich typ otaczający, a ponieważ wiele języków obsługuje instrukcję foreach, zmienne modułu wyliczającego rzadko muszą być deklarowane przez użytkownika końcowego.</span><span class="sxs-lookup"><span data-stu-id="53e21-114">Enumerators are usually instantiated by their enclosing type, and because many languages support the foreach statement, enumerator variables rarely have to be declared by the end user.</span></span>  
  
 <span data-ttu-id="53e21-115">**✓ DO** używać zagnieżdżonych typów w przypadku relacji między typu zagnieżdżonego i jej typu zewnętrznego taki sposób, że dostępność elementu członkowskiego semantyka pożądane.</span><span class="sxs-lookup"><span data-stu-id="53e21-115">**✓ DO** use nested types when the relationship between the nested type and its outer type is such that member-accessibility semantics are desirable.</span></span>  
  
 <span data-ttu-id="53e21-116">**X DO NOT** Użyj publicznego typów zagnieżdżonych jako logiczne grupowanie skonstruować; Użyj przestrzeni nazw dla tego.</span><span class="sxs-lookup"><span data-stu-id="53e21-116">**X DO NOT** use public nested types as a logical grouping construct; use namespaces for this.</span></span>  
  
 <span data-ttu-id="53e21-117">**X AVOID** publicznie udostępnione typy zagnieżdżone.</span><span class="sxs-lookup"><span data-stu-id="53e21-117">**X AVOID** publicly exposed nested types.</span></span> <span data-ttu-id="53e21-118">Jedynym wyjątkiem jest to, że zmienne typu zagnieżdżonego muszą być deklarowane tylko w rzadkich scenariuszach, takich jak podklasy lub inne zaawansowane scenariusze dostosowywania.</span><span class="sxs-lookup"><span data-stu-id="53e21-118">The only exception to this is if variables of the nested type need to be declared only in rare scenarios such as subclassing or other advanced customization scenarios.</span></span>  
  
 <span data-ttu-id="53e21-119">**X DO NOT** Użyj zagnieżdżone typy, jeśli typ może odwoływać się poza typu zawierającego.</span><span class="sxs-lookup"><span data-stu-id="53e21-119">**X DO NOT** use nested types if the type is likely to be referenced outside of the containing type.</span></span>  
  
 <span data-ttu-id="53e21-120">Na przykład Wyliczenie przesłane do metody zdefiniowanej w klasie nie powinno być zdefiniowane jako typ zagnieżdżony w klasie.</span><span class="sxs-lookup"><span data-stu-id="53e21-120">For example, an enum passed to a method defined on a class should not be defined as a nested type in the class.</span></span>  
  
 <span data-ttu-id="53e21-121">**X DO NOT** używać zagnieżdżonych typów, jeśli muszą zostać utworzone przez kod klienta.</span><span class="sxs-lookup"><span data-stu-id="53e21-121">**X DO NOT** use nested types if they need to be instantiated by client code.</span></span>  <span data-ttu-id="53e21-122">Jeśli typ ma konstruktora publicznego, prawdopodobnie nie powinien być zagnieżdżony.</span><span class="sxs-lookup"><span data-stu-id="53e21-122">If a type has a public constructor, it should probably not be nested.</span></span>  
  
 <span data-ttu-id="53e21-123">Jeśli typ może być skonkretyzowany, który wydaje się wskazywać, że typ ma miejsce w strukturze na własny (można go utworzyć, działać z nim i zniszczyć bez użycia typu zewnętrznego) i dlatego nie powinien być zagnieżdżony.</span><span class="sxs-lookup"><span data-stu-id="53e21-123">If a type can be instantiated, that seems to indicate the type has a place in the framework on its own (you can create it, work with it, and destroy it without ever using the outer type), and thus should not be nested.</span></span> <span data-ttu-id="53e21-124">Typy wewnętrzne nie powinny być szeroko ponownie używane poza typem zewnętrznym bez żadnej relacji z typem zewnętrznym.</span><span class="sxs-lookup"><span data-stu-id="53e21-124">Inner types should not be widely reused outside of the outer type without any relationship whatsoever to the outer type.</span></span>  
  
 <span data-ttu-id="53e21-125">**X DO NOT** zagnieżdżony typ jako element członkowski interfejsu.</span><span class="sxs-lookup"><span data-stu-id="53e21-125">**X DO NOT** define a nested type as a member of an interface.</span></span> <span data-ttu-id="53e21-126">W wielu językach nie jest obsługiwana taka konstrukcja.</span><span class="sxs-lookup"><span data-stu-id="53e21-126">Many languages do not support such a construct.</span></span>
