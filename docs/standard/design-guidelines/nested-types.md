# <a name="nested-types"></a><span data-ttu-id="2b17a-102">Zagnieżdżone typy</span><span class="sxs-lookup"><span data-stu-id="2b17a-102">Nested Types</span></span>
<span data-ttu-id="2b17a-103">Typ zagnieżdżony jest typem zdefiniowanym w zakresie innego typu, który jest nazywany typem otaczającym.</span><span class="sxs-lookup"><span data-stu-id="2b17a-103">A nested type is a type defined within the scope of another type, which is called the enclosing type.</span></span> <span data-ttu-id="2b17a-104">Typ zagnieżdżony ma dostęp do wszystkich elementów członkowskich jego typu otaczającego.</span><span class="sxs-lookup"><span data-stu-id="2b17a-104">A nested type has access to all members of its enclosing type.</span></span> <span data-ttu-id="2b17a-105">Na przykład ma dostęp do prywatnych pól zdefiniowanych w typie otaczającym i do chronionych pól zdefiniowanych we wszystkich elementów nadrzędnych typu otaczającego.</span><span class="sxs-lookup"><span data-stu-id="2b17a-105">For example, it has access to private fields defined in the enclosing type and to protected fields defined in all ascendants of the enclosing type.</span></span>

 <span data-ttu-id="2b17a-106">Ogólnie rzecz biorąc, zagnieżdżone typy powinny być używane oszczędnie.</span><span class="sxs-lookup"><span data-stu-id="2b17a-106">In general, nested types should be used sparingly.</span></span> <span data-ttu-id="2b17a-107">Istnieje kilka przyczyn tego.</span><span class="sxs-lookup"><span data-stu-id="2b17a-107">There are several reasons for this.</span></span> <span data-ttu-id="2b17a-108">Niektórzy deweloperzy nie są w pełni zaznajomieni z koncepcją.</span><span class="sxs-lookup"><span data-stu-id="2b17a-108">Some developers are not fully familiar with the concept.</span></span> <span data-ttu-id="2b17a-109">Tacy deweloperzy mogą na przykład mieć problemy z składnią deklarowania zmiennych typów zagnieżdżonych.</span><span class="sxs-lookup"><span data-stu-id="2b17a-109">These developers might, for example, have problems with the syntax of declaring variables of nested types.</span></span> <span data-ttu-id="2b17a-110">Zagnieżdżone typy są również bardzo ściśle sprzężone z ich otaczającymi typami i nie są odpowiednie do typów ogólnego przeznaczenia.</span><span class="sxs-lookup"><span data-stu-id="2b17a-110">Nested types are also very tightly coupled with their enclosing types, and as such are not suited to be general-purpose types.</span></span>

 <span data-ttu-id="2b17a-111">Typy zagnieżdżone są najlepiej dopasowane do szczegółów implementacji modelowania ich typów otaczających.</span><span class="sxs-lookup"><span data-stu-id="2b17a-111">Nested types are best suited for modeling implementation details of their enclosing types.</span></span> <span data-ttu-id="2b17a-112">Użytkownik końcowy powinien rzadko musieli zadeklarować zmienne typu zagnieżdżonego i prawie nigdy nie powinien mieć jawnie utworzyć wystąpienia zagnieżdżonych typów.</span><span class="sxs-lookup"><span data-stu-id="2b17a-112">The end user should rarely have to declare variables of a nested type and almost never should have to explicitly instantiate nested types.</span></span> <span data-ttu-id="2b17a-113">Na przykład moduł wyliczający kolekcji może być typem zagnieżdżonym tej kolekcji.</span><span class="sxs-lookup"><span data-stu-id="2b17a-113">For example, the enumerator of a collection can be a nested type of that collection.</span></span> <span data-ttu-id="2b17a-114">Moduły wyliczające są zwykle tworzone przez ich typ otaczający, a ponieważ wiele języków obsługuje instrukcję foreach, zmienne modułu wyliczającego rzadko muszą być deklarowane przez użytkownika końcowego.</span><span class="sxs-lookup"><span data-stu-id="2b17a-114">Enumerators are usually instantiated by their enclosing type, and because many languages support the foreach statement, enumerator variables rarely have to be declared by the end user.</span></span>

 <span data-ttu-id="2b17a-115">✔️ Używaj zagnieżdżonych typów, gdy relacja między typem zagnieżdżonym a jego zewnętrznym typem jest taka, że semantyka dostępności elementu członkowskiego jest pożądana.</span><span class="sxs-lookup"><span data-stu-id="2b17a-115">✔️ DO use nested types when the relationship between the nested type and its outer type is such that member-accessibility semantics are desirable.</span></span>

 <span data-ttu-id="2b17a-116">❌ nie używać publicznych typów zagnieżdżonych jako konstrukcji grupowania logicznego; Użyj przestrzeni nazw dla tego programu.</span><span class="sxs-lookup"><span data-stu-id="2b17a-116">❌ DO NOT use public nested types as a logical grouping construct; use namespaces for this.</span></span>

 <span data-ttu-id="2b17a-117">❌ unikać publicznie uwidocznionych typów zagnieżdżonych.</span><span class="sxs-lookup"><span data-stu-id="2b17a-117">❌ AVOID publicly exposed nested types.</span></span> <span data-ttu-id="2b17a-118">Jedynym wyjątkiem jest to, że zmienne typu zagnieżdżonego muszą być deklarowane tylko w rzadkich scenariuszach, takich jak podklasy lub inne zaawansowane scenariusze dostosowywania.</span><span class="sxs-lookup"><span data-stu-id="2b17a-118">The only exception to this is if variables of the nested type need to be declared only in rare scenarios such as subclassing or other advanced customization scenarios.</span></span>

 <span data-ttu-id="2b17a-119">❌ nie używaj zagnieżdżonych typów, jeśli typ jest prawdopodobnie przywoływany poza typem zawierającym.</span><span class="sxs-lookup"><span data-stu-id="2b17a-119">❌ DO NOT use nested types if the type is likely to be referenced outside of the containing type.</span></span>

 <span data-ttu-id="2b17a-120">Na przykład Wyliczenie przesłane do metody zdefiniowanej w klasie nie powinno być zdefiniowane jako typ zagnieżdżony w klasie.</span><span class="sxs-lookup"><span data-stu-id="2b17a-120">For example, an enum passed to a method defined on a class should not be defined as a nested type in the class.</span></span>

 <span data-ttu-id="2b17a-121">❌ nie używaj zagnieżdżonych typów, jeśli muszą one być tworzone przez kod klienta.</span><span class="sxs-lookup"><span data-stu-id="2b17a-121">❌ DO NOT use nested types if they need to be instantiated by client code.</span></span>  <span data-ttu-id="2b17a-122">Jeśli typ ma konstruktora publicznego, prawdopodobnie nie powinien być zagnieżdżony.</span><span class="sxs-lookup"><span data-stu-id="2b17a-122">If a type has a public constructor, it should probably not be nested.</span></span>

 <span data-ttu-id="2b17a-123">Jeśli typ może być skonkretyzowany, który wydaje się wskazywać, że typ ma miejsce w strukturze na własny (można go utworzyć, działać z nim i zniszczyć bez użycia typu zewnętrznego) i dlatego nie powinien być zagnieżdżony.</span><span class="sxs-lookup"><span data-stu-id="2b17a-123">If a type can be instantiated, that seems to indicate the type has a place in the framework on its own (you can create it, work with it, and destroy it without ever using the outer type), and thus should not be nested.</span></span> <span data-ttu-id="2b17a-124">Typy wewnętrzne nie powinny być szeroko ponownie używane poza typem zewnętrznym bez żadnej relacji z typem zewnętrznym.</span><span class="sxs-lookup"><span data-stu-id="2b17a-124">Inner types should not be widely reused outside of the outer type without any relationship whatsoever to the outer type.</span></span>

 <span data-ttu-id="2b17a-125">❌ nie definiują typu zagnieżdżonego jako elementu członkowskiego interfejsu.</span><span class="sxs-lookup"><span data-stu-id="2b17a-125">❌ DO NOT define a nested type as a member of an interface.</span></span> <span data-ttu-id="2b17a-126">W wielu językach nie jest obsługiwana taka konstrukcja.</span><span class="sxs-lookup"><span data-stu-id="2b17a-126">Many languages do not support such a construct.</span></span>
