# <a name="nested-types"></a><span data-ttu-id="c1bba-102">Zagnieżdżone typy</span><span class="sxs-lookup"><span data-stu-id="c1bba-102">Nested Types</span></span>
<span data-ttu-id="c1bba-103">Typ zagnieżdżony jest typ zdefiniowany w zakresie innego typu, która jest wywoływana typ otaczający.</span><span class="sxs-lookup"><span data-stu-id="c1bba-103">A nested type is a type defined within the scope of another type, which is called the enclosing type.</span></span> <span data-ttu-id="c1bba-104">Zagnieżdżony typ ma dostęp do wszystkich elementów członkowskich w jego typie otaczającym.</span><span class="sxs-lookup"><span data-stu-id="c1bba-104">A nested type has access to all members of its enclosing type.</span></span> <span data-ttu-id="c1bba-105">Na przykład ma dostęp do prywatnych pól zdefiniowanych w typie otaczającym i chronione pól zdefiniowanych w wszystkich nadrzędnych typu otaczającego.</span><span class="sxs-lookup"><span data-stu-id="c1bba-105">For example, it has access to private fields defined in the enclosing type and to protected fields defined in all ascendants of the enclosing type.</span></span>  
  
 <span data-ttu-id="c1bba-106">Ogólnie rzecz biorąc zagnieżdżone typy powinny być używane rzadko.</span><span class="sxs-lookup"><span data-stu-id="c1bba-106">In general, nested types should be used sparingly.</span></span> <span data-ttu-id="c1bba-107">Istnieje kilka przyczyn.</span><span class="sxs-lookup"><span data-stu-id="c1bba-107">There are several reasons for this.</span></span> <span data-ttu-id="c1bba-108">Niektórzy deweloperzy nie są w pełni zapoznać się z pojęciem.</span><span class="sxs-lookup"><span data-stu-id="c1bba-108">Some developers are not fully familiar with the concept.</span></span> <span data-ttu-id="c1bba-109">Tych deweloperów na przykład, Niewykluczone, problemy ze składnią zadeklarowania zmiennych typów zagnieżdżonych.</span><span class="sxs-lookup"><span data-stu-id="c1bba-109">These developers might, for example, have problems with the syntax of declaring variables of nested types.</span></span> <span data-ttu-id="c1bba-110">Zagnieżdżone typy są również bardzo ściśle powiązane z ich typ otaczający i jako takie nie są odpowiednie, jako typów ogólnego przeznaczenia.</span><span class="sxs-lookup"><span data-stu-id="c1bba-110">Nested types are also very tightly coupled with their enclosing types, and as such are not suited to be general-purpose types.</span></span>  
  
 <span data-ttu-id="c1bba-111">Zagnieżdżone typy są dopasowane do modelowania szczegóły implementacji ich typ otaczający.</span><span class="sxs-lookup"><span data-stu-id="c1bba-111">Nested types are best suited for modeling implementation details of their enclosing types.</span></span> <span data-ttu-id="c1bba-112">Użytkownik końcowy rzadko powinni być zmuszeni do deklarowania zmiennych typu zagnieżdżonego oraz prawie nigdy nie powinny mieć jawnie utworzyć typy zagnieżdżone.</span><span class="sxs-lookup"><span data-stu-id="c1bba-112">The end user should rarely have to declare variables of a nested type and almost never should have to explicitly instantiate nested types.</span></span> <span data-ttu-id="c1bba-113">Na przykład moduł wyliczający kolekcji może być zagnieżdżony typ tej kolekcji.</span><span class="sxs-lookup"><span data-stu-id="c1bba-113">For example, the enumerator of a collection can be a nested type of that collection.</span></span> <span data-ttu-id="c1bba-114">Moduły wyliczające są zwykle tworzone przez ich typ otaczający. Ponadto ponieważ wiele języków obsługują instrukcji foreach, zmiennych modułu wyliczającego rzadko muszą być zgłaszane przez użytkownika końcowego.</span><span class="sxs-lookup"><span data-stu-id="c1bba-114">Enumerators are usually instantiated by their enclosing type, and because many languages support the foreach statement, enumerator variables rarely have to be declared by the end user.</span></span>  
  
 <span data-ttu-id="c1bba-115">**✓ DO** używać zagnieżdżonych typów w przypadku relacji między typu zagnieżdżonego i jej typu zewnętrznego taki sposób, że dostępność elementu członkowskiego semantyka pożądane.</span><span class="sxs-lookup"><span data-stu-id="c1bba-115">**✓ DO** use nested types when the relationship between the nested type and its outer type is such that member-accessibility semantics are desirable.</span></span>  
  
 <span data-ttu-id="c1bba-116">**X DO NOT** Użyj publicznego typów zagnieżdżonych jako logiczne grupowanie skonstruować; Użyj przestrzeni nazw dla tego.</span><span class="sxs-lookup"><span data-stu-id="c1bba-116">**X DO NOT** use public nested types as a logical grouping construct; use namespaces for this.</span></span>  
  
 <span data-ttu-id="c1bba-117">**X AVOID** publicznie udostępnione typy zagnieżdżone.</span><span class="sxs-lookup"><span data-stu-id="c1bba-117">**X AVOID** publicly exposed nested types.</span></span> <span data-ttu-id="c1bba-118">Jedynym wyjątkiem jest, jeśli zmienne typu zagnieżdżonego musi być zadeklarowany tylko w rzadkich scenariuszach, takich jak tworzenie podklasy lub innych scenariuszy zaawansowanych dostosowań.</span><span class="sxs-lookup"><span data-stu-id="c1bba-118">The only exception to this is if variables of the nested type need to be declared only in rare scenarios such as subclassing or other advanced customization scenarios.</span></span>  
  
 <span data-ttu-id="c1bba-119">**X DO NOT** Użyj zagnieżdżone typy, jeśli typ może odwoływać się poza typu zawierającego.</span><span class="sxs-lookup"><span data-stu-id="c1bba-119">**X DO NOT** use nested types if the type is likely to be referenced outside of the containing type.</span></span>  
  
 <span data-ttu-id="c1bba-120">Na przykład wyliczenia przekazany do metody zdefiniowanej w klasie nie powinien być zdefiniowany jako typ zagnieżdżony w klasie.</span><span class="sxs-lookup"><span data-stu-id="c1bba-120">For example, an enum passed to a method defined on a class should not be defined as a nested type in the class.</span></span>  
  
 <span data-ttu-id="c1bba-121">**X DO NOT** używać zagnieżdżonych typów, jeśli muszą zostać utworzone przez kod klienta.</span><span class="sxs-lookup"><span data-stu-id="c1bba-121">**X DO NOT** use nested types if they need to be instantiated by client code.</span></span>  <span data-ttu-id="c1bba-122">Jeśli typ ma on publicznego konstruktora, prawdopodobnie nie powinny być zagnieżdżone.</span><span class="sxs-lookup"><span data-stu-id="c1bba-122">If a type has a public constructor, it should probably not be nested.</span></span>  
  
 <span data-ttu-id="c1bba-123">Jeśli można utworzyć wystąpienia typu, który wydaje się, że wskazuje typ ma miejsce w ramach samodzielnie (możesz można go utworzyć, pracować z nim i zniszczyć bez kiedykolwiek przy użyciu typu zewnętrznego), a zatem nie powinien być zagnieżdżone.</span><span class="sxs-lookup"><span data-stu-id="c1bba-123">If a type can be instantiated, that seems to indicate the type has a place in the framework on its own (you can create it, work with it, and destroy it without ever using the outer type), and thus should not be nested.</span></span> <span data-ttu-id="c1bba-124">Typy wewnętrzne nie powinna być powszechnie używana poza typu zewnętrznego bez żadnych relacji jakiejkolwiek do typu zewnętrznego.</span><span class="sxs-lookup"><span data-stu-id="c1bba-124">Inner types should not be widely reused outside of the outer type without any relationship whatsoever to the outer type.</span></span>  
  
 <span data-ttu-id="c1bba-125">**X DO NOT** zagnieżdżony typ jako element członkowski interfejsu.</span><span class="sxs-lookup"><span data-stu-id="c1bba-125">**X DO NOT** define a nested type as a member of an interface.</span></span> <span data-ttu-id="c1bba-126">Wiele języków nie obsługują takich konstrukcji.</span><span class="sxs-lookup"><span data-stu-id="c1bba-126">Many languages do not support such a construct.</span></span>
