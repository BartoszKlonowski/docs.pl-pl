# <a name="property-design"></a><span data-ttu-id="e5f6a-102">Właściwości projektu</span><span class="sxs-lookup"><span data-stu-id="e5f6a-102">Property Design</span></span>
<span data-ttu-id="e5f6a-103">Choć z technicznego punktu widzenia są bardzo podobne do metody właściwości są zupełnie różne pod względem scenariusze ich użycia.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="e5f6a-104">Powinny one widoczne, jako pola inteligentne.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-104">They should be seen as smart fields.</span></span> <span data-ttu-id="e5f6a-105">Mają one, składnia wywoływania pól i swobodne korzystanie z metod.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>  
  
 <span data-ttu-id="e5f6a-106">**✓ DO** utworzenie właściwości tylko do pobrania, jeśli element wywołujący nie ma być można zmienić wartości właściwości.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-106">**✓ DO** create get-only properties if the caller should not be able to change the value of the property.</span></span>  
  
 <span data-ttu-id="e5f6a-107">Należy pamiętać, że jeśli typ właściwość jest typu referencji zmiennej, nawet jeśli właściwość jest tylko do pobierania można zmienić wartości właściwości.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>  
  
 <span data-ttu-id="e5f6a-108">**X DO NOT** Podaj setter o dostępności szerszym niż metoda pobierająca właściwości tylko do zestawu lub właściwości.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-108">**X DO NOT** provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>  
  
 <span data-ttu-id="e5f6a-109">Na przykład nie za pomocą właściwości publicznej metody ustawiającej ani pobierającej chronionych.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-109">For example, do not use properties with a public setter and a protected getter.</span></span>  
  
 <span data-ttu-id="e5f6a-110">Jeśli metoda pobierająca właściwości nie można podać, należy zaimplementować funkcje jako metody.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="e5f6a-111">Rozważ rozpoczęcie nazwę metody z `Set` i postępuj zgodnie ze co będzie mieć nazwę właściwości.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="e5f6a-112">Na przykład <xref:System.AppDomain> ma metodę o nazwie `SetCachePath` zamiast właściwość tylko do zestawu o nazwie `CachePath`.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>  
  
 <span data-ttu-id="e5f6a-113">**✓ DO** podać za pośrednictwem domyślne wartości dla wszystkich właściwości, zapewniając, że wartości domyślne nie powodują luka w zabezpieczeniach lub poważny niewydajny kod.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-113">**✓ DO** provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>  
  
 <span data-ttu-id="e5f6a-114">**✓ DO** umożliwia właściwości można ustawić w dowolnej kolejności, nawet jeśli spowoduje to tymczasowe nieprawidłowy stan obiektu.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-114">**✓ DO** allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>  
  
 <span data-ttu-id="e5f6a-115">To częsty problem w dwóch lub więcej właściwości, aby być powiązane ze sobą do punktu, w której niektóre wartości w jedną właściwość może być nieprawidłowy podane wartości innych właściwości dla tego samego obiektu.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="e5f6a-116">W takich przypadkach wyjątki wynikłe ze nieprawidłowy stan powinien zostać odroczony do momentu wzajemnie powiązanych właściwości są rzeczywiście używane razem z obiektu.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>  
  
 <span data-ttu-id="e5f6a-117">**✓ DO** zachować poprzedniej wartości, jeśli metody ustawiającej właściwość zgłasza wyjątek.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-117">**✓ DO** preserve the previous value if a property setter throws an exception.</span></span>  
  
 <span data-ttu-id="e5f6a-118">**X AVOID** zgłaszanie wyjątków z pobierających właściwości.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-118">**X AVOID** throwing exceptions from property getters.</span></span>  
  
 <span data-ttu-id="e5f6a-119">Metody pobierające właściwości powinny być proste operacje i nie powinny mieć wszystkie warunki wstępne.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="e5f6a-120">Jeśli metoda pobierająca może zgłosić wyjątek, prawdopodobnie należy przeprojektowany jako metodę.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="e5f6a-121">Należy zauważyć, że ta zasada nie ma zastosowania do indeksatory, w którym oczekujemy, że wyjątki w wyniku sprawdzania poprawności argumentów.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>  
  
### <a name="indexed-property-design"></a><span data-ttu-id="e5f6a-122">Właściwość indeksowana projektu</span><span class="sxs-lookup"><span data-stu-id="e5f6a-122">Indexed Property Design</span></span>  
 <span data-ttu-id="e5f6a-123">Indeksowana właściwość jest właściwością specjalne, która może mieć parametrów i może być wywoływana przy użyciu specjalnej składni podobnie jak indeksowanie tablicy.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>  
  
 <span data-ttu-id="e5f6a-124">Właściwości indeksowane są często nazywane indeksatorów.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="e5f6a-125">Indeksatory należy używać tylko w przypadku interfejsów API, które zapewniają dostęp do elementów w kolekcji logiczne.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="e5f6a-126">Na przykład ciąg to zbiór znaków i indeksatora w <xref:System.String?displayProperty=nameWithType> została dodana do dostęp do jego znaków.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>  
  
 <span data-ttu-id="e5f6a-127">**✓ CONSIDER** przy użyciu indeksatorów w celu zapewnienia dostępu do danych przechowywanych w tablicy wewnętrznej.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-127">**✓ CONSIDER** using indexers to provide access to data stored in an internal array.</span></span>  
  
 <span data-ttu-id="e5f6a-128">**✓ CONSIDER** udostępnia indeksatory na typy reprezentujące kolekcje elementów.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-128">**✓ CONSIDER** providing indexers on types representing collections of items.</span></span>  
  
 <span data-ttu-id="e5f6a-129">**X AVOID** przy użyciu indeksowane właściwości z więcej niż jeden parametr.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-129">**X AVOID** using indexed properties with more than one parameter.</span></span>  
  
 <span data-ttu-id="e5f6a-130">Jeśli projekt wymaga wielu parametrów, ponowne rozpatrzenie czy właściwość reprezentuje jest naprawdę akcesora czyli logicznej kolekcji.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="e5f6a-131">Jeśli nie, należy użyć metody.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-131">If it does not, use methods instead.</span></span> <span data-ttu-id="e5f6a-132">Rozważ rozpoczęcie nazwę metody z `Get` lub `Set`.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-132">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="e5f6a-133">**X AVOID** indeksatory z typami parametrów innych niż <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, lub wyliczenia.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-133">**X AVOID** indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>  
  
 <span data-ttu-id="e5f6a-134">Jeśli projekt wymaga innych typów lub parametrów, a silnie ponownie oceń czy interfejs API reprezentuje jest naprawdę akcesora czyli logicznej kolekcji.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="e5f6a-135">Jeśli nie, należy użyć metody.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-135">If it does not, use a method.</span></span> <span data-ttu-id="e5f6a-136">Rozważ rozpoczęcie nazwę metody z `Get` lub `Set`.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-136">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="e5f6a-137">**✓ DO** Użyj nazwy `Item` dla właściwości indeksowanych, chyba że istnieje lepsze oczywiście nazwa (np. zobacz <xref:System.String.Chars%2A> właściwość `System.String`).</span><span class="sxs-lookup"><span data-stu-id="e5f6a-137">**✓ DO** use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>  
  
 <span data-ttu-id="e5f6a-138">W języku C# indeksatory są domyślnie nazwanego elementu.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="e5f6a-139"><xref:System.Runtime.CompilerServices.IndexerNameAttribute> Można dostosować tę nazwę.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>  
  
 <span data-ttu-id="e5f6a-140">**X DO NOT** zapewniają zarówno indeksatora, jak i metody, które są semantycznie równoważne.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-140">**X DO NOT** provide both an indexer and methods that are semantically equivalent.</span></span>  
  
 <span data-ttu-id="e5f6a-141">**X DO NOT** zapewniają więcej niż jednej rodziny przeciążone indeksatorów w jednym typie.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-141">**X DO NOT** provide more than one family of overloaded indexers in one type.</span></span>  
  
 <span data-ttu-id="e5f6a-142">Ta wartość jest wymuszana przez kompilator języka C#.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-142">This is enforced by the C# compiler.</span></span>  
  
 <span data-ttu-id="e5f6a-143">**X DO NOT** niestandardowy użycie właściwości indeksowanych.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-143">**X DO NOT** use nondefault indexed properties.</span></span>  
  
 <span data-ttu-id="e5f6a-144">Ta wartość jest wymuszana przez kompilator języka C#.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-144">This is enforced by the C# compiler.</span></span>  
  
### <a name="property-change-notification-events"></a><span data-ttu-id="e5f6a-145">Zdarzenia powiadomień zmiany właściwości</span><span class="sxs-lookup"><span data-stu-id="e5f6a-145">Property Change Notification Events</span></span>  
 <span data-ttu-id="e5f6a-146">Czasami przydatne jest zapewnienie zdarzenie powiadamiania użytkownika o zmianach w wartości właściwości.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="e5f6a-147">Na przykład `System.Windows.Forms.Control` zgłasza `TextChanged` zdarzeń po wartości jego `Text` właściwości została zmieniona.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>  
  
 <span data-ttu-id="e5f6a-148">**✓ CONSIDER** wywoływanie zmienić zdarzenia powiadomień po zmodyfikowaniu wartości właściwości ogólnych interfejsów API (zazwyczaj projektanta składników).</span><span class="sxs-lookup"><span data-stu-id="e5f6a-148">**✓ CONSIDER** raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>  
  
 <span data-ttu-id="e5f6a-149">W przypadku dobrej scenariusz dla użytkownika dowiedzieć się, gdy zmienia się właściwość obiektu, obiekt powinna podnieść zdarzenia powiadomienia o zmianie właściwości.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>  
  
 <span data-ttu-id="e5f6a-150">Jest mało prawdopodobne, aby być warte włożonej obciążenie, które można wywoływać zdarzenia, takiego niskiego poziomu interfejsy API, takich jak typy podstawowe lub kolekcji.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="e5f6a-151">Na przykład <xref:System.Collections.Generic.List%601> nie zainicjowałaby takie zdarzenia, gdy nowy element zostanie dodany do listy i `Count` zmiany właściwości.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>  
  
 <span data-ttu-id="e5f6a-152">**✓ CONSIDER** wywoływanie zmienić zdarzenia powiadomień za pomocą zewnętrznego wymusza po zmianie wartości właściwości.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-152">**✓ CONSIDER** raising change notification events when the value of a property changes via external forces.</span></span>  
  
 <span data-ttu-id="e5f6a-153">Jeśli wartość właściwości zmieni się za pośrednictwem niektórych siły zewnętrzne (w sposób inny niż przez wywołanie metody dla obiektu), należy zgłosić zdarzenia wskazują dla dewelopera, że wartość zmienia się i został zmieniony.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="e5f6a-154">Dobrym przykładem jest `Text` właściwość kontrolkę pola tekstowego.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="e5f6a-155">Gdy użytkownik wpisze tekst w `TextBox`, automatycznie ulega zmianie wartość właściwości.</span><span class="sxs-lookup"><span data-stu-id="e5f6a-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
