# <a name="property-design"></a><span data-ttu-id="f91de-102">Projekt właściwości</span><span class="sxs-lookup"><span data-stu-id="f91de-102">Property Design</span></span>
<span data-ttu-id="f91de-103">Chociaż właściwości są technicznie bardzo podobne do metod, są one bardzo odmienne pod względem scenariuszy użycia.</span><span class="sxs-lookup"><span data-stu-id="f91de-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="f91de-104">Powinny być one widoczne jako pola inteligentne.</span><span class="sxs-lookup"><span data-stu-id="f91de-104">They should be seen as smart fields.</span></span> <span data-ttu-id="f91de-105">Mają one składnię wywołującą pól i elastyczność metod.</span><span class="sxs-lookup"><span data-stu-id="f91de-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>

 <span data-ttu-id="f91de-106">✔️ utworzyć właściwości tylko DO pobrania, jeśli obiekt wywołujący nie powinien mieć możliwości zmiany wartości właściwości.</span><span class="sxs-lookup"><span data-stu-id="f91de-106">✔️ DO create get-only properties if the caller should not be able to change the value of the property.</span></span>

 <span data-ttu-id="f91de-107">Należy pamiętać, że jeśli typ właściwości jest modyfikowalnym typem odwołania, wartość właściwości można zmienić, nawet jeśli właściwość jest tylko do odczytu.</span><span class="sxs-lookup"><span data-stu-id="f91de-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>

 <span data-ttu-id="f91de-108">❌ Nie udostępniaj właściwości "Set-Only" ani właściwości z Setter, która ma szerszy dostęp niż metoda pobierająca.</span><span class="sxs-lookup"><span data-stu-id="f91de-108">❌ DO NOT provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>

 <span data-ttu-id="f91de-109">Na przykład nie należy używać właściwości z publiczną setter i chronioną metodę pobierającą.</span><span class="sxs-lookup"><span data-stu-id="f91de-109">For example, do not use properties with a public setter and a protected getter.</span></span>

 <span data-ttu-id="f91de-110">Jeśli nie można podać metody pobierającej właściwość, zamiast tego należy zaimplementować funkcję jako metodę.</span><span class="sxs-lookup"><span data-stu-id="f91de-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="f91de-111">Rozważ uruchomienie nazwy metody z `Set` i po nazwie właściwości.</span><span class="sxs-lookup"><span data-stu-id="f91de-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="f91de-112">Na przykład <xref:System.AppDomain> ma metodę o nazwie, która `SetCachePath` nie ma właściwości "Set-Only" `CachePath` .</span><span class="sxs-lookup"><span data-stu-id="f91de-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>

 <span data-ttu-id="f91de-113">✔️ zapewnić rozsądne wartości domyślne dla wszystkich właściwości, co gwarantuje, że wartości domyślne nie powodują, że w żadnym przypadku nie są napoważny zabezpieczenia ani nieefektywny kod.</span><span class="sxs-lookup"><span data-stu-id="f91de-113">✔️ DO provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>

 <span data-ttu-id="f91de-114">✔️ Zezwalaj na ustawianie właściwości w dowolnej kolejności, nawet jeśli spowoduje to tymczasowy nieprawidłowy stan obiektu.</span><span class="sxs-lookup"><span data-stu-id="f91de-114">✔️ DO allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>

 <span data-ttu-id="f91de-115">Jest ona wspólna dla dwóch lub więcej właściwości, które mają być powiązane z punktem, w którym niektóre wartości jednej właściwości mogą być nieprawidłowe z uwzględnieniem wartości innych właściwości w tym samym obiekcie.</span><span class="sxs-lookup"><span data-stu-id="f91de-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="f91de-116">W takich przypadkach wyjątki, które wynikają z nieprawidłowego stanu, powinny być odroczone do momentu, gdy właściwości międzypowiązane są rzeczywiście używane razem z obiektem.</span><span class="sxs-lookup"><span data-stu-id="f91de-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>

 <span data-ttu-id="f91de-117">✔️ zachować poprzedniej wartości, jeśli Metoda ustawiająca Właściwość zgłosi wyjątek.</span><span class="sxs-lookup"><span data-stu-id="f91de-117">✔️ DO preserve the previous value if a property setter throws an exception.</span></span>

 <span data-ttu-id="f91de-118">❌ UNIKAj zgłaszania wyjątków z metod pobierających właściwości.</span><span class="sxs-lookup"><span data-stu-id="f91de-118">❌ AVOID throwing exceptions from property getters.</span></span>

 <span data-ttu-id="f91de-119">Metody pobierające właściwości powinny być prostymi operacjami i nie powinny mieć żadnych warunków wstępnych.</span><span class="sxs-lookup"><span data-stu-id="f91de-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="f91de-120">Jeśli metoda pobierająca może zgłosić wyjątek, prawdopodobnie powinna zostać przeprojektowana tak, aby była metodą.</span><span class="sxs-lookup"><span data-stu-id="f91de-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="f91de-121">Należy zauważyć, że ta zasada nie ma zastosowania do indeksatorów, w przypadku których oczekujemy wyjątków w wyniku walidacji argumentów.</span><span class="sxs-lookup"><span data-stu-id="f91de-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>

### <a name="indexed-property-design"></a><span data-ttu-id="f91de-122">Projekt właściwości indeksowanej</span><span class="sxs-lookup"><span data-stu-id="f91de-122">Indexed Property Design</span></span>
 <span data-ttu-id="f91de-123">Indeksowana właściwość jest właściwością specjalną, która może mieć parametry i może być wywoływana przy użyciu specjalnej składni podobnej do indeksowania tablicy.</span><span class="sxs-lookup"><span data-stu-id="f91de-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>

 <span data-ttu-id="f91de-124">Właściwości indeksowane są często określane jako indeksatory.</span><span class="sxs-lookup"><span data-stu-id="f91de-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="f91de-125">Indeksatory powinny być używane tylko w interfejsach API, które zapewniają dostęp do elementów w kolekcji logicznej.</span><span class="sxs-lookup"><span data-stu-id="f91de-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="f91de-126">Na przykład ciąg jest kolekcją znaków, a indeksator <xref:System.String?displayProperty=nameWithType> został dodany w celu uzyskania dostępu do jego znaków.</span><span class="sxs-lookup"><span data-stu-id="f91de-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>

 <span data-ttu-id="f91de-127">✔️ Rozważ użycie indeksatorów, aby zapewnić dostęp do danych przechowywanych w tablicy wewnętrznej.</span><span class="sxs-lookup"><span data-stu-id="f91de-127">✔️ CONSIDER using indexers to provide access to data stored in an internal array.</span></span>

 <span data-ttu-id="f91de-128">✔️ Rozważ dostarczenie indeksatorów dla typów reprezentujących kolekcje elementów.</span><span class="sxs-lookup"><span data-stu-id="f91de-128">✔️ CONSIDER providing indexers on types representing collections of items.</span></span>

 <span data-ttu-id="f91de-129">❌ Należy unikać używania właściwości indeksowanych z więcej niż jednym parametrem.</span><span class="sxs-lookup"><span data-stu-id="f91de-129">❌ AVOID using indexed properties with more than one parameter.</span></span>

 <span data-ttu-id="f91de-130">Jeśli projekt wymaga wielu parametrów, należy rozważyć, czy właściwość naprawdę reprezentuje metodę dostępu do kolekcji logicznej.</span><span class="sxs-lookup"><span data-stu-id="f91de-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="f91de-131">Jeśli tak nie jest, zamiast tego użyj metod.</span><span class="sxs-lookup"><span data-stu-id="f91de-131">If it does not, use methods instead.</span></span> <span data-ttu-id="f91de-132">Rozważ uruchomienie nazwy metody z `Get` lub `Set` .</span><span class="sxs-lookup"><span data-stu-id="f91de-132">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="f91de-133">❌ Unikaj indeksatorów z typami parametrów innymi niż <xref:System.Int32?displayProperty=nameWithType> , <xref:System.Int64?displayProperty=nameWithType> , <xref:System.String?displayProperty=nameWithType> , <xref:System.Object?displayProperty=nameWithType> lub wyliczeniem.</span><span class="sxs-lookup"><span data-stu-id="f91de-133">❌ AVOID indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>

 <span data-ttu-id="f91de-134">Jeśli projekt wymaga innych typów parametrów, należy silnie oszacować, czy interfejs API naprawdę reprezentuje metodę dostępu do logicznej kolekcji.</span><span class="sxs-lookup"><span data-stu-id="f91de-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="f91de-135">Jeśli tak nie jest, użyj metody.</span><span class="sxs-lookup"><span data-stu-id="f91de-135">If it does not, use a method.</span></span> <span data-ttu-id="f91de-136">Rozważ uruchomienie nazwy metody z `Get` lub `Set` .</span><span class="sxs-lookup"><span data-stu-id="f91de-136">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="f91de-137">✔️ należy używać nazwy `Item` indeksowanych właściwości, chyba że jest to wyraźnie lepsza nazwa (np. <xref:System.String.Chars%2A> Właściwość na `System.String` ).</span><span class="sxs-lookup"><span data-stu-id="f91de-137">✔️ DO use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>

 <span data-ttu-id="f91de-138">W języku C# Indeksatory są domyślnie nazwanymi elementami.</span><span class="sxs-lookup"><span data-stu-id="f91de-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="f91de-139"><xref:System.Runtime.CompilerServices.IndexerNameAttribute>Można użyć, aby dostosować tę nazwę.</span><span class="sxs-lookup"><span data-stu-id="f91de-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>

 <span data-ttu-id="f91de-140">❌ NIE należy podawać zarówno indeksatora, jak i metod, które są semantycznie równoważne.</span><span class="sxs-lookup"><span data-stu-id="f91de-140">❌ DO NOT provide both an indexer and methods that are semantically equivalent.</span></span>

 <span data-ttu-id="f91de-141">❌ Nie udostępniaj więcej niż jednej rodziny przeciążonych indeksatorów w jednym typie.</span><span class="sxs-lookup"><span data-stu-id="f91de-141">❌ DO NOT provide more than one family of overloaded indexers in one type.</span></span>

 <span data-ttu-id="f91de-142">Jest to wymuszane przez kompilator języka C#.</span><span class="sxs-lookup"><span data-stu-id="f91de-142">This is enforced by the C# compiler.</span></span>

 <span data-ttu-id="f91de-143">❌ NIE używaj właściwości indeksowanych innych niż domyślne.</span><span class="sxs-lookup"><span data-stu-id="f91de-143">❌ DO NOT use nondefault indexed properties.</span></span>

 <span data-ttu-id="f91de-144">Jest to wymuszane przez kompilator języka C#.</span><span class="sxs-lookup"><span data-stu-id="f91de-144">This is enforced by the C# compiler.</span></span>

### <a name="property-change-notification-events"></a><span data-ttu-id="f91de-145">Zdarzenia powiadamiania o zmianie właściwości</span><span class="sxs-lookup"><span data-stu-id="f91de-145">Property Change Notification Events</span></span>
 <span data-ttu-id="f91de-146">Czasami warto podać zdarzenie powiadamiające użytkownika o zmianach w wartości właściwości.</span><span class="sxs-lookup"><span data-stu-id="f91de-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="f91de-147">Na przykład `System.Windows.Forms.Control` wywołuje `TextChanged` zdarzenie po zmianie wartości jego `Text` właściwości.</span><span class="sxs-lookup"><span data-stu-id="f91de-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>

 <span data-ttu-id="f91de-148">✔️ ROZWAŻYĆ podnoszenie poziomu powiadomień o zmianach, gdy wartości właściwości w interfejsach API wysokiego poziomu (zazwyczaj składniki projektanta) są modyfikowane.</span><span class="sxs-lookup"><span data-stu-id="f91de-148">✔️ CONSIDER raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>

 <span data-ttu-id="f91de-149">Jeśli istnieje dobry scenariusz dla użytkownika, aby wiedzieć, kiedy zmienia się właściwość obiektu, obiekt powinien zgłosić zdarzenie zmiany powiadomienia dla właściwości.</span><span class="sxs-lookup"><span data-stu-id="f91de-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>

 <span data-ttu-id="f91de-150">Nie jest jednak mało prawdopodobne, aby wywoływać te zdarzenia dla interfejsów API niskiego poziomu, takich jak typy podstawowe lub kolekcje.</span><span class="sxs-lookup"><span data-stu-id="f91de-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="f91de-151">Na przykład <xref:System.Collections.Generic.List%601> nie będzie zgłaszać takich zdarzeń, gdy nowy element zostanie dodany do listy i `Count` Zmiana właściwości.</span><span class="sxs-lookup"><span data-stu-id="f91de-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>

 <span data-ttu-id="f91de-152">✔️ ROZWAŻYĆ podnoszenie poziomu powiadomień o zmianach, gdy wartość właściwości zostanie zmieniona za pośrednictwem sił zewnętrznych.</span><span class="sxs-lookup"><span data-stu-id="f91de-152">✔️ CONSIDER raising change notification events when the value of a property changes via external forces.</span></span>

 <span data-ttu-id="f91de-153">Jeśli wartość właściwości zostanie zmieniona za pośrednictwem niektórych zewnętrznych sił (w inny sposób niż wywoływanie metod w obiekcie), zdarzenia zdarzeń wskazują deweloperowi, że wartość zmienia się i została zmieniona.</span><span class="sxs-lookup"><span data-stu-id="f91de-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="f91de-154">Dobrym przykładem jest `Text` Właściwość kontrolki pola tekstowego.</span><span class="sxs-lookup"><span data-stu-id="f91de-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="f91de-155">Gdy użytkownik wpisze tekst w `TextBox` , wartość właściwości jest automatycznie zmieniana.</span><span class="sxs-lookup"><span data-stu-id="f91de-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
