## <a name="type-promotion"></a><span data-ttu-id="aca5b-161">Promocja typu</span><span class="sxs-lookup"><span data-stu-id="aca5b-161">Type Promotion</span></span>  
 <span data-ttu-id="aca5b-162"><xref:System.Xml.Schema.XmlSchemaInference> Klasy sprawdza, czy wartości atrybutu i elementu jednego naraz.</span><span class="sxs-lookup"><span data-stu-id="aca5b-162">The <xref:System.Xml.Schema.XmlSchemaInference> class examines attribute and element values one at a time.</span></span> <span data-ttu-id="aca5b-163">Po napotkaniu wartości jest wnioskowany typ najbardziej restrykcyjne, bez znaku.</span><span class="sxs-lookup"><span data-stu-id="aca5b-163">As values are encountered, the most restrictive, unsigned type is inferred.</span></span> <span data-ttu-id="aca5b-164">Jeśli typem została wykryta dla atrybutu lub elementu, a nowa wartość okaże się, że jest niezgodny z aktualnie wnioskowany typ, wnioskowany typ zostanie podwyższony do nowego typu, który dotyczy zarówno obecnie wnioskowany typ, jak i nową wartość.</span><span class="sxs-lookup"><span data-stu-id="aca5b-164">If a type has been inferred for an attribute or element, and a new value is encountered that does not match the currently inferred type, the inferred type is promoted to a new type that applies to both the currently inferred type and the new value.</span></span> <span data-ttu-id="aca5b-165"><xref:System.Xml.Schema.XmlSchemaInference> Klasy należy wziąć pod uwagę poprzednie wartości podczas podwyższania poziomu wnioskowany typ.</span><span class="sxs-lookup"><span data-stu-id="aca5b-165">The <xref:System.Xml.Schema.XmlSchemaInference> class does consider previous values when promoting the inferred type.</span></span>  
  
 <span data-ttu-id="aca5b-166">Na przykład należy wziąć pod uwagę następujące fragmenty XML z dwóch dokumentów XML:</span><span class="sxs-lookup"><span data-stu-id="aca5b-166">For example, consider the following XML fragments from two XML documents:</span></span>

 `<MyElement1 attr1="52344" />`

 <span data-ttu-id="aca5b-170">Po pierwsze `attr2` napotkano wartość typu `attr2` jest wnioskowany jako `unsignedByte` na podstawie wartości `0`.</span><span class="sxs-lookup"><span data-stu-id="aca5b-170">When the first `attr2` value is encountered, the type of `attr2` is inferred as `unsignedByte` based on the value `0`.</span></span> <span data-ttu-id="aca5b-171">Podczas drugiego `attr2` jest napotkano typ zostanie podwyższony do `string` oparte na aktualnie wnioskowany typ `unsignedByte` i bieżącą wartość `true` ponieważ <xref:System.Xml.Schema.XmlSchemaInference> klasy należy wziąć pod uwagę poprzednie wartości podczas podwyższania poziomu wnioskowany typ.</span><span class="sxs-lookup"><span data-stu-id="aca5b-171">When the second `attr2` is encountered, the type is promoted to `string` based on the currently inferred type of `unsignedByte` and the current value `true` because the <xref:System.Xml.Schema.XmlSchemaInference> class does consider previous values when promoting the inferred type.</span></span> <span data-ttu-id="aca5b-172">Jednak jeśli oba wystąpienia elementu `attr2` napotkano w dokumencie XML, a nie w dwóch różnych dokumentów XML, jak pokazano powyżej, `attr2` może wywnioskować jako `boolean`.</span><span class="sxs-lookup"><span data-stu-id="aca5b-172">However, if both instances of `attr2` were encountered in the same XML document and not in two different XML documents as illustrated above, `attr2` would have been inferred as `boolean`.</span></span>
