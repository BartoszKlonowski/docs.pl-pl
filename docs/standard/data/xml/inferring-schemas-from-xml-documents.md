 <span data-ttu-id="7d453-112">W przykładzie powyżej, kiedy `attribute1` napotkano atrybutu o wartości `6` przez <xref:System.Xml.Schema.XmlSchemaInference> procesu, zakłada się być typu `xs:unsignedByte`.</span><span class="sxs-lookup"><span data-stu-id="7d453-112">In the example above, when the `attribute1` attribute is encountered with a value of `6` by the <xref:System.Xml.Schema.XmlSchemaInference> process, it is assumed to be of type `xs:unsignedByte`.</span></span> <span data-ttu-id="7d453-113">Podczas drugiego `parent` napotkane elementu przez <xref:System.Xml.Schema.XmlSchemaInference> procesu ograniczenie się zmniejszyć, modyfikując typu, który ma `xs:string` ponieważ wartość `attribute1` atrybut jest teraz `A`.</span><span class="sxs-lookup"><span data-stu-id="7d453-113">When the second `parent` element is encountered by the <xref:System.Xml.Schema.XmlSchemaInference> process, the constraint is loosened by modifying the type to `xs:string` because the value of the `attribute1` attribute is now `A`.</span></span> <span data-ttu-id="7d453-114">Podobnie `minOccurs` atrybutu dla wszystkich `child` wywnioskować w schemacie elementy są zmniejszyć do `minOccurs="0"` ponieważ drugi element nadrzędny nie ma elementów podrzędnych.</span><span class="sxs-lookup"><span data-stu-id="7d453-114">Similarly, the `minOccurs` attribute for all the `child` elements inferred in the schema are loosened to `minOccurs="0"` because the second parent element has no child elements.</span></span>
