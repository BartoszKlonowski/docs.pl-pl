# <a name="developing-for-multiple-platforms-with-the-net-framework"></a>Tworzenie aplikacji dla wielu platform przy użyciu programu .NET Framework

Możesz opracowywać aplikacje zarówno firmy Microsoft, jak i platform innych niż firmy Microsoft, za pomocą środowiska .NET Framework i programu Visual Studio.
  
## <a name="options-for-cross-platform-development"></a>Opcje dla wieloplatformowego opracowywania aplikacji

[!INCLUDE[standard](../../../includes/pcl-to-standard.md)]
  
 Aby opracowywanie dla wielu platform, możesz udostępniać kod źródłowy lub pliki binarne i wykonywać wywołania między kodem .NET Framework i interfejsów API środowiska wykonawczego Windows.  
  
|Jeśli chcesz...|Użyj...|  
|-----------------------|------------|  
|Udostępniać kod źródłowy między aplikacjami systemu Windows Phone 8.1 i Windows 8.1|**Projekty udostępnione** (aplikacje uniwersalne szablonu w programie Visual Studio 2013 Update 2).<br /><br /> -Aktualnie brak obsługi języka Visual Basic.<br />— Można oddzielić kodu specyficznego dla platformy przy użyciu #`if` instrukcji.<br /><br /> Aby uzyskać więcej informacji zobacz:<br /><br /> -   [Rozpocznij kodowanie](/windows/uwp/get-started/create-uwp-apps)<br />-   [Za pomocą programu Visual Studio do tworzenia aplikacji uniwersalnych XAML](https://devblogs.microsoft.com/visualstudio/using-visual-studio-to-build-universal-xaml-apps/) (wpis na blogu)<br />-   [Za pomocą programu Visual Studio do tworzenia aplikacji XAML zbieżności](https://channel9.msdn.com/Events/Build/2014/3-591) (wideo)|  
|Udostępnianie plików binarnych między aplikacjami, które przeznaczonych dla różnych platform|**Przenośne biblioteki klas projektów** dla kodu, który jest niezależny od platformy.<br /><br /> — Ta metoda jest zazwyczaj używany dla kodu, który implementuje logikę biznesową.<br />— Używając Visual Basic lub C#.<br />— Obsługa interfejsu API zależy to od platformy.<br />-Przenośnej biblioteki klas projektów dla Windows 8.1 i Windows Phone 8.1 obsługują interfejsów API środowiska wykonawczego Windows i XAML. Te funkcje nie są dostępne w starszych wersjach Portable Class Library.<br />— Jeśli to konieczne, możesz abstrakcyjna kodu specyficznego dla platformy, przy użyciu interfejsy lub abstrakcyjne klasy.<br /><br /> Aby uzyskać więcej informacji zobacz:<br /><br /> -   [Biblioteki klas przenośnych](cross-platform-development-with-the-portable-class-library.md)<br />-   [Jak pracować biblioteki klas przenośnych upewnij dla Ciebie](https://blogs.msdn.microsoft.com/dsplaisted/2012/08/27/how-to-make-portable-class-libraries-work-for-you/) (wpis na blogu)<br />-   [Korzystanie z przenośnej biblioteki klas z modelem MVVM](using-portable-class-library-with-model-view-view-model.md) <br />-   [Zasoby aplikacji dla bibliotek przeznaczonych dla wielu platform](app-resources-for-libraries-that-target-multiple-platforms.md) <br />-   [Narzędzia .NET portability Analyzer](https://marketplace.visualstudio.com/items?itemName=ConnieYau.NETPortabilityAnalyzer) (z rozszerzeniem programu Visual Studio)|  
|Udostępniać kod źródłowy między aplikacjami na platformach innych niż Windows 8.1 i Windows Phone 8.1|**Dodaj jako link** funkcji.<br /><br /> — Ta metoda jest odpowiednia dla logiki aplikacji, która jest wspólny dla obu aplikacji, ale nie jest przenośny, przyczyny. Funkcja ta jest przydatna dla języka C# lub kod języka Visual Basic.<br />     Na przykład Windows Phone 8 i Windows 8 korzystają interfejsów API środowiska wykonawczego Windows, ale przenośnej biblioteki klas nie obsługują środowiska wykonawczego Windows w przypadku tych platform. Możesz użyć `Add as link` można udostępnić wspólny kod środowiska uruchomieniowego Windows między aplikacji Windows Phone 8 oraz aplikacji Windows Store, który jest przeznaczony dla systemu Windows 8.<br /><br /> Aby uzyskać więcej informacji zobacz:<br /><br /> -   [Udostępnianie kodu za pomocą Dodaj jako Link](https://docs.microsoft.com/previous-versions/windows/apps/jj714082(v=vs.105))<br />-   [Jak: Dodaj istniejące elementy do projektu](https://docs.microsoft.com/previous-versions/visualstudio/visual-studio-2010/9f4t9t92(v=vs.100))|  
|Pisanie Windows Store apps za pomocą programu .NET Framework lub wywoływanie interfejsów API środowiska wykonawczego Windows z poziomu kodu .NET Framework|**Interfejsów API środowiska wykonawczego Windows** z kodu .NET Framework w języku C# lub Visual Basic i użyj programu .NET Framework do tworzenia aplikacji Windows Store. Należy pamiętać o interfejs API różnice między dwiema platformami. Istnieją jednak klasy, aby pomóc Ci pracować z różnic.<br /><br /> Aby uzyskać więcej informacji zobacz:<br /><br /> -   [Obsługa programu .NET framework dla aplikacji Windows Store i środowiska wykonawczego Windows](support-for-windows-store-apps-and-windows-runtime.md) <br />-   [Przekazywanie adresu URI do środowiska wykonawczego Windows](passing-a-uri-to-the-windows-runtime.md) <br />-   <xref:System.IO.WindowsRuntimeStreamExtensions><br />-    <xref:System.WindowsRuntimeSystemExtensions>|
