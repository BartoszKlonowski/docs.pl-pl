# <a name="developing-for-multiple-platforms-with-the-net-framework"></a>Tworzenie aplikacji dla wielu platform przy użyciu programu .NET Framework
Można tworzyć aplikacje firmy Microsoft i platform firmy Microsoft przy użyciu programu .NET Framework i Visual Studio.  
  
## <a name="options-for-cross-platform-development"></a>Opcje dla aplikacji dla wielu platform  
 Aby opracować dla wielu platform, można udostępniać kodu źródłowego lub plików binarnych, i może wykonywać wywołania między kodem platformy .NET Framework i interfejsów API środowiska wykonawczego systemu Windows.  
  
|Jeśli chcesz...|Użyj...|  
|-----------------------|------------|  
|Udostępnianie kodu źródłowego między aplikacjami systemu Windows Phone 8.1 i Windows 8.1|**Udostępnionych projektów** (szablon uniwersalnych aplikacji w programie Visual Studio 2013 Update 2).<br /><br /> -Aktualnie nie obsługuje języka Visual Basic.<br />— Można oddzielić specyficzne dla platformy kodu za pomocą #`if` instrukcje.<br /><br /> Aby uzyskać więcej informacji zobacz:<br /><br /> -   [Tworzenie aplikacji, które odnoszą się do systemu Windows i Windows Phone przy użyciu programu Visual Studio](https://msdn.microsoft.com/library/windows/apps/dn609832.aspx) (artykuł w witrynie MSDN)<br />-   [Za pomocą programu Visual Studio do tworzenia uniwersalnych aplikacji XAML](https://blogs.msdn.microsoft.com/visualstudio/2014/04/14/using-visual-studio-to-build-universal-xaml-apps/) (wpis w blogu)<br />-   [Za pomocą programu Visual Studio do tworzenia aplikacji XAML zbieżność](https://channel9.msdn.com/Events/Build/2014/3-591) (klip wideo)|  
|Udostępnianie plików binarnych między aplikacjami, które mają różne platformy docelowe|**Projektów biblioteki klas przenośnych** kod, który jest niezależny od platformy.<br /><br /> — Ta metoda jest zwykle używany dla kodu, który implementuje logiki biznesowej.<br />-Można używać języka Visual Basic lub C#.<br />-Obsługa interfejsu API zależy od platformy.<br />-Przenośne projektach biblioteki klas, które są przeznaczone Windows 8.1 i Windows Phone 8.1 obsługują interfejsów API środowiska wykonawczego systemu Windows i języka XAML. Te funkcje nie są dostępne w starszych wersjach przenośnej biblioteki klas.<br />— W razie potrzeby można abstrakcyjnej specyficzne dla platformy kodu za pomocą interfejsów lub klas abstrakcyjnych.<br /><br /> Aby uzyskać więcej informacji zobacz:<br /><br /> -   [Biblioteka klas przenośnych](../../../docs/standard/cross-platform/cross-platform-development-with-the-portable-class-library.md)<br />-   [Jak pracować biblioteki klasy przenośnych upewnij dla Ciebie](https://blogs.msdn.microsoft.com/dsplaisted/2012/08/27/how-to-make-portable-class-libraries-work-for-you/) (wpis w blogu)<br />-   [Przy użyciu przenośnej biblioteki klas z modelem MVVM](../../../docs/standard/cross-platform/using-portable-class-library-with-model-view-view-model.md) <br />-   [Zasoby aplikacji dla bibliotek przeznaczonych dla wielu platform](../../../docs/standard/cross-platform/app-resources-for-libraries-that-target-multiple-platforms.md) <br />-   [Analizator przenośność .NET](http://visualstudiogallery.msdn.microsoft.com/1177943e-cfb7-4822-a8a6-e56c7905292b) (rozszerzenie programu Visual Studio)|  
|Udostępnianie kodu źródłowego między aplikacjami na platformach innych niż Windows 8.1 i Windows Phone 8.1|**Dodaj jako link** funkcji.<br /><br /> — Ta metoda jest odpowiednia dla logiki aplikacji, która jest często obie aplikacje, ale nie przenośne jakiegoś powodu. Funkcja ta jest przydatna dla języka C# lub kod Visual Basic.<br />     Na przykład Windows Phone 8 i Windows 8 udostępniać interfejsów API środowiska wykonawczego systemu Windows, ale przenośnej biblioteki klas nie obsługują środowiska wykonawczego systemu Windows dla tych platform. Można użyć `Add as link` udostępnianie typowy kod środowiska uruchomieniowego systemu Windows między aplikacji Windows Phone 8 i aplikacji ze Sklepu Windows przeznaczonego dla systemu Windows 8.<br /><br /> Aby uzyskać więcej informacji zobacz:<br /><br /> -   [Udostępnianie kodu z Dodaj jako Link](http://msdn.microsoft.com/library/windowsphone/develop/jj714082\(v=vs.105\).aspx) (artykuł w witrynie MSDN)<br />-   [Porady: Dodawanie istniejących elementów do projektu](http://msdn.microsoft.com/library/vstudio/9f4t9t92\(v=vs.100\).aspx) (artykuł w witrynie MSDN)|
