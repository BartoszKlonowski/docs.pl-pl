    public int ProductId { get; private set; }

### <a name="defining-an-event-bus-interface"></a><span data-ttu-id="5f122-168">Definiowanie interfejsu magistrali zdarzeń</span><span class="sxs-lookup"><span data-stu-id="5f122-168">Defining an event bus interface</span></span>

<span data-ttu-id="5f122-169">Zacznijmy możliwe implementacji dla celów eksploracji i niektóre kod implementacji interfejsu zdarzenia magistrali.</span><span class="sxs-lookup"><span data-stu-id="5f122-169">Let’s start with some implementation code for the event bus interface and possible implementations for exploration purposes.</span></span> <span data-ttu-id="5f122-170">Interfejs powinna być rodzajowa i proste, jak interfejsu.</span><span class="sxs-lookup"><span data-stu-id="5f122-170">The interface should be generic and straightforward, as in the following interface.</span></span>

```csharp
public interface IEventBus
{
    void Publish(IntegrationEvent @event);

    void Subscribe<T, TH>()
        where T : IntegrationEvent
        where TH : IIntegrationEventHandler<T>;

    void SubscribeDynamic<TH>(string eventName)
        where TH : IDynamicIntegrationEventHandler;

    void UnsubscribeDynamic<TH>(string eventName)
        where TH : IDynamicIntegrationEventHandler;

    void Unsubscribe<T, TH>()
        where TH : IIntegrationEventHandler<T>
        where T : IntegrationEvent;
}
```

<span data-ttu-id="5f122-171">`Publish` Metoda jest prosta.</span><span class="sxs-lookup"><span data-stu-id="5f122-171">The `Publish` method is straightforward.</span></span> <span data-ttu-id="5f122-172">Magistrala zdarzeń będzie emisji zdarzeń integracji przekazanego do dowolnego mikrousługi lub nawet aplikacji zewnętrznych, subskrybowane dla tego zdarzenia.</span><span class="sxs-lookup"><span data-stu-id="5f122-172">The event bus will broadcast the integration event passed to it to any microservice, or even an external application, subscribed to that event.</span></span> <span data-ttu-id="5f122-173">Ta metoda jest używana przez mikrousługi, która publikuje zdarzenia.</span><span class="sxs-lookup"><span data-stu-id="5f122-173">This method is used by the microservice that is publishing the event.</span></span>

<span data-ttu-id="5f122-174">`Subscribe` Metody (może mieć wielu implementacji w zależności od argumentów) są używane przez mikrousług, który chcesz otrzymywać zdarzeń.</span><span class="sxs-lookup"><span data-stu-id="5f122-174">The `Subscribe` methods (you can have several implementations depending on the arguments) are used by the microservices that want to receive events.</span></span> <span data-ttu-id="5f122-175">Ta metoda ma dwa argumenty.</span><span class="sxs-lookup"><span data-stu-id="5f122-175">This method has two arguments.</span></span> <span data-ttu-id="5f122-176">Pierwsza to zdarzenie integracji subskrybować (`IntegrationEvent`).</span><span class="sxs-lookup"><span data-stu-id="5f122-176">The first is the integration event to subscribe to (`IntegrationEvent`).</span></span> <span data-ttu-id="5f122-177">Drugi argument jest integracja zdarzenia obsługi (lub metoda wywołania zwrotnego), o nazwie `IIntegrationEventHandler<T>`, która zostanie wykonana po mikrousługi odbiornik pobiera ten komunikat o zdarzeniu integracji.</span><span class="sxs-lookup"><span data-stu-id="5f122-177">The second argument is the integration event handler (or callback method), named `IIntegrationEventHandler<T>`, to be executed when the receiver microservice gets that integration event message.</span></span>
