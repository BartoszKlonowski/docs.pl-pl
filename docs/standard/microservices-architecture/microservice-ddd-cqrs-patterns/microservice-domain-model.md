### <a name="the-aggregate-pattern"></a><span data-ttu-id="441e8-176">Wzorzec agregacji</span><span class="sxs-lookup"><span data-stu-id="441e8-176">The Aggregate pattern</span></span>

<span data-ttu-id="441e8-177">Model domeny zawiera klaster jednostek różne dane i procesy, które można kontrolować istotny obszar funkcji, takich jak spełnienia kolejności lub magazynu.</span><span class="sxs-lookup"><span data-stu-id="441e8-177">A domain model contains clusters of different data entities and processes that can control a significant area of functionality, such as order fulfilment or inventory.</span></span> <span data-ttu-id="441e8-178">Do bardziej szczegółowych DDD jest funkcję agregacji, w tym artykule opisano klastra lub grupy jednostek i zachowań, które mogą być traktowane jako jednostka cohesive.</span><span class="sxs-lookup"><span data-stu-id="441e8-178">A more fine-grained DDD unit is the aggregate, which describes a cluster or group of entities and behaviors that can be treated as a cohesive unit.</span></span>

<span data-ttu-id="441e8-179">Zazwyczaj zdefiniujesz agregacją, w oparciu o transakcji, które są potrzebne.</span><span class="sxs-lookup"><span data-stu-id="441e8-179">You usually define an aggregate based on the transactions that you need.</span></span> <span data-ttu-id="441e8-180">Klasycznym przykładem jest zamówienie, zawierający listę elementów w kolejności.</span><span class="sxs-lookup"><span data-stu-id="441e8-180">A classic example is an order that also contains a list of order items.</span></span> <span data-ttu-id="441e8-181">Przedmiot zamówienia zwykle jest jednostką.</span><span class="sxs-lookup"><span data-stu-id="441e8-181">An order item will usually be an entity.</span></span> <span data-ttu-id="441e8-182">Ale będzie ona jednostce podrzędnej w agregacji kolejność, która będzie również zawierać jednostki zamówienia jego jednostkę główną, zwykle nazywane głównego agregacji.</span><span class="sxs-lookup"><span data-stu-id="441e8-182">But it will be a child entity within the order aggregate, which will also contain the order entity as its root entity, typically called an aggregate root.</span></span>

<span data-ttu-id="441e8-183">Identyfikowanie zagregowanych danych może okazać się trudne.</span><span class="sxs-lookup"><span data-stu-id="441e8-183">Identifying aggregates can be hard.</span></span> <span data-ttu-id="441e8-184">Wartość zagregowana jest grupy obiektów, które muszą być zgodne ze sobą, ale nie można po prostu wybierz grupę obiektów i nadaj im etykiety na wartość zagregowana.</span><span class="sxs-lookup"><span data-stu-id="441e8-184">An aggregate is a group of objects that must be consistent together, but you cannot just pick a group of objects and label them an aggregate.</span></span> <span data-ttu-id="441e8-185">Musi rozpoczynać się koncepcja domeny i myśleć o jednostek, które są używane w najbardziej typowych transakcje powiązane z związany z tą koncepcją.</span><span class="sxs-lookup"><span data-stu-id="441e8-185">You must start with a domain concept and think about the entities that are used in the most common transactions related to that concept.</span></span> <span data-ttu-id="441e8-186">Te jednostki, które muszą być transakcyjnie spójne to, co stanowi agregację.</span><span class="sxs-lookup"><span data-stu-id="441e8-186">Those entities that need to be transactionally consistent are what forms an aggregate.</span></span> <span data-ttu-id="441e8-187">Myśleć o operacji transakcji jest prawdopodobnie najlepszym sposobem identyfikacji agregacji.</span><span class="sxs-lookup"><span data-stu-id="441e8-187">Thinking about transaction operations is probably the best way to identify aggregates.</span></span>

### <a name="the-aggregate-root-or-root-entity-pattern"></a><span data-ttu-id="441e8-188">Wzorzec głównego agregacji lub jednostki głównej</span><span class="sxs-lookup"><span data-stu-id="441e8-188">The Aggregate Root or Root Entity pattern</span></span>

<span data-ttu-id="441e8-189">Wartość zagregowana składa się z co najmniej jedną jednostkę: głównego agregacji, nazywane również jednostki głównej lub podstawowej jednostki.</span><span class="sxs-lookup"><span data-stu-id="441e8-189">An aggregate is composed of at least one entity: the aggregate root, also called root entity or primary entity.</span></span> <span data-ttu-id="441e8-190">Ponadto może mieć wiele podrzędnych jednostek i obiektów wartości wszystkich jednostek i obiektów współpracując do zaimplementowania transakcji i wymagane zachowanie.</span><span class="sxs-lookup"><span data-stu-id="441e8-190">Additionally, it can have multiple child entities and value objects, with all entities and objects working together to implement required behavior and transactions.</span></span>

<span data-ttu-id="441e8-191">Główny agregacji ma na celu zapewnienie spójności agregacji; należy go tylko punkt wejścia dla aktualizacji do agregacji za pomocą metod lub operacji w całkowitym Klasa główna.</span><span class="sxs-lookup"><span data-stu-id="441e8-191">The purpose of an aggregate root is to ensure the consistency of the aggregate; it should be the only entry point for updates to the aggregate through methods or operations in the aggregate root class.</span></span> <span data-ttu-id="441e8-192">Należy wprowadzić zmiany do jednostki w ramach agregacji tylko przy użyciu głównego agregacji.</span><span class="sxs-lookup"><span data-stu-id="441e8-192">You should make changes to entities within the aggregate only via the aggregate root.</span></span> <span data-ttu-id="441e8-193">Jest ochrona spójności agregacji, biorąc pod uwagę invariants i zasady spójności, może być potrzebnych do wykonania w swojej agregacji.</span><span class="sxs-lookup"><span data-stu-id="441e8-193">It is the aggregate’s consistency guardian, taking into account all the invariants and consistency rules you might need to comply with in your aggregate.</span></span> <span data-ttu-id="441e8-194">Jeśli zmienisz niezależnie jednostka lub wartość obiektu podrzędnego, głównego agregacji nie upewnij się, że agregacja w prawidłowym stanie.</span><span class="sxs-lookup"><span data-stu-id="441e8-194">If you change a child entity or value object independently, the aggregate root cannot ensure that the aggregate is in a valid state.</span></span> <span data-ttu-id="441e8-195">Jest podobna do tabeli za pomocą nie będzie gałęzi.</span><span class="sxs-lookup"><span data-stu-id="441e8-195">It would be like a table with a loose leg.</span></span> <span data-ttu-id="441e8-196">Związane z utrzymaniem spójności jest głównym celem głównego agregacji.</span><span class="sxs-lookup"><span data-stu-id="441e8-196">Maintaining consistency is the main purpose of the aggregate root.</span></span>
