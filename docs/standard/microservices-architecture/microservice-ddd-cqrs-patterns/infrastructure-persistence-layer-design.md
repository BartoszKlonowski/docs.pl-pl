## <a name="the-specification-pattern"></a><span data-ttu-id="ffd11-184">Wzorzec specyfikacji</span><span class="sxs-lookup"><span data-stu-id="ffd11-184">The Specification pattern</span></span>

<span data-ttu-id="ffd11-185">Wzorzec specyfikacji (jego pełna nazwa jest wzorzec specyfikacji zapytania) jest wzorzec DDD, zaprojektowane jako miejsce, w którym można umieścić definicji zapytania za pomocą opcjonalnych, sortowanie i stronicowanie logiki.</span><span class="sxs-lookup"><span data-stu-id="ffd11-185">The specification pattern (its full name would be Query-specification pattern) is a DDD pattern designed as the place where you can put the definition of a query with optional sorting and paging logic.</span></span>

<span data-ttu-id="ffd11-186">Wzorzec specyfikacja definiuje zapytanie w obiekcie.</span><span class="sxs-lookup"><span data-stu-id="ffd11-186">The specification pattern defines a query in an object.</span></span> <span data-ttu-id="ffd11-187">Na przykład, aby hermetyzować stronicowane zapytanie, które wyszukuje niektórych produktów, należy utworzyć `PagedProduct` specyfikacja, która przyjmuje niezbędne parametry wejściowe, takie jak `pageNumber`, `pageSize`, `filter`itp. <span data-ttu-id="ffd11-188">Ma wiele zalet tego podejścia:</span><span class="sxs-lookup"><span data-stu-id="ffd11-188">There are several benefits to this approach:</span></span>
