## <a name="encapsulate-data-in-the-domain-entities"></a><span data-ttu-id="03db1-145">Hermetyzuj dane w jednostkach domeny</span><span class="sxs-lookup"><span data-stu-id="03db1-145">Encapsulate data in the Domain Entities</span></span>

<span data-ttu-id="03db1-146">Typowy problem w modelach jednostki jest eksponowanie właściwości nawigacji kolekcji jako typy list dostępny publicznie.</span><span class="sxs-lookup"><span data-stu-id="03db1-146">A common problem in entity models is that they expose collection navigation properties as publicly accessible list types.</span></span> <span data-ttu-id="03db1-147">Dzięki temu każdy Deweloper współpracownika do manipulowania zawartość tych kolekcji typów, które może pominąć reguł biznesowych ważne związanych z kolekcji, prawdopodobnie pozostawienie obiektu w nieprawidłowym stanie.</span><span class="sxs-lookup"><span data-stu-id="03db1-147">This allows any collaborator developer to manipulate the contents of these collection types, which may bypass important business rules related to the collection, possibly leaving the object in an invalid state.</span></span> <span data-ttu-id="03db1-148">Rozwiązaniem tego problemu do prezentowania dostęp tylko do odczytu do powiązanej kolekcji i jawnie określić metody, które definiują sposób, w którym klienci nimi manipulować.</span><span class="sxs-lookup"><span data-stu-id="03db1-148">The solution to this is to expose read-only access to related collections and explicitly provide methods that define ways in which clients can manipulate them.</span></span>

<span data-ttu-id="03db1-149">W poprzednim kodzie Zauważ, że wiele atrybutów są tylko do odczytu "lub" prywatnych i są tylko nadaje się do aktualizacji za pomocą metod klasy, więc dowolna aktualizacja uwzględnia invariants domeny biznesowej i logikę określoną w ramach metod klasy.</span><span class="sxs-lookup"><span data-stu-id="03db1-149">In the previous code, note that many attributes are read-only or private and are only updatable by the class methods, so any update considers business domain invariants and logic specified within the class methods.</span></span>

<span data-ttu-id="03db1-150">Na przykład, następujące wzorców DDD **należy *nie* wykonaj następujące czynności** z dowolnego polecenia obsługi metody lub aplikacji warstwy klasy (w rzeczywistości powinien to być niemożliwe, należy wykonać czynności):</span><span class="sxs-lookup"><span data-stu-id="03db1-150">For example, following DDD patterns, **you should *not* do the following** from any command handler method or application layer class (actually, it should be impossible for you to do so):</span></span>

```csharp

// COMMAND HANDLERS
// Code in command handler methods or Web API controllers
//... (WRONG) Some code with business logic out of the domain classes ...

    pictureUrl, unitPrice, discount, units);

//... (WRONG) Accessing the OrderItems colletion directly from the application layer // or command handlers
myOrder.OrderItems.Add(myNewOrderItem);
//...
```

<span data-ttu-id="03db1-151">W tym przypadku metoda Add jest wyłącznie operacji dodawania danych bezpośredni dostęp do kolekcji OrderItems.</span><span class="sxs-lookup"><span data-stu-id="03db1-151">In this case, the Add method is purely an operation to add data, with direct access to the OrderItems collection.</span></span> <span data-ttu-id="03db1-152">W związku z tym większość logika domeny, zasady lub walidacji dotyczące czy operację, używając obiektów podrzędnych będą rozkłada warstwy aplikacji (programy obsługi poleceń i kontrolerów internetowych interfejsów API).</span><span class="sxs-lookup"><span data-stu-id="03db1-152">Therefore, most of the domain logic, rules, or validations related to that operation with the child entities will be spread across the application layer (command handlers and Web API controllers).</span></span>

<span data-ttu-id="03db1-153">Po przejściu wokół głównego agregacji głównego agregacji nie gwarantuje jego invariants, jego ważności lub jego zgodności.</span><span class="sxs-lookup"><span data-stu-id="03db1-153">If you go around the aggregate root, the aggregate root cannot guarantee its invariants, its validity, or its consistency.</span></span> <span data-ttu-id="03db1-154">Po pewnym czasie będziesz mieć postaci spaghetti kod lub kod skryptu transakcji.</span><span class="sxs-lookup"><span data-stu-id="03db1-154">Eventually you will have spaghetti code or transactional script code.</span></span>

<span data-ttu-id="03db1-155">Aby skorzystać z wzorców DDD, jednostki nie może mieć publiczne metody ustawiające w dowolnej właściwości jednostki.</span><span class="sxs-lookup"><span data-stu-id="03db1-155">To follow DDD patterns, entities must not have public setters in any entity property.</span></span> <span data-ttu-id="03db1-156">Zmiany w jednostce powinien opierać się za pomocą jawnych metod za pomocą jawnego wszechobecne języka o zmiany, które wykonują w jednostce.</span><span class="sxs-lookup"><span data-stu-id="03db1-156">Changes in an entity should be driven by explicit methods with explicit ubiquitous language about the change they are performing in the entity.</span></span>
