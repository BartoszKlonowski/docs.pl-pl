# <a name="docker-terminology"></a><span data-ttu-id="6f23c-104">Terminologia docker</span><span class="sxs-lookup"><span data-stu-id="6f23c-104">Docker terminology</span></span>

<span data-ttu-id="6f23c-105">W tej sekcji wymieniono terminy i definicje, należy zapoznać się z przed pobraniem głębiej do rozwiązania Docker.</span><span class="sxs-lookup"><span data-stu-id="6f23c-105">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="6f23c-106">Dla dalszego definicje, zobacz szeroką gamę [słownik](https://docs.docker.com/v1.11/engine/reference/glossary/) dostarczonych przez Docker.</span><span class="sxs-lookup"><span data-stu-id="6f23c-106">For further definitions, see the extensive [glossary](https://docs.docker.com/v1.11/engine/reference/glossary/) provided by Docker .</span></span>

<span data-ttu-id="6f23c-107">**Kontener obrazu**: pakiet ze wszystkich zależności i informacje potrzebne do utworzenia kontenera.</span><span class="sxs-lookup"><span data-stu-id="6f23c-107">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="6f23c-108">Obraz zawiera wszystkie zależności (na przykład struktury) oraz konfiguracji wdrożenia i wykonywania mają być używane przez moduł wykonawczy kontenera.</span><span class="sxs-lookup"><span data-stu-id="6f23c-108">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="6f23c-109">Zazwyczaj obraz pochodzi z wielu obrazów podstawowej są warstwy skumulowany na siebie w celu utworzenia kontenera systemu plików.</span><span class="sxs-lookup"><span data-stu-id="6f23c-109">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container’s filesystem.</span></span> <span data-ttu-id="6f23c-110">Obraz nie można modyfikować po jego utworzeniu.</span><span class="sxs-lookup"><span data-stu-id="6f23c-110">An image is immutable once it has been created.</span></span>

<span data-ttu-id="6f23c-111">**Kontener**: wystąpienie obrazu Docker.</span><span class="sxs-lookup"><span data-stu-id="6f23c-111">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="6f23c-112">Kontener reprezentuje wykonanie jednej aplikacji, procesu lub usługi.</span><span class="sxs-lookup"><span data-stu-id="6f23c-112">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="6f23c-113">Składa się z zawartości obrazu Docker środowiska wykonawczego i standardowy zestaw instrukcji.</span><span class="sxs-lookup"><span data-stu-id="6f23c-113">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="6f23c-114">Podczas skalowania usługi, tworzy się wiele wystąpień kontenera w ten sam obraz.</span><span class="sxs-lookup"><span data-stu-id="6f23c-114">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="6f23c-115">Lub zadania wsadowego można utworzyć wiele kontenerów na podstawie takiego samego obrazu, przekazywanie różnych parametrów do każdego wystąpienia.</span><span class="sxs-lookup"><span data-stu-id="6f23c-115">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="6f23c-116">**Tag**: znak lub etykiety można stosować do obrazów, dzięki czemu można zidentyfikować różnych obrazy lub wersji tego samego obrazu (w zależności od numeru wersji lub środowisko docelowe).</span><span class="sxs-lookup"><span data-stu-id="6f23c-116">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="6f23c-117">**Plik Dockerfile**: plik tekstowy, który zawiera instrukcje dotyczące sposobu tworzenia obrazu Docker.</span><span class="sxs-lookup"><span data-stu-id="6f23c-117">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span>

<span data-ttu-id="6f23c-118">**Tworzenie**: Akcja tworzenia obrazu kontenera na podstawie informacji i kontekstu dostarczonych przez jego plik Dockerfile, a także dodatkowe pliki w folderze, w którym obraz jest tworzony.</span><span class="sxs-lookup"><span data-stu-id="6f23c-118">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="6f23c-119">Można tworzyć obrazy przy użyciu polecenia Docker docker kompilacji.</span><span class="sxs-lookup"><span data-stu-id="6f23c-119">You can build images with the Docker docker build command.</span></span>

<span data-ttu-id="6f23c-120">**Repozytorium (repozytorium)**: kolekcja powiązanych obrazy usługi Docker, etykietą tag, który wskazuje wersję obrazu.</span><span class="sxs-lookup"><span data-stu-id="6f23c-120">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="6f23c-121">Niektóre repozytoriów zawiera wiele odmian określonego obrazu, takich jak obraz zawierający zestawy SDK (większych) obraz zawierającego tylko środowisk uruchomieniowych (jaśniejszy) itp. Warianty te mogą być oznaczane przy użyciu tagów.</span><span class="sxs-lookup"><span data-stu-id="6f23c-121">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="6f23c-122">Pojedynczy repozytorium może zawierać wariantów platform, takich jak obraz systemu Linux i obrazu systemu Windows.</span><span class="sxs-lookup"><span data-stu-id="6f23c-122">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="6f23c-123">**Rejestru**: usługa, która zapewnia dostęp do repozytoriów.</span><span class="sxs-lookup"><span data-stu-id="6f23c-123">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="6f23c-124">Rejestr domyślne najbardziej publicznego obrazów jest [Centrum Docker](https://hub.docker.com/) (właściciel Docker jako organizacja).</span><span class="sxs-lookup"><span data-stu-id="6f23c-124">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="6f23c-125">Rejestr zawiera zwykle repozytoria kilka zespołów.</span><span class="sxs-lookup"><span data-stu-id="6f23c-125">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="6f23c-126">Firmy muszą często prywatnej rejestrów do przechowywania obrazów i zarządzanie nimi one utworzone.</span><span class="sxs-lookup"><span data-stu-id="6f23c-126">Companies often have private registries to store and manage images they’ve created.</span></span> <span data-ttu-id="6f23c-127">Rejestru kontenera platformy Azure jest inny przykład.</span><span class="sxs-lookup"><span data-stu-id="6f23c-127">Azure Container Registry is another example.</span></span>

<span data-ttu-id="6f23c-128">**Centrum docker**: publiczny rejestru w celu przekazania obrazów i pracować z nimi.</span><span class="sxs-lookup"><span data-stu-id="6f23c-128">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="6f23c-129">Centrum docker zapewnia Docker hosting obrazu, rejestrów publicznych lub prywatnych kompilacji wyzwalaczy i punkty zaczepienia sieci web i integracja z usługą GitHub i Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="6f23c-129">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="6f23c-130">**Azure rejestru kontenera**: publiczny zasobów do pracy z obrazy usługi Docker i jego składniki na platformie Azure.</span><span class="sxs-lookup"><span data-stu-id="6f23c-130">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="6f23c-131">Zapewnia to rejestru jest bliski wdrażanie na platformie Azure i zapewniającą kontrolę dostępu, umożliwiając przy użyciu grup usługi Azure Active Directory i uprawnienia.</span><span class="sxs-lookup"><span data-stu-id="6f23c-131">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="6f23c-132">**Docker zaufane rejestru (DTR)**: usługi rejestru A Docker (z Docker), które mogą być zainstalowane lokalnie, znajduje się w obrębie centrum danych i sieci organizacji.</span><span class="sxs-lookup"><span data-stu-id="6f23c-132">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization’s datacenter and network.</span></span> <span data-ttu-id="6f23c-133">Jest wygodną metodą prywatnej obrazów, które mają być zarządzane w przedsiębiorstwie.</span><span class="sxs-lookup"><span data-stu-id="6f23c-133">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="6f23c-134">Docker zaufane rejestru wchodzi w skład produktu Docker centrum danych.</span><span class="sxs-lookup"><span data-stu-id="6f23c-134">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="6f23c-135">Aby uzyskać więcej informacji, zobacz [Docker zaufane rejestru (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="6f23c-135">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="6f23c-136">**Docker Community Edition (CE)**: narzędzia deweloperskie dla systemu Windows i macOS służący do tworzenia, uruchamiania i testowania kontenery lokalnie.</span><span class="sxs-lookup"><span data-stu-id="6f23c-136">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="6f23c-137">CE docker dla systemu Windows zapewnia środowisk deweloperskich dla systemu Linux i kontenery systemu Windows.</span><span class="sxs-lookup"><span data-stu-id="6f23c-137">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="6f23c-138">Host Docker systemu Linux w systemie Windows jest oparta na [funkcji Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) maszyny wirtualnej.</span><span class="sxs-lookup"><span data-stu-id="6f23c-138">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) virtual machine.</span></span> <span data-ttu-id="6f23c-139">Host dla kontenerów systemu Windows opiera się bezpośrednio w systemie Windows.</span><span class="sxs-lookup"><span data-stu-id="6f23c-139">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="6f23c-140">Docker CE dla komputerów Mac jest oparta na funkcji Hypervisor Apple framework i [funkcji hypervisor xhyve](https://github.com/mist64/xhyve), zapewniające hosta maszyny wirtualnej systemu Linux Docker na Mac OS X. Docker CE dla systemu Windows i Mac zastępuje Docker przybornika, które było oparte na Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="6f23c-140">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="6f23c-141">**Docker Enterprise Edition (EE)**: wersja skali przedsiębiorstwa Docker narzędzi do tworzenia aplikacji systemu Linux i Windows.</span><span class="sxs-lookup"><span data-stu-id="6f23c-141">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="6f23c-142">**Redagowanie**: narzędzie wiersza polecenia i yaml programu pliku w formacie z metadanymi do definiowania i uruchamiania usługi kontenera aplikacji.</span><span class="sxs-lookup"><span data-stu-id="6f23c-142">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="6f23c-143">Należy zdefiniować pojedynczej aplikacji oparte na wiele obrazów z jednego lub więcej plików .yml, które można zastąpić wartości w zależności od środowiska.</span><span class="sxs-lookup"><span data-stu-id="6f23c-143">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="6f23c-144">Po utworzeniu definicje, możesz wdrożyć aplikację całego kontenera usługi za pomocą jednego polecenia (docker-tworzą w górę) tworzącą kontener na obraz na hoście Docker.</span><span class="sxs-lookup"><span data-stu-id="6f23c-144">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="6f23c-145">**Klaster**: zbiór hostów Docker widoczne tak, jakby była pojedynczego wirtualnego hosta Docker, dzięki czemu aplikacja można skalować do wielu wystąpień usług rozprzestrzeniające się na wielu hostach w klastrze.</span><span class="sxs-lookup"><span data-stu-id="6f23c-145">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="6f23c-146">Docker klastrów mogą być tworzone z Docker Swarm, Mesosphere DC/OS Kubernetes i sieci szkieletowej usług Azure.</span><span class="sxs-lookup"><span data-stu-id="6f23c-146">Docker clusters can be created with Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="6f23c-147">(Jeśli używasz Docker Swarm do zarządzania klastrem, zwykle odwołujesz się do klastra jako *swarm* zamiast klastra.)</span><span class="sxs-lookup"><span data-stu-id="6f23c-147">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

<span data-ttu-id="6f23c-148">**Orchestrator**: narzędzie, które upraszcza zarządzanie klastrów i hostów Docker.</span><span class="sxs-lookup"><span data-stu-id="6f23c-148">**Orchestrator**: A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="6f23c-149">Orchestrators umożliwiają zarządzanie obrazów, kontenery i hostami za pomocą interfejsu wiersza polecenia (CLI) lub graficznego interfejsu użytkownika.</span><span class="sxs-lookup"><span data-stu-id="6f23c-149">Orchestrators enable you to manage their images, containers, and hosts through a command line interface (CLI) or a graphical UI.</span></span> <span data-ttu-id="6f23c-150">Można zarządzać kontenera sieci, konfiguracji równoważenia obciążenia, odnajdywania usługi, wysoką dostępność, Konfiguracja hosta Docker i więcej.</span><span class="sxs-lookup"><span data-stu-id="6f23c-150">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="6f23c-151">Orchestrator jest odpowiedzialny za uruchomione, dystrybucja skalowanie i naprawianie obciążeń przez zbiór węzłów.</span><span class="sxs-lookup"><span data-stu-id="6f23c-151">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="6f23c-152">Zazwyczaj produkty programu orchestrator są te same produkty, które udostępniają infrastrukturę klastra, takie jak Mesosphere DC/OS, Kubernetes Docker Swarm i sieci szkieletowej usług Azure.</span><span class="sxs-lookup"><span data-stu-id="6f23c-152">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
