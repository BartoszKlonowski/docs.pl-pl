<span data-ttu-id="1525f-108">Rysunek 4-20 pokazuje prostszej metody tylko używania mikrousług z aplikacji klienckiej monolitycznego.</span><span class="sxs-lookup"><span data-stu-id="1525f-108">Figure 4-20 shows the simpler approach of just consuming microservices from a monolithic client application.</span></span> <span data-ttu-id="1525f-109">Oczywiście może mieć usługi platformy ASP.NET MVC w zakresie od tworzenia kodu HTML i JavaScript.</span><span class="sxs-lookup"><span data-stu-id="1525f-109">Of course, you could have an ASP.NET MVC service in between producing the HTML and JavaScript.</span></span> <span data-ttu-id="1525f-110">Rysunek jest uproszczeniem wyróżniający się, że masz jednego klienta (monolityczną) korzystanie z mikrousług, która po prostu skoncentrować się na logikę i dane, a nie na kształt interfejsu użytkownika (HTML i JavaScript) w interfejsie użytkownika.</span><span class="sxs-lookup"><span data-stu-id="1525f-110">The figure is a simplification that highlights that you have a single (monolithic) client UI consuming the microservices, which just focus on logic and data and not on the UI shape (HTML and JavaScript).</span></span>

<span data-ttu-id="1525f-116">W czasie uruchamiania aplikacji klienckich poszczególne składniki interfejsu użytkownika klienta (na przykład klasy TypeScript) rejestruje się przy użyciu mikrousług infrastruktury w stanie dostarczać modele widoków dla danego scenariusza.</span><span class="sxs-lookup"><span data-stu-id="1525f-116">At client application start-up time, each of the client UI components (TypeScript classes, for example) registers itself with an infrastructure microservice capable of providing ViewModels for a given scenario.</span></span> <span data-ttu-id="1525f-117">Mikrousługi zmienia kształt, interfejs użytkownika zostanie również zmiany.</span><span class="sxs-lookup"><span data-stu-id="1525f-117">If the microservice changes the shape, the UI changes also.</span></span>

<span data-ttu-id="1525f-124">Złożone podejście interfejsu użytkownika, który jest wymuszany przez mikrousług może być trudniejszym wyzwaniem lub mniej tak, w zależności od jakich technologii interfejsu użytkownika używasz.</span><span class="sxs-lookup"><span data-stu-id="1525f-124">A composite UI approach that is driven by microservices can be more challenging or less so, depending on what UI technologies you are using.</span></span> <span data-ttu-id="1525f-125">Na przykład nie będzie używać tych samych technik do tworzenia aplikacji sieci web tradycyjnych, którego używasz do tworzenia SPA lub dla natywnych aplikacji mobilnych (tak jak podczas opracowywania aplikacji platformy Xamarin, które mogą być trudniejsze dla tej metody).</span><span class="sxs-lookup"><span data-stu-id="1525f-125">For instance, you will not use the same techniques for building a traditional web application that you use for building an SPA or for native mobile app (as when developing Xamarin apps, which can be more challenging for this approach).</span></span>
