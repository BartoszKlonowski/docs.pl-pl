<span data-ttu-id="7f973-109">Rysunek 4-20 pokazuje prostsze eksploatującego właśnie mikrousług z wbudowanymi klienta aplikacji.</span><span class="sxs-lookup"><span data-stu-id="7f973-109">Figure 4-20 shows the simpler approach of just consuming microservices from a monolithic client application.</span></span> <span data-ttu-id="7f973-110">Oczywiście może zostać usługi platformy ASP.NET MVC Between produkujących HTML i JavaScript.</span><span class="sxs-lookup"><span data-stu-id="7f973-110">Of course, you could have an ASP.NET MVC service in between producing the HTML and JavaScript.</span></span> <span data-ttu-id="7f973-111">Liczba jest uproszczenie, który wyróżnia się, że masz jednego klienta (wbudowanymi) mikrousług, który właśnie skupić się na logikę i dane, a nie na kształtu interfejsu użytkownika (HTML i JavaScript) korzystanie z interfejsu użytkownika.</span><span class="sxs-lookup"><span data-stu-id="7f973-111">The figure is a simplification that highlights that you have a single (monolithic) client UI consuming the microservices, which just focus on logic and data and not on the UI shape (HTML and JavaScript).</span></span>

<span data-ttu-id="7f973-114">Z kolei złożonego interfejsu użytkownika jest dokładnie generowane i składane przez mikrousług, samodzielnie.</span><span class="sxs-lookup"><span data-stu-id="7f973-114">In contrast, a composite UI is precisely generated and composed by the microservices themselves.</span></span> <span data-ttu-id="7f973-115">Niektóre mikrousług stacji visual kształt określonych obszarów interfejsu użytkownika.</span><span class="sxs-lookup"><span data-stu-id="7f973-115">Some of the microservices drive the visual shape of specific areas of the UI.</span></span> <span data-ttu-id="7f973-116">Najważniejsza różnica polega na składniki interfejsu użytkownika klienta (na przykład klasy TS) na podstawie szablonów, czy ViewModel danych kształtowania-interfejsu użytkownika dla tych szablonów pochodzi z każdym mikrousługi.</span><span class="sxs-lookup"><span data-stu-id="7f973-116">The key difference is that you have client UI components (TS classes, for example) based on templates, and the data-shaping-UI ViewModel for those templates comes from each microservice.</span></span>

<span data-ttu-id="7f973-117">W momencie uruchamiania aplikacji klienta poszczególne składniki interfejsu użytkownika klienta (na przykład klasy TypeScript) rejestruje się o stanie dostarczać ViewModels w danym scenariuszu mikrousługi infrastruktury.</span><span class="sxs-lookup"><span data-stu-id="7f973-117">At client application start-up time, each of the client UI components (TypeScript classes, for example) registers itself with an infrastructure microservice capable of providing ViewModels for a given scenario.</span></span> <span data-ttu-id="7f973-118">W przypadku mikrousługi zmiany kształtu, również zmiany interfejsu użytkownika.</span><span class="sxs-lookup"><span data-stu-id="7f973-118">If the microservice changes the shape, the UI changes also.</span></span>

<span data-ttu-id="7f973-119">Rysunek 4-21 zawiera wersję tego podejścia złożonego interfejsu użytkownika.</span><span class="sxs-lookup"><span data-stu-id="7f973-119">Figure 4-21 shows a version of this composite UI approach.</span></span> <span data-ttu-id="7f973-120">To jest uproszczone, ponieważ może mieć inne mikrousług, który agregowania szczegółowego części oparte na różnych technik — zależy on od jest konstruowany podejście tradycyjnych sieci web (platformy ASP.NET MVC) lub SPA (jednej strony aplikacji).</span><span class="sxs-lookup"><span data-stu-id="7f973-120">This is simplified, because you might have other microservices that are aggregating granular parts based on different techniques—it depends on whether you are building a traditional web approach (ASP.NET MVC) or an SPA (Single Page Application).</span></span>

<span data-ttu-id="7f973-125">Złożone podejście interfejsu użytkownika, który jest wymuszany przez mikrousług może być wyzwaniem więcej lub mniej tak, w zależności od tego, jakie technologii interfejsu użytkownika używasz.</span><span class="sxs-lookup"><span data-stu-id="7f973-125">A composite UI approach that is driven by microservices can be more challenging or less so, depending on what UI technologies you are using.</span></span> <span data-ttu-id="7f973-126">Na przykład nie będzie używać tych samych metod do tworzenia aplikacji sieci web tradycyjnych, używanej do tworzenia SPA lub natywnych aplikacji mobilnej (tak jak podczas tworzenia aplikacji platformy Xamarin, które mogą być trudniejsze dla tej metody).</span><span class="sxs-lookup"><span data-stu-id="7f973-126">For instance, you will not use the same techniques for building a traditional web application that you use for building an SPA or for native mobile app (as when developing Xamarin apps, which can be more challenging for this approach).</span></span>
