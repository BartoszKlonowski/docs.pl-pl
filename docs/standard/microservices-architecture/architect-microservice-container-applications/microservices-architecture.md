# <a name="microservices-architecture"></a><span data-ttu-id="593fb-103">Architektura Mikrousług</span><span class="sxs-lookup"><span data-stu-id="593fb-103">Microservices architecture</span></span>

<span data-ttu-id="593fb-104">Jak wskazuje nazwę, architektura mikrousług jest podejście do tworzenia aplikacji serwera jako zestaw usług mała.</span><span class="sxs-lookup"><span data-stu-id="593fb-104">As the name implies, a microservices architecture is an approach to building a server application as a set of small services.</span></span> <span data-ttu-id="593fb-105">Każda usługa jest uruchamiana we własnym procesie i komunikuje się z innymi procesami przy użyciu protokołów, takich jak HTTP/HTTPS, Websocket, lub [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span><span class="sxs-lookup"><span data-stu-id="593fb-105">Each service runs in its own process and communicates with other processes using protocols such as HTTP/HTTPS, WebSockets, or [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span></span> <span data-ttu-id="593fb-106">Każdy mikrousługi implementuje określonej domeny end-to-end lub możliwości biznesowe w obrębie granicy kontekstu oraz poszczególnych musi być opracowana autonomicznie i możliwych do wdrożenia niezależnie.</span><span class="sxs-lookup"><span data-stu-id="593fb-106">Each microservice implements a specific end-to-end domain or business capability within a certain context boundary, and each must be developed autonomously and be deployable independently.</span></span> <span data-ttu-id="593fb-107">Na koniec każdego mikrousługi powinien być właścicielem jego modelu danych powiązanych domeny i logika domeny (suwerenności i zarządzanie zdecentralizowane danych) na podstawie technologii magazynowania danych (SQL, NoSQL) i różnych języków programowania.</span><span class="sxs-lookup"><span data-stu-id="593fb-107">Finally, each microservice should own its related domain data model and domain logic (sovereignty and decentralized data management) based on different data storage technologies (SQL, NoSQL) and different programming languages.</span></span>

<span data-ttu-id="593fb-108">Jaki rozmiar powinna być mikrousługi?</span><span class="sxs-lookup"><span data-stu-id="593fb-108">What size should a microservice be?</span></span> <span data-ttu-id="593fb-109">Podczas tworzenia mikrousługi, rozmiar nie może być istotne.</span><span class="sxs-lookup"><span data-stu-id="593fb-109">When developing a microservice, size should not be the important point.</span></span> <span data-ttu-id="593fb-110">Zamiast tego istotne należy utworzyć słabo sprzężona usług, więc autonomię tworzenia, wdrażania i skalowania dla każdej usługi.</span><span class="sxs-lookup"><span data-stu-id="593fb-110">Instead, the important point should be to create loosely coupled services so you have autonomy of development, deployment, and scale, for each service.</span></span> <span data-ttu-id="593fb-111">Oczywiście podczas identyfikacji i mikrousług projektowania, należy spróbować były możliwie najmniejsze tak długo, jak nie ma zbyt wiele zależności bezpośrednich z innych mikrousług.</span><span class="sxs-lookup"><span data-stu-id="593fb-111">Of course, when identifying and designing microservices, you should try to make them as small as possible as long as you do not have too many direct dependencies with other microservices.</span></span> <span data-ttu-id="593fb-112">Większe znaczenie niż rozmiar mikrousługi spójności wewnętrznego, który musi mieć i jego niezależność od innych usług.</span><span class="sxs-lookup"><span data-stu-id="593fb-112">More important than the size of the microservice is the internal cohesion it must have and its independence from other services.</span></span>

<span data-ttu-id="593fb-113">Dlaczego architektura mikrousług?</span><span class="sxs-lookup"><span data-stu-id="593fb-113">Why a microservices architecture?</span></span> <span data-ttu-id="593fb-114">Krótko mówiąc zapewnia elastyczność długoterminowej.</span><span class="sxs-lookup"><span data-stu-id="593fb-114">In short, it provides long-term agility.</span></span> <span data-ttu-id="593fb-115">Mikrousług włączyć lepsze utrzymanie w systemach złożonych, duże i wysoko skalowane przez umożliwienie tworzenia aplikacji z wielu usług niezależnie do wdrożenia że mieć cykle szczegółowego i autonomicznego.</span><span class="sxs-lookup"><span data-stu-id="593fb-115">Microservices enable better maintainability in complex, large, and highly-scalable systems by letting you create applications based on many independently deployable services that each have granular and autonomous lifecycles.</span></span>

<span data-ttu-id="593fb-116">Dodatkowa korzyść mikrousług można skalować w poziomie niezależnie.</span><span class="sxs-lookup"><span data-stu-id="593fb-116">As an additional benefit, microservices can scale out independently.</span></span> <span data-ttu-id="593fb-117">Zamiast aplikacją wbudowanymi pojedynczego musi skalowanie w poziomie jako jednostki, możesz zamiast tego skalować w poziomie określonych mikrousług.</span><span class="sxs-lookup"><span data-stu-id="593fb-117">Instead of having a single monolithic application that you must scale out as a unit, you can instead scale out specific microservices.</span></span> <span data-ttu-id="593fb-118">W ten sposób można skalować tylko funkcjonalności wymaga więcej przetwarzania zasilania lub sieci przepustowość do obsługi żądanie, a nie skalowania innych obszarach aplikacji, które nie muszą być skalowane obszaru.</span><span class="sxs-lookup"><span data-stu-id="593fb-118">That way, you can scale just the functional area that needs more processing power or network bandwidth to support demand, rather than scaling out other areas of the application that do not need to be scaled.</span></span> <span data-ttu-id="593fb-119">Oznacza to, że oszczędności, ponieważ potrzebna mniej sprzętu.</span><span class="sxs-lookup"><span data-stu-id="593fb-119">That means cost savings because you need less hardware.</span></span>

<span data-ttu-id="593fb-123">Zaprojektowanie szczegółowych aplikacji opartych na mikrousług umożliwia ciągłej integracji i rozwiązań ciągłego dostarczania.</span><span class="sxs-lookup"><span data-stu-id="593fb-123">Architecting fine-grained microservices-based applications enables continuous integration and continuous delivery practices.</span></span> <span data-ttu-id="593fb-124">Przyspiesza również dostarczania nowych funkcji, do aplikacji.</span><span class="sxs-lookup"><span data-stu-id="593fb-124">It also accelerates delivery of new functions into the application.</span></span> <span data-ttu-id="593fb-125">Szczegółowe kompozycji aplikacji umożliwia również uruchamianie i testowanie mikrousług w izolacji i rozwijać je samodzielnie, przy zachowaniu wyczyść kontraktów między nimi.</span><span class="sxs-lookup"><span data-stu-id="593fb-125">Fine-grained composition of applications also allows you to run and test microservices in isolation, and to evolve them autonomously while maintaining clear contracts between them.</span></span> <span data-ttu-id="593fb-126">Tak długo, jak nie zmienisz interfejsów lub umowy, można zmienić wewnętrzny wykonania dowolnego mikrousługi lub dodawanie nowych funkcji bez przerywania innych mikrousług.</span><span class="sxs-lookup"><span data-stu-id="593fb-126">As long as you do not change the interfaces or contracts, you can change the internal implementation of any microservice or add new functionality without breaking other microservices.</span></span>

-   <span data-ttu-id="593fb-128">Monitorowanie i kondycji kontroli usług i infrastruktury.</span><span class="sxs-lookup"><span data-stu-id="593fb-128">Monitoring and health checks of the services and infrastructure.</span></span>

-   <span data-ttu-id="593fb-129">Skalowalnej infrastruktury usług (czyli chmury i orchestrators).</span><span class="sxs-lookup"><span data-stu-id="593fb-129">Scalable infrastructure for the services (that is, cloud and orchestrators).</span></span>

-   <span data-ttu-id="593fb-130">Zabezpieczenia projektowania i wdrażania na wielu poziomach: uwierzytelniania, autoryzacji, zarządzanie klucze tajne, bezpiecznej komunikacji, itp.</span><span class="sxs-lookup"><span data-stu-id="593fb-130">Security design and implementation at multiple levels: authentication, authorization, secrets management, secure communication, etc.</span></span>

-   <span data-ttu-id="593fb-131">Dostarczanie szybkie aplikacji, zwykle za pomocą różnych zespołów koncentrujących się na różnych mikrousług.</span><span class="sxs-lookup"><span data-stu-id="593fb-131">Rapid application delivery, usually with different teams focusing on different microservices.</span></span>

-   <span data-ttu-id="593fb-132">DevOps i CI/CD rozwiązań i infrastruktury.</span><span class="sxs-lookup"><span data-stu-id="593fb-132">DevOps and CI/CD practices and infrastructure.</span></span>
