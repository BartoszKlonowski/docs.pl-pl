## <a name="container-design-principles"></a><span data-ttu-id="5e514-113">Zasady projektowania kontenera</span><span class="sxs-lookup"><span data-stu-id="5e514-113">Container design principles</span></span>

<span data-ttu-id="5e514-116">Podczas projektowania obraz kontenera, zobaczysz [punktu wejścia](https://docs.docker.com/engine/reference/builder/#entrypoint) definicję w pliku Dockerfile.</span><span class="sxs-lookup"><span data-stu-id="5e514-116">When you design a container image, you'll see an [ENTRYPOINT](https://docs.docker.com/engine/reference/builder/#entrypoint) definition in the Dockerfile.</span></span> <span data-ttu-id="5e514-117">Definiuje proces, którego okres istnienia określa okres istnienia kontenera.</span><span class="sxs-lookup"><span data-stu-id="5e514-117">This defines the process whose lifetime controls the lifetime of the container.</span></span> <span data-ttu-id="5e514-118">Po zakończeniu procesu, kończy się cyklu życia kontenera.</span><span class="sxs-lookup"><span data-stu-id="5e514-118">When the process completes, the container lifecycle ends.</span></span> <span data-ttu-id="5e514-119">Kontenery może reprezentować długotrwałe procesy takie jak serwery sieci web, ale można również reprezentują krótkotrwałe procesy, takie jak zadania usługi batch, które wcześniej może zostały zaimplementowane jako Azure [WebJobs](https://github.com/Azure/azure-webjobs-sdk/wiki).</span><span class="sxs-lookup"><span data-stu-id="5e514-119">Containers might represent long-running processes like web servers, but can also represent short-lived processes like batch jobs, which formerly might have been implemented as Azure [WebJobs](https://github.com/Azure/azure-webjobs-sdk/wiki).</span></span>

<span data-ttu-id="5e514-120">Jeśli proces zakończy się niepowodzeniem, kończy się kontenera i koordynatora przejmuje.</span><span class="sxs-lookup"><span data-stu-id="5e514-120">If the process fails, the container ends, and the orchestrator takes over.</span></span> <span data-ttu-id="5e514-121">Jeśli koordynatora został skonfigurowany do zachowania pięć wystąpień i zawiedzie, koordynatora utworzy innego wystąpienia kontenera w celu zastąpienia procesu nie powiodło się.</span><span class="sxs-lookup"><span data-stu-id="5e514-121">If the orchestrator was configured to keep five instances running and one fails, the orchestrator will create another container instance to replace the failed process.</span></span> <span data-ttu-id="5e514-122">W ramach zadania usługi batch proces zostanie uruchomiony przy użyciu parametrów.</span><span class="sxs-lookup"><span data-stu-id="5e514-122">In a batch job, the process is started with parameters.</span></span> <span data-ttu-id="5e514-123">Po zakończeniu tego procesu roboczego zostało ukończone.</span><span class="sxs-lookup"><span data-stu-id="5e514-123">When the process completes, the work is complete.</span></span> <span data-ttu-id="5e514-124">Niniejsze wskazówki awarii rozwijaną koordynatorów później.</span><span class="sxs-lookup"><span data-stu-id="5e514-124">This guidance drills-down on orchestrators, later on.</span></span>

<span data-ttu-id="5e514-125">Może się okazać scenariusz, w którym ma się wiele procesów w jeden kontener.</span><span class="sxs-lookup"><span data-stu-id="5e514-125">You might find a scenario where you want multiple processes running in a single container.</span></span> <span data-ttu-id="5e514-126">W tym scenariuszu ponieważ może istnieć tylko jeden wpis punktu na kontener, może uruchomić skrypt w kontenerze, który uruchamia programy tyle zgodnie z potrzebami.</span><span class="sxs-lookup"><span data-stu-id="5e514-126">For that scenario, since there can be only one entry point per container, you could run a script within the container that launches as many programs as needed.</span></span> <span data-ttu-id="5e514-127">Na przykład, można użyć [nadzorca](http://supervisord.org/) lub podobnego narzędzia, aby zadbać o uruchamianie wielu procesów w jednym kontenerze.</span><span class="sxs-lookup"><span data-stu-id="5e514-127">For example, you can use [Supervisor](http://supervisord.org/) or a similar tool to take care of launching multiple processes inside a single container.</span></span> <span data-ttu-id="5e514-128">Jednak mimo że można znaleźć architektur, zawierających wiele procesów na kontener, takie podejście nie jest bardzo często.</span><span class="sxs-lookup"><span data-stu-id="5e514-128">However, even though you can find architectures that hold multiple processes per container, that approach isn't very common.</span></span>
