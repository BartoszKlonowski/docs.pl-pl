## <a name="behavioral-change"></a><span data-ttu-id="1d500-107">Zmiana zachowania</span><span class="sxs-lookup"><span data-stu-id="1d500-107">Behavioral change</span></span>

<span data-ttu-id="1d500-110">Gdy zmiany behawioralne są widoczne na zewnątrz i modyfikują publiczną umowę typu, można je łatwo oszacować, ponieważ wpływają na zgodność binarną.</span><span class="sxs-lookup"><span data-stu-id="1d500-110">When behavioral changes are externally visible and modify a type's public contract, they are easy to evaluate since they affect binary compatibility.</span></span> <span data-ttu-id="1d500-111">Zmiany implementacji są znacznie trudniejsze do oszacowania; w zależności od rodzaju zmiany i częstotliwości i wzorców używania interfejsu API, wpływ zmiany może przedziały od pozostałej do niewielkiej ilości.</span><span class="sxs-lookup"><span data-stu-id="1d500-111">Implementation changes are much more difficult to evaluate; depending on the nature of the change and the frequency and patterns of use of the API, the impact of a change can range from severe to innocuous.</span></span>  

## <a name="binary-compatibility"></a><span data-ttu-id="1d500-112">Zgodność binarna</span><span class="sxs-lookup"><span data-stu-id="1d500-112">Binary compatibility</span></span>

<span data-ttu-id="1d500-113">Zgodność binarna odnosi się do zdolności konsumenta interfejsu API do korzystania z interfejsu API w nowszej wersji bez ponownej kompilacji.</span><span class="sxs-lookup"><span data-stu-id="1d500-113">Binary compatibility refers to the ability of a consumer of an API to use the API on a newer version without recompilation.</span></span> <span data-ttu-id="1d500-114">Takie zmiany jak dodanie metod lub dodanie nowej implementacji interfejsu do typu nie wpływa na zgodność binarną.</span><span class="sxs-lookup"><span data-stu-id="1d500-114">Such changes as adding methods or adding a new interface implementation to a type do not affect binary compatibility.</span></span> <span data-ttu-id="1d500-115">Jednak usunięcie lub zmiana podpisów publicznych zestawu w taki sposób, aby klienci nie mogli już uzyskiwać dostępu do tego samego interfejsu uwidocznionego przez zestaw, ma wpływ na zgodność binarną.</span><span class="sxs-lookup"><span data-stu-id="1d500-115">However, removing or altering an assembly's public signatures so that consumers can no longer access the same interface exposed by the assembly does affect binary compatibility.</span></span> <span data-ttu-id="1d500-116">Zmiana tego rodzaju jest określana jako *binarna niezgodna zmiana*.</span><span class="sxs-lookup"><span data-stu-id="1d500-116">A change of this kind is termed a *binary incompatible change*.</span></span>

## <a name="source-compatibility"></a><span data-ttu-id="1d500-117">Zgodność ze źródłem</span><span class="sxs-lookup"><span data-stu-id="1d500-117">Source compatibility</span></span>

 <span data-ttu-id="1d500-118">Zgodność ze źródłem dotyczy możliwości istniejących odbiorców interfejsu API w celu ponownego skompilowania do nowszej wersji bez żadnych zmian źródłowych.</span><span class="sxs-lookup"><span data-stu-id="1d500-118">Source compatibility refers to the ability of existing consumers of an API to recompile against a newer version without any source changes.</span></span> <span data-ttu-id="1d500-119">*Niezgodna zmiana źródła* występuje, gdy konsument musi zmodyfikować kod źródłowy, aby pomyślnie skompilować go do nowszej wersji interfejsu API.</span><span class="sxs-lookup"><span data-stu-id="1d500-119">A *source incompatible change* occurs when a consumer needs to modify source code for it to build successfully against a newer version of an API.</span></span>

## <a name="design-time-compatibility"></a><span data-ttu-id="1d500-120">Zgodność czasu projektowania</span><span class="sxs-lookup"><span data-stu-id="1d500-120">Design-time compatibility</span></span>

<span data-ttu-id="1d500-121">Zgodność czasu projektowania odnosi się do zachowania środowiska czasu projektowania między wersjami programu Visual Studio i innymi środowiskami czasu projektowania.</span><span class="sxs-lookup"><span data-stu-id="1d500-121">Design-time compatibility refers to preserving the design-time experience across versions of Visual Studio and other design-time environments.</span></span> <span data-ttu-id="1d500-122">Chociaż może to wiązać się z zachowaniem lub interfejsem użytkownika projektantów, najważniejszym aspektem zgodności w czasie projektowania jest zgodność projektów.</span><span class="sxs-lookup"><span data-stu-id="1d500-122">While this can involve the behavior or the UI of designers, the most important aspect of design-time compatibility concerns project compatibility.</span></span> <span data-ttu-id="1d500-123">Projekt lub rozwiązanie musi być możliwe do otwarcia i użycia w nowszej wersji środowiska czasu projektowania.</span><span class="sxs-lookup"><span data-stu-id="1d500-123">A project or solution must be able to be opened and used on a newer version of the design-time environment.</span></span>

## <a name="backwards-compatibility"></a><span data-ttu-id="1d500-124">Zgodność z poprzednimi wersjami</span><span class="sxs-lookup"><span data-stu-id="1d500-124">Backwards compatibility</span></span>

<span data-ttu-id="1d500-125">Zgodność z poprzednimi wersjami odnosi się do zdolności klienta interfejsu API do uruchamiania w nowej wersji, zachowując się w ten sam sposób.</span><span class="sxs-lookup"><span data-stu-id="1d500-125">Backwards compatibility refers to the ability of an existing consumer of an API to run against a new version while behaving in the same way.</span></span> <span data-ttu-id="1d500-126">Obie zmiany w zachowaniu i zmiany w zgodności binarnej wpływają na zgodność z poprzednimi wersjami.</span><span class="sxs-lookup"><span data-stu-id="1d500-126">Both behavioral changes and changes in binary compatibility affect backwards compatibility.</span></span> <span data-ttu-id="1d500-127">Jeśli klient nie może działać lub działa inaczej w przypadku uruchamiania w nowszej wersji interfejsu API, interfejs API nie jest *zgodny z poprzednimi*wersjami.</span><span class="sxs-lookup"><span data-stu-id="1d500-127">If a consumer is not able to run or behaves differently when running against the newer version of the API, the API is *backwards incompatible*.</span></span>

<span data-ttu-id="1d500-128">Zmiany wpływające na zgodność z poprzednimi wersjami są zdecydowanie odradzane, ponieważ deweloperzy domyślnie oczekują wsteczną zgodność w nowszych wersjach interfejsu API.</span><span class="sxs-lookup"><span data-stu-id="1d500-128">Changes that affect backwards compatibility are strongly discouraged since developers by default expect backwards compatibility in newer versions of an API.</span></span>

## <a name="forward-compatibility"></a><span data-ttu-id="1d500-129">Zgodność dalej</span><span class="sxs-lookup"><span data-stu-id="1d500-129">Forward compatibility</span></span>

<span data-ttu-id="1d500-130">Zgodność ze starszymi wersjami odnosi się do zdolności klienta interfejsu API do uruchamiania w starszej wersji, przy jednoczesnym zachowaniu tego samego zachowania.</span><span class="sxs-lookup"><span data-stu-id="1d500-130">Forward compatibility refers to the ability of an existing consumer of an API to run against an older version while exhibiting the same behavior.</span></span> <span data-ttu-id="1d500-131">Jeśli użytkownik nie może uruchomić lub działa inaczej w przypadku uruchamiania w starszej wersji interfejsu API, interfejs API nie jest *zgodny*.</span><span class="sxs-lookup"><span data-stu-id="1d500-131">If a consumer is not able to run or behaves differently when run against an older version of the API, the API is *forward incompatible*.</span></span> 

<span data-ttu-id="1d500-132">Utrzymywanie zgodności w sposób niemal wyklucza wszelkie zmiany lub dodatki z wersji do wersji, ponieważ te zmiany uniemożliwiają uruchomienie odbiorcy, który jest przeznaczony dla nowszej wersji w ramach wcześniejszej wersji.</span><span class="sxs-lookup"><span data-stu-id="1d500-132">Maintaining forward compatibility virtually precludes any changes or additions from version to version, since those changes prevent a consumer that targets a later version from running under an earlier version.</span></span> <span data-ttu-id="1d500-133">Deweloperzy oczekują, że klient korzystający z nowszego interfejsu API może nie działać prawidłowo w porównaniu ze starszym interfejsem API.</span><span class="sxs-lookup"><span data-stu-id="1d500-133">Developers expect that a consumer that relies on a newer API may not function correctly against the older API.</span></span>
