## <a name="behavioral-change"></a><span data-ttu-id="b739a-112">Zmiana zachowania</span><span class="sxs-lookup"><span data-stu-id="b739a-112">Behavioral change</span></span>

<span data-ttu-id="b739a-115">Gdy zmiany behawioralne są widoczne na zewnątrz i modyfikują publiczną umowę typu, można je łatwo oszacować, ponieważ wpływają na zgodność binarną.</span><span class="sxs-lookup"><span data-stu-id="b739a-115">When behavioral changes are externally visible and modify a type's public contract, they are easy to evaluate since they affect binary compatibility.</span></span> <span data-ttu-id="b739a-116">Zmiany implementacji są znacznie trudniejsze do oszacowania; w zależności od rodzaju zmiany i częstotliwości i wzorców używania interfejsu API, wpływ zmiany może przedziały od pozostałej do niewielkiej ilości.</span><span class="sxs-lookup"><span data-stu-id="b739a-116">Implementation changes are much more difficult to evaluate; depending on the nature of the change and the frequency and patterns of use of the API, the impact of a change can range from severe to innocuous.</span></span>

## <a name="binary-compatibility"></a><span data-ttu-id="b739a-117">Zgodność binarna</span><span class="sxs-lookup"><span data-stu-id="b739a-117">Binary compatibility</span></span>

<span data-ttu-id="b739a-118">Zgodność binarna odnosi się do zdolności konsumenta interfejsu API do korzystania z interfejsu API w nowszej wersji bez ponownej kompilacji.</span><span class="sxs-lookup"><span data-stu-id="b739a-118">Binary compatibility refers to the ability of a consumer of an API to use the API on a newer version without recompilation.</span></span> <span data-ttu-id="b739a-119">Takie zmiany jak dodanie metod lub dodanie nowej implementacji interfejsu do typu nie wpływa na zgodność binarną.</span><span class="sxs-lookup"><span data-stu-id="b739a-119">Such changes as adding methods or adding a new interface implementation to a type do not affect binary compatibility.</span></span> <span data-ttu-id="b739a-120">Jednak usunięcie lub zmiana podpisów publicznych zestawu w taki sposób, aby klienci nie mogli już uzyskiwać dostępu do tego samego interfejsu uwidocznionego przez zestaw, ma wpływ na zgodność binarną.</span><span class="sxs-lookup"><span data-stu-id="b739a-120">However, removing or altering an assembly's public signatures so that consumers can no longer access the same interface exposed by the assembly does affect binary compatibility.</span></span> <span data-ttu-id="b739a-121">Zmiana tego rodzaju jest określana jako *binarna niezgodna zmiana*.</span><span class="sxs-lookup"><span data-stu-id="b739a-121">A change of this kind is termed a *binary incompatible change*.</span></span>

## <a name="source-compatibility"></a><span data-ttu-id="b739a-122">Zgodność ze źródłem</span><span class="sxs-lookup"><span data-stu-id="b739a-122">Source compatibility</span></span>

## <a name="design-time-compatibility"></a><span data-ttu-id="b739a-125">Zgodność czasu projektowania</span><span class="sxs-lookup"><span data-stu-id="b739a-125">Design-time compatibility</span></span>

<span data-ttu-id="b739a-126">Zgodność czasu projektowania odnosi się do zachowania środowiska czasu projektowania między wersjami programu Visual Studio i innymi środowiskami czasu projektowania.</span><span class="sxs-lookup"><span data-stu-id="b739a-126">Design-time compatibility refers to preserving the design-time experience across versions of Visual Studio and other design-time environments.</span></span> <span data-ttu-id="b739a-127">Chociaż może to wiązać się z zachowaniem lub interfejsem użytkownika projektantów, najważniejszym aspektem zgodności w czasie projektowania jest zgodność projektów.</span><span class="sxs-lookup"><span data-stu-id="b739a-127">While this can involve the behavior or the UI of designers, the most important aspect of design-time compatibility concerns project compatibility.</span></span> <span data-ttu-id="b739a-128">Projekt lub rozwiązanie musi być możliwe do otwarcia i użycia w nowszej wersji środowiska czasu projektowania.</span><span class="sxs-lookup"><span data-stu-id="b739a-128">A project or solution must be able to be opened and used on a newer version of the design-time environment.</span></span>

## <a name="backwards-compatibility"></a><span data-ttu-id="b739a-129">Zgodność z poprzednimi wersjami</span><span class="sxs-lookup"><span data-stu-id="b739a-129">Backwards compatibility</span></span>

<span data-ttu-id="b739a-130">Zgodność z poprzednimi wersjami odnosi się do zdolności klienta interfejsu API do uruchamiania w nowej wersji, zachowując się w ten sam sposób.</span><span class="sxs-lookup"><span data-stu-id="b739a-130">Backwards compatibility refers to the ability of an existing consumer of an API to run against a new version while behaving in the same way.</span></span> <span data-ttu-id="b739a-131">Obie zmiany w zachowaniu i zmiany w zgodności binarnej wpływają na zgodność z poprzednimi wersjami.</span><span class="sxs-lookup"><span data-stu-id="b739a-131">Both behavioral changes and changes in binary compatibility affect backwards compatibility.</span></span> <span data-ttu-id="b739a-132">Jeśli klient nie może działać lub działa inaczej w przypadku uruchamiania w nowszej wersji interfejsu API, interfejs API nie jest *zgodny z poprzednimi*wersjami.</span><span class="sxs-lookup"><span data-stu-id="b739a-132">If a consumer is not able to run or behaves differently when running against the newer version of the API, the API is *backwards incompatible*.</span></span>

<span data-ttu-id="b739a-133">Zmiany wpływające na zgodność z poprzednimi wersjami nie są zalecane, ponieważ deweloperzy oczekują wstecznej zgodności w nowszych wersjach interfejsu API.</span><span class="sxs-lookup"><span data-stu-id="b739a-133">Changes that affect backwards compatibility are discouraged, since developers expect backwards compatibility in newer versions of an API.</span></span>

## <a name="forward-compatibility"></a><span data-ttu-id="b739a-134">Zgodność dalej</span><span class="sxs-lookup"><span data-stu-id="b739a-134">Forward compatibility</span></span>

<span data-ttu-id="b739a-135">Zgodność ze starszymi wersjami odnosi się do zdolności klienta interfejsu API do uruchamiania w starszej wersji, przy jednoczesnym zachowaniu tego samego zachowania.</span><span class="sxs-lookup"><span data-stu-id="b739a-135">Forward compatibility refers to the ability of an existing consumer of an API to run against an older version while exhibiting the same behavior.</span></span> <span data-ttu-id="b739a-136">Jeśli użytkownik nie może uruchomić lub działa inaczej w przypadku uruchamiania w starszej wersji interfejsu API, interfejs API nie jest *zgodny*.</span><span class="sxs-lookup"><span data-stu-id="b739a-136">If a consumer is not able to run or behaves differently when run against an older version of the API, the API is *forward incompatible*.</span></span>

<span data-ttu-id="b739a-137">Utrzymywanie zgodności w sposób niemal wyklucza wszelkie zmiany lub dodatki z wersji do wersji, ponieważ te zmiany uniemożliwiają uruchomienie odbiorcy, który jest przeznaczony dla nowszej wersji w ramach wcześniejszej wersji.</span><span class="sxs-lookup"><span data-stu-id="b739a-137">Maintaining forward compatibility virtually precludes any changes or additions from version to version, since those changes prevent a consumer that targets a later version from running under an earlier version.</span></span> <span data-ttu-id="b739a-138">Deweloperzy oczekują, że klient korzystający z nowszego interfejsu API może nie działać prawidłowo w porównaniu ze starszym interfejsem API.</span><span class="sxs-lookup"><span data-stu-id="b739a-138">Developers expect that a consumer that relies on a newer API may not function correctly against the older API.</span></span>
