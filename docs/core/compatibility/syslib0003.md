# <a name="syslib0003-code-access-security-is-not-supported"></a>SYSLIB0003: zabezpieczenia dostępu kodu nie są obsługiwane

[Zabezpieczenia dostępu kodu (CAS)](../../framework/data/adonet/code-access-security.md) to nieobsługiwana, Starsza technologia. Infrastruktura umożliwiająca obsługę urzędów certyfikacji, która istnieje tylko w .NET Framework 2. x-4. x, jest przestarzała i nie otrzymuje poprawek do obsługi lub zabezpieczeń.

W związku z tym większość typów związanych z zabezpieczeniami dostępu kodu (CAS) w programie .NET jest przestarzałych, począwszy od platformy .NET 5,0. Obejmuje to atrybuty CAS, takie jak <xref:System.Security.Permissions.SecurityPermissionAttribute> obiekty uprawnień urzędów certyfikacji, takie <xref:System.Net.SocketPermission> jak <xref:System.Security.Policy.EvidenceBase> Typy pochodne i inne pomocnicze interfejsy API. Użycie tych interfejsów API generuje ostrzeżenie `SYSLIB0003` w czasie kompilacji.

## <a name="workarounds"></a>Obejścia

- W przypadku potwierdzenia uprawnień zabezpieczeń Usuń atrybut lub wywołanie, które potwierdza uprawnienie.

  ```csharp
  // REMOVE the attribute below.
  [SecurityPermission(SecurityAction.Assert, ControlThread = true)]
  public void DoSomething()
  {
  }
  public void DoAssert()
  {
      // REMOVE the line below.
      new SecurityPermission(SecurityPermissionFlag.ControlThread).Assert();
  }
  ```

- Jeśli masz odmowę lub ograniczenie (za pośrednictwem `PermitOnly` ) dowolnych uprawnień, skontaktuj się z doradcą zabezpieczeń. Ponieważ atrybuty urzędów certyfikacji nie są uznawane przez środowisko uruchomieniowe programu .NET 5.0 +, aplikacja może mieć otwór bezpieczeństwa, jeśli nie opiera się on na infrastrukturze CAS w celu ograniczenia dostępu do tych metod.

  ```csharp
  // REVIEW the attribute below; could indicate security vulnerability.
  [SecurityPermission(SecurityAction.Deny, ControlThread = true)]
  public void DoSomething()
  {
  }
  public void DoPermitOnly()
  {
      // REVIEW the line below; could indicate security vulnerability.
      new SecurityPermission(SecurityPermissionFlag.ControlThread).PermitOnly();
  }
  ```

- Jeśli masz jakieś uprawnienia (z wyjątkiem <xref:System.Security.Permissions.PrincipalPermission> ), Usuń żądanie. Wszystkie wymagania zakończą się pomyślnie w czasie wykonywania.

  ```csharp
  // REMOVE the attribute below; it will always succeed.
  [SecurityPermission(SecurityAction.Demand, ControlThread = true)]
  public void DoSomething()
  {
  }
  public void DoDemand()
  {
      // REMOVE the line below; it will always succeed.
      new SecurityPermission(SecurityPermissionFlag.ControlThread).Demand();
  }
  ```

- Jeśli masz wymaganie <xref:System.Security.Permissions.PrincipalPermission> , zapoznaj się z tematem wskazówki dotyczące [SYSLIB0002: PrincipalPermissionAttribute jest przestarzałe](syslib0002.md#workarounds). Te wskazówki dotyczą zarówno programu <xref:System.Security.Permissions.PrincipalPermission> , jak i <xref:System.Security.Permissions.PrincipalPermissionAttribute> .
