|---|---|
|Szczegóły|Implementacja oryginalnego <xref:System.ComponentModel.MemberDescriptor.Equals%2A?displayProperty=nameWithType> metody porównanie dwóch właściwości inny ciąg z obiektów są porównywane: Nazwa kategorii i ciąg opisu. |Sugestia|Jeśli aplikacja jest zależna od <xref:System.ComponentModel.MemberDescriptor.Equals%2A?displayProperty=nameWithType> czasami zwracanie <code>false</code> po deskryptory są równoważne i przeznaczona dla programu .NET Framework 4.6.2 lub później, masz kilka opcji:<ol><li>Zmiany kodu do porównania <xref:System.ComponentModel.MemberDescriptor.Category> i <xref:System.ComponentModel.MemberDescriptor.Description> pól ręcznie oprócz wywołania <xref:System.ComponentModel.MemberDescriptor.Equals%2A?displayProperty=nameWithType> metody.</li><li>Opcja rezygnacji z tę zmianę, dodając następującą wartość do pliku app.config:</li></ol><pre><code class="lang-xml">&lt;runtime&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.System.MemberDescriptorEqualsReturnsFalseIfEquivalent=true&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;</code></pre>Jeśli aplikacja jest przeznaczony dla platformy .NET Framework 4.6.1 lub starszym i działa w programie .NET Framework 4.6.2 lub nowszym i chcesz, aby zmiana ta jest włączona, można ustawić przełącznika zgodności <code>false</code> przez dodanie następującą wartość do pliku app.config:<pre><code class="lang-xml">&lt;runtime&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.System.MemberDescriptorEqualsReturnsFalseIfEquivalent=false&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;</code></pre>|
