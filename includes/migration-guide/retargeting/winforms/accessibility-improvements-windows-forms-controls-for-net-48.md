### <a name="accessibility-improvements-in-windows-forms-controls-for-net-48"></a>Ulepszenia ułatwień dostępu w formantach formularzy systemu Windows dla platformy .NET 4.8

|   |   |
|---|---|
|Szczegóły|Struktura formularzy systemu Windows nadal ulepsza sposób działania z technologiami ułatwień dostępu, aby lepiej obsługiwać klientów formularzy Windows. Należą do nich następujące zmiany:<ul><li>Zmiany w celu poprawy wyświetlania w trybie wysokiego kontrastu.</li><li>Zmiany w interakcji z Narratorem.</li><li>Zmiany w hierarchii dostępne (poprawa nawigacji za pośrednictwem drzewa automatyzacji interfejsu użytkownika).</li></ul>|
|Sugestia|**Jak zgłosić się lub zrezygnować z tych zmian**<br>Aby aplikacja korzystać z tych zmian, musi działać na .NET Framework 4.8. Aplikacja może wyrazić zgodę na te zmiany w jeden z następujących sposobów:<ul><li>Jest ponownie skompilowany do celu .NET Framework 4.8. Te zmiany ułatwień dostępu są domyślnie włączone w aplikacjach Windows Forms, które są przeznaczone dla programu .NET Framework 4.8.</li><li>Jest przeznaczony dla platformy .NET Framework 4.7.2 lub starszej wersji i rezygnuje z starszych <code>&lt;runtime&gt;</code> zachowań ułatwień dostępu, dodając <code>false</code>następujący [przełącznik AppContext](https://docs.microsoft.com/dotnet/framework/configure-apps/file-schema/runtime/appcontextswitchoverrides-element) do sekcji pliku konfiguracji aplikacji i ustawiając go na , jak pokazano w poniższym przykładzie.</li></ul><pre><code class="lang-xml">&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot;?&gt;&#13;&#10;&lt;configuration&gt;&#13;&#10;&lt;startup&gt;&#13;&#10;&lt;supportedRuntime version=&quot;v4.0&quot; sku=&quot;.NETFramework,Version=v4.7&quot;/&gt;&#13;&#10;&lt;/startup&gt;&#13;&#10;&lt;runtime&gt;&#13;&#10;&lt;!-- AppContextSwitchOverrides value attribute is in the form of &#39;key1=true/false;key2=true/false  --&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.UseLegacyAccessibilityFeatures=false;Switch.UseLegacyAccessibilityFeatures.2=false;Switch.UseLegacyAccessibilityFeatures.3=false&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;&lt;/configuration&gt;&#13;&#10;</code></pre>Należy pamiętać, że aby zdecydować się na funkcje ułatwień dostępu dodane w .NET Framework 4.8, należy również zdecydować się na funkcje ułatwień dostępu .NET Framework 4.7.1 i 4.7.2, jak również. Aplikacje, które są przeznaczone dla programu .NET Framework 4.8 i chcą zachować starsze zachowanie ułatwień dostępu, mogą zdecydować się na korzystanie ze starszych funkcji ułatwień dostępu, jawnie ustawiając przełącznik AppContext na <code>true</code>. Włączenie obsługi wywołania etykietek narzędzi klawiatury wymaga dodania wiersza <code>Switch.System.Windows.Forms.UseLegacyToolTipDisplay=false</code> do wartości AppContextSwitchOverrides:<pre><code class="lang-xml">&#39;&lt;AppContextSwitchOverrides value=&quot;Switch.UseLegacyAccessibilityFeatures=false;Switch.UseLegacyAccessibilityFeatures.2=false;Switch.UseLegacyAccessibilityFeatures.3=false;Switch.System.Windows.Forms.UseLegacyToolTipDisplay=false&quot; /&gt;&#39;&#13;&#10;</code></pre>Należy zauważyć, że włączenie tej funkcji wymaga włączenia wyżej wymienionych funkcji ułatwień dostępu programu .NET Framework 4.7.1 - 4.8. Ponadto jeśli którakolwiek z funkcji ułatwień dostępu nie jest wybrana, ale <xref:System.NotSupportedException> funkcja wyświetlania etykietki narzędzia jest wprowadzona, środowisko uruchomieniowe zostanie uruchomione przy pierwszym dostępie do tych funkcji. Komunikat o wyjątku wskazuje, że etykietki narzędzi klawiatury wymagają poprawy ułatwień dostępu poziomu 3 do włączenia. **Używanie kolorów zdefiniowanych przez system operacyjny w motywach o wysokim kontraście**<ul><li>Ulepszone motywy o wysokim kontraście.</li></ul>**Ulepszona obsługa Narratora**<ul><li>Narrator ogłasza teraz kierunek <xref:System.Windows.Forms.DataGridViewColumn> sortowania podczas ogłaszania <xref:System.Windows.Forms.DataGridViewCell>dostępnej nazwy pliku .</li></ul>**Ulepszona obsługa ułatwień dostępu CheckedListBox**<ul><li>Ulepszona obsługa <xref:System.Windows.Forms.CheckedListBox> narratora dla formantu. Podczas przechodzenia <xref:System.Windows.Forms.CheckedListBox> do formantu za pomocą <xref:System.Windows.Forms.CheckedListBox> klawiatury Narrator skupia go i ogłasza.</li><li>Pusty formant CheckedListBox ma teraz prostokąt fokusu narysowany dla wirtualnego pierwszego elementu, gdy formant staje się skoncentrowany.</li></ul>**Ulepszona obsługa ułatwień dostępu combobox**<ul><li>Włączona automatyzacja interfejsu <xref:System.Windows.Forms.ComboBox> użytkownika obsługuje formant, z możliwością korzystania z powiadomień automatyzacji interfejsu użytkownika i innych funkcji automatyzacji interfejsu użytkownika.</li></ul>**Ulepszona obsługa ułatwień dostępu DataGridView**<ul><li>Włączona automatyzacja interfejsu <xref:System.Windows.Forms.DataGridView> użytkownika obsługuje kontrolę z możliwością korzystania z powiadomień automatyzacji interfejsu użytkownika i innych funkcji automatyzacji interfejsu użytkownika.</li><li>Element automatyzacji interfejsu użytkownika, <xref:System.Windows.Forms.DataGridViewComboBoxEditingControl> który <xref:System.Windows.Forms.DataGridViewTextBoxEditingControl> odpowiada lub jest teraz elementem podrzędnym odpowiedniej komórki edycji.</li></ul>**Ulepszona pomoc techniczna dotycząca ułatwień dostępu LinkLabel**<ul><li>Ulepszona <xref:System.Windows.Forms.LinkLabel> dostępność formantu: Narrator ogłasza stan <xref:System.Windows.Forms.LinkLabel> wyłączeń dla łącza, jeśli odpowiedni formant jest wyłączony.</li></ul>**Ulepszona obsługa ułatwień dostępu progressbar**<ul><li>Włączona automatyzacja interfejsu <xref:System.Windows.Forms.ProgressBar> użytkownika obsługuje formant z możliwością korzystania z powiadomień automatyzacji interfejsu użytkownika i innych funkcji automatyzacji interfejsu użytkownika. Deweloperzy mogą teraz używać powiadomień automatyzacji interfejsu użytkownika, które Narrator może ogłosić, aby wskazać postęp.</li></ul>Aby zapoznać się z omówieniem zdarzeń automatyzacji interfejsu użytkownika, w tym zdarzeniami powiadomień automatyzacji interfejsu użytkownika, zobacz [Omówienie zdarzeń automatyzacji interfejsu użytkownika.](https://docs.microsoft.com/windows/desktop/WinAuto/uiauto-eventsoverview) **Ulepszona pomoc techniczna dotycząca ułatwień dostępu wg.**<ul><li>Włączona automatyzacja interfejsu <xref:System.Windows.Forms.PropertyGrid> użytkownika obsługuje formant, z możliwością korzystania z powiadomień automatyzacji interfejsu użytkownika i innych funkcji automatyzacji interfejsu użytkownika.</li><li>Element automatyzacji interfejsu użytkownika, który odpowiada aktualnie edytowanej właściwości jest teraz elementem podrzędnym odpowiedniego elementu właściwości interfejsu użytkownika.</li><li>Element właściwości automatyzacji interfejsu użytkownika jest teraz elementem podrzędnym <xref:System.Windows.Forms.PropertyGrid> odpowiedniego elementu kategorii, jeśli formant nadrzędny jest ustawiony na widok kategorii.</li></ul>**Ulepszona obsługa ToolStrip**<ul><li>Włączona automatyzacja interfejsu <xref:System.Windows.Forms.ToolStrip> użytkownika obsługuje formant, z możliwością korzystania z powiadomień automatyzacji interfejsu użytkownika i innych funkcji automatyzacji interfejsu użytkownika.</li><li>Ulepszona <xref:System.Windows.Forms.ToolStrip> nawigacja po elementach.</li><li>W trybie elementów fokus Narratora nie znika i nie przechodzi do ukrytych elementów.</li></ul>**Ulepszone wskazówki wizualne**<ul><li>Pusty <xref:System.Windows.Forms.CheckedListBox> formant wyświetla teraz wskaźnik ostrości po odebraniu fokusu.</li></ul>Uwaga: Obsługa automatyzacji interfejsu użytkownika jest włączona dla formantów w czasie wykonywania, ale nie jest używana w czasie projektowania.
