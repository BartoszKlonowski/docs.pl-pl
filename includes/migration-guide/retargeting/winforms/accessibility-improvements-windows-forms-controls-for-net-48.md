### <a name="accessibility-improvements-in-windows-forms-controls-for-net-48"></a>Ulepszenia ułatwień dostępu w kontrolkach Windows Forms dla programu .NET 4,8

|   |   |
|---|---|
|Szczegóły|Windows Forms Framework kontynuuje ulepszanie sposobu działania z technologiami ułatwień dostępu w celu lepszego wsparcia Windows Forms klientów. Należą do nich następujące zmiany:<ul><li>Zmiany w celu usprawnienia wyświetlania w trybie duży kontrast.</li><li>Zmiany interakcji z narratorem.</li><li>Zmiany w dostępnej hierarchii (ulepszenie nawigacji przez drzewo automatyzacji interfejsu użytkownika).</li></ul>|
|Sugestia|**Jak wybrać lub wycofać te zmiany**<br>Aby aplikacja mogła korzystać z tych zmian, należy ją uruchomić na .NET Framework 4,8. Aplikacja może wyrazić zgodę na te zmiany w jeden z następujących sposobów:<ul><li>Zostanie ponownie skompilowana w celu przekierowania .NET Framework 4,8. Te zmiany ułatwień dostępu są domyślnie włączone w Windows Forms aplikacjach przeznaczonych dla .NET Framework 4,8.</li><li>Jest ona przeznaczona dla .NET Framework 4.7.2 lub starszej wersji, a także [wyłączają](https://docs.microsoft.com/dotnet/framework/configure-apps/file-schema/runtime/appcontextswitchoverrides-element) się ze starszych zachowań ułatwień dostępu, dodając następujący przykład do <code>&lt;runtime&gt;</code> w pliku konfiguracji aplikacji i ustawiając go na <code>false</code>, jak pokazano w poniższym przykładzie.</li></ul><pre><code class="lang-xml">&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot;?&gt;&#13;&#10;&lt;configuration&gt;&#13;&#10;&lt;startup&gt;&#13;&#10;&lt;supportedRuntime version=&quot;v4.0&quot; sku=&quot;.NETFramework,Version=v4.7&quot;/&gt;&#13;&#10;&lt;/startup&gt;&#13;&#10;&lt;runtime&gt;&#13;&#10;&lt;!-- AppContextSwitchOverrides value attribute is in the form of &#39;key1=true/false;key2=true/false  --&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.UseLegacyAccessibilityFeatures=false;Switch.UseLegacyAccessibilityFeatures.2=false;Switch.UseLegacyAccessibilityFeatures.3=false&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;&lt;/configuration&gt;&#13;&#10;</code></pre>Należy pamiętać, że aby skorzystać z funkcji ułatwień dostępu dodanych w .NET Framework 4,8, należy również zrezygnować z funkcji ułatwień dostępu .NET Framework 4.7.1 i 4.7.2. Aplikacje przeznaczone dla .NET Framework 4,8 i chcą zachować starsze zachowanie ułatwień dostępu mogą zrezygnować z używania starszych funkcji ułatwień dostępu przez jawne ustawienie tego przełącznika AppContext na <code>true</code>. Włączenie obsługi wywołania etykietki narzędzia klawiatury wymaga dodania wiersza <code>Switch.System.Windows.Forms.UseLegacyToolTipDisplay=false</code> do wartości AppContextSwitchOverrides:<pre><code class="lang-xml">&#39;&lt;AppContextSwitchOverrides value=&quot;Switch.UseLegacyAccessibilityFeatures=false;Switch.UseLegacyAccessibilityFeatures.2=false;Switch.UseLegacyAccessibilityFeatures.3=false;Switch.System.Windows.Forms.UseLegacyToolTipDisplay=false&quot; /&gt;&#39;&#13;&#10;</code></pre>Należy pamiętać, że włączenie tej funkcji wymaga skorzystania z powyższych funkcji ułatwień dostępu .NET Framework 4.7.1-4,8. Ponadto jeśli jakakolwiek z funkcji ułatwień dostępu nie zostanie włączona, ale funkcja wyświetlania etykietki narzędzi zostanie włączona, podczas pierwszego dostępu do tych funkcji zostanie wygenerowany <xref:System.NotSupportedException> środowiska uruchomieniowego. Komunikat o wyjątku wskazuje, że etykietki narzędzi klawiatury wymagają usprawnienia ułatwienia dostępu w celu włączenia poziomu 3. **Używanie kolorów zdefiniowanych w systemie operacyjnym w duży kontrast motywach**<ul><li>Ulepszone motywy o wysokim kontraście.</li></ul>**Ulepszona obsługa Narratora**<ul><li>Teraz program Narrator ogłasza kierunek sortowania <xref:System.Windows.Forms.DataGridViewColumn> podczas ogłaszania dostępnej nazwy <xref:System.Windows.Forms.DataGridViewCell>.</li></ul>**Ulepszona obsługa ułatwień dostępu CheckedListBox**<ul><li>Ulepszona obsługa programu Narrator dla kontrolki <xref:System.Windows.Forms.CheckedListBox>. Podczas przechodzenia do kontrolki <xref:System.Windows.Forms.CheckedListBox> przy użyciu klawiatury program Narrator koncentruje się na elemencie <xref:System.Windows.Forms.CheckedListBox> i ogłasza go.</li><li>Pusta kontrolka CheckedListBox ma teraz prostokąt fokusu rysowany dla wirtualnego pierwszego elementu, gdy formant zmieni się na fokus.</li></ul>**Ulepszona obsługa ułatwień dostępu ComboBox**<ul><li>Włączono obsługę automatyzacji interfejsu użytkownika dla kontrolki <xref:System.Windows.Forms.ComboBox>, z możliwością korzystania z powiadomień automatyzacji interfejsu użytkownika i innych funkcji automatyzacji interfejsu użytkownika.</li></ul>**Ulepszona obsługa ułatwień dostępu DataGridView**<ul><li>Włączono obsługę automatyzacji interfejsu użytkownika dla kontrolek <xref:System.Windows.Forms.DataGridView> z możliwością używania powiadomień automatyzacji interfejsu użytkownika i innych funkcji automatyzacji interfejsu użytkownika.</li><li>Element automatyzacji interfejsu użytkownika, który odnosi się do <xref:System.Windows.Forms.DataGridViewComboBoxEditingControl> lub <xref:System.Windows.Forms.DataGridViewTextBoxEditingControl> jest teraz elementem podrzędnym odpowiedniej komórki edycji.</li></ul>**Ulepszona obsługa ułatwień dostępu LinkLabel**<ul><li>Ulepszona dostępność kontroli <xref:System.Windows.Forms.LinkLabel>: program Narrator ogłasza stan wyłączenia dla łącza, Jeśli odpowiednia kontrolka <xref:System.Windows.Forms.LinkLabel> jest wyłączona.</li></ul>**Ulepszona obsługa ułatwień dostępu ProgressBar**<ul><li>Włączona obsługa automatyzacji interfejsu użytkownika dla kontrolki <xref:System.Windows.Forms.ProgressBar> z możliwością używania powiadomień automatyzacji interfejsu użytkownika i innych funkcji automatyzacji interfejsu użytkownika. Deweloperzy mogą teraz używać powiadomień automatyzacji interfejsu użytkownika, które narrator może ogłosić, aby wskazać postęp.</li></ul>Omówienie zdarzeń automatyzacji interfejsu użytkownika, w tym zdarzenia powiadomień automatyzacji interfejsu użytkownika, można znaleźć w temacie [Omówienie zdarzeń automatyzacji interfejsu użytkownika](https://docs.microsoft.com/windows/desktop/WinAuto/uiauto-eventsoverview). **Ulepszona obsługa ułatwień dostępu** do elementu PropertyGrid<ul><li>Włączono obsługę automatyzacji interfejsu użytkownika dla kontrolki <xref:System.Windows.Forms.PropertyGrid>, z możliwością korzystania z powiadomień automatyzacji interfejsu użytkownika i innych funkcji automatyzacji interfejsu użytkownika.</li><li>Element automatyzacji interfejsu użytkownika, który odnosi się do obecnie edytowanej właściwości, jest teraz elementem podrzędnym odpowiadającego elementu automatyzacji interfejsu użytkownika elementu właściwości.</li><li>Element elementu właściwości Automatyzacja interfejsu użytkownika jest teraz elementem podrzędnym odpowiadającego elementu kategorii, jeśli nadrzędna kontrolka <xref:System.Windows.Forms.PropertyGrid> jest ustawiona na widok kategorii.</li></ul>**Ulepszona obsługa ToolStrip**<ul><li>Włączono obsługę automatyzacji interfejsu użytkownika dla kontrolki <xref:System.Windows.Forms.ToolStrip>, z możliwością korzystania z powiadomień automatyzacji interfejsu użytkownika i innych funkcji automatyzacji interfejsu użytkownika.</li><li>Ulepszona nawigacja za <xref:System.Windows.Forms.ToolStrip> elementów.</li><li>W trybie elementów fokus Narratora nie znika i nie przechodzi do elementów ukrytych.</li></ul>**Ulepszone podpowiedzi wizualne**<ul><li>Pusta kontrolka <xref:System.Windows.Forms.CheckedListBox> teraz wyświetla wskaźnik fokusu po odebraniu fokusu.</li></ul>Uwaga: Obsługa automatyzacji interfejsu użytkownika jest włączona dla formantów w środowisku uruchomieniowym, ale nie jest używana w czasie projektowania. Aby zapoznać się z omówieniem automatyzacji interfejsu użytkownika, zobacz [Omówienie automatyzacji interfejsu użytkownika](https://docs.microsoft.com/dotnet/framework/ui-automation/ui-automation-overview).</p>**Wywoływanie etykietek narzędzi formantów przy użyciu klawiatury**<ul><li>Etykietka narzędzia kontrolki może teraz być wywoływana przez skoncentrowanie kontrolki z klawiaturą. Ta funkcja musi zostać włączona jawnie dla aplikacji (patrz sekcja **&quot;jak zrezygnować z tych zmian lub z nich&quot;** )</li></ul>|
