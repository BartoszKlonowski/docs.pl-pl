### <a name="accessibility-improvements-in-windows-forms-controls-for-net-48"></a>Ulepszenia ułatwień dostępu w kontrolkach formularzy Windows Forms dla 4,8 platformy .NET

|   |   |
|---|---|
|Szczegóły|Środowiska formularzy Windows kontynuuje poprawić, jak to działa przy użyciu technologii ułatwień dostępu w celu skuteczniejszej obsługi klientów Windows Forms. Obejmują one następujące zmiany:<ul><li>Zmiany w celu zwiększenia wyświetlanych podczas trybu wysokiego kontrastu.</li><li>Zmiany w interakcję z Narrator.</li><li>Zmiany w hierarchii dostępny (zwiększanie nawigację drzewa automatyzacji interfejsu użytkownika).</li></ul>|
|Sugestia|<strong>Sposób korzystania z opcji na lub poza te zmiany</strong>w kolejności dla aplikacji do korzystania z tych zmian, należy uruchomić na .NET Framework 4.8. Aplikacja mogą wyrazić zgodę w tych zmian w jednej z następujących sposobów:<ul><li>Jest ponownie kompilowana pod kątem .NET Framework 4.8. Te zmiany ułatwień dostępu są włączone domyślnie w aplikacji Windows Forms, przeznaczonych dla 4,8 Framework .NET.</li><li>Jest przeznaczony dla .NET Framework 4.7.2 lub starszej wersji, a zdecyduje poza zachowania starszych ułatwień dostępu przez dodanie poniższego [przełącznika AppContext](https://docs.microsoft.com/dotnet/framework/configure-apps/file-schema/runtime/appcontextswitchoverrides-element) do <code>&lt;runtime&gt;</code> części pliku konfiguracyjnego aplikacji i ustawieniem dla niego <code>false</code> , jak pokazano w poniższym przykładzie.</li></ul><pre><code class="lang-xml">&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot;?&gt;&#13;&#10;&lt;configuration&gt;&#13;&#10;&lt;startup&gt;&#13;&#10;&lt;supportedRuntime version=&quot;v4.0&quot; sku=&quot;.NETFramework,Version=v4.7&quot;/&gt;&#13;&#10;&lt;/startup&gt;&#13;&#10;&lt;runtime&gt;&#13;&#10;&lt;!-- AppContextSwitchOverrides value attribute is in the form of &#39;key1=true/false;key2=true/false  --&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.UseLegacyAccessibilityFeatures=false;Switch.UseLegacyAccessibilityFeatures.2=false;Switch.UseLegacyAccessibilityFeatures.3=false&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;&lt;/configuration&gt;&#13;&#10;</code></pre>Należy pamiętać, że do korzystania z funkcji ułatwień dostępu, dodany do programu .NET Framework 4,8, musi również zgody na uczestnictwo w .NET Framework 4.7.1 i 4.7.2 także funkcje ułatwień dostępu. Aplikacje docelowe .NET Framework 4.8, które chcesz zachować zachowanie ułatwień dostępu w starszej wersji zgodzić się na korzystanie z funkcji ułatwień dostępu w starszej wersji przez jawne ustawienie tego parametru AppContext <code>true</code>. Włączanie klawiatury, obsługa wywołania etykietki narzędzia wymaga dodania <code>Switch.System.Windows.Forms.UseLegacyToolTipDisplay=false</code> wiersz z wartością AppContextSwitchOverrides:<pre><code class="lang-xml">&#39;&lt;AppContextSwitchOverrides value=&quot;Switch.UseLegacyAccessibilityFeatures=false;Switch.UseLegacyAccessibilityFeatures.2=false;Switch.UseLegacyAccessibilityFeatures.3=false;Switch.System.Windows.Forms.UseLegacyToolTipDisplay=false&quot; /&gt;&#39;&#13;&#10;</code></pre>Należy pamiętać, że włączenie tej funkcji wymaga zgody na korzystanie z funkcji wyżej ułatwień dostępu programu .NET Framework 4.7.1 - 4.8. Ponadto jeśli dowolny ułatwień dostępu, nie zgłoszono funkcji, ale etykietka narzędzia wyświetla funkcji jest wyrażeniu zgody na uczestnictwo w środowisku uruchomieniowym <xref:System.NotSupportedException> zostanie zgłoszony przy pierwszym dostępie do tych funkcji. Komunikat o wyjątku wskazuje, że klawiatura etykietki narzędzi wymagają ulepszenia ułatwień dostępu z poziomu 3, aby włączyć. <strong>Kolorów zdefiniowanych przez użycie systemu operacyjnego w wysokiego kontrastu, motywów</strong><ul><li>Ulepszone motywy o wysokim kontraście.</li></ul><strong>Ulepszona obsługa Narrator</strong><ul><li>Narrator teraz ogłasza kierunek sortowania <xref:System.Windows.Forms.DataGridViewColumn> podczas ogłoszenie dostępna nazwa <xref:System.Windows.Forms.DataGridViewCell>.</li></ul><strong>Ulepszona obsługa CheckedListBox ułatwień dostępu</strong><ul><li>Ulepszona obsługa Narrator <xref:System.Windows.Forms.CheckedListBox> kontroli. Podczas przechodzenia do <xref:System.Windows.Forms.CheckedListBox> kontrolować za pomocą klawiatury, Narrator koncentruje się <xref:System.Windows.Forms.CheckedListBox> element i ogłasza go.</li><li>Pusty CheckedListBox, kontrolka ma teraz fokus prostokąt dla wirtualnych pierwszy element staje się fokusem formantu.</li></ul><strong>Ulepszona obsługa ComboBox ułatwień dostępu</strong><ul><li>Włączona obsługa automatyzacji interfejsu użytkownika dla <xref:System.Windows.Forms.ComboBox> kontroli możliwość użycia powiadomień automatyzacji interfejsu użytkownika i inne funkcje automatyzacji interfejsu użytkownika.</li></ul><strong>Ulepszona obsługa ułatwień dostępu w formancie DataGridView</strong><ul><li>Włączona obsługa automatyzacji interfejsu użytkownika dla <xref:System.Windows.Forms.DataGridView> formantu o możliwość używania powiadomień automatyzacji interfejsu użytkownika i inne funkcje automatyzacji interfejsu użytkownika.</li><li>Element automatyzacji interfejsu użytkownika, który odpowiada <xref:System.Windows.Forms.DataGridViewComboBoxEditingControl> lub <xref:System.Windows.Forms.DataGridViewTextBoxEditingControl> teraz jest elementem podrzędnym odpowiedniego komórki edycji.</li></ul><strong>Ulepszona obsługa LinkLabel ułatwień dostępu</strong><ul><li>Ulepszone <xref:System.Windows.Forms.LinkLabel> kontrolować ułatwień dostępu: Narrator ogłasza wyłączona dla tego połączenia, jeśli odpowiedni <xref:System.Windows.Forms.LinkLabel> kontrolka jest wyłączona.</li></ul><strong>Ulepszona obsługa ProgressBar ułatwień dostępu</strong><ul><li>Włączona obsługa automatyzacji interfejsu użytkownika dla <xref:System.Windows.Forms.ProgressBar> formantu o możliwość używania powiadomień automatyzacji interfejsu użytkownika i inne funkcje automatyzacji interfejsu użytkownika. Deweloperzy mogą teraz korzystanie z powiadomień dotyczących automatyzacji interfejsu użytkownika, które Narrator, można anonsować wskazują postęp.</li></ul>Aby uzyskać przegląd Przegląd zdarzeń automatyzacji interfejsu użytkownika, w tym zdarzenia powiadomień automatyzacji interfejsu użytkownika, zobacz [Przegląd zdarzeń automatyzacji interfejsu użytkownika](https://docs.microsoft.com/en-us/windows/desktop/WinAuto/uiauto-eventsoverview). <strong>Ulepszona obsługa PropertyGrid ułatwień dostępu</strong><ul><li>Włączona obsługa automatyzacji interfejsu użytkownika dla <xref:System.Windows.Forms.PropertyGrid> kontroli możliwość użycia powiadomień automatyzacji interfejsu użytkownika i inne funkcje automatyzacji interfejsu użytkownika.</li><li>Element automatyzacji interfejsu użytkownika, który odpowiada właściwości aktualnie edytowanej teraz jest elementem podrzędnym elementu odpowiednie właściwości elementu automatyzacji interfejsu użytkownika.</li><li>Właściwości elementu automatyzacji interfejsu użytkownika jest teraz elementem podrzędnym odpowiedniego elementu kategorii, jeśli element nadrzędny <xref:System.Windows.Forms.PropertyGrid> kontrolki jest ustawiony na wartość view kategorii.</li></ul><strong>Ulepszona obsługa ToolStrip</strong><ul><li>Włączona obsługa automatyzacji interfejsu użytkownika dla <xref:System.Windows.Forms.ToolStrip> kontroli możliwość użycia powiadomień automatyzacji interfejsu użytkownika i inne funkcje automatyzacji interfejsu użytkownika.</li><li>Lepsza Nawigacja za pomocą <xref:System.Windows.Forms.ToolStrip> elementów.</li><li>W trybie elementów fokus Narrator nie znika i nie przechodzi ukryte elementy.</li></ul><strong>Ulepszone podpowiedzi wizualne</strong><ul><li>Pusta <xref:System.Windows.Forms.CheckedListBox> kontroli teraz wyświetla wskaźnik fokus, gdy staje się aktywny.</li></ul>Uwaga: Obsługa automatyzacji interfejsu użytkownika jest włączona dla formantów w czasie wykonywania, ale nie jest używany w czasie projektowania. Aby zapoznać się z omówieniem automatyzacji interfejsu użytkownika, zobacz [Przegląd automatyzacji interfejsu użytkownika](https://docs.microsoft.com/dotnet/framework/ui-automation/ui-automation-overview).</p><strong>Wywoływanie kontrolki etykietki narzędzi, za pomocą klawiatury</strong><ul><li>Kontrolka etykietki narzędzi może być wywoływany teraz, koncentrując się kontrolki z klawiatury.
