<span data-ttu-id="966e5-103">W narzędziach MSBuild 12.0 (dołączonych do programu Visual Studio 2013) zostały zmienione lokalizacje plików programu MSBuild, co powoduje, że starsze pliki obiektów docelowych programu Entity Framework są nieprawidłowe.</span><span class="sxs-lookup"><span data-stu-id="966e5-103">MSBuild 12.0 tools (included in Visual Studio 2013) changed MSBuild file locations, causing older Entity Framework targets files to be invalid.</span></span> <span data-ttu-id="966e5-104">W wyniku tego zadania `EntityDeploySplit` i `EntityClean` kończą się niepowodzeniem, ponieważ nie można odnaleźć biblioteki `Microsoft.Data.Entity.Build.Tasks.dll`.</span><span class="sxs-lookup"><span data-stu-id="966e5-104">The result is that `EntityDeploySplit` and `EntityClean` tasks fail because they are unable to find `Microsoft.Data.Entity.Build.Tasks.dll`.</span></span> <span data-ttu-id="966e5-105">Należy pamiętać, że ten problem wynika ze zmiany dotyczącej zestawu narzędzi (MSBuild/VS), a nie ze zmiany w programie .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="966e5-105">Note that this break is because of a toolset (MSBuild/VS) change, not because of a .NET Framework change.</span></span> <span data-ttu-id="966e5-106">Wystąpi on tylko w przypadku uaktualniania narzędzi dla deweloperów, a nie w przypadku uaktualniania samego programu .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="966e5-106">It will only occur when upgrading developer tools, not when merely upgrading the .NET Framework.</span></span>
