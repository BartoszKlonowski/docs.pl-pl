### <a name="accessibility-improvements-in-wpf"></a>Ulepszenia ułatwień dostępu w WPF

|   |   |
|---|---|
|Szczegóły|<strong>Ulepszenia usługi wysoki kontrast</strong><br><ul><li>Fokus dla <xref:System.Windows.Controls.Expander> kontroli jest teraz widoczna. W poprzednich wersjach programu .NET Framework nie było.</li><li>Tekst w <xref:System.Windows.Controls.CheckBox> i <xref:System.Windows.Controls.RadioButton> kontrolki, gdy są zaznaczone jest teraz bardziej widoczny niż w poprzednich wersjach systemu .NET Framework.</li><li>Obramowanie wyłączone <xref:System.Windows.Controls.ComboBox> jest teraz kolor wyłączonego tekstu. W poprzednich wersjach programu .NET Framework nie było.</li><li>Przyciski wyłączone i konkretną teraz używać kolor motywu poprawne. W poprzednich wersjach programu .NET Framework ich nie zmieniła się.</li><li>Przycisk listy rozwijanej jest teraz widoczna gdy <xref:System.Windows.Controls.ComboBox> stylu kontrolki jest ustawiona na <xref:System.Windows.Controls.ToolBar.ComboBoxStyleKey?displayProperty=nameWithType>, w poprzednich wersjach programu .NET Framework nie było.</li><li>Strzałka wskaźnika sortowania w <xref:System.Windows.Controls.DataGrid> formant teraz używa kolorów motywu. W poprzednich wersjach programu .NET Framework go nie zmieniła się.</li><li>Domyślny styl hiperłącze teraz zmienia kolor motywu poprawne na myszy za pośrednictwem. W poprzednich wersjach programu .NET Framework go nie zmieniła się.</li><li>Fokus klawiatury na przyciski radiowe są teraz widoczne. W poprzednich wersjach programu .NET Framework nie było.</li><li><xref:System.Windows.Controls.DataGrid> Formantu checkbox kolumny używa teraz oczekiwanego kolory opinii fokus klawiatury. W poprzednich wersjach programu .NET Framework go nie zmieniła się.</li><li>wizualizacje fokus klawiatury są teraz widoczne na <xref:System.Windows.Controls.ComboBox> i <xref:System.Windows.Controls.ListBox> kontrolki. W poprzednich wersjach programu .NET Framework nie było.</p></li></ul><strong>Ulepszenia interakcji z czytnika ekranu</strong><br><ul><li><xref:System.Windows.Controls.Expander> Formanty są teraz prawidłowo od niedawna grup (Rozwiń/Zwiń) przez czytniki zawartości ekranu.</li><li><xref:System.Windows.Controls.DataGridCell> Formanty teraz są poprawnie ogłaszany jako komórce siatki danych (zlokalizowany) przez czytniki zawartości ekranu.</li><li>Czytniki zawartości ekranu teraz ogłosi nazwy można edytować <xref:System.Windows.Controls.ComboBox>.</li><li><xref:System.Windows.Controls.PasswordBox> formanty nie są anonsowane jako &quot;żaden element w widoku&quot; przez czytniki zawartości ekranu.</p></li></ul><strong>Obsługa LiveRegion</strong><br>Czytniki zawartości ekranu, takich jak program Narrator pomagają sprawdzić zawartość interfejsu użytkownika aplikacji, zwykle poprzez opisanie czegoś dotyczące interfejsu użytkownika, który aktualnie ma fokus, ponieważ prawdopodobnie jest to element najbardziej przydatnych dla użytkownika. Jednak jeśli element interfejsu użytkownika gdzieś zmieni się na ekranie, a nie ma fokus, użytkownik może nie dostęp do informacji i pominąć ważne informacje. LiveRegions są przeznaczone do rozwiązania tego problemu. Korzystając z deweloper może je poinformować czytnika zawartości ekranu lub inne [automatyzacji interfejsu użytkownika](~/docs/framework/ui-automation/ui-automation-overview.md) klienta, który ważnych zmian z elementem interfejsu użytkownika. Czytnik zawartości ekranu możesz zdecydować, jak i kiedy poinformowania użytkownika o tej zmianie. Właściwość LiveSetting umożliwia również wiedzieć, jak ważne jest, aby poinformować użytkownika o zmianach wprowadzonych w interfejsie użytkownika czytnika zawartości ekranu.|
|Sugestia|<strong>Sposób korzystania z opcji do lub z tych zmian</strong><br>Aby dla aplikacji do korzystania z tych zmian, należy uruchomić w środowisku .NET Framework 4.7.1 lub nowszej. Aplikacji mogą korzystać z tych zmian w jednej z następujących sposobów:<ul><li>Dla środowiska .NET Framework 4.7.1. Jest to zalecane podejście. Te zmiany ułatwień dostępu są włączone domyślnie w aplikacji WPF, przeznaczonych dla platformy .NET Framework 4.7.1 lub nowszy.</li><li>Jego oznacza brak zgody na zachowania starszych ułatwień dostępu przez dodanie poniższego [przełącznika AppContext](~/docs/framework/configure-apps/file-schema/runtime/appcontextswitchoverrides-element.md) w <code>&lt;runtime&gt;</code> części pliku konfiguracyjnego aplikacji i ustawieniem dla niego <code>false</code>, jak pokazano w poniższym przykładzie.</li></ul><pre><code class="lang-xml">&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot;?&gt;&#13;&#10;&lt;configuration&gt;&#13;&#10;&lt;startup&gt;&#13;&#10;&lt;supportedRuntime version=&quot;v4.0&quot; sku=&quot;.NETFramework,Version=v4.7&quot;/&gt;&#13;&#10;&lt;/startup&gt;&#13;&#10;&lt;runtime&gt;&#13;&#10;&lt;!-- AppContextSwitchOverrides value attribute is in the form of &#39;key1=true/false;key2=true/false  --&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.UseLegacyAccessibilityFeatures=false&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;&lt;/configuration&gt;&#13;&#10;</code></pre>Aplikacji przeznaczonych dla platformy .NET Framework 4.7.1 lub nowszej i chcesz zachować starsze zachowanie dostępności zgodzić się na korzystanie z funkcji ułatwień dostępu w starszej wersji przez jawne ustawienie tego parametru AppContext <code>true</code>.<br>Aby zapoznać się z omówieniem automatyzacji interfejsu użytkownika, zobacz [Przegląd automatyzacji interfejsu użytkownika](~/docs/framework/ui-automation/ui-automation-overview.md).|
