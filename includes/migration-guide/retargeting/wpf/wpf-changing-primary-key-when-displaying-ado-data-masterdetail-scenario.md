### <a name="wpf-changing-a-primary-key-when-displaying-ado-data-in-a-masterdetail-scenario"></a>Klucz podstawowy przy wyświetlaniu danych ADO w scenariuszu wzorzec/szczegół zmiana WPF

|   |   |
|---|---|
|Szczegóły|Załóżmy, że masz kolekcję elementów typu ADO <code>Order</code>, z relacji o nazwie &quot;SzczegółyZamówienia&quot; dotyczą kolekcję elementów typu <code>Detail</code> za pośrednictwem klucza podstawowego &quot;OrderID&quot;. W aplikacji WPF można powiązać formant listy szczegóły dla podanej kolejności:<pre><code class="lang-xml">&lt;ListBox ItemsSource=&quot;{Binding Path=OrderDetails}&quot; &gt;&#13;&#10;</code></pre>w przypadku DataContext <code>Order</code>. WPF pobiera wartość <code>OrderDetails</code> właściwość - D Kolekcja wszystkich <code>Detail</code> elementów, których <code>OrderID</code> odpowiada <code>OrderID</code> elementu głównego. Zmiana zachowania powstaje podczas zmiany klucza podstawowego <code>OrderID</code> elementu głównego. Automatycznie zmieni ADO <code>OrderID</code> każdego odpowiednie rekordy w kolekcji szczegóły (to znaczy te skopiowane do kolekcji D).  Ale co się dzieje z D?<ul><li>Zachowanie starego: kolekcja D jest wyczyszczone.   Element główny jest <em>nie</em> podnieść powiadomienie o zmianie właściwości <code>OrderDetails</code>.  Element ListBox w dalszym ciągu używać kolekcji D, który jest obecnie pusta.</li><li>Nowe zachowanie: kolekcja D jest bez zmian.   Każdy z elementów jego zgłasza powiadomienia o zmianie <code>OrderID</code> właściwości.  Pole listy będzie później nadal używać kolekcji D i wyświetla szczegóły z nowym <code>OrderID</code>.</li></ul>WPF implementuje nowe zachowanie przez utworzenie kolekcji D w inny sposób: po wywołaniu metody ADO <xref:System.Data.DataRowView.CreateChildView(System.Data.DataRelation,System.Boolean)?displayProperty=nameWithType> z <code>followParent</code> argument wartość <code>true</code>.|
|Sugestia|Aplikacja pobiera nowe zachowanie przy użyciu następującego przełącznika AppContext.<pre><code class="lang-xml">&lt;configuration&gt;&#13;&#10;&lt;runtime&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.System.Windows.Data.DoNotUseFollowParentWhenBindingToADODataRelation=false&quot;/&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;&lt;/configuration&gt;&#13;&#10;&#13;&#10;</code></pre>Domyślnie przełącznika <code>true</code> (stare zachowanie) dla aplikacji przeznaczonych dla platformy .NET 4.7.1 lub w dół i <code>false</code> (nowe zachowanie) dla aplikacji przeznaczonych dla platformy .NET 4.7.2 lub nowszej.|
|Zakres|Pomocnicza|
|Wersja|4.7.2|
