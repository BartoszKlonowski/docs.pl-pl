### <a name="managed-cryptography-classes-do-not-throw-a-cryptographyexception-in-fips-mode"></a>Klasy zarządzane kryptografii nie zgłaszają wyjątków CryptographyException w trybie FIPS

|   |   |
|---|---|
|Szczegóły|W programie .NET Framework 4.7.2 i wcześniejszych wersjach zarządzanego dostawcy usług kryptograficznych klasy takich jak <xref:System.Security.Cryptography.SHA256Managed> throw <xref:System.Security.Cryptography.CryptographicException> po bibliotek kryptograficznych systemu są skonfigurowane w trybie FIPS. Te wyjątki zostaną zgłoszone, ponieważ wersje zarządzanego nie zostały poddane FIPS (Federal Information przetwarzania normy) w oparciu o zasady FIPS 140-2 certyfikacji, jak również do blokowania algorytmów kryptograficznych, które nie były uwzględniane w celu uzyskania zatwierdzenia.  Ponieważ deweloperzy kilka swoich komputerach deweloperskich w trybie FIPS, często zgłoszeniu tych wyjątków tylko na systemy produkcyjne. Aplikacji przeznaczonych dla platformy .NET Framework 4.8 i nowszych wersjach automatycznie Przełącz się do nowszej, swobodna zasady, tak aby <xref:System.Security.Cryptography.CryptographicException> generowany jest już domyślnie w takich przypadkach. Zamiast tego klasy kryptografii zarządzane przekierowania operacje kryptograficzne bibliotekę kryptografii systemu. Ta zmiana zasad skutecznie usunięcie potencjalnie mylące różnica między środowisk deweloperskich i środowisk produkcyjnych i sprawia, że składnikami macierzystymi i składniki zarządzane działają na podstawie tych samych zasad kryptograficznych.|
|Sugestia|Jeśli to zachowanie jest niepożądany, możesz zrezygnować z jej i Przywróć poprzednie zachowanie tak, <xref:System.Security.Cryptography.CryptographicException> jest zgłaszany w trybie FIPS przez dodanie poniższego [AppContextSwitchOverrides](~/docs/framework/configure-apps/file-schema/runtime/appcontextswitchoverrides-element.md) ustawienia konfiguracji do [ <runtime> ](~/docs/framework/configure-apps/file-schema/runtime/runtime-element.md) sekcji w pliku konfiguracji aplikacji:<pre><code class="lang-xml">&lt;runtime&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.System.Security.Cryptography.UseLegacyFipsThrow=true&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;</code></pre>Jeśli aplikacja jest przeznaczony dla .NET Framework 4.7.2 lub wcześniej, można również zdecydować się w do tej zmiany przez dodanie poniższego [AppContextSwitchOverrides](~/docs/framework/configure-apps/file-schema/runtime/appcontextswitchoverrides-element.md) ustawienia konfiguracji do [ <runtime> ](~/docs/framework/configure-apps/file-schema/runtime/runtime-element.md) sekcja pliku konfiguracji aplikacji:<pre><code class="lang-xml">&lt;runtime&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.System.Security.Cryptography.UseLegacyFipsThrow=false&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;</code></pre>|
