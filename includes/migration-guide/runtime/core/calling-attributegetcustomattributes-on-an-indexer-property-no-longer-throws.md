### <a name="calling-attributegetcustomattributes-on-an-indexer-property-no-longer-throws-ambiguousmatchexception-if-the-ambiguity-can-be-resolved-by-indexs-type"></a>Podczas wywoływania Attribute.GetCustomAttributes w właściwości indeksatora już zgłasza ambiguousmatchexception —, jeśli niejednoznaczności może zostać rozpoznana przez typ indeksu

|   |   |
|---|---|
|Szczegóły|Przed programu .NET Framework 4.6, wywołanie <code>GetCustomAttribute(s)</code> w indeksatorze, właściwości, które odbiegają od innej właściwości tylko przez typ indeksu mogłoby spowodować <xref:System.Reflection.AmbiguousMatchException?displayProperty=name>. Począwszy od programu .NET Framework 4.6, atrybuty właściwości zostaną prawidłowo zwrócone.|
|Sugestia|Należy pamiętać, że GetCustomAttribute(s) będą działać częściej teraz. Jeśli aplikacja została wcześniej opierając się na <xref:System.Reflection.AmbiguousMatchException?displayProperty=name>, odbicia powinien być teraz używane w celu jawnie zamiast poszukaj wiele indeksatorów.|
|Zakres|Krawędź|
|Wersja|4.6|
