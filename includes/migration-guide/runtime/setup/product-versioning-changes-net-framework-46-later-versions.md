|---|---|
|Szczegóły|Przechowywanie wersji produktu uległo zmianie w stosunku do poprzednich wersji programu .NET Framework, a w szczególności z programu .NET Framework 4, 4.5, 4.5.1 i 4.5.2. Poniżej przedstawiono szczegółowe zmiany:<ul><li>Wartość <code>Version</code> wpisu w <code>HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full</code> kluczu została <code>4.6.xxxxx</code> zmieniona na dla .NET Framework 4.6 i jego wydania punktowe oraz dla <code>4.7.xxxxx</code> .NET Framework 4.7 i 4.7.1. W .NET Framework 4.5, 4.5.1 i 4.5.2 <code>4.5.xxxxx</code>miał format .</li><li>Przechowywanie wersji plików i produktów dla plików .NET Framework uległo zmianie z wcześniejszego schematu przechowywania wersji 4.0.30319.x na 4.6.X.0 dla programu .NET Framework 4.6 i jego wersji punktowych oraz na 4.7.X.0 dla programu .NET Framework 4.7 i 4.7.1. Te nowe wartości można zobaczyć podczas wyświetlania właściwości pliku po kliknięciu prawym przyciskiem myszy na pliku.</li><li><xref:System.Reflection.AssemblyFileVersionAttribute> Atrybuty <xref:System.Reflection.AssemblyInformationalVersionAttribute> i dla zestawów zarządzanych mają wartości version w postaci 4.6.X.0 dla .NET Framework 4.6 i jego wersji punktowych oraz 4.7.X.0 dla .NET Framework 4.7 i 4.7.1.</li><li>W .NET Framework 4.6, 4.6.1, 4.6.2, 4.7 i 4.7.1 <xref:System.Environment.Version?displayProperty=nameWithType> właściwość zwraca ciąg <code>4.0.30319.42000</code>wersji stałej . W .NET Framework 4, 4.5, 4.5.1 i 4.5.2 zwraca ciągi wersji w formacie <code>4.0.30319.xxxxx</code> (na przykład &quot;4.0.30319.18010&quot;). Należy zauważyć, że nie zaleca się kod aplikacji przy żadnej nowej zależności od Environment.Version właściwości.</li></ul>Aby uzyskać więcej informacji, zobacz [Jak: Określić, które wersje programu .NET Framework są zainstalowane](~/docs/framework/migration-guide/how-to-determine-which-versions-are-installed.md).|
