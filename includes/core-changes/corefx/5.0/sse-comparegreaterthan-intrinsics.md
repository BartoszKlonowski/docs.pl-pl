### <a name="sse-and-sse2-comparegreaterthan-methods-properly-handle-nan-inputs"></a><span data-ttu-id="d9763-101">Metody SSE i SSE2 CompareGreaterThan prawidłowo obsługują dane wejściowe NaN</span><span class="sxs-lookup"><span data-stu-id="d9763-101">SSE and SSE2 CompareGreaterThan methods properly handle NaN inputs</span></span>

<span data-ttu-id="d9763-102">Następujące <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> metody i zostały <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> naprawione w celu prawidłowego obsłużenia `NaN` danych wejściowych i dopasowania sprzętowego do odpowiednich metod w <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> klasie:</span><span class="sxs-lookup"><span data-stu-id="d9763-102">The following <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> and <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> methods have been fixed to properly handle `NaN` inputs and match the hardware behavior of the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> class:</span></span>

* `CompareGreaterThan`
* `CompareGreaterThanOrEqual`
* `CompareNotGreaterThan`
* `CompareNotGreaterThanOrEqual`
* `CompareScalarGreaterThan`
* `CompareScalarGreaterThanOrEqual`
* `CompareScalarNotGreaterThan`
* `CompareScalarNotGreaterThanOrEqual`

#### <a name="change-description"></a><span data-ttu-id="d9763-103">Zmień opis</span><span class="sxs-lookup"><span data-stu-id="d9763-103">Change description</span></span>

<span data-ttu-id="d9763-104">Wcześniej `NaN` dane wejściowe na liście <xref:System.Runtime.Intrinsics.X86.Sse> i <xref:System.Runtime.Intrinsics.X86.Sse2> metody zwróciły niepoprawny wynik.</span><span class="sxs-lookup"><span data-stu-id="d9763-104">Previously, `NaN` inputs to the listed <xref:System.Runtime.Intrinsics.X86.Sse> and <xref:System.Runtime.Intrinsics.X86.Sse2> methods returned an incorrect result.</span></span> <span data-ttu-id="d9763-105">Wynik również różni się od wyniku wygenerowanego przez odpowiadającą metodę w <xref:System.Runtime.Intrinsics.X86.Avx> klasie.</span><span class="sxs-lookup"><span data-stu-id="d9763-105">The result also differed from the result generated by the corresponding method in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="d9763-106">Począwszy od platformy .NET 5,0, te metody poprawnie obsługują `NaN` dane wejściowe i zwracają te same wyniki, co odpowiednie metody w <xref:System.Runtime.Intrinsics.X86.Avx> klasie.</span><span class="sxs-lookup"><span data-stu-id="d9763-106">Starting in .NET 5.0, these methods correctly handle `NaN` inputs and return the same results as the corresponding methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="d9763-107">Streaming SIMD Extensions (SSE) i Streaming SIMD Extensions 2 (SSE2) Industry Architecture (ISAs) nie zapewniają bezpośredniej obsługi sprzętu dla tych metod porównywania, dzięki czemu są one zaimplementowane w oprogramowaniu.</span><span class="sxs-lookup"><span data-stu-id="d9763-107">The Streaming SIMD Extensions (SSE) and Streaming SIMD Extensions 2 (SSE2) industry standard architectures (ISAs) don't provide direct hardware support for these comparison methods, so they're implemented in software.</span></span> <span data-ttu-id="d9763-108">Wcześniej metody były niepoprawnie zaimplementowane i nieprawidłowo obsługiwały `NaN` dane wejściowe.</span><span class="sxs-lookup"><span data-stu-id="d9763-108">Previously, the methods were improperly implemented, and they incorrectly handled `NaN` inputs.</span></span> <span data-ttu-id="d9763-109">W przypadku kodu portowego z macierzystego nieprawidłowe zachowanie może spowodować błędy.</span><span class="sxs-lookup"><span data-stu-id="d9763-109">For code ported from native, the incorrect behavior may introduce bugs.</span></span> <span data-ttu-id="d9763-110">W przypadku ścieżki kodu 256-bitowego metody mogą również generować różne wyniki do równoważnych metod w <xref:System.Runtime.Intrinsics.X86.Avx> klasie.</span><span class="sxs-lookup"><span data-stu-id="d9763-110">For a 256-bit code path, the methods can also produce different results to the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="d9763-111">Przykład sposobu, w jaki metody były wcześniej nieprawidłowe, można zaimplementować `CompareNotGreaterThan(x,y)` jako `CompareLessThanOrEqual(x,y)` dla zwykłych liczb całkowitych.</span><span class="sxs-lookup"><span data-stu-id="d9763-111">As an example of how the methods were previously incorrect, you can implement `CompareNotGreaterThan(x,y)` as `CompareLessThanOrEqual(x,y)` for regular integers.</span></span> <span data-ttu-id="d9763-112">Jednak w przypadku `NaN` danych wejściowych logika oblicza niewłaściwy wynik.</span><span class="sxs-lookup"><span data-stu-id="d9763-112">However, for `NaN` inputs, that logic computes the wrong result.</span></span> <span data-ttu-id="d9763-113">Zamiast tego, używając `CompareNotLessThan(y,x)` porównuje cyfry poprawnie *i* pobiera `NaN` dane wejściowe.</span><span class="sxs-lookup"><span data-stu-id="d9763-113">Instead, using `CompareNotLessThan(y,x)` compares the numbers correctly *and* takes `NaN` inputs into consideration.</span></span>

- <span data-ttu-id="d9763-118">Jeśli poprzednie zachowanie było wymagane, można zachować takie zachowanie, zmieniając odpowiednie wywołanie w następujący sposób:</span><span class="sxs-lookup"><span data-stu-id="d9763-118">If the previous behavior was desired, you can maintain that behavior by changing the relevant invocation as follows:</span></span>

  * `CompareScalarGreaterThanOrEqual(x,y)` -> `CompareScalarNotLessThan(x,y)`
  * `CompareScalarNotGreaterThan(x,y)` -> `CompareScalarLessThanOrEqual(x,y)`
